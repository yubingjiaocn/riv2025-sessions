# AWS re:Invent 2025 网络防火墙会议总结

## 会议概述

本次AWS re:Invent 2025分会场由AWS网络防火墙服务产品经理Amish Sha主讲，重点介绍了AWS Network Firewall的最新功能和增强特性。会议回应了客户在去年re:Invent上提出的核心诉求：如何在不增加复杂性的前提下保持领先于新兴威胁，以及如何在简化安全管理的同时不降低保护水平。演讲者强调了安全简化与风险控制之间的微妙平衡——过度简化可能导致防火墙策略中的规则遗漏，从而使网络变得脆弱。

AWS Network Firewall是一项完全托管的防火墙服务，客户无需担心防火墙基础设施的部署或持续的软件升级。该服务具有高可靠性和可扩展性，每个可用区可提供高达100 Gbps的吞吐量。服务内置有状态检测引擎，支持基于IP地址、应用程序、完全限定域名(FQDN)编写规则，甚至可以编写自定义规则来匹配特定的TCP头部。此次发布的核心亮点包括主动威胁防御(Active Threat Defense)功能和AWS合作伙伴托管规则(Partner Managed Rules)的推出，后者整合了七家顶级安全厂商的威胁情报，使客户能够通过几次点击即可获得企业级的威胁防护能力。

## 详细时间线与关键要点

[00:00 - 01:30] 开场与背景介绍
- 演讲者Amish Sha介绍自己是AWS网络防火墙服务的产品经理
- 回顾去年re:Invent客户提出的主要需求：保持领先于新兴威胁、简化安全而不妥协保护
- 强调安全简化的挑战：防火墙策略中一条遗漏的规则就可能使网络变得脆弱

[01:30 - 03:00] AWS Network Firewall服务概述
- 介绍AWS Network Firewall是完全托管的防火墙服务
- 无需担心基础设施部署和软件升级，动态自动完成
- 高可靠性和可扩展性：每个可用区提供高达100 Gbps吞吐量
- 内置状态检测引擎，支持多种规则类型
- 与AWS Firewall Manager完全集成，可跨账户部署一致的安全策略

[03:00 - 05:00] 常见使用场景
- **入站过滤(Ingress Filtering)**：使用状态检测、协议检测和IDS/IPS功能防止入侵
- **出站过滤(Egress Filtering)**：最常见的使用场景，监控工作负载的出站连接，防止与命令控制中心的连接，保护数据不被泄露
- **VPC到VPC安全(东西向保护)**：防止流量的横向移动，在VPC工作负载之间创建逻辑边界，在银行和金融服务等受监管行业中非常常见

[05:00 - 08:00] 主动威胁防御(Active Threat Defense)
- 这是AWS托管的规则，基于Amazon Madpot基础设施生成威胁情报
- Madpot是AWS的全球数字诱饵或蜜罐网络，伪装成易受攻击的AWS服务
- 通过分析攻击者的战术、技术和程序(TTP)来策划规则
- 规则每10分钟更新一次，确保防护活跃攻击并清理不再活跃的威胁规则
- 与Amazon GuardDuty集中集成，提供新兴和活跃威胁的中央可见性
- 客户无需编写规则或构建自定义Lambda解决方案来保持规则更新

[08:00 - 10:00] AWS合作伙伴托管规则发布
- 重大发布：AWS Network Firewall现在支持AWS合作伙伴托管规则
- 可以轻松利用AWS Marketplace合作伙伴的威胁情报
- 七家首发合作伙伴：Checkpoint、Fortinet、Infoblox、Lumen、Rapid7、Trend Micro和ThreatSTOP
- 只需几次点击即可直接从控制台部署，无需在Marketplace和第三方平台之间来回切换
- 规则由合作伙伴持续更新，提供主动保护

[10:00 - 12:00] Checkpoint托管规则
- Checkpoint是全球网络安全领导者，首创状态防火墙
- 提供AI驱动的云交付安全解决方案
- 规则来自Threat Cloud AI专家精心策划
- 增强对数百个常见漏洞和暴露(CVE)的保护
- 包含针对OWASP Top 10漏洞的策划规则
- 提供全面覆盖，无需手动管理规则集

[12:00 - 14:30] Fortinet托管规则
- Fortinet是全球网络安全领导者和下一代防火墙领域的可信品牌
- 将AI驱动的威胁情报原生集成到AWS Network Firewall
- Fortinet托管IPS规则提供针对恶意软件和命令控制威胁的保护
- 基于AI驱动的威胁情报，目前为全球数十万Fortinet客户提供服务
- 规则来自FortiGuard Labs
- 帮助维护合规性，Fortinet IPS规则提供PCI DSS合规性和其他需要内联检测的受监管工作负载合规性

[14:30 - 16:30] Infoblox托管规则
- Infoblox将网络、安全和云与其保护性DDI平台统一
- 受到13,000多家客户信任，包括大多数财富100强企业

- 90%的检测在DNS查询前完成，误报率仅0.2%
- 可以轻松阻止新注册域名或已知高风险域名，只需在网络防火墙策略中启用规则集

[18:00 - 19:30] Rapid7托管规则
- Rapid7是威胁检测和响应领域的领导者

  - **勒索软件和网络犯罪**：针对以经济利益为动机的威胁

- 所有策划的托管规则列表可以快速选择并直接从控制台应用到网络防火墙策略
- 无需担心持续更新防火墙规则或可能造成安全风险的错误配置

[23:00 - 24:30] 网络防火墙策略最佳实践指南
- 网络防火墙策略通常是允许列表和拒绝列表规则的组合
- **拒绝列表创建**：使用AWS和合作伙伴的托管规则
- 使用GeoIP过滤阻止流向特定国家的流量
- 使用状态检测和威胁签名规则
- **允许列表创建**：从宽松的允许列表开始，允许某些可信的顶级域名
- 逐步构建更窄的可信目的地允许列表
