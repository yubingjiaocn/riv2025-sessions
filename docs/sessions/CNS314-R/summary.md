# AWS re:Invent 2025 技术会议总结

## 会议概述

本次技术会议由Tiffany和Olivier主讲，主题为"如何使用Strands Agent构建多智能体角色扮演游戏主持人"。会议展示了如何利用AWS的Strands Agents SDK构建一个AI驱动的龙与地下城(D&D)游戏主持人助手。演讲者通过实时编程演示，展示了从零开始构建多智能体工作流的完整过程，包括智能体间通信、工具集成以及MCP(模型上下文协议)和A2A(智能体到智能体)协议的应用。

会议的核心理念是解决传统桌游中游戏主持人难找的问题，通过AI技术降低游戏主持的门槛。演讲者展示了如何创建一个包含编排器、角色管理智能体、规则智能体和骰子工具的完整系统，每个组件都有专门的职责和功能。整个演示过程生动地展现了现代AI智能体技术在实际应用中的强大能力。

## 详细时间线与关键要点

### 0:00-5:00 - 会议开场与背景介绍
- 介绍桌面角色扮演游戏概念，特别是龙与地下城
- 解释游戏主持人(Game Master)的重要作用和挑战
- 阐述项目动机：利用AI技术辅助游戏主持人

### 5:00-10:00 - 智能体概念与发展历程
- 回顾生成式AI的发展历程，从ChatGPT发布开始
- 解释智能体与传统LLM的区别：智能体具有执行能力
- 介绍智能体工具生态系统：API、函数、数据库等
- 展示多智能体协作模式：编排器+专业化智能体

### 10:00-15:00 - MCP和A2A协议详解
- 详细介绍MCP(模型上下文协议)：标准化智能体与工具的连接方式
- 解释A2A(智能体到智能体)协议：规范智能体间通信
- 对比两种协议的应用场景和通信方式差异
- 强调协议标准化对AI生态系统的重要意义

### 15:00-20:00 - Strands Agents SDK介绍
- 介绍Strands Agents作为AWS内部使用的Python SDK
- 展示SDK的简洁性：仅需几行代码即可创建智能体
- 演示安装过程：pip install strands-agents
- 说明SDK在Amazon Q等产品中的实际应用

### 20:00-30:00 - 实时编程演示开始
- 创建基础智能体：三行代码实现LLM调用
- 演示多种模型支持：Amazon Bedrock、Anthropic、Gemini等
- 展示本地模型集成：使用Ollama运行本地LLM
- 介绍工具集成：添加current_time工具解决实时信息问题

### 30:00-40:00 - 工具开发与MCP服务器
- 演示自定义工具开发：创建roll_dice函数
- 使用@tool装饰器将函数转换为智能体工具
- 创建MCP服务器：使用FastMCP库外部化工具
- 展示MCP客户端连接：智能体通过MCP协议调用外部工具

### 40:00-50:00 - 多智能体架构实现
- 创建角色管理智能体：专门处理D&D角色创建和管理
- 实现A2A服务器：使智能体可被其他智能体调用
- 演示智能体间通信：游戏主持人调用角色管理智能体
- 展示完整工作流：骰子投掷、角色创建、文件操作的协同工作

### 50:00-52:00 - 完整演示与总结
- 展示最终产品：包含前端界面的完整游戏系统
- 演示多个智能体协作：MCP服务器、角色智能体、规则智能体
- 展示实际游戏场景：角色创建、情节推进、规则检查
- 提供代码资源：AWS Samples中的完整示例代码
