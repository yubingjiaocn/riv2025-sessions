# AWS re:Invent 2025 - Amazon EKS Auto Mode 会议总结

## 会议概述

本次 AWS re:Invent 2025 分会场重点介绍了 Amazon EKS Auto Mode 这一重大功能更新。演讲者 Sai Venom(AWS 首席解决方案架构师)和 Alex Kessner(EKS Auto Mode 首席产品经理)通过现场演示和技术讲解,展示了 EKS Auto Mode 如何彻底简化 Kubernetes 集群的部署和运维工作。

EKS Auto Mode 代表了 AWS 对 Kubernetes 云端运行方式的愿景实现。它提供了完全托管的计算、网络和存储资源,让用户能够在几分钟内从零开始部署生产就绪的集群。这项服务将 AWS 八年来运营数千万个集群的最佳实践内置其中,大幅降低了运维开销。会议还邀请了 Capital One 的 Dan Lavine 分享了他们在高度合规的金融环境中采用 EKS Auto Mode 的实践经验,展示了该技术在企业级场景中的实际应用价值。

核心价值在于将共享责任模型的边界大幅移动——AWS 承担了从集群能力、计算生命周期管理到操作系统补丁、监控和修复等大量底层工作,让客户能够专注于应用程序本身和业务创新。

### 关键技术亮点
- **AWS Backup 集成**:一键备份 EKS 集群,建议在升级前使用
- **版本升级策略**:客户控制控制平面升级时机,AWS 自动处理节点和组件升级
- **成本优化**:自动整合节点,支持 Spot 实例,按需扩缩容
- **安全性增强**:临时计算资源、自动 OS 补丁、AWS 控制的访问管理
