# AWS re:Invent 2025 技术会议总结

## 会议概述

本次会议由三位AWS解决方案架构师Adrian Begg、Dirk和Robert主讲，主题为"三种重要的转型模式：绞杀者模式、Saga模式和断路器模式"。会议通过一个名为"New Science"的科学新闻应用案例，详细介绍了如何从单体应用向微服务架构转型的实践方法。

演讲者强调每个架构决策都涉及权衡取舍，重点是选择最不痛苦的选项，专注于改善客户体验、减少运营负担或提高应用的敏捷性和可演化性。会议内容涵盖了从单体应用解耦、分布式系统集成到API保护的完整转型路径。

## 详细时间线与关键要点

### 0:00-10:00 会议开场与问题背景
- **演讲者介绍**：三位AWS解决方案架构师分别介绍自己的专业领域
- **核心理念**：每个架构决策都涉及权衡，需要在简单性和复杂性之间做出选择
- **案例背景**：介绍"New Science"应用从三个朋友的简单想法开始，最初只有用户注册、文章发布和评论功能
- **早期成功**：24小时内获得用户反馈，快速添加搜索功能

### 10:00-20:00 单体应用的挑战
- **成长痛点**：随着业务增长，成本激增，开发团队陷入"救火"模式
- **技术债务**：原始开发者离职，缺乏文档，代码难以理解和维护
- **运营困境**：任何变更都需要所有产品经理批准，团队恐惧和不快乐
- **重构困境**：完全重建预计需要两年时间，风险过高

### 20:00-30:00 绞杀者模式介绍
- **生物学类比**：通过澳大利亚昆士兰的绞杀榕树解释模式原理
- **模式核心**：在现有系统边缘增量构建新系统，逐步"绞杀"旧系统
- **关键优势**：提高单体应用稳定性的同时逐步替换组件，降低风险
- **实施策略**：引入代理/门面组件来控制流量路由

### 30:00-40:00 绞杀者模式实践
- **搜索服务重构**：首先解决最大问题点，使用Amazon API Gateway作为代理
- **技术选择**：采用AWS Lambda处理可变负载，Amazon OpenSearch Service优化搜索功能
- **文章管理服务**：引入反腐败层处理组件间依赖，使用Amazon S3存储文章数据
- **数据迁移**：使用AWS DMS进行异构数据库迁移

### 40:00-50:00 Saga模式详解
- **业务场景**：Premium Tier订阅流程，涉及多个下游系统协作
- **集成策略**：从同步请求-响应模式转向异步消息传递
- **消息队列优势**：使用Amazon SQS缓冲消息，解决可用性依赖问题
- **发布-订阅模式**：通过Amazon SNS简化系统间通信

### 50:00-60:00 断路器模式与API保护
- **问题识别**：下游API不稳定导致整个Saga流程回滚，级联故障
- **断路器状态**：关闭（正常）、开启（故障）、半开（测试）三种状态
- **AWS实现**：使用AWS Step Functions实现断路器逻辑，CloudWatch告警触发状态变更
- **API限流**：通过EventBridge管道和使用计划保护API，避免超载第三方服务
- **综合保护**：结合AWS WAF、API Gateway限流和使用计划提供多层防护
