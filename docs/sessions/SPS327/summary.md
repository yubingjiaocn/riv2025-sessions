# AWS re:Invent 2025 - 曼彻斯特机场集团应用代理AI优化航班准点率

## 会议概述

本次技术分享会由AWS高级AI策略师Tom Chester、团队成员Margarita以及曼彻斯特机场集团(MAG)的Steve共同主讲，重点介绍了MAG如何应用代理AI技术解决机场运营复杂性问题，确保航班准点起飞。作为英国最大的机场集团，MAG每天处理超过1000个航班，面临着巨大的运营挑战。

演讲者详细阐述了MAG的愿景：创建由AI代理驱动的统一数字化同事工作平台，减少运营复杂性并为组织创造价值。他们选择了"大胆思考、小步开始、快速扩展"的框架，首先从员工意外缺勤报告这一关键业务场景入手，逐步构建更复杂的代理AI系统。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与背景
- 欢迎辞，介绍演讲团队成员
- MAG作为英国最大机场集团的背景介绍
- 机场运营面临的日常挑战：员工临时缺勤对安检流程的影响
- 这是MAG第三次在re:Invent上分享AI解决方案

### 5:00-10:00 MAG运营复杂性分析
- MAG拥有3个机场，约9000名员工，校园内总计40000名工作人员
- 战略目标：为乘客提供无缝、安全的机场体验
- COVID-19期间的挑战：航班量降至1-2%，主要用于货运
- 多方协作复杂性：MAG员工、边境部队、航空公司合作伙伴、地勤处理商

### 10:00-15:00 AI愿景与用例选择
- 数字化同事工作平台的愿景
- 选择意外缺勤报告作为首个用例的原因
- 业务价值：减少加班费用、提高安检效率、降低第三方服务成本
- 安全考虑：用户身份验证、数据准确性、系统可靠性

### 15:00-20:00 为什么选择代理AI
- 生成式AI解决方案的演进：从AI助手到完全自主的代理系统
- 代理AI的核心能力：目标导向、工具访问、记忆功能、推理能力
- 机场环境的非确定性特征需要动态适应能力
- 与传统自动化工具相比的优势：处理异常情况、模块化扩展

### 20:00-25:00 当前工作流程分析
- 员工缺勤报告的现有流程：联系点、信息收集、HR政策检查、系统记录
- 不同机场间流程差异带来的标准化挑战
- 轮班员工的额外复杂性：班次记录系统、多重通知需求

### 25:00-30:00 技术实现深度解析
- 从确定性工作流到代理工作流的转变
- 工具适配的重要性：详细错误捕获、人类可读输出、连续工具合并
- Amazon Bedrock Agent Core Runtime的应用
- 模型上下文协议(MCP)服务器的使用

### 30:00-35:00 安全与合规考虑
- 社会工程攻击的防护
- 上下文感知护栏的实现
- 最小权限原则的应用
- 多轮对话模式的安全监控

### 35:00-40:00 语音交互解决方案
- Amazon Nova Sonic语音模型的集成
- 实时流媒体复杂性处理
- 语音提示优化：避免思维链、枚举模式、拼写考虑
- 代理层次结构：语音模型调用文本模型

### 40:00-45:00 用户体验与未来展望
- UI设计的重要性：保持用户参与度
- 实际演示：从身份验证到缺勤记录的完整流程
- 成果：99%的缺勤报告一致性，90%的记录时间减少
- 未来计划：自动排班、航站楼运营优化、更多代理协作
