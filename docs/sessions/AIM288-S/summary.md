# AWS re:Invent 2025 - 智能AI代理在云安全中的应用

## 会议概述

本次技术分享会由Sysdig公司代表主讲，重点探讨了智能AI代理（Agentic AI）在云安全领域的革命性应用。演讲者深入分析了传统云安全管理面临的挑战，特别是漏洞管理中的海量数据处理难题，并展示了如何通过AI代理技术将安全专业人员从繁重的重复性工作中解放出来。

会议强调了AI代理不是要取代安全专业人员，而是作为"新队友"协助工作，通过自主分析、学习适应和智能协作，帮助安全团队更高效地处理云环境中的安全威胁。演讲者特别展示了Sysdig平台在容器和Kubernetes环境中的漏洞管理自动化解决方案，演示了如何将数千个漏洞快速筛选到可管理的数量级。

## 详细时间线与关键要点

### 0:00-2:00 开场与议题介绍
- 会议主题：智能AI代理与云安全
- 演讲大纲：AI代理特性、作为新队友的价值、云安全工作流程变革
- 承诺展示Sysdig实际工作流程实现

### 2:00-4:00 智能AI代理核心特征
- **自主性**：全天候独立工作，无需人工干预每个步骤
- **学习适应能力**：能够学习、适应并使用可用资源
- **协作性**：不是取代工作而是提升效率，成为团队成员
- **未来愿景**：从分析建议逐步发展到自动执行修复

### 4:00-6:00 云安全现状挑战
- **数据过载问题**：云规模扩大导致安全信号激增
- **传统工作流程痛点**：
  - 需要手动分析仪表板数据
  - 难以优先级排序业务影响
  - 缺乏针对性修复建议
  - 需要在多个工具间切换查找信息

### 6:00-8:00 AI代理驱动的新工作流程
- **传统流程**：告警 → 调查问题 → 验证真实性 → 确定影响 → 制定修复方案
- **AI增强流程**：告警 → 立即获得影响分析和修复建议 → 快速决策 → 可选验证
- **关键优势**：绕过耗时的前期分析步骤，直接进入解决阶段

### 8:00-10:00 Sysdig平台介绍与漏洞管理挑战
- **Sysdig定位**：云安全平台，涵盖漏洞管理、态势管理、实时威胁检测
- **容器环境漏洞管理痛点**：
  - 数千个漏洞发现难以处理
  - 需要回答多个关键问题：生产环境？业务关键？实际使用？外部暴露？
  - 优先级排序、工单分配、修复方案制定耗时巨大

### 10:00-12:00 AI代理解决方案架构
- **环境分析代理**：识别生产集群、业务应用重要性、数据风险评估
- **运行时分析代理**：确定漏洞组件是否实际使用、内存加载状态
- **暴露性分析代理**：检测外部网络暴露情况
- **可利用性分析代理**：查询已知漏洞利用方式和修复可行性

### 12:00-15:00 实际演示环节
- **漏洞筛选演示**：从1200+漏洞筛选到可管理数量
  - 关键级别筛选
  - 内存加载状态过滤
  - 生产环境识别（1000+）
  - 互联网暴露筛选（显著减少）
  - 可利用性验证
- **智能优先级排序**：基于重要性和普遍性的二维图表分析
- **自动修复建议生成**：
  - 提供具体修复命令
  - 区分应用层和基础层修复
  - 避免破坏性版本升级
  - 生成结构化Jira工单

### 15:00-17:00 总结与展望
- **应用领域扩展**：从漏洞管理扩展到态势管理、威胁检测等所有云安全领域
- **核心价值**：
  - 帮助防护云环境
  - 加速威胁响应决策
  - 简化安全工具使用复杂度
- **邀请互动**：鼓励参会者到展台深入交流
- **未来方向**：寻找更多安全工具和方法中的AI代理应用机会
