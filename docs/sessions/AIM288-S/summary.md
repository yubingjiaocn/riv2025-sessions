# AWS re:Invent 2025 - 代理式AI在云安全中的应用

## 会议概述

本次分会场主要探讨了代理式AI（Agentic AI）如何革新云安全工作流程。演讲者来自Sysdig公司，重点介绍了AI代理如何从传统的问答工具演变为能够自主执行任务的"团队成员"。在云安全领域，安全团队面临着海量的告警、漏洞和配置问题，传统方式需要花费大量时间进行分析、优先级排序和修复。代理式AI通过自动化这些繁琐的流程，能够立即提供影响分析、修复建议和上下文信息，将原本需要数小时甚至数天的工作缩短到几分钟。

演讲者特别强调，AI不是要取代安全人员，而是作为协作伙伴帮助他们更高效地工作。通过Sysdig平台的实际案例演示，展示了如何利用AI代理将数千个容器漏洞快速筛选到只需关注的少数关键问题，并自动生成详细的修复方案。这种转变让安全团队能够从繁重的数据分析工作中解放出来，专注于真正重要的安全决策和响应工作。

## 详细时间线

00:00 - 01:30 开场介绍
- 演讲者欢迎观众，介绍会议主题：代理式AI与云安全
- 说明会议将涵盖AI代理的定义、作为新团队成员的角色，以及如何改变云安全工作方式

01:30 - 03:45 什么是代理式AI
- 解释代理式AI的核心特征：从简单问答转向执行具体任务
- 强调AI能够自主行动、全天候工作、学习适应并使用可用工具
- 突出"协作"概念，消除AI会取代工作的担忧

03:45 - 05:30 AI作为团队成员
- 描述AI如何提供分析、呈现发现、给出建议
- 讨论未来AI将能够自动执行修复操作的愿景
- 承认目前在让AI自动进行更改方面仍存在谨慎态度

05:30 - 07:15 云安全面临的挑战
- 询问现场安全专业人员和开发人员的情况
- 描述云环境中数据量庞大的问题：告警、漏洞、配置问题
- 列举传统工作流的痛点：分析、过滤、优先级排序、通知、获取修复建议

07:15 - 09:00 传统vs AI驱动的工作流对比
- 传统流程：告警→调查→确定影响→寻找修复方案（耗时数小时到数天）
- AI驱动流程：告警→立即获得影响分析和修复建议→快速决策→可选的深入验证
- 强调AI将漫长的前期工作压缩，让团队直接进入修复阶段

09:00 - 11:30 漏洞管理用例介绍
- 聚焦容器和Kubernetes环境的漏洞管理
- 询问现场使用容器/Kubernetes的人员（大多数举手）
- 描述常见问题：扫描发现大量漏洞，难以优先级排序，需要在多个系统间切换获取上下文

11:30 - 14:00 Sysdig平台介绍
- 介绍Sysdig作为云安全平台，涵盖多个安全领域
- 说明平台的核心能力：漏洞管理、安全态势、实时威胁检测
- 强调平台拥有深度遥测数据和丰富上下文，可供AI利用

14:00 - 16:30 漏洞管理的具体挑战
- 列举需要回答的关键问题：是否真正需要担心？是否在生产环境？是否是业务关键应用？
- 继续列举：漏洞是否正在使用？是否已加载到内存？是否暴露在外？是否有修复方案？谁负责修复？
- 描述优先级排序、开票、确定修复方法、进度报告等后续工作

16:30 - 18:00 Sysdig的AI解决方案概述
- 说明AI如何通过整合多个信息源来解决这些问题
- 介绍三个关键能力：环境分析、运行时洞察、修复策略生成

18:00 - 20:30 实时演示开始
- 切换到Sysdig平台演示界面
- 展示平台的多个功能模块：漏洞、运行时事件、合规等
- 进入漏洞管理的AI代理功能区域

20:30 - 24:00 演示：漏洞过滤漏斗
- 展示初始状态：数千个漏洞需要处理
- 第一步过滤：仅显示严重（Critical）漏洞
- 第二步：筛选已加载到内存中的漏洞
- 第三步：识别生产环境中的漏洞（从1200降至1000+）
- 第四步：筛选暴露在互联网上的漏洞（大幅减少）
- 第五步：确认可被利用的漏洞（已知exploit）
- 最后：自动识别例外情况（如Windows包但环境不使用Windows）

24:00 - 27:00 演示：优先级排序和可视化
- 展示漏洞重要性与普遍性的二维图表
- 识别一个异常值：虽然不是最严重但部署范围很广
- 演示"解释为什么重要"功能，AI生成详细说明
- 展示可以进一步筛选仅显示严重漏洞的选项

27:00 - 30:30 演示：AI生成修复方案
- 点击"修复"按钮，AI生成详细的修复"配方"
- 说明AI理解用户环境，不会建议破坏性的大版本升级
- 展示修复建议包括应用层和基础镜像层的具体操作
- 强调AI提供的是可执行的命令和步骤，而非仅仅列出CVE编号

30:30 - 32:30 演示：工单集成
- 展示创建Jira工单的功能
- 说明所有AI生成的洞察和修复建议会自动格式化并填入工单
- 强调这大大简化了从发现到修复的整个流程
- 如果工单已存在，系统会智能识别并更新

32:30 - 34:00 演示：进度跟踪
- 展示漏洞趋势图表，理想情况应该向下向右发展
- 说明AI帮助跟踪修复进度，便于向管理层汇报
- 可以用于识别是否需要投入更多资源

34:00 - 36:30 总结AI代理的价值
- 强调AI代理如何节省大量时间和精力
- 减少在不同系统间切换和手动搜索的需求
- 让团队能够专注于真正的修复工作而非繁琐的分析

36:30 - 38:00 扩展应用场景
- 说明这种方法适用于云安全的所有领域
- 包括安全态势管理、威胁检测等
- AI可以承担那些重要但耗时的过滤和分析工作
