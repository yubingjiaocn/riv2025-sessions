# AWS re:Invent 2025 分会场总结

## 会议概述

本次演讲由来自日本东京的初创公司 InfinMind 的 Aza 主讲，主题是如何构建 AI 引擎以大规模生成视频的近实时洞察。该公司正在开发下一代基于 AI 的视频搜索引擎，专门针对大规模视频数据集。演讲者指出，当前全球超过 80% 的互联网数据是视频格式，而企业存储了 3-8 PB 的视频数据却从未被分析，这被称为"暗数据危机"。现有技术无法以经济且准确的方式分析视频内容并使其可搜索。

InfinMind 正在开发基础视频理解模型，具备四大核心能力：因果事件推理、全模态检索、精确时间定位和长上下文理解。该技术已在日本电视广播领域部署，可追踪所有电视广播内容并使其可搜索，包括产品、品牌、位置、姓名、电话号码等。公司计划将该技术扩展到零售智能、制造业质量控制和安防等领域。

技术架构方面，InfinMind 与 AWS 深度合作，使用 AWS Parallel Cluster、Slurm、S3、EC2 和 FSx 等服务训练模型，并利用 Nvidia H100、L40S、B200 等 GPU。公司正在推进 SOC 2 安全认证，并开发可部署在客户安全环境中的应用程序，确保数据隐私。

## 详细时间线

[00:00 - 01:30] 开场介绍
- 演讲者 Aza 介绍自己从东京飞行 16 小时来到 AWS re:Invent
- 演讲主题：构建 AI 引擎以大规模生成视频近实时洞察
- 公司定位：下一代 AI 视频搜索引擎

[01:30 - 03:45] 问题陈述：暗数据危机
- 全球正在被持续录制，但大部分视频被存储后从未分析
- 互联网数据中超过 80% 是视频格式
- 企业拥有 3-8 PB 的未分析视频数据
- 约 90% 的视频内容从未被分析，称为"暗数据危机"
- 核心问题：现有技术无法以经济且准确的方式分析和搜索视频内容

[03:45 - 05:30] 技术愿景：从点到线
- 现有 AI 模型只能做基础理解（物体检测、人脸检测）- "点对点"理解
- InfinMind 目标：构建"线"的理解 - 理解视频随时间的变化
- 理解事件之间的因果关系
- 提供视频中发生事件的完整故事
- 达到人类级别的视频理解能力

[05:30 - 08:00] 四大核心能力介绍
- **因果事件推理**：回答视频中的"为什么"问题
- **全模态检索**：将视频的所有模态（视觉、音频、文本）整合到单一空间中进行搜索
- **精确时间定位**：理解视频的时间特性，用自然语言搜索特定时刻的特定事件
- **长上下文理解**：不仅理解 30 分钟片段，还能理解数小时、数天、数周甚至数年的视频内容

[08:00 - 09:30] 用例 1：因果事件推理
- 场景：安防摄像头录制到有人奔跑
- 问题：为什么这个人在奔跑？
- 解决方案：视频理解模型推理过去事件并提供准确答案

[09:30 - 11:00] 用例 2：精确时间定位
- 场景：4 小时会议录像，一年前签署了合同
- 需求：找到合同签署的确切时刻和当时的对话
- 解决方案：从 4 小时视频中精确提取合同签署的 5 秒片段

[11:00 - 12:30] 用例 3：全模态检索
- 视频包含多种模态：图像帧、文本、音频等
- 场景：查找玻璃破碎声音与人奔跑同时发生的事件
- 解决方案：模型需要同时理解音频和视觉内容，并在单一空间中搜索

[12:30 - 14:00] 用例 4：长上下文理解
- 应用场景：安防取证研究
- 场景：警方需要在全城数百小时的监控录像中追踪嫌疑人
- 解决方案：模型记忆所有时间线上的事件，追踪嫌疑人在 6 小时内跨越 12 个摄像头的行踪

[14:00 - 16:00] 日本市场部署：电视广播
- 公司已在日本部署初始技术和基础设施
- 应用：追踪日本所有电视广播内容并使其可搜索
- 可搜索内容：产品、品牌、位置、姓名、电话号码等
- 功能：视频摘要、情感提取、内容分段、事件追踪等

[16:00 - 17:30] 模型适配工厂
- 发现技术适用于多个行业：安防、制造、零售等
- 开发"微调/适配工厂"
- 基础模型可快速微调以适应不同用例
- 为特定用例提供最佳视频理解模型

[17:30 - 18:30] 其他行业用例
- **零售智能**：理解购物者在零售空间中的行为旅程
- **制造业**：质量控制、安全控制、生产力控制
- **安防**：取证研究

[18:30 - 20:30] AWS 技术栈

- 存储：S3 和 FSx
