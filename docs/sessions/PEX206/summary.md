# AWS re:Invent 2025 Greenfield 合作伙伴项目会议总结

## 会议概述

本次会议重点介绍了AWS的Greenfield合作伙伴项目,旨在帮助合作伙伴获取新客户。会议由Satel Garcia(DTM合作伙伴战略负责人)和Rahan Sha(Greenfield合作伙伴销售负责人)主讲。Greenfield在AWS的定义中指的是尚未充分使用AWS服务的新客户,包括从大型企业到中小企业的各个客户群体。

会议强调了生成式AI和代理AI(Agentic AI)作为客户切入点的巨大机遇。研究显示,53%的Greenfield客户缺乏安全使用AI的技能,25%的企业基础设施尚未为AI做好准备,这为合作伙伴创造了巨大的市场空间。AWS推出了全新的合作伙伴Greenfield项目(PGP),这是一个多年期的战略性项目,通过合同协议、专项激励、培训赋能和联合销售支持,帮助合作伙伴建立可持续的Greenfield业务实践。

会议还分享了实际案例,展示了系统集成商Tropolite如何帮助机电管道领域的企业客户,通过Amazon Bedrock和SageMaker构建智能资产管理系统和AI驱动的事件管理系统,实现业务转型。演讲者强调,成功的Greenfield策略需要以业务成果为导向、提供系统化的客户旅程、与AWS战略对齐,并提供集成化的解决方案体验。

## 详细时间线与关键要点

### 开场介绍 (0:00-2:30)
- **0:00** - 会议开始,主持人介绍客户获取的挑战性和成本问题
- **0:15** - Satel Garcia自我介绍,担任DTM合作伙伴战略负责人,负责Greenfield、SMB和间接模式
- **0:30** - Rahan Sha介绍,担任Greenfield合作伙伴销售负责人

### Greenfield定义与范围 (2:30-4:00)
- **2:30** - 解释Greenfield概念:不仅是新标志客户,而是尚未充分参与AWS服务旅程的客户
- **3:00** - 强调Greenfield涵盖所有客户细分市场:大型企业、中端市场、中小企业、公共部门和ISV
- **3:30** - 介绍会议议程:如何与合作伙伴共同吸引新客户

### AI机遇与市场研究 (4:00-6:30)
- **4:00** - 提及Matt Garman主题演讲中关于AI的内容,强调与本次会议的关联性
- **4:30** - 引用Synov研究公司的数据
- **5:00** - 关键数据:53%的Greenfield客户缺乏安全使用代理AI或生成式AI的技能
- **5:30** - 25%的企业基础设施尚未为AI做好准备
- **6:00** - 中小企业正在大量投资AI测试和试点项目
- **6:15** - Gartner预测:未来几年15%的自主决策将由AI代理完成

### Greenfield成功的四大关键要素 (6:30-9:30)
- **6:30** - 介绍AWS在Greenfield领域的经验和学习成果
- **7:00** - 要素一:以业务成果为导向 - 不要首先谈论技术(如QuickSight或SageMaker),而要关注解决的业务问题和业务转型
- **7:45** - 要素二:系统化的客户旅程 - 明确范围,提供可扩展、可预测、可重复的解决方案,建立客户信任
- **8:15** - 强调首个工作负载的关键性,通常迁移不是第一个工作负载,代理AI是绝佳的切入点
- **8:45** - 要素三:战略性AWS对齐 - 客户希望合作伙伴了解AWS项目和优惠,能够帮助降低首个工作负载的成本
- **9:15** - 要素四:集成化体验和解决方案 - 提供无缝体验,可能需要引入ISV或其他合作伙伴

### 客户细分策略 (9:30-11:30)
- **9:30** - 强调不同客户细分需要不同的策略
- **10:00** - 大型企业:处理复杂的遗留工作负载,需要变更管理实践和现代化方案
- **10:30** - 中型企业:希望采用AI但缺乏资源,需要合作伙伴专业知识识别用例
- **11:00** - 中小企业:缺乏所有资源,需要可扩展的解决方案和数字化营销方法
- **11:15** - 强调结合联合销售方法和现场合作模式的重要性

### 客户案例:Tropolite (11:30-13:00)
- **11:30** - 介绍合作伙伴Tropolite(系统集成商)的案例
- **11:45** - 客户背景:机电管道(MEP)领域的大型企业,面临遗留系统现代化挑战
- **12:15** - 解决方案:利用Amazon Bedrock和SageMaker构建智能资产管理系统
- **12:30** - 加速生命周期管理,开发AI驱动的事件管理系统
- **12:45** - 成果:全面转型业务流程和工作流,为传统企业创造业务成果

### 三大注意事项 (13:00-15:30)
- **13:00** - 注意事项一:不要忽视自助式小客户 - 投资营销和SEO,通过数字化和自动化流程引导客户
- **13:45** - 这对SMB细分市场尤为重要
- **14:15** - 注意事项二:理解现有关系 - 客户可能与传统合作伙伴有深厚关系,销售周期可能长达12-18个月
- **14:45** - 强调长期陪伴客户的重要性,真正的ROI在客户采用AI并迁移数据到云端后实现
- **15:15** - 注意事项三:AI优先战略 - 建立AI优先的市场策略,重新调整组织、消息传递和解决方案

### 如何开始Greenfield合作 (15:30-17:00)
- **15:30** - 介绍启动Greenfield合作的基本要素
- **15:45** - 制定明确的市场策略
- **16:00** - 充分利用合作伙伴Greenfield项目(PGP)
- **16:15** - 利用Greenfield专项资金
- **16:30** - 配置专门的人员团队
- **16:45** - 建立围绕Greenfield的盈利业务路径,认识到这是长期业务

### 整合AI到合作关系 (17:00-18:30)
- **17:00** - 强调为不同客户细分(企业、中型企业、SMB)构建可扩展的市场策略
- **17:30** - 帮助客户识别用例,展示业务价值和ROI
- **17:45** - 白手套式服务:从概念验证(POC)到工作负载落地全程陪伴客户
- **18:00** - 利用AWS的各种机制:Gen AI快速资金、合作伙伴Greenfield项目等激励措施
- **18:15** - 通过POC阶段引导客户并持续支持

### 合作伙伴Greenfield项目(PGP)详解 (18:30-21:00)
- **18:30** - 介绍全新的合作伙伴Greenfield项目方法
- **18:45** - 核心转变:从单个机会层面转向可持续的多年业务实践
- **19:00** - 这是一个基于合同协议的高标准项目,合作伙伴需要承诺具体目标
- **19:30** - 项目特点:多年期(至少三年)、共同投资、端到端所有权
- **19:45** - 提供能力建设、指导、培训赋能和额外福利
- **20:00** - 帮助合作伙伴建立专门的P&L(损益表)和Greenfield专业能力
- **20:30** - 建议从特定专业领域开始(如代理AI解决方案或安全),而非覆盖所有领域
- **20:45** - 强调这是高标准项目,合作伙伴需要评估自身准备度

### PGP项目旅程 (21:00-22:30)
- **21:00** - 构建阶段:识别适合Greenfield客户的解决方案
- **21:15** - AWS分享数据和最佳实践,帮助合作伙伴了解有效策略
- **21:30** - 探索与其他合作伙伴或SaaS解决方案的合作机会
- **21:45** - 市场推广阶段:与AWS框架保持一致的价值主张
- **22:00** - 在机会层面利用其他项目(如MAP迁移加速计划、POC项目)
- **22:15** - 联合销售团队专注于Greenfield客户
- **22:30** - 持续增长客户,推动第二、第三个工作负载

- **22:30** - 首要行动:在离开会场前注册PGP项目
- **22:45** - 与AWS销售团队、合作伙伴销售经理、客户经理积极互动
- **23:00** - AWS团队拥有按行业或用例分类的无合作伙伴附加的潜在客户管道
