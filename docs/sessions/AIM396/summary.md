# AWS re:Invent 2025 AIM396 会议总结

## 会议概述

本次会议主题为"如何将开源框架与 Amazon Bedrock Agent Core 集成"，由 AWS 的首席数据科学家 Shree Subramaniam 和高级生成式 AI 专家解决方案架构师 Aris 主讲，并邀请了 Cohere Health 的工程副总裁 Keith 分享实践经验。

会议深入探讨了从概念验证（POC）到生产环境的完整路径。演讲者强调，虽然开源框架（如 LangGraph、CrewAI 和 Strands）使构建基础 Agent 变得简单，但将其投入生产环境需要完全不同的方法。AWS 通过 Agent Core 平台提供了端到端的企业级解决方案，该平台包括 Runtime（运行时）、Memory（内存）、Identity（身份管理）、Policy（策略）和 Eval（评估）等模块化服务。

会议的核心内容围绕"Agent Ops"概念展开——这是传统 DevOps 在 AI Agent 时代的演进。演讲者详细介绍了如何通过开放标准（如 MCP 用于工具集成、A2A 用于 Agent 间通信、OpenTelemetry 用于可观测性）实现无缝集成。特别强调了使用 Langfuse 等开源工具进行可观测性管理，以及建立包括实验、测试和生产监控在内的完整 CI/CD 流程的重要性。

## 详细时间线与关键要点

### **开场与背景介绍 (00:00 - 05:30)**
- **00:00** - 会议开始，介绍主题：将开源框架与 Amazon Bedrock Agent Core 集成
- **00:30** - 演讲者自我介绍：Shree Subramaniam（AWS 首席数据科学家）、Aris（AWS 高级生成式 AI 专家 SA）、Keith（Cohere Health 工程副总裁）
- **01:00** - 现场调查：询问有多少人使用过开源框架构建 Agent
- **01:30** - 展示常见的开源框架和协议（LangGraph、CrewAI、Strands、MCP、A2A）
- **02:30** - 强调构建基础 Agent 与投入生产环境的巨大差异
- **03:30** - AWS 对开源的承诺：长期贡献 Linux Foundation、GitHub 等开源项目

### **Agent Core 平台介绍 (05:30 - 10:00)**
- **05:30** - Agent Core 概述：端到端的企业级 Agent 平台
- **06:00** - Agent Core Runtime：可部署任何开源 Agent 和 MCP/A2A 服务器，无需或仅需最小代码更改
- **06:45** - Runtime 特性：会话隔离环境、支持对话式和长时间运行的 Agent
- **07:30** - 集成服务介绍：Identity（身份凭证管理）、Memory（短期和长期内存）、Policy（策略约束）、Eval（评估）
- **08:30** - Observability（可观测性）：基于 OpenTelemetry 标准，连接所有服务
- **09:30** - 模块化设计：可根据需求选择使用的服务组件

### **实际应用场景演示 (10:00 - 18:00)**
- **10:00** - 场景设定：客户服务订单跟踪系统
- **10:45** - 传统方式：人工客服需要查询多个系统，可能需要数天时间
- **11:30** - Agent 解决方案：使用客户服务 Agent 和技术支持 Agent 替代人工
- **12:00** - 框架选择指南：
  - LangGraph：图形化执行，适合复杂编排，学习曲线陡峭
  - CrewAI：易于上手，适合基于角色的团队 Agent
  - Strands：AWS 自研框架，模型驱动，适合企业级应用
- **13:30** - 示例架构：客户服务 Agent 使用 Strands，技术支持 Agent 使用 LangGraph
- **14:30** - 本地开发挑战：本地凭证、硬编码集成、无法扩展

### **Agent Core Runtime 部署 (18:00 - 24:00)**
- **18:00** - Runtime 解决方案：自动扩展、专用端点、快速从零到生产
- **18:45** - 支持任何模型提供商（不限于 Bedrock）
- **19:30** - 安全特性：VPC 模式、OAuth、基于资源的访问策略
- **20:00** - 新功能发布：
  - 完整支持 MCP 规范
  - 双向流式传输（适用于语音接口）
- **21:00** - 两种部署方法：
  1. Docker 容器推送到 ECR
  2. ZIP 文件上传到 S3
- **22:00** - 代码修改示例：仅需几行代码（导入 SDK、初始化应用、添加入口点装饰器）
- **23:00** - Agent Core CLI 工具：
  - agent-core configure：配置部署参数
  - agent-core deploy：一键部署
  - agent-core invoke：调用 Agent

### **上下文工程与集成挑战 (24:00 - 30:00)**
- **24:00** - Agent 的上下文工程：用户指令、系统指令、RAG 检索信息、短期内存、工具响应
- **25:00** - 集成复杂性：开发者需要编写大量集成代码连接不同节点
- **26:00** - 问题：集成代码占据代码库大部分，而非业务逻辑
- **27:00** - 解决方案：使用开放协议简化集成

### **MCP 协议集成 (30:00 - 33:00)**
- **30:00** - MCP（Model Context Protocol）介绍：通过统一接口安全访问外部数据和工具
- **30:45** - 两种 MCP 连接方式：
  1. 在 Runtime 上托管自有 MCP 服务器（零代码更改）
  2. 使用 Agent Core Gateway 为现有 API 创建 MCP 接口
- **32:00** - Gateway 支持：外部 API、Smithy 模型、API Gateway

### **A2A 协议集成 (33:00 - 35:30)**
- **33:00** - A2A（Agent-to-Agent）协议：标准化 Agent 间通信接口
- **33:30** - 由 Google 开发，后捐赠给 Linux Foundation
- **34:00** - 支持不同框架、语言、模型的 Agent 原生通信
- **34:45** - Runtime 原生支持 A2A，无需代码更改

### **完整架构与深度集成 (35:30 - 40:00)**
- **35:30** - 更新后的架构：
  - 蓝线（A2A）：连接客户服务 Agent 和技术支持 Agent
  - 红线（MCP）：连接 Agent 与订单管理系统和外部物流 API
- **36:30** - 框架深度集成：
  - Strands：内置 Session Manager，支持 Agent Core Memory（短期和长期）
  - LangGraph：Agent Core Memory Saver，支持检查点保存和加载
- **38:00** - 开源合作伙伴关系：
  - LangGraph：通过 langchain-aws 库集成
  - Strands：AWS 自研，4000+ GitHub stars，100 万次下载
  - CrewAI：直接贡献代码到其仓库
  - LlamaIndex：RAG 实现和 Agent Core 集成

### **Agent Ops 概念引入 (40:00 - 48:00)**
- **40:00** - Aris 接棒，介绍从 DevOps 到 Agent Ops 的演进
- **41:00** - DevOps 的多维度：基础设施、工具、自动化、流程、文化
- **42:00** - DevOps 早期挑战：缺乏托管服务、工具不完善、瀑布式文化

- **44:30** - 开源生态系统：GitHub、Docker 等公司的崛起
- **45:30** - 新的敏捷文化：CI/CD、自动化
- **46:30** - Agent Ops 目标：将 DevOps 成功经验应用到 Agent 时代

### **基础设施层 (48:00 - 51:00)**

- **49:00** - Serverless 特性：快速启动、自动管理
