# AWS re:Invent 2025 欧洲主权云技术会议总结

## 会议概述

本次技术会议由来自德国的数字主权专家 Armen Schneider 主讲,重点介绍了即将推出的 AWS 欧洲主权云(European Sovereign Cloud, ESC)。会议深入探讨了数字主权的概念框架,包括数据主权(数据驻留和操作员访问控制)和运营主权(弹性和独立性)两大核心维度。演讲者强调,ESC 是 AWS 在三年前发布数字主权承诺后的重要里程碑,旨在满足欧盟高度监管行业和公共部门客户的特殊合规需求。

AWS 欧洲主权云将于 2025 年底在德国勃兰登堡(柏林附近)正式启动,这是一个物理隔离的全新分区(partition),采用完全独立的运营模式。该云平台不仅确保客户内容和元数据完全保留在欧盟境内,还建立了独立的信任根、PKI 基础设施、IAM 系统和 DNS 系统。ESC 的设计目标是实现"无限期独立运营",即在任何情况下都能持续运行,无需依赖欧盟以外的任何资源。这一新产品是对现有 AWS 商业云、Outposts 和专用本地区域的补充,为客户提供更多主权选择。

会议还详细介绍了 ESC 的治理结构,包括四家新成立的欧洲公司,分别负责基础设施、日常运营、信任根管理和整体控股。所有运营人员必须是欧盟居民并受雇于这些欧洲实体,未来还将要求具备欧盟公民身份。ESC 将在启动时提供超过 80 项服务,这是 AWS 新区域启动时服务数量最多的一次,充分体现了打造全功能云平台的承诺。

## 详细时间线与关键要点

00:00:00 - 会议开场与背景介绍
- 演讲者 Armen Schneider 自我介绍,来自德国的数字主权专家
- 会议目标:在 20 分钟内快速介绍欧洲主权云

00:01:00 - 数字主权概念框架
- 数字主权包含多个维度:数据驻留、操作员访问、弹性、透明度、独立性和生存能力
- 将这些需求归纳为两大类:数据主权(数据位置和访问控制)和运营主权(弹性和独立性)

00:02:30 - AWS 数字主权承诺
- 三年前(2022 年)AWS 发布数字主权承诺
- 核心承诺:客户控制数据位置、控制数据访问权限、全面加密(传输中、使用中、静态)、提供最高云弹性

00:03:30 - 数字主权时间线回顾
- 2022 年发布数字主权承诺后,AWS 推出多项技术和合同增强
- 重要里程碑:Nitro 系统的第三方认证,证明零操作员访问能力,并将其纳入服务条款成为合同承诺

00:05:00 - AWS 云主权产品组合
- AWS 商业云本身就是"设计即主权"(sovereign by design)
- 针对不同需求提供多层次解决方案:
  - Outposts:将数据保留在国家边界内或客户自有数据中心
  - 专用本地区域:为需要更多服务和更大规模的客户提供专用资源
  - 欧洲主权云:针对欧盟特定监管要求的全新产品

00:07:00 - 欧洲主权云核心特性
- 计划于 2025 年底启动,首个区域位于德国勃兰登堡
- 物理隔离的全新分区,具有大量额外组件
- 客户内容和客户创建的元数据均保留在欧盟境内
- 由欧盟本地实体的人员进行日常运营

00:08:30 - 主权参考框架
- AWS 收集了欧盟各地客户和监管机构的需求,定义了"主权参考框架"
- 该框架将公开发布,并纳入审计流程
- 建立独立的治理结构

00:09:30 - 无限期运营设计
- ESC 设计目标:在任何情况下都能无限期独立运营
- 建立全新的信任根:独立的证书颁发机构、顶级域名、IAM 系统
- 日常运营不依赖世界其他地区的任何资源
- 在欧盟境内保留源代码副本和专用安全运营中心

00:11:00 - 治理结构详解
- 成立四家新公司运营 ESC:
  - 基础设施子公司:拥有数据中心、网络、冷却、电力等物理资产
  - 运营团队:负责日常运营,包括 24x7 技术支持和技术客户经理
  - 信任根管理公司:拥有和管理 PKI 基础设施
  - 控股公司:设有管理董事和独立咨询委员会

00:13:00 - 运营人员要求
- 所有运营人员必须受雇于欧盟实体并居住在欧盟
- 运营时必须位于欧盟境内
- 未来将要求具备欧盟公民身份
- 客户可来自全球任何地方,ESC 是互联网连接的云

00:14:30 - 运营控制权
- 欧盟本地团队拥有完全运营控制权
- 使用与全球相同的代码库以保持云的敏捷性
- 本地团队可以停止、回滚部署,拥有完全运营控制权

00:15:30 - 数据驻留对比
- 商业云:客户内容保留在所选区域,但客户创建的元数据(如 S3 存储桶名称)可能离开区域
- ESC:所有内容和元数据均保留在欧盟境内
- IAM 系统、DNS 系统、角色和权限均独立且保留在 ESC 区域内

00:16:30 - 独立计费系统
- ESC 拥有独立的计费系统
- 客户可选择以欧元计费
- 计费数据不会离开区域

00:17:00 - 服务可用性
- 启动时将提供超过 80 项服务,是 AWS 新区域启动时服务数量最多的一次
- 目标是打造全功能云,而非功能受限的云
- 后续将持续增加更多服务并发布路线图

00:18:00 - 合作伙伴生态系统
- 已有大量启动合作伙伴
- 将提供独立的 Marketplace
- 合作伙伴包括运营合作伙伴和 SaaS 提供商
- 引入"主权能力认证",帮助合作伙伴满足主权要求

00:19:00 - 适用场景与扩展性
- ESC 不是"一刀切"的解决方案,是现有产品的补充
- 客户可将 Outposts 或专用本地区域连接到 ESC,作为其扩展
- 建议客户评估是否真正需要 ESC 的额外主权组件

00:20:00 - 启动计划与账户管理
- 2025 年底启动,面向所有客户开放
- 任何人都可以注册 ESC 账户
- 由于采用新的 IAM 系统,需要创建新账户和新组织结构
- 如果同时在商业云和 ESC 运营,需要复制治理结构
- 不允许跨分区进行角色假设(assume role)
- 网络物理隔离,Direct Connect 需要使用 ESC 专用接入点

00:21:00 - 会议结束
