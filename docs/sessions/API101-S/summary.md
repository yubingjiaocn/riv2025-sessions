# AWS re:Invent 2025 会议总结：LMI 如何将商业创新与联邦安全要求相结合

## 会议概述

本次会议由 LMI 公司的首席技术官 Jared Summers 主讲，重点介绍了如何将商业现货软件(COTS)的创新速度与联邦政府严格的安全要求相结合。LMI 是一家为联邦政府提供下一代解决方案和产品的供应商，已有 65 年历史。在过去三年中，公司成功转型，从传统服务模式转向以知识产权驱动的解决方案和产品模式，投资数亿美元收购和整合公司，实现了 68% 的复合年增长率。

演讲核心围绕联邦政府对 COTS 解决方案日益增长的需求展开。联邦政府希望利用商业领域的技术投资和创新，但同时必须满足 CMMC、FedRAMP、ATO、NIST 等严格的安全标准。LMI 通过建立可重用的安全平台、模块化架构和共同投资模式，成功解决了这一挑战，能够在满足所有安全要求的前提下，在 72 小时内交付 MVP 应用程序，并在数周内完成客户反馈的迭代。

演讲者通过三个具体案例展示了 LMI 的方法论：Spectre（物联网资产追踪解决方案）、Iron Sled（符合 FedRAMP High 和 IL5 标准的安全平台）以及 Raptor（模块化建模仿真工具包）。这些案例证明了在联邦环境中实现商业速度和创新的可行性，关键在于提前投资安全基础设施、采用模块化设计，以及展现真正的合作伙伴精神。

0:00 - 0:45 - 开场介绍
- Jared Summers 自我介绍，LMI 公司 CTO
- LMI 是为联邦政府提供下一代解决方案和产品的供应商

0:45 - 2:00 - 联邦市场的 COTS 需求背景
- 过去 12-18 个月联邦政府对商业现货解决方案需求强劲
- 政府希望利用商业市场的技术投资和创新
- 挑战：如何在满足联邦安全要求的同时保持创新速度

2:00 - 2:30 - 联邦安全要求的复杂性
- 联邦市场充满各种安全标准缩写：CMMC、FedRAMP、ATO、NIST 等
- 核心问题：如何在保证安全的同时实现快速创新

2:30 - 3:15 - LMI 公司背景
- 公司已有 65 年历史
- 过去 3 年进行重大转型
- 从服务模式转向解决方案和产品模式
- 投资数亿美元收购公司并以安全方式整合
- 目标：第一天就提供任务就绪的技术

3:15 - 5:00 - 案例一：Spectre 物联网解决方案
- 商业领域已经解决了物联网、库存追踪、资产定位等问题数十年
- 联邦客户（特别是国防部、国土安全部）需要数据安全
- 当天上午宣布推出 Spectre 商业解决方案
- 收购商业技术并在可重用平台中进行安全加固
- 提供从硬件、数据传输到分析的端到端安全解决方案，第一天即可部署

5:00 - 7:30 - 案例二：Iron Sled 安全平台与快速开发
- 在受限环境中部署的挑战
- LMI 通过研发资金建立了符合 FedRAMP High 和 IL5 标准的自有平台 Iron Sled
- 所有内部解决方案从第一天起就按照客户所需的安全基线构建
- 容器和应用程序能够通过客户的安全检查
- 实际案例：从想法到 MVP 仅需 72 小时，一周后完成客户反馈迭代
- 该应用程序本周已在客户环境中用于实战演习

7:30 - 9:30 - 案例三：Raptor 模块化平台
- 强调可重用性和可重复性的重要性
- Raptor 平台已为太空军使用超过十年
- 采用模块化工具包架构
- 可以解决多种不同的客户挑战：太空通信网络、印太地区资产可用性优化、医疗物流等
- 模块化组件可以根据具体问题灵活组合

9:30 - 11:00 - 成功的关键要素
- 政府寻找愿意共同投资的合作伙伴
- 不同于传统模式（客户付费开发，等待 18-24 个月或 3-5 年）
- 展示正确的合作伙伴特质：共同投资、第一天就提供可用能力
- 解决方案必须是模块化、可重用的，同时满足安全标准

11:00 - 11:45 - LMI 的增长成果
- 采用这种方法后，LMI 实现了 68% 的复合年增长率
- 对于一家 65 年历史的传统公司来说，这是惊人的转型成就

11:45 - 13:30 - 定制化的必要性
- 成功不意味着跳过定制化
- 后台流程（如人力资源）可能有直接的双重用途
- 但任务关键型应用（CBP、陆军、太空军、邮政服务等）需要定制化
- COTS 不意味着没有定制，而是第一天就带来安全解决方案并提前投资

13:30 - 17:00 - Spectre 详细案例：海军造船厂项目
- 海军希望提高所有海军造船厂的吞吐量
- 造船厂面积巨大且复杂
- 拆解海军舰艇时，大量零部件需要维修或更换
- 某些组件（如核推进相关）极其重要，丢失会导致整个船厂停工
- Spectre 追踪整个生命周期的资产可见性：从拆卸、维修、更换到重新安装
- 数据集成到调度和规划系统中
- 避免零件订购重复、确保正确的安装顺序
- 在正确的时间调度正确的人员
- 目标：更快、更高效、更低成本地完成舰艇维护

17:00 - 18:30 - Spectre 详细案例：陆军仓库项目
- 陆军仓库资源受限但资产众多
- 与陆军合作解决资产定位问题
- 追踪资产位置、状态和组件配置
- 支持父子关系和不同精度的标签
- 实际应用：快速找到特定配置的传动装置，而不是在 230 英亩的场地上寻找
- 提高运营效率
