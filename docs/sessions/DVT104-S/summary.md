# AWS re:Invent 2025 Harness平台与Marriott DevOps转型会议总结

## 会议概述

本次会议是AWS re:Invent 2025的一场技术分享会，主要介绍了Harness平台如何帮助企业实现DevOps和软件交付的现代化转型。会议由Harness公司代表主讲，并邀请了Marriott国际酒店集团的技术高管进行炉边谈话，分享他们使用Harness平台进行DevOps转型的实际经验。

会议重点讨论了在AI时代，虽然代码编写效率大幅提升（AI帮助下代码量增加25%），但软件交付稳定性却下降了7.2%，交付吞吐量下降1.5%的现象。Harness平台致力于通过AI和自动化解决软件交付过程中60-70%的外循环工作，包括测试、安全、部署和优化等复杂工作流程。

Marriott作为拥有近10,000家酒店、遍布143个国家的全球酒店集团，分享了他们如何通过Harness平台实现从6个月的环境部署时间缩短到35分钟，CI时间从12分钟减少到5分钟的显著成果。

## 详细时间线与关键要点

### 0:00-10:00 会议开场与Harness平台介绍
- Harness公司使命：帮助全球开发者快速、可靠、高效、安全地交付代码
- AI对软件工程的影响：代码量增加25%，但交付稳定性和吞吐量反而下降
- 软件工程时间分配：内循环（编码）占30-40%，外循环（测试、安全、部署、优化）占60-70%
- 外循环包含30多个复杂工作流程，传统方式存在大量手动步骤和低效率

### 10:00-20:00 Harness平台架构与AI基础
- Harness平台模块化设计：CICD、基础设施即代码、数据库DevOps、工件注册表等
- AI基础架构三大组成部分：
  - AI代理库：执行各种DevOps任务
  - 知识图谱：捕获企业上下文信息
  - 编排层：控制AI代理的自动化程度
- 平台规模数据：1000+企业客户，过去12个月处理1.28亿次部署、8100万次构建

### 20:00-35:00 Harness平台现场演示
- 演示智能测试优化：根据代码变更关联性只运行相关测试，节省约8分钟
- 安全漏洞自动化处理：聚合、规范化、去重漏洞，AI自动生成修复PR
- 基础设施成本预估：部署前显示成本影响，避免意外的19倍成本增长
- 金丝雀部署：自动监控APM指标和日志，检测异常时自动回滚
- 策略治理：通过OPA策略确保安全扫描等必要步骤无法被跳过

### 35:00-45:00 Marriott嘉宾介绍与转型背景
- Sean Corkum：DevOps和自动化高级总监
- Adnan Haq：基础设施平台负责人
- Marriott规模：近10,000家酒店，143个国家，每16小时开设一家新酒店
- 转型定义：不仅是技术现代化，更是流程、文化的全面变革
- COVID-19成为转型催化剂：I型复苏带来的快速增长需求

### 45:00-52:00 Marriott转型策略与成果
- 转型原则：
  - 从基础设施团队转向平台团队
  - 标准化部署流水线
  - 敏捷思维模式，保持变更灵活性
  - 自动化审批流程，默认批准除非有问题
- 关键成果：
  - 环境部署：从6个月缩短到35分钟
  - CI时间：从12分钟减少到5分钟（功能更多的情况下）
  - 开发者自助服务：无需依赖其他团队
- 选择Harness的原因：内置OPA治理、高级部署能力、强大的API系统

### 52:00-57:30 AI应用前景与建议总结
- AI在DevOps中的应用前景：
  - 自动修复单元测试失败
  - 处理新出现的CVE漏洞
  - 减少开发者认知负荷
  - 让开发者专注于核心业务价值创造
- 给企业的建议：
  - 标准化模板设计
  - 不要迁移旧的技术债务
  - 寻找前瞻性团队作为试点
  - 快速获得早期成功
  - 保持敏捷思维，准备调整方向
  - 选择愿意成为合作伙伴的供应商
