# AWS re:Invent 2025 - 使用 Amazon Bedrock AgentCore 构建智能代理系统

## 会议概述

本次技术会议由 Deepali Talande、Franz Stefan 和 Jan Thewes 共同主讲，重点演示如何使用 Amazon Bedrock AgentCore 为现有遗留应用程序创建智能 AI 代理，而无需重写任何现有代码。会议采用实践编程的方式，展示了从零开始构建代理到集成遗留系统的完整流程。

演讲者首先介绍了智能代理循环的核心概念，包括提示处理、模型调用和工具使用等关键组件。随后通过实际编码演示，展示了如何使用 Strands SDK 创建 AI 代理，并将其部署到 AgentCore 运行时环境。最后通过一个独角兽商店的遗留应用案例，演示了如何在不修改原有系统的情况下，为其添加智能代理功能。

## 详细时间线与关键要点

### 0:00-10:00 - 核心概念介绍
- **智能代理循环机制**：用户提示 → 代理控制器 → 模型处理 → 工具调用 → 响应生成
- **Strands SDK 特性**：开源 Python SDK，支持模型无关的代理构建方法
- **AgentCore 功能**：AWS 托管的运行时基础设施，提供扩展性、安全性和会话管理
- **MCP 协议**：开放标准，定义代理与应用程序和服务之间的通信方式

### 10:00-25:00 - 基础代理创建演示
- **环境准备**：安装必需的 Python 包（strands、strands-tools、bedrock-agentcore）
- **简单代理实现**：仅需 4 行代码创建基本代理
- **工具集成**：添加 HTTP 请求工具以获取实时信息（如天气查询）
- **系统提示配置**：设置代理行为边界和指导原则

### 25:00-35:00 - AgentCore 部署流程
- **本地到云端转换**：使用 Bedrock AgentCore 包装器创建 HTTP 服务器
- **部署工具使用**：通过 AgentCore CLI 工具包简化部署配置
- **配置选项**：执行角色、S3 存储桶、授权方式和内存设置
- **云端验证**：在 AWS 控制台中查看已部署的代理

### 35:00-45:00 - 遗留系统集成准备
- **遗留应用介绍**：独角兽商店 Java 应用，运行在 ECS 上
- **API 规范生成**：使用 Amazon Q Developer 自动生成 OpenAPI 规范
- **网关配置**：在 AgentCore 网关中添加 REST API 目标
- **架构说明**：代理通过网关与遗留应用通信，无需修改原有代码

### 45:00-55:00 - 高级功能实现
- **会话管理**：实现多用户会话隔离和上下文管理
- **内存集成**：添加 AgentCore 内存功能以保持对话历史
- **实际购物演示**：通过代理完成账户创建和商品购买流程
- **可观测性展示**：使用分布式追踪查看代理执行过程和性能指标

### 55:00-56:30 - 总结与资源
- **GitHub 资源**：提供完整的教程和用例模板
- **CloudFormation 模板**：可直接部署的示例配置
- **会议反馈**：鼓励参会者提供反馈并在实际环境中尝试
