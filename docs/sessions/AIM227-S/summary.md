# AWS re:Invent 2025 GenAI FinOps 会议总结

## 会议概述

本次会议由KPMG澳大利亚的TBM和FinOps实践负责人JJ Sharma以及IBM Cloudability的首席产品营销经理Andrew Midgley主讲，重点探讨了如何将FinOps实践应用于生成式AI投资管理。会议指出，尽管Gartner预测2025年全球生成式AI支出将达到6640亿美元，比2024年增长75%以上，但只有不到30%的AI领导者表示其CEO对投资回报感到满意。

演讲者通过电力发展的历史类比，阐述了AI技术的三个发展阶段：任务替代、人机协作增强，以及系统级变革。会议强调了在AI快速发展和成本急剧下降的环境中，组织需要建立财务纪律来确保AI投资能够产生可衡量的业务价值。

## 详细时间线与关键要点

### 0:00-10:00 开场介绍与AI市场现状
- 介绍演讲者背景和会议目标
- 强调AI投资规模巨大但ROI挑战严峻
- 提出通过FinOps实践来管理AI支出的必要性
- 概述会议议程：FinOps基础、AI趋势、部署模式、AWS服务定价、FinOps实践和单位经济学

### 10:00-15:00 FinOps基础与社区洞察
- 介绍FinOps Foundation的65,000名从业者社区
- 分析2025年FinOps状态报告：AI和ML支出从最低优先级跃升至中等优先级
- 强调治理现在比浪费减少更重要，反映出从战术优化向战略价值对齐的转变
- 声明本次演讲纯属教育性质，不推荐任何特定产品或服务

### 15:00-25:00 GenAI战略视角分析
- 通过"为什么"、"什么"、"如何"三个问题分析GenAI
- 展示ChatGPT-4价格在18个月内下降100倍的对数图表
- 将AI发展与电力发展进行类比：点解决方案→应用级改进→系统级变革
- 强调成本和治理考虑在三个不同规模下呈非线性变化

### 25:00-35:00 成本驱动因素与KPMG案例研究
- 用汽车运营成本类比说明AI成本的复杂性
- 详细分析AI成本结构：推理成本只是冰山一角，还包括安全、设计、可观测性、支持维护等
- 分享KPMG工作台的成功案例：从各国独立开发到全球统一平台
- 强调规模经济的重要性和FinOps在全球部署中的战略作用

### 35:00-45:00 AWS GenAI部署模式与服务详解
- 详细对比三种部署模式：SaaS API、PaaS（SageMaker和Bedrock）、IaaS（EC2/EKS）
- 分析客户数据：15%的EC2支出用于GPU实例，大部分AI支出仍在自托管GPU基础设施
- 深入讲解Amazon Bedrock的无服务器特性和基于令牌的定价
- 详细介绍SageMaker的端到端ML平台功能，包括构建、训练、推理三个阶段的成本结构

### 45:00-51:30 FinOps实践与单位经济学
- 强调成本分配是FinOps成功的基础，通过账户和标签实现直接映射
- 介绍共享AI基础设施的成本分配策略，使用令牌等遥测数据
- 详解优化策略：选择合适的基础模型、实例右调、缓存、批处理
- 介绍承诺折扣：EC2储蓄计划、SageMaker储蓄计划、Bedrock预置吞吐量
- 阐述三种规模的单位经济学：实验成本、生产力提升成本、战略优势价值
- 总结可行动建议：从高层次AI支出跟踪开始，逐步深入到成本分配和单位指标
