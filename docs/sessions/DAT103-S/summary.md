# AWS re:Invent 2025 会议总结：将孤立的企业系统数据转化为AI驱动的创新

## 会议概述

本次闪电演讲由MongoDB行业解决方案团队的高级专家Andrew Gomez主讲,时长20分钟。会议重点探讨了如何将分散在各个企业系统中的孤立数据整合起来,转化为AI驱动的创新应用。演讲者通过两个实际演示案例,展示了生成式AI、AI驱动搜索、检索增强生成(RAG)以及AI代理等技术如何改善客户体验和提升运营效率。

会议深入分析了当前许多AI项目无法投入生产的根本原因,特别强调了数据碎片化和数据质量问题。为解决这些挑战,演讲者介绍了操作数据层(Operational Data Layer, ODL)的架构方法,并详细阐述了如何使用MongoDB Atlas作为核心数据层来构建统一、可靠的数据基础设施。通过将所有数据和AI能力整合到单一平台,企业可以显著降低开发复杂度,加快创新速度,并为未来的技术浪潮做好准备。

## 详细时间线

0:00-1:30 - 开场介绍
- 演讲者Andrew Gomez自我介绍,来自MongoDB行业解决方案团队
- 团队职责:通过行业特定解决方案帮助组织最大化数据价值
- 会议议程预告:AI创新挑战、操作数据层解决方案、MongoDB Atlas的优势

1:30-3:45 - AI创新概念介绍
- 生成式AI(GenAI):自动化流程,如基于产品图片生成多语言描述
- AI驱动搜索:通过语义含义而非关键词进行智能搜索
- 检索增强生成(RAG):减少幻觉,通过专有数据扩展LLM能力
- AI代理和多代理系统:具备决策和推理能力,能够执行端到端流程

3:45-5:00 - 客户成功案例
- Nova Nordisk案例:使用LLM和专有数据生成临床研究报告,从12周缩短至10分钟
- 技术栈:AWS Bedrock(Claude 3和Titan)、MongoDB Atlas向量搜索
- DevRev案例:为内部团队提供支持聊天机器人,使用MongoDB Atlas、EKS和VPC配对

5:00-8:30 - 演示1:电子商务个性化推荐
- 用户在电商平台下单后触发发票和推荐微服务
- 数字收据包含订单交易和忠诚度信息
- 基于购买商品实时生成个性化产品推荐
- 用户可下载PDF版本用于财务管理
- 线下购买历史同步到在线账户,用于个性化首页推荐

8:30-10:30 - 演示2:金融投资组合管理系统
- 系统持续扫描市场洞察,展示绩效和资产图表
- 多样化ETF投资组合,包含VIX、GDP、利率、失业率等敏感度指标
- 三个每日定时代理持续扫描市场
- AI助手支持自然语言查询,获取最新市场数据洞察
- 代理自动处理非结构化和半结构化数据源(财务分析、新闻、社交媒体)
- 显示生成响应所使用的工具,保持透明度

10:30-12:30 - AI项目失败的原因分析
- 许多AI项目被放弃或未能达成目标
- 原因1:与组织战略目标不一致
- 原因2(重点):数据不完整和碎片化
- 数据质量差、数据不足或分散在遗留系统中难以整合
- 后果:不准确的结果、幻觉、糟糕的用户体验、品牌声誉受损
- 数据显示:超过60%的客户会在一次糟糕体验后更换品牌

12:30-15:00 - 操作数据层(ODL)架构介绍
- ODL定义:将孤立的企业数据整合到单一可靠数据层的架构蓝图
- 五层架构详解:
  1. 源系统层:数据原始来源,通常为单一用途构建的遗留系统
  2. 数据摄取层:通过ETL、CDC等机制将数据导入ODL,可使用MongoDB Atlas流处理
  3. ODL核心层:MongoDB Atlas整合结构化、非结构化和半结构化数据
  4. 处理层:负责安全性、加密、访问控制和策略过滤
  5. 应用层:包括操作应用、商业智能工具、GenAI和代理应用

15:00-18:00 - MongoDB Atlas作为ODL的关键优势
- 核心优势:简化开发,所有数据和能力集成在单一平台
- 灵活的文档模型:支持向量、图等复杂数据类型,操作数据与AI数据共存
- 代理内存能力:Atlas作为代理的内存层,存储上下文和学习历史
- 强大的查询能力:全文搜索、语义搜索、混合搜索、地理空间查询原生集成
- 向量搜索查询示例(来自数字收据演示)

18:00-19:30 - 集成便利性和企业就绪性
- 与Amazon Bedrock、LangGraph等框架的无缝集成
- 金融演示案例:使用Bedrock扩展能力,Atlas作为内存层,LangGraph作为编排器
- 与对象存储集成:数字收据存储在Amazon S3,URL嵌入文档元数据
- 企业级特性:在30多个AWS区域运行,数据加密(静态、传输和使用中)
- 集中化优势:更易于安全维护和团队入职

19:30-20:30 - 总结和关键要点
- 持续创新,利用AI技术,但要关注业务价值
- AI只是众多创新浪潮之一,投资正确的数据基础设施才能应对未来
