# AWS re:Invent 2025 会议总结：使用GenAI打破遗留系统壁垒并加速现代化

## 会议概述

本次AWS re:Invent 2025技术分会由高级专家解决方案架构师Anand Bilgaiyan和Sreekumar Nair主讲，主题为"打破遗留系统壁垒并使用GenAI加速现代化"。会议重点探讨了如何将单体遗留应用程序转换为基于微服务的云原生架构，通过结合架构方法论、最佳实践和GenAI的强大功能来实现应用程序现代化。

演讲者以Unicorn电商商店作为示例应用程序，展示了从单体架构到微服务架构的完整转换过程。该应用程序基于.NET Core 3.1框架构建，包含用户登录、产品搜索、购物车管理、支付和订单处理等标准电商功能。会议采用七阶段方法论，结合AWS的GenAI服务（如Kiro AI IDE和AWS Transform）来加速现代化进程。

## 详细时间线与关键要点

### 0:00-10:00 会议开场与问题定义
- 介绍演讲者和会议主题
- 阐述数字化转型的重要性以及遗留IT应用程序成为业务增长障碍的问题
- 提出使用GenAI加速现代化的解决方案
- 介绍Unicorn电商商店示例应用程序及其业务功能

### 10:00-20:00 单体架构挑战分析
- 详细分析单体应用程序的架构特性：所有模块捆绑在单一应用程序中
- 识别单体架构的主要挑战：
  - 扩展性问题（需要扩展整个应用程序而非单个模块）
  - 显著的变更时间影响业务敏捷性
  - 单点故障影响业务连续性
  - 运营挑战和技术债务
- 介绍七阶段现代化方法论

### 20:00-30:00 GenAI工具介绍
- 详细介绍Kiro AI驱动的IDE功能：
  - 规范驱动的软件开发方法
  - 支持新应用程序构建、现有应用程序修改和重构现代化
- 介绍AWS Transform服务的四大核心能力：
  - VMware迁移
  - 全栈Windows现代化
  - 大型机现代化
  - 自定义现代化

### 30:00-40:00 应用程序分析与框架升级演示
- 使用Kiro进行代码库分析演示
- 识别当前.NET 3.1框架并推荐升级到.NET 8
- 展示AWS Transform for .NET的三个关键阶段：
  - 分析阶段：识别需要修改的Windows依赖组件
  - 转换阶段：AI代理将应用程序转换为.NET 8
  - 验证阶段：确保转换后的应用程序正常运行
- 实际演示框架升级过程和验证

### 40:00-50:00 分解分析与架构方法论
- 介绍常见的分解陷阱：分布式单体和纳米服务
- 详细解释领域驱动设计（DDD）概念：
  - 域、子域的定义和分类（核心、支持、通用子域）
  - 有界上下文的重要性
- 介绍事件风暴方法论：
  - 七个关键元素：领域事件、命令、参与者、聚合、业务策略、外部系统、视图
  - 通过事件驱动的分解方法识别微服务边界
- 使用Kiro进行事件风暴分析演示

### 50:00-55:00 架构设计与最终重构
- 介绍模型上下文协议（MCP）及其在架构图生成中的应用
- 使用AWS架构图MCP服务器生成draw.io格式的架构图
- 最终的应用程序分解和重构演示：
  - 将单体应用程序分解为独立的微服务
  - 生成Web API、实体框架模型和数据库组件
  - 创建Kubernetes部署清单和CloudFormation模板
- 总结七阶段现代化流程的完整实现和关键收获
