# AWS re:Invent 2025 网络混沌工程会议总结

## 会议概述

本次会议由Forward Networks的首席解决方案架构师Craig Johnson主讲，主题是在AWS网络中引入受控混沌以提高网络弹性。Johnson拥有丰富的数据中心网络经验，曾在Cisco工作多年并持有CCIE认证。他将传统数据中心网络的混沌工程理念应用到AWS云网络环境中。

会议的核心理念是"永远不要在网络故障状态下退出变更窗口"。Johnson强调网络工程师经常被指责为系统问题的根源，因为网络是连接所有组件的胶水。为了应对这一挑战，他提出了基于意图验证的网络测试方法，使用AWS的可达性分析器等工具来主动验证网络配置的正确性，而不是被动等待应用团队确认系统状态。
