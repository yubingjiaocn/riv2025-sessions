# AWS re:Invent 2025 - 通过 AI Ops 加速事件响应

## 会议概述

本次会议由 Prattal Chakra(AWS 全球云运营专家团队负责人)和 Andres Silva(云运营团队首席解决方案架构师)主讲,重点介绍了如何通过 AI Ops 加速云运营中的事件响应。

演讲者巧妙地使用了 Formula 1 赛车的类比来说明技术演进的重要性。Formula 1 通过持续采用新技术,将进站时间从 1950 年代的 67 秒缩短到现在的平均 2.3 秒,实现了 96% 的改进。这一演进过程包括:采用最新技术(如气动扳手)、数据驱动决策(每辆赛车配备 300 个传感器)、以及精心编排的流程。同样,企业也需要通过采用 AI Ops 技术来缩短事件响应时间,将原本需要数小时解决的问题缩短到分钟甚至秒级。

会议强调了 CloudWatch 作为 AWS 旗舰可观测性服务的演进历程,从 2014 年的日志服务发展到今天支持统一数据管理、自然语言查询、应用程序映射、生成式 AI 可观测性等功能。AWS 自身就是 CloudWatch 最大的客户,每月处理 17 EB 的日志数据、32 千万亿次指标观测和 9.32 亿次 Canary 运行。演讲者详细介绍了多项 AI 创新功能,包括统一遥测数据存储、MCP 服务器、CloudWatch Investigations 以及 AWS DevOps Agent 等,展示了如何通过这些技术实现人机协作的 AI 辅助决策。

21:00 - 23:00 - 数据挑战

- 企业面临类似挑战:基础设施日志、应用日志、数据库、流服务等数据分散

23:00 - 25:30 - CloudWatch 统一数据管理发布
- Matt Garman 当天上午宣布的新功能
- 统一三大用例:运营、安全、合规
- 提供丰富的查询和搜索界面
- 支持分析层级和遥测管道

25:30 - 28:30 - 关键特性

- 完整的遥测管道:数据摄取、转换、消费
- S3 Tables 集成:通过 Apache Iceberg 和 Athena 访问数据

### MCP 服务器 (41:30 - 48:00)

41:30 - 43:00 - MCP 服务器概念
- 类比 Formula 1 的 1993 年双向遥测技术突破
- 之前只能获取数据,1993 年后可以远程调整赛车设置
- 带来每圈 0.3 秒优势,帮助车队赢得多个冠军

43:00 - 45:00 - MCP 服务器重要性
- 被描述为"AI 模型与 API 之间的 USB-C 标准化连接"
- 实现双向通信:获取遥测数据 + 执行操作
- 对 AI Ops 至关重要:需要安全高效地获取数据并采取行动

45:00 - 48:00 - AWS MCP 服务器
- 目前提供三个 CloudWatch 相关 MCP 服务器:
  - CloudWatch MCP Server
  - Application Signals MCP Server
  - CloudTrail MCP Server
- 可与 Kira、VS Code Q Developer 等工具集成
- 可构建自定义代理应用程序
- 支持自动修复场景
