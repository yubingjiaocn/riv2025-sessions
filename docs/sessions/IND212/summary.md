# AWS re:Invent 2025 会议总结：利用生成式AI推进医疗业务运营分析

## 会议概述

本次会议由Huron Consulting Group的Shane O'Connor和Kendra主讲，重点介绍了如何在医疗保健领域应用生成式AI技术来改善患者体验和业务运营分析。Huron是一家在教育和医疗行业具有深厚影响力的全球咨询公司，致力于通过AWS架构和生成式AI能力为客户提供创新服务。

会议主要聚焦两个核心领域：一是患者体验管理，通过Huron Rounding工具实现结构化的循证方法来改善患者体验；二是业务运营优化，涵盖收入周期、供应链、药房和人力资源等方面。演讲者强调，生成式AI使他们能够突破传统结构化数据的局限，深入挖掘非结构化数据中的洞察，从而为客户提供更具可操作性的建议。通过AWS Bedrock、Nova LLM模型和Redshift等服务，Huron成功实现了情感分析自动化，并计划推出实时转录和智能巡房等创新功能。

会议还展示了具体的应用成果，包括90%的情感分析准确率、每周处理10,000条笔记的能力，以及如何通过优化HCAHPS评分来直接影响医院收入。未来路线图包括将情感数据整合到商业智能平台、提供实时教练指导，以及连接患者体验与市场份额和净收入机会之间的关系。

## 详细时间线

0:00 - 0:45 会议开场与介绍
- Shane O'Connor和Kendra介绍会议主题：利用生成式AI推进患者业务运营分析
- 介绍Huron Consulting Group在医疗和教育行业的全球咨询地位

0:45 - 2:15 Huron Rounding工具介绍
- Kendra详细介绍Huron Rounding：一种结构化的循证方法，通过领导者、护理人员、患者和家属之间的有意义互动来改善患者体验
- 强调护士领导和管理人员的持续巡房如何加强员工关系、识别绩效障碍并快速升级问题

2:15 - 3:30 业务运营领域应用
- Shane介绍业务运营咨询重点：改善客户财务绩效，涵盖收入周期、供应链、药房和人力资源运营
- 强调数据在Huron DNA中的核心地位
- 说明生成式AI如何帮助突破结构化数据限制，利用LLM理解非结构化数据

3:30 - 5:00 生成式AI在患者体验中的应用
- 介绍平台如何分析巡房会话的调查反馈，识别影响HCAHPS绩效的关键患者体验驱动因素
- 解释HCAHPS（医院消费者评估）：出院患者调查，评分越高医院获得的Medicare资金越多
- 指出传统HCAHPS数据的缺陷：需要3-4个月才能收到反馈，错失早期干预机会

5:00 - 6:30 患者体验架构详解
- 展示当前架构：使用Amazon Bedrock、Nova LLM模型和Redshift
- Event Bridge作为调度器触发每日情感分析的Step Function
- Nova Micro模型在Redshift中注册为可调用函数
- 模型根据问答文本返回四种情感之一

6:30 - 8:00 业务运营架构对比
- Shane介绍业务运营架构流程：从客户接收原始数据（平面文件等）
- 数据存储在S3，转移到Glue Catalog表中
- 通过DBT在Redshift中进行数据转换和策划
- 整合不同数据源，构建从账单到解决的完整索赔路径
- 输出方式：通过Amazon QuickSight可视化或连接到低代码应用进行临时Bedrock调用

8:00 - 9:30 患者体验提升策略
- 分析调查响应以发现潜在情绪
- 快速检测不满意和改善机会，提高患者满意度和忠诚度
- 识别患者互动模式以指导改进和优化护理交付

9:30 - 11:00 真实案例展示
- 展示患者体验调查问卷模板示例
- 案例分析：患者对预约系统表达沮丧（负面情感），但对预约前信息和现场服务表示满意（正面情感）
- 提取关于预约安排和医护人员互动的主题

11:00 - 13:30 Rounding AI的未来愿景
- 录音和转录流程：巡房人员可以开启录音，应用程序在后台无缝记录
- 消除护士面对设备和繁重笔记工作的不便
- 三种处理方式：
  1. 实时流式处理：问答即时记录
  2. 批处理：适用于低网络连接区域
  3. 语音备忘录上传：作为备用方案

13:30 - 15:00 AI流程的灵活性与配置
- 巡房人员可以用自己的话以任何顺序提问，无需照本宣科
- 完全可按地点、主题和模板配置
- 适用于患者体验和员工巡房，提供教练指导机会
- 自动记录、输入和分类关键细节（问题、表彰、备注）

15:00 - 16:30 情感分析增强功能
- 继续提供每个问题的情感分析
- 新增整体巡房情感评分
- 安全存储转录记录供未来审查
- 提供AI选择答案的解释，帮助医护人员理解AI决策
- 用户拥有完全自主权更新AI选择的答案

16:30 - 18:30 业务运营深度洞察
- 整合不同数据源构建财务绩效历史
- 使用LLM突出显示付款方关系、员工绩效和拒付管理中的改进领域
- 原子级别（单个索赔或患者历史）和群体健康级别（跨付款方和服务线的趋势）分析
- 识别关键问题、拒付模式和员工交接中的深层洞察
- 开发评分系统帮助客户理解机会

18:30 - 20:00 员工效能管理
- 重点关注从账单到全额支付和解决的工作流程效能
- 将结果输出到QuickSight可视化的表格
- 提供员工级别绩效和效能评分的整体图景
- 利用架构处理自由文本笔记和非结构化数据，提供以前需要人工审查的额外洞察

20:00 - 22:00 成果展示
- 情感分析准确率达到90%（从75%提升到98%）
- 目标接近100%准确率
- 创建智能巡房：使用实时情感分析识别有不良体验风险的患者
- 优先处理之前有不良体验的患者，确保问题得到解决
- 通过优化HCAHPS资金直接影响医院收入（HCAHPS反馈通常需要3-4个月，早期检测可提高评分）

22:00 - 23:30 业务运营规模化成果
- 每周自动审查10,000条笔记
- 保持约90%的准确率
- 随着客户群扩大，处理能力将大幅提升
- 识别净收入机会和更深层次的拒付趋势

23:30 - 25:30 未来路线图
- 通过商业智能分析和平台（Amazon QuickSight）聚合情感数据与业务运营输出
- 寻找增强教练指导的机会：将结果连接到实时教练指导，改善患者互动和员工满意度
- 强调教练指导是Huron核心价值：不仅提供仪表板，更要与客户合作制定战略
- 语言模型总结自由文本使战略制定更快速

25:30 - 27:00 数据整合与未来机会
- 计划连接患者体验和业务运营数据
- 探索数据间的隐藏关联
- 研究患者体验如何转化为市场份额
- 分析患者情感如何反映潜在的净收入机会
