# AWS re:Invent 2025 技术会议总结：使用无服务器架构和MCP服务器构建AI代理

## 会议概述

本次技术分享会由AWS解决方案架构师Rahul Popot和Ed Schaeffer主讲，重点介绍如何使用Strands框架和Model Context Protocol (MCP)服务器构建AI代理。会议涵盖了代理式AI的核心概念、Strands开源框架的使用方法，以及如何通过MCP协议集成外部工具和服务。演讲者通过实际编码演示展示了在AWS无服务器架构上部署AI代理的完整流程，包括Lambda函数、API Gateway和安全认证等关键组件。

## 详细时间线与关键要点

### 00:00-05:00 开场介绍与背景
- 演讲者自我介绍：Rahul Popot（6年AWS经验，新泽西）和Ed Schaeffer（5年AWS经验，费城地区）
- 会议主题：使用无服务器架构和MCP服务器构建AI代理
- 议程预览：代理式AI概念、Strands框架、外部工具集成、部署选项和实时编码演示

### 05:00-10:00 AI代理的必要性分析
- LLM的局限性：被动响应、知识局限于训练数据、缺乏实时信息和上下文感知
- AI代理的优势：能够访问外部系统、执行操作、构建复杂工作流程
- 代理循环核心机制：推理→工具发现与选择→工具执行→响应生成的循环过程
- 现场调研：部分参与者已使用AI代理进行日常任务自动化

### 10:00-15:00 Strands框架介绍
- Strands Agent SDK：2024年7月发布的开源Python和TypeScript框架
- 设计理念：几行代码即可构建生产就绪的代理
- 模型优先方法：利用先进LLM进行推理、规划和工作流设计
- 支持多种模型提供商：Bedrock、SageMaker、Gemini、OpenAI等

### 15:00-20:00 代理的三大核心组件
- 系统提示词：定义代理的角色和行为模式
- 模型访问：支持工具使用的任何模型
- 工具/资源：代理可调用的功能集合
- 代码示例：展示基础聊天机器人的实现方式

### 20:00-25:00 工具集成与自定义
- 内置工具：计算器、文件系统交互、HTTP请求等
- 自定义工具：使用@tool装饰器创建Python方法
- 工具元数据提取：自动生成OpenAPI兼容规范
- 天气工具示例：演示简单的自定义工具实现

### 25:00-30:00 Model Context Protocol (MCP)详解
- MCP定义：连接AI应用程序的开源标准
- 问题解决：统一AI代理与各种工具的通信接口
- 类比说明：如同微服务的REST API标准，为AI应用提供"USB-C接口"
- 架构优势：从M×N问题简化为M+N问题

### 30:00-35:00 MCP集成与Strands支持
- Strands内置MCP客户端：无需手动实现协议规范
- 传输层支持：STDIO本地连接、HTTP流式客户端
- 动态工具发现：自动加载和更新可用工具列表
- 团队协作优势：各团队独立更新MCP服务器，无需重新部署客户端

### 35:00-40:00 无服务器架构选择理由
- 按需付费：仅在使用时收费，避免空闲资源浪费
- 自动扩缩容：应对峰值流量（如黑色星期五）
- 内置安全性和可观测性：提供完整的监控和调试能力
- 架构设计：两个核心组件 - MCP服务器和Strands代理

### 40:00-45:00 实时编码演示
- MCP服务器实现：掷骰子功能，集成DynamoDB状态管理
- Lambda授权器：使用"good token"进行安全认证
- MCP Inspector测试：验证服务器功能和安全性
- Strands代理部署：使用SAM模板在Lambda上部署

### 45:00-48:30 演示测试与总结
- 天气查询测试：成功获取拉斯维加斯实时天气信息
- 通勤建议测试：AI代理自动调用天气API并提供出行建议
- 综合功能测试：同时调用天气、通勤和掷骰子三个工具
- 代理智能推理：无需指定调用顺序，自动确定工具使用序列
