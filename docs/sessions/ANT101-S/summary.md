# AWS re:Invent 2025 会议总结：Evercommerce 数据现代化之旅

## 会议概述

本次会议由 Fivetran 企业销售副总裁 John Sedlan 和 Evercommerce 数据与分析副总裁 Kulpit Patel 共同主持,重点讨论了 Evercommerce 公司的数据现代化转型历程。Evercommerce 是一家领先的服务商务平台,为全球 40 多个国家的约 70 万小企业客户提供 SaaS 解决方案。该公司通过收购整合了 50 多家独立运营的公司,面临着严重的数据孤岛、数据重复和缺乏统一治理等挑战。

Kulpit Patel 在加入公司 18 个月后,通过前 3 个月内进行的 60 多次访谈,发现 Evercommerce 是一家"数据饥渴"的公司。为了将公司从"数据饥渴"转变为"数据驱动型企业",他们启动了名为"数据工厂"(Data Factory)的战略项目。该项目采用 Fivetran 的托管数据湖服务(Managed Data Lake),基于 AWS S3 构建开放数据平台,使用 Apache Iceberg 架构。这一解决方案不仅解决了当前的数据整合问题,还为未来的 AI/ML 应用和数据货币化奠定了基础,实现了"构建一次,多次消费"的目标。

## 详细时间线与关键要点

[00:00:00 - 00:02:30] 开场介绍与 Fivetran 公司概况
- John Sedlan 介绍自己是 Fivetran 企业销售副总裁
- Fivetran 是一家专注于自动化、安全和可扩展数据移动的数据摄取公司
- 公司拥有约 1500 名员工,提供 700 多个预构建连接器
- 服务全球 7000 多家客户,覆盖北美、欧洲和亚太地区

[00:02:30 - 00:04:00] Fivetran 技术架构说明
- 支持从操作数据存储、SaaS 应用(如 Salesforce、Zendesk、Workday)到大型数据库(Oracle、SQL、DB2)的数据源
- 提供自动化数据管道,无需手动维护,自动处理架构漂移
- 数据存储在 AWS S3 中,支持实时分析、业务运营、云迁移和 AI/ML 应用

[00:04:00 - 00:05:30] Kulpit Patel 介绍 Evercommerce
- Kulpit Patel 是 Evercommerce 企业数据副总裁
- Evercommerce 是数字原生服务商务平台,为小企业主提供 SaaS 解决方案
- 业务覆盖 40 多个国家,服务约 70 万客户
- 公司使命是让小企业主专注于业务运营,而非技术问题

[00:05:30 - 00:07:30] 数据基础设施的初始状态
- 通过收购整合了 50 多家公司,面临人员、流程和技术三方面的挑战
- 存在严重的数据孤岛和数据重复问题
- Kulpit 加入公司 18 个月,前 3 个月进行了 60 次访谈
- 发现公司是"数据饥渴型"企业,需要以光速提供数据访问能力

[00:07:30 - 00:10:00] 数据现代化的主要挑战
- 三大核心挑战:理解业务、提供数据可访问性、建立数据治理
- 在 AI 时代,缺乏数据整合和高质量数据基础是最大障碍
- 提出"数据工厂"概念,分三个阶段:建立工厂基础、部署机器设备、投入生产运营

[00:10:00 - 00:12:00] 与 Fivetran 的战略合作
- Evercommerce 与 Fivetran 合作已有 8-10 年
- 去年采取战略性转变,从在各个公司单独使用 Fivetran 转向企业级统一使用
- 采用 Fivetran 托管数据湖(Managed Data Lake)产品
- 基于 Apache Iceberg 构建完全开放的数据平台,底层使用 AWS S3

[00:12:00 - 00:14:30] 跨品牌可见性与业务影响
- 通过数据工厂实现"构建一次,多次消费"的目标
- 统一数据资产管理、治理和质量控制
- 实现营销效果、成就评分、漏斗分析、潜在客户生成的统一视图
- 建立标准化数据定义和企业级报告能力
- 重点不仅是技术实施,更是变革管理和业务认同

[00:14:30 - 00:16:30] 选择 AWS S3 的决策过程
- 公司使用多个云平台(AWS、Azure 等),但选择 AWS 作为核心平台
- 需要坚如磐石的基础层来支持数据转换和处理
- 运行约 100 个不同的连接器,频率从每分钟到每小时不等
- S3 提供无缝集成能力,是市场上最佳的存储引擎之一
- 结合 Fivetran 托管数据湖和 Apache Iceberg,构建面向未来的开放架构

[00:16:30 - 00:19:00] 未来项目规划
- 完成数据工厂基础建设后,重点转向 AI/ML 就绪的基础设施
- 应用场景包括:客户流失预测、客户留存分析、交叉销售和向上销售倾向分析
- 计划全面利用 Fivetran 生态系统,特别是与 dbt 的集成
- 采用"左移"(Shift Left)策略,在数据加载时就进行治理和质量控制
- 理念:坏数据就是坏数据,必须在源头控制质量以避免技术债务

[00:19:00 - 00:21:30] 360 度数据旅程与技术整合
- Fivetran 正在与 dbt 合并,并收购了反向 ETL 公司 Census
- Evercommerce 创建了"DNA 飞轮"(DNA Flywheel)概念
- 关注数据生产者和消费者的全生命周期
- 不以技术为先,而是以业务转型为导向
- 最终目标:通过 API 即服务和数据货币化达到理想状态(Nirvana State)
