# AWS re:Invent 2025 会议总结：Graphite - AI驱动的代码审查平台

## 会议概述

本次会议介绍了Graphite公司开发的AI驱动代码审查平台，该平台专为新时代的AI辅助软件开发而设计。演讲者指出，世界上许多发展最快的公司和工程团队已经不再使用GitHub进行传统的代码审查，因为AI生成代码的时代需要一个全新的、能够跟上软件编写速度的代码审查平台。Graphite目前服务于Shopify、Snowflake、Robinhood、Duolingo等顶级企业客户，并取得了显著成效。

演讲的核心论点是：虽然AI工具（如Claude、Cursor、Windsurf等）极大地提升了代码编写速度，但代码审查、测试、合并和部署等"外循环"流程已成为新的瓶颈。Shopify在使用Graphite后，每位开发者合并的PR数量增加了33%，ASA公司每位工程师交付的代码行数增加了21%。Graphite不仅提供AI审查代理，还重新设计了整个代码审查体验，包括统一的工作流收件箱、自我修复CI、合并冲突解决等功能。演讲者强调，仅靠基础大语言模型无法胜任代码审查工作，需要专门的工程化平台来解决"生成器-验证器差距"问题。

## 详细时间线

0:00-0:30 - 开场与核心观点
- 揭示行业秘密：许多顶尖公司已不再使用GitHub进行代码审查

0:30-1:30 - 客户案例与成果展示

- ASA成果：每位工程师交付代码增加21%
- 服务数十万工程师，取得可验证的企业价值

1:30-3:00 - 为什么代码审查至关重要
- 编写代码只是第一步，外循环流程（审查、测试、合并、部署）才是真正的瓶颈
- 在Meta和Google等大公司，等待代码审查的时间常常超过开发功能本身
- AI工具使问题恶化10倍：工程师能更快生成代码，但审查需求激增
- 引用推文："新瓶颈不再是完成工作，而是审查工作"

3:00-4:00 - AI代码生成需要AI代码审查
- 结论一：AI代码生成必须配合AI代码审查才能真正释放LLM的潜力
- 不能只在IDE中应用AI，需要在整个工具链中应用AI

4:00-5:30 - 为什么基础LLM不够用
- 初期尝试：直接将代码差异提交给Claude进行审查
- 失败案例：AI建议更新代码但实际没有改变、CSS理解错误、建议添加无意义注释、建议回退代码
- 类比：就像让实习生第一周就做复杂重构
- 核心问题：LLM擅长生成代码，但审查和理解代码是更难的问题
- 引入概念："生成器-验证器差距"（generator-validator gap）

5:30-7:30 - Graphite的多维度解决方案
- AI审查代理：几秒内扫描每个PR，发现bug、安全漏洞等问题
- 统一工作流收件箱：显示待审查PR和进行中的更改
- 自我修复CI：测试失败时AI建议修复方案
- 即将推出：合并冲突自动解决
- 开发者生产力度量
- 整合为统一平台
