# AWS re:Invent 2025：使用Amazon Bedrock和Kiro加速开发和DevSecOps

## 会议概述

本次会议由AWS EMEA合作伙伴解决方案架构团队负责人Dhana Vadivelan主持，联合巴西的Fabio Oliveira和AWS大使、Xebia Cloud CTO Joris Conijn共同演讲。会议重点介绍了如何通过智能体AI（Agentic AI）和平台驱动方法来变革软件开发生命周期（SDLC），不仅限于代码生成，而是覆盖从需求收集到部署运维的全流程。

演讲者分享了AWS团队与合作伙伴网络合作一年来的实践经验，展示了20多个架构模式和最佳实践。会议通过AnyCompanyReads虚拟公司案例，演示了如何将传统18个月的软件发布周期缩短至6-8个月。核心内容包括Amazon Bedrock AgentCore平台、Kiro智能体IDE和CLI工具，以及Xebia为Delta Airlines开发的ACE平台成功案例。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍和背景
- 演讲者自我介绍，观众构成调查（开发者、技术领导、解决方案架构师）
- 说明会议目标：展示如何使用智能体超越代码生成来加速SDLC
- 介绍团队一年来创建架构模式和最佳实践的历程

### 5:00-15:00 传统SDLC挑战和财务影响
- 量化生产力损失：78%软件工程师花费30%时间处理重复手工任务
- 财务影响分析：250人团队年损失1340万美元生产力
- Atlassian研究：20%开发时间因低效率而损失
- 更好的开发者体验可带来30%生产力提升

### 15:00-25:00 智能体AI的必要性和Gartner预测
- Gartner魔力象限：到2028年90%企业软件工程团队将使用AI代码助手
- 从2024年14%到2028年90%的6倍增长
- 三大挑战类别：工具集和开发环境碎片化、集成和互操作性、企业级安全可观测性和可扩展性

### 25:00-35:00 智能体AI工作原理和设计最佳实践
- 智能体四大核心组件：记忆、工具集成、规划能力、行动执行
- 设计最佳实践四个维度：框架选择、工具和知识库、LLM选择、执行技术
- AWS智能体AI三种方法：专业化解决方案、完全托管服务、DIY方法

### 35:00-45:00 平台驱动架构方法
- 解决方案架构概述：ReactJS UI + Cognito认证 + Bedrock Agent后端
- 多个架构模式：业务分析师、项目知识库、架构设计、开发者、测试、运维
- 项目知识库作为中央存储，支持跨角色信息传递和追溯

### 45:00-60:00 实际演示和案例研究
- **Emily（业务分析师）**：使用Confluence和Jira集成，自动提取需求创建史诗和验收标准
- **Adam（解决方案架构师）**：生成架构图和OpenAPI规范，从组织最佳实践出发
- **Anna（开发者）**：在Kiro IDE中进行规范驱动开发，从需求到生产就绪应用
- **Chris（DevSecOps工程师）**：使用Kiro CLI进行日志分析和问题分类

### 60:00-61:30 客户成功案例和总结

- **其他案例**：Iskaan建筑管理公司、Zonevoice现代化项目
