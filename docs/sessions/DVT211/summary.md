# AWS re:Invent 2025 - 使用生成式 AI 加速 .NET 应用现代化

## 会议概述

本次 AWS re:Invent 2025 分会场由 AWS 高级全球专家 Derek Dejarden 和 Thomson Reuters 平台工程副总裁 Matt Dimich 共同主讲,主题聚焦如何利用生成式 AI 工具加速 .NET 应用程序的现代化改造。会议深入探讨了企业在应用现代化过程中面临的核心挑战——资源短缺问题,并展示了 AWS 如何通过 AWS Transform for .NET 和 Kiro AI IDE 这两个创新工具来解决这一痛点。

演讲者首先通过现场调查发现,几乎所有与会者都有正在进行的现代化项目,但相当一部分项目因缺乏工程资源而停滞。传统的 .NET Framework 应用现代化需要大量资深工程师投入,而这些工程师通常被分配到更关键的功能开发项目中。AWS Transform for .NET 的推出改变了这一局面——它是一个异步的、基于 AI 代理的服务,能够在云端自动完成 40-70% 的代码转换工作,无需占用本地开发资源。会议通过实际演示展示了完整的现代化工作流程:从使用 AWS Transform for .NET 自动转换代码,到使用 Kiro IDE 完成剩余工作和验证,最终成功将一个 .NET Framework 4.8 应用迁移到 .NET Core 8 并在 Mac 上运行。

演讲还分享了 Hartford、Experian 和 Thomson Reuters 等企业的成功案例,这些企业通过使用 AWS 的现代化工具,显著缩短了项目周期,节省了数百个工程日,并将更多工程师资源释放到创新和功能开发上。会议强调,通过将应用现代化与云迁移同步进行,企业不仅能够降低 40% 的运营成本,还能获得 1.5-2 倍的性能提升和 50% 的可扩展性增强,并为容器化、Kubernetes 和无服务器架构等现代云技术打开大门。

- 现代化项目 ROI 实现周期长
- 技术债务持续累积,阻碍创新和 AI 集成

00:03:45 - 现代化演进历程
- 过去:先"提升并迁移"到云,应用现代化推迟到后期
- 现在:利用生成式 AI 工具,可以在迁移过程中同步完成应用现代化
- 新方法从一开始就消除技术债务,立即启用节省成本的云技术

00:05:00 - 现代化的价值主张
- 成本:Linux 运行成本通常降低 40%
- 性能:Linux 应用性能提升 1.5-2 倍
- 可扩展性:提升 50%,需要更少的虚拟机
- 兼容性:支持 ARM64/Graviton 处理器
- 现代技术:容器化、Kubernetes 编排、Lambda 无服务器

00:06:30 - 项目停滞的根本原因
- 展示典型现代化流程:分析阶段 → 转换阶段 → 验证阶段
- 核心问题:缺乏工程资源
- 现代化需要熟悉遗留技术栈和业务逻辑的资深工程师
- 这些工程师通常被分配到功能开发项目

00:08:00 - 生成式 AI 的局限性
- 编码助手(如 Q Developer、Kiro)可以加速工程师工作
- 但这些工具仍需要工程师操作,无法解决资源短缺问题

00:09:00 - AWS Transform for .NET 介绍
- 作为异步"离键盘"服务推出
- 使用云端运行的 AI 代理替代部分资深工程资源需求
- 自动化 40-70% 的转换任务
- 始终保持人工参与(分析和验证阶段)

00:11:00 - 服务架构详解
- 首个用于大规模现代化 .NET 应用的异步代理体验
- 结合两个组件:
  - 规则引擎(10 年现代化经验积累,运行快速且成本低)
  - 生成式 AI(使用 Amazon Bedrock,提高完成率,持续改进规则引擎)
- 提供 IDE 体验(Visual Studio 2022/2026)和 Web 体验(批量转换)

00:13:30 - IDE 开发者体验演示
- 在 Visual Studio 中选择要转换的项目
- 执行自动评估,生成报告
- 报告显示转换步骤、潜在问题和难度评估
- 可自定义转换计划
- 右键点击执行转换,代码上传到云端处理
- 异步执行,不占用本地机器或 IDE 资源
- 支持百万行级别的大型单体应用转换

00:16:00 - Web 体验介绍
- 用于大规模批量转换
- 通过代码连接或 S3 连接源代码仓库
- 发现并转换整个仓库(而非单个项目)
- 可同时运行 5-10 个转换任务
- 支持上传私有 NuGet 包或指向制品仓库

00:18:00 - 完整工作流演示预告
- 将演示端到端的 .NET 现代化工作流
- 使用 AWS Transform for .NET 和 Kiro AI IDE

00:19:30 - 实际演示开始
