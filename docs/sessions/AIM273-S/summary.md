# AWS re:Invent 2025 金融服务风险管理平台建设分享

## 会议概述

本次分享会由普华永道合伙人Lakshman主持，邀请了Capital One的风险技术领导团队分享其自主构建云原生风险管理平台的经验。Matt作为产品管理副总裁负责产品战略，Solomon作为软件工程副总裁专注于架构实施。Capital One在2019年就做出了在AWS上原生构建GRC（治理、风险与合规）平台的前瞻性决策，而非采购传统供应商解决方案。

该平台服务于Capital One内部约10,000名风险从业者，涵盖各业务线和各道防线。Capital One将自己定位为"恰好从事银行业务的科技公司"，将风险管理视为核心业务而非仅仅是控制职能。他们的愿景是构建实时智能风险管理平台，从传统的事后记录转向主动风险预测和管理。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与公司理念
- Lakshman介绍嘉宾背景和会议目的
- Matt阐述Capital One作为科技公司的理念：风险管理即业务本身
- Solomon强调云原生转型如何改变了解决业务问题的方式

### 5:00-10:00 平台愿景与用户规模
- **核心愿景**：构建实时智能风险管理，从被动记录转向主动预测
- **用户规模**：约10,000名内部用户使用该平台
- **设计理念**：不模仿市场现有产品，而是创造全新的解决方案
- 引入净推荐值(NPS)评估用户满意度，这在风险管理软件中极为罕见

### 10:00-15:00 架构设计与对象模型
- **核心架构**：基于对象模型设计，包括风险、控制、问题和事件四大对象
- **关键转变**：从产品思维转向平台思维
- **对象关系**：建立对象间关系，实现交互和事务处理
- 通过评估作为与对象的交易，使整个系统变得"活跃"

### 15:00-20:00 实时风险管理的实现
- **实时需求**：全球互联环境下，风险管理必须实时响应市场变化
- **技术支撑**：利用社交媒体、情感分析等实时数据源
- **监管服务**：将监管机构视为客户，实时证明安全性
- **云原生优势**：能够快速采用AWS新服务，专注于差异化功能开发

### 20:00-25:00 安全工程实践
- **数据安全**：采用高级加密技术保护静态和传输中的数据
- **应用安全**：实施复杂的身份管理和应用级安全控制
- **开发安全**：将安全嵌入开发生命周期的第一步，而非事后添加
- **自动化扫描**：从代码提交到GIT开始就进行安全扫描

### 25:00-30:00 自建vs采购的决策逻辑
- **战略考量**：GRC是风险管理的核心，影响公司最重要的风险管理功能
- **用户规模**：10,000用户覆盖所有业务线，具有重塑风险管理的巨大潜力
- **灵活性需求**：现有供应商解决方案缺乏业务所需的灵活性
- **分层迁移**：采用逐步迁移策略，先迁移风险对象，再迁移控制对象

### 30:00-35:00 监管合作与AI应用
- **监管透明**：向监管机构开放平台访问，将其视为客户
- **AI风险管理**：既要管理AI引入的风险，也要利用AI提升风险管理能力
- **监管科技**：预期监管机构将采用更先进的技术进行监管
- **全文件测试**：澳大利亚已开始实施全文件测试，预计将推广到其他地区

### 35:00-41:00 未来展望与经验分享
- **用户反馈**：每半年进行一次NPS调查，持续改进产品
- **变更管理**：在不中断监管报告的前提下更换系统的挑战
- **功能实验**：使用功能标志进行A/B测试，获取用户反馈
- **关键建议**：充分了解投资规模，做好长期承诺准备
- **未来愿景**：实现真正的实时智能风险管理，从被动应对转向主动预测
