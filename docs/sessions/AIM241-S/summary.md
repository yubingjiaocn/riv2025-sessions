# AWS re:Invent 2025 医疗AI与数据转型专题讨论会总结

## 会议概述

本次专题讨论会聚焦医疗行业的AI和数据转型，由PwC云数据与AI健康服务负责人Abbas Mooraj主持，邀请了四位来自美国不同地区医疗机构的技术领导者。参会嘉宾包括Banner Healthcare首席技术官Mike Regan、Nuvance Health首席信息官Geoff Hook、WellSense Health Plan首席信息官Raj Nair，以及Advocate Health首席AI与数字官Andy Crowder。这些机构代表了美国16个州的医疗服务。

讨论会围绕"从数据到AI再到价值实现"的主题展开，重点探讨了医疗行业如何通过现代化数据平台和AI技术应对当前面临的边际压力、监管压力以及人力短缺等挑战。与会者强调，在医疗行业实现真正的AI转型需要良好的数据基础、战略合作伙伴关系以及组织文化的根本性变革。

## 详细时间线与关键要点

### 开场介绍 (0:00-5:00)
- Abbas Mooraj介绍会议背景，强调这是连续第四年举办此类讨论
- 四位嘉宾分别介绍各自组织和职责范围
- Mike Regan负责Banner Healthcare的传统IT、数字化和数据技术
- Geoff Hook管理Nuvance Health的数据、AI、分析和安全技术栈
- Raj Nair担任WellSense Health Plan CIO，同时参与Boston Medical Center技术领导
- Andy Crowder负责Advocate Health的AI战略和数字解决方案

### 数据平台现代化之路 (5:00-15:00)
- **Geoff分享Nuvance的数据之旅**：
  - 认识到医疗数据存在孤岛问题，各应用系统都希望保留自己的数据
  - 决定将所有数据迁移到安全的云平台，采用"将逻辑带到数据"而非"将数据带到系统"的理念
  - 选择AWS原生服务以避免技术债务，让有限的团队资源更高效
- **Raj的双重策略**：
  - 发现组织存在多个数据仓库，数据质量极差
  - 采用云现代化策略，同时启动AI工厂概念
  - 专注于数据质量高或非结构化数据的AI用例

### AI实施与价值实现 (15:00-30:00)
- **Banner的数据平台战略**：
  - Mike强调从以系统为中心转向以基础设施为中心
  - 构建Banner数据平台，基于AWS上的Databricks
  - 目标是将所有临床数据实时更新到统一平台
- **Advocate Health的治理框架**：
  - Andy介绍"最佳护理地点"战略，整合750名变革推动者
  - 开发Fair AI框架，建立闭环流程和多学科审查团队
  - 强调临床医生正在引领AI应用，而非后台部门

### 合作伙伴关系与创新 (30:00-40:00)
- **战略合作的重要性**：
  - Andy分享与Aidoc等公司的合作经验，部署15个算法
  - 强调选择正确合作伙伴进行共同开发和创新的重要性
- **AI工厂模式**：
  - Raj解释AI工厂概念：证明某个能力的价值后，快速部署到其他类似用例
  - 目前有40个用例在生产环境，AI项目完全自筹资金并实现2倍回报
- **多模型策略**：
  - 采用成本效益最优的解决方案，主要使用Anthropic的Claude模型
  - 保持灵活性，能够快速切换到更好的选择

### 生产力提升与流程重构 (40:00-50:00)
- **医疗行业生产力挑战**：
  - Geoff指出医疗行业生产力数十年来保持平稳，而其他行业持续增长
  - AI成为提升生产力的关键杠杆，特别是在人力短缺的背景下
- **收入周期自动化**：
  - 讨论AI在收入周期管理中的应用，包括预授权、拒付申诉等
- **呼叫中心转型**：
  - Mike分享使用Nova Sonic 2进行对话式AI的经验
  - 将呼叫中心视为整个组织沟通的缩影，具有广泛自动化潜力

### 组织变革与未来展望 (50:00-56:00)
- **文化变革的重要性**：
  - Andy强调需要CEO支持，将AI作为未来运营模式
  - 与首席临床官和首席人力官建立合作关系
  - 开发"未来工作"框架，系统性地重新设计工作角色
- **2026年预测与目标**：
  - Mike设定到2028年减少50%行政负担的目标
  - Andy预测每个临床护理人员都将拥有增强型AI助手
  - 强调需要像COVID期间一样快速行动和部署技术
- **最终建议**：
  - 优先考虑行动速度和价值实现
  - 启用公民主导的AI开发
  - 专注于业务流程重新设计而非简单自动化
  - 建立灵活的技术架构以适应快速变化
