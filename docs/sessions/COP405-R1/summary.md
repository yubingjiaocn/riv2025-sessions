# AWS re:Invent 2025 观测性智能代理技术分享会总结

## 会议概述

本次技术分享会由AWS的Kevin和Andrea主讲，主题为"使用智能代理工具改善观测性态势"。会议重点介绍了如何利用AI代理技术解决现代分布式系统中观测性数据过载的问题。演讲者展示了一个基于Python和Strandz Agent SDK构建的观测性代理系统，该系统能够自动分析AWS基础设施，生成智能化的观测性报告，并主动创建监控告警。

演讲采用代码演示的形式，通过一个名为"宠物收养诊所"的示例应用，详细展示了如何构建能够查询CloudWatch指标、分析日志、关联行业事件并生成可操作建议的智能代理。该解决方案旨在让工程师在凌晨3点被生产故障唤醒之前，就能通过AI代理提前发现和处理潜在问题。

## 详细时间线与关键要点

### 00:00-05:00 问题背景与解决方案概述
- Kevin开场提问：有多少人曾在凌晨3点被生产故障唤醒查看日志
- 观测性领域的演变：从缺乏数据到数据过载的转变
- AI代理在观测性场景中的应用价值
- 代理系统的核心组件：LLM处理、上下文提供、自然语言响应

### 05:00-10:00 MCP工具与系统架构
- 模型上下文协议(MCP)的重要性和安全边界作用
- 宠物收养网站演示应用介绍
- 分布式系统观测性数据爆炸式增长的挑战
- 构建代理的核心策略：提供丰富上下文、定义SLO、趋势分析

### 10:00-15:00 代理系统设计理念
- 自然语言查询到HTML报告的完整流程
- Strandz Agent SDK的简洁性（10行代码即可构建基础代理）
- 系统组件：LLM、上下文提供、MCP访问、可用输出
- Andrea接手演示，强调代码的开源性质和预生产环境测试的重要性

### 15:00-25:00 观测性报告深度分析
- 执行摘要：EKS宠物网站系统、Lambda函数、安全扫描发现
- 关键发现：系统缺乏告警，代理自动创建4个关键告警
- 指标分析：错误率、故障率、延迟分析（400毫秒平均延迟）
- 运营审计：EKS应用性能问题、Lambda功能分析、合成监控

### 25:00-35:00 安全分析与行业关联

- 三层行动建议：立即行动、短期改进、长期优化
- CloudWatch日志分析：1.36GB应用日志的成本优化建议

### 35:00-45:00 代码实现详解
- Python代理主文件结构和参数配置

- 代理创建的两步过程：上下文设置和交互执行
- 系统提示的重要性：观测性专家角色定义和能力描述

### 45:00-52:00 提示工程与工具创建
- 详细的分析步骤定义：资源发现、健康分析、深度链接生成
- 告警管理策略：保守创建原则、避免重复告警

### 52:00-57:30 部署策略与问答环节
- 自动化部署选项：基于事件触发或定期执行
- 预防性评估：在人工告警前触发代理分析
- 观众问答：令牌使用效率、规模化执行、模型兼容性、结构化输出处理
