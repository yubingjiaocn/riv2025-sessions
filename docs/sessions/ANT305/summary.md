# AWS re:Invent 2025 数据分析创新会议总结

## 会议概述

本次会议由AWS分析产品组合负责人Kinshuk Pahare主持，与会者包括Neil Mukerje和Netflix工程经理Anjali。会议重点讨论了数据处理领域的最新创新，涵盖AI代理、Apache Iceberg、易用性改进以及治理安全等四大主题。

会议展示了AWS数据处理服务的强大规模：每周执行超过3亿次Redshift Serverless查询、10亿次Athena查询和3亿个Spark作业，处理超过100PB的S3数据。这些服务基于S3数据湖构建，体现了客户对AWS数据处理服务的信任。Netflix作为重要客户分享了他们评估和采用EMR的实际经验，展示了从自建平台向AWS托管服务迁移的成功案例。

## 详细时间线与关键要点

### 0:00-5:00 会议开场与服务概览
- 介绍演讲嘉宾和议程安排
- AWS数据处理服务组合：Glue ETL、EMR、Athena、Redshift
- 服务使用规模统计：每周3亿+查询，处理100PB+数据
- S3作为数据湖的核心地位，存储EB级Parquet数据

### 5:00-10:00 性能优化与创新主题
- AWS Spark性能比开源版本快4.5倍，写入快2倍
- Spark 4.0版本性能提升5.4倍
- 四大创新主题：Iceberg标准化、AI代理、易用性、治理安全

### 10:00-20:00 AI代理在Spark升级中的应用
- Spark版本升级的挑战：代码升级和数据一致性问题
- AI代理工作流程：构建升级计划→执行和调试→数据质量检查
- 代理功能演示：自动识别项目结构，生成升级计划，处理错误信息
- FINRA使用案例：将原本需要6-12个月的升级任务缩短至分钟级别

### 20:00-25:00 Iceberg V3支持与新特性
- 基于Iceberg 1.10版本，支持删除向量和行血缘功能
- EMR Spark 7.12运行时共享给Glue ETL和Athena
- EMR Spark 4.0发布，性能提升5.4倍
- Iceberg物化视图：自动加速查询，支持增量刷新

### 25:00-35:00 易用性改进（Neil主讲）
- SageMaker Notebooks：新的现代化笔记本体验，内置AI代理
- 支持Python、SQL、Spark多语言，秒级启动大规模Spark集群
- Spark Connect架构：客户端与远程驱动分离，实现无缝远程执行
- AI代理功能：理解数据目录，生成SQL/Python/Spark代码，创建完整分析计划

### 35:00-40:00 托管服务与安全治理
- 完全无服务器的托管Airflow，按使用付费
- 工作流级别的IAM角色配置，增强安全模型
- EMR Serverless无服务器存储：消除本地磁盘配置，节省20%成本
- 多层级访问控制：S3访问授权、表级权限、细粒度行列级安全

### 40:00-43:30 Netflix案例分享（Anjali主讲）
- Netflix大数据平台规模：11,000个工作流，250,000个作业，EB级数据
- EMR评估动机：安全隔离需求、运维负担、新技术采用挑战
- POC测试结果：PySpark性能提升最显著，资源消耗大幅降低
- 迁移计划：2025年上半年技术准备，下半年平台迁移，2027年用户工作流迁移
