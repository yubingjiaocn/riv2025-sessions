# AWS Lambda Managed Instances 技术会议总结

## 会议概述

本次AWS re:Invent 2025技术会议由AWS首席解决方案架构师Stephen Leedig和AWS Lambda团队首席工程师Archana Shikantha主讲，重点介绍了本周刚刚发布的全新功能——Lambda Managed Instances (LMI)。该功能允许用户在保持熟悉的Lambda编程模型的同时，将函数运行在自己账户中的EC2实例上，为高流量、稳定状态的工作负载提供更多控制权和成本优化选项。

Lambda Managed Instances并非Lambda默认服务的替代品，而是针对特定用例的补充解决方案。它特别适用于具有平滑、可预测流量模式的高流量工作负载，以及需要特定计算、内存或网络吞吐量要求的应用程序。该功能消除了冷启动问题，支持多并发调用，并允许用户应用EC2定价机制如预留实例和节省计划。

## 详细时间线与关键要点

### 00:00-05:00 开场介绍与背景
- Stephen Leedig介绍AWS在无服务器领域的持续创新历程
- 展示Lambda从早期Python、Node.js、Java运行时到现在支持几乎所有运行时的发展
- 介绍VPC集成、网络优化、层和并发等功能的演进
- 强调Step Functions和EventBridge等服务在构建事件驱动架构中的作用

### 05:00-10:00 客户用例与需求分析
- 金融机构利用Lambda迁移核心银行应用，降低运营成本
- 医疗保健提供商使用Lambda自动化预约预订和保险理赔处理
- 零售公司采用事件驱动架构应对购物高峰
- 政府机构案例：澳大利亚统计局完全基于无服务器构建人口普查系统
- 客户反馈：需要更多控制权、专用计算选项和成本优化能力

### 10:00-15:00 Lambda Managed Instances核心概念
- LMI允许在用户账户中的EC2实例上运行Lambda函数
- 支持400多种实例类型，包括通用、计算优化和内存优化实例族
- AWS负责实例生命周期、操作系统和运行时补丁管理
- 支持EC2定价构造如节省计划、预留实例等
- 适用场景：高流量稳定工作负载vs传统Lambda适用的不可预测流量场景

### 15:00-25:00 技术实现深度解析
- Archana详细介绍LMI的三步实现流程：创建容量提供商、创建函数、发布版本
- 容量提供商配置：VPC配置、实例类型选择、扩展设置
- 支持C、M、R实例族的最新一代，从large规格开始
- 支持Intel、AMD x86架构和ARM Graviton实例
- 网络配置：实例获得VPC IP地址，所有出站流量通过实例网络接口

### 25:00-35:00 函数配置与特性支持
- 函数创建过程与传统Lambda几乎相同，仅需关联容量提供商
- 支持特性：OCI容器和ZIP格式、最新版本的Java/Python/Node/.NET运行时
- 支持层、扩展、函数URL、响应流、15分钟超时和持久函数
- 内存配置范围扩展至32GB，新增执行环境内存与vCPU比率设置
- 比率选项：2:1（计算优化）、4:1（通用）、8:1（内存优化）

### 35:00-45:00 并发与扩展机制
- 多并发支持：单个执行环境可同时处理多个调用
- 默认并发限制：Java(10)、Python(100)、Node.js(1000)、.NET(25)
- 无冷启动：执行环境在函数激活时预初始化
- 基于资源的异步扩展：监控CPU利用率进行扩展决策
- LMI代理提供负载均衡和资源压力保护

### 45:00-52:00 合作伙伴集成与定价
- DataDog提供完整可观测性支持，包括指标监控和自动追踪
- Spot.io通过自主优化平台支持性能和成本优化
- AWS AppConfig、CloudWatch Lambda Insights完全支持
- PowerTools for AWS提供线程安全的实用工具套件
- 基础设施即代码支持：CloudFormation、SAM、CDK、Terraform
- 定价模式：EC2实例成本+15%管理费+每次调用费用（无持续时间费用）
