# AWS re:Invent 2025 基因组学会议总结

## 会议概述

本次会议是一场20分钟的闪电演讲，重点介绍了AWS与阿斯利康(AstraZeneca)合作，在AWS最新一代FPGA实例(F2)上运行基因组学流水线的工作成果。演讲者包括AWS高性能计算生命科学专家解决方案架构师Marissa Powers和阿斯利康基因组学研究中心的Sean Odell。

该项目展示了令人瞩目的性能提升和成本优化成果。通过将基因组学处理流水线从上一代F1实例迁移到最新的F2实例,实现了62%的速度提升和71%的成本降低。这些改进对于阿斯利康每年需要处理数万个样本的大规模基因组学分析工作流至关重要。项目使用了Illumina DRAGEN加速工具、AWS Batch容器化作业编排服务以及Seqera Nextflow工作流管理器,构建了一个可扩展、可重现的基因组学数据处理架构。

会议还强调了等效性测试的重要性,确保在新硬件平台上运行的结果与原平台完全一致,这对于药物发现和临床试验等应用场景至关重要。演讲通过实际演示展示了如何使用AWS Batch动态配置资源,实现从零扩展到数百万vCPU再回到零的弹性计算能力。

00:00 - 00:45 - 会议开场与介绍
- 欢迎参会者,介绍这是一场20分钟的闪电演讲
- 主题:AWS与阿斯利康合作在最新一代FPGA实例上运行基因组学流水线

00:45 - 01:30 - 演讲者介绍
- Marissa Powers自我介绍:AWS高性能计算生命科学专家解决方案架构师,专注于大型制药客户的药物发现研发工作
- 介绍合作伙伴Sean Odell,来自阿斯利康基因组学研究中心
- 介绍项目团队成员,包括AWS的Manu Palai、Illumina的团队以及阿斯利康的Gabriel Hernandez

01:30 - 02:00 - 会议议程概览
- 将讨论三个主要问题:为什么(Why)、发现了什么(What)、如何构建(How)
- 承诺会议结束时进行演示

02:00 - 03:30 - 阿斯利康的基因组学处理需求(Sean Odell)
- 介绍阿斯利康基因组学处理的三大类别
- 第一类:开发跨不同模态的标准化工作流和流水线(如外显子组和全基因组测序使用不同工作流)
- 强调结果可重现性的重要性,特别是对药物发现和临床试验的影响

03:30 - 04:30 - 数据管理挑战
- 处理海量数据:PB级数据量,数千万个文件
- 需要以成本高效的方式存储和检索数据
- 需要数据目录来跟踪所有文件
- 需要将提取的洞察提供给科学家进行下游分析

04:30 - 05:00 - 工作负载特征
- 工作负载具有明显的峰值特征,非稳态处理
- 每年接收大批量样本
- 需要快速处理以最大化投资回报

05:00 - 06:00 - 测试方案设计
- 从每种模态(外显子组和全基因组)选择样本
- 使用已知数据集
- 在F1和F2上运行完全相同的DRAGEN命令行
- 使用完全相同的参考文件

06:00 - 07:00 - 性能测试结果
- 实现高达62%的速度提升
- DRAGEN软件版本、样本和参考文件完全相同
- 性能提升在外显子组和全基因组测序中保持一致
- 可以在一半时间内完成相同工作量

07:00 - 07:30 - 成本优化结果
- 实现71%的成本降低
- 节省的资金可以重新投资到业务的其他领域和科学研究

07:30 - 08:00 - 等效性测试验证
- 使用生物信息学工具确认F1和F2的结果完全匹配
- 变异检测结果完全相同
- DRAGEN生成的所有指标完全相同(除了依赖日期的头部信息)

08:00 - 09:30 - 实例规格对比分析(Marissa Powers)
- F1实例:16个vCPU,2个芯片,每芯片8核
- F2.6xlarge实例:24个vCPU,1个芯片,每芯片16核
- vCPU数量增加50%
- 每芯片核心数增加200%,这是性能提升的最大贡献因素
