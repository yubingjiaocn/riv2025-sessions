# AWS re:Invent 2025 Kiro 会议总结

## 会议概述

本次会议由AWS Next Generation Developer Experience团队的两位专家解决方案架构师Ryan Yanchuleff和Ryan Bachman主讲，主题为"使用Kiro构建应用程序"。Kiro是AWS在2025年发布的新产品，在会议前两周刚刚正式发布。会议采用互动式现场编程演示的形式，演讲者计划根据观众建议现场构建一个完整的应用程序。

会议重点介绍了Kiro的核心理念——规范驱动开发（spec-driven development），这是一种回归软件开发基本原则的方法。与传统的生成式AI工具不同，Kiro不会直接从简单提示生成代码，而是首先创建需求文档、设计文档和项目计划，让开发者有机会审查和修改这些规范，然后再进行实际的代码实现。这种方法解决了传统AI开发工具在规模化、控制性和代码质量方面的三大挑战。

## 详细时间线与关键要点

### 0:00-10:00 会议开场与背景介绍
- 两位Ryan介绍自己的角色和团队职责
- 强调会议的互动性质，鼓励观众参与讨论
- 说明将现场构建观众推荐的应用程序
- 介绍Kiro作为相对较新的产品，刚刚正式发布

### 10:00-20:00 生成式AI开发工具的演进历程
- 回顾生成式AI工具的发展阶段：从简单的自动补全到智能助手，再到当前的代理开发
- 解释"代理开发"（agentic development）的概念和承诺
- 分析传统AI开发工具面临的三大挑战：
  - 规模化问题：如何处理大型代码库和大型开发团队
  - 控制问题：缺乏真正的协作，开发者无法控制AI的决策过程
  - 质量问题：生成的代码往往不符合预期

### 20:00-30:00 规范驱动开发理念
- 介绍规范驱动开发作为Kiro的核心特色
- 对比传统软件开发流程：需求分析→设计文档→项目计划→实施
- 解释Kiro如何将这一流程应用到AI开发中
- 强调通过预先生成和审查规范文档来确保一致性和质量

### 30:00-40:00 现场演示准备与应用构思
- 观众提出应用想法：健身应用、AWS事件管理应用、酒店虚拟礼宾应用
- 最终确定构建一个健身活动管理Web应用
- 演示如何从简单的一句话提示开始
- 展示Kiro的两种模式：Vibe模式（传统方式）和Spec模式（规范驱动）

### 40:00-50:00 需求文档生成与分析
- 实时演示需求文档的自动生成过程
- 展示生成的需求文档包含的内容：
  - 高级介绍和术语表
  - 七个详细的用户故事和验收标准
  - 遵循EARS（Easy Accessible Requirements System）行业标准
- 强调开发者可以审查和修改这些需求

### 50:00-60:00 设计文档与技术栈选择
- 展示设计文档的生成过程
- 介绍自动选择的技术栈：React + TypeScript + Tailwind CSS
- 解释基于属性的测试（property-based testing）概念
- 展示架构设计的详细分层结构

### 60:00-65:00 问答环节与会议总结
- 回答观众关于MCP服务器、代理引导、技术限制等问题
- 解释Kiro的200,000 token上下文窗口限制
- 讨论与Jira等第三方工具的集成方案
- 介绍代理钩子（Agent Hooks）功能用于自动化工作流
- 会议因时间限制未能完成完整的应用构建演示
