# AWS re:Invent 2025 AI安全会议总结

## 会议概述

本次会议由Trend Micro的资深专家主讲，这是他第12次参加re:Invent大会。会议主要探讨了AI如何成为创新驱动力的同时，也成为了攻击者的新战场。演讲者强调，随着AI基础设施和云原生服务的发展，组织需要制定全新的安全策略来保护这些环境。

会议深入分析了AI安全的多层防护理念，不仅仅是模型安全，而是涵盖数据、模型、AI管道、运行时环境等完整生命周期的安全防护。演讲者通过实际案例和威胁研究，展示了攻击者如何利用AI应用的各个环节进行攻击，并提出了相应的防护策略和最佳实践。

## 详细时间线与关键要点

### 0:00-5:00 开场与背景介绍
- 演讲者分享这是他第12次参加re:Invent，见证了网络安全在AWS生态中的演进
- 强调AI正在成为创新驱动力，同时也扩大了攻击面
- 提出现代安全需要在拥抱创新的同时保持对威胁的领先优势

### 5:00-10:00 AI威胁模型分析
- 介绍AI攻击的"飞轮模型"：数据、模型、AI管道、运行时
- 强调用户是关键因素，包括企业内部用户和外部应用用户
- 讨论数据投毒攻击：通过向训练集注入恶意数据来操纵模型行为
- 模型盗窃：攻击者可以窃取模型并在其他基础设施中重新部署

### 10:00-15:00 AI管道安全挑战
- AI管道中的多个攻击点：模型构建时的投毒、供应链后门注入
- 影子AI问题：组织缺乏对所有AI技术使用的完整可见性
- 分享Reddit案例：某组织因仅依赖模型安全而忽略运行时防护栏导致项目失败
- 强调传统防御过于被动，AI威胁演进速度过快

### 15:00-20:00 数据层安全防护
- 大多数AI风险源于数据层
- 数据可见性问题：大部分数据处于"水面下"，未分类或未受保护
- 数据泄露可能发生在训练、推理日志、API通信等各个环节
- 提出数据安全态势管理的重要性

### 20:00-25:00 供应链与基础设施安全
- AI组件依赖大量开源库和外部服务
- 三大攻击向量：依赖投毒、未验证镜像、管道妥协
- 强调容器镜像扫描的重要性，不仅查找漏洞还要检查泄露的密钥
- 介绍CI/CD管道验证和代码完整性检查

### 25:00-30:00 模型与AI代理安全
- 模型和代理本身可能成为攻击向量
- AI代理的自主行为引入全新风险类别
- 分享与大型软件公司的对话：缺乏控制代理范围的方法
- 介绍新推出的AI Scanner工具，可扫描敏感数据泄露、提示注入等风险

### 30:00-35:00 基础设施与网络安全
- AI基础设施面临资源耗尽、GPU滥用、数据泄露等威胁
- 强调从零开始构建安全的重要性
- 网络层防护：API拦截、横向移动、数据渗透
- 披露40%的API端点缺乏身份验证，存在严重安全隐患

### 35:00-41:00 用户因素与总结
- 人为因素仍是最大安全挑战
- 深度伪造攻击和钓鱼攻击日益增加
- 强调AI安全需要统一的安全视角，从数据到用户的全链条防护
- 介绍基于威胁研究"Silent Sabotage"的案例分析
- 推广AI安全蓝图和Trend Vision One Vault挑战赛
