# AWS re:Invent 2025 迁移会议总结

## 会议概述

本次会议由AWS高级解决方案架构师Sassan Hajra Sulikha及其团队成员Mangesh Blay和Adi Simon主讲,重点介绍了如何利用数据驱动的方法将Microsoft工作负载迁移到AWS云平台。会议强调了在迁移旅程中需要回答的关键问题,包括迁移目标、原因、时间、预算和所需资源等。演讲者展示了AWS如何通过创新的AI驱动工具和服务,特别是AWS Transform服务,简化和加速大规模迁移项目。

会议涵盖了完整的迁移生命周期,从发现和评估阶段开始,深入探讨了不同类型工作负载的迁移策略,包括服务器和虚拟机、Active Directory、SQL Server数据库以及.NET应用程序。演讲者通过多个现场演示展示了如何使用AWS的各种工具和服务来自动化迁移流程,减少人工干预,并利用AI代理(Agentic AI)来处理复杂的迁移任务。整个会议强调了"人在回路"(human-in-the-loop)的概念,确保在自动化和人工监督之间取得平衡。

会议特别关注了2025年AI时代的迁移方法,展示了如何利用生成式AI和代理AI来革新传统的迁移流程,使迁移项目更快速、更高效、更少出错。

- **01:03:00 - 01:05:00**: 介绍三种AWS上的Active Directory设计选项:自管理EC2域控制器、AWS Managed Microsoft AD和AD Connector

- **01:09:00 - 01:10:30**: 解释AD Connector作为网关的角色和一对一关系限制

- **01:12:00 - 01:14:00**: 介绍Hybrid Edition的关键优势:可扩展现有域到托管服务,同时获得托管体验
