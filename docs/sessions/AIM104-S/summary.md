# AWS re:Invent 2025 技术会议总结

## 会议概述

本次技术会议由Pegasystems公司的Launchpad负责人John Huehn和高级产品工程副总裁Steve Bixby主讲，主题为"构建企业级智能体SaaS应用程序"。会议演示了如何使用Pega Launchpad平台在短短五个步骤内构建一个能够处理复杂监管流程的企业级智能体SaaS应用，并将其销售给数千个B2B客户。

演讲者以一个名为"Homeland"的虚构金融科技公司为例，展示了如何为银行构建一个智能体应用程序，将抵押贷款预批准处理时间从数天缩短到数秒。该应用程序使用工作流来管理AI智能体，确保在银行等受监管行业中提供可控、确定性和可预测的结果。整个演示突出了Launchpad作为AI驱动的低代码应用开发平台的能力，包括其在AWS上的安全可扩展云架构。

## 详细时间线与关键要点

### 0:00-3:00 会议开场与场景介绍
- John Huehn和Steve Bixby介绍会议主题和目标
- 提出构建企业级智能体SaaS应用的五个步骤
- 介绍"Homeland"金融科技公司案例和Sarah的抵押贷款预批准需求场景
- 强调银行业对AI自动化的需求以及对决策可变性的担忧

### 3:00-6:00 Launchpad平台介绍与应用创建
- Steve开始使用Launchpad平台进行实时演示
- 输入应用名称"Homeland"并启动应用构建过程
- 系统自动识别行业、位置等信息并开始构建
- 展示平台如何使用生成式AI创建行业模板和自定义工作流

### 6:00-9:00 工作流设计与数据模型
- 展示系统生成的八个工作流，包括抵押贷款预批准流程
- 详细查看抵押贷款预批准工作流的各个阶段：收集申请数据、验证信息、财务评估、预批准决策、最终确定
- 展示自动生成的数据模型，包括申请人姓名、出生日期、W2表格、年收入等字段
- 强调所有内容都是使用AWS Bedrock模型的生成式AI创建

### 9:00-12:00 工作流定制与规则配置
- John建议为超过500万美元的贷款申请添加人工审批
- Steve演示如何通过自然语言输入直接修改工作流
- 系统自动插入人工审批步骤和相应的决策规则
- 展示如何添加AWS Nova智能体来自动化就业验证流程

### 12:00-15:00 应用预览与测试
- 展示应用的对话界面预览功能
- Steve作为测试用户与虚拟助手互动，演示抵押贷款预批准流程
- 展示同一逻辑如何在不同渠道部署：对话界面、员工桌面应用、网站自助服务
- 演示应用的后台管理界面和工作流实例创建

### 15:00-18:00 应用部署与客户使用演示
- 展示如何将应用部署给订阅者（银行客户）
- 演示提供商仪表板，显示订阅者使用情况和收入数据
- 切换到U Plus Bank网站，展示真实客户Sarah使用部署的应用
- 完整演示从申请到批准的端到端流程，包括文档上传和即时批准结果
