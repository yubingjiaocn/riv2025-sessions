# AWS re:Invent 2025 技术会议总结

## 会议概述

本次会议由Mary Technology的CEO兼联合创始人Dan主讲，重点介绍了法律科技领域中大型语言模型在文档审查方面的挑战及解决方案。Mary Technology是一家总部位于悉尼的法律科技公司，现已拓展至全球市场，专门帮助律师事务所自动化文档审查流程。

Dan详细阐述了大型语言模型在法律争议解决工作负载中面临的四个核心问题：训练数据可用性问题、压缩机制导致的信息丢失、事实提取的复杂性，以及法律专业人员对AI输出结果信心不足的问题。会议展示了Mary Technology如何通过构建专门的事实处理管道来解决这些挑战，实现了75-85%的文档审查时间减少。

## 详细时间线与关键要点

### 0:00-2:00 开场介绍与问题概述
- Dan介绍Mary Technology背景：悉尼法律科技公司，专注文档审查自动化
- 询问现场法律运营负责人和律师事务所从业者人数
- 提出核心论点：大型语言模型不适用于法律争议解决工作负载

### 2:00-5:00 第一个问题：训练数据挑战
- 法律争议数据高度敏感，无法公开获取用于训练
- 法律案件没有单一正确答案，至少存在两方观点
- 需要理解所有潜在叙述和正确答案，取决于代表哪一方

### 5:00-8:00 第二个问题：压缩机制问题
- 大型语言模型本质上是压缩机器
- 文档处理流程：页面→图像→文本→标记→嵌入→上下文压缩→分块和摘要
- 每层压缩都会丢失法律细节和重要含义，如手写笔记等关键信息
- 承认LLM在通用能力方面表现优秀

### 8:00-12:00 第三个问题：事实提取复杂性
- 以"A. Smith报告错误"为例说明挑战
- 多个同名人员识别问题（Alice vs Andrew Smith）
- 日期格式歧义（澳大利亚vs美国格式）
- 事实相关性判断和文档来源重要性评估
- 医疗文档中"PT"（患者）缩写识别问题

### 12:00-15:00 第四个问题：法律专业信心问题
- 展示匿名信件分析案例：无署名、无收件人、间接描述犯罪
- LLM无法建立"Daniel偷车"的关联
- 需要通过笔迹分析、时间线对比等方法建立证据链
- 法律专业人员不会直接使用AI生成的完美法律文档

### 15:00-17:00 解决方案展示
- 播放Mary Technology平台演示视频
- 展示从文档混乱到结构化处理的完整流程
- 关键功能：文档分割、分类、实体提取、时间线构建、案例摘要

### 17:00-19:00 技术方法与成果展示
- 构建事实制造处理管道，将事实作为一等公民对待
- 每个事实对象包含丰富元数据，支持关系构建和矛盾检测
- 所有决策过程可解释，提供推理依据
- 客户成果：与澳大利亚最大律师事务所合作，文档审查时间减少75-85%
- 客户满意度：96/100 NPS评分
- 开放问答环节
