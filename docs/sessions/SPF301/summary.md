# AWS re:Invent 2025 会议总结：Catapult Sports 利用 AWS IoT 改变体育数据分析

## 会议概述

本次会议由来自澳大利亚的体育解决方案架构师 Mike 和 IoT 专家解决方案架构师 Greg 主讲，重点介绍了 Catapult Sports 如何利用 AWS IoT 服务构建下一代运动员监测系统 Vector 8。

Catapult Sports 是一家在澳大利亚证券交易所上市的体育科技公司，在全球 24 个地点拥有超过 400 名员工，服务于 128 个国家的 5000 多支精英运动队伍，涵盖 40 多种体育项目。公司的核心使命是通过数据驱动的方式优化运动员表现、改善恢复能力并量化重返赛场的准备度。

会议深入探讨了 Vector 8 可穿戴设备系统的技术架构，展示了如何通过 AWS IoT Core、AWS IoT Greengrass、Amazon MSK、Amazon ECS 等服务实现设备健康监测、实时数据流传输（10Hz）和赛后详细数据分析（100Hz）。特别值得注意的是，通过 AWS IoT 的空中更新（OTA）功能，Catapult 将设备更新速度提升了 32 倍，97% 的设备始终运行最新固件，客户支持案例大幅减少。系统能够在 30 秒内上传一场 2 小时训练的 30 名运动员数据，一场 NHL 比赛可捕获 3600 万个数据点。

### AWS IoT Core 与数据采集 (9:30-13:00)

9:30 - AWS IoT Core 服务介绍：
- 托管服务，提供可扩展的云端点
- 设备认证与授权
- 传输加密
- 大规模设备管理
- 规则引擎（连接应用与数据的关键）

10:30 - 三种数据采集路径：
1. 设备健康数据：电池状态、固件版本
2. 热数据：10Hz 采样的实时比赛/训练数据（加速度、速度、位置、心率）
3. 冷数据：100Hz 采样的赛后原始惯性数据（用于机器学习推理）

11:15 - 设备健康监测架构：
- Vector 8 Tag → UWB 传输 → Catapult Receiver
- Receiver 通过 MQTT 发布到 AWS IoT Core
- IoT 规则引擎路由到 Amazon MSK
- MSK 流式传输到 Amazon ECS
- 通过 API Gateway 展示到 Web 界面

12:00 - 热数据实时架构：
- 10Hz 更新频率（每秒 10 次）
- 标签通过 UWB 传输到接收器
- 实时流式传输到移动设备和 Web 应用
- 教练可在 iOS 应用上查看即时可视化数据

### 设备软件架构 (26:00-30:00)

26:00 - Greg 接手讲解技术细节

26:30 - AWS IoT Greengrass 介绍：
- Catapult 使用 Greengrass 作为 Vector 8 Dock 和 Receiver 的基础
- 边缘运行时 + 云服务
- 帮助构建、部署和管理大规模多进程 IoT 应用
- 开源边缘运行时（GitHub 可用）

27:30 - Greengrass 组件模型：
- 可执行文件、二进制文件、脚本、容器镜像
- 支持组件版本控制和依赖解析
- 日志记录用于诊断
- 大量预构建 AWS 组件
- 进程间通信（IPC）

28:30 - 软件架构高层视图：
- 底层：嵌入式 Linux 操作系统
- 中层：Greengrass 边缘运行时
- 顶层：Catapult 自定义组件 + AWS 提供的组件

### Receiver 与 Dock 架构详解 (30:00-35:00)

30:00 - Receiver 软件架构：
- 标签通过 UWB 无线通信到接收器
- 数据处理器自定义组件接收数据
- 通过 IPC 发布设备健康消息到 AWS IoT Core
- Greengrass 消息缓冲器处理连接中断

31:30 - Receiver 的其他组件：
- 数据库、RESTful API、Web 服务器
- 向场边 iOS 应用提供 10Hz 实时数据

32:30 - Dock 架构：
- 标签放回 Dock 进行充电和数据传输
- 有线连接（非无线）
- Retriever 组件检索冷数据（100Hz 赛后数据）
- Raw File Upload 组件将数据文件发布到 S3
- 整个过程通常只需数十秒

### 设备入网 (35:00-38:00)

35:00 - Catapult 的设备入网目标：从开箱到运动员追踪少于 10 分钟的自助服务体验

35:30 - Vector 7（上一代产品）的入网流程：
- 需要在笔记本电脑上安装桌面应用
- 预约 Catapult 客户支持
- 配置设备、应用固件更新、测试
- 通常需要 2 天或至少数小时
- 客户支持和客户都需要大量人工参与

36:30 - Vector 8 的入网流程：
- 类似消费级电子设备设置
- 开箱 → 在手机上安装 Catapult 应用
- 在应用中注册设备
- 配置 Wi-Fi 凭证
- 设备连接云端，自动拉取配置和固件更新
- 发布状态
- 从开始到完成 10 分钟，无需客户支持参与

37:30 - AWS IoT 带来的价值：设备配置、更新和状态报告的规模化和自动化

### 空中更新（OTA）(38:00-44:00)

38:00 - OTA 更新的重要性：在 IoT 领域，OTA 是"零号特性"，是构建其他一切的基础

38:30 - OTA 更新的优势：
- 设备出货后轻松添加新功能
- 大规模修补设备
- 解决 bug 和客户支持问题
- 防止客户重复报告相同问题
- 支持创新和快速迭代

39:30 - Greengrass 部署功能：
- 开箱即用支持 OTA 更新
- 定义应部署到设备的组件集
- 可针对单个设备或设备组
- 可大规模操作

40:30 - Greengrass 部署的弹性特性：
- 对间歇性连接具有弹性
- 自动处理重试、暂停和恢复
- 下载组件工件（程序二进制文件、可执行文件、容器镜像）

41:00 - 更新的粒度：
- Greengrass 计算当前状态与部署需求之间的差异
- 仅下载所需内容

41:30 - 推出和回滚配置：
- 如果部署包含故障组件
- Greengrass 自动恢复到之前的部署状态
- 保持设备处于良好工作状态

42:00 - 与操作系统更新集成：Catapult 能够使用 Greengrass 部署修补和管理嵌入式 Linux 操作系统

42:30 - 标签（Tag）的更新流程（更复杂）：
- 标签运行 FreeRTOS（微控制器设备）
- UWB 接口是定制协议
- 首先通过 Greengrass 部署将标签固件二进制文件从云端传输到 Dock
- 使用 AWS IoT Jobs 触发更新过程
- Device Manager 组件订阅 MQTT 主题接收作业通知
- 标签返回 Dock 时，Device Manager 将固件分发到每个标签

### OTA 更新的影响 (44:00-47:00)

44:00 - 与 Vector 7 相比速度提升 32 倍（基于 Catapult 自己的数据）

44:30 - 速度提升的原因：
- Vector 7 需要客户参与（笔记本电脑 + 桌面应用）
- Vector 8 是自动化流程（标签放回 Dock 即自动更新）

45:00 - 几乎所有标签始终运行最新固件：
- 标签放回 Dock 后即获得最新固件
- 只有未返回 Dock 的标签（丢失、放错位置）未运行最新固件

45:45 - 客户支持案例大幅减少：
- 根据 Catapult 历史数据，超过 50% 的客户支持问题通过更新到最新固件即可解决
- Vector 8 中这一过程自动发生

46:15 - 支持创新：OTA 更新支撑快速迭代

46:45 - 开发流程改进：
- 理念规划 → 构建新价值 → 部署 → 测试、测量、学习 → 收集反馈 → 循环
- 更快的部署 + 97% 的渗透率 = 更快的反馈和学习

### 开发流程转型 (47:00-49:00)

47:00 - Catapult 固件团队的工作方式变革：
- 从瀑布流程转向敏捷冲刺
- 发布节奏从数月缩短到数周
- 某些情况下甚至是数天（特别是开发新功能和 Beta 测试时）

48:00 - Beta 测试优势：
- 与 Beta 客户合作时能够高速迭代
- 帮助开发客户真正想要的功能
- 对 Catapult 和客户都有利

### 设备配置 (49:00-52:00)

49:00 - 设备配置需求：
- 每个可穿戴设备有大量配置参数
- Catapult 希望从云端配置标签
- 无需边缘应用程序

49:30 - 配置设置包括：
- 个人运动员设置
- 运动专项动作的指标设置
- 不同团队、教练、运动科学家的特定要求
- 射频设置（调整标签与接收器之间的无线通信）

50:00 - AWS IoT Shadows（影子）功能：
- JSON 文档
- AWS IoT 帮助在云端和设备之间同步
- Greengrass 上的 Shadow Manager 组件实现设备端功能
- 开箱即用

51:00 - 配置流程：
- Vector Web 用户修改配置
- Shadow Manager 检测到云端影子文档变化
- 自动同步到本地影子数据库
- Device Manager 组件通过 IPC 检索影子文档
- 标签下次返回 Dock 时自动接收最新配置

51:45 - 价值：解耦架构
- 可在云端进行配置更改
- 即使 Dock 离线、标签使用中也可安全更改
- 标签下次返回 Dock 时自动获取更改
