# AWS re:Invent 2025 Lightning Talk CNS 428 - Human-in-the-Loop for Agentic AI Systems

## 会议概述 (Session Overview)

本次闪电演讲由AWS首席专业解决方案架构师Juraj Mahapatra主讲，重点探讨了在Amazon Bedrock和Agent Core环境下实施人机协作(Human-in-the-Loop, HITL)的重要性。演讲者拥有超过6年的AWS工作经验，专注于初创企业客户服务。

演讲核心围绕从传统生成式AI助手向自主代理系统的演进过程，以及为什么在代理AI系统中实施人机协作至关重要。通过现场调研发现，虽然大多数与会者已在生产环境中使用LLM推理，但尚未有人在生产中部署代理系统，这凸显了本次演讲的实用价值。演讲涵盖了HITL的实施机制、关键要点以及相关资源推荐。

## 详细时间线与关键要点 (Detailed Timeline)

### 0:00-2:00 开场介绍与现状调研
- 演讲者自我介绍：Juraj Mahapatra，AWS首席专业解决方案架构师
- 专注领域：Amazon Bedrock、Agent Core、初创企业客户
- 现场调研结果：无人在生产环境使用代理，但大多数人使用LLM推理

### 2:00-5:00 AI系统演进时间线
- 展示从简单生成式AI助手到自主代理AI系统的演进过程
- 强调代理AI系统不必局限于聊天机器人前端，可异步调用
- 讨论人工监督程度的变化：从高度监督到低度监督的转变
- 强调开发者责任：代理出错时，责任在于开发者而非代理本身

### 5:00-8:00 AWS模型推理架构演进
- 早期选项：SageMaker、Bedrock、Hugging Face模型、EC2/EKS部署
- 第一阶段：简单聊天GPT模式的问答系统
- 第二阶段：引入持久化存储维护会话历史
- 第三阶段：通过向量存储实现上下文增强响应(RAG)

### 8:00-11:00 实时数据处理与代理工具
- 从静态数据向实时数据处理的转变
- 代理和工具的引入：API调用、数据库查询、实时信息获取
- 安全性考虑：入站/出站身份验证、护栏机制
- 可观测性需求：CloudWatch或第三方工具监控

### 11:00-13:00 现代代理架构与复杂性
- MCP服务器集成和A2A(Agent-to-Agent)通信协议
- 单代理架构：计算层、内存、MCP服务器、可观测性
- 生产环境复杂性：多代理系统、跨部门集成、外部工具整合
- 人机协作检查点的必要性

### 13:00-15:00 HITL实施场景与策略
- **高风险决策点**：医生开药处方等关键决策
- **不可逆操作**：资金转账、交易完成等操作
- **合规要求**：某些国家和标准要求人工监督
- **信任建立阶段**：新系统部署初期的渐进式自主化
- **边缘案例处理**：模糊情况下的人工干预

### 15:00-17:00 技术实现与工具
- **MCP协议**：elicitations功能实现人机交互
- **Step Functions**：wait for callback机制
- **Lambda持久化函数**：类似Step Functions的功能
- **LangGraph**：状态管理和工具执行控制
- **A2A协议示例**：旅行代理与天气代理的通信案例

### 17:00-17:30 总结与资源
- 渐进式自主化：从高度人工控制到完全自主的演进
- Agent Core评估功能预览版介绍
- 关键资源推荐：
  - 哈佛技术论文：人机算法半人马系统概念
  - GitHub示例应用：A2A客户端/服务器实现
  - 相关博客和技术会议资源
