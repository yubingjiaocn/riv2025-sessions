# AWS re:Invent 2025 会议总结：缩小发布速度与安全性之间的差距

## 会议概述

本次会议由LaunchDarkly CEO兼联合创始人Edith Harbaugh主持，邀请了Ally Financial的Nate Gray和Qlik的Jeffrey Goldberg作为嘉宾。会议主要探讨了如何在保证软件可靠性的同时提高发布速度，打破传统认为速度与可靠性相互冲突的观念。

演讲者们分享了各自组织在过去十年中如何从季度发布转向持续交付，通过功能标志（feature flags）实现部署与发布的解耦，从而让开发团队获得更多自主权并提高开发者满意度。会议强调了现代软件开发中测试生产环境的重要性，以及如何通过自动化和精细化控制来管理风险。

## 详细时间线与关键要点

### 0:00-10:00 开场介绍与背景
- Edith Harbaugh介绍LaunchDarkly的创立背景和核心理念
- 分享了从2014年创立公司的初衷：解决发布过程中的焦虑和风险
- 介绍LaunchDarkly作为运行时功能控制平面的核心价值
- 强调通过20多个SDK支持各种编程语言和平台

### 10:00-20:00 发布节奏的演变
- Jeffrey分享Qlik从本地产品转向云端的经历
- 从每年5次发布转向持续交付，显著提升开发者满意度
- Nate介绍Ally Financial从月度发布转向每月68次发布的转变
- 强调通过功能标志实现API优先开发和分阶段推出

### 20:00-30:00 开发者体验的改善
- 讨论如何让开发团队重新获得夜晚和周末的休息时间
- 通过在生产环境中测试来提高软件质量
- 分享从股市收盘后5点发布的信心来源
- 强调部署与发布分离带来的灵活性和控制力

### 30:00-40:00 风险管理与合规要求
- 讨论在高度监管行业中的非协商性护栏
- Ally Financial在金融监管环境下的合规实践
- Qlik在FedRAMP合规产品中的职责分离
- 分享过度谨慎反而导致事故的经验教训

### 40:00-50:00 AI集成与未来发展
- 将AI模型视为功能进行管理和分阶段推出
- 讨论数据主权和跨区域推理的重要性
- 分享通过功能标志控制AI能力的实践经验
- 强调上下文感知在提高系统可靠性中的作用

### 50:00-61:00 自动化趋势与最佳实践
- 讨论向左移动安全扫描以防止最后时刻的紧急情况
- 分享独立可发布架构的重要性
- 强调开发者自主权在现代发布管理中的核心地位
- 回答观众关于版本控制、功能标志治理和可靠性的问题
