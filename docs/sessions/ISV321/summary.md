# AWS re:Invent 2025 - Zoom AI Companion 3.0 会议总结

## 会议概述

本次会议由 Zoom 的 AI 基础设施负责人 Shin Wang 和 AWS 解决方案架构师 Dimitri 共同主讲，重点介绍了 Zoom 如何通过 AI Companion 3.0 实现从对话到任务完成的转变。Zoom AI Companion 经历了三个发展阶段：从最初的会议摘要功能（1.0），到可交互式问答（2.0），再到如今能够主动规划、推理并代表用户执行任务的智能代理系统（3.0）。这一演进体现了 AI 从被动响应到主动协作的重大转变。

AI Companion 3.0 的核心能力包括跨会议分析、智能日程安排、自动化工作流程等。演讲者通过实际案例展示了该系统如何在几分钟内完成原本需要数天的协调工作——从 Jira 检索项目瓶颈信息，到自动重新安排日历并协调跨时区团队会议，再到准备会议议程。技术架构方面，Zoom 采用了模块化的多智能体框架，结合 AWS 微服务架构和 Amazon Bedrock，通过联邦 AI 策略（将自研小语言模型与第三方大语言模型结合）实现了高质量、低延迟和成本优化的平衡。

该解决方案构建在 AWS 云服务之上，利用 Amazon Bedrock 提供的 API 访问多种模型，使用 OpenSearch 进行向量存储和上下文检索，通过微服务架构实现全球范围的弹性扩展。Zoom 的创新之处在于其联邦 AI 方法：小语言模型首先理解用户意图并处理输入，然后将优化后的提示传递给大语言模型，从而为用户提供高度定制化的输出结果。

- 展示全新的 AI Companion 3.0 Web 界面（Zoomtopia 2025 发布）
- 演示聊天历史查看和对话延续功能
- 展示会后跟进功能：自动生成个性化摘要、关键决策、行动项、风险关注点和后续步骤
- 演示跨会议分析功能：可选择多个会议进行综合分析
- 展示针对特定项目系列会议的深度分析，包括跨主题线索追踪、长期监控和高优先级事项识别

03:30 - 05:15 | AI Companion 演进历程
- AI Companion 1.0：会议摘要，捕捉关键要点
- AI Companion 2.0：引入交互功能，用户可提问并获得更多信息
- AI Companion 3.0：智能代理 AI，能够代表用户进行规划、推理和执行

05:15 - 07:00 | 实际应用案例
- Shin Wang 分享个人使用经验：项目时间线面临风险的处理过程
- 智能检索：AI Companion 连接 Jira 检索相关工单并总结瓶颈
- 智能调度：自动查找可用时间段，根据复杂指令重新安排日历，在当天内安排紧急会议
- 会议准备：基于 Jira 摘要和历史会议记录自动生成聚焦议程
- 结果：原本需要数天的协调工作在几分钟内完成

07:00 - 09:00 | 技术架构核心原则
- 构建在原生多智能体框架之上，处理内存、上下文、提示历史等
- 三大核心原则：
  1. 模块化：可轻松扩展新能力，即插即用
  2. 可定制化：可定义自定义模板和工作流，适应不同垂直领域
  3. 租户隔离：提供增强的安全性

09:00 - 10:30 | 高级技术特性
- 先进的上下文工程：结合会话信息、长期记忆和个人偏好
- 联邦 AI 策略：Zoom 自研小语言模型与第三方前沿大语言模型配合使用
- 目标：为客户提供最高质量服务，同时以可扩展的方式控制成本

10:30 - 13:00 | AWS 技术架构深度解析（Dimitri）
- 可扩展性优先：采用模块化微服务架构，支持不同区域和时段的弹性扩展
- Amazon Bedrock 集成：通过 API 访问多种模型，无需担心可扩展性
- OpenSearch 应用：用于索引和上下文管理，通过向量存储服务对话并快速检索信息
- 联邦 AI 工作流程：小语言模型首先理解用户隐含意图（如输出格式需求），处理后传递给大语言模型，实现高度定制化输出

13:00 - 14:30 | 架构总结
- 专门构建的智能体框架，每个代理根据特定用户需求做出独立决策
- 基于 AWS 微服务架构实现按需扩展
- 通过 Amazon Bedrock 访问不同模型，为特定任务选择最佳工具
- 未来展望：AI Companion 3.0 只是起点，正在探索 Agent Core 等新架构
