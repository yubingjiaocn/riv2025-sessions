# AWS re:Invent 2025 技术会议总结

## 会议概述

本次会议由AWS金融服务首席解决方案架构师Richard Chester和伦敦证券交易所集团(LSEG)云工程负责人Dean Brighton共同主讲。会议重点介绍了LSEG在多云环境下的战略实践,以及如何利用AWS服务解决金融行业的复杂技术挑战。

LSEG作为一家拥有超过200年历史的全球金融基础设施提供商,目前运营着超过1300个AWS账户,同时在Azure和GCP上也有显著部署,并保持着相当规模的本地基础设施。作为一个多云组织,LSEG面临着如何在保持监管合规的同时,充分利用各云服务商优势的挑战。会议通过两个实际用例展示了LSEG如何通过结构化的多云框架和AWS Outposts超低延迟解决方案,实现了性能提升、成本优化和监管合规的多重目标。

会议特别强调了在受监管行业中实施多云战略的实践经验,包括如何建立统一的云能力框架、标准化跨云服务商的操作,以及如何利用AWS Outposts满足极端性能要求和监管弹性需求。这些实践使LSEG的开发效率提升了80%,同时增强了安全合规性并降低了运营成本。

### 用例一:FX价格流系统升级 (13:30-20:00)
- **13:30** - Richard介绍FX价格流系统:全球交易商对客户的流式报价服务
- **14:00** - 服务超过150个货币对,连接100多个流动性提供商和1200多个客户
- **14:30** - 挑战一:现代化需求,需要定期升级基础设施,探索内核旁路等新技术
- **15:00** - 挑战二:延迟和性能,包括宏观延迟(地理位置接近流动性提供商)和微观延迟(前后端流量快速流转)
- **15:30** - 性能改进可以降低交易拒绝率、提高交易量、缩小利润率、增加市场份额
- **16:00** - 挑战三:基础设施和成本,市场碎片化增加竞争,高昂的前期资本支出
- **16:30** - 解决方案:使用AWS超低延迟Outposts
- **17:00** - 超低延迟Outposts特性:优化的裸机计算、确定性性能、物理隔离网络、超低延迟网卡
- **17:30** - 提供等距电缆和二层组播(区域内不可用),确保公平公正访问
- **18:00** - AWS Nitro技术通过专用硬件提供卓越性能,加密连接和防篡改检测保证安全
- **18:30** - 性能结果:49百分位延迟1毫秒(本地2毫秒),2倍改进
- **19:00** - 中位数延迟181微秒,峰值速率165,000条消息/秒(本地1.1毫秒,峰值3.3倍)
- **19:30** - 扇出性能:在双倍峰值的持续负载下,速度快2倍

### 用例二:LCH清算所弹性架构 (20:00-27:00)
- **20:00** - Dean介绍伦敦清算所(LCH):LSEG多数持股的全球金融市场基础设施组织
- **20:30** - LCH作为中央对手方清算和结算各类金融交易(掉期、股票、回购等)
- **21:00** - LCH在买卖双方之间充当中介,降低风险,确保即使一方违约交易也能完成
- **21:30** - 清算工作负载在AWS区域运行,大多从本地迁移而来
- **22:00** - 需求:可扩展性、弹性、创新、快速实验、缩短上市时间
- **22:30** - 最重要因素:弹性 - LCH工作负载被视为英国关键国家基础设施
- **23:00** - 2023年3月英格兰银行发布新指令:必须能够应对云服务商完全中断
- **23:30** - 要求能够完全独立于云服务商运营
- **24:00** - 解决方案:主区域运行Amazon EKS、Oracle RDS、S3存储、Amazon FSX for NetApp ONTAP
- **24:30** - 第一层弹性:复制到次要区域(标准多区域架构)
- **25:00** - 第二层弹性:在AWS超低延迟Outposts上构建第三套环境
- **25:30** - Outposts上运行EKS、EC2上的Oracle、本地NetApp设备
- **26:00** - 最终弹性:数据复制到Azure
- **26:30** - 已成功测试5天断连场景,向英格兰银行证明满足要求,包括数据库迁移
