# AWS re:Invent 2025 后量子密码学会议总结

## 一、会议概述

本次AWS re:Invent 2025分组会议由AWS密码学总监Ken Beer主讲,并邀请了Vanguard公司的Rajie Sharma分享实践经验。会议主题聚焦于后量子密码学(Post-Quantum Cryptography, PQC)的迁移策略和加密敏捷性(Crypto Agility)。

Ken Beer在会议开场介绍了自己在AWS工作超过13年的经历,从2012年帮助推出密钥管理服务(KMS)开始,目前负责从开源密码学库、算法实现到增值服务(如KMS、Cloud HSM、AWS Payments Cryptography)的全栈密码学业务。会议的核心信息是:量子计算机虽然尚未出现,但"现在收集、未来解密"(Harvest Now, Decrypt Later)的威胁模型要求企业立即开始准备后量子密码学迁移。AWS正在通过其服务和开源工具帮助客户完成这一"无差异化的繁重工作",让客户能够专注于业务创新而非底层密码学实现。

Vanguard作为管理11.9万亿美元资产的全球资产管理公司,分享了他们如何将量子风险纳入企业风险管理框架,并通过四大支柱(战略响应、风险嵌入、可见性工具、IT迁移)来规划其量子安全之旅。这为其他企业提供了宝贵的早期采用者经验。

## 二、详细时间线与关键要点

### **00:00 - 开场介绍**
- Ken Beer自我介绍,AWS密码学总监,13年AWS工作经验
- 2012年在re:Invent推出KMS服务
- 目前负责从开源密码库到增值服务的完整密码学栈

### **02:30 - AWS密码学服务概览**
- **KMS(密钥管理服务)**:AWS最大的"锁和钥匙"服务,平均每小时处理300亿次加密/解密操作,高峰期可增长30%以上
- 12年前发明了专为云设计的硬件安全模块(HSM)
- KMS改变了密钥管理范式:客户无需关心密钥物理位置,只需管理访问控制

### **05:45 - 其他密码学服务**
- **Cloud HSM**:为需要使用传统HSM协议的银行、政府、制造业客户提供过渡方案
- **Certificate Manager (ACM)**:简化PKI管理,2025年推出可导出证书功能
- **Private Certificate Authority (PCA)**:管理私有信任存储的内部证书
- **Secrets Manager**:管理数据库凭证等秘密信息,所有秘密都通过KMS加密

### **08:20 - 2025年服务更新**
- ACM推出可导出公共证书
- Secrets Manager深化与第三方系统集成(如Cyber Arc、Salesforce)
- PCA增强Kubernetes集群证书管理能力

### **10:15 - 后量子密码学威胁模型**
- **量子计算威胁**:虽然密码学相关量子计算机尚未出现,但Shor算法可破解RSA和椭圆曲线算法
- **"现在收集、未来解密"风险**:攻击者可以现在截获加密流量,等待量子计算机出现后解密
- 美国政府要求秘密信息保密50年,推动PQC标准化

### **13:40 - NIST标准化进展**
- NIST发布基于格的密码学算法标准:ML-KEM、ML-DSA、SLH-DSA
- 这些算法将成为未来标准,如同今天的RSA和ECC

### **15:30 - 三大风险领域**
1. 机密性风险:传输中的数据加密(优先级最高)
2. 认证风险(长期):固件签名等长期数字签名
3. 认证风险(短期):数字证书中的身份验证

### **18:00 - AWS迁移策略博客**
- 2024年11月发布AWS安全博客,概述迁移思路
- 强调大多数组织30年未经历密码学迁移(上次是1990年代DES到AES)
- 必须进行密码学资产清单盘点和优先级排序

### **20:45 - 迁移优先级建议**
1. 首要任务:公共端点的量子安全加密(解决"现在收集、未来解密"威胁)
2. 第二优先:代码签名等长期数字签名
3. 第三优先:数字证书中的短期信任根(需等待浏览器支持)

### **24:10 - 标准化工作**
- **NIST**:核心算法标准(FIPS 203/204/205)
- **IETF**:协议实现标准(TLS、SSH、IPSec)
- **ETSI**:电信行业协议标准
- AWS参与所有标准化工作,确保互操作性

### **26:30 - AWS开源工具**
- **AWS-LC**:开源密码学库,支持x86和ARM64优化
- **S2N**:开源TLS实现,最小化延迟
- 所有工具免费且开源,体现透明度承诺

### **29:00 - 客户责任分层**
- **客户自定义代码**:需自行查找和替换RSA/ECC算法
- **TLS终止端点**:如使用ALB、API Gateway、CloudFront等AWS服务,AWS已自动提供PQ支持
- **托管服务**:S3等服务端点已支持ML-KEM

### **32:15 - 部署现状与性能**
- Cloudflare报告:超过50%流量已使用ML-KEM,无延迟问题,几乎无故障报告
- TLS 1.3添加新密码套件不会像协议升级那样造成兼容性问题

### **35:40 - AWS PQC时间线**
- **2024年8月**:NIST认证ML-KEM和ML-DSA为FIPS 203/204
- **2024年末**:AWS-LC发布ML-KEM初始实现
- **2024年末**:KMS、ACM、Secrets Manager部署PQ支持
- **2025年**:KMS推出ML-DSA原生签名操作
- **2025年9月**:CloudFront默认支持ML-KEM
- **2025年10月**:发布优化版AWS-LC并提交FIPS验证

### **38:20 - 最新服务更新(2025年末)**
- ALB、NLB、API Gateway支持量子安全TLS策略
- S3端点支持ML-KEM(几周前发布)
- Private CA支持ML-DSA证书签名
- 2026年优先级:所有公共端点支持ML-KEM

### **42:00 - 加密敏捷性建议**
- 不要将PQC视为一次性项目,而是建立可持续的密码学升级流程
- 未来可能需要迁移到更优化的算法(特别是数字签名领域)
- 建立自动化管道,简化未来算法替换

### **44:30 - Vanguard案例分享开始**
- Rajie Sharma介绍Vanguard:5000万投资者,11.9万亿美元资产管理
- 2万名员工,无物理分支机构,完全依赖安全加密数字渠道
- 几年前识别量子计算为重大威胁

### **46:15 - 监管环境**
- 美国:NIST 2024年8月发布标准
- 欧盟:NIS 2和DORA法规要求"最强加密形式"
- 加拿大:政府发布量子安全实施计划
- **关键信息**:如果尚未开始PQC迁移,现在就是启动时机

### **48:00 - Vanguard技术环境**
- 使用AWS但也运营数据中心和其他云服务
- 需要与第三方SaaS合作伙伴集成
- 会议重点:AWS环境内的实施

### **49:30 - Vanguard四大支柱战略**
1. 战略响应:将PQC定位为业务问题而非技术问题
2. 风险嵌入:将量子风险纳入企业风险管理框架
3. 可见性工具:识别所有密码学资产位置
4. IT迁移:与IT团队合作迁移到格基算法

### **51:45 - 战略响应细节**
- 用业务风险和机遇的语言沟通
- 类比韧性管理等大型IT风险
- 强调监管合规和客户期望
- 突出早期行动的紧迫性:迁移可能需要多年时间
- 必须在"Q日"(量子计算机出现日)之前完成

### **54:00 - 会议结束**
- 强调PQC是团队运动,需要跨组织协作和高管支持
- 鼓励参会者立即开始规划和实施

- 建立加密敏捷性以应对未来算法演进
- Vanguard等早期采用者提供了可借鉴的企业级实施框架
