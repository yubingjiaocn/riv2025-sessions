# AWS VMware迁移：成功案例、路线图与策略

## 会议概述

本次AWS re:Invent 2025分组会议聚焦于VMware工作负载向AWS云平台的迁移策略与实践经验。会议邀请了三位来自不同行业的客户代表：Envision Healthcare云与基础设施交付副总裁Tyler Barhydt、墨西哥航空基础设施副总监Angel Gonzalez，以及加州大学首席信息官Molly Greek。

AWS作为最早帮助客户迁移VMware工作负载到云端的提供商，强调没有一刀切的解决方案。每个组织都有独特的应用程序、数据和基础设施需求，因此AWS提供多种迁移路径：通过Amazon EVS进行快速迁移、使用AWS Transform for VMware加速重新托管到EC2、重新平台化到容器服务，以及完全重构为云原生服务。三位客户分享了各自的迁移历程、面临的挑战以及获得的经验教训。

## 详细时间线与关键要点

### 0:00-10:00 开场介绍与AWS迁移策略概述
- AWS提供最全面的VMware工作负载迁移和现代化路径
- 四种主要迁移方式：重新定位(EVS)、重新托管(EC2)、重新平台化(容器)、重构(云原生)
- AWS Transform for VMware新增强化代理AI能力，自动化发现、依赖映射和网络转换
- 现场调研显示大部分与会者正在考虑VMware工作负载迁移

### 10:00-15:00 客户背景介绍
- **Envision Healthcare (Tyler Barhydt)**：专注医生EMHM服务，拥有25年IT经验，负责云和基础设施交付
- **墨西哥航空 (Angel Gonzalez)**：墨西哥最大航空公司之一，航线覆盖90多个目的地，负责云、本地服务、网络等基础设施
- **加州大学 (Molly Greek)**：为22个不同组织提供共享服务，包括25万学生申请系统、六大医疗系统、加州第二大雇主的HR薪资系统

### 15:00-25:00 Envision Healthcare迁移经验
- 面临11个以上数据中心的碎片化问题，大量M&A活动造成基础设施分散
- 采用云优先策略，主要使用AWS，部分工作负载在Azure
- 利用Transform、应用发现服务、EC2、RDS等工具
- 2025年成功退出一个数据中心，计划2026年底前退出剩余两个主要数据中心
- 通过EBA(体验式加速)研讨会和Presidio合作伙伴获得支持
- 整合本地和云团队，利用Skill Builder进行技能提升

### 25:00-32:00 墨西哥航空EVS迁移案例
- 四年前开始云迁移战略，目标是关闭三个主要数据中心，350个应用程序迁移
- 采用重构和重新平台化策略处理关键工作负载
- 面临技术更新和过时问题，选择EVS作为快速迁移方案
- 利用HCX技术扩展网络，保持IP地址不变，避免应用程序重大更改
- 一个月内成功迁移96台服务器、50多个应用程序
- 获得性能提升、网络安全改善等多重收益，客户反馈良好

### 32:00-38:00 加州大学应急驱动的迁移
- 从紧急情况开始：28万员工HR薪资系统性能问题，每次运行时间越来越长
- 六个月内迁移到AWS，性能问题完全解决
- 本科生申请系统：25万申请者，连续两年在截止前15分钟崩溃
- 与AWS合作解决高峰负载问题，AWS团队在午夜时分现场支持
- 大型机迁移：保留50-99年数据要求，与Deloitte合作，节省300万美元
- 最终实现100% AWS部署，及时避免VMware成本上涨

### 38:00-44:00 经验教训与未来规划
- **发现和清理**：识别大量僵尸服务器和未使用应用程序，Envision从5000核心减少到22核心
- **治理重要性**：避免无序的云使用，建立标准、治理和FinOps实践
- **合作伙伴关系**：将供应商视为业务合作伙伴，坦诚沟通预算和挑战
- **安全改善**：加州大学解决6万个漏洞，移除15年历史的遗留系统
- **未来规划**：墨西哥航空2026年实现100%云部署，继续现代化tier 1和tier 3应用
- **技能建设**：持续利用Skill Builder培训，招聘云专业人才，推进基础设施即代码
