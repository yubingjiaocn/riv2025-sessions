# AWS re:Invent 2025 - 使用 Kiro 和 MCP 服务器加速构建无服务器应用程序

## 会议概述

本次技术分享会由 AWS 的两位主要解决方案架构师 Sean Kendall 和 Brian Zambrano 主讲。Sean 专注于无服务器和生成式 AI 技术，拥有超过 6 年的 AWS 工作经验；Brian 则来自生成式 AI 创新中心，负责为客户免费构建生成式 AI 概念验证和 MVP 项目。

会议的核心主题是展示如何使用 Amazon Kiro（原 Amazon Q Developer）和模型上下文协议（MCP）服务器来快速构建无服务器应用程序。演讲者通过实际构建一个井字棋游戏的后端来演示整个开发流程，展现了 AI 辅助开发工具如何显著提升开发效率。会议强调了无服务器架构结合 AI 工具的强大优势，能够在极短时间内完成复杂应用的构建和部署。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍和工具概述
- 演讲者自我介绍，Brian 来自生成式 AI 创新中心，为客户免费构建 AI 解决方案
- 介绍会议目标：展示如何使用 Kiro 和 MCP 服务器加速无服务器应用开发
- 现场调研：大部分听众了解 Kiro，但实际使用者较少；多数人在构建无服务器应用

### 5:00-15:00 开发工具详细介绍
- **Amazon Kiro CLI**：命令行界面，支持完全自动化开发流程
- **Kiro IDE**：集成开发环境，提供可视化界面和更丰富的交互体验
- **MCP 服务器概念**：模型上下文协议，为 AI 模型提供最新信息和执行能力的标准接口
- 重要说明：Q 品牌正在迁移到 Kiro，功能基本相同

### 15:00-25:00 MCP 服务器深入解析
- **核心概念**：MCP 服务器作为后端系统的通用接口，提供最新知识和执行操作能力
- **实际应用场景**：
  - AWS 最新文档访问
  - 实时股价数据
  - 天气信息
  - 公司内部 API 集成
- **演示中使用的 MCP 服务器**：
  - AWS Serverless MCP：SAM 集成和无服务器指导
  - AWS CDK MCP：包含 CDK Nag 安全检查和 Power Tools 集成
  - AWS Core MCP：提示理解和其他 MCP 服务器路由
  - AWS Documentation MCP：最新 AWS 服务文档
  - Fetch MCP：网页内容获取能力

### 25:00-35:00 目标架构和提示工程
- **目标架构展示**：使用 AI 生成的井字棋游戏架构图
- **提示工程最佳实践**：
  - 明确指定"构建和部署"以实现完全自动化
  - 指导使用特定 MCP 服务器获取 OpenAPI 规范
  - 指定部署到默认 AWS 账户
  - 要求自动测试 API 功能
  - 处理 CORS 头部配置
  - 添加项目标签和自动删除保护

### 35:00-45:00 Kiro CLI 实际演示
- **Agent 配置**：展示 JSON 配置文件结构，包括系统提示和 MCP 服务器设置
- **安全考虑**：讨论 allowed_tools 设置和 -a 标志的使用风险
- **MCP 服务器添加**：从 AWS Labs GitHub 仓库复制配置到本地 agent 文件
- **实际构建过程**：
  - 启动 Kiro CLI 并加载预配置的 re:Invent agent
  - 输入完整提示开始自动化构建流程
  - 展示实验性功能如待办事项列表和检查点

### 45:00-55:00 Kiro IDE 演示和故障排除
- **IDE 界面介绍**：文件系统导航、聊天界面、MCP 服务器配置
- **Steering Documents**：创建项目特定的指导文档，使用 Refine 功能自动改进
- **实际构建过程**：使用简化提示构建相同的井字棋后端
- **错误处理演示**：
  - 两个演示都遇到了验证错误
  - 展示如何将错误信息反馈给 AI 进行自动修复
  - 强调 CORS 配置仍然是常见问题点

### 55:00-60:00 总结和问答
- **性能对比**：两种方法都在约 5 分钟内完成构建和部署
- **成功验证**：修复错误后，井字棋游戏成功运行
- **算法改进**：演示如何要求 AI 改进游戏算法使其更具挑战性
- **资源链接**：提供 QR 码访问 Kiro CLI、Kiro IDE 和 MCP 服务器文档
- **最终建议**：填写会议调查问卷提供反馈
