# AWS re:Invent 2025 Metronome 分会场总结

## 会议概述

本次分会场由 Metronome 的解决方案工程师 Austin 主讲,主题为"为什么使用量计费必须被视为基础设施"。这是 Metronome 首次参加 AWS re:Invent,公司服务于包括 Salesforce、OpenAI、Anthropic、Nvidia、Confluent 和 Langchain 在内的行业领先企业。

演讲核心观点是:在 AI 时代,软件货币化正在从传统的许可证和订阅模式转向实时的使用量计费模式。这种转变要求将计费系统从后台财务功能提升为核心产品基础设施。Metronome 通过实时事件处理、模块化定价模型和高可用性架构,解决了使用量计费在正确性、低延迟和高可用性方面的技术挑战。演讲强调,现代计费系统必须具备可扩展性、模块化、可编排性和可扩展性四大基础设施原则,才能支持快速迭代的 AI 产品定价策略。

## 详细时间线

00:00 - 开场介绍
- Austin 介绍自己来自 Metronome,担任解决方案工程师
- 这是 Metronome 首次参加 AWS re:Invent
- 公司服务客户涵盖从 Salesforce 到 OpenAI、Anthropic 等 AI 前沿公司

01:30 - 客户案例展示
- 列举主要客户:OpenAI、Confluent、Anthropic、Nvidia、Langchain
- 在展会现场看到许多合作伙伴的标志

02:15 - 软件时代演进
- 从许可证模式到订阅/席位模式
- 当前进入代理(Agentic)时代:工作流、行动、混合定价模式
- 新的定价模型几乎每天都在被创造

03:30 - 混合定价模式兴起
- 公司从纯席位制转向席位+配额的混合模式
- LLM 是纯使用量模式,使用即产生成本
- 代理定价难以简单套用订阅模式

04:45 - 货币化的实时化转变
- 过去:计费是缓慢的后台批处理,月度账单,与产品体验脱节
- 现在:实时工作流、通知系统、权限控制、产品体验核心化
- 以 ChatGPT 的 Token 为例,计费单位成为产品核心

06:00 - 货币化作为战略增长杠杆
- 连接客户体验与产品使用的最终状态
- 成为产品开发和客户匹配的持续飞轮

06:30 - 事件驱动架构
- Metronome 通过业务事件处理:API 调用、推理、服务器启停等
- 实时归因和控制:访问限制、超额通知、90% 预付提醒
- 这些必须是实时的,不能按日或按月处理

08:15 - 为什么这很困难
- 传统上,事务可观测性是"三选二"问题:正确性、低延迟、高可用性
- 大多数公司只能选择两个

08:45 - 正确性挑战
- 金融服务需要每一分钱都准确归因
- 需要通过审计(无论是上市公司还是融资初创公司)
- 事件可能乱序到达,必须按时间戳正确归因

09:30 - 低延迟需求
- 实时服务告警、权限控制、欺诈检测
- PLG(产品驱动增长)模式下客户即时访问,需要防止坏账
- 客户期望实时信息,不能等 24 小时更新
- 财务团队需要实时了解所有客户的月度状态

10:45 - 高可用性要求
- 计费是基础设施,永远不能宕机
- 产品可以宕机,但计费必须始终可用
- 支持各种仪表板的数据必须随时可访问

11:15 - Metronome 的解决方案
- 同时实现了正确性、低延迟和高可用性三者

11:30 - 四大基础设施原则
- 可扩展性(Scalable)
- 模块化(Modular)
- 可编排性(Orchestrable)
- 可扩展性(Extensible)
- 没有公司第一次就能做对定价,系统必须支持迭代

12:15 - 可扩展性详解
- 实时摄取信息和发票计算
- 处理使用量激增,不会崩溃
- 以 OpenAI 为例,灵活扩缩容

13:00 - 模块化详解
- 解耦计费模型的每个部分
- 为大客户提供复杂定制合同
- 同时支持完全独立的 PLG 细分市场
- 实时更新定价,无需迁移(迁移是"脏词")

14:00 - 可编排性详解
- 发票发送到支付提供商
- 仪表板通知客户
- 支出告警:90% 阈值警告,150% 限制访问
- 这些成为核心产品体验
- 信用账本管理,用于收入确认和消耗追踪

15:15 - 可扩展性详解
- Metronome 位于整个货币化的核心
- 使用事件/交易输入系统
- 实时计量和定价评级
- 灵活的定价和打包
- 合同生命周期管理:了解何时续约、转为免费试用或欠费合同

16:30 - 系统集成
- 集成 CRM 和计费门户
- 提供数据链接供团队深入了解财务和客户状态
- 评估、理解、分析和迭代定价策略
- 最终目标:确保收款(不只是为了兴趣,而是业务健康)
- 支持 AWS Marketplace 和 Stripe

17:45 - AI 时代的速度
- AI 不再只是流行词,而是业务现实
- OpenAI 在过去 12 个月进行了 10 多次定价调整
- 85% 的软件公司正在采用使用量定价
- 可能不是核心模式,但作为混合模式的一部分(如在平台上添加代理或集成 LLM)

19:00 - 货币化工程师的新角色
- 这是一个新兴职位头衔
- 需要评估平台能否:
  - 实时处理数百万事件
  - 处理 PLG 规模
  - 保持高可用性
  - 无需部署和迁移即可更改定价
  - 让非工程师(RevOps)进行定价实验
  - 成为财务和产品的单一真实来源

20:30 - Metronome 的核心能力
- 实时发票计算
- 模块化定价模型
- 创建、迭代、推出定价变更
- 发布使用量报表和发票数据

21:00 - OpenAI 案例研究

- 实时部署给数百万用户
- 定价更新快速,与财务协调,基于 Metronome 分析匹配价值
- 超高效的计费运营
