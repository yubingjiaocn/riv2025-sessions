# AWS re:Invent 2025 云治理会议总结

## 会议概述

本次AWS re:Invent 2025分组会议聚焦于云治理的实际应用，特别展示了日本数字厅如何在大规模云环境中平衡敏捷性与合规性。会议由AWS云治理团队的Nivas Durairaj主持，日本专业解决方案架构师经理Ohmura和日本数字厅首席云官Yamamoto作为特邀嘉宾分享了实践经验。

日本数字厅作为本次会议的核心案例，展示了如何为1700多个地方政府提供服务，管理超过6000个AWS账户的大规模云环境。该机构成立于2020年9月COVID-19疫情后，致力于推动日本政府的数字化转型，包括My Number Card国民身份证系统和政府云平台的建设。会议深入探讨了在高度监管环境中实现创新、组织敏捷性与安全合规之间平衡的最佳实践。

## 详细时间线与关键要点

### 0:00-10:00 会议开场与云治理基础概念
- 以幽默的方式定义云治理：不是简单地将服务器和500页政策文档一起迁移到云端
- 介绍演讲嘉宾：AWS云治理团队Nivas Durairaj、日本专业SA经理Ohmura、日本数字厅首席云官Yamamoto
- 阐述高度监管行业客户需求：医疗、公共部门、金融服务等领域对创新、组织敏捷性、安全合规的平衡需求

### 10:00-20:00 云治理架构与最佳实践
- 定义AWS云治理：指导组织遵循最佳实践的规则、流程和报告集合
- 介绍多账户环境架构：使用AWS Organizations和AWS Control Tower构建安全、可扩展的登陆区
- 详解关键组件：Identity Center身份联合、日志归档账户、审计账户、账户自动化配置
- 强调多账户设计原则：提高运营效率、团队管理和降低安全风险

### 20:00-30:00 治理控制类型与实施策略
- 三种治理控制类型：预防性控制（限制用户操作）、主动控制（资源配置前扫描）、检测性控制（合规性评估）
- AWS Organizations预防性控制：服务控制策略(SCP)、资源控制策略(RCP)、声明性策略
- AWS Config检测性控制：合规包、聚合器、自定义规则创建
- 推荐使用Amazon Q Developer简化自定义规则创建过程

### 30:00-40:00 日本数字厅案例介绍
- 播放数字厅宣传视频，展示数字化服务如何改善民众生活
- Yamamoto介绍数字厅背景：2020年COVID-19后成立，解决疫苗追踪等数字化挑战
- 重要成就：My Number Card覆盖1亿人口（80%普及率），政府云平台支持1700个地方政府
- 规模数据：超过6000个AWS账户，每月新增300-400个账户

### 40:00-50:00 政府云三大目标实现策略
- **治理目标**：制度层面通过直接合同和日本法律保障数据主权；技术层面通过用户自有加密密钥和HSM硬件安全模块确保数据安全
- **地方自治**：预定义自动化流程访问环境，数字厅员工无法直接访问地方政府环境，年度审计确保合规
- **敏捷性与可扩展性**：通过Control Tower等云技术实现，开发真实组织结构到AWS Organizations的映射功能

### 50:00-56:30 技术实施细节与未来规划
- Ohmura详解四大最佳实践：安全框架对齐、强身份基础、账户自动化配置、持续监控测试
- 身份管理优化：使用My Number Card实现用户创建时间从5天缩短至1小时
- 账户创建自动化：通过Step Functions、SQS、Lambda实现每月300+账户的自动化创建
- 多云策略原则：一个系统使用一个CSP、一个运营团队对应一个CSP、确保数据可移植性
- 未来计划：2026年4月发布GCAS DevStack Git服务，准备政府AI环境

### 总结与新功能发布
- AWS新功能发布：账户转移简化、多组织账单集中管理、Control Tower独立控制体验
- 核心理念：云治理为实验和执行提供正确基础，通过护栏替代路障，自动化实现合规
