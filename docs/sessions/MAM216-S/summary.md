# AWS re:Invent 2025 - 加速传统系统现代化方法

## 会议概述

本次技术分享由Slalom公司AWS云能力团队的Alex Hatcher主讲，重点介绍了他们的"零遗留系统"(Zero Legacy)战略方法。演讲深入探讨了如何利用生成式AI技术来加速传统系统现代化，特别是大型机和AS/400工作负载的迁移。Hatcher分享了Slalom在帮助客户从单体、有状态的传统架构转向微服务、无状态现代架构方面的实践经验。

演讲的核心主题围绕传统系统现代化面临的挑战，包括成本上升、技能人才短缺等问题，以及生成式AI如何为这些挑战提供解决方案。通过Amazon Q、Bedrock知识库、Kendra等AWS服务，团队能够快速理解和逆向工程传统代码库。演讲还详细介绍了与La-Z-Boy公司的成功合作案例，展示了如何将z系列大型机完全迁移到AWS云环境。

## 详细时间线与关键要点

### 0:00-2:30 - 开场与零遗留系统概念介绍
• Alex Hatcher介绍Slalom的AWS云能力团队背景
• 定义传统系统：任何阻碍创新和现代化的系统
• 零遗留系统战略目标：消除技术债务对业务创新的拖累
• 传统系统特征：单体架构、有状态；现代架构：微服务、无状态

### 2:30-5:00 - 传统系统现状与挑战分析
• 大型机(z/OS、AS/400)工作负载的特点：高度有状态、单体化、集中式
• 客户面临的主要问题：成本持续上升、技能人才招聘困难
• 真实案例：政府客户培训子女维护父辈开发的大型机系统
• 技能短缺问题的严重性：原开发者退休后由子女接手维护

### 5:00-8:30 - 生成式AI在现代化中的应用
• AI技术推动现代化进程的关键作用
• 传统系统的"黑盒"特性：代码、数据访问困难，实时监控受限
• 生成式AI逆向工程能力：
  - 使用Amazon Kendra索引和Bedrock知识库
  - 将代码上传至S3存储桶并建立索引
  - 通过MCP和Q Developer进行代码查询和理解
• 一小时内即可开始代码分析，大幅降低人工成本

### 8:30-11:00 - 生成式AI的多重应用场景
• 逆向工程：深度理解和文档化传统系统
• SDLC流程增强：使用Q CLI等工具提升开发效率30%以上
• 前向工程：结合Kiro CLI等规范驱动开发工具包
• 系统分解能力：
  - 提取高级域解释
  - 分解业务域
  - 定义端到端执行路径
• 微服务转换：将单一执行路径转换为微服务架构

### 11:00-13:30 - La-Z-Boy成功案例详解
• 项目背景：客户初期对大型机现代化持抗拒态度
• AWS资助的评估阶段：
  - 使用Amazon Q分析现有代码
  - 映射第三方系统依赖关系
  - 识别关键利益相关者
• 动员阶段实施：
  - 使用Q加速基础设施部署
  - 构建Step Functions替代JCL作业
  - COBOL转Python并部署为ECS任务
  - DB2迁移至Aurora Postgres
• 全面迁移与生产就绪验证

### 13:30-15:00 - 项目成果与总结
• 建立DB2与Postgres数据同步比较流程
• 发现并修复未知业务流程缺陷
• 实现真正的零遗留系统目标
• 业务流程现代化的连锁效应
