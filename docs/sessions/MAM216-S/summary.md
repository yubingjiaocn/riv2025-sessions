# AWS re:Invent 2025 会议总结：加速遗留系统现代化

## 会议概述

本次会议由 Slalom 公司 AWS 云能力团队的 Alex Hatcher 主讲，重点介绍了他们的"零遗留系统"(Zero Legacy) 战略及其在加速遗留系统现代化方面的实践方法。演讲者强调，遗留系统是指任何阻碍创新和现代化的系统，特别是那些单体架构、有状态的系统，如大型机 Z/OS 和 AS400 工作负载。这些系统往往导致企业在尝试创新时因技术债务而退缩，无法实现业务目标。

Slalom 的"零遗留系统"活动旨在帮助客户摆脱遗留系统的束缚，使其能够自由创新而不受技术限制。演讲者指出，当前推动现代化的关键因素包括不断上升的维护成本、技能人才短缺（甚至出现开发人员培训子女维护系统的情况），以及生成式 AI 技术的突破性进展。生成式 AI 特别是在逆向工程、代码理解和规范驱动开发方面展现出巨大潜力，使得现代化项目的效率大幅提升。

会议还分享了 Lazy Boy 家具公司的成功案例，展示了如何通过 AWS 服务和生成式 AI 工具（特别是 Amazon Q）成功将其 Z 系列大型机迁移到云端，实现了真正的"零遗留系统"。该项目不仅完成了技术迁移，还发现并修复了多年未被发现的业务流程问题，为企业开启了持续创新的新篇章。

## 详细时间线

0:00 - 0:45 - 开场介绍
- 演讲者 Alex Hatcher 自我介绍，来自 Slalom 的 AWS 云能力团队
- 介绍会议主题：加速遗留系统现代化的方法

0:45 - 2:00 - 会议议程概览
- 介绍"零遗留系统"活动的战略重要性
- 预告将讨论当前客户面临的挑战和机遇
- 提及 Lazy Boy 客户成功案例

2:00 - 3:30 - 定义遗留系统
- 遗留系统定义：任何需要修改、阻碍现代化或创新的系统
- 描述典型场景：产品团队想要创新，但因触及遗留系统而放弃
- "零遗留系统"的目标：消除创新障碍，实现自由创造

3:30 - 5:00 - 遗留系统的特征
- 遗留系统通常是单体架构和有状态的
- 现代架构则是微服务和无状态的
- 大型机（Z/OS、AS400）是典型的有状态、单体、集中式工作负载

5:00 - 6:30 - 客户面临的挑战
- 成本不断上升
- 招聘困难，技能人才短缺
- 案例：政府客户的原始开发人员培训自己的子女维护大型机系统
- 退休开发人员在海滩度假时仍需接听女儿关于 COBOL 代码的咨询电话

6:30 - 8:30 - 为什么是现在？AI 的推动作用
- 生成式 AI 成为现代化的重要推动力
- 遗留系统如大型机的代码、数据和监控信息相对封闭
- 生成式 AI 能够以前所未有的方式逆向工程这些系统

8:30 - 10:30 - 传统方法 vs. AI 方法
- 传统方法：多名解决方案架构师分析大型代码库，描述、综合、构建图表并验证，成本高昂且耗时
- AI 方法：使用 Amazon Kendra 索引、Bedrock 知识库，将代码上传到 S3，通过 MCP 和 Q Developer 访问
- 在一小时内即可开始询问代码相关问题，极大降低开销

10:30 - 12:00 - 生成式 AI 在现代化中的应用
- 逆向工程：理解和文档化遗留系统
- 传统 SDLC 增强：使用 Q Developer 或 Q CLI 提高生产力约 30% 或更多
- 能够将遗留代码上下文引入开发过程

12:00 - 14:00 - 前向工程与规范驱动开发
- 使用生成式 AI 提取系统知识
- 利用 Kiro CLI 等规范驱动开发工具
- 能够分解领域、定义执行路径、识别子程序
- 类似于微服务的单一执行路径概念
- 实现可追溯的规范驱动开发流程

14:00 - 16:30 - 人员与组织对齐的重要性
- 技术很重要，但 Slalom 的优势在于与客户建立联系
- 避免孤立的现代化尝试（仅 IT 部门或仅产品部门）
- 遗留系统通常是核心系统，存在数十年，业务流程围绕其构建
- 必须协调所有利益相关者，建立跨组织的一致性
- 基于共同目标定义战略

16:30 - 19:00 - Lazy Boy 案例：初期接触
- 在半天研讨会中了解到 Lazy Boy 有大型机
- 客户最初有抵触情绪，曾多次尝试现代化但未成功
- 团队正在缓慢推进现代化工作
- Slalom 建议至少进行对话，探讨可能性
- 发现真正的转型机会

19:00 - 21:00 - Lazy Boy 案例：评估阶段
- 结合 AWS 专业知识和生成式 AI（客户当时未采用）
- 进行 AWS 资助的评估
- 使用 Amazon Q 理解代码
- 映射第三方依赖关系和外部系统
- 绘制完整生态系统图
- 识别并联系各部门关键利益相关者

21:00 - 23:00 - Lazy Boy 案例：动员阶段
- 获得所有人的支持，决定推进项目
- 使用 Amazon Q 加速基础设施部署
- 使用 Q 构建 Step Functions 替代 JCL 作业
- 将 COBOL 转换为 Python 并部署为 ECS 任务
- 将 DB2 环境现代化为 Aurora PostgreSQL

23:00 - 24:30 - Lazy Boy 案例：迁移与测试
- 动员阶段使用工作负载子集验证流程
- 完全迁移阶段现代化全部工作负载
- 达到生产就绪状态
- 建立测试流程确保功能等效性
- 设置 DB2 和 PostgreSQL 同步运行和数据比对流程

24:30 - 26:00 - Lazy Boy 案例：发现与优化
- 在测试过程中发现并修复了一些 bug
- 更重要的是：发现了客户从未知道的历史 bug
- 需要重新思考某些业务流程
- 这是组织首次意识到可以自由改变代码和流程
- 开启了"我们还能现代化什么其他流程？"的思考

26:00 - 26:45 - 总结与邀请
- 强调"零遗留系统"的力量在于解放创新能力
