# AWS re:Invent 2025 技术会议总结：AI SRE 代理解决方案

## 会议概述

本次技术会议介绍了 NeuBird 公司开发的 AI SRE 代理解决方案 Hawkeye，专注于通过人工智能自动化解决生产环境中的问题。演讲者展示了如何利用代理技术分析根本原因、减少平均解决时间，并与编程助手（如 Claude Code、GitHub Copilot 等）协作实施解决方案。

该解决方案的核心价值在于帮助企业在不改变现有工具的情况下，通过智能代理自动分析复杂的可观测性数据，从手动的故障排查流程转向自动化的根本原因分析。演讲者强调了这种方法能够显著提高生产力、减少系统停机时间，并为客户带来巨大的成本节约。

## 详细时间线与关键要点

### 0:00-5:00 - 开场介绍与问题定义
- 演讲者介绍 AI SRE 概念，强调解决生产环境问题的重要性
- 阐述当前挑战：大量可观测性数据需要在压力下快速分析
- 目标：通过代理自动化根本原因调查过程，减少平均解决时间
- 展示与编程代理（Cloud Code、Cursor、GitHub Copilot）的协作能力

### 5:00-8:00 - 客户案例展示
- 保险公司案例：部署在 VPC 中的 Hawkeye，分析跨本地和 AWS 的复杂环境
- 物流公司案例：管理 Databricks 上的大量夜间作业和 EKS 上的 Flink 作业
- 澳大利亚银行案例：从备份管理开始，现已扩展到多个用例场景
- 强调解决方案适用于安全敏感的企业环境

### 8:00-12:00 - 产品演示：Hawkeye 界面
- 展示 Hawkeye 仪表板：左侧显示未调查的事件和警报，右侧显示已调查的
- 演示从 Datadog 监控器触发的自动分析流程
- 展示丰富的上下文信息：事件描述、应用程序工件、事件时间线、证据、根本原因和纠正措施
- 强调深度数据分析能力：访问 traces、遥测数据、指标、日志和配置信息

### 12:00-16:00 - 编程助手集成演示
- 展示与 Claude Code 的集成，通过 MCP（模型上下文协议）服务器
- 演示自文档化的 API 接口，支持多种工作流程
- 展示典型操作：列出警报、启动调查、监控进度、获取结果
- 演示自动化修复步骤和 AWS 连接创建过程

### 16:00-19:00 - 产品发布与市场策略
- 宣布昨日发布的自助服务入门功能
- 介绍 AWS Marketplace 上的两周免费试用
- 对比竞争对手的孤立解决方案，强调跨平台整合能力
- 支持多种可观测性工具：CloudWatch、Datadog、Dynatrace、Grafana、Prometheus

### 19:00-20:00 - 总结与行动号召
- 鼓励参会者通过 AWS Marketplace 试用产品
- 提供展位信息（1219 号展位）和现场咨询机会
- 强调产品成熟度和公开可用性
- 按需付费模式，无长期承诺要求
