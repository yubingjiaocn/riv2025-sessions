# AWS re:Invent 2025 会议总结：使用欺骗技术让攻击者抓狂

## 会议概述

本次会议由 Fastly 公司的两位演讲者 Rick 和 Memo 主讲，重点介绍了如何利用欺骗技术（Deception Technology）来改变网络安全防御策略。演讲者通过生动的类比——盲人高尔夫球手 Charlie Boswell 在夜间挑战世界冠军 Ben Hogan 的故事，以及橄榄球比赛中进攻方与防守方的优势对比，阐述了如何通过创新手段"拉平竞争环境"，让防守方获得更多主动权。

会议的核心理念是：传统的网络安全防御往往只关注攻击手段本身，而忽略了攻击者也是人类，他们的决策过程会受到心理因素的影响。通过引入行为科学中的"助推"（Nudge）和"阻力"（Sludge）概念，Fastly 提出了一种全新的防御思路——不是简单地阻止攻击者，而是通过增加摩擦、制造混乱和挫败感，让攻击者在不知情的情况下浪费时间和资源，最终放弃攻击目标。

Fastly 将这一理念应用到其 Next-Gen WAF 产品中，推出了针对账户接管攻击的欺骗功能。该功能的独特之处在于它不需要单独的蜜罐基础设施，而是直接集成在 WAF 工作流中，通过篡改攻击者提交的用户名和密码，让所有凭证验证都失败，但不会明确告知攻击者被阻止，从而让攻击者误以为是凭证列表本身有问题，无法进行针对性的调整和反制。

## 详细时间线

### 开场故事：拉平竞争环境的艺术（0:00-2:30）

0:00-0:15 - 演讲者 Rick 询问现场是否有高尔夫爱好者，准备分享一个高尔夫故事

0:15-1:00 - 讲述 1958 年德克萨斯州沃斯堡盲人高尔夫球手协会锦标赛的故事，介绍全国冠军 Charlie Boswell

1:00-1:30 - 介绍 Charlie Boswell 的背景：二战期间在试图营救战友时失明，战后开始学习高尔夫

1:30-2:00 - Charlie 挑战当时的世界顶级高尔夫球手 Ben Hogan，提出每洞赌注 1000 美元

2:00-2:20 - Ben Hogan 认为与盲人比赛不公平而拒绝，但在 Charlie 的激将法下最终同意

2:20-2:30 - 揭晓关键转折：Charlie 提议比赛时间为"今晚 10 点"，在黑暗中比赛让双方处于同等条件

### 攻防优势对比（2:30-3:30）

2:30-3:00 - 用橄榄球比赛类比网络安全：进攻方（四分卫）拥有明显优势，可以制定战术并观察防守方部署

3:00-3:30 - 防守方只能猜测进攻方意图，处于被动地位；提出问题：如何拉平这个竞争环境，进入攻击者的思维模式

### 行为科学在网络安全中的应用（3:30-6:30）

3:30-4:00 - Memo 接手演讲，强调传统防御策略过于关注攻击手段而非攻击者本身

4:00-4:30 - 核心观点：攻击者也是人类，他们的大脑运作方式与我们相同，理解这一点是实施高效防御策略的关键

4:30-5:30 - 介绍行为科学的两个概念：
  - **Nudge（助推）**：以积极方式影响决策，使过程更简单（例如：密码强度指示器、运动提醒应用）
  - **Sludge（阻力）**：增加复杂性和摩擦（例如：验证码、订阅服务的复杂取消流程）

5:30-6:30 - 提出核心策略：将 Sludge 用于正当目的，通过增加摩擦来影响攻击者的行为和决策

### 对攻击者施加的成本分析（6:30-8:00）

6:30-7:00 - 量化成本：增加摩擦会让攻击者付出更多金钱和精力，失去其他攻击机会

7:00-7:30 - 声誉成本：攻击失败会导致攻击者失去信誉，难以招募新成员或获得新客户

7:30-7:45 - 信息成本：攻击者需要投入更多精力收集目标信息，如果提供无效信息会增加其成本

7:45-8:00 - 心理成本：通过制造挫败感、羞耻感和困惑来增加攻击者的心理负担——这也是本次演讲标题的由来

### 欺骗技术的历史与应用（8:00-9:30）

8:00-8:30 - 欺骗技术在历史上的应用，特别是战争中的应用（例如：特洛伊木马）

8:30-9:00 - 网络安全中的蜜罐（Honeypot）概念：通过欺骗攻击者来获取其战术信息

9:00-9:15 - 自然界中的欺骗：动物改变外观以捕猎或躲避捕食者

9:15-9:30 - 强调欺骗不是"耍阴招"，而是利用对敌人战术的了解来为己方谋利

### Fastly 的欺骗技术实现（9:30-13:00）

9:30-10:00 - Rick 介绍 Fastly 产品副总裁的背景（深厚的安全和行为经济学背景），这是该技术的创意来源

10:00-10:30 - 目标：进入攻击者的思维，让他们感到挫败从而放弃或转移目标

10:30-11:00 - 当前实现：针对账户接管攻击的 Next-Gen WAF 产品功能

11:00-11:30 - 工作原理：用户可配置触发欺骗的参数（恶意机器人信号、特定路径、IP 地址、国家等）

11:30-12:15 - 核心机制：一旦触发欺骗，系统会篡改攻击者提交的用户名和密码（即使是有效凭证），发送到源服务器后返回"用户名或密码错误"，让所有尝试都失败

12:15-12:45 - 关键优势：攻击者不知道发生了什么，无法获得反馈来调整策略（不像被明确阻止或速率限制那样可以调整）

12:45-13:00 - 攻击者会误以为是购买的凭证列表本身有问题，而不是防御系统在起作用

### 与传统蜜罐的对比（13:00-14:30）

13:00-13:30 - 传统蜜罐的问题：需要单独的基础设施，必须与生产环境保持同步

13:30-14:00 - 传统蜜罐的成本：需要运行两套实例，工程工作量大，维护成本高

14:00-14:30 - Fastly 方案的优势：直接集成在 WAF 工作流中，无需额外基础设施，易于调整和维护

### 未来发展与深度防御（14:30-16:30）

14:30-15:00 - 当前实现只是起点，从账户接管开始

15:00-15:30 - 未来计划：扩展到其他领域（API 安全、其他攻击向量），采用不同的欺骗方式

15:30-16:00 - 易用性优势：通过 GUI 界面操作，安全人员无需深入代码，可以轻松启用、记录日志，甚至在正式部署前进行测试

16:00-16:30 - 深度防御理念：欺骗技术只是 Fastly WAF 的一部分，还包括速率限制、DDoS 防护、CDN 和 WAF 的安全设计

### 总结（16:30-17:00）

16:30-16:45 - 所有功能都有日志记录，支持 API 和 Terraform 驱动，易于部署、回滚和迭代管理

16:45-17:00 - 最终目标：像 Charlie 在高尔夫比赛中那样拉平竞争环境，在攻击者造成损害之前迷惑他们
