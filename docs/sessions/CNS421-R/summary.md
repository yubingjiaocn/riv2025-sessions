# AWS re:Invent 2025 - 使用智能代理简化EKS运维操作

## 会议概述

本次会议是一场现场编程演示（Code Talk），由AWS首席解决方案架构师Sai Vennam和Lucas Duarte主讲。会议主题为"使用智能代理简化EKS运维操作"，展示了如何构建超越传统聊天机器人的AI代理系统。

演讲者通过现场编程演示了从去年基于RAG（检索增强生成）的方法演进到今年基于智能代理架构的解决方案。他们构建了一个名为"Kube Agent"的系统，能够自动监听Slack告警消息，执行Kubernetes集群故障排除，并将解决方案存储为"部落知识"供未来使用。整个系统采用微代理架构，使用多个专门的LLM模型来处理不同任务，实现了从被动响应到主动解决问题的转变。

## 详细时间线与关键要点

### 0:00-5:00 会议开场与演示预览
- 介绍演讲者和Code Talk形式的现场编程会议
- 展示Kube Agent的完整工作流程演示
- 演示智能代理如何自动响应Slack中的告警消息并执行故障排除

### 5:00-15:00 技术演进对比：从RAG到智能代理
- 回顾去年基于RAG的解决方案架构
- 解释RAG的局限性：上下文窗口限制、数据更新延迟、缺乏实时数据
- 介绍Strands开源SDK用于构建智能代理
- 展示新的多代理架构：编排代理 + 专家代理的组合

### 15:00-25:00 基础代理实现与MCP集成
- 现场编程演示基础Strands代理架构
- 展示手动编码的工具：describe_pod和get_pods
- 演示代理在缺乏工具时的局限性
- 介绍AWS托管的EKS MCP服务器
- 现场集成MCP，展示代理能力的显著提升

### 25:00-35:00 智能消息分类系统
- 从关键词匹配升级到AI驱动的消息分类
- 使用Nova Micro模型进行轻量级消息意图分类
- 实现Before Invocation Event回调机制
- 演示智能分类的效果：技术问题回复，日常对话忽略

### 35:00-45:00 部落知识存储系统
- 引入NASA Voyager探测器案例说明部落知识的重要性
- 设计内存代理架构，使用S3 Vectors作为知识库
- 实现自动知识存储和检索功能
- 展示微代理模式：独立的内存代理服务

### 45:00-55:00 代理间通信与集成
- 实现Agent-to-Agent (A2A)通信协议
- 构建代理发现和工具共享机制
- 演示编排代理如何智能路由请求到专门代理
- 现场测试知识存储和检索功能

### 55:00-64:00 完整故障排除演示与总结
- 部署故障应用进行端到端演示
- 展示代理如何结合历史知识和实时MCP数据
- 演示自动问题修复和知识存储循环
- 分享GitHub资源和会议反馈信息
