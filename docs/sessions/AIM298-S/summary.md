# AWS re:Invent 2025 生命科学领域的代理式AI与数据管理

## 会议概述

本次AWS re:Invent 2025分会场聚焦于生命科学行业中代理式AI(Agentic AI)在数据管理领域的应用与转型。演讲者来自ZS公司,分享了制药行业CIO们当前最关注的议题:如何从概念验证(POC)阶段过渡到真正的规模化转型。根据ZS的调研,十分之九的生命科学CIO正在转变思维方式,不再仅仅是将现有流程自动化,而是重新构想和优化整个业务流程,以充分发挥代理式AI的潜力。

演讲核心围绕两大数据管理范式展开:"AI所需的数据"(Data for AI)和"数据所需的AI"(AI for Data)。前者强调如何准备高质量、高准确度的数据以支持AI洞察,通过构建全面的元数据湖(metadata lake)将洞察准确率从70%提升至98%。后者则关注如何利用AI优化数据工程生命周期,包括软件开发效率提升40%、测试效率提升75%,以及通过AI实现自适应数据治理、自我修复的数据质量管理等。演讲者强调,成功的代理式AI转型需要跨职能协作、业务与IT深度融合,以及重新思考操作模型和基础设施架构。

## 详细时间线与关键要点

00:00:00 - 开场与CIO优先事项
- 介绍会议主题:生命科学领域的代理式AI数据管理与用例
- 生命科学CIO的首要目标:从代理式AI中看到实际价值
- 过去两年主要进行试点项目,现在需要实现规模化转型

00:01:30 - 行业转型趋势
- 十分之九的CIO正在转变思维:从自动化现有流程转向重新构想流程
- CIO面临的核心问题:如何弥合实验与实际执行/实施之间的差距
- ZS研究显示:十分之九的CIO希望加快数字化AI技术创新的步伐

00:03:00 - 思维方式转变的必要性
- 不能再用传统方式思考技术转型或AI转型
- 需要关注最终价值和所需的人员流程变革
- 高层已开始转变思维,但需要在整个组织中渗透

00:04:00 - 数据管理的两大核心范式
- **AI所需的数据(Data for AI)**:如何准备数据以确保准确的高质量洞察
- **数据所需的AI(AI for Data)**:
  - AI用于数据工程:优化软件工程生命周期,效率提升可达40%
  - AI用于数据运营:数据质量、治理、访问管理、策略、安全等

00:05:30 - 元数据治理的挑战
- 历史上元数据治理、可发现性和验证一直未能成功
- 原因:创建和维护元数据需要大量工作
- 现在可以使用AI不仅创建数据,还能维护和持续管理数据

00:06:30 - 三大转型领域
1. 重新构想数据消费:改变人机交互方式,转变分析工作流程
2. 重新定义数据工程:利用AI加速数据工程生命周期
3. 数据治理:自适应治理、自我修复的数据质量、自动化信息检索

00:08:00 - Data for AI:元数据湖的重要性
- 传统表级和列级元数据描述不足以达到所需准确度
- 需要创建全面的"元数据湖"(metadata lake),包括:
  - 血缘关系和连接条件
  - 业务规则的编码化
  - 领域特定的细微差别(如"销售"在市场准入、现场代表、财务部门的不同含义)

00:10:00 - 操作模型的范式转变
- 从中心化模型转向数据产品操作模型
- 需要业务深度参与,与IT协作
- 只有领域专家(如财务、市场准入人员)才能提供准确的业务上下文
- 可以使用代理创建元数据,但仍需提供初始上下文

00:11:30 - 准确度提升成果
- 通过实施元数据湖,洞察准确率从70%提升至98%
- ZS的目标是在明年达到100%的准确度

00:12:00 - 用例一:分析引擎转型
- 制药公司拥有大量分析团队执行各类分析(预测、绩效洞察等)
- 传统上需要数百人执行分析工作
- 分析类型包括:描述性分析、确定性分析、预测性分析

00:13:30 - 代理式分析实施方法
- 创建按领域划分的分析就绪数据集
- 构建深度上下文层,包括数据元数据和人类工作流程知识
- 创建智能层,通过API访问传统经典AI模型
- 部署针对不同用例的代理,遵循人类工作流程执行模型
- 根据分析复杂度保持人类参与(描述性分析可完全自动化)

00:15:00 - 分析转型成果
- 效率提升40%
- 大幅缩短洞察时间
- 通过编码算法、模型和人类智能实现自动化

00:16:00 - 用例二:文档生成
- 制药行业需要生成大量文档(营销内容、供应链文档、临床文档等)
- 传统流程复杂且耗时
- 使用生成式AI构建知识中心,包含协议、知情同意书等

00:17:30 - 文档生成的优势
- 显著节省资金和时间
- 确保遵循验证程序
- 可在整个生态系统中嵌入合规性
- 适用于临床、供应链和商业领域

00:18:30 - 用例三:软件工程生命周期优化
- 构建和测试阶段是最大的效率提升点
- 生成式AI擅长构建全新代码(对Python等新语言效果更好)
- 测试环节效率提升可达75%
- 整个软件工程生命周期效率提升可达40%

00:20:00 - 关键要点总结
1. 规模化洞察:不再停留在POC阶段,要实现真正的组织转型
2. 持续ROI:提前思考ROI来源,不仅是效率提升,还包括价值创造
3. 流程重塑:不要简单自动化现有流程,要重新思考和设计流程
4. 基础设施韧性:确保基础设施稳健,避免成本失控
5. 跨职能协作:数字化和AI团队必须协同工作,业务与IT深度融合

00:22:30 - 最终建议
- 业务流程转型是核心目标
- 上下文(Context)是代理式AI时代的关键
- 必须跨职能协作才能实现转型
