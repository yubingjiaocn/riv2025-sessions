# AWS re:Invent 2025 量子计算会议总结

## 会议概述

本次会议主题为"Amazon Braket量子计算：从探索到企业应用"，由AWS量子计算服务Amazon Braket总经理Eric Kessler和AWS量子计算市场推广负责人Michael Brett共同主讲。会议深入探讨了量子计算的基本概念、当前发展状态以及AWS在量子计算领域的战略布局。

演讲者首先澄清了关于量子计算的常见误解，强调量子计算机并非运行更快的经典计算机，而是能够运行更丰富算法类别的计算设备。会议重点介绍了AWS的四大量子技术支柱：Amazon Braket服务、高级解决方案实验室、AWS量子计算中心以及后量子密码学团队。通过与AstraZeneca的药物设计案例研究，展示了量子-经典混合工作负载的实际应用潜力。

## 详细时间线与关键要点

### 0:00-5:00 会议开场与量子计算基础概念
- Eric Kessler介绍会议议程和演讲者背景
- 澄清量子计算常见误解：量子计算机不是更快的经典计算机
- 强调量子计算机能运行指数级更高效的算法类别
- 引用费曼1981年名言："自然不是经典的，如果要模拟自然，最好使用量子力学方法"

### 5:00-10:00 量子计算的应用领域与挑战
- 解释量子计算的三个核心要点：不会取代经典计算机、许多用例目前隐藏、需要全新算法
- 介绍量子计算应用垂直领域：物理化学模拟、材料科学、密码学、优化和机器学习
- 强调当前技术仍处于早期阶段，尚无量子计算机能在工业相关问题上超越经典方案

### 10:00-15:00 AWS量子技术四大支柱
- 介绍AWS量子计算中心：位于加州理工学院，开发Cat量子比特技术
- 展示Ocelot原型芯片成果：错误纠正资源需求降低10倍
- Amazon Braket服务概述：2020年推出，提供多种量子计算机访问
- 支持五家供应商：IonQ、AQT、Rigetti、IQM、QuEra

### 15:00-20:00 Amazon Braket服务架构
- 三层架构：基础设施访问层、接口能力层、工具层
- 支持按需付费和预约模式
- 兼容主要量子计算框架：Qiskit、Penny Lane、CUDA Quantum
- 新功能Program Sets：多程序批量提交，性能提升最高24倍

### 20:00-25:00 政府合作与研究应用
- NERSC研究显示50%生产工作负载可通过量子计算指数级加速
- 与多国量子计划合作：新加坡NQCH、英国NQCC、爱尔兰QCloud、澳大利亚POSI
- 澳大利亚量子中心门户网站启动，提供量子和经典计算资源访问

### 25:00-30:00 企业量子计算策略（Michael Brett主讲）
- 当前客户主要为研究人员：国家实验室、大学、研究中心
- 摩根大通案例：Marco Pistoia强调提前布局的重要性
- 企业量子计算采用流程：用例发现、算法研究基准测试、混合机会探索

### 30:00-35:00 Quantum Embark项目与高级解决方案实验室
- Quantum Embark：12周固定价格企业入门项目
- 高级解决方案实验室：长期研发合作，开发量子算法知识产权
- 量子计算集成到AWS高性能计算服务组合中

### 35:00-40:00 AstraZeneca药物设计案例研究
- 四方合作：AstraZeneca、IonQ、NVIDIA、AWS
- 过渡金属催化问题：氧化加成和Suzuki-Miyaura偶联
- 量子-经典混合工作流：IonQ Forte-1量子计算机 + AWS Parallel Cluster
- 使用40个P5 EN实例（H100），实现656倍性能提升

### 40:00-41:20 行动建议与资源
- 领先企业策略：生成知识产权、了解供应链、建设人才队伍
- AWS量子计算参与方式：Braket服务、数字徽章、Quantum Embark、高级解决方案实验室
