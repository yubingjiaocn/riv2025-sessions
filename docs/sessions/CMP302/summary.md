# AWS re:Invent 2025 CMP302 会议总结

## 会议概述

本次会议主题为"利用AWS上的HPC加速跨行业工程创新"（CMP302）。会议由AWS HPC团队主持，并邀请了ARM公司生产力工程副总裁Karima Madridi和DTN公司大气科学建模工程团队负责人Taylor Gowan作为嘉宾演讲者。

会议重点介绍了HPC（高性能计算）和AI市场的快速增长趋势。根据Hyperion Research的数据，2024年HPC和AI市场增长了23.5%，达到近600亿美元规模，其中云计算占约100亿美元。传统HPC工作负载仍保持8.4%的增长，而AI基础设施增长高达166%。云采用预计将从2024年的100亿美元增长到2029年的237亿美元，占整体市场的20%。

会议详细展示了AWS为HPC优化的基础设施，包括专用计算实例（HPC 6A、7A、8A等）、加速器（从Nvidia L4到GB200，以及AWS自研的Trainium和Inferentia芯片）、高性能存储解决方案（FSx for Lustre等）和编排服务（PCS、Batch、Parallel Cluster）。特别值得注意的是，AWS宣布即将推出基于AMD第五代EPYC处理器的HPC 8A实例，以及Graviton 4处理器的持续创新。两位行业嘉宾分享了各自公司从本地数据中心迁移到AWS云端的转型历程，展示了云HPC在实际生产环境中带来的显著优势。

### AWS HPC基础架构 (13:00 - 23:00)

13:00 - 16:00 - HPC优化计算实例
- HPC 6A、7A、6ID、7G系列（数字代表代次，字母代表技术：A=AMD，I=Intel，G=Graviton）
- 加速计算：Nvidia L4到GB200系列
- 预告即将推出HPC 8A（基于AMD第五代EPYC处理器Turin）

16:00 - 18:00 - AWS自研芯片
- Graviton：基于ARM的处理器
- Trainium和Inferentia：加速器芯片
- 宣布Rainier项目：将部署50万颗Trainium 2芯片用于训练Anthropic Claude下一代模型
- Graviton 4：基于ARM Neoverse核心，7个chiplet，12个DDR5通道，96条PCIe Gen 5通道

18:00 - 20:00 - 存储解决方案
- 对象存储：S3
- 文件存储：EFS、FSx for Windows、FSx for NetApp ONTAP、FSx for Lustre、FSx for OpenZFS
- 块存储：Amazon EBS
- 通过EFA fabric连接到计算实例

20:00 - 23:00 - 编排服务
- PCS（Parallel Computing Service）：基于Slurm的全托管服务
- Batch：调度和运行容器化工作负载
- Parallel Cluster：开源传统服务，被大多数HPC客户使用
- 量子计算：AWS Bracket服务，提供离子阱、超导和中性原子三种模态

### ARM云转型之旅 (28:00 - 45:00)

28:00 - 31:00 - ARM业务概况
- ARM全球计算覆盖范围无与伦比，100%的全球互联人口依赖ARM
- 已出货超过3250亿颗基于ARM的芯片
- 拥有2200万ARM软件开发者
- 基础设施方面：已向数据中心出货超10亿颗Neoverse核心
- 2025年预计占超大规模云提供商50%的单位出货量

31:00 - 34:00 - ARM与AWS战略合作
- 合作始于2018年
- 共同愿景：高效可扩展的计算
- Neoverse和Graviton的联合价值主张
- 已帮助70,000家公司在云端采用ARM

34:00 - 37:00 - ARM面临的四大挑战
1. 行业压力：复杂性增加、上市时间紧迫、行业动态变化
2. 计算需求爆炸：AI工作负载驱动，可持续性需求
3. 战略目标：可扩展计算、创新速度、转型需求
4. 云敏捷性：需要在所有工作中采用云敏捷性

37:00 - 40:00 - ARM产品开发周期
- 产品开发周期至少一年
- 分为前端和后端两个阶段
- 前端：大规模回归测试，较小内存占用，单线程
- 后端：更长作业时间，需要大量内存，多线程
- 利用AWS不同类型实例优化工作负载放置

40:00 - 43:00 - 云原生平台架构
四大要素：
1. 专用平台：正确的平台配合正确的作业，高效的数据和存储管理
2. 自动化：提高可靠性、降低成本、消除人为瓶颈
3. 可观测性：提供实时洞察、故障分析、成本控制
4. CI/CD：简化应用开发和发布

43:00 - 45:00 - CloudRunner平台
- 针对前端作业优化
- 基于AWS Batch调度器
- 关键数据：
  - 50万个并发作业
  - 100个项目同时执行
  - 1000名工程师使用
  - 1小时内环境就绪
  - 支持安全和出口管制技术
- 利用Spot实例节省90%成本
- 存储成本降低70%

45:00 - 48:00 - Cloud Foundation Platform (CFP)
- 针对后端作业优化
- 使用LSF调度器和FSx for NetApp ONTAP存储
- 功能完全等同于本地HPC
- 支持LDAP用户认证、Unix权限
- 可连接本地Git仓库
- 支持批处理和交互式作业
- 关键优势：云弹性，存储可从5TB按需扩展

48:00 - 52:00 - 业务影响
- 从CPU受限转变为人力受限
- 前端验证：本地4个月4000个槽位 vs 云端1个月20000个槽位（5倍提升）
- 吞吐量提升66%
- 4年内产品数量从15个增加到25个

52:00 - 58:00 - 优化和效率
三大优化领域：
1. 节省计划：
   - Spot实例：用于可中断作业
   - 实例节省计划：用于可预测作业
   - 计算节省计划：用于不断变化的作业类型
   - 采用混合平衡策略

2. EDA工具优化：
   - 不同工具有不同行为和需求
   - 需要分析运行时特性和内核调优
   - 根据工具特性匹配合适的EC2实例
   - 前端作业优化示例显示显著性能提升

3. 存储优化：
   - 工作负载可能非常突发，读写性能动态变化
   - 前端受元数据IOPS限制
   - 后端受数据吞吐量限制
   - FSx for NetApp ONTAP提供弹性和可扩展性

58:00 - 59:00 - Graviton性能优势
- 在前端工作负载中表现出色
- 在性价比和运行时性能方面优于AMD和Intel
- Graviton 3和4相比其他实例提供商性能更优

## 总结

本次会议全面展示了AWS HPC生态系统如何推动跨行业工程创新。从市场趋势来看，HPC和AI的融合正在加速，云计算在其中扮演越来越重要的角色。AWS提供的完整HPC解决方案栈——从底层优化的计算和存储基础设施，到灵活的消费模型，再到高级编排服务——为企业提供了强大的工具集。

ARM和DTN的实际案例证明，云HPC不仅能够提供与本地数据中心相当的性能，还能通过弹性扩展、快速访问最新技术、优化的成本结构等优势，显著提升工程效率和业务敏捷性。这些成功案例为其他考虑云迁移的HPC用户提供了宝贵的参考。
