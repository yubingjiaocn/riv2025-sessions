# AWS re:Invent 2025 ScyllaDB 会议总结

## 会议概述

本次会议由ScyllaDB技术总监Felipe主持，重点介绍了ScyllaDB数据库在高性能、大规模数据密集型应用中的优势。会议邀请了两家重要合作伙伴：SAS和Freshworks，分享了他们使用ScyllaDB解决实际业务挑战的成功案例。

Felipe首先介绍了ScyllaDB的核心价值主张：为需要可预测性能的大规模应用提供数据库解决方案。ScyllaDB采用贴近硬件的架构设计，能够最大化底层基础设施的性能，一个三节点集群就能在AWS EC2上支持每秒数百万次操作，同时保持可预测的个位数毫秒延迟。会议展示了ScyllaDB在实时个性化广告投放和AI驱动的客户服务平台中的实际应用效果。

## 详细时间线与关键要点

### 0:00-10:00 ScyllaDB产品介绍
- Felipe介绍ScyllaDB解决的核心挑战：数据密集型应用在扩展时面临的性能瓶颈问题
- ScyllaDB的三个主要适用场景：响应时间不可预测、数据库无法跟上吞吐量需求、数据库扩展成本失控
- 用户群体分析：大多数用户需要接近百万级每秒操作数，P99响应时间在个位数到20毫秒之间
- ScyllaDB的核心优势：高可用架构、支持事务和分析工作负载、原生向量搜索功能

### 10:00-25:00 SAS实时广告个性化案例
- SAS高级软件开发经理Brian Jones分享SAS 360 Match平台的技术架构
- 数字视频广告的核心组件：广告决策、播放信号反馈、优化算法
- SAS 360 Match处理规模：每天数十亿次展示、视频广告和程序化竞价请求
- 广告决策流程：会话启动、数据加载、请求处理、创意选择、响应渲染
- ScyllaDB在SAS中的应用：提供低延迟读取、线性扩展、数据冗余、工作负载优先级功能

### 25:00-35:00 SAS技术深度解析
- 数据扩展策略：会话启动时激活数据、必要时写入更改、减少集群内交互
- 流量峰值处理：负载均衡、自动扩展、预定扩展、CPU利用率缓冲、负载削减
- ScyllaDB存储的数据类型：观众数据、注册数据、事件计数、广告曝光记录、标识符映射
- 客户案例：ITV数字广告平台负责人Alex Maison对SAS 360 Match灵活性的评价

### 35:00-45:00 Freshworks AI架构重设计
- Freshworks工程副总裁Sreedhar和高级经理Premkumar介绍公司背景
- Freshworks规模：75,000客户、40种语言支持、接近10亿美元收入
- 数据技术栈演进：从单一MySQL到多种数据技术的组合
- Dataverse平台要求：数据工程、大规模可靠性、灾难恢复、数据驻留、安全性

### 45:00-56:00 Freshworks迁移策略与未来规划
- 三种数据库迁移场景：Cassandra到ScyllaDB、MySQL到ScyllaDB、DynamoDB到ScyllaDB
- 零停机迁移策略：使用ZDM代理进行双重读取、CDM迁移器处理历史数据
- 性能提升成果：延迟降低95%、支持10倍增长、显著成本节约
- AI用例规划：特征存储缓存、模型训练数据源、通知服务、URL缩短服务
- BYOA（自带账户）模式的价值：数据驻留合规、区域化部署、无缝工程师访问
