# AWS re:Invent 2025 - 构建AI代理工厂技术会议总结

## 会议概述

本次技术会议由AWS高级解决方案架构师Girish和PSA Justin主讲，重点介绍了如何构建一个AI代理工厂系统。该系统能够根据用户请求自动创建单个代理或多代理系统，从原型开发一直到生产部署。会议展示了四个核心概念的实际应用：Strands SDK用于构建代理、MCP（模型上下文协议）用于工具连接、A2A（代理间通信协议）用于多代理协作，以及AgentCore作为生产级运行环境。

演讲者通过现场编码演示了从简单聊天机器人到复杂多代理系统的完整构建过程，包括创建自定义工具、集成MCP服务器、实现代理间通信，以及部署到AgentCore生产环境。会议的亮点是展示了一个能够理解自然语言请求并自动生成、部署代理的"构建代理"，大大简化了代理开发流程。

## 详细时间线与关键要点

### 0:00-10:00 - 会议开场与核心概念介绍
- Girish介绍会议目标：学习四个关键概念构建代理工厂
- 定义AI代理：运行循环的模型加工具集，用于实现特定目标
- 介绍Strands SDK的三大优势：
  - 几行代码即可创建代理
  - 内置常用工具集
  - 原生支持MCP和A2A协议
- 解释MCP作为"通用翻译器"的作用
- 介绍A2A协议的三大功能：代理发现、能力识别、通信语言

### 10:00-20:00 - AgentCore生产环境介绍
- 详细介绍AgentCore的模块化服务架构
- 核心功能包括：
  - 无服务器运行时环境
  - 身份认证与权限管理
  - 短期和长期内存管理
  - 代码解释器功能
  - 安全浏览器工具
  - AgentCore Gateway用于Lambda函数集成
  - 可观测性功能
- 展示代理工厂架构：构建代理通过三个工具创建Lambda函数、网关和代理部署

### 20:00-30:00 - Strands SDK基础编码演示
- 演示三行代码创建简单聊天机器人
- 展示模型选择灵活性：从Claude Sonnet到Nova Premier
- 介绍多模型提供商支持：Anthropic、OpenAI、Ollama
- 演示代理工具集成：
  - http_request用于网络访问
  - use_aws用于AWS服务交互
  - current_time用于时间感知
- 实际创建能够搜索AWS新闻并上传到S3的代理

### 30:00-40:00 - 自定义工具创建与MCP集成
- 演示动态工具创建：复利计算工具
- 展示@tool装饰器的使用方法
- MCP客户端创建演示：
  - 构建解决方案架构师代理
  - 集成AWS文档和定价MCP服务器
  - 生成TCO计算报告
- 回答观众关于上下文窗口管理的技术问题

### 40:00-50:00 - 多代理系统与A2A协议演示
- 构建HR系统多代理架构：
  - HR代理作为A2A客户端（端口8000）
  - 员工代理作为A2A服务器（端口8001）
  - MCP服务器提供员工数据库访问
- 演示机器学习项目员工推荐功能
- 展示AgentCore部署流程：
  - 安装AgentCore CLI
  - 配置代理文件和依赖
  - 自动创建运行时、角色和内存资源
