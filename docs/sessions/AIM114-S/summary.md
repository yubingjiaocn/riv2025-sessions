# AWS re:Invent 2025 - Stripe与AWS合作的代理商务解决方案

## 会议概述 (Session Overview)

本次技术分享会重点介绍了Stripe与AWS合作推出的代理商务(Agentic Commerce)解决方案。演讲者阐述了"代理是新的店面"这一核心观点，强调发现模式正在向GenAI辅助代理转变。会议展示了两种代理商务模式：企业对代理(B2A)和代理对消费者(A2C)，以及AWS和Stripe如何在这两种模式中实现即时、安全、可信的支付体验。

演讲者首先介绍了Stripe作为互联网金融基础设施的定位，强调其不仅仅是支付处理商，而是可编程的资金流动平台。通过与AWS的合作，Stripe为全球400多万客户提供五个九(99.999%)的可用性保障，在刚刚结束的黑色星期五和网络星期一期间处理了近400亿美元的交易，达到六到七个九的可用性。会议还详细介绍了Stripe的支付基础模型，这是世界上首个基于数百亿关键信号训练的支付AI模型。

## 详细时间线与关键要点 (Detailed Timeline)

### 0:00-2:00 开场与Stripe平台介绍
- 提出核心观点：代理是新的店面，发现模式向GenAI辅助代理转变
- Stripe定位为互联网金融基础设施，类似"支付领域的AWS"
- 服务范围涵盖全球支付接受、自动化收入管理、税务计算、市场支持等

### 2:00-5:00 AWS合作伙伴关系与性能指标
- AWS与Stripe合作实现五个九的可用性保障
- 服务400多万客户，具备高度可扩展性和可靠性
- 黑色星期五/网络星期一处理近400亿美元交易，达到六到七个九可用性
- 优化结账套件平均带来12%的营收增长

### 5:00-8:00 支付基础模型技术详解
- 发布世界首个支付基础模型，基于数百亿关键信号训练
- 使用高维向量空间量化不同集群及其相互关系
- 2024年处理1.4万亿美元年度支付量
- 92%的卡片通过该模型已被识别，实现主动欺诈预防

### 8:00-10:00 实际应用案例与效果
- 95%的卡片测试攻击被实时检测，比之前方法提升21%
- 智能争议功能：使用GenAI自动生成争议文档
- DoorDash实施后退款成本降低10%
- Vimeo和Squarespace争议恢复率提升13%

### 10:00-13:00 代理商务模式介绍
- 三种模式：企业对代理(B2A)、代理对消费者(A2C)、代理对代理(A2A)
- 现场调查显示超过一半观众使用过AI购物推荐
- 重点关注支付作为代理商务中不可妥协的信任层
- 与OpenAI、Perplexity、Gemini、Microsoft Copilot等AI公司建立合作

### 13:00-16:00 代理商务协议(ACP)技术实现
- 9月底发布开源Apache-2许可的代理商务协议
- 与OpenAI共同开发，解决人类可读结账向机器可读转换问题
- 创建抽象层，使代理能够实现确定性结果
- 与Google AP2框架的区别：ACP注重实际构建和市场投放

### 16:00-18:00 共享支付令牌与安全保障
- 共享支付令牌作为预算限制机制，防止代理超额消费
- 多层逻辑结构包含支付方法、预算限制和欺诈信号
- 目前在Etsy平台实时运行，正在与Walmart和Shopify实施
- 用户可在ChatGPT中直接完成购买，无需离开聊天界面

### 18:00-20:00 代理对消费者模式与总结
- 介绍Amazon Rufus、Walmart Sparky、Ralph Lauren Ask Ralph等案例
- Stripe与AWS合作提供参考架构，使用Bedrock构建现场购物顾问
- 提供MCP服务器和代理商务SDK，支持直接API调用
- 开放社区开发模式，鼓励开发者参与协议完善
