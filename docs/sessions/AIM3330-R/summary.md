# AWS re:Invent 2025 - 如何确保AI代理安全运行

## 会议概述

本次技术分享由AWS Bedrock AgentCore团队的首席产品经理Sarthak Handa和首席工程师Vivek Bhadauria主讲，重点探讨了AI代理安全控制的关键问题。随着AI代理在企业工作流程中的广泛应用，如何确保代理在执行真实世界操作时保持在预期边界内成为关键挑战。

演讲者介绍了一个分层的代理安全框架，包含运行时隔离、工具访问控制、身份管理、可观测性评估和策略执行等多个层面。通过AgentCore平台的各项功能，企业可以在不牺牲代理自主性的前提下，实现对AI代理的全面安全控制。会议还通过保险代理的实际案例演示了策略执行的具体实现过程。

## 详细时间线与关键要点

### 0:00-5:00 - 会议开场与问题定义
- 介绍演讲者：Sarthak Handa（首席产品经理）和Vivek Bhadauria（首席工程师）
- 核心议题：如何让AI代理远离麻烦，确保安全运行
- AI代理的独特能力：理解用户意图、规划行动、执行多步骤工作流程
- 企业应用场景：IT工单处理自动化、结构化内容生成、决策支持

### 5:00-10:00 - AI代理的风险与挑战
- 代理运行时行为的不可预测性
- 代理可能执行超出预期范围的操作
- 访问和修改敏感数据的风险
- 误解业务规则的可能性
- 分层保护的心理模型：控制代理运行方式、限制允许操作、验证实际行为

### 10:00-15:00 - AgentCore安全框架介绍
- Vivek上台演示现场调研：构建AI代理vs生产部署vs安全控制的现状
- 代理定义：循环中的LLM系统
- AgentCore Runtime提供的强隔离边界
- 每个用户会话的独立内存、文件系统和CPU资源
- 使用Firecracker microVM技术实现隔离

### 15:00-20:00 - 工具管理与访问控制
- 工具策划的重要性：最小权限原则
- AgentCore Gateway作为MCP兼容服务器
- Lambda拦截器实现细粒度访问控制
- 身份管理：AgentCore Identity的三大功能
- 入站认证、出站认证和凭证管理

### 20:00-25:00 - 可观测性与评估
