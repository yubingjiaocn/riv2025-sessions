# AWS re:Invent 2025 会议总结

## 会议概述

本次会议由Tech Mahindra的创新与新兴技术业务负责人Shini主讲，重点探讨了Tech Mahindra与AWS如何通过生成式AI领域的联合解决方案帮助企业降低总体拥有成本(TCO)。会议揭示了一个令人震惊的现实：尽管众多IT公司在生成式AI领域进行了大量概念验证(POC)，但只有5%能够真正转化为生产环境。此外，企业在技术投资方面存在严重浪费，约三分之一的GPU、技术和软件投资未得到有效利用。

为解决这些问题，Tech Mahindra与AWS共同制定了一套将POC转化为生产级应用的原则框架，并推出了TM Orion平台。该平台与Nvidia合作开发，托管在AWS上，提供模型微调、拖拽式代码库创建和智能代理自动化功能。会议通过两个实际案例展示了如何通过AWS服务（如Amazon Lex、AWS Lambda、Claude等）实现业务成果，强调了关注业务结果而非单纯技术实现的重要性。

Tech Mahindra的AI交付策略围绕四个核心领域展开：生产力提升、业务转型、创新和治理。公司不仅专注于AI，还在元宇宙和量子计算领域与AWS展开合作，探索数字孪生和量子计算在欺诈检测、路线优化等场景中的应用。

[03:45 - 06:15] 生产级POC的五大原则
1. 将POC视为完整项目，关注交付成果、数据需求和延迟指标
2. 从POC阶段就融入安全性和治理机制
3. 采用分阶段生产策略，在每个阶段中学习和改进
4. 规划人机协作模式，让AI代理处理大部分工作，逐步减少人工干预
5. 建立评估和基准测试机制，明确预期成果

[09:30 - 13:00] 案例一：美国工程服务公司
- 客户背景：成立于1950年的工程表面开发公司
- 问题：75名客服人员应对大量客户咨询，导致长时间排队，影响生产力
- 解决方案技术栈：
  - Amazon Lex：使用自然语言理解识别客户查询意图
  - AWS Lambda：协调信息流，获取库存、定价等数据
  - Claude (Anthropic模型)：处理复杂用例，将信息转化为人类可读格式
- 成果：自动化50%的客户来电，75名客服人员能够投入更多时间进行销售，额外创造50,000美元收入

[13:00 - 17:30] 案例二：菲律宾金融科技公司
- 客户背景：主营移动支付和汇款业务，2024年拥有9400万用户，市值50亿美元
- 市场环境：菲律宾汇款市场年增长率13.3%（至2030年）
- 问题：账户接管(ATO)欺诈案件解决时间长达7天，严重损害公司信誉
- 解决方案：开发Fraud Sentinel平台
  - 使用AWS Step Functions自动化欺诈检测流程
  - 创建生成式AI聊天机器人，让欺诈分析师用自然业务语言与系统交互
- 成果：ATO案件解决时间从7天缩短至24小时以内，SLA降至1天，客户满意度(CSAT)提升10%

[17:30 - 20:45] 业务成果导向的方法论
- 强调关注业务成果而非技术本身
- 三大关注点：
  1. 客户寻求的业务成果是什么？有时普通自动化方案即可解决问题
  2. 成本经济性：从方案设计阶段就考虑成本，包括token计费等因素
  3. 将POC规划为生产级项目
- 最重要原则：衡量所有工作成果，建立明确的输出目标

[20:45 - 22:30] 问题分析与解决流程
- 与客户讨论时深入理解问题本质及其业务影响
- 案例一：客服人员忙于处理查询而无法进行销售和交叉销售
- 案例二：快速增长市场中的信誉损失风险
- 先确定业务成果，再决定技术方案（AI、生成式AI、RAG等）

[22:30 - 24:00] 大语言模型的选择策略

- 关键洞察：模型质量不仅取决于参数规模，更取决于微调方式和构建方法
