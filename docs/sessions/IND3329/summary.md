# AWS re:Invent 2025 - Cox Automotive 的企业级 AI 代理实施蓝图

## 会议概述

本次会议由 AWS 高级解决方案架构师 Ravi 主持，邀请了 Cox Automotive 企业架构副总裁 Brian-Lloyd Newberry (BLN) 和首席架构师 Tabare Gowon 分享企业级 AI 代理的实施经验。会议重点探讨了如何将 AI 代理从原型阶段成功推进到生产环境并实现规模化运营。

Cox Automotive 作为汽车行业的主导软件提供商，拥有 autotrader.com、kellybluebook.com 等知名消费者门户网站，每天处理数万亿车辆洞察数据和数亿客户交互。该公司已在生产环境中部署了 150 多个 AI 模型，并在 2024 年成功启动了一项雄心勃勃的挑战：在 5 周内将 5 个代理 AI 产品推向市场。

## 详细时间线与关键要点

### 0:00-5:00 - 开场与挑战定义
- Ravi 介绍 Amazon Bedrock Agent Core 的核心价值
- 现场调研显示：许多组织在实验 AI 代理，但很少有成功部署到生产环境
- 定义 AI 代理的五大生产化挑战：运行时、内存管理、安全性、工具发现、可观测性

### 5:00-10:00 - Agent Core 解决方案架构
- 通过服务技师助手案例说明复杂的多模态交互场景
- Agent Core 提供五大完全托管服务：Runtime、Memory、Identity、Gateway、Observability
- 两个专用工具：Browser 和 Code Interpreter
- 支持多模态输入（最大 100MB 负载）和长时间会话（最长 8 小时）

### 10:00-20:00 - Cox Automotive 公司背景与 AI 历程
- BLN 介绍 Cox Automotive 的市场地位和数据规模
- 2018 年全面迁移至 AWS，从 50 个数据中心缩减至 3 个
- 2023 年 3 月全面投入生成式 AI，一年内推出三个主要产品
- 其中一个产品每年节省 75 万美元，另一个产品将客户响应率提升 50%

### 20:00-25:00 - "疯狂实验"的启动
- 2024 年 7 月底，首席产品官提出挑战：劳动节前推出 5 个代理 AI 产品
- 面临团队零经验、时间紧迫的"空白页问题"
- 成功要素：明确目标、设定截止日期、营造兴奋氛围

### 25:00-30:00 - 技术架构决策
- 选择 Amazon Bedrock 和 Agent Core 作为基础设施
- 决定使用 Strands 框架（AWS 支持的开源产品）
- 强调专注性的重要性：所有团队使用同一框架以集中学习

### 30:00-35:00 - Strands 框架实践
- 代理开发的核心要素：编写提示文本、配置参数、连接模型和工具
- 通过简单的 Python 配置实现复杂功能
- 重点关注功能特性而非框架和基础设施

### 35:00-40:00 - 实际产品案例：虚拟客服助手
- Tabare 介绍从 Predictive Insights 到全自动虚拟助手的演进
- 解决经销商下班后无人响应客户咨询的痛点
- 多代理架构：编排器 + 销售/服务子代理的协作模式

### 40:00-43:00 - 安全与评估最佳实践
- 红队测试：主动尝试让系统偏离轨道
- 双重护栏策略：硬护栏（完全阻止）+ 软护栏（温和引导）
- 自动化评估：使用 LLM 作为评判者，跟踪相关性、完整性、语调等指标
- 熔断器机制：设置成本和轮次限制，优雅失败

### 43:00-43:42 - 项目成果与经验总结
- 5 个项目中 3 个已投入生产，1 个即将上线，1 个重新设计
- 核心经验：立即行动、颠覆性思考、超越聊天机器人的想象
- 行动建议：今天就开始红队测试、为最坏情况设计、设置硬性限制
- 强调"第一天"哲学：持续创新，避免停滞
