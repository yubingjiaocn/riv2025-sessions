# AWS re:Invent 2025 会议总结：使用 Amazon Redshift 实现分析数据仓库现代化

## 会议概述

本次会议主题为"使用 Amazon Redshift 实现分析数据仓库现代化"，由 Amazon Redshift 团队的首席产品经理 Manin Goyel、首席解决方案架构师 Sat 以及来自罗氏制药的 Yanik Mistelli 共同主讲。

会议重点探讨了现代数据仓库如何帮助企业打破数据孤岛、整合多源数据、支持多种格式分析，并最终为业务带来变革性价值。演讲者分享了当前数据分析领域的关键趋势，特别是生成式 AI 和代理式 AI 的战略重要性。根据哈佛商业评论的调查，超过 83% 的数据和分析领导者认为生成式 AI 是其组织的关键战略举措，但超过 50% 的企业仍面临数据基础设施方面的挑战。

Amazon Redshift 作为现代数据仓库平台，提供了存储与计算分离、湖仓一体架构、多仓库架构等核心能力，并通过 Zero ETL、MCP 服务器等创新功能，帮助企业简化数据管道、实现实时分析，并支持代理式 AI 应用场景。会议还展示了 Charter Communications 通过迁移到 Redshift 实现 35% 成本降低和 18% SLA 改进的成功案例，以及罗氏制药整合全球多个数据仓库环境的现代化实践。

### 实时演示 2：Redshift MCP 服务器 (25:00 - 33:00)
- **25:00** - 第二个挑战：营销分析师依赖 IT 团队运行查询
- **25:30** - MCP（Model Context Protocol）介绍：LLM 与应用通信的标准化协议
- **26:00** - Redshift MCP 服务器于 2025 年 6 月发布
- **26:30** - 工作流程：自然语言提示 → Bedrock LLM → 识别工具 → MCP 客户端 → Redshift
- **27:00** - 演示开始：Amazon Q 作为 MCP 客户端
- **27:30** - 查询可用工具：execute_query、list_databases、list_schemas、list_tables
- **28:00** - 自然语言查询："显示所有可用的 Redshift 集群"
- **28:30** - LLM 调用 list_clusters 工具，返回两个集群信息
- **29:00** - 查询数据库和表："analytics 集群中有哪些数据库和表?"
- **29:30** - LLM 依次调用 list_databases、list_schemas、list_tables
- **30:00** - 查询表结构："显示 customers 和 orders 表的结构"
- **30:30** - LLM 调用 list_columns 工具两次
- **31:00** - 业务查询："分析客户购买模式，给出前 10 名客户及购买频率"
- **31:30** - LLM 调用 execute_query 两次，自动生成 SQL 查询
- **32:00** - 返回结果和自然语言摘要
- **32:30** - 解决方案总结：Zero ETL 简化数据管道 + MCP 实现自助分析
