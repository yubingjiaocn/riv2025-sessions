# AWS re:Invent 2025 会议总结：通过集成云服务推进行为健康分析

## 会议概述

本次会议聚焦于弗吉尼亚州通过实施现代化企业数据仓库来推进行为健康分析的转型之旅。会议由弗吉尼亚州行为健康与发展服务部（DBHDS）的实施经理Bryant Jenkins主持，邀请了三位技术领导者：弗吉尼亚州首席信息官Bob Osmond、DBHDS首席信息官Russell Accashian，以及德勤数字健康领导者Don Hoag。

该项目旨在通过无服务器事件驱动架构实现数据管理的现代化，提高数据质量，实现跨多个健康系统的个人关联，并优化为弱势群体服务的关键项目的结果报告。弗吉尼亚州作为一个中等规模的州，服务于870万公民和67个不同的机构，面临着数据孤岛、手工流程繁重、数据透明度缺失等挑战。

## 详细时间线与关键要点

### 0:00-10:00 开场介绍与弗吉尼亚州IT概况
- 会议开场，介绍演讲嘉宾和议程安排
- Bob Osmond介绍弗吉尼亚州IT基础设施：
  - 服务870万公民，管理67个机构
  - 运营1700个不同地点
  - 采用共享运营模式，在基础设施层面标准化
  - 四大工作主题：基础设施、网络安全、治理、采购

### 10:00-20:00 云转型历程与挑战分析
- 弗吉尼亚州云转型分为两个阶段：
  - 2018-2022年："虚假云"阶段，仅将云作为私有数据中心使用
  - 2022年后：真正拥抱云计算，利用无服务器架构
- AWS选择原因：成熟度高、选择丰富（从50种计算选项增至450种）
- Russell Accashian分享DBHDS面临的挑战：
  - 过时的架构和孤岛式运营
  - 12个设施、40个社区服务委员会缺乏数据集成
  - 手工流程导致数据延迟数天至数月
  - 数据质量差，缺乏透明度

### 20:00-30:00 解决方案架构与创新特点
- Don Hoag详细介绍数据仓库平台的三大支柱：
  - 信息速度：实现实时API连接，摆脱批处理依赖
  - 准确完整的数据：通过HL7标准获取护理点完整信息
  - 联邦合规：满足SAMHSA等组织的报告要求
- 技术架构特点：
  - 尽可能使用无服务器技术
  - 使用Glue实现数据透明度和元数据管理
  - 集成主数据管理服务和实体解析服务
  - 实现纵向记录视图，支持多年历史数据分析

### 30:00-40:00 团队协作与项目管理
- 多学科团队协作模式：
  - 敏捷开发方法，持续迭代和回顾
  - 联邦合规、数据治理、安全策略等多方面专家参与
  - AWS架构审查和FinOps结构确保预算对齐
- Russell强调私营部门与政府部门协作的差异
- 实现了12个设施的实时床位管理和入院出院监控

### 40:00-50:00 工具选择、AI应用与未来规划
- 工具选择策略：
  - 收入差异化领域自主开发
  - 标准功能使用现有解决方案（如Mirth、Snowflake）
  - 重点使用AWS原生服务和事件驱动架构
- AI应用前景：
  - 使用AI支持HL7标准演进的编码过程
  - 生成护理连续性警报，实现主动干预
  - 成本预测和预算预测
  - 与本地大学合作开发代理AI改进元数据管理
- 人才培养：与大学合作编写课程，建立招聘渠道

### 50:00-52:48 问答环节与总结
- 观众提问涵盖数据共享合规性、社会健康决定因素、911系统集成等话题
- 强调数据安全的重要性，遵循HIPAA、CJIS、FERPA等法规
- 介绍数据清洁室、数据匿名化、合成数据生成等安全共享机制
- 会议在热烈掌声中结束，演讲者表示愿意会后继续交流
