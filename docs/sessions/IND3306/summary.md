# AWS re:Invent 2025 技术会议总结

## 会议概述

本次AWS re:Invent 2025第一天的技术分享会议聚焦于从分散系统到统一数据平台的转型，以加速和改进机器学习与AI体验。会议由AWS架构师主持，邀请了Rocket Companies的数据副总裁Garima Sharma和工程总监Ilia Fisher分享实际转型经验。

Rocket Companies作为美国最大的抵押贷款提供商之一，服务着全美六分之一的抵押贷款业务，面临着数百个系统和数千个流程产生的数据碎片化挑战。会议深入探讨了如何通过建立统一的数据基础设施，将传统需要数周处理的抵押贷款申请缩短至8分钟内完成，并实现了210个机器学习模型的全自动化运行。

## 详细时间线与关键要点

### 0:00-5:00 开场与问题定义
- 主持人分享旅行经历，引出数据碎片化问题的类比
- 提出核心问题：当数据增长速度超过使用速度时会发生什么
- 阐述企业普遍面临的挑战：数据积累速度超过团队整理能力

### 5:00-10:00 数据碎片化成因分析
- 解释组织创新过程中碎片化的自然产生
- 三大组织目标：创新、快速扩展、缩短上市时间
- 每个新产品发布都带来独立的数据架构和管道
- 现场调研：询问听众是否面临类似挑战

### 10:00-15:00 碎片化影响与Rocket背景介绍
- 碎片化对下游系统的影响
- AI准确性受数据质量影响的重要性
- 介绍Rocket Companies：美国最大抵押贷款提供商之一
- Garima Sharma登台分享转型经验

### 15:00-25:00 Rocket的创新历程与挑战识别
- Rocket的创新里程碑：1998年首个在线抵押贷款、2015年全数字化抵押贷款
- 2019年实现全美50州电子结算，2023年宣布AI驱动的房屋所有权策略
- 识别三大数据挑战：数据孤岛和重复、数据发现困难、大规模利用困难
- 决策：迁移至统一数据湖，采用开放表格格式

### 25:00-35:00 统一数据基础架构设计
- 数据规模：从10+PB增长至30+PB
- 三步实施策略：数据注入、自动化、安全性
- 采用S3、EMR、Glue和Parquet格式
- 创建Customer 360、Mortgage 360、Transaction 360统一视图

### 35:00-45:00 Ilia Fisher技术深度分享
- 现代数据平台三层架构：摄取层、处理层、消费层
- 技术栈详解：API Gateway、Kubernetes、Kinesis、EMR、Glue
- 数据湖三区域：原始数据、处理数据、符合业务标准数据
- MLOps和DevOps自动化：210个机器学习模型零人工干预运行

### 45:00-49:30 生成式AI应用演示与总结
- 现场演示基于统一数据平台的AI代理应用
- 展示自然语言查询、数据分析和报告生成功能
- 强调数据来源的可信度和一致性
- 总结三大成功模式：聚合、策划、API化
- 会议结束，提供后续联系方式
