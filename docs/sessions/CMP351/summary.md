# AWS re:Invent 2025 性能测试专场总结

## 会议概述

本次会议由AWS基础设施解决方案负责人Luis主讲，重点介绍了如何构建可靠和高效的应用程序和系统。Luis首先澄清了三个经常被混淆的核心概念：弹性(Resiliency)、可靠性(Reliability)和效率(Efficiency)。弹性是指系统从故障中恢复的能力，可靠性是指预防故障发生的所有措施，而效率则是指在满足服务级别目标(SLO)的前提下以最少的资源消耗完成任务。演讲者强调，在可靠性上投入越多，就越不需要担心弹性问题。

会议的核心内容聚焦于性能测试的重要性和实施方法。Luis详细介绍了六种不同类型的性能测试：负载测试(Load Testing)、压力测试(Stress Testing)、耐久性测试(Endurance/Soak Testing)、可扩展性测试(Scalability Testing)、峰值测试(Spike Testing)和容量测试(Volume Testing)。他特别强调性能测试不应该只在产品发布前进行一次，而应该成为日常开发和运维流程中的常规实践。最后，Luis演示了AWS的开源解决方案"Distributed Load Testing on AWS"，这是一个可以在5分钟内部署完成的单租户性能测试平台，支持JMeter、K6和Locust三种主流测试框架，并能够从全球多个区域同时进行分布式负载测试。

## 详细时间线

00:00 - 开场与自我介绍
- 演讲者Luis介绍自己是AWS基础设施解决方案负责人，来自迈阿密
- 询问现场观众背景：开发者和云架构师

01:30 - 核心概念定义
- 弹性(Resiliency)：从崩溃中恢复的能力
- 可靠性(Reliability)：预防故障发生的所有措施
- 效率(Efficiency)：以最少资源消耗达成目标的能力

03:00 - 性能测试定义
- 使用汽车类比解释性能测试概念
- 性能测试是理解系统在不同负载条件下行为表现的能力

04:30 - 六种性能测试类型详解
- 负载测试：测试预期负载（如100万用户）
- 压力测试：找到性能降级点或突破点，或测试2倍、5倍、10倍负载能力
- 耐久性测试：检测长时间运行下的问题（如内存泄漏）
- 可扩展性测试：验证从零到目标负载的扩展能力
- 峰值测试：测试突然的负载变化（如促销活动从0到100万用户在1分钟内）
- 容量测试：测试大量数据摄入或读取时的系统行为

09:00 - 性能测试的重要性
- 可以自信地扩展和部署
- 提高可靠性和效率
- 提供更好的用户体验（响应时间应在3秒以内）
- 成本优化：通过测试找到最佳配置（small/medium/large实例选择）

11:30 - 何时进行性能测试
- 重大事件前
- 单元测试阶段
- 基础设施变更时（如从x86迁移到Graviton）
- 建议：尽早测试、事后测试、持续测试
- 建立基线并每日运行，监控5-10%的性能偏差

14:00 - 测量指标
- 百分位数(Percentiles)：建议关注95%和99%而非100%
- 每分钟/每秒事务数
- 带宽
- 延迟：特别强调全球多区域测试的重要性
- 错误率
- 资源使用情况：CPU、内存等

16:30 - 主流性能测试框架
- JMeter：支持Java、XML、YAML
- K6 (Grafana)：支持JavaScript
- Locust：支持Python

17:30 - AWS分布式负载测试解决方案介绍
- 开源解决方案，由AWS完全支持
- 单租户架构，部署在用户自己的环境中
- 通过CloudFormation模板5分钟内即可部署完成
- 版本4.0.1最新发布

19:00 - 现场演示
- 展示Web UI界面和测试配置
- 演示全球体育赛事场景测试案例
- 支持单次运行或定期调度（每日/每周）
- 可通过CI/CD管道和API调用
- 支持单个HTTP端点或JMeter/K6/Locust脚本

21:00 - 多区域测试演示
- 展示4个区域的全球分布式测试配置
- 可配置每个区域的独立IP地址数量和并发用户数
- 演示从0到目标负载在1分钟内的爬坡测试
- 系统自动在全球区域部署基础设施、执行测试并清理资源

23:30 - 实时测试结果展示
- 显示平均响应时间、成功请求数和错误数
- 演示自动扩展场景中的错误分析
- 展示负载爬坡曲线和持续负载阶段

25:00 - 总结与互动邀请
- 鼓励观众部署并测试该解决方案
- 邀请观众参加AWS村的深度演示（下午3-6点）
