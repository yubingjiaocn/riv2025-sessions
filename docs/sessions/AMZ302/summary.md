# Amazon Leo：基于AWS构建的低地球轨道卫星系统

## 会议概述

本次AWS re:Invent 2025分组会议由Gary和Andy主讲，详细介绍了Amazon Leo（原Project Kuiper）低地球轨道卫星系统的构建过程。Amazon Leo是Amazon开发的全球卫星互联网服务，旨在为全球消费者、企业和政府用户提供可靠的网络连接。

会议重点强调了Amazon作为AWS客户的独特地位——他们使用与其他AWS客户相同的服务和资源，没有特殊待遇。这意味着AWS客户能够获得与Amazon相同的技术能力和服务质量。通过Amazon在Prime Day等大型活动中推动AWS服务的极限（如8.9万亿次GuardDuty日志事件、1.5千万亿次ElastiCache请求），所有AWS客户都能从这些技术突破中受益。

演讲分为两个主要部分：首先详细介绍Amazon Leo如何利用AWS服务构建其卫星系统，然后展示客户如何在自己的AWS账户中利用Amazon Leo服务扩展网络能力到全球任何地点。

## 详细时间线与关键要点

### 0:00-5:00 会议开场与背景介绍
- Gary和Andy介绍会议主题：Amazon Leo低地球轨道卫星系统
- 强调Amazon作为AWS普通客户的地位，使用相同的AWS服务
- 展示Prime Day数据：8.9万亿GuardDuty日志事件、1.5千万亿ElastiCache请求
- 播放卫星发射视频，展示Kuiper卫星成功部署

### 5:00-15:00 Amazon Leo系统架构概述
- Andy介绍Kuiper系统的5个"不可能"问题
- 系统架构类似蜂窝网络：客户终端→卫星→地面网关→网络核心→互联网
- 卫星设计：类似太空中的数据中心机架，使用商用以太网交换机和Linux系统
- 自研RF调制解调器ASIC芯片，基于ARM指令集
- 卫星运行在600公里低地球轨道，速度17,000英里/小时

### 15:00-25:00 卫星仿真与测试系统
- 构建基于物理的游戏引擎进行卫星仿真
- 使用AWS扩展仿真能力：API Gateway、Step Functions、EC2实例
- 在Graviton实例上运行未修改的飞行代码（ARM架构）
- 在带FPGA的裸机实例上运行FPGA代码
- 仿真结果与实际太空运行几乎完全一致

### 25:00-35:00 卫星控制与通信系统
- 太空IoT系统：必须始终能够与数百颗卫星通信
- 优化低吞吐量通信，基于预测状态进行通信
- 构建低地球轨道所有物体的精确模型（数万个物体）
- 使用无服务器应用程序构建轨道预测服务
- 通过索引查询系统优化卫星-地面站通信窗口

### 35:00-45:00 网络规划与优化
- 3200颗卫星组成的星座网络
- 每30-40秒进行卫星切换（handoff）
- 服务全球160万个六边形区域
- 大规模优化问题：数百万种连接组合
- 使用SageMaker Studio管理订户数据
- 通过Managed Kafka处理实时网络状态数据
- 在Fargate实例上运行分布式网络规划计算

### 45:00-51:00 客户应用场景与AWS集成
- Gary介绍Leo的客户使用场景
- 客户虚拟网络（CVN）概念：每个天线对应一个虚拟网络接口
- 三种网关类型：公共互联网、Direct to AWS (D2A)、私有网络互连
- 通过IAM临时委托实现Leo与AWS账户的无缝集成
- 应用案例：
  - 海上风电场监控（IoT Core + TimeStream + Grafana + EKS）
  - 远程站点出口流量检查（Outposts + Network Firewall）
  - 远程拍摄现场安全监控（IoT Core + Kinesis Video Streams）

### 安全特性
- 自研芯片内置硬件安全模块，支持千兆级加密/解密
- 端到端加密：从客户终端到网络边缘POP
- 密钥管理：AWS KMS + Nitro Enclaves
- 地面网络使用MACsec加密
- 基于AWS标准加密原语构建
