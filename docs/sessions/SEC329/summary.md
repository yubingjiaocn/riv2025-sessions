# AWS 安全事件响应服务会议总结

## 会议概述

本次会议重点介绍了Infor公司如何通过采用AWS安全事件响应服务(AWS Security Incident Response Service)来转型其安全运营能力。Infor是一家为多个行业提供AI驱动的云应用ERP系统的公司,其平台完全构建在AWS之上,使用数百种AWS服务。会议由AWS安全副总裁Hart和Infor首席信息安全官(CISO)Yoana共同参与,分享了双方合作的成功案例。

通过部署AWS安全事件响应服务,Infor实现了从数千个安全告警中筛选出真正需要关注的事件,将调查时间从数天缩短至数分钟,并将需要人工处理的案例减少到仅1%。更重要的是,该服务帮助Infor的安全团队从繁重的一线分类工作中解放出来,能够专注于更复杂、更有价值的安全问题。会议还展示了AWS如何将Agentic AI集成到事件响应流程中,通过自动化分析和智能推荐进一步提升响应效率。

## 详细时间线与关键要点

### 开场介绍 (00:00 - 02:30)
- **会议主题**: Infor通过AWS安全事件响应服务转型安全运营的旅程
- **演讲嘉宾**: Lindsay(AWS)、Hart(AWS安全副总裁)、Yoana(Infor CISO)
- **议程预告**: 背景介绍、挑战分析、解决方案、服务增强、未来展望

### Infor公司背景 (02:30 - 04:00)
- **业务特点**: 提供行业特定的AI驱动云应用,服务于复杂运营环境
- **客户案例**: 阿斯顿·马丁使用Infor系统定制每辆车的具体配置(如线程颜色)
- **技术架构**: 平台完全构建在AWS上,使用数百种AWS服务
- **业务价值**: 为客户提供独特的业务智能和安全保障

### 面临的三大挑战 (04:00 - 06:30)
- **挑战一**: 如何从大量安全告警中识别真正重要的威胁,做出智能决策
- **挑战二**: 如何让安全团队、托管安全服务提供商和供应商在统一环境中协作
- **挑战三**: 如何维持足够的云安全专业知识(AWS有300多项服务)

### 解决方案架构 (06:30 - 12:00)
- **主动防御策略**: 从响应转向预防,提前介入安全运营
- **数据整合**: 
  - 接入AWS Guard Duty告警
  - 整合第三方工具(Security Hub)
  - 定制连接器接入Microsoft Defender和CrowdStrike数据
- **智能分类**: 
  - 通过Event Bridge将告警发送到AWS环境
  - 自动化分析和丰富告警信息
  - 基于环境学习进行智能分类
- **调查加速**: 从数天缩短到数分钟完成调查
- **精准升级**: 只将真正需要关注的事件通知客户

### DNS数据泄露检测案例演示 (12:00 - 16:00)
- **攻击模式**: DNS数据泄露是Guard Duty常见告警,但并非都是恶意行为
- **处理流程**:
  1. Guard Duty检测到异常并生成告警
  2. 通过Event Bridge发送到AWS安全事件响应服务
  3. 结合EDR解决方案进行跨源分析
  4. 应用自动化逻辑和响应者智能判断
  5. 人工审查无法自动判定的告警
  6. 创建抑制规则或升级给客户
- **情境感知**: 系统能识别渗透测试、并购活动等预期行为

### 服务增强功能 (16:00 - 18:30)
- **Agentic AI集成**: 开启案例时自动运行AI代理,提供关键建议和日志提取
- **灵活定价**: 从固定入口改为分层定价,随业务规模调整
- **通信过滤**: 允许客户过滤不需要的自动通知
- **合规认证**: 获得HiTrust认证,持续增加新认证
- **ITSM集成**: 支持与现有票务管理系统集成
- **灵活部署**: 支持组织单元(OU)级别的独立部署

### 实施成果 (18:30 - 21:00)
- **规模**: 管理大量AWS账户、端点和应用程序
- **效率提升**: 大幅节省安全运营团队的时间
- **精准度**: 仅1%的告警需要调查,0.3%需要升级
- **误报率**: 升级的案例都是预期行为(渗透测试、演练等)
- **团队价值**: 安全团队可专注于更复杂的问题

### AI代理深度应用 (21:00 - 25:00)
- **技术背景**: 应对日益复杂和快速的攻击模式,AI成为必要手段
- **自动化能力**:
  - 开启案例后立即自动运行
  - 分析CloudTrail日志
  - 识别与攻击模式相关的环境区域
  - 主动发现未明确指出的潜在问题
  - 提供调查建议和起点
- **默认启用**: 无需额外配置,所有客户自动获得
- **实战验证**: 在Infor的实际案例中取得显著效果

### 渗透测试协作案例 (25:00 - 27:30)
- **Yoana的初期焦虑**: 不确定会发现什么问题
- **实际结果**: 环境非常干净,发现的都是渗透测试活动
- **响应速度**: 周六收到通知后立即确认环境安全
- **检测能力**: 成功识别多次渗透测试活动
- **快速确认**: 几秒钟内完成全面分析并给出结论

### 合作伙伴生态系统 (27:30 - 32:00)
- **团队协作理念**: 安全是团队运动,不只是AWS和Infor
- **多方协作**: 
  - CrowdStrike提供MDR和事件响应
  - Microsoft Defender提供端点保护
  - AWS提供云原生安全能力
- **数据融合**: 
  - AWS覆盖互联网流量的三分之一(2019年数据)
  - 不同工具从不同视角观察
  - 综合分析提供更高可信度
- **客户价值传递**: Infor向其客户和合作伙伴传递这种安全生态系统的价值

### 业务沟通策略 (32:00 - 35:00)
- **受众特点**: ERP客户通常是项目经理、业务运营人员和CFO,而非安全人员
- **沟通挑战**: 需要向非技术受众解释安全价值
- **信息来源**: 董事会和高管从新闻、审计和监管要求了解安全
- **价值主张**: 强调"顶级安全公司"(AWS、CrowdStrike、Defender)的实时监控和响应能力
- **业务影响**: 帮助客户理解"这会发生在我们身上吗?"的问题

### Amazon Q在事件响应中的应用 (35:00 - 38:00)
- **实际案例**: Lindsay使用Q分析客户的威胁指标(IoC)
- **功能演示**:
  - 搜索特定IoC在环境中的所有出现
  - 生成IoC行为的完整总结
  - 提供跨案例的关联分析
- **决策支持**: 帮助团队快速决定是否需要广泛阻断
- **MCP服务器**: 可自定义MCP服务器连接内部票务系统
- **扩展性**: 客户可以构建自己的MCP服务器扩展Q的能力

### 未来展望讨论 (38:00 - 43:00)
- **2030愿景问题**: 多方安全合作如何为未来做准备
- **Lindsay的观点**:
  - 威胁行为者之间相互协作
  - 安全团队必须同样协作
  - 定期测试合作伙伴间的信息流转
  - 无法用今天的方式做2030年的安全
  - 需要考虑大规模场景(如2万个账户的事件响应)
- **安全角色转变**:
  - 从后台职能转为核心业务流程
  - 需要在业务决策中占有一席之地
  - 必须提前了解新技术(OpenSearch、Bedrock等)
  - 从"阻碍者"转变为"推动者"

### Yoana的未来愿景 (43:00 - 45:30)
- **"不再关注徽章"**: 团队协作超越组织边界
- **消除传统任务**: 希望不再需要扫描、密码管理等重复性工作
- **类比**: 当前的安全流程像小学生罚抄"我不会在课堂上睡觉"
- **安全编码化**: 将安全内置到系统中,而非事后检查
- **业务赋能**: 帮助公司更高效地赚钱,同时保持安全
- **自动化愿景**: 通过AI和自动化消除所有繁琐任务

### 会议总结
本次会议展示了AWS安全事件响应服务如何通过智能自动化、AI代理和生态系统协作,帮助大型企业客户实现安全运营的根本性转型,从被动响应转向主动防御,从人工分类转向智能决策,最终让安全团队能够专注于真正重要的战略性安全工作。
