# AWS re:Invent 2025 Smartsheet AI项目组合管理会议总结

## 会议概述

本次会议由Smartsheet产品总监Olen Ronning主讲，重点介绍了如何通过AI技术提升项目和组合管理的可视性。Ronning分享了Smartsheet正在开发的"智能工作管理"平台，该平台将AI深度集成到项目管理的整个生命周期中。

会议探讨了当前项目组合管理面临的核心挑战，包括缺乏可视性、手动流程过多以及人力资源容量限制等问题。Smartsheet提出通过代理AI(Agentic AI)来扩展工作团队能力，实现从洞察到行动的自动化转换，并通过AI驱动的工作流程自动化来提升生产力。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与问题定义
- 演讲者自我介绍：Olen Ronning，Smartsheet产品总监，6年产品经验
- 提出AI概念：将AI比作"思维的光轮摩托"，比传统"思维自行车"更快更远
- 识别项目组合管理痛点：缺乏可视性、手动流程繁重、资源容量限制
- 现场调研：了解听众在项目组合管理中遇到的具体挑战

### 5:00-10:00 解决方案框架
- 介绍"智能工作管理"概念：基于AI的协作工作管理演进
- 强调数据基础的重要性：需要可靠的共享真实数据源
- 知识图谱构建：分析人员、内容和工作之间的连接关系
- 现有AI功能展示：自然语言公式生成、图表创建、仪表板转换

### 10:00-15:00 端到端生命周期AI应用
- 项目规划自动化：从项目简介到完整项目计划的分钟级生成
- AI驱动的列处理：情感分析、摘要提取等LLM功能
- 场景规划功能：实时资源容量分析和多场景对比
- Smart Assist副驾驶体验：风险分析、活动摘要、执行报告生成

### 15:00-20:00 代理AI和自动化监控
- 自动化更新收集：从Slack/Teams聚合每日站会信息
- 智能代理监控：后台持续风险监控和预警系统
- RAID日志自动维护：风险、行动、决策的自动记录
- 持续学习改进：基于历史数据的模式分析和最佳实践推荐

### 20:00-22:00 AWS技术架构与总结
- AWS基础设施：Amazon Neptune知识图谱、PostgreSQL对话历史、数据湖聚合
- 代理工具层：MCP服务器支持外部AI服务访问
- Amazon Bedrock和Claude模型：代理编排和专业化子代理
- 负责任AI层：Amazon GuardRails、OpenSearch审计、管理控制台
- 未来愿景：从小时级规划到分钟级执行，实现持续学习的良性循环
