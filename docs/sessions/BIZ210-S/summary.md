# AWS re:Invent 2025 分会场总结：利用AI提升项目和组合管理可见性

## 会议概述

本次分会场由Smartsheet产品总监Ola Ronning主讲，重点介绍了如何将人工智能技术应用于项目和组合管理，以提升工作可见性和效率。演讲者首先分享了自己在Smartsheet六年的工作经验，并用"AI是思维的光速摩托"这一比喻来说明AI相比传统计算机工具的巨大提升潜力。

Ola详细阐述了当前项目组合管理面临的核心挑战，包括缺乏可见性、手动流程繁重、以及资源容量管理困难等问题。他指出，许多项目经理需要同时管理数百个项目，这使得传统的手动跟踪方式难以为继。Smartsheet提出的解决方案是通过"智能工作管理"（Intelligent Work Management）平台，将AI能力深度集成到项目管理的全生命周期中，从战略规划、优先级排序、执行监控到交付报告，实现端到端的智能化管理。

演讲的技术部分重点介绍了Smartsheet如何利用AWS服务构建AI驱动的项目管理平台。该平台基于知识图谱、AI代理和负责任AI治理层构建，能够自动识别风险、生成项目计划、优化资源分配，并通过持续学习不断改进管理流程。这种方法不仅能将项目规划时间从数周缩短到数分钟，还能让项目经理的工作效率成倍提升。

## 详细时间线

[00:00 - 01:30] 开场与演讲者介绍
- Ola Ronning自我介绍，担任Smartsheet产品总监已有6年
- 分享个人背景：计算机科学和工业设计复合背景
- 展示今年万圣节的Tron主题设计作品

[01:30 - 03:00] AI与项目管理的愿景
- 引用Steve Jobs的"计算机是思维的自行车"名言
- 提出AI是"思维的光速摩托"概念，能够更快更远地推进工作
- 询问现场观众中有多少人在管理项目或组合

[03:00 - 05:30] 当前项目组合管理的挑战
- 展示项目管理中常见的痛点清单
- 强调"看不见就无法管理"的核心问题
- 讨论如何利用AI和工作管理工具更快速、更主动地获取洞察

[05:30 - 08:00] 可见性和流程挑战
- 缺乏可见性：工作状态、人员分配、资源可用性不清晰
- 数据不及时导致风险遗漏和延迟响应
- 手动流程繁重：审批、分类等需要大量人工监督
- 容量限制：举例说明3名项目经理管理200个项目的极端情况

[08:00 - 10:00] AI解决方案的机会
- 使用Agentic AI扩展劳动力规模
- 自动化后台流程，让项目经理能够管理更大规模的组合
- 从洞察到行动：AI不仅提供分析，还能代表用户执行操作

[10:00 - 12:00] 智能工作管理平台介绍
- 基础是数据：需要可靠的共享真实数据源
- 构建知识图谱分析人员、内容和工作之间的关系
- 为AI提供强大的上下文支持

[12:00 - 14:30] 现有AI功能展示
- 自然语言生成公式计算预算和风险
- 通过提问生成图表并转换为仪表板
- 未来方向：将AI作为贯穿整个应用生命周期的基础能力

[14:30 - 17:00] 项目规划与生成
- 文档提取和处理：从项目简报或工作说明书自动生成项目计划
- 结合模板和最佳实践指南
- 将数周的规划工作缩短到几分钟
- 该功能的早期版本已在试用体验中上线

[17:00 - 19:00] 优先级排序与AI列
- AI驱动的列处理：使用LLM直接在表格中处理数据
- 支持情感分析、摘要提取等细微分析
- 自动对组合中的每一行执行LLM提示功能

[19:00 - 21:30] 场景规划与容量管理
- 场景规划功能（目前处于Beta测试）
- 组合级视图：查看所有资源和项目
- 实时查看工作调整对容量的影响
- AI生成不同优化场景：平衡容量、优化成本、及时交付

[21:30 - 24:00] Smart Assist智能助手
- AI协同驾驶体验，专为项目组合管理调优
- 功能包括：风险分析、近期活动摘要、执行进度报告
- 可在几秒钟内跨多个项目完成分析
- 计划扩展到组合级视图

[24:00 - 26:00] 自动化更新收集
- 从Slack或Teams自动聚合人员更新
- 总结更新内容，突出显示风险
- 自动发送摘要更新，消除繁琐的信息收集工作
- 构建每日站会报告

[26:00 - 29:00] AI代理与自动监控
- 将自动化操作与风险分析结合
- 代理在后台持续监控项目风险
- 演示：在Microsoft Teams中接收资源风险警报
- 代理提供选项、重新分配工作并代表用户执行操作

[29:00 - 31:00] RAID日志自动维护
- RAID日志：风险、行动、问题、决策
- 代理自动跟踪项目变更：日期变动、风险发生、采取的行动
- 支持持续学习和改进

[31:00 - 33:00] 持续改进与模式分析
- 分析组合中的模式和趋势
- 基于最佳实践和类似客户经验提供建议
- 能够立即将改进部署到进行中的项目

[33:00 - 35:00] 完整项目生命周期视图
- 从战略设定到执行再到交付的完整流程
- AI和代理为每个阶段提供支持
- 事后学习反馈到数据中形成良性循环
- 知识图谱不断增强，提供更精准的业务定制洞察

[35:00 - 38:30] AWS技术架构详解
- 数据基础层：
  - Amazon Neptune支持知识图谱
  - PostgreSQL数据库存储对话历史和长期记忆
  - 数据湖聚合所有数据
- 代理工具层：产品内工具和MCP服务器外部访问
- 代理编排层：使用Amazon Bedrock和Claude模型，通过Anthropic SDK构建专业子代理

[38:30 - 40:00] 负责任AI治理层
- Guardrails管理输入输出审核
- OpenSearch跟踪日志和审计
- 完整的可追溯性和可审计性
- 管理员体验：数据访问控制、用户访问控制、功能启用/禁用

[40:00 - 41:30] 工具生态系统
- 当前Smart Assist体验中已提供多种工具
- 持续添加新工具
- 通过外部MCP服务器提供访问能力

[41:30 - 43:00] 负责任AI四大原则
- 数据隐私和安全：控制权
- 问责制：查看所有操作并治理每个行动
- 透明度和可解释性：理解每个行动及其推荐原因
- 公平合理：确保对所有输入提供公平合理的响应

[43:00 - 44:30] 未来愿景总结
- 项目规划从数小时缩短到数分钟
- 从卡顿流程到智能端到端流程
- 从手动追踪到代理自动更新和响应
- 从现状维持到持续学习的良性循环
