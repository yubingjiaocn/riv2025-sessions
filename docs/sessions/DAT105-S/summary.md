# AWS re:Invent 2025 会议总结：使用智能内存构建弹性生成式AI和智能体应用

## 会议概述

本次技术分享由Cockroach Labs高级经理David Joy主讲，重点探讨了如何使用CockroachDB构建可扩展、弹性的生成式AI和智能体应用。会议深入分析了当前企业在AI应用落地过程中面临的挑战，包括基础设施准备不足、95%的项目停留在POC阶段、以及缺乏可扩展性等问题。

演讲者通过实际演示展示了CockroachDB作为分布式SQL数据库如何支持RAG（检索增强生成）、智能体工作流和记忆系统，特别强调了其在多区域部署、故障恢复和向量数据处理方面的独特优势。会议还包含了一个引人注目的实时演示，展示了在节点故障情况下系统的持续可用性。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与背景设定
- David Joy自我介绍，来自Cockroach Labs，与AWS合作3.5-4年
- 调查现场观众对CockroachDB和数据库的熟悉程度
- 强调会议将提供可实际应用的技术方案和架构

### 5:00-10:00 生成式AI用户行为变化分析
- 对比传统搜索引擎与现代答案引擎的用户行为差异
- 介绍从搜索链接到直接获取答案的转变
- 强调大语言模型已发展为完整的数据生态系统

### 10:00-15:00 企业AI应用现状与挑战
- 70%企业运行传统基础设施，50%的AI项目因基础设施问题受阻
- 95%的AI项目停留在POC阶段，难以投入生产
- 企业面临ROI不确定性和大规模部署的技术挑战
- 强调需要构建支持数百万用户的可扩展、弹性系统

### 15:00-20:00 AI应用的三大核心能力
- **知识访问**：通过RAG技术获取上下文相关的事实数据
- **智能体行动**：使用工具自主做出决策和执行操作
- **学习记忆**：基于用户行为和偏好持续优化响应

### 20:00-25:00 CockroachDB架构介绍
- 对比传统单实例数据库与分布式数据库架构
- 介绍CockroachDB的水平扩展能力和数据分片机制
- 展示多区域、多云部署的灾难恢复能力
- 强调数据本地化和GDPR合规性支持

### 25:00-30:00 RAG应用实时演示
- 展示银行应用中的AI助手功能
- 演示"咖啡消费查询"的完整流程：缓存检查→向量搜索→AI处理→结果返回
- 介绍缓存优化策略，显著提升响应速度并降低成本
- 展示数据库操作日志和向量搜索过程

### 30:00-35:00 多智能体工作流演示
- 展示收据处理的三智能体协作系统：处理智能体、欺诈检测智能体、预算智能体
- 演示OCR处理、欺诈检测和预算分析的并行执行
- 通过重复上传收据展示欺诈检测的记忆功能
- 强调智能体的"思考-行动-记忆"三步骤模式

### 35:00-40:00 系统弹性测试与总结
- 实时演示：在10,000笔交易处理过程中关闭两个数据库节点
- 展示系统在节点故障情况下的零停机时间表现
- 介绍CockroachDB的分布式向量索引技术，解决向量数据存储和搜索的性能问题
- 总结产品特性：量化模型减少94%存储负载，支持按用户和区域的向量索引
- 提供产品试用信息和联系方式
