# AWS re:Invent 2025 交易平台技术分享会总结

## 会议概述

本次分享会由LPL Financial的技术平台负责人Abhishek Sharma主讲，重点介绍了LPL Financial如何构建下一代云原生交易平台。LPL Financial是美国最大的独立券商，管理着2.3万亿美元资产，服务32,000名财务顾问。

会议深入探讨了财富管理行业面临的技术挑战，特别是在当前万亿美元财富代际转移的背景下，新一代客户对个性化、实时响应和智能洞察的需求不断增长。为满足这些期望，LPL正在从传统架构向完全智能化的云原生交易平台转型，该平台不仅要跟上市场变化，更要能够预测和适应市场需求。

演讲者详细介绍了他们的云转型之旅，从2024年开始的渐进式迁移到未来的多区域、多可用区架构设计。重点展示了Client Works Rebalancer这一旗舰产品如何利用AWS服务实现模型化交易，以及AI在市场中心故障检测和ETF分类等场景中的应用。

## 详细时间线与关键要点

### 开场与背景介绍 (0:00-5:00)
- **演讲者介绍**: Abhishek Sharma，LPL Financial交易和投资管理技术平台负责人
- **互动环节**: 通过举手问答了解参会者背景，包括re:Invent参会经验和交易经验
- **感谢AWS**: 强调AWS合作伙伴关系在现代化和构建下一代交易平台中的关键作用

### 财富管理行业概览 (5:00-10:00)
- **行业规模**: 144万亿美元资产管理规模，30万财务顾问，6800万客户
- **代际财富转移**: 100万亿美元资产正从婴儿潮一代转移到更具技术素养的下一代
- **技术挑战**: 新一代客户要求个性化、即时响应和智能洞察，对现有数十年历史的系统构成挑战

### LPL Financial公司介绍 (10:00-15:00)
- **市场地位**: 美国排名第一的券商，财富500强公司
- **业务模式**: 独立券商，为财务顾问提供技术、运营、合规等后台支持
- **核心优势**: 灵活性和规模化，32,000名财务顾问，2.3万亿美元资产管理
- **技术挑战**: 平台需要适应从小企业到大型机构的各种规模需求

### 交易平台技术目标 (15:00-20:00)
- **业务目标**: 零安全事件、100%合规、超高弹性、10倍扩展能力、亚毫秒级性能
- **技术策略**: 利用AWS云提供速度、规模和弹性；通过SageMaker、Bedrock等AI平台注入智能
- **设计原则**: 为峰值而非平均值设计，确保在任何交易量下都能提供机构级弹性

### 云转型之旅 (20:00-30:00)
- **渐进式方法**: 采用迭代增量方式，每年每步都为下一步奠定基础
- **建筑类比**: 如同建造摩天大楼，先建基础，再加强核心，然后添加各种功能
- **2024年重点**: 自动扩展，通过解耦平台架构实现EKS自动扩展，某些场景下实现10倍扩展
- **未来规划**: 从多可用区向多区域多可用区转型，减少本地数据中心依赖

### 当前架构设计 (30:00-40:00)
- **混合架构**: 本地数据中心与多可用区云环境协同工作
- **核心组件**: S3存储、EKS Kubernetes、PostgreSQL、DynamoDB、MemoryDB、Kafka消息代理
- **数据策略**: 根据需求选择不同数据库，关系型用PostgreSQL，文档型用DynamoDB，低延迟用MemoryDB
- **监控体系**: CloudWatch端到端可观测性，自愈能力和合成事务监控

### 下一代架构愿景 (40:00-50:00)
- **去本地化**: 一级关键应用完全移除本地依赖，充分利用云的全部能力
- **多区域设计**: 从主动-被动向主动-主动多区域环境演进
- **数据一致性**: 使用Aurora DB解决多区域写入场景下的数据一致性问题
- **AI集成**: 采用Bedrock配合SageMaker，更好地集成GenAI和LLM到核心工作流

### Client Works Rebalancer深度解析 (50:00-70:00)
- **产品概念**: 基于模型的交易平台，14,000名顾问使用，完成超过7000万笔交易
- **业务价值**: 解决市场变化导致的投资组合偏离问题，自动化重新平衡过程
- **技术架构**: 
  - 编排器将大型重平衡交易分解为小批次
  - Kafka消息代理确保可靠交付
  - EKS集群上的重平衡算法自动扩展
  - 进程内缓存实现200+交易检查的亚毫秒级验证
- **性能优化**: CQRS模式分离读写操作，自适应扩展，每秒处理数千笔交易

### AI应用案例 (70:00-85:00)
- **市场中心故障检测**: 
  - 使用Random Cut Forest时间序列异常检测算法
  - 实时监控交易执行模式，提前发现市场中心问题
  - 避免订单堆积和重复提交的运营噩梦
- **ETF分类自动化**:
  - 使用人工神经网络(ANN)无监督模型
  - 自动分类新ETF以满足监管和合规要求
  - 夜间批处理工作流，人工审核确认

### AI架构实现 (85:00-95:00)
- **训练流程**: RDS Aurora存储历史数据 → Lambda函数数据预处理 → S3存储训练数据 → SageMaker训练和部署模型
- **实时推理**: Lambda函数调用训练好的模型端点，实时处理交易数据流
- **反馈循环**: 模型决策结果反馈到RDS用于未来学习改进

### 未来AI应用规划 (95:00-结束)
- **后台自动化**: 多智能体解决方案处理税务优化重平衡
  - 重平衡智能体: 基于50+参数和历史运行数据训练
  - 验证智能体: 检查生态系统级联依赖和影响
  - 交易智能体: 执行与市场中心的交易
  - 初步研究显示吞吐量提升10倍
- **模型建议系统**: 为账户投资组合提供AI驱动的模型建议
