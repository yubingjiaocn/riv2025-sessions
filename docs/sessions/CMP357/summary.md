# AWS re:Invent 2025 量子计算技术会议总结

## 会议概述

本次闪电演讲由AWS量子计算市场战略团队负责人Michael Brett和网络安全初创企业项目负责人Anna Sonic共同主持,重点介绍了AWS在量子技术领域的布局和进展。会议涵盖了AWS在量子计算方面的四大核心领域:Amazon Braket服务、高级解决方案实验室、AWS量子计算中心以及后量子密码学技术。

Michael Brett详细介绍了量子计算的应用前景,包括药物发现、材料设计、密码学、优化和机器学习等领域。AWS通过Amazon Braket服务提供按需访问的量子计算资源,目前已接入8台来自5家不同供应商的量子计算机,包括离子阱、超导和中性原子等多种技术路线。AWS还在加州理工学院建立了自己的量子计算中心,专注于超导量子比特技术,并在芯片级错误纠正方面取得突破性进展,其Ocelot芯片成功将错误纠正所需资源降低了几个数量级。

Anna Sonic则重点介绍了初创企业在后量子密码学领域的创新,包括Qsecure和Pero等公司如何开发端到端的后量子加密保护技术,帮助企业在现有基础设施中部署这些技术,为未来量子计算可能带来的安全威胁做好准备。AWS目前支持数千家网络安全初创企业在其平台上构建解决方案。

## 详细时间线

开场介绍 (0:00-2:30)
- Michael Brett介绍自己作为AWS量子计算市场战略团队负责人的角色
- 介绍同事Anna Sonic,她负责网络安全初创企业项目
- 说明这是本周闪电演讲剧场参与人数最多的一场会议

量子技术应用前景 (2:30-4:00)
- 介绍量子计算的潜在应用领域:模拟量子物理、药物发现、材料设计
- 讨论量子计算在密码学领域的双重影响:既是潜在威胁也推动新标准发展
- 提及优化和机器学习等其他应用场景

AWS量子技术四大支柱 (4:00-6:30)
- **Amazon Braket服务**:类似EC2的量子计算服务,5年前在re:Invent发布
- **高级解决方案实验室**:专业服务团队,专注量子算法研发
- **AWS量子计算中心**:自主研发量子计算硬件
- **后量子密码学**:保护资产免受未来量子威胁

AWS量子计算中心介绍 (6:30-9:00)
- 位于加州帕萨迪纳的加州理工学院校园内
- 采用超导量子比特技术路线
- 重点研发芯片级错误纠正技术
- 介绍Ocelot芯片的突破性成果:将错误纠正资源需求降低数个数量级

Amazon Braket服务详解 (9:00-12:00)
- 提供一致的用户体验,类似其他AWS服务
- 支持按需付费,无需预付费承诺
- 统一的API和SDK支持多种量子计算机
- 当前主要用户为研究人员:大学、国家实验室等

Braket架构层次 (12:00-15:00)
- **基础设施层**:量子处理单元(QPU)、模拟器、CPU/GPU协同处理
- **软件层**:AWS标准API、SDK、Jupyter Lab笔记本
- **第三方集成**:支持OpenQASM、Qiskit、CUDA-Q等编程语言
- **合作伙伴生态**:如QC Ware等公司提供增值服务和错误纠正软件

量子计算定位 (15:00-17:00)
- 将量子计算视为高性能计算组合中的另一种加速器
- 不会取代现有计算,而是作为补充和增强
- 采用组合策略,将合适的工作负载分配给合适的处理器

当前发展阶段 (17:00-19:00)
- 目前主要应用是研究量子计算机本身
- 正从硬件表征阶段过渡到算法研究阶段
- 下一步将应用于科学发现:高能物理、化学模拟等
- 最终目标是解决行业相关问题

可用量子计算机 (19:00-21:00)
- 目前提供8台来自5家供应商的量子计算机
- **离子阱技术**:包括两周前刚上线的Alpine Quantum Technologies(AQT)设备
- **超导技术**:Rigetti和IQM的设备
- **中性原子技术**:支持模拟编程的量子计算机

研发合作路径 (21:00-23:00)
- 发现用例:咨询潜在应用
- 研究设计:开发算法并进行基准测试
- 突破边界:提升性能和效率
- 奠定基础:建立知识产权、供应链和人才储备

初创企业项目介绍 (23:00-25:00)
- Anna Sonic介绍AWS全球支持数千家网络安全初创企业
- 后量子密码学是今年五大热门话题之一
- 团队负责连接顶级投资者、收集安全从业者反馈、识别优秀创始人

初创企业案例一:Qsecure (25:00-26:30)
- 提供端到端的后量子密码学保护
- 可部署在混合环境:本地和云端
- 帮助企业在现有基础设施中采用后量子技术

初创企业案例二:Pero (26:30-28:00)
- 开发后量子密码技术应对"现在窃取,以后解密"风险
- 为端点开发零信任架构
- 可在现有环境中部署
