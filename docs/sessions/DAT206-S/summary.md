# AWS re:Invent 2025 会议总结：Salesforce Data 360 与 AWS 零拷贝集成

## 会议概述

本次会议由 Salesforce 战略合作伙伴团队的 Natalya Leon、Data 360 平台产品经理 Sid 以及 AWS 解决方案架构师 Bill 共同主讲。会议重点介绍了 Salesforce 与 AWS 如何通过合作帮助企业客户统一数据并在整个企业范围内激活数据，从而提供更好的客户体验并为 AI 应用奠定坚实基础。

核心主题围绕"零拷贝"（Zero Copy）技术展开，这是 Data 360 连接套件中的关键功能。该技术允许企业在不复制数据的情况下，直接访问存储在 AWS 数据仓库和数据湖中的数据，实现实时数据激活。会议强调了在当今客户期望个性化体验的时代，企业必须打破数据孤岛，统一分散在多个系统中的数据，才能满足客户需求并在竞争中脱颖而出。

通过 Salesforce Data 360 与 AWS 服务（如 S3、Glue、Athena、Redshift 等）的深度集成，企业可以实现双向数据流动，既能将 AWS 数据引入 Salesforce 进行客户 360 度视图构建和营销活动，又能将 Salesforce 处理后的数据回传至 AWS 进行进一步分析和 AI 应用。

## 详细时间线与关键要点

### 00:00 - 开场介绍
- **演讲者介绍**：Natalya Leon（Salesforce 战略合作伙伴团队）、Sid（Data 360 产品经理）、Bill（AWS 解决方案架构师）
- **会议议程**：解决方案概述、架构深入探讨、演示展示、客户成功案例

### 01:30 - 客户期望与数据挑战
- **客户期望**：企业客户期望公司了解他们、提供个性化体验、预测需求
- **现实挑战**：企业数据分散在多个系统中形成数据孤岛，导致数据不可靠且难以采取行动
- **Black Friday 购物案例**：演讲者分享个人购物体验，强调个性化服务的重要性

### 04:00 - Salesforce 与 AWS 合作愿景
- **核心目标**：帮助企业统一数据并在整个企业范围内激活
- **四大支柱**：
  - 为 AI Agent 提供必要的模型、灵活性和控制
  - 支持多 Agent 生态系统
  - 提供端到端的信任与合规保障

### 05:30 - Data 360 平台介绍
- **定位**：Data 360 不是数据仓库或数据湖，而是"激活引擎"
- **核心目的**：统一、协调并激活数据资产中的每个数据点
- **三大目标**：
  1. 理解客户
  2. 加速销售增长
  3. 提高生产力

### 07:00 - Data 360 三大构建模块
1. 数据访问：以最适合任务的方式访问数据，优化性能和成本
2. 统一客户档案：创建跨系统的单一客户视图
3. 构建 Agent 世界：为 AI Agent 定义可执行的操作和知识边界

### 09:00 - 数据激活五步流程
1. 连接：通过丰富的连接器套件连接各种数据源
2. 协调：标准化不同模式、解决身份冲突、构建统一档案
3. 治理：创建权限管理和合规策略
4. 激活：通过细分、文档 AI、预测 AI 等服务激活数据
5. 应用：支持 Customer 360 和 Agent Force 360

### 12:00 - Agent Force 与 Data 360 的关系
- **架构基础**：Agent Force 构建在 Data 360 之上
- **三大原因**：
  1. **非结构化数据管道**：文档加载、分块、向量化、RAG 流程
  2. **结构化数据查询**：自然语言转 SQL（NL to SQL）
  3. **Agent 记忆**：确保对话连续性

### 15:00 - 连接套件详解
- **四大类别**：
  1. 非结构化数据管道
  2. 批量摄取（ETL）
  3. 实时数据管道（毫秒级推送）
  4. **零拷贝**（运行时访问，无需持久化外部数据）

### 16:30 - 零拷贝技术核心概念
- **传统方式问题**：需要复制大量数据，导致数据过时、成本高昂
- **零拷贝优势**：
  - 运行时安全访问数据
  - 始终使用最新数据
  - 最大化现有数据产品投资价值
- **使用统计**：自 2024 年推出以来，零拷贝已占 Data 360 流量的 50% 以上

### 19:00 - AWS 与 Salesforce 合作路线图
- **客户痛点**：客户希望数据自由流动，特别是在 AI Agent 时代
- **合作原则**：双方共享"客户至上"的领导原则
- **集成服务**：RDS、Aurora、Athena、S3、Glue、EMR、Redshift 等

### 21:00 - 金融科技客户案例
- **客户背景**：使用多个 Salesforce 产品（Marketing Cloud、Data Cloud、Agent Force）
- **演进路径**：
  1. 最初使用查询联邦（Query Federation）连接 RDS 数据库
  2. 随着数据湖建设，采用文件联邦（File Federation）连接 S3 和 Glue
  3. 通过数据共享（Data Sharing）将 Customer 360 数据回传至 AWS
- **业务价值**：实现营销活动的精准客户定位和数据湖增强

### 24:00 - 查询联邦 vs 文件联邦对比
查询联邦（Query Federation）：
- 使用 JDBC 驱动连接传统数据库
- 每次查询返回完整数据集
- 适用于 RDS、Aurora 等关系型数据库

文件联邦（File Federation）：
- 基于 Iceberg 目录功能
- 支持变更数据捕获（CDC）
- 仅传输增量变更数据
- 更高性能和成本效益
- 支持并行化查询
- 避免双重计费（AWS 查询 + Data 360）
- 2025 年 2 月正式 GA

### 28:00 - 架构详解
- **数据流入**：通过查询联邦和文件联邦将 AWS 数据引入 Data 360
- **数据流出**：通过 Glue 数据目录将 Data 360 数据共享回 AWS
- **双向流动**：支持 Agent Force（Salesforce 侧）和 Agent Core（AWS 侧）的 AI 应用
- **Iceberg 表**：Data 360 内部也使用 Iceberg 表存储

### 31:00 - Lake Formation 安全机制
- **细粒度权限控制**：在 AWS 数据源上应用精细权限
- **短期凭证**：使用 Lake Formation 颁发短期凭证，避免长期密钥泄露
- **SIG V4 签名**：通过 AWS Signature Version 4 实现安全 API 访问
- **最佳实践**：避免使用长期访问密钥，优先使用 IAM 角色

### 34:00 - AWS Glue 核心功能
- **Apache Iceberg 支持**：将 S3 对象转换为表格
- **无服务器架构**：无需管理计算资源
- **高可用性**：基于 S3 的持久性和可扩展性
- **目录集成**：与多种数据目录集成
- **Athena 查询**：通过 Athena 提供额外查询能力

### 36:00 - 演示环节
- **Data 360 设置界面**：展示连接配置
- **支持的 AWS 服务**：
  - Athena（查询引擎）
  - Aurora MySQL（关系型数据库）
  - Amazon RDS（数据库服务）
  - Amazon Redshift（数据仓库）
  - AWS Glue（数据目录和 ETL）

### 38:00 - 总结
- **零拷贝是关键**：Data 360 连接套件的核心功能
- **双向数据流**：实现 Salesforce 与 AWS 之间的无缝数据交换
- **AI 就绪**：为 Agent Force 和 Agent Core 提供统一数据基础
- **客户价值**：降低成本、提高性能、确保数据新鲜度、增强安全性
