# AWS re:Invent 2025 - Salesforce与AWS数据集成解决方案

## 会议概述

本次技术分享会由Salesforce战略合作伙伴团队的Natalia Leon、Data 360平台产品经理Sid以及AWS解决方案架构师Bill共同主讲。会议重点介绍了Salesforce与AWS如何通过零拷贝(Zero Copy)技术帮助企业客户统一数据并激活全企业范围的数据价值，为AI应用奠定坚实基础。

演讲者强调了现代客户对个性化体验的期望，以及企业在数据孤岛问题上面临的挑战。通过Salesforce Data 360平台与AWS服务的深度集成，企业能够在不移动数据的情况下实现实时数据访问和激活，同时保持数据安全性和成本效益。会议包含了详细的技术架构讲解、现场演示以及客户成功案例分享。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与客户期望分析
- Natalia Leon介绍演讲团队和议程安排
- 强调客户对个性化体验的期望：了解客户身份、提供个性化体验、预测客户需求
- 分析企业面临的数据孤岛挑战：数据分散在多个系统中，难以提供统一的客户体验
- 介绍Salesforce与AWS合作解决方案的四大核心价值

### 5:00-15:00 Data 360平台核心功能介绍
- Sid详细介绍Data 360作为Salesforce激活引擎的定位和作用
- 阐述Data 360的三大核心目标：理解客户、加速销售增长、提高工作效率
- 介绍Data 360的三大构建模块：
  - 数据访问：支持多种连接方式，零拷贝技术为核心
  - 客户统一档案：创建跨系统的统一客户视图
  - 智能体世界构建：为AI智能体定义操作规则和能力边界

### 15:00-25:00 数据激活流程与连接套件
- 详细说明数据激活的五步流程：连接、协调、治理、激活、应用
- 介绍丰富的连接套件：支持结构化、半结构化、非结构化数据
- 强调零拷贝技术在连接套件中的核心地位
- 解释数据协调(harmonization)概念：标准化模式、解决身份冲突、构建统一档案
- 介绍治理框架：表级、记录级、列级的细粒度权限控制

### 25:00-35:00 AWS零拷贝技术深度解析
- Bill介绍AWS与Salesforce合作的技术路线图
- 通过金融科技客户案例说明技术演进：从查询联邦到文件联邦再到数据共享
- 详细对比查询联邦与文件联邦的技术差异：
  - 查询联邦：基于JDBC的传统数据库连接
  - 文件联邦：基于Iceberg目录的数据湖访问，支持变更数据流
- 强调文件联邦的优势：并行化处理、成本效益、实时数据访问
