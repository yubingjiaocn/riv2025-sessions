# AWS re:Invent 2025 Oracle Database at AWS 会议总结

## 会议概述

本次会议由 Oracle 数据平台销售组织集团副总裁 Doug Palmer 主讲，重点介绍了 Oracle 与 AWS 之间的战略合作伙伴关系。Palmer 在会上分享了这一合作如何为医疗保健和生命科学领域的大型企业客户解锁巨大的市场机遇。他强调，Oracle Database at AWS 服务的推出是他在 Oracle 14-15 年职业生涯中见证的最大市场机遇之一，这在几年前是难以想象的合作。

会议深入探讨了医疗保健行业面临的核心挑战：过去 50 年的技术演进虽然实现了现代化，但也造成了大量的应用程序和数据孤岛。这些孤岛不仅增加了成本和运营开销，还阻碍了企业充分利用 AI 技术的能力。Palmer 详细阐述了 Oracle Database at AWS 如何通过五大核心场景解决这些问题：从 IaaS 迁移到数据库服务、在云端重新整合应用和数据库、将本地大型工作负载迁移到云端、在超大规模云开发栈中构建应用，以及实现企业级 AI 能力。

Oracle 自 2019 年开创多云概念以来，已经与 Azure、GCP 和 AWS 建立了数据库服务合作。目前 AWS 已有两个区域上线（US East 1 和 US West 2），未来计划扩展到 20 多个区域。这项服务将 Oracle 48 年的企业级数据库技术与 AWS 的云服务无缝集成，为客户提供了前所未有的灵活性和选择权。

00:07:15 - 场景一：从 IaaS 迁移到数据库服务
- 许多客户在过去 5-10 年在原生 IaaS 上部署了 Oracle 数据库
- 现在有机会利用 Oracle Database at AWS 原生服务
- 改善可扩展性、安全性、维护和运营，降低成本

00:08:00 - 场景二和三：重新整合和迁移工作负载
- 许多客户在本地拥有大型 Oracle 环境，但选择 AWS 作为首选超大规模云
- 应用栈与数据库分离导致延迟问题和用户体验不佳
- 现在可以将数据库与应用在云端整合，改善性能
- 支持将本地运行的大型工作负载（理赔和保单管理系统、自定义应用、大型数据仓库）迁移到 AWS
- 支持 Exadata 和 Real Application Clusters (RAC) 技术

00:09:30 - 场景四和五：应用开发和企业 AI
- 利用 Oracle 融合 23ai 数据库的原生能力（向量、JSON、图形）
- 在单一数据库环境中部署，直接从 AWS 控制台操作
- 利用 AWS 高级服务构建 AI 代理系统
- 将 AI 带到数据，而不是将数据移动到 AI
- 允许客户在 AWS 内原生部署，将大型语言模型带到数据
