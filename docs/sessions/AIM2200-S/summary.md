# AWS re:Invent 2025 会议总结

## 会议概述

本次分会是关于联合航空公司(United Airlines)与塔塔咨询服务公司(TCS)以及AWS三方合作,共同推进机组运营系统现代化转型的案例分享。会议由TCS云解决方案负责人Ayand Data主持,联合航空机组现代化总监Sean Carson和TCS联合航空客户总监Subramanyan(Subu)担任主讲嘉宾。

这是一个真实的业务现代化案例,展示了如何通过技术、客户和服务提供商三方紧密合作,重新构想业务流程并实现转型。联合航空与TCS已有超过20年的合作历史,特别是在过去5年中,双方在机组运营领域采取了激进的转型策略,借助AWS云平台实现了从传统大型机系统到现代化云架构的彻底转变。这一转型不仅涉及技术平台的升级,更重要的是改善了运营可靠性、提升了员工体验,并为未来的智能化运营奠定了基础。

联合航空目前是全球按可用座位里程计算最大的航空公司,运营超过1000架飞机,服务380个目的地,每日执行超过5000个航班,年运送旅客近1.75亿人次。公司拥有超过11万名员工,其中包括18000名飞行员和28000名乘务员。机组运营系统正是服务于这两个核心工作群体的关键系统。

## 详细时间线与关键要点

### **00:00 - 开场介绍**
- TCS的Ayand Data介绍会议主题和演讲嘉宾
- 强调这是一个三方合作的真实业务现代化案例,而非单纯的技术现代化

### **02:30 - TCS与联合航空的合作背景**
- Subu介绍TCS与联合航空超过20年的合作关系
- 合作的核心优势:领域专业知识、业务知识、情境知识、创新能力
- 过去10年在机组运营领域深度合作,最近5年采取激进转型策略

### **04:15 - Sean Carson自我介绍与行业背景**
- Sean在旅游和酒店行业工作约30年,其中25年在联合航空
- 联合航空的战略重点是创新和演进
- 该项目体现了公司鼓励的韧性和创新精神

### **05:30 - 联合航空公司概况**
- 按可用座位里程计算的全球最大航空公司
- 今年首次运营超过1000架飞机
- 服务380个目的地,拥有7个国内枢纽和至少1个国际枢纽
- 每日超过5000个航班起降
- 年运送旅客近1.75亿人次
- 员工超过11万人,包括18000名飞行员和28000名乘务员

### **07:00 - 机组运营系统介绍**
- 飞行员和乘务员都有集体谈判协议管理工作规则
- 基于资历和工作规则每月竞标航班排班
- 培训一名新的机组调度员达到熟练水平需要约18个月

### **09:45 - 机组运营面临的挑战**
- **传统大型机系统**:技术老化,难以适应生态系统变化,创新和自动化受限
- **监管合规要求**:需要向FAA披露数据,确保符合安全规定
- **资源稀缺**:熟悉大型机技术的人才越来越少
- **运营可靠性问题**:天气事件、行业挑战、机场老化、人员配置问题
- **机组利用率优化**:正处于大规模增长期,需要优化现有资源
- **中断管理**:不规则运营时需要解决复杂的调度难题
- **数据孤岛**:数据结构分散,需要集中化以支持分析

### **14:20 - 转型的核心原则**
- **提升运营可靠性**:主动预测和调配机组,进行假设场景分析
- **赋能调度员**:提供决策支持工具,使工具更直观易用,缩短学习曲线
- **合同规则合规**:适应每4-6年更新一次的劳工合同变化
- **改善员工体验**:从传统绿屏界面转向现代化用户界面
- **减少中断**:避免因机组调度问题导致的二次航班取消

### **18:30 - 转型方法论**
- **平台重构**:从单体大型机系统转向分布式架构
- **集中化合法性规则引擎**:将嵌入在COBOL代码中的规则提取为可查询的中央引擎
- **微服务架构**:将单一工作队列拆分为多个微服务,提高系统韧性
- **自动化为中心**:建立自动化工具并培养团队对自动化的信任
- **面向未来**:支持"United Next"计划,目标是机队规模增长100%

### **23:00 - 云迁移战略**
- 约4-5年前,联合航空开始将关键应用迁移到AWS云平台
- 目前约80个关键应用已在AWS平台上运行
- AWS和TCS成为重要合作伙伴

### **25:15 - 云平台带来的优势**
- **计算能力大幅提升**:支持更复杂的计算需求
- **数据管理现代化**:标准化的数据目录和管理方式
- **更高的韧性**:应用程序更加稳定可靠
- **竞争优势**:内部构建的先进机组调度系统成为竞争优势

### **28:00 - 新系统的核心功能**
- **告警引擎**:基于合法性规则提供决策支持,按优先级处理而非先进先出
- **集中化规则引擎**:为自动化策略提供平台基础
- **自动扩展能力**:根据不规则运营时的数据流量自动调整资源
- **冗余和韧性**:数据缓冲机制,改善下游系统的恢复策略
- **自动化灾难恢复**:利用AWS云的自动恢复能力

### **32:45 - 系统架构变革**
- 彻底革新前端界面,不是简单的"提升和迁移"
- 目前有4个独立的机组应用程序,由不同团队管理但相互集成
- 将单一队列拆分为**超过380个微服务**
- 微服务架构确保单个队列故障不会阻塞整个管道

### **35:30 - 转型成果**
- **交易工作量显著减少**:调度员工作更轻松高效
- **数据集中化**:整合到联合航空的中央数据中心,支持企业级分析
- **培训成本降低**:系统更直观,员工流失率下降
- **角色细分**:新员工可以从影响较小的角色开始学习

### **38:00 - 未来发展方向**
- **智能工作空间**:整合运营中心、机组、维护和地面操作的信息
- **预测性分析**:为各团队提供更高效的协作工具
- **AI集成**:探索TCS的Mastercraft平台,嵌入大语言模型
- **AI决策支持**:让AI环境辅助甚至自动做出调度决策
- **可观测性和可靠性**:建立数字孪生等概念,持续提升系统可靠性

### **42:00 - 关键启示**
- 转型是一个旅程而非终点
- 停止创新就意味着停滞不前
- 通过打破传统大型机的束缚,为AI和先进分析工具的应用创造了可能
- 5年前在传统平台上无法实现的功能,现在已成为现实

会议总结:这是一个典型的企业数字化转型成功案例,展示了如何通过云技术、微服务架构和合作伙伴生态系统,将一个有着数十年历史的传统航空公司核心运营系统转变为现代化、智能化的平台。转型不仅解决了技术债务问题,更重要的是为未来的AI驱动运营和持续创新奠定了坚实基础。
