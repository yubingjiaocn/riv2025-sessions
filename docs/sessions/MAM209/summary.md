# AWS re:Invent 2025 SAP 转型客户分享会议总结

## 会议概述

本次会议是AWS re:Invent 2025的一场客户分享会，主题聚焦于SAP工作负载在AWS上的转型之旅。会议由AWS的Soulat Khan主持，邀请了三位来自不同行业的客户代表分享他们的实际经验：Mondelez International的云工程师Pete Rauchenstein、Colgate-Palmolive的基础设施和数据库架构总监Purnachand Aravapalli，以及澳大利亚联邦银行核心银行业务总经理Simon Davies。

会议深入探讨了这些企业选择AWS作为SAP转型合作伙伴的原因、实施过程中的挑战与收获，以及现代化创新的实践经验。三位客户代表分别来自消费品制造、个人护理用品和金融服务行业，为不同规模和需求的企业提供了宝贵的参考经验。会议强调了文化变革、技术创新和合作伙伴关系在SAP云转型中的重要性。

## 详细时间线与关键要点

### 00:00-05:00 开场介绍与嘉宾自我介绍
- 主持人Soulat Khan介绍会议目的：分享客户SAP工作负载AWS转型经验
- Pete Rauchenstein（Mondelez International）：全球最大消费品公司之一，生产奥利奥、吉百利等知名品牌
- Purnachand Aravapalli（Colgate-Palmolive）：基础设施和数据库架构总监，公司年营业额200亿美元
- Simon Davies（澳大利亚联邦银行）：核心银行业务总经理，服务澳大利亚三分之一人口

### 05:00-15:00 选择AWS的原因分析
- **澳大利亚联邦银行的考量**：
  - 处理澳大利亚40%的交易量，系统极其关键
  - 选择AWS的三个主要原因：规模和弹性、工程合作伙伴关系、全栈视角的现代化机会
  - 与AWS共同开发了EC2新功能，惠及所有SAP客户
  - 从孤立的8个团队运维模式转向统一的全栈运营

- **Mondelez的转型背景**：
  - 公司成立16年，由Kraft北美业务和Cadbury欧洲业务合并而成
  - 面临不同系统、不同云平台、不同操作系统的整合挑战
  - 4年前建立内部云工程组织，除第一个根账户外全部使用基础设施即代码
  - 与AWS专业服务团队建立深度合作关系

### 15:00-25:00 Colgate-Palmolive的云优先策略
- **公司背景**：250年历史的全球性公司，服务200个国家和地区
- **当前状态**：6个生产环境已在AWS RISE平台运行，计划2027年中期前迁移另外8个系统
- **选择标准**：
  - 性能要求：达到或超越现有系统性能
  - 与SAP路线图保持一致
  - 云多样化战略的一部分
- **独特优势**：2019年就完成S/4HANA转型，业务案例从一开始就包含云迁移

### 25:00-35:00 AWS服务项目与经验分享
- **Mondelez的合作经验**：
  - 与AWS专业服务团队保持4年深度合作，至今仍有每周办公时间
  - 通过re:Invent结识合作伙伴Lemongrass
  - 重视自动化和基础设施即代码原则
  - 专注于可扩展、可重复的解决方案

- **澳大利亚联邦银行的量化改进**：
  - 性能提升30%：从DB2大型机R/3迁移到AWS上的S/4HANA
  - 弹性改进：故障转移时间从90分钟缩短到16分钟，进一步优化后实现更快恢复
  - 效率提升：基础设施成本降低30%，每月减少数百个手动操作点

### 35:00-40:00 Colgate-Palmolive的实施经验
- **快速部署**：8个月内迁移6个生产环境
- **关键支持服务**：
  - 沉浸式培训日和AWS架构完善框架
  - 连接性研讨会，快速部署互联解决方案
  - AWS客户团队的持续支持和安全态势建设
- **成功因素**：AWS团队能够理解企业特定需求，用客户语言沟通

### 40:00-45:00 现代化与创新展望
- **澳大利亚联邦银行的AI应用**：
  - 利用Amazon Q进行自动化测试用例构建和代码修复
  - 实时事件流处理，支持每秒5000笔交易
  - 通过Amazon MSK、Kinesis、SageMaker构建数据管道
  - 探索Bedrock和AgentCore在智能银行服务中的应用

- **Colgate-Palmolive的服务整合**：
  - 使用AWS互补服务支持RISE平台
  - 品牌网站已在AWS上运行，提供全球覆盖和高性能
  - 逐步将更多组件迁移到AWS自有账户

- **Mondelez的自动化工具**：
  - 使用Spacelift进行Terraform自动化
  - Amazon Q提高开发效率
  - 与Lemongrass合作开发自动化部署解决方案

### 45:00-49:30 经验总结与建议
- **Pete的建议**：
  - 团队合作至关重要，这是一场开卷考试
  - 不要害怕挑战现状，勇于尝试新方法
  - 建立正确的合作伙伴关系是成功的关键

- **Purna的经验**：
  - 让数据和证据引导决策
  - 与合作伙伴保持紧密协作和信任关系
  - 重视变更管理和团队沟通
  - 为上线和日常运营做充分准备

- **Simon的文化洞察**：
  - 文化转型与技术转型同等重要
  - 勇气意味着采取行动，保持前进
  - 保持"合理的不合理性"和专注的执行力
  - 团队始终保持紧迫感，最终成功完成转型
