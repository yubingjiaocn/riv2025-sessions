# AWS re:Invent 2025 Rubrik 云弹性与AI代理保护会议总结

## 会议概述

本次会议由Rubrik公司的AI总经理Dev Rishi和云专家总监Adam主讲，重点探讨了在AI时代云弹性的真正含义以及企业如何应对AI代理带来的新挑战。Dev Rishi分享了与180多个企业组织的对话洞察，揭示了代理AI（Agentic AI）将成为2026年及以后企业AI部署的主要趋势。

会议核心围绕Rubrik新发布的Agent Cloud产品展开，该产品旨在解决企业在部署AI代理时面临的监控、治理和修复三大挑战。Adam则从云原生数据保护角度，详细阐述了在日益云化的环境中如何确保数据弹性，特别是针对S3存储和DevOps环境的保护策略。

## 详细时间线与关键要点

### 0:00-5:00 开场与AI代理趋势介绍
- Dev Rishi介绍Rubrik在AI弹性方面的思考
- 定义代理AI：具有工具访问权限的大语言模型，能够代表用户采取行动
- 强调代理AI在工作流自动化和生产力提升方面的巨大潜力

### 5:00-10:00 AI代理风险分析
- AI代理具有超人能力，可以10倍速度造成10倍损害
- 使用非人类身份和凭证运行，在非AI设计的企业生态系统中操作
- 实际案例：聊天机器人推荐竞争对手产品、编码代理删除生产数据库

### 10:00-15:00 企业部署挑战
- 缺乏统一视图了解运行中的AI代理
- 无法有效监控代理可访问的工具和数据
- 从AI错误中恢复的困难
- 构建代理容易（数天到数周），但部署审批流程漫长（数月）

### 15:00-20:00 Rubrik解决方案介绍
- 结合数据、身份和LLM平台三大核心能力
- 发布Rubrik Agent Cloud产品
- 与AWS Bedrock AgentCore集成，自动发现原生构建的代理

### 20:00-25:00 Agent Cloud三大支柱
- **监控与可观测性**：自动发现AI代理，建立代理清单
- **治理**：将策略转化为软件配置，实时执行治理
- **修复**：AI Agent Rewind功能，一键撤销破坏性操作

### 25:00-30:00 云数据弹性责任模型
- Adam接手讲解云环境中数据弹性责任分工问题
- 介绍共享责任模型：AWS负责云的安全，客户负责数据安全
- 分析不同团队间责任推诿现象

### 30:00-35:00 弹性策略分层
- **高可用性**：应对常见故障的基础弹性
- **灾难恢复**：应对自然灾害或区域性故障
- **网络弹性**：防护网络攻击并确保有效恢复
- 介绍3-2-1-1-0数据弹性原则

### 35:00-40:00 工作负载分层保护
- **关键任务（Tier 0）**：需要本地、远程和网络保险库三重保护
- **业务关键（Tier 1）**：本地和远程保护，可选网络保险库
- **重要应用（Tier 2）**：本地和远程保护
- **开发测试（Tier 3）**：基础保护，可通过基础设施即代码重建

### 40:00-46:00 S3保护与DevOps环境
- S3数据指数级增长，从存档用途转向云数据库
- 版本控制和复制不足以应对自动化事故和网络事件
- DevOps环境成为关键盲点，包含代码仓库、CI/CD管道、ML管道
- Rubrik推出DevOps保护功能，集成到统一平台

### 结束语与作业
- 提出四个思考问题：AI代理治理策略、网络RTO定义、DevOps环境保护、恢复测试频率
- 强调定期恢复测试的重要性
