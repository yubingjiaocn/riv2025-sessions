# AWS re:Invent 2025 - Amazon Linux 会议总结

## 会议概述

本次会议庆祝了 Amazon Linux 发布 15 周年（2010-2025），由产品经理 Shri 和首席工程师 Frederick 共同主持。会议重点介绍了 Amazon Linux 2023 (AL2023) 在过去一年中的重要创新和功能更新。Amazon Linux 作为 AWS 上最受欢迎的操作系统，专为云环境优化，提供了卓越的安全性、稳定性和与 AWS 服务的深度集成。

会议强调了 Amazon Linux 的核心价值：AWS 优化、高度安全、简化运维、合规就绪以及降低总拥有成本（无许可费用）。从 AL1（2010）到 AL2（2017）再到 AL2023（2023），Amazon Linux 不断演进，适应上游变化和客户需求。本次会议详细介绍了 2024 年在安全性、功能特性和客户反馈响应方面的重大进展。

会议特别关注了三大支柱：安全性（包括 CVE 修复、FIPS 验证、DISA STIG 合规）、稳定性（内核维护策略调整、向后兼容性）以及实用性（图形桌面、NVIDIA 仓库、ESPAL 补充软件包）。这些更新旨在帮助客户更安全、更高效地运行工作负载，同时简化从 AL2 到 AL2023 的迁移路径。

## 详细时间线

### 开场与历史回顾
[00:00 - 02:30]
- 欢迎致辞，介绍演讲者 Shri（产品经理）和 Frederick（首席工程师）
- 宣布 2025 年是 Amazon Linux 发布 15 周年（AL1 于 2010 年首次发布）
- 会议议程：Amazon Linux 演进历程、AL2023 关键功能、AL2 支持终止时间表、未来规划

[02:30 - 05:00] Amazon Linux 核心价值
- AWS 优化：预配置 cloud-init、预装 AWS 工具
- 高度安全：定期提供安全补丁和漏洞修复
- 简化运维：与 EC2、IAM、EBS、CloudWatch 等服务原生集成
- 合规就绪：支持 FIPS、DISA STIG 等合规标准
- 降低 TCO：无许可费用，内置 AWS 支持

[05:00 - 06:30] 版本演进
- AL1（2010）：专为云构建，集成 AWS 服务
- AL2（2017）：增强稳定性和安全性，提供长期支持，成为最常用的操作系统
- AL2023（2023）：最新版本，提供现代功能和改进的安全默认设置

### 安全性创新

[06:30 - 10:00] CVE 安全修复
- 2024 年为 AL2023 发布约 1,500 个安全修复
- 修复分布：500+ 重要级别 CVE、800+ 中等级别 CVE、少量严重级别 CVE
- 关键发现：超过 60% 的重要和中等级别 CVE 来自 Linux 内核（而非用户空间软件包）
- Amazon Linux 团队每日评估新 CVE，确保及时修复并控制误报

[10:00 - 15:30] Linux 内核维护策略调整
- 上游变化：Linux 内核长期支持（LTS）从 6 年缩短至 2 年
- AL2023 策略：6.1 内核将持续维护至 AL2023 支持结束
- 新策略：从 6.12 内核开始，新内核维护期为 4 年（而非操作系统全生命周期）
- 影响：未来 Amazon Linux 版本用户需要计划至少一次内核升级
- 新 SSM 参数：引入内核特定参数（如 al2023-kernel-6.1、al2023-kernel-6.12）和 kernel-default 参数
- kernel-default 参数：自动迁移到最新推荐内核，减少回归风险，建议配合可预测更新功能使用

[15:30 - 19:00] FIPS 验证
- AL2023 获得最终 FIPS 140 验证（2024 年早期）
- 验证模块：OpenSSL、内核、NSS、GnuTLS、libgcrypt
- 用户可通过内核命令行或应用配置启用 FIPS 模式
- 注意事项：6.1 内核和 OpenSSL 3.0 已验证，6.12 内核和 OpenSSL 3.2 正在验证中
- 承诺：持续对 AL2023 更新进行 FIPS 重新验证

[19:00 - 22:00] DISA STIG 合规
- 2024 年 9 月发布 AL2023 DISA STIG 配置文档
- 自动化支持：
  - EC2 Image Builder：可在镜像构建时启用 DISA STIG 配置
  - AWS Systems Manager：提供托管文档，可在运行实例上应用 DISA STIG 配置
- 简化合规性操作，无需手动配置

### 性能与功能改进

[22:00 - 26:30] OpenSSL 性能提升
- 问题背景：AL2023 初始版本使用 OpenSSL 3.0，相比 AL2 的 OpenSSL 1.0.2 存在性能下降
- 解决方案：AL2023.7 版本升级至 OpenSSL 3.2
- 性能改进：
  - 合成基准测试显示性能提升高达 15 倍
  - 真实 TLS 工作负载显示 2-3 倍性能提升（TLS 会话建立）
- FIPS 模块分离：将 FIPS 提供程序独立打包，允许用户在使用最新 OpenSSL 的同时运行已验证的 FIPS 模块

[26:30 - 30:00] 图形桌面支持
- 客户需求：测试图形应用、系统管理员需要图形环境
- 实现：基于 GNOME 的图形桌面
- 设计考虑：
  - 最小化依赖树
  - 排除常见 CVE 来源的应用
  - 与 Amazon DCV 集成，提供安全远程访问
- 意外发现：视频编解码器软件包下载量超过桌面本身，团队希望了解更多使用场景

[30:00 - 34:00] NVIDIA 仓库
- 问题：在 EC2 加速实例上运行 Amazon Linux 时，难以选择合适的 NVIDIA 软件栈
- 解决方案：发布 AL2023 NVIDIA 仓库
- 特点：
  - 使用 NVIDIA 官方 RPM 包
  - 与 EC2 平台团队合作，对每个更新进行资格验证和性能测试
  - 添加安全元数据和 CVE 公告
  - 在所有 AWS 区域和可用区复制，提供低延迟访问
- 启用方式：dnf install nvidia-release 后按 NVIDIA 文档安装所需软件包

[34:00 - 37:30] ESPAL 补充软件包
- 背景：客户从 AL2 迁移到 AL2023 时，依赖 EPEL 仓库的软件包不可用
- 解决方案：与 SUSE 合作，发布 ESPAL（Supplementary Packages for Amazon Linux）
- 内容：从 EPEL 9 重新构建数千个软件包用于 AL2023
- 持续更新：未来几周和几个月将继续添加新软件包
- 注意事项：
  - 支持级别低于核心仓库
  - 仅同步上游 Fedora 项目的更新，不独立维护
  - 不提供安全公告
  - 建议仅用于迁移过渡，长期应迁移到核心仓库软件包
