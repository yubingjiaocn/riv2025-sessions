# AWS re:Invent 2025 会议总结：通过 AWS Marketplace 和 Amazon Bedrock Agent Core 部署 AI 代理

## 会议概述

本次技术分享会由 AWS Marketplace 的 AI 代理和工具业务开发负责人 Mike Levy 和产品经理 Idan Mislitz 共同主讲。会议重点介绍了如何通过 AWS Marketplace 快速发现、部署和使用 AI 代理解决方案，并将其集成到 Amazon Bedrock Agent Core 工作流中。

演讲者强调了 Agentic AI（代理式人工智能）的实际应用价值，并展示了如何帮助客户克服从概念验证（POC）到生产环境部署的挑战。通过 AWS Marketplace 提供的第三方解决方案和 Amazon Bedrock Agent Core 的模块化服务，企业可以更快速地实现业务流程自动化、提高生产力并降低成本。会议还包含了一个完整的实际演示，展示了如何使用 LLM IQ 代理工具来帮助编码代理选择最适合特定用例的大语言模型。

根据 Gartner 预测，到 2028 年，超过 33% 的企业软件将包含 Agentic AI 功能，而目前这一比例仅为 1%。然而，也有超过 40% 的 Agentic AI 项目可能在 2027 年前因无法看到业务成果或成本过高而被取消，这凸显了快速部署和实现价值的重要性。

### 实际演示：部署到 Gateway（20:00-24:00）
- **20:00** - 进入启动向导，选择 Bedrock Agent Core 部署选项（而非 API 选项）
- **20:30** - 第一步：从卖家获取访问密钥（通过 Quick Launch 功能自动配置到 AWS 账户）
- **21:00** - 第二步：设置身份验证，告诉 Gateway 调用工具时使用特定 API 密钥
- **21:30** - 复制 API 密钥并粘贴到 Agent Core 的身份验证界面
- **22:00** - 点击"添加产品到新网关"按钮，跳转到 Agent Core Gateway 控制台
- **22:30** - 在 Gateway 控制台中，所有必需信息已预填（得益于 Marketplace 集成）
- **23:00** - 提供网关名称，选择 Cognito 身份验证，命名目标（工具），OpenAPI 规范已自动提供
- **23:30** - 选择相关 API 密钥，点击创建网关
