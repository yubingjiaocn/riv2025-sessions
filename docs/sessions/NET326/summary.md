# AWS re:Invent 2025 安全边界保护会议总结

## 会议概述

本次会议主题为"在AWS上构建安全边界保护"，由AWS网络团队的三位专家主讲。产品经理Shovan Das负责零信任网络和WAN领域，Megha Kande领导CloudFront团队专注边缘安全，首席解决方案架构师Sohaib Tahir展示了使用GenAI和Kiro CLI构建端到端边界保护的实际演示。

会议围绕虚构客户SecureShop展开，该客户拥有公共零售网站和私有财务应用程序。演讲者详细介绍了四个流量方向的保护策略：互联网入口、应用程序出口、应用程序间通信以及员工访问。通过CloudFront、WAF、Shield、Network Firewall和Verified Access等服务的组合使用，构建了完整的安全边界防护体系。

## 详细时间线与关键要点

### 0:00-10:00 会议开场与安全边界概述
- **安全边界的重要性**：构建和扩展应用程序后，保护应用程序同样重要
- **四个流量方向**：
  - 互联网入口：用户通过互联网连接，需防范DDoS和恶意攻击
  - 应用程序出口：应用程序访问外部资源，需确保合规性
  - 应用程序间通信：防止横向移动攻击
  - 员工访问：确保正确权限的用户访问
- **SecureShop架构介绍**：包含公共零售网站和私有财务应用，部署在多个VPC中

### 10:00-25:00 CloudFront边缘安全详解
- **流量威胁分析**：超过50%的流量来自机器人，DDoS攻击量在过去两年增长400%
- **AWS边缘基础设施**：750个存在点，分布在100多个城市和50多个国家
- **多层安全防护**：
  - Layer 3/4 Shield防护：IP黑名单、SYN flood攻击防护
  - TLS连接安全：高级加密套件、后量子计算密钥支持
  - 双向TLS认证：完全托管的客户端证书验证
- **WAF托管规则**：OWASP前10安全风险防护、机器人控制、DDoS防护

### 25:00-35:00 私有VPC源站保护与演示准备
- **私有源站连接**：CloudFront支持连接私有VPC中的应用程序
- **VPC到VPC NAT**：CloudFront自动建立并管理VPC连接，无额外费用
- **Layer 7 DDoS防护**：新的托管规则，响应时间从分钟级降至秒级
- **Shield Advanced功能**：支持500亿请求检查，提供更精细的控制选项

### 35:00-45:00 Kiro CLI演示 - 部署边缘安全
- **Kiro CLI介绍**：AI代理工具，支持自然语言与AWS资源交互
- **自动化部署**：使用简单提示词部署完整的安全配置
- **实施的安全措施**：
  - SQL注入和跨站脚本防护
  - 速率限制：每5分钟200请求限制
  - 地理封锁：仅允许美国地区访问
  - HTTPS强制重定向
  - 源访问控制(OAC)启用
- **监控指标**：1400个请求中51个被阻止，展示了WAF规则的有效性

### 45:00-55:00 网络防火墙与零信任访问
- **Network Firewall功能**：
  - 基于Suricata规则引擎
  - 托管服务，自动扩展
  - 集成AWS网关，支持bump-in-the-wire部署
- **MadPot威胁情报**：全球蜜罐网络，每日分析超过1亿个签名模式
- **客户端VPN vs Verified Access**：
  - Client VPN：提供网络级访问，支持SSH/Telnet
  - Verified Access：基于零信任原则的HTTP应用访问
- **零信任原则**：非网络信号验证 + 持续验证

### 55:00-59:00 Verified Access演示与总结
- **FinanceCore应用演示**：
  - IAM Identity Center身份验证
  - WAF规则保护内部应用
  - 策略助手工具用于细化访问控制
- **Network Firewall规则测试**：
  - 阻止HTTP流量（端口80）
  - 阻止SSH连接（端口22）
  - 恶意域名防护
- **完整安全架构**：结合所有服务构建SecureShop企业级安全边界
- **会议总结**：通过多层安全服务组合，实现了全方位的网络边界保护
