# AWS re:Invent 2025 - 从原型到生产：Amazon Bedrock Agent Core

## 会议概述

本次会议主题为"弥合从原型到生产的鸿沟"，由AWS Amazon Bedrock Agent Core产品管理负责人Vivek Singh、AWS AI代理技术负责人Mark Roy以及爱立信AI技术负责人Sarbishes Das共同主讲。

会议深入探讨了AI代理从原型开发到生产部署过程中面临的核心挑战。尽管过去一年中AI代理原型数量激增，但真正投入生产环境的比例却极低，仅为个位数百分比。演讲者通过现场调查证实了这一现象：大量观众构建过代理原型，但能够将其成功部署到生产环境并稳定运行的寥寥无几。这种"原型到生产的鸿沟"成为当前组织实现AI代理愿景的最大障碍。

为解决这些挑战，AWS推出了Amazon Bedrock Agent Core平台，这是一套完整的无服务器服务集合，专门用于构建、部署和运营高性能、安全且可扩展的AI代理系统。该平台提供了模块化的服务组件，支持任何框架、任何模型，并内置了安全性、可观测性和治理能力。会议还展示了爱立信如何利用Agent Core将网络工程师的研究时间从数天缩短到数分钟的实际案例。

### 核心挑战分析 (05:30 - 15:00)

05:30 - 原型易构建的原因
- 框架如LangGraph、Strands Agents、Crew AI提供了现成的抽象
- 开发者可以在几天内构建出编码助手、客户支持机器人或销售代理
- 框架处理了底层管道工作，让开发者专注于代理行为定义

07:00 - "Sales Assist"案例引入
- 假设场景：构建一个帮助销售代表的AI代理
- 功能包括：回答产品问题、从Salesforce提取客户历史、起草个性化邮件、建议下一步行动
- 为单个销售代表本地运行原型只需几天

08:30 - 生产环境的复杂性
- 需要支持10,000名销售代表，分布在5个国家
- 每个代理每天处理数十次对话
- 访问多个系统：Salesforce、Confluence、DocuSign、内部定价API
- 必须遵守每个用户的个人权限

10:00 - 四大挑战类别
1. 性能挑战：代理必须响应迅速、可靠且准确
2. 可扩展性挑战：需要处理数千个并发会话
3. 安全挑战：必须遵守细粒度的身份和访问控制
4. 治理挑战：需要可见性和防护措施来控制不安全操作

11:30 - 性能挑战详解
- 工具集成：生产环境中有数十甚至数百个工具，每个都有自己的模式、速率限制和不可预测的故障
- 内存管理：LLM是无状态的，但代理需要记住交易历史和先前对话的关键事实
- 持续评估：需要评估正确性和有用性，防止代理性能在生产中漂移

13:00 - 可扩展性挑战
- 需要安全、弹性和可扩展的运行时
- 在高峰工作负载时扩展，空闲时缩减
- 保持每个代理会话隔离
- 在长时间运行的多步推理工作流中维护状态
- 从API超时或服务中断等故障中快速恢复

14:00 - 安全与治理挑战
- 安全：每个操作必须遵守细粒度的身份和访问控制（例如：Sarah看到企业账户，Tom看不到）
- 治理：需要审计能力（代理访问了什么数据、发送了什么邮件、建议了什么价格）
- 实时业务规则执行（例如：永不分享未公布的功能、折扣不超过20%）

### Amazon Bedrock Agent Core介绍 (15:00 - 25:00)

15:00 - Agent Core发布
- 正式发布的通用服务，今年早些时候推出
- 提供完整的服务集，用于构建、部署和运营高性能代理
- 目标：安全且大规模地运行代理

16:30 - Agent Core三大特性
1. 优化价值实现时间：完全无服务器，按使用付费，甚至不收取IO等待时间费用（通常占代理会话时间的60-70%）
2. 灵活性：每个服务完全模块化，可独立或组合使用，支持任何框架和任何模型
3. 企业级控制：提供访问管理和可观测性，满足企业部署需求

18:00 - 核心服务概览

构建高性能代理的服务：
- **Memory（内存）**：自动提取和存储短期记忆（多轮交互）和长期记忆（语义记忆、用户偏好）
- **Gateway（网关）**：将工具和代理组合成单一MCP服务器接口，统一不同工具源
- **托管工具**：
  - Code Interpreter：在隔离环境中安全生成和执行代码
  - Browser：使代理能够大规模与Web应用交互

部署和扩展代理的服务：
- **Runtime（运行时）**：无服务器运行时，支持任何框架、任何协议（MCP、A2A）、任何模型
- **Identity（身份）**：简化代理身份和访问管理，允许代理代表用户安全访问AWS和第三方服务
- **Policy（策略）**：今日发布，提供实时确定性控制，用自然语言定义策略

运营可信代理的服务：
- **Observability（可观测性）**：提供代理工作流每个步骤的丰富可视化和操作指标，支持OpenTelemetry格式
- **Evaluations（评估）**：今日发布，持续检查代理行为质量，在问题造成广泛影响前发现

### Mark Roy深入讲解 (25:00 - 50:00)

25:00 - Mark Roy登场
- 自我介绍：AWS AI代理全球技术负责人，30年构建经验
- 过去两年与数百个客户合作构建和部署代理
- 观察到三个共同点：出色的原型、令人信服的商业价值、数月的繁重工作

27:00 - 挑战一：代理托管
- 问题：在哪里运行代理？不能在笔记本电脑上运行并分享给用户
- 压力来源：需要支持10,000用户、立即交付、安全官要求、平台团队处理多个框架（LangGraph、Crew AI、OpenAI）
- Agent Core Runtime解决方案：支持任何框架、任何模型，安全且大规模托管

29:30 - Runtime特性
- 从零扩展到数千个并发会话
- 内置MCP托管、A2A代理互操作性、OpenTelemetry可观测性、OAuth安全性
- 支持多种代理类型：智能聊天、语音代理、长时间运行的深度研究、大负载处理
- 只需4行代码即可让现有代理准备好在Agent Core Runtime上运行

32:00 - 安全隔离
- 问题：如何确保不同用户的敏感数据（如社会安全号码、信用评分）在并发会话中安全隔离
- 现场调查：只有一位观众举手表示100%确信会话安全
- Agent Core Runtime解决方案：开箱即用的安全会话隔离，使用microVM架构（不仅是容器级隔离）
- 会话结束后自动清理，无临时文件残留，无权限提升风险

35:00 - 挑战二：代理记忆
- 问题：真正的代理需要学习、记忆和改进能力
- 需要记住最近几分钟的对话（短期记忆）和过去几个月的内容（长期记忆）
- 必须可靠（代理重启后仍保留）且安全

36:30 - Agent Core Memory
- 支持任何框架，即插即用
- 短期记忆：低延迟、安全，用于良好的对话体验
- 长期记忆：自动提取知识、事实、对话摘要
- 昨日新增：情节记忆（episodic memory），能够跨情节反思并获得洞察
- 实现超个性化体验，完全托管、无服务器、安全

39:00 - 挑战三：工具集成
- 问题：代理需要访问企业数据、API和服务才能产生真正价值
- 数据是差异化因素，但如何整合MCP服务器和现有能力是繁重工作

40:30 - Agent Core Gateway
- 自动将现有资源映射为代理就绪工具
- 将API暴露为MCP，插入Strands、Crew AI、LangGraph等框架
- 可创建多个网关，每个网关添加多个目标（API端点、Lambda函数、现有MCP服务器）
- 大幅提升价值实现时间

42:00 - 动态工具选择
- 问题：工具过多（数百甚至数千个）会导致代理性能下降
- 解决方案：内置语义搜索，工具添加时自动索引
- 根据上下文动态选择少量相关工具，而非提供全部300个工具
- 结果：更快、更便宜、更准确的"三重优势"

44:00 - Policy for Gateway（今日发布）
- 问题：平衡代理自主性与控制需求，避免"登上华尔街日报头版"的百万美元错误
- 解决方案：拦截每个工具调用，应用管理员定义的策略
- 用自然语言定义策略，低延迟实时执行
- 虽然代理是非确定性的，但策略执行是确定性的
- 提供可观测性和审计能力

46:30 - 内置工具
- **Code Interpreter**：安全沙箱中生成和执行代码，使代理成为即时数据分析师，可生成可视化
- **Browser**：自动化访问遗留应用（5年或25年前构建的关键任务系统），LLM解释截图并执行点击、导航、滚动、复制等操作

48:00 - 跨领域关注点一：安全
- 挑战：用户→应用→代理→其他代理→内部和第三方工具，必须确保正确的用户访问正确的数据
- Agent Core解决方案：
  - 入站：集成任何身份提供商（Microsoft Entra、Ping、Okta、Auth0）
  - 出站：集成凭证提供商（Salesforce、ServiceNow、Workday、Jira等），支持API密钥、OAuth、IAM
  - 端到端安全访问，代表用户或代理自主执行
- 警告：不要自己尝试构建，底层非常复杂

49:30 - 跨领域关注点二：可观测性
- 原则：永远不要信任代理，必须知道它在做什么
- 需要追踪：请求、计划、执行步骤、重定向、工具输入/输出、错误和重试
- Agent Core提供：
  - 所有服务的完整可观测性
  - 仪表板显示延迟、错误率等指标
  - 端到端可视化、层次化时间线、完整轨迹
  - 支持OpenTelemetry，可集成Dynatrace、Datadog、Langfuse等

51:00 - 跨领域关注点三：评估（今日发布）
- 问题：如何知道代理表现如何？是否安全、负责任、礼貌、正确、使用正确的工具和参数？
- Agent Core Evaluations：
  - 不到1分钟即可在控制台配置：选择代理、选择指标、设置采样率
  - 自动评估所有流量，提供评分和推理解释
  - 提供仪表板和按需评估
  - 今日公开预览发布

53:00 - Agent Core平台总览
- 展示完整平台架构图
- 强调模块化：可按需选择服务
- 支持所有正确的协议：OAuth、OpenTelemetry、A2A、MCP
- 构建在客户已信任的AWS基础之上

54:00 - 演示：多框架多模型集成
- 场景：展示"任何框架、任何模型、A2A"的真实性
- 架构：
  - 编排器：使用Google ADK和Gemini
  - 子代理1：使用Strands和Bedrock模型
  - 子代理2：使用OpenAI和GPT-4
  - 通过A2A通信
- 使用Agent Core Gateway访问工具，集成Memory和Observability
- UI演示：JavaScript界面显示A2A代理卡片、任务委派、短期和长期记忆
