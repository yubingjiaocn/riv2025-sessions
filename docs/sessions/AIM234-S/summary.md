# AWS re:Invent 2025 技术会议总结：可信任的自主代理

## 会议概述

本次会议由IBM的Neil LeBlanc（watsonx.governance市场负责人）和KPMG的Diana Griffin（AI治理专家）共同主讲，主题为"可信任的自主代理"。会议深入探讨了AI代理技术的快速发展及其带来的治理挑战，重点介绍了如何通过有效的AI治理框架来确保代理系统的可信性和安全性。

演讲者强调，随着代理技术变得更加强大和复杂，组织面临着透明度缺失、行为无法解释、测试困难等挑战。会议展示了watsonx.governance平台如何通过监控、评估和优化功能来解决这些问题，并强调AI治理应被视为创新的推动力而非阻碍。

## 详细时间线与关键要点

### 0:00-2:00 开场介绍与背景
- 演讲者自我介绍：Neil LeBlanc（IBM）和Diana Griffin（KPMG）
- 强调AI代理技术在过去2-3年的快速发展超出预期
- 指出代理技术虽然强大但带来伦理和社会影响的担忧

### 2:00-5:00 代理技术的价值与市场预测
- 代理AI的核心价值：增强人类智能、自动化繁琐任务、提高效率和决策质量
- 关键市场数据：
  - McKinsey预测代理AI将贡献4.4万亿美元年度利润价值
  - BCG预测未来5年将实现45%的增长
  - Gartner预测到2028年，三分之一的生成式AI交互将通过自主代理进行

### 5:00-8:00 治理挑战分析
- 代理来源追踪困难：组织难以掌握代理的构建者、测试方法和数据来源
- 透明度缺失：从传统ML到生成式AI再到代理系统，可见性逐步降低
- 行为解释性问题：多代理系统自主交互时缺乏决策过程的可解释性
- 测试和质量控制挑战：偏见、漂移和整体健康状况监控困难

### 8:00-12:00 watsonx.governance解决方案演示
- 三大核心能力：
  - 可观测性：持续监控代理性能和输出质量
  - 评估能力：构建时、测试时和生产环境的持续评估
  - 优化功能：识别最适合的代理并提供故障修复指导
- 实时演示治理控制台功能：
  - 代理清单管理和元数据查看
  - 风险评估问卷和自动风险识别
  - 用例审批和代理上线流程

### 12:00-16:00 治理理念转变讨论
- Diana Griffin强调治理应作为"护栏"而非"阻碍"
- 明确的治理框架能够加速创新而非阻碍创新
- 缺乏治理导致的问题：
  - 审批流程冗长（可能长达9个月）
  - 影子AI的产生增加风险
  - 开发者不知道如何合规地进行创新

### 16:00-19:00 实施挑战与最佳实践
- 组织面临的主要挑战：
  - 缺乏AI治理专业知识
  - 治理委员会角色不明确
  - 传统工作方式不再适用
- 代理评估建议：
  - 关注流程痛点而非单纯任务自动化
  - 以结果为导向设计代理系统
  - 避免自动化糟糕的现有流程
- 强调混合开放平台的重要性，支持全环境监控而不限制构建方式

### 19:00-19:24 总结与展望
- 宣布watsonx.governance正在开发AgentOps开发套件
- 强调AI治理工具对于代理系统成功部署的必要性
