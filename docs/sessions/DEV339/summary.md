# AWS re:Invent 2025 技术分享会总结

## 会议概览

本次分享会由来自电气公司的 Yuki Oava 主讲，主题为"为智能家居设备构建聊天应用"。演讲者介绍了一个名为"Cadengers"的创新项目，这个名称结合了日语中的"家电"（Kaden）和英雄团队"Avengers"的概念。该项目旨在重新定义人与智能家居设备之间的关系，从传统的命令-响应模式转变为更温暖、更自然的家庭成员式互动。

Cadengers 是一个聊天应用，让用户能够像与家人交流一样与智能家电互动。每个家电都被赋予了独特的个性和头像，能够以符合其功能特点的自然方式回应用户。例如，冰箱可能会提醒烤箱关于食谱所需的食材。该项目充分利用了 AWS 的多项服务，特别是 Amazon Bedrock Agent，构建了一个包含个性化知识库、记忆日历和智能协调器的完整架构。

演讲者 Yuki Oava 是一位在物联网云平台和智能家居应用领域工作的开发者，同时也是 AWS 社区建设者和日本 AWS 用户组的活跃成员。他在分享中详细展示了项目的概念、用户体验、系统架构以及四大核心功能特性。

## 详细时间线

00:00 - 开场介绍
- 演讲者自我介绍：Yuki Oava，来自电气公司的物联网开发者
- 介绍演讲主题：为智能家居设备构建聊天应用

01:30 - 项目名称解释
- 介绍项目名称"Cadengers"的由来
- 解释这是日语"家电"（Kaden）和"Avengers"（复仇者联盟）的组合词
- 说明 Cadengers 是一个让用户像与家人互动一样与智能家电交流的聊天应用

02:45 - 演讲者背景
- 介绍工作经历：在名为 Lenova 的物联网云平台工作
- 开发智能家居应用 My MU
- 获得 AWS 和 AWS Japan 的多项认证和荣誉
- 活跃于 AWS 社区，包括日本 AWS 用户组

04:00 - 会议议程
- 介绍四个主要议题：产品概念、应用和用户体验、演示视频、整体架构

04:30 - 产品概念阐述
- 说明物联网智能家电让用户可以随时随地轻松控制设备
- 指出传统关系主要是命令和响应模式
- 提出愿景：在人类和家电之间建立更温暖、更自然的关系

06:00 - 设计理念
- 核心理念：让用户更爱他们的家电
- 想象一个用户不仅仅是发出命令，而是与家电对话并建立亲密关系的世界
- 设计目标：鼓励友好且有用的互动，而非令人烦恼的交互

07:15 - 个性化设计
- 每个家电都有反映其功能的个性
- 头像以符合设备特点的自然方式回应用户
- 家电之间也可以相互对话，例如冰箱可能会提醒烤箱关于食谱的食材

08:00 - 演示视频播放
- 播放约两分钟的演示视频
- 展示三个家电角色：Big Chill Taro（冰箱）、Cooney Breeze（空调）、Bounty Vision（电视）
- 演示场景：在寒冷的冬天，家电们推荐火锅和温暖的乌冬面
- 展示家电之间的互动和协作

10:30 - 整体架构介绍
- 说明为每个家电头像部署了独立的 Agent
- 还有一个管理记忆系统和记忆日历的 Agent
- 前端使用 React Native 和 Amplify，由 Amazon Developer 托管和构建
- 对话数据存储在 DynamoDB 中，通过 AppSync 的 GraphQL 连接到前端

12:00 - 无服务器架构
- 后端主要基于 Lambda 和 Step Functions 构建
- 介绍四大核心功能特性

13:00 - 第一个特性：独特个性
- 每个家电 Agent 包含系统提示词和个性化配置文件
- 维护允许列表，Agent 只能调用特定设备的 API
- 个性可以根据用户偏好进行调整
- 使用响应模板和安全约束确保回复安全和适当
- Agent 可以调用外部 API（如开关板 API）执行各种操作

15:30 - 第二个特性：个性化知识库
- 使用 Amazon Neptune 图数据库构建结构化知识模型
- 节点代表用户、家电和性能数据
- 边捕获关系和时间上下文
- 结合基于规则的启发式和学习模型存储重要信息
- 通过分析对话流程，Agent 逐渐学习上下文并改进未来响应

17:45 - 第三个特性：记忆日历
- 对话以日历形式保存，包括摘要、缩略图等
- 为难忘时刻保存图像和短视频片段
- 创建类似故事的记忆，用户可以像浏览图片日记一样浏览
- 按主题分类，Agent 可以检索相关的过去互动以丰富未来对话

19:30 - 第四个特性：协调器（Mediator）
- 保持对话流畅和舒适
- 为候选回复评分，考虑相关性、置信度和个人适配度
- 执行约束条件，例如每个头像限制 10 次回复，防止单个头像主导对话
- 确保自然的轮流发言，防止交叉对话或重复回复

21:00 - 关于 Amazon Bedrock Agent Core 的讨论
- 解释为什么没有使用 Amazon Bedrock Agent Core
- 说明开发时该服务仍处于预发布阶段
- 认为许多功能可以被 Amazon Bedrock Agent Core 替代
- 介绍 Agent Core Runtime 可以运行代理框架
- Agent Core Gateway 可以将现有 API 包装成 MCP 兼容工具
- Memory Agent 提供短期和长期存储，为 Agent 提供适时的上下文

23:30 - 典型交互流程示例
- 用户询问："今晚应该做什么？"
- 冰箱头像检查可用食材并提供建议
- 通知烤箱头像，烤箱处理食谱并安排加热
- 协调器确保每个头像按顺序发言，动作安全且协调
- 展示上下文感知建议和个性化如何让互动更愉快

25:00 - 总结
- Cadengers 是一个与智能家居设备聊天的应用
- 将家用电器转变为具有独特个性的友好 Agent
- 结合 Agent Runtime、MCP 和多模态记忆提供上下文感知和安全的互动
- 结果是更高的用户参与度和更具挑战性的用户体验
- 使用可重用的工具和在 AWS 上可扩展的架构
