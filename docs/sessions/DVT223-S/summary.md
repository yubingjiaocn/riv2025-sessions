# AWS re:Invent 2025 会议总结：使用 Amazon Q Developer 最大化生产力

## 会议概述

本次 AWS re:Invent 2025 分会场由 Pluralsight 的云架构师主讲，重点介绍了如何使用 Amazon Q Developer（现已更名为 Kiro CLI）来提升日常工作效率。演讲者分享了 Pluralsight 最新的技术预测报告，该报告基于对 1500 名技术高管、IT 人员和业务领导者的访谈，指出 AI 的下一阶段将聚焦于治理、道德规范以及投资回报率。报告还显示，连续三年排名前列的技术技能缺口包括网络安全、云基础技能、云架构技能以及 AI 和机器学习。

演讲者强调，在这个快速变化的技术环境中，持续学习和掌握新技能变得前所未有的重要。Kiro CLI（原 Q Developer CLI Agent）作为一个终端聊天代理工具，为开发者和云架构师提供了强大的生产力提升能力。它不仅可以回答问题，还能执行实际操作，如管理 AWS 资源、读写本地文件、审查云配置模板、提供故障排除建议等。演讲者通过多个实际演示展示了如何在日常工作中使用 Kiro CLI，包括技术实现、文档处理、研究开发等多个场景。

## 详细时间线

00:00 - 开场介绍
- 会议主题：使用 Amazon Q Developer 最大化生产力
- 工具更名：Q Developer CLI Agent 现更名为 Kiro CLI

00:30 - 演讲者背景与行业洞察
- 演讲者来自 Pluralsight，专注于帮助企业提升员工技能
- 分享技术预测报告：访谈了 1500 名技术高管和 IT 专业人员
- AI 下一阶段重点：治理、道德规范和投资回报率
- 关键技术缺口：网络安全、云基础技能、云架构、AI/机器学习

02:00 - 现场调查
- 询问观众使用 Amazon Q Developer、Kiro CLI、Claude 模型或其他编码助手的经验

02:30 - 会议议程预览
- Kiro CLI 入门指南
- 示例提示词分享
- 实际演示
- 常见错误及避免方法
- 如何在 AI 时代蓬勃发展
- 学习资源推荐

03:00 - Kiro CLI 入门方式
- 三种使用方式：AWS 控制台、Kiro IDE（VS Code 分支）、Kiro CLI（终端聊天代理）
- Kiro CLI 特点：简洁、无繁琐功能、适合终端工作流

04:00 - 示例提示词场景
- 执行 AWS 账户操作（如克隆 EC2 实例）
- 故障排除支持（AWS 错误、Python 等编程语言错误）
- 读写本地文件（审查 CloudFormation 模板、IAM 策略等）

04:30 - 演讲者工作角色介绍
- 角色定位：云架构师、构建者、内容创作者
- 工作内容：文档处理、研究、幻灯片开发、教程开发、技术实现、AWS 故障排除
- Kiro CLI 应用：覆盖所有工作领域

05:00 - 演示 1：AWS CLI 命令查询
- 场景：忘记如何停止 Amazon Comprehend 情感分析作业
- 使用 AWS 文档 MCP 服务器获取准确、最新的命令信息

06:00 - 演示 2：故障排除访问拒绝错误
- 场景：S3 访问拒绝错误排查
- Kiro 提供分步排查方法：检查 IAM 策略、S3 存储桶策略、验证存储桶所有权、对象锁定等
- 提供可用的 AWS CLI 命令辅助排查

07:00 - 演示 3：IAM 策略安全审查
- Kiro 读取本地 IAM 策略文件
- 识别安全漏洞（如过度宽松的资源规范）
- 提供改进建议和修正代码
- 可选择让 Kiro 直接重写策略文件

08:00 - 演示 4：使用 Boto3 调用 Amazon Bedrock
- 场景：学习如何使用 SDK 调用 Bedrock 基础模型
- Kiro 提供准确的 Python 代码示例
- 使用 AWS 文档 MCP 服务器减少幻觉风险
- 说明不同模型需要不同参数

09:00 - 演示 5：生成合成测试数据
- 场景：为测试用例创建合成数据
- Kiro 快速生成 100 条示例数据
- 询问是否写入本地文件系统

10:00 - 演示 6：Bedrock 最佳实践查询
- 查询常见请求体参数的最佳实践（如 top K、top P）
- 使用 AWS 文档 MCP 服务器多次读取文档
- 提供参数说明、最佳实践和示例配置
- 对 AI 从业者认证考试有帮助

11:00 - 演示 7：创建架构图
- 场景：为三层应用创建架构图（ALB + ECS + Aurora）
- 使用 AWS 图表 MCP 服务器
- 自动生成并保存架构图到本地
- 用于与团队沟通架构设计思路

12:00 - 常见错误 1：幻觉问题
- AI 仍会犯错，不应盲目相信所有输出
- 避免方法：使用高质量、最新的相关数据；使用 MCP 服务器确保答案有依据；要求提供信息来源引用

13:00 - 常见错误 2：过度依赖
- 不应在未验证的情况下过度依赖 AI
- 正确态度：将 AI 视为助手而非权威；AI 支持决策而非替代人类决策

14:00 - 常见错误 3：数据共享风险
- 免费层服务可能使用用户数据重新训练模型
- 建议：检查所有工具的设置选项；提供机密数据前需谨慎

15:00 - 如何在 AI 时代蓬勃发展
- 市场需求：能够使用 AI 工具获得"超能力"的人才
- 关键能力：理解 AI 的优势和局限性；快速适应技术变化；有效使用新工具；将人文关怀、道德和同理心融入技术应用；将 AI 与人类专业知识相结合

16:00 - 行动挑战

- 验证输出结果
- 跟踪生产力提升情况
