# AWS re:Invent 2025 技术会议总结：使用 Amazon Q Developer 最大化生产力

## 会议概述

本次技术分享会由 Pluralsight 的云架构师主讲，主题为"使用 Amazon Q Developer 最大化生产力"。演讲者介绍了 Amazon Q Developer CLI Agent 现已更名为 Kiro CLI，这是一个基于 Claude 模型的终端聊天代理工具。会议重点探讨了如何在日常工作中有效使用 Kiro CLI 来提升生产力，包括 AWS 资源管理、故障排除、代码生成等多个应用场景。

演讲者分享了 Pluralsight 最新的技术预测报告，该报告基于对 1500 名技术高管、IT 人员和业务领导者的调研。报告显示，AI 的下一阶段将专注于治理、伦理和投资回报率，而云服务（特别是 AWS 的 Bedrock 和 SageMaker）将成为 AI 普及的关键推动力。网络安全、基础云技能、云架构技能、云安全技能以及 AI 和机器学习仍然是最重要的技术技能缺口。

## 详细时间线与关键要点

### 0:00-3:00 开场介绍与背景
- 介绍 Amazon Q Developer CLI Agent 更名为 Kiro CLI
- 分享 Pluralsight 技术预测报告的主要发现
- 强调持续学习新技能和趋势的重要性
- 现场调研观众对各种 AI 编程助手的使用经验

### 3:00-6:00 Kiro CLI 入门指南
- 介绍三种使用 Q Developer 的方式：
  - AWS 控制台中的故障排除建议
  - Kiro IDE（VS Code 的增强版本）
  - Kiro CLI（终端聊天代理）
- 重点推荐 Kiro CLI 适合喜欢在终端工作的用户
- 分享实用的示例提示词，包括：
  - 创建 EC2 实例并指定安全组和 VPC
  - AWS 和编程语言错误故障排除
  - 读写本地文件，审查 CloudFormation 模板和 IAM 策略

### 6:00-12:00 实际演示环节
- **AWS CLI 命令查询**：演示如何获取停止 Amazon Comprehend 情感分析作业的命令
- **故障排除指导**：展示如何处理 AccessDenied 错误，获得详细的分步排查方法
- **IAM 策略安全审查**：演示本地文件读取和安全最佳实践建议功能
- **Bedrock 模型调用**：展示如何使用 boto3 调用 Amazon Bedrock 基础模型
- **合成数据生成**：快速创建 100 个测试用例的合成数据
- **Bedrock 最佳实践**：获取请求体参数（如 top-k、top-p）的最佳实践指导
- **架构图生成**：使用 MCP 服务器创建三层应用架构图

### 12:00-15:00 避免常见错误
- **幻觉问题**：强调 AI 仍会犯错，需要事实核查，特别是在生产系统中
- **过度依赖**：将 AI 视为助手而非权威，支持而非替代人类决策
- **数据共享风险**：注意免费版本可能使用用户数据重新训练模型，建议检查设置并谨慎处理机密数据

### 15:00-18:00 AI 时代的成功策略与总结
- 强调对具备 AI 工具使用能力人才的需求不断增长
- 建议从低风险的日常任务开始实验 Kiro CLI
- 重要建议包括：
  - 验证 AI 输出结果
  - 跟踪生产力提升情况
  - 与团队分享成功和失败经验
  - 持续实验，因为技术在不断发展
- 提供学习资源：Pluralsight 课程内容和相关文章
- 会议结束后提供现场交流机会
