# AWS re:Invent 2025 - 云端安全强化：Meta规模化安全实践

## 会议概述

本次技术分享会由AWS解决方案架构师Robin Rodriguez主持，邀请了Meta高级工程经理Peter Neuhausen和AWS安全解决方案架构师Syed Sharif，深入探讨了如何为全球35亿日活用户构建安全的云基础设施。会议重点介绍了AWS云安全手册的核心原则，以及Meta在超大规模基础设施中实施的创新安全解决方案。

Meta作为全球最大的社交媒体平台之一，面临着独特的安全挑战：支持150个国家的用户，处理比传统互联网站点高出数个数量级的交易率，同时确保数据隐私和合规性。会议分享了Meta如何通过混合云架构、自动化安全控制和隐私感知基础设施来应对这些挑战，特别是在AI集群安全方面的实践经验。

## 详细时间线与关键要点

### 0:00-10:00 开场介绍与规模化挑战
- Robin Rodriguez介绍会议背景，强调Meta每日为35亿用户提供安全数字体验的挑战
- 介绍演讲嘉宾：Meta的Peter Neuhausen和AWS的Syed Sharif
- 概述会议议程：AWS云安全手册、Meta的超大规模基础设施挑战、AI集群安全实践

### 10:00-20:00 AWS云安全手册核心原则
- **规模化安全特征**：短暂工作负载、分布式系统和自动化的融合
- **嵌入式安全**：从设计阶段到部署和运行时的全流程安全集成
- **联邦化安全**：安全不再是看门人角色，而是融入云部署结构中
- **弹性设计**：不仅防止问题发生，更要能够承受和恢复
- **学习循环**：从过往事件中学习，创建无责备文化的迭代改进过程

### 20:00-30:00 身份域安全控制
- **身份提供商和单点登录**：使用AWS Identity Center进行统一身份管理
- **组织级策略**：在组织层面阻止恶意行为，在OU和账户层面应用特定策略
- **服务控制策略(SCP)**：防止修改CloudTrail、限制特定加密类型等
- **分层方法**：沙盒环境让开发者自由实验，30天自动清理机制
- **自动化护栏**：用自动化控制替代手动控制以实现安全扩展

### 30:00-40:00 网络域和零信任架构
- **零信任访问**：基于身份而非网络位置的访问控制
- **流量检查模式**：出站、入站和东西向流量的三种检查模式
- **AWS Network Firewall**：限制敏感数据外泄和命令控制活动
- **身份感知服务授权**：通过AWS VPC Lattice实现应用间安全连接
- **跨栈零信任机制**：从边缘到云API的一致授权评估

### 40:00-50:00 数据治理和资源保护
- **数据分类和保护**：根据数据敏感性匹配相应的安全措施
- **KMS密钥管理**：将密钥靠近数据存放，使用SCP和RCP进行治理
- **基础设施即代码验证**：多层验证从开发到部署的全流程
- **不可变基础设施**：部署后不变更，消除未授权修改和点击操作
- **数据边界**：定义可信身份、资源和网络的边界控制

### 50:00-57:00 Meta实际案例和总结
- **Meta规模挑战**：35亿日活用户，全球分布，高交易率特征
- **混合云架构**：结合自有数据中心和AWS云资源
- **AI集群安全**：GPU稀缺性和大数据传输的独特挑战
- **隐私感知架构**：实时应用隐私策略的复杂系统
- **威胁检测和响应**：流式日志分析、实时异常检测和自动化修复

会议强调无论组织规模如何，有效云安全的基本原则保持一致：了解关键资产、实施适当控制、从一开始就将安全构建到流程中。Meta的经验表明，规模化安全不仅是保护更多资源，更是让企业能够自信地增长，确保安全态势与业务同步发展。
