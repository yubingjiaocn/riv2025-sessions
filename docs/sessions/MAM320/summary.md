# AWS re:Invent 2025 技术会议总结：使用 AWS Transform for .NET 进行应用现代化

## 会议概述

本次会议展示了墨西哥领先的人力资源管理公司 Grupores Internacional 如何使用 AWS Transform for .NET 服务成功实现应用现代化的案例。Grupores 是一家成立于1991年的公司，专注于人力资源管理、工资处理和考勤控制解决方案，服务于墨西哥68%的制造业员工，覆盖1200多家客户和440万员工。

会议由 AWS 团队和 Grupores 工程负责人 Armando Valenzuela 共同主讲，详细介绍了从传统 .NET Framework 应用迁移到现代化无服务器架构的完整过程。演讲涵盖了技术挑战、解决方案架构、实施过程以及取得的显著成果，包括70%的人力成本节省、40%的基础设施成本降低和零停机时间的成功迁移。

## 详细时间线与关键要点

### 00:00-05:00 会议开场与公司介绍
- 介绍会议议程和演讲者
- Grupores Internacional 公司背景：成立于1991年，总部位于蒂华纳
- 业务规模：覆盖墨西哥68%制造业员工，服务1200+客户，440万员工
- 技术挑战：拥有超过10年的遗留代码，运行在 Windows 本地应用程序上

### 05:00-15:00 技术挑战与现代化需求
- 系统压力场景：400万用户在周五晚上同时访问应用程序
- 现代化三大支柱：保持客户中心、成本效益、零停机要求
- 应用场景：墨西哥政府要求所有工资单必须向税务部门报告
- 系统架构：传统 XML PDF 引擎处理每日200万请求，峰值400万请求

### 15:00-25:00 迁移准备与代码分析
- 项目准备建议：分析最适合迁移的项目，整理遗留项目依赖
- 代码规模：135,000行代码，70个 NuGet 包依赖
- 清理成果：移除23个不必要的包依赖
- AWS Transform 自动迁移：从 .NET Framework 4.6 升级到 .NET 8

### 25:00-35:00 迁移实施过程
- 时间线：4月开始，2周内完成代码转换和手动修复
- 技术路径：从 Elastic Beanstalk → AWS Fargate → Lambda
- 架构解耦：将单体应用拆分为微服务，避免分布式单体
- 自动化构建：使用 CDK 和 CodePipeline 实现 CI/CD

### 35:00-45:00 解决方案架构与成果
- 最终架构：Lambda 函数 + 独立数据库 + S3 存储
- 性能提升：响应时间保持在0.5秒以内，即使在峰值期间
- 成本节省：使用 Graviton Lambda 降低40%基础设施成本
- 弹性扩展：日均处理1130万份工资文档，无需手动扩容

### 45:00-55:00 AWS Transform 服务演示
- 服务介绍：支持 .NET Framework 到 .NET 8/10 的自动迁移
- 两种体验：IDE 集成版本和 Web 界面版本
- 新功能：多代理能力，允许用户驱动现代化过程
- 技术演示：Visual Studio 中的实际迁移操作流程

### 55:00-57:00 经验总结与未来规划
- 关键经验：现代化不仅是代码迁移，需要考虑架构重构
- 团队协作：涉及架构团队、开发团队和基础设施团队
- 未来计划：继续使用绞杀者模式迁移其他 .NET Framework 项目
- 技术演进：采用可组合的无服务器架构，考虑数据库迁移
