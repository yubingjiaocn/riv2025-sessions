# AWS re:Invent 2025 安全团队扩展会议总结

## 会议概述

本次会议由AWS首席信息安全官Amy主持，与同事Neha Runta和来自礼来公司的Andrea Abel共同探讨了如何扩展安全团队以应对快速变化的技术环境，特别是生成式AI带来的挑战和机遇。会议重点关注了三个核心策略：将安全专业知识嵌入到业务开发工作流程中、保持对风险的敏锐洞察并快速适应变化，以及与业务部门紧密合作解决实际问题。

Amy分享了AWS内部如何通过构建安全原语、嵌入式安全指导和工具，以及适应性安全体验来实现大规模安全运营。Neha介绍了AWS安全代理这一前沿AI工具，它能够将安全意图转化为自动化机制。Andrea则从企业客户角度分享了礼来公司在供应链安全威胁建模方面的实践经验，展示了如何通过与AWS合作来解决复杂的安全挑战。

## 详细时间线与关键要点

### 0:00-10:00 开场与AI对安全的影响
- Amy介绍会议主题和演讲者
- 讨论AI特别是生成式AI如何改变安全游戏规则
- 传统确定性系统防护方法在非确定性系统中的局限性
- 威胁行为者利用Gen AI降低定向攻击成本，提高钓鱼攻击的说服力
- 攻击者开始针对AI工具进行攻击，包括提示注入等新型威胁

### 10:00-20:00 防御者的AI优势与内部实践
- Gen AI帮助防御者快速处理和分析大量信息
- AWS内部将深度日志分析时间从4小时缩短至11分钟
- 自动化凭证泄露响应流程，包括CloudTrail数据检索和实体图生成
- 强调需要在开发速度和威胁行为者进步速度之间保持竞争优势
- 介绍三个核心策略：嵌入安全专业知识、适应性调整、与业务紧密合作

### 20:00-30:00 嵌入安全专业知识的实践
- 批评"左移"概念，强调现代产品开发的非线性特性
- AWS内部三种投资方式：安全原语、生命周期嵌入式指导、适应性构建体验
- S2N TLS作为安全原语的经典案例：从OpenSSL的50万行代码简化到6000行
- 2022年发布S2N Quic，支持后量子密钥交换
- API安全测试自动化：代理分析API变更并自动创建全面测试用例

### 30:00-40:00 合规自动化与主动防御系统
- 利用Gen AI自动化合规评估准备工作，减少构建者参与需求
- 介绍AWS主动防御系统组件：
  - Blackfoot：每日处理312万亿网络流
  - Madpot：每日监控5.5亿恶意活动
  - Mithra：每日处理20万恶意域名
  - Sonaris：每日阻止近50亿次扫描
- 这些系统为AWS Shield、Route 53等服务提供威胁情报

### 40:00-50:00 AWS安全代理发布与演示
- Neha介绍意图与机制的区别概念
- AWS安全代理作为前沿AI代理，能够自主工作并提供上下文感知的安全检查
- 演示设计文档安全需求分析：1分钟内分析11个不适用、5个合规、2个需要更多信息、1个不合规的要求
- 代码审查集成：自动检查是否使用内部安全原语Daffodil
- 支持GitHub集成，可应用于设计审查和代码审查

### 50:00-58:48 适应性安全与业务合作案例
- 讨论外部威胁环境变化：APT 29水坑攻击案例和供应链攻击检测
- 内部开发工作流程变化：Gen AI增加开发者与运行代码之间的距离
- 强调测试质量的重要性和安全团队的新机遇
- Andrea分享礼来公司供应链威胁建模实践
- 通过与AWS合作建立三方协作模式解决供应链安全问题
- 总结三个核心建议：嵌入专业知识、保持风险关注、加强业务合作
