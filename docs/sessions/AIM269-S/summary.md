# AWS re:Invent 2025 技术会议总结

## 会议概述 (Session Overview)

本次技术会议由 Gopal 主讲，重点探讨了人工智能在企业数据应用中面临的核心挑战——AI系统"自信地犯错"的问题。演讲者分析了MIT关于95%AI试点项目失败的报告，并提出了解决方案。会议展示了PromptQL公司如何通过构建能够承认"不知道"的AI系统来解决信任问题，从而实现真正可用的企业级AI数据分析产品。

演讲者强调，虽然AI在原始智能方面已经超越人类，但人类仍然更值得信赖，因为人类具备持续学习和承认无知的能力。关键洞察是：只有当AI承认不知道时，才能对其进行教学和改进，这是构建可信AI系统的核心产品循环。

## 详细时间线与关键要点 (Detailed Timeline)

### 0:00-3:00 问题定义与背景
- MIT 95%AI试点失败报告分析
- AI"自信地犯错"问题的核心挑战
- AI技术快速普及但信任度不足的现状
- 企业用户无法信任AI做出重要业务决策的根本原因

### 3:00-6:00 理论基础与解决思路
- 引用Dwarkesh Patel播客和Andrej Karpathy、Ilya等专家观点
- 人类价值不在于原始智能而在于持续学习能力
- 关键洞察：只有承认"不知道"才能进行教学
- AI产品缺失的核心循环：承认无知→接受教学→持续改进

### 6:00-9:00 PromptQL产品演示准备
- 传统数据分析流程的复杂性问题
- "与数据对话"产品的普遍局限性
- 大多数产品最终沦为需要技术人员验证的文本转SQL工具
- PromptQL的核心运营模式：让AI承认无知，专家进行教学

### 9:00-15:00 核心产品演示
- **知识展示**：蓝色链接显示AI已知概念及来源
- **Wiki系统**：支撑AI的协作式知识库
- **不确定性处理**：红色链接标识未知概念和假设
- **专家协作**：人类专家实时加入对话进行指导
- **学习循环**：对话自动转化为Wiki条目，实现知识积累
- **高级概念处理**：展示复杂业务逻辑和预测模型的处理能力

- **大规模处理能力**：支持10万张表、1万个指标的企业级应用
- **产品差异化**：相比DataBricks Genie、Snowflake Cortex等竞品的优势
