# AWS re:Invent 2025 金融转型会议总结

## 会议概述

本次会议由来自PWC加拿大和蒙特利尔银行(Bank of Montreal)的三位专家共同主持，分享了他们在过去两年中利用AI技术推动金融转型的实践经验。演讲者包括PWC加拿大金融转型实践负责人Reshett、蒙特利尔银行风险与财务首席数据分析官Ram Pedu，以及PWC金融服务云数据和AI实践负责人Ravi。

会议重点探讨了如何将AI作为强大工具应用于金融机构的转型之旅。演讲者强调，在受监管的银行业环境中实施AI面临独特挑战，包括AI治理、数据准确性要求、以及如何在快速变化的技术环境中保持合规性。他们分享了从概念验证到生产部署的完整历程，特别强调了业务采用和用户赋能的重要性。

核心主题围绕三个支柱展开：提高流程效率、增强决策有效性、以及通过AI增强员工体验。演讲者指出，AI的目标不是替代人类，而是增强人类能力，使财务专业人员能够成为更好的故事讲述者和战略思考者。会议还讨论了从单个用例扩展到组织级转型的挑战，以及如何在监管环境中平衡创新与合规。

## 详细时间线

[00:00 - 01:30] 开场与演讲者介绍
- 会议开始，介绍将在20分钟内分享两年的AI转型之旅
- Reshett介绍自己领导PWC加拿大金融转型实践，专注金融服务和财务技术优化
- Ram Pedu介绍自己担任蒙特利尔银行风险与财务首席数据分析官
- Ravi介绍自己领导PWC金融服务云数据和AI实践

[01:30 - 04:00] 银行AI之旅的四个关键问题
- 讨论为什么选择在企业职能部门进行转型
- 探讨过去两年面临的挑战
- 强调从概念到生产实施的障碍
- 重点关注业务用户采用和赋能的重要性

[04:00 - 07:30] AI治理与信任框架
- Ram强调作为受监管金融机构，首要任务是建立AI信任框架
- 讨论AI治理的关键要素：公平性、透明度和问责制
- 指出从试点到生产的最大挑战是缩短交付周期
- 强调在金融和风险领域，数字准确性至关重要，甚至要精确到小数点
- 讨论如何为AI输出建立信任和护栏机制

[07:30 - 10:00] 从单一用例到组织转型
- Ravi讨论从个别用例转向整个组织转型的转变
- 强调不同用例之间的协同效应，如内容生成和内容摘要
- 介绍如何在多个用例之间构建通用能力和模式
- Ram补充说明构建可跨用例重用的能力（如基于Amazon Bedrock的RAG和代理架构）

[10:00 - 13:00] 金融转型的三大支柱
- Reshett回顾金融领域自1999年Y2K以来的首次重大转型
- 介绍用例识别的三个支柱：
  1. 提高流程效率（如结账流程、差异分析）
  2. 增强有效性（确保准确性和精确度）
  3. 增强员工体验（赋能人才，培养软技能）

[13:00 - 16:30] 用例优先级排序框架
- Ram详细说明用例评估的四个维度：
  1. 业务价值（效率提升）
  2. 可行性（技术能力和实施时间）
  3. 业务一致性（团队准备度和高管支持）
  4. 风险评估（监管压力和合规要求）
- 强调随着经验积累，在每个维度上都在不断改进

[16:30 - 19:00] 监管环境中的技术创新挑战
- Reshett提出在快速变化的技术环境中的挑战
- 讨论如何平衡"AI爱好"与"AI实践应用"
- Ravi从两个角度分析：用例生命周期和软件开发生命周期(SDLC)
- 强调需要批判性思考在开发生命周期的哪个阶段使用新工具
- 指出在需求生成和动态模型选择阶段使用新工具风险较高

[19:00 - 22:00] 加速交付速度
- Ram分享将3个月工作压缩到8小时的政策管理案例
- 讨论如何挑战传统交付时间：将月级工作压缩到周级，周级压缩到天级
- 强调快速迭代的重要性，避免用户在3个月后才首次看到产品
- 提到黑客马拉松思维：在受限环境和时间内快速构建可展示的原型

[22:00 - 24:30] 用户教育与变革管理
- Ram强调用户期望在工作中使用日常生活中的AI工具
- 介绍BMO内部的"AI for All"和"AI for Finance and Risk"培训课程
- 讨论从基础到高级（包括代理AI）的分层学习路径
- 强调变革管理的核心是教育和赋能用户
- 提到Microsoft Co-pilot在全行的推广

[24:30 - 26:30] 跨职能协作的重要性
- Ram强调从第一天起就需要引入正确的利益相关者
- 列举关键合作伙伴：监管部门、风险团队、模型治理、法务、内部审计
- 指出成功不在于单独构建用例，而在于建立正确的协作机制
- 强调不能孤立开发后期望产品自然成功

[26:30 - 28:00] AI对传统工作流程的影响
- Ravi讨论数据迁移和云迁移工作流程的变革
- 指出源目标映射、测试脚本生成、目标代码生成都可以端到端自动化
- 强调不再需要相同数量的数据工程师和测试人员
- 目标是更快、更便宜、更高效和更准确地完成工作

[28:00 - 30:30] 数据敏感性与治理
- Reshett提出金融未发布数据被视为注册机密的挑战
- 讨论不能在低级环境中使用生产数据集的监管要求
- Ram回应称基于角色的访问控制已存在，但代理AI可能需要增强护栏
- 举例说明需要为代理设置交易审批阈值策略（如100美元以下自动批准）
- 强调AI治理的核心：公平性、透明度、问责制

[30:30 - 32:00] 合成数据的应用
- Ram讨论在MVP阶段使用高质量合成数据的可能性
- 指出当前合成数据能力比3-4年前有显著提升
- Ravi补充说明与Anthropic合作构建代理系统
- 讨论不仅生成合成数据，还要使其尽可能具有代表性
- 介绍使用代理生成ML模型微调参数的创新方法

[32:00 - 33:30] 未来展望与总结
- Reshett询问未来5年金融领域的发展方向
- Ram回应：去年是实验年，今年将聚焦投资回报率(ROI)
- 强调从业务战略出发，识别高价值、高可行性的用例
- Ram感谢PWC和AWS在加速工作中提供的支持
- Reshett总结：从"人员、流程、技术"转向"人员、流程、绩效"
- 强调技术如同电力，关键在于如何最佳利用

[33:30 - 34:00] 会议结束
