# AWS re:Invent 2025 会议总结：使用 Slack 加速 AI 辅助开发

## 会议概述

本次会议主要探讨了如何利用 Slack 平台来增强 AI 辅助开发流程，以及如何在 Slack 中部署和构建 AI 代理（agents）。演讲者介绍了 Slack 作为事件驱动平台的核心优势，特别强调了其在 AI 开发领域的天然适配性。Slack 平台已有 8-10 年历史，最初称为"机器人"（bots）的非人类参与者，现在演变为集成了大语言模型的"代理"（agents），但底层技术架构基本保持一致。

Slack 在 AI 开发中的最大优势在于提供开箱即用的用户界面，让开发者可以专注于业务价值和新技术的实现，而无需担心 UI 层面的问题。平台支持多种 AI 解决方案的集成，包括现成的产品、自建系统、以及第三方合作伙伴的服务。此外，Slack 提供了丰富的上下文信息（用户关系、频道关联、历史消息等），这对于构建真正有用的上下文感知型代理至关重要。演讲者通过实际演示展示了 GitHub Copilot 代理如何在 Slack 频道中自动处理代码问题、创建 PR，以及如何利用多个 AI 工具（Claude、Cursor、Codex 等）协同工作。

会议还重点介绍了 Slack 平台的开发便利性，特别是通过 WebSocket 模式和 AI 辅助编码工具（如 Cursor），开发者可以在几分钟内从零开始构建一个功能完整的 Slack 应用。这种快速迭代能力对于 AI 代理开发尤为重要，因为它允许团队快速测试想法、获取用户反馈，避免陷入长期的规划和架构设计而错失市场机会。

- 新 UI 元素：流式输出、加载状态、思考字符串等

- Channel Expert 代理：自动回答频道中的常见问题，减少干扰

- 解决企业环境中开放端点的审批难题
