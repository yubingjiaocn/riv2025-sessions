# AWS re:Invent 2025 技术会议总结：使用 KNAI 和 Hyper-Execute 扩展自动化测试

## 会议概述

本次技术会议由 Lambda Test 产品管理总监 Shanta Uwali 主讲，重点介绍了如何通过 KNAI 和 Hyper-Execute 平台解决现代测试策略中的关键挑战。会议深入探讨了当前测试环境面临的三大核心问题：分散的监督管理、不一致的测试标准，以及脆弱且不可扩展的基础设施。

演讲者强调了生成式 AI 如何重塑工程实践，特别是在代码量激增的背景下，传统测试方法已无法满足现代需求。Lambda Test 提出的解决方案是基于意图驱动的现代测试工作流，通过 KNAI 进行测试规划和编写，结合 Hyper-Execute 进行执行和分析，支持功能测试、API 测试、可访问性测试、视觉回归测试和性能测试等多种测试类型。

## 详细时间线与关键要点

### 0:00-2:30 - 问题识别与背景介绍
- 介绍演讲者身份和会议主题
- 阐述当前测试策略的三大核心问题：
  - 分散的监督管理导致测试流程断裂和盲点
  - 不一致的测试标准造成团队间差异性
  - 脆弱的传统基础设施无法满足现代测试需求

### 2:30-4:00 - 测试现状深度分析
- 揭示测试工程师时间分配的真实情况
- 指出大部分时间消耗在修复不稳定测试、测试优化和调试上
- 强调生成式 AI 带来的代码量激增对测试的新挑战

### 4:00-6:00 - Lambda Test 解决方案介绍
- 展示基于 KNAI 和 Hyper-Execute 的现代意图驱动测试工作流
- 说明支持多种测试类型的综合能力
- 介绍系统如何处理多媒体输入、PRD 文档和 Jira 需求

### 6:00-10:00 - 实时演示：从需求到自动化

- 演示去重功能和人工审核流程
- 展示跨平台支持（Web、移动原生、混合应用）

### 10:00-13:00 - 高级功能展示
