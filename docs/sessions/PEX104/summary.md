# AWS re:Invent 2025 - 生成式AI卓越实践分享会总结

## 会议概述

本次分享会由AWS AI能力认证项目全球负责人主讲,重点探讨了生成式AI和智能体AI(Agentic AI)的采用趋势、客户面临的挑战以及合作伙伴在AI转型中扮演的关键角色。演讲者基于McKinsey、BCG、IDC和Deloitte等机构的调研数据,以及AWS团队连续三年委托进行的专项研究,深入分析了企业从生成式AI向智能体AI过渡的现状。

会议强调,虽然AI技术的采用势头强劲,近75%的企业已在至少一个业务功能中使用生成式AI应用,50%的企业已将概念验证(POC)转化为生产环境,但客户在规模化部署过程中仍面临数据基础、模型可扩展性、技术复杂性等多重障碍。演讲者特别指出,85%的客户预计在未来1-3年内将维持或增加对合作伙伴的依赖,合作伙伴在从构思到部署的每个阶段都发挥着不可或缺的作用。

此外,会议还介绍了AWS为合作伙伴提供的资源和支持体系,包括AI能力认证计划、技能培训课程、知识中心以及合作伙伴成长路径图,帮助合作伙伴提升技术能力、获得市场认可并与AWS销售团队协同服务客户。随着智能体AI时代的到来,90%的企业计划在未来6-12个月内实施智能体AI解决方案,这为合作伙伴创造了巨大的市场机遇。

## 详细时间线与关键要点

开场与背景介绍 (0:00-2:30)
- 演讲者介绍自己担任AWS AI能力认证项目全球负责人,该项目旨在帮助客户找到具有专业技术能力和商业成功经验的合作伙伴
- 最近宣布推出智能体AI类别认证,欢迎具备智能体AI能力的合作伙伴加入
- 现场调查显示听众主要为AWS合作伙伴

AI采用趋势与数据 (2:30-5:00)
- 近75%的客户在至少一个业务功能中使用生成式AI应用
- 50%的客户已将POC转化为生产环境
- 40%已部署生产的客户正在实现预期收益
- AI采用从战略阶段进入实际行动阶段,高管层承诺增强

智能体AI的兴起 (5:00-7:00)
- 60%的企业设立了首席AI官(Chief AI Officer)职位
- 智能体AI已经到来并投入使用,合作伙伴拥有成功案例
- 90%的企业计划在未来6-12个月内实施智能体AI
- 生成式AI技术已趋于成熟,部署时间缩短,快速见效成为可能

技术转型的挑战 (7:00-9:30)
- 智能体AI被认为更加复杂,需要技术、人员和流程的全面转型
- 虽然许多客户认为自己能够实现规模化,但实际上只有40%真正达到规模化生产
- 常见障碍包括:数据基础薄弱、模型可扩展性问题、技术挑战
- 客户反馈:制作MVP容易,但规模化部署和广泛采用具有挑战性

部署痛点分析 (9:30-11:00)
- 技术痛点:实施周期长、变更管理困难、解决方案难以使用、令牌限制、客户支持不足
- 50%的主要关注点与人员和流程相关,而非纯技术问题
- 需要合作伙伴针对每个痛点提供解决方案和流程

合作伙伴的关键作用 (11:00-14:00)
- 85%的客户预计在未来1-3年内维持或增加对合作伙伴的依赖
- 合作伙伴在AI采用的每个阶段都至关重要:从构思、探索用例、开发POC到全面部署
- 94%的客户使用多个合作伙伴,包括基础模型提供商、技术公司、ISV、服务提供商、系统集成商等
- 90%的客户认为合作伙伴是克服部署挑战的关键

生产路径上的挑战总结 (14:00-16:00)
- 构思阶段:创造业务案例价值
- 数据策略:建立强大的治理流程
- 安全与合规:不能事后考虑,需要前置
- 明确成功指标和交付预期
- 变更管理的重要性
- 技能差距问题突出

AWS合作伙伴资源与支持 (16:00-20:00)
- 推出全新的AI合作伙伴成长路径图(类似"蛇梯棋"游戏设计)
- 四大支柱:构建(Build)、市场推广(Market)、联合销售(Co-sell)、成长(Grow)
- 帮助合作伙伴开发AI就绪解决方案、制定市场策略、与AWS销售团队协同、获得高级市场推广机会
- 提供QR码链接到合作伙伴中心获取详细资源

AI能力认证的价值 (20:00-22:00)
- AI领域竞争激烈且嘈杂,能力认证是"噪音中的信号"
- 帮助客户识别经过严格验证并具有商业成功记录的合作伙伴
- 调研显示客户期望合作伙伴具备:快速执行AI项目的能力、专门的AI实践团队、云服务商(如AWS)的验证
- 推出新的AI能力认证徽章(取代生成式AI能力认证),区分服务类和软件类合作伙伴

个人技能认证 (22:00-23:00)
- 除组织级认证外,还可为开发人员和个人获得认证徽章
- 通过AWS Skill Builder提供高管、销售和技术路径的学习课程
- 新发布的智能体AI课程值得关注
- 客户数据显示对个人认证的期望不断增加
