# AWS re:Invent 2025 会议总结：使用 Check Point CloudGuard WAF 保护现代 Web 应用程序、API 和 GenAI 环境

## 会议概述

本次会议由 Check Point 的 Eduardo Redondo Garcia（全球云安全架构负责人）和 Rolando Panez（拥有 15 年以上经验的资深专家）共同主讲，重点介绍了如何在现代云环境中保护 Web 应用程序、API 和生成式 AI 应用。

会议强调了传统基于签名的 WAF 解决方案在面对现代威胁时的局限性，特别是在生成式 AI 时代。演讲者详细介绍了 Check Point CloudGuard WAF 如何通过双层 AI 模型提供零日保护，同时实现极低的误报率。会议还涵盖了最近收购的 Lakera 公司如何增强对 GenAI 应用的保护能力。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与背景
- Eduardo 介绍会议主题：保护现代 Web 应用程序、API 和 GenAI 环境
- 强调云环境为企业创新带来机遇的同时也带来了新的安全挑战
- 提出三个主要学习目标：AI 驱动的 WAF 方法、灵活的部署模式、GenAI 应用保护创新

### 5:00-10:00 团队介绍与市场观察
- Rolando 分享 15 年来客户从本地迁移到云端的经验
- 对比去年和今年客户对 GenAI 的态度变化：从"不知道如何使用"到"已有解决方案但担心安全问题"
- 引出对 GenAI 应用安全保护需求的讨论

### 10:00-20:00 CloudGuard WAF 的独特 AI 方法
- Eduardo 详细解释双层 AI 架构：
  - 第一层：攻击指标分析，基于数百万 HTTP 流量训练的 AI 模型
  - 第二层：上下文分析，学习特定应用环境和用户行为
- 强调完全摒弃基于签名的方法，实现零日保护
- 展示性能数据：99.4% 的检测率，低于 1% 的误报率

### 20:00-30:00 客户案例与零日保护能力
- 分享电商客户案例：零调优情况下部署，两周内仅出现一次误报
- 重点讨论 Log4Shell 事件：CloudGuard WAF 在漏洞公开前就已提供保护
- 解释 AI 模型如何在没有签名更新的情况下检测变种攻击

### 30:00-40:00 AI 引擎工作原理与 API 保护
- Rolando 详细介绍双层 AI 的工作机制
- 第一层：8000+ 攻击指标的监督学习模型
- 第二层：无监督学习模型，分析用户行为、群体行为、可信用户和应用内容
- 通过 SQL 注入误报示例展示 AI 如何实时学习和调整
- Eduardo 介绍 API 保护功能：自动发现、模式推荐、影子 API 检测

### 40:00-50:00 部署选项与客户成功案例
- 介绍多种部署模式：
  - WAF 即服务：利用 AWS 基础设施，15 分钟内完成部署
  - 网关模式：VM 部署，支持自动扩展
  - 代理集成：nginx、Kong 等反向代理的纳米代理
  - 容器化部署：Docker、AWS Fargate、EKS 入口控制器
- BBVA 银行案例：解决遗留应用保护、可扩展性和可视性需求

### 50:00-56:00 GenAI 安全挑战与 Lakera 解决方案
- Eduardo 对比传统 Web 应用与 GenAI 应用的根本差异
- 传统应用：代码是可执行文件，攻击通过代码进行
- GenAI 应用：自然语言是可执行文件，通过提示注入攻击
- 介绍 Lakera 收购：专门针对 GenAI 环境的原生 AI 安全解决方案
- Gandalf 游戏：世界最大的 AI 红队对抗提示注入攻击平台
- 双层 GenAI 保护：提示注入检测 + 语义上下文分析
- 支持 100+ 语言的提示注入保护，延迟低于 50 毫秒
- 产品演示：展示 WAF 控制台、攻击检测和防护效果
