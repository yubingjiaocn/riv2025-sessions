# AWS re:Invent 2025 SentinelOne 自主SOC会议总结

## 会议概述

本次技术会议由SentinelOne的产品管理负责人Nick和高级解决方案工程总监Joe Poyner主讲，主题为"自主SOC：从工具集合转向自主安全运营中心"。会议深入探讨了当前安全运营中心面临的挑战，包括手动流程繁重、分析师工作不可持续等问题，以及攻击者如何利用生成式AI技术提升攻击效率。

演讲者重点介绍了SentinelOne的Purple AI平台如何通过生成式AI和智能工作流实现SOC自动化，从传统的工具辅助模式转向AI主导的自主安全模型。会议展示了从数据收集、威胁检测到事件响应的完整自动化流程，并通过实际演示说明了AI如何将原本需要数小时的安全分析工作缩短至几分钟。

## 详细时间线与关键要点

### 0:00-5:00 开场与问题定义
- 演讲者介绍：Nick（产品管理负责人）和Joe Poyner（高级解决方案工程总监）
- 现场调研：询问观众中有多少人在SOC工作过
- **核心问题识别**：
  - SOC分析师工作极度不可持续
  - 手动流程繁重，需要处理大量告警
  - 威胁调查和响应耗时数小时甚至数天
  - 缺乏主动风险管理能力

### 5:00-10:00 攻击态势分析
- **攻击者优势**：不受治理、政策和组织架构限制
- **新兴威胁**：
  - Google威胁情报报告的多态恶意软件Prompt FLUX
  - Anthropic报告显示攻击者使用AI自动化攻击
  - 攻击者能够自动化90%的基础工作
- **社会工程攻击演进**：
  - 传统垃圾邮件通过拼写错误识别的时代已过
  - 生成式AI创建的钓鱼邮件和图像难以识别
  - 攻击质量显著提升

### 10:00-15:00 Purple AI解决方案介绍
- **Purple AI核心能力**：
  - 自然语言界面

  - 基于AWS Bedrock训练的专有模型Ultraviolet
- **自主SOC成熟度模型**：
  - Level 0：完全手动
  - Level 1-2：自动化剧本和AI辅助
  - Level 3：部分自主（当前水平）
  - Level 4：高度自主（开发中）

### 15:00-20:00 技术架构与性能
- **数据管理**：通过Observo AI实现"完美数据，零噪音"
- **查询引擎**：
  - 行业领先的AI SIEM查询性能
  - 20倍性能提升

- **实际案例分享**：
  - Joe分享Azure基础设施误报事件
  - 传统调查需要4-5小时人工工作
  - AI可将此类工作自动化

### 20:00-25:00 实时演示
