# AWS re:Invent 2025 数据库现代化会议总结

## 会议概述

本次会议由CCC Intelligent Solutions的技术团队和战略合作伙伴Cognizant共同分享了他们从Oracle数据库迁移到AWS Aurora PostgreSQL的完整现代化之旅。CCC作为汽车保险理赔和碰撞修复行业的SaaS解决方案领导者，每年处理约1800万个理赔案例，每日处理5万笔交易，数据库每秒处理5万次事务。

演讲者Subrat详细介绍了CCC如何通过三年的现代化项目，成功将超过150TB的数据从Oracle RAC集群迁移到Aurora PostgreSQL，实现了运营效率的显著提升。Cognizant的Chandana Paul作为战略合作伙伴，补充说明了整个迁移过程中的协作模式和最佳实践。

## 详细时间线与关键要点

### 0:00-5:00 公司背景与数据重要性
- CCC Intelligent Solutions是汽车理赔和碰撞修复行业的SaaS解决方案领导者
- 服务对象包括汽车保险公司、修理厂、汽车制造商和零部件供应商
- 每年处理1800万个理赔案例，日均5-5.5万个案例
- 与300家顶级汽车保险公司、35000家修理厂、5000家供应商和12家主要汽车制造商集成

### 5:00-10:00 云现代化三阶段策略
- 第一阶段：基础设施现代化，从本地迁移到AWS云
- 第二阶段：应用现代化，从Oracle WebLogic迁移到微服务和JBUS架构
- 第三阶段：数据库现代化，从Oracle迁移到Aurora PostgreSQL
- 数据库作为最后阶段的原因：核心业务数据的关键性

### 10:00-15:00 数据库现代化挑战分析
- 识别所有数据库集成点和依赖关系
- 分析超过5万个Oracle原生SQL语句的兼容性
- 处理超过150TB的数据迁移，其中单个数据库接近100TB
- 确保迁移过程中SLA/SLO不受影响

### 15:00-20:00 风险缓解与测试策略
- 构建独立的测试环境模拟生产负载
- 分析并迁移30-40TB的二进制数据（BLOB/CLOB）到S3
- 处理Oracle到PostgreSQL的数据类型转换挑战
- 解决无主键表的数据传输问题

### 20:00-25:00 迁移工具与技术实施
- 使用AWS DMS进行数据传输，需要大量定制配置
- 利用AWS SCT工具进行初始模式转换
- 处理跨15个数据库、50个模式的复杂环境
- 分阶段迁移策略，优先处理独立应用

### 25:00-30:00 现代化收益与成果
- 实现运营效率提升和性能优化
- 优化成本策略，改善FinOps预测能力
- 增强自动化和DevOps能力
- 改善RTO/RPO指标和灾难恢复能力
- 提升安全合规性和扩展能力

### 30:00-35:00 基础设施对比分析
- 迁移前：9节点Oracle RAC集群，100多个WebLogic虚拟机
- 迁移后：2实例Aurora PostgreSQL（1个写入实例+1个读取实例）
- 实现水平扩展能力和无头灾难恢复
- 数据同步延迟降至毫秒级别

### 35:00-40:00 运维效率提升
- 自动化补丁管理，减少30%以上的补丁工作量
- DBA团队转型为数据库可靠性工程师
- 简化灾难恢复测试，减少30%以上的工作量
- 摆脱供应商锁定，采用开源技术栈

### 40:00-44:30 合作伙伴经验分享与总结
- Cognizant作为战略合作伙伴的端到端参与
- 现代化超过200个应用程序，修复500多个SQL查询
- 通过EKS环境实现零停机时间部署
- 24/7 SRE支持确保平台可靠性
- 为计划数据库现代化的组织提供建议和最佳实践
