# AWS re:Invent 2025 分会场总结：使用无服务器架构和生成式AI构建智能制造系统

## 会议概述

本次会议由AWS解决方案架构师Muhammad Salah主讲，重点探讨了如何利用AWS无服务器架构结合生成式AI技术解决制造业中的计划外停机问题。演讲者以烘焙饼干的个人经历引入主题，指出制造业中普遍存在的数据孤岛、知识断层和自动化不足等问题。据统计，全球前500家制造商每年因计划外停机损失高达1.4万亿美元，相当于西班牙的GDP总量，约占其收入的11%。

演讲详细介绍了一套完整的解决方案架构，通过AWS IoT Garnet框架统一不同数据源的集成，利用Amazon Bedrock进行模型微调和部署，构建数字孪生系统和智能对话应用。该方案能够实现生产线数据的实时连接、专家知识的共享传承，以及基于AI的自动化决策和通知系统。整个实施过程仅需7天即可完成，为制造企业提供了快速落地的智能化转型路径。

核心价值在于打破数据孤岛，将分散在各个机器和系统中的数据整合到统一的数据湖中，通过微调的AI模型为操作员提供实时指导和自动化建议，并支持持续学习机制不断优化模型准确性。

## 详细时间线

00:00:00 - 开场故事：烘焙饼干的启示
- 演讲者分享与女儿烘焙饼干时遇到的问题：一半饼干被烤焦
- 引出制造业中的类似困境：数据存在但相互孤立，无法诊断根本原因

00:01:30 - 问题定义：制造业的"沉默时刻"
- 介绍计划外停机（unplanned downtime）的严重性
- 全球前500家制造商每年损失1.4万亿美元，占收入的11%
- 相当于西班牙的GDP总量

00:02:45 - 三大核心问题分析
- 数据孤岛：生产线上每台机器有独立数据库，缺乏端到端关联
- 技能断层：依赖资深专家经验，知识无法有效传承给初级操作员
- 可见性缺失：多数据源但缺乏分析能力，导致订单延误

00:04:20 - 解决方案起点：数据湖架构
- 整合多种数据源：遥测数据、告警、环境数据、文档和标准操作程序
- 面临集成挑战：现代IoT协议（MQTT、LoRaWAN）与传统协议（SFTP）并存

00:05:30 - AWS IoT Garnet框架引入
- 提供统一API吸收不同类型的集成
- 将数据转换为建模数据，在图数据库中建立IoT设备间的关系
- 支持多种数据消费者：QuickSight仪表板和数字孪生系统

00:06:45 - 数字孪生展示
- 左侧显示实际生产线状态
- 饼干检测器报告饼干破裂，机器自5点起停机
- 实现数据点连接，打破数据孤岛

00:07:30 - 生成式AI应用：弥合技能差距
- 在Amazon Bedrock上部署GenAI应用
- 微调模型并导入Bedrock
- 提供文本和语音交互界面，让操作员可以与数据对话
- 通过Garnet框架实时流式传输数据到Bedrock引擎

00:09:00 - 自动化通知系统
- 每5分钟自动分析生产线状态
- 根据捕获的数据提供建议
- 向操作员发送通知以采取相应行动
- 支持AI直接调用API控制机器

00:10:15 - 工作流程详解
- 饼干检测器使用视觉语言模型（VLM）检测缺陷：破裂、变形、气泡
- 调用Bedrock获取多数据源：生产线SOP、机器告警、遥测数据、设备手册
- 自动发送通知给操作员
- 提供文本和语音聊天应用
- 自动执行纠正措施

00:11:30 - 详细架构：数据采集层
- 设备测量数据通过AWS IoT Core发送
- Lambda函数和SQS进行数据模型转换
- 标准化数据模型存储到S3数据湖
- SQS实现解耦，确保数据一致性

00:13:00 - 详细架构：操作执行层
- 通过API Gateway更新生产线操作
- 可调整混合器温度、扭矩等参数

00:14:00 - 数据准备与模型微调
- 目标：让模型理解如何响应查询
- 结构化响应以指导操作员
- 确保指令合规、安全并分类
- 从多种文档（PDF、CSV、文本）读取数据
- 使用Step Functions作为数据管道
- 调用模型生成结构化数据（避免"幻觉机器"）
- 输出训练数据和测试数据

00:16:30 - SageMaker AI模型操作
- 使用SageMaker AI Foundation Model Operations进行指令微调
- 生成模型工件和更新的权重
- 将自定义模型导入Amazon Bedrock
- 选择无服务器实现而非EC2部署，避免GPU利用率计算复杂性
- 提供统一API调用不同版本的导入模型

00:18:00 - Web应用层
- React前端托管在S3上
- Lambda函数调用模型
- 使用Converse API提供灵活性和可扩展性
- 支持模型版本切换和回滚，无需更改API签名

00:19:30 - 通知系统详解
- 每5分钟向模型提供捕获的数据
- 模型消化信息并与微调数据比较
- 提供明确建议，考虑合规性、严重性和所需步骤
- 通过SNS向操作员发送通知
- 支持自动执行所需操作

00:21:00 - 完整架构总结
- 数据采集：Garnet框架统一集成
- 基础层：数据准备和模型微调
- 应用层：Web应用连接Bedrock自定义模型
- 实现机器、传感器和系统使用同一语言

00:22:00 - 方案价值
- 连接的数据点和机器集中在一处
- 通过微调模型获得智能洞察
- 理解并解释生产线实际情况
- 提供下一步建议和推荐

00:23:00 - 持续学习机制
- 系统持续生成更多数据
- 操作员反馈机制（点赞/点踩）
- 基于反馈更新微调模型
- 不断提高模型准确性

00:24:00 - 资源分享
- 提供三个QR码：
  1. Garnet框架GitHub仓库
  2. 小语言模型实现和Bedrock自定义模型导入指南
  3. 端到端机器学习管道和持续微调实现

00:25:00 - 行动呼吁与总结
- 建议从小规模开始：单一生产线、多数据源
- 实施微调和模型导入
- 整个实施过程仅需7天
- 获得更多洞察并持续增强模型
- 会议结束
