# AWS re:Invent 2025 Redis 现代化缓存技术分享会总结

## 会议概述

本次技术分享会由Redis产品管理团队负责人Pieter Cailliau和ifood机器学习工程师Heber Scachetti联合主讲，主题为"使用现代化缓存驱动实时应用"。会议深入探讨了Redis从传统缓存到现代化数据平台的演进历程，重点介绍了Redis在解决热键问题、数据同步、向量搜索等方面的最新技术突破。

Pieter首先从Redis的核心价值出发，分析了传统OLTP工作负载中80-90%为读操作的现状，说明了缓存技术的重要性。随后详细介绍了Redis查询引擎、客户端缓存、数据集成等关键技术创新。Heber则从实际应用角度，分享了ifood作为拉丁美洲最大食品配送公司如何利用Redis支撑每月1.6亿订单的技术实践，包括GenAI平台、RAG系统和特征存储等核心应用场景。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与Redis概述
- Redis作为内存数据结构存储，支持缓存、会话存储、消息代理等多种用途
- 每日2.25百万Docker拉取量，总计超过100亿次下载
- Stack Overflow评选为最受欢迎的非关系型数据库
- 产品团队专注三个方向：核心Redis、开发者体验、数据迁移

### 5:00-10:00 热键问题分析
- Microsoft数据显示80%的OLTP工作负载为纯读查询
- Oracle声称典型OLTP工作负载中70-90%为读操作
- 热键问题的两种典型场景：高频访问键（如社交媒体影响者资料）和索引交集操作
- Redis单线程架构的扩展限制

### 10:00-15:00 客户端缓存解决方案
- 客户端缓存将热键副本保存在客户端应用本地内存
- 避免网络往返，适用于读密集型工作负载
- Redis跟踪每个客户端的缓存键，写入时发送失效通知
- Redis 7.4版本开始在主要客户端库中提供支持

### 15:00-20:00 Redis查询引擎
- 原名Redis Search，现称Redis查询引擎
- 支持定义模式和索引字段，索引不占用键空间
- 提供协调器进行分布式查询处理
- 支持文本、标签、数值、地理位置和向量等字段类型

### 20:00-25:00 向量搜索与语义缓存
- Redis向量搜索性能基准测试显示优于竞争对手
- LangCache语义缓存产品，缓存LLM响应
- 支持OpenAI、Anthropic等嵌入模型
- Asurion客户案例：70%缓存命中率，响应时间减半

### 25:00-30:00 缓存模式演进
- Cache-aside模式的局限性：陈旧数据和雷群问题
- Refresh-ahead模式的优势：数据一致性更好
- 传统解决方案的复杂性：Chrome作业、Kafka架构、ETL工具

### 30:00-35:00 Redis数据集成(RDI)
- 基于Apache Flink构建的低代码解决方案
- 支持CDC捕获和数据转换
- Axis Bank案例：响应时间从173ms降至29ms，吞吐量从500 RPS提升至数万RPS
- 现已进入公开预览阶段

### 35:00-40:00 Redis Flex与JSON数据结构
- 自动分层存储，成本降低75%
- 支持最大50TB数据库，最小250GB起步
- 原生JSON数据结构，支持原子更新和部分检索
- 向量压缩技术，内存使用减少92%

### 40:00-45:00 性能优化与ifood实践案例
- Redis 8.4版本单进程超过125万操作/秒
- TLS性能提升2倍，延迟改善高达90%
- ifood应用场景：GenAI平台速率限制、RAG向量存储、特征存储
- 支撑每月1.6亿订单，特征存储提供500万请求/秒服务
