# AWS re:Invent 2025 中小企业AI转型指南会议总结

## 会议概述

本次会议由AWS全球合作伙伴中小企业专家Nikki Mosheri主讲，重点探讨中小企业如何通过AWS合作伙伴开启AI转型之旅。会议强调，中小企业在全球范围内占据重要地位——占全球企业总数的98%，雇佣全球67%的劳动力，创造45%的全球GDP。尽管77%的中小企业认识到AI的价值，并将17%的IT预算投入AI（计划增长超25%），但超过80%的企业因复杂性、人才短缺和担心犯错而错失AI机会。

Nikki提出了一个基于成功案例的七步框架，帮助中小企业系统化地实施AI战略。她特别强调中小企业不应独自进行AI转型，而应借助AWS SMB能力合作伙伴的专业支持。这些合作伙伴经过严格的技术审核，专门为中小企业提供可自部署的解决方案，并在全球范围内提供服务。会议的核心信息是：AI转型触手可及，中小企业可以通过正确的方法和合作伙伴支持引领AI革命。

## 详细时间线与关键要点

00:00:00 - 开场介绍
- 演讲者Nikki Mosheri自我介绍，说明其作为AWS全球合作伙伴中小企业专家的职责
- 会议目标：为中小企业提供AI实施清晰路径，帮助AWS合作伙伴了解如何支持中小企业

00:01:15 - 中小企业的全球重要性
- 中小企业占全球企业总数的98%
- 雇佣全球67%的劳动力
- 创造45%的全球GDP

00:02:00 - 互动环节：AI挑战现状调查
- 现场调查显示大多数参会者被告知AI将改变业务
- 许多人不知从何开始
- 普遍存在对犯错的担忧

00:02:45 - 中小企业AI投资现状
- 77%的中小企业期望AI帮助完成端到端工作流程
- 平均将17%的IT预算分配给AI
- 计划增加超过25%的AI投资

00:03:30 - 中小企业面临的挑战
- 超过80%的中小企业因AI复杂性错失机会
- 主要障碍：技术复杂性、人才短缺、担心犯错
- 需要直接明了的AI实施方法

00:04:30 - 七步框架介绍
- 强调这是基于成功中小企业实践的经验证方法论
- 针对中小企业和AWS合作伙伴两类受众

00:05:00 - 第一步：识别业务问题
- 从业务挑战出发，而非技术导向
- 关注运营痛点：什么流程耗时过长？什么任务需要17-20次点击？
- 识别业务瓶颈
- AWS合作伙伴可帮助评估AI相关性

00:06:30 - 第二步：从简单智能开始
- 避免立即选择定制化方案
- 优先选择预打包的、内置AI的解决方案
- 快速实现价值
- AWS合作伙伴提供可自部署的预打包解决方案

00:07:30 - 第三步：建立坚实的数据基础
- 强调这是最重要的步骤
- 数据质量和数据卫生至关重要
- 清理过时、重复、无关的数据
- AI的效果取决于训练数据的质量
- 用烹饪比喻：没有准备好食材就无法做出好菜
- AWS合作伙伴可帮助评估和改善数据质量

00:09:00 - 第四步：弥补人才缺口
- 超过80%的中小企业难以找到AI人才
- 建议利用AWS合作伙伴的AI专业知识
- 合作伙伴拥有证书、培训和专业AI解决方案
- 比自行招聘更高效

00:10:00 - 第五步：通过综合解决方案简化集成
- 超过70%的中小企业担心集成问题
- AWS合作伙伴提供集成的综合解决方案
- 确保系统间的互操作性

00:11:00 - 第六步：通过混合AI保障未来安全
- 数据安全是业务成功的基础
- 混合AI提供灵活的安全选项
- 在预算范围内获得最安全的方案

00:12:00 - 第七步：拥抱代理AI（Agentic AI）
- 强调必须先完成前六步
- 传统聊天机器人vs代理AI的区别：
  - 传统聊天机器人：提供指导和最佳实践
  - 代理AI：独立完成任务
- 完成前六步后可探索自主化路线图

00:14:00 - 合作伙伴选择的重要性
- 用手术比喻：不会给自己做手术，同样不应独自进行AI转型
- 引出关键问题：如何选择合适的合作伙伴？

00:15:00 - SMB能力合作伙伴介绍
- 专门为中小企业创建的合作伙伴计划
- 简化中小企业寻找相关、有能力的合作伙伴的过程

00:16:00 - SMB能力合作伙伴的特点
- 经过严格的技术审核
- 验证解决方案是否适合中小企业（如可自部署）
- 要求拥有成熟的中小企业实践
- 强制要求最低数量的活跃中小企业用户
- 全球覆盖：从北美到中国

00:17:30 - 成功的标志
- 主题：更快实现价值
- 运营成本下降
- 完成任务的时间减少
- 生产力提高
- 收入增加
- 客户满意度提高

00:18:30 - 避免的错误
- 不要独自行动
- 不要从太大的问题开始（10年都解决不了的问题）
- 专注短期可行的业务问题
- 不要被新技术吸引而忽视业务挑战
- 不要调整业务挑战以适应技术用例（应该反过来）
- 再次强调：不要忽视数据质量

00:20:00 - 愿景与行动号召
- AI革命触手可及
- 中小企业有能力引领AI革命
- 不要让这成为会后待办事项清单上的又一项任务

00:20:45 - 现场行动要求
- 要求听众立即拿出手机/电脑/笔记本
- 写下两个最大的业务挑战
- 思考：什么流程耗时长？什么任务有人工负担？
- 邀请参会者到884号展位（SMB Accelerate展位）讨论

00:21:30 - 结束语
- 强调未来由中小企业构建
- 邀请通过LinkedIn联系
- 全天在展位提供咨询
