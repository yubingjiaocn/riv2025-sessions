# AWS re:Invent 2025 - Amazon EBS 创新功能详解

## 会议概述 (Session Overview)

本次会议由Amazon EBS产品经理Aaron Chen和Sooraj Prasannan主讲，全面介绍了过去12-18个月Amazon Elastic Block Store (EBS)的重要创新功能。会议以一家快速增长的金融科技公司为例，从三个核心业务场景展示EBS的技术进步：实时事务处理系统、分析数据仓库系统和开发环境。

演讲者详细阐述了EBS作为网络附加持久存储的核心价值，包括卷类型、快照备份和数据服务三大支柱。会议重点强调EBS不仅仅是简单的块存储，而是由AWS工程师维护和运营的大规模分布式系统，能够提供不同级别的性能、持久性和功能特性。

## 详细时间线与关键要点 (Detailed Timeline)

### 0:00-5:00 - 会议开场与EBS基础介绍
- Aaron Chen介绍EBS基本概念：网络附加持久存储
- EBS三大支柱：卷类型(IO2/GP3)、快照备份(基于S3)、数据服务
- EBS作为分布式系统的架构优势
- 服务各种工作负载：从关键业务ERP/CRM到EC2启动卷

### 5:00-10:00 - 客户用例场景设定
- 以金融科技公司为例，介绍三大业务场景
- 实时事务处理系统：核心业务数据库
- 分析数据仓库：业务洞察和报告生成
- 开发环境：新功能构建和测试

### 10:00-15:00 - 实时事务处理系统性能优化
- 亚毫秒级延迟要求：从平均亚毫秒优化到500微秒以下
- IO2卷在主要云提供商中提供最佳P99.9 I/O延迟
- 全球基础设施平等性：IO2 Block Express扩展到所有商业和政府云区域
- 性能监控指标：CloudWatch中的平均延迟、IOPS和吞吐量指标

### 15:00-20:00 - 系统弹性和故障注入测试
- 卷级和实例级超限检查指标
- AWS故障注入服务(FIS)中的延迟注入功能
- 四种延迟模板：持续延迟、延迟增加、间歇性延迟、延迟减少
- EBS卷回收站功能：防止意外删除，支持恢复操作

### 20:00-25:00 - 分析数据仓库系统优化
- Sooraj接管演讲，介绍分析系统的存储需求
- GP3卷性能大幅提升：IOPS从16K增至80K，吞吐量从1GB增至2GB
- 卷大小扩展：从16TB增至64TB
- R8gb实例：150 Gbps带宽，18.75 GB/s吞吐量，720K IOPS

### 25:00-30:00 - 开发环境和数据复制优化
- EBS卷克隆功能：直接卷到卷复制，无需快照中间层
- 蓝绿部署支持：降低软件部署风险
- 基于时间的AMI复制：最快15分钟跨区域复制
- 卷初始化预配置IOPS：最高300 MB/s初始化速度
