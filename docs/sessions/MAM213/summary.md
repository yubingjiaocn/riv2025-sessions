# AWS re:Invent 2025 - SAP数据管理与创新策略会议总结

## 会议概述

本次会议由AWS技术负责人Carlos Karus和瑞典零售商Axfood首席技术架构师Gustaf Hilen共同主讲，重点探讨了在AWS上运行SAP工作负载的数据管理策略。会议强调数据是企业数字化转型的核心基础，而非简单的IT项目。Carlos指出，90%的企业在理解客户需求方面存在困难，但45%却声称已完成数字化转型，这表明许多企业的数字化战略存在根本性问题。

会议的核心观点是：没有"银弹"式的通用解决方案。企业需要根据自身需求、风险承受能力、成本预算和技能储备，在SAP原生解决方案（如BDC）、AWS原生服务和第三方工具（如Databricks、Snowflake）之间找到最佳组合。Gustaf分享了Axfood的实践经验，该公司通过在AWS上构建数据栈，运行超过100个机器学习模型，在预测准确度上比标准系统提升30%，成功实现了业务差异化竞争。

## 详细时间线与关键要点

### 开场与背景介绍 (00:00 - 05:30)
- **00:00** - 会议开始，Carlos Karus自我介绍，在AWS工作12年，专注于业务应用和SAP领域
- **01:15** - 介绍嘉宾Gustaf Hilen，来自瑞典零售商Axfood，这是他第10次参加re:Invent
- **02:30** - 会议议程说明：数据重要性、产品方案、最佳实践模式、客户案例分享

### 数据战略的重要性 (05:30 - 15:00)
- **05:30** - 核心观点：数据是业务创新的基础，IT不应与业务分离
- **07:45** - 调查数据揭示：90%的企业难以理解客户需求，但45%声称已完成数字化转型，存在明显矛盾
- **09:20** - 数字化转型的三大误区：
  - 将其视为纯IT部门职责
  - 设定固定时间框架（如12个月项目）
  - IT与业务割裂
- **11:30** - 强调数据战略的核心价值：速度、可见性、控制力、支持新技术（如生成式AI）
- **13:45** - AWS的差异化价值：提供专家团队协助客户定制最适合的解决方案，而非提供"银弹"

### SAP在AWS上的应用场景 (15:00 - 25:00)
- **15:00** - 数据管理的三种主要模式：
  - 使用SAP BDC（Business Data Cloud）
  - 使用AWS原生解决方案
  - 使用第三方工具（Databricks、Snowflake等）
- **17:30** - 实际应用中通常是混合模式：根据不同用例选择不同方案
- **19:00** - SAP BW（Business Warehouse）在2027年终止支持，企业需要制定迁移策略
- **21:00** - BDC在AWS上的优势：
  - 使用Graviton处理器（ARM架构，高性价比）
  - 利用AWS骨干网络
  - 基于Amazon S3存储
- **23:00** - AWS原生方案优势：开源标准（如Apache Iceberg、Hudi）、企业级安全性、高并发支持

### 第三方集成方案 (25:00 - 30:00)
- **25:00** - Databricks集成：支持Unity Catalog、Delta Lake，适合Spark工作负载
- **27:00** - Snowflake集成：易于与AWS生成式AI服务和LLM集成
- **28:30** - 合作伙伴案例：Celonis等专注于SAP和AWS的合作伙伴可提供评估和迁移服务

### Axfood客户案例分享 (30:00 - 50:00)
- **30:00** - Gustaf介绍Axfood背景：瑞典零售商，800家门店，占瑞典食品市场25%份额
- **32:00** - SAP应用情况：
  - 15年前开始使用SAP实现系统整合
  - 运行S/4HANA和SAP CAR（Customer Activity Repository）
  - 所有门店销售数据流入CAR系统
- **35:00** - 数据架构：SAP存储主数据和核心流程，AWS构建数据栈用于AI/ML和分析应用
- **37:30** - 机器学习应用场景：
  - 运行超过100个ML模型
  - 其中20个模型专门用于门店和仓库配送
  - 预测准确度比标准系统提升30%
- **40:00** - 零售业务优化：
  - 商品分类规划的AI模拟
  - 价格优化和交叉销售预测
  - 电商推荐系统（购物篮平均50件商品）
- **43:00** - 数据战略核心：
  - 必须从所有系统（SAP和非SAP）复制数据到统一数据主库
  - 使用开放格式和开放标准
  - 避免供应商锁定

### 未来展望与建议 (50:00 - 58:00)
- **50:00** - 数据战略挑战：应用供应商（如SAP、Salesforce）正在构建基于大数据的新模块，需要平衡云端生态系统
- **52:00** - 迁移建议：ECC到S/4迁移应采用一对一范围，先建立稳定基础再创新
- **54:00** - 未来重点方向：
  - 数据产品化和治理
  - 代理式AI（Agentic AI）需要清洁数据和MCP服务器支持
  - 数据层中的传感器和自动化操作
- **56:00** - 成功经验分享：
  - 尽早展示高价值用例获得管理层支持
  - 与供应商寻找共同点
  - 选择合适的技术平台
- **57:30** - 会议结束，提醒参会者可参加Gustaf下午的深度分享会议（M214）

## 核心要点总结

1. 数据是业务核心：数据战略不是IT项目，而是持续的业务转型过程
2. 没有通用方案：需要根据企业实际情况定制SAP、AWS和第三方工具的组合方案
3. 开放标准优先：使用开放格式存储数据，建立统一数据主库，避免供应商锁定
4. 先稳定后创新：大型迁移项目应先确保稳定性，再逐步引入创新
5. 价值驱动投资：通过早期高价值用例获得管理层支持和持续投资
