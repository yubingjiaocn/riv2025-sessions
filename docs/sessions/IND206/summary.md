# AWS re:Invent 2025 会议总结：使用生成式AI简化电信网络安全运营

## 会议概述

本次闪电演讲由AWS解决方案架构师Anur Ali和Rei共同主持，主题聚焦于如何利用生成式AI技术来简化电信行业的网络安全运营。演讲深入探讨了电信服务提供商在分析网络安全日志时面临的常见挑战，特别是在诊断用户请求被阻止或安全事件发生时的困境。传统方式下，安全分析师需要花费数小时手动梳理来自多个来源的海量日志数据，这不仅耗时且容易出错。

演讲重点介绍了基于Amazon Bedrock Agent Core构建的智能代理解决方案——"网络追踪专家助手"。该方案通过AI代理自动化地从多个日志源获取、关联和分析安全日志，并利用推理能力快速定位问题根源。演讲中展示了一个实际案例：员工David Lee在访问公司应用时遇到安全阻断，AI代理在几分钟内就完成了跨Zscaler、Checkpoint和Alumio三个数据源的日志分析，成功识别出流量被阻断的具体位置和原因。这种解决方案不仅大幅缩短了事件响应时间（从数小时降至数分钟），还能作为自助服务工具提供给最终用户，显著提升了开发团队和安全运营团队的协作效率。

## 详细时间线

00:00 - 01:30 - 会议开场与演讲者介绍
- Anur Ali和Rei作为AWS解决方案架构师介绍本次闪电演讲
- 概述会议议程：电信行业面临的挑战、AI代理解决方案、演示和架构设计

01:30 - 04:00 - 电信行业安全挑战分析
- 电信服务提供商面临日益增多的网络安全威胁和严格的监管要求
- 安全架构不断演进导致监控和安全控制工具复杂度增加
- 展示典型的用户访问内部应用的网络流程，涉及多个互联网络段和安全工具

04:00 - 06:30 - 日志分析的常见痛点
- 日志信息冗长且分散在AWS云、多区域、第三方云和本地环境
- 日志格式不标准，各工具使用专有格式，缺乏统一的关联标准
- 大型企业每天产生TB级别的日志数据，存储和查询成本高昂
- 日志关联困难，需要学习多种专有查询工具

06:30 - 09:00 - AI代理解决方案介绍
- 推出"网络追踪专家助手"虚拟助手（聊天机器人形式）
- 基于Amazon Bedrock Agent Core构建，具备自动获取、关联和分析日志的能力
- 利用AI推理能力快速诊断并找出问题根源
- 将原本需要数小时的工作缩短至几分钟内完成

09:00 - 11:00 - Amazon Security Lake最佳实践
- 建议将日志集中存储在Amazon Security Lake
- 支持开放网络安全架构框架(OCSF)标准格式
- 提供生命周期管理策略，以Iceberg格式高效存储
- 虽非必需但强烈推荐与AI代理架构结合使用
- 成熟后可作为自助服务工具提供给最终用户

11:00 - 15:30 - 解决方案架构详解（Rei讲解）
- 左侧展示用户角色（主要是安全分析师）
- 使用Amazon Bedrock Agent Core Runtime运行生成式AI代理
- 采用Anthropic Claude 3.7 Sonnet模型和Strands SDK构建
- Amazon Bedrock Agent Core Gateway作为MCP网关暴露工具
- 通过JWT令牌进行身份验证（使用Amazon Cognito，也支持Okta、Entra ID等）
- Lambda函数实现具体工具功能，连接到Zscaler、Checkpoint、Alumio等日志源

15:30 - 20:00 - 实际案例演示
- 场景：Acme电信公司员工David Lee在家使用公司笔记本访问账单应用时遇到安全阻断
- 演示前端界面，输入自然语言查询
- AI代理自动连接到Agent Core Gateway，进行身份验证并列出可用工具
- 代理自动检查三个数据源（Checkpoint、Alumio、Zscaler）
- 生成详细报告：识别流量阻断时间戳、分析各层日志、追踪完整流量路径
- 结论：用户成功通过ZPA认证并连接到Elasticsearch，但后续连接被第三层网络防火墙阻断

20:00 - 23:00 - 代码实现细节
- 展示提示词工程的重要性，包含各日志源的架构信息
- 代理可动态编写查询语句来诊断事件
- 连接到Amazon Bedrock中的Claude 3.7 Sonnet模型
- 建立MCP客户端连接到Gateway URL，传递JWT Bearer令牌
- 强调只需添加4行代码即可将本地代理迁移到Agent Core Runtime

23:00 - 26:00 - 解决方案价值（Anur讲解）
- 显著加快事件检测和解决时间（从数小时降至数分钟）
- 提升安全运营团队生产力，特别是对新团队成员友好
- 可作为自助服务工具提供给内部用户，减少工单量
- 改善开发团队与安全运营团队的协作
- 提升开发敏捷性，减少因安全问题导致的开发延迟
- 支持自然语言查询，无需学习复杂的查询工具

26:00 - 30:00 - 关键经验与总结（Rei讲解）
- AWS生成式AI的强大能力，使用Strands SDK大幅缩短解决时间
- Amazon Bedrock Agent Core Runtime显著缩短生产部署时间（仅需4行代码）
- 采用"爬行-行走-奔跑"渐进式方法：从安全分析师门户开始，发展到自助服务门户，最终用于调查危险安全事件
- Amazon Security Lake集中管理安全态势的最佳实践
- Amazon Bedrock Agent Core Gateway的强大功能：支持Lambda、REST API和MCP服务器目标类型，可连接本地或其他云平台的日志源

30:00 - 31:00 - 会议结束
- 推广AWS Skill Builder学习平台
- 请求观众填写会议调查问卷
- 开放现场问答环节
