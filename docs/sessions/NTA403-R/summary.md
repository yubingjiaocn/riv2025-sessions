# AWS re:Invent 2025 高级RAG架构会议总结

## 会议概述

本次会议是AWS re:Invent 2025的400级技术深度会议，主题为"高级RAG架构"。会议由两位AWS解决方案架构师主讲：Vivek Mittal（来自坦帕，专注于AI/ML，在AWS工作3.5年）和Pallavi Nargund（首席解决方案架构师，IT行业25年经验，AWS工作7.5年）。两位讲师都是AI/ML专家组成员，专门研究检索增强生成（RAG）架构。

会议采用代码驱动的演示方式，重点关注如何提高RAG系统的准确性。讲师们通过实际的Jupyter Notebook演示，展示了从基础RAG到高级自纠正代理RAG的完整技术演进。会议使用了一个虚构的金融公司Octank的10-K报告作为演示数据，该数据完全由LLM生成，用于展示各种RAG优化技术。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与背景设定
- 讲师自我介绍和会议定位
- 确认听众对RAG的熟悉程度和生产环境使用情况
- 强调这是一个代码驱动的400级会议，将直接进入代码演示

### 5:00-10:00 会议议程与RAG准确性的重要性
- 介绍Amazon Bedrock Knowledge Bases概述
- 讨论高级RAG技术和代码演练计划
- 强调RAG准确性的重要性：企业数据决策和自主系统查询的需求

### 10:00-15:00 RAG基础架构回顾
- RAG的三个核心组件：数据摄取、检索和生成
- 重点关注非结构化数据（文档、图像、视频）
- Amazon Bedrock Knowledge Bases作为端到端RAG工作流的托管服务介绍

### 15:00-25:00 标准RAG vs 高级RAG架构模式
- 标准RAG（朴素RAG）的适用场景和局限性
- 高级RAG的两种主要分支模式：
  - 条件分支：根据查询类型选择不同的向量存储
  - 并行分支：从多个数据源获取信息并进行排序整合
- 查询重构作为最常见的优化技术

### 25:00-35:00 代码演示环境设置
- Jupyter Notebook环境配置
- 使用Octank虚构公司的10-K报告作为演示数据
- 预配置的S3存储桶、Knowledge Base和DynamoDB表
- 使用Haiku 4.5作为生成模型，Titan embedding作为嵌入模型

### 35:00-40:00 基础RAG流程演示
- 展示retrieve和retrieve_and_generate两种API
- 测试三个基础问题：HTM投资组合公允价值、主要业务部门、员工数量
- 基础RAG成功回答了明确定义的术语问题

### 40:00-45:00 复杂查询挑战与查询分解技术
- 测试更复杂的查询：Octank Tower和举报人丑闻
- 演示查询分解（decompose and generate）技术
- 通过orchestration configuration启用查询分解功能

### 45:00-50:00 查询扩展技术与自纠正代理RAG
- 处理专有缩写词（如DCM）的查询扩展技术
- 使用DynamoDB存储术语词汇表
- Vivek介绍自纠正代理RAG架构：
  - 中央代理协调多种技术
  - 上下文相关性检查和响应质量评估
  - 自动选择最适合的RAG策略

### 50:00-52:30 其他优化技术与总结
- 数据摄取流程优化：语义分块、多模态解析、元数据标记
- 检索流程优化：元数据过滤、重新排序、混合搜索
- 总结核心思路：通过上下文相关性检查和检索相关性检查来提高RAG准确性
- 开放问答环节
