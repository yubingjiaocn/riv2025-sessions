# AWS re:Invent 2025 - 爱立信通过AWS AI优化移动网络体验

## 会议概述

本次技术分享会由AWS电信业务部门高级解决方案架构师Christian Finelli主持，重点介绍了爱立信如何利用AWS AI技术转型其内部开发体验并优化客户移动网络性能。会议邀请了爱立信的Ibrahim和Dag两位专家，分别从认知网络解决方案(CNS)和系统理解实验室两个角度，展示了爱立信在AI驱动的网络运营和产品开发方面的创新实践。

爱立信作为全球领先的5G网络设备提供商，运营着全球超过50%的5G网络，拥有超过60,000项移动技术专利。通过与AWS的深度合作，爱立信正在利用传统机器学习和多智能体AI解决方案的组合来优化CSP(通信服务提供商)的移动网络性能，同时通过AI赋能其庞大的软件开发团队提高产品开发效率。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与AWS-CSP合作框架
- Christian Finelli介绍会议主题和演讲嘉宾
- AWS与CSP合作的五大业务目标：增加收入、优化成本、减少客户流失
- CSP正在AWS上迁移和现代化复杂的BSS/OSS系统栈
- AI是所有这些领域的关键组件，爱立信作为AWS关键合作伙伴最适合规模化这些举措

### 5:00-10:00 成功的智能体AI解决方案关键要素
- 业务必须能够信任智能体及其输出结果
- 数据访问和知识授权需要在最底层实现
- 数据血缘在传统ML训练中的重要性
- 可解释性的重要性：能够剖析智能体行为、推理过程和工具调用
- 上下文为王：在正确时间向正确智能体提供简洁的知识和数据
- 多智能体解决方案中的价值归因和ROI确定

### 10:00-15:00 可观测性与AWS AI技术栈
- Ibrahim强调可解释性对CSP的重要性，特别是在国家基础设施运营中
- AWS AI技术栈的多层架构：
  - 底层：数据科学家使用AI计算进行训练和推理
  - 中层：Amazon SageMaker AI用于完整的MLOps流程
  - 上层：Amazon Bedrock用于大规模安全运行智能体
- 爱立信的两个主要用例：认知网络解决方案(CNS)和系统理解实验室

### 15:00-20:00 爱立信的全球影响力与自主网络框架
- 爱立信运营全球340个5G网络中的189个(超过50%)
- 全球5G流量(中国以外)超过50%运行在爱立信设备上
- 拥有150年创新传统和60,000+移动技术专利
- 74%的全球CSP在使用爱立信设备时在市场竞争中排名第一
- 自主网络框架：TMF定义的五级网络运营标准，目前不到10%的CSP达到三级以上

### 20:00-25:00 认知循环与智能体AI架构
- 认知循环的五个阶段：测量、保障、提议、评估、执行
- 智能体AI架构基于开放性和水平化两大目标
- 通过AWS服务为智能体提供不同能力
- R应用作为智能体工具，提供网络理解和执行变更的能力
- 可观测性在整个AI技术栈中的重要性

### 25:00-30:00 技术架构深入解析
- 从无线接入网络摄取大量数据的ETL层
- 使用AWS无服务器服务、容器和完全托管服务(如Athena和Glue)
- Amazon SageMaker AI构建ML管道：数据准备、模型训练、评估和推理托管
- 智能体通过RAG技术依赖其他数据源和知识库
- Amazon Bedrock AgentCore提供内存和可观测性能力

### 30:00-35:00 Cell Shapers强化学习智能体
- Cell Shapers是基于网络数字孪生表示的强化学习智能体
- 解决从2G到6G的多代网络覆盖和质量优化问题
- 基于实时流量重构无线环境，持续从网络中学习
- 多目标强化学习算法在覆盖、容量和质量之间提供最佳权衡
- 展示多个智能体的实际部署结果和性能提升数据

### 35:00-40:00 系统理解实验室的使命
- Dag介绍移动网络的巨大复杂性挑战
- 爱立信面临"理解力约束"问题：数万名研发人员处理分布式系统复杂性
- AI使命：在复杂系统工程极限下使AI发挥作用
- 三层方法：整合内部数据资产、结合实时网络数据、理解人类意图
- 前沿AI能力与工业环境AI影响之间存在巨大差距

### 40:00-45:00 信息融合与MCP架构
- 信息融合贯穿需求收集、系统化、开发、测试集成、发布部署全流程
- 通过MCP(模型上下文协议)实现前端工具与内部研发资产的解耦
- 专业化AI智能体、模型和开发工具的构建
- Bastian介绍系统理解实验室的关键支柱：正确数据、多步推理、多模态处理

### 45:00-49:24 AgentCore架构与评估框架
- 下游工具集成通过MCP连接各种工程工具
- 可观测性：理解工具调用、上下文相关性、复杂系统监控
- 可扩展架构：双边市场模式，生产者和消费者的快速入门
- 评估框架的重要性：智能体AI项目中唯一的常量是高速变化
- 数据处理复杂性：多格式、多结构、不同权威性数据的处理
- 自定义大语言模型训练：针对爱立信专有硬件和框架的代码生成
- AgentCore运行时架构：MCP网关、专业化智能体、知识集成和身份管理
