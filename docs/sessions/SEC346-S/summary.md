# AWS re:Invent 2025 Vega安全运营架构重构会议总结

## 会议概述

本次会议由初创公司Vega的联合创始人兼CTO Ellie和威胁检测研究主管Ian主讲。Vega成立于2024年1月，已获得6500万美元融资，致力于彻底改变安全运营和安全分析的方式。会议重点讨论如何利用AWS S3存储桶重新构想SIEM架构，在不增加成本的前提下提高可见性并充分利用数据价值。

演讲者分享了他们与数百家企业交流后发现的三个核心问题：不断增长的安全遥测数据量、多个数据存储库的管理复杂性，以及传统单体SIEM架构的局限性。Vega提出了一种联邦化架构方案，挑战传统的数据集中化假设，允许企业在多个数据存储库中查询数据，而无需将所有数据传输到单一集中位置。

## 详细时间线与关键要点

### 0:00-3:00 公司介绍与背景
- Vega公司简介：2024年1月成立的初创公司，获得6500万美元融资
- 演讲者背景：Ellie（CTO，15年网络安全经验）和Ian（威胁检测主管，10年事件响应经验）
- 会议目标：展示如何使用AWS S3重新构想SIEM架构

### 3:00-8:00 当前安全运营面临的三大挑战
- **挑战一**：安全遥测数据量持续增长，云迁移和多云环境加剧了这一问题
- **挑战二**：传统单体架构要求为每字节日志付费，导致成本约束
- **挑战三**：组织维护多个数据存储库，有些企业甚至使用9个不同的SIEM系统

### 8:00-12:00 传统架构的根本问题
- 质疑安全遥测数据集中化的必要性
- 挑战存储与检测工程耦合在同一单体系统中的模式
- 提出利用多个数据存储库（包括S3存储桶）实现传统SIEM功能的可能性
- 探讨跨多个数据存储库进行关联分析的需求

### 12:00-16:00 AWS客户实际案例分析
- **传统方式成本结构**：服务费用、AWS存储、数据传输费用、SIEM摄取费用、SIEM存储费用、安全模块费用
- **联邦化架构优势**：直接在S3存储桶中建立索引，无需数据传输
- **成本节省**：750+AWS账户的组织可实现60-80%的成本降低

### 16:00-18:00 客户成功案例
- **案例一**：大型电商平台通过S3索引实现70%成本降低，完全替代传统SIEM
- **案例二**：财富500强制药公司，500+AWS账户，实现零数据传输成本的安全运营

### 18:00-20:30 Vega解决方案架构
- **核心能力**：跨存储关联、混合数据湖支持、统一安全视图
- **Security Analytics Mesh**：连接多种数据源（SIEM、数据湖、对象存储）
- **技术特点**：KQL查询接口、AI/LLM集成、本地索引技术
- **多云支持**：支持三大主要云服务商的数千个云账户
