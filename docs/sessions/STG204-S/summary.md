# AWS re:Invent 2025 会议总结

## 会议概述

本次会议主要介绍了H2O.ai与Datafi合作解决企业AI平台在AWS EBS存储方面的优化问题。H2O.ai是一家全球领先的企业AI公司，提供生成式AI和预测式AI平台，可部署在本地环境、云端和气隙环境中。该公司的整个平台运行在Kubernetes上，在AWS云端使用EKS，并严重依赖EBS存储来支持AI模型训练和数据处理。

H2O.ai面临的核心挑战是EBS存储利用率极低的问题。他们存储了超过2PB的数据，但大量存储空间未被充分利用，导致云存储成本浪费严重且无法有效扩缩容。传统解决方案需要将数据从旧EBS卷迁移到新卷，过程复杂且痛苦。Datafi作为一个自主存储解决方案，能够自动管理AWS客户的云存储，根据需求自动调整EBS卷容量，实现无缝扩缩容且不影响性能。

通过部署Datafi解决方案，H2O.ai成功将存储利用率从25%提升至80%，将存储成本从2PB降低至不到1PB，实现了显著的成本节约。整个部署过程零停机，无需修改现有基础设施，完美集成到GitOps流程中，同时还提升了客户的存储性能。

## 详细时间线

0:00 - 会议开始与H2O.ai公司介绍
- H2O.ai是企业AI领域的全球领导者
- 提供生成式AI和预测式AI解决方案
- 平台可部署在本地、云端和气隙环境
- 在Gaia基准测试中处于领先地位

0:45 - H2O.ai技术栈介绍
- 整个平台完全运行在Kubernetes上
- 在AWS云端使用EKS
- 严重依赖EBS存储支持AI引擎和模型训练
- 需要快速且高质量的存储性能

1:30 - 存储问题描述
- 存储了超过2PB的EBS存储
- 存储量快速增长
- 无法有效扩缩容
- 大量云存储被浪费
- 传统解决方案需要数据迁移，过程复杂痛苦

2:30 - Datafi解决方案介绍
- Datafi是自主存储解决方案
- 自动管理AWS客户的云存储
- 可即时部署
- 根据需求自动调整EBS卷容量
- 数据填满时自动扩容，删除文件时自动缩容
- 对性能无影响

3:30 - Datafi核心功能
- 动态自动扩缩容
- 完全自主运行
- 克服EBS限制
- 支持无限扩展EBS容量（向上和向下）
- 零停机时间
- 无需修改应用程序或文件系统

4:30 - 无缝集成特性
- 可在不影响应用的情况下安装或卸载
- 无需修改技术栈
- 与Kubernetes无缝集成
- 支持Terraform
- 支持任何Linux操作系统
- 自动完成，无需人工干预

5:15 - Datafi工作原理
- 基于安装在EC2服务器或Kubernetes集群上的底层代理
- 代理自动动态更改EBS目标卷
- 不影响应用程序运行
- SaaS控制平面运行在Datafi的VPC中
- 监控代理并发送扩缩容命令
- 提供分析功能，显示EBS部署详情和效率指标

6:15 - 基础设施即代码集成
- 深度集成Kubernetes和其他IaC环境
- 无需修改即可使用产品
- 完全集成到CI/CD生命周期

6:45 - 实施挑战与解决方案
- **挑战1：Bottlerocket兼容性** - 与Datafi合作确保代理在现有Bottlerocket基础设施上运行
- **挑战2：安全性** - 确保客户数据安全，数据不离开EKS集群，SaaS仅用于管理
- **挑战3：备份兼容性** - 与Velero备份解决方案保持兼容，无缝继续备份和恢复PV

8:00 - 优化效果展示
- **初始状态**：容量利用率仅25%，总容量2PB，实际使用0.5PB
- **优化过程**：随着Datafi部署到更多环境，利用率持续提升
- **最终结果**：利用率稳定在80%左右（目标缓冲区）
- **成本节约**：从支付2PB降至不到1PB

9:30 - 实施成果总结
- Datafi解决方案部署到所有客户环境
- 使用现有工具（Terraform）部署
- 集成到GitOps流程
- 无需对基础设施做重大修改
- 显著降低成本
- 提高存储利用率
- 提升客户性能

10:30 - 零停机部署
- Datafi代理以只读模式部署到所有集群
- 只需切换开关即可激活
- 自动开始优化存储
- 无需人工干预
- 立即开始降低存储成本
