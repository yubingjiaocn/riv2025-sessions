# AWS re:Invent 2025 可持续计算气候解决方案会议总结

## 会议概述

本次会议由AWS高级解决方案架构师顾宇和技术客户经理Nathan Pathak主讲，主题为"可持续计算气候解决方案"。会议深入探讨了气候挑战的复杂性以及高性能计算(HPC)在解决气候问题中的重要作用。演讲者通过实际案例展示了如何在AWS云平台上构建可持续的HPC工作负载，并分享了牛津大学在空气污染检测项目中的成功实践。

会议强调了气候系统的复杂性，包括大气、水圈、冰冻圈、岩石圈和生物圈五个相互作用的组成部分，以及人类社会在应对气候变化时面临的社会经济、政治和文化挑战。演讲者指出，技术在加速气候解决方案方面发挥着关键作用，特别是在数据获取、存储和大规模计算处理方面。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与气候挑战现状
- 演讲者自我介绍：顾宇(AWS社会责任与影响现代化高级解决方案架构师)和Nathan Pathak(AWS技术客户经理)
- 通过现场互动了解参会者受气候变化影响情况
- Nathan分享印度新德里空气污染问题的亲身经历
- 介绍当前极端天气事件：夏季更热更长、飓风持续时间更长、降水模式改变导致洪水和干旱、野火季节延长

### 5:00-10:00 气候挑战的复杂性分析
- 气候系统复杂性：五个相互作用的组成部分(大气、水圈、冰冻圈、岩石圈、生物圈)
- 人类社会复杂性：社会经济、政治、文化因素以及全球性问题的地区差异
- 时间紧迫性：需要在有限时间窗口内将全球温升控制在1.5°C以下
- 当前进展：绿色建筑、可再生能源、可持续农业、交通电气化

### 10:00-15:00 技术栈要求与HPC介绍
- 气候解决方案技术栈三大核心要求：
  - 数据：来自供应链、卫星图像、海洋声音等多源大数据
  - 存储：能够容纳不同类型大数据并高效检索
  - 计算：大规模数据处理和仿真能力
- HPC在气候研究中的重要性和传统本地部署的能耗挑战
- AWS共享责任模型：AWS负责云的可持续性，客户负责云中的可持续性

### 15:00-20:00 AWS基础设施可持续性
- AWS全球基础设施：38个地理区域、120个可用区、245个国家和地区
- 根据埃森哲研究，AWS工作负载比本地部署效率高4.1倍
- 数据中心设计优化：PUE值1.15，机械能耗减少46%，混凝土碳排放减少35%
- AWS Well-Architected框架2021年新增可持续性支柱
- 六大可持续性基础支柱：区域选择、需求对齐、数据、硬件和服务、软件和架构、流程和文化

### 20:00-25:00 可持续HPC构建模块
- 三层架构：
  - 基础层：Amazon FSx for Lustre文件系统、Elastic Fabric Adapter网络接口
  - 计算调度层：AWS Parallel Cluster、AWS Batch、AWS Parallel Computing Service
  - 可视化层：Amazon DCV、Research and Engineering Studio
- 数据管理：AWS开放数据平台减少数据传输排放，提供即用型数据访问

### 25:00-30:00 存储和计算优化
- Amazon S3生命周期策略：根据数据访问模式自动迁移和删除数据
- Amazon FSx for Lustre优势：智能分层、自动生命周期管理、与S3集成，存储占用减少50%
- 三种计算解决方案对比：
  - AWS Parallel Cluster：适合OpenMP和MPI工作负载，碳足迹减少57%
  - AWS Batch：适合松耦合容器化工作负载，计算时间减少60%
  - AWS Parallel Computing Service：全托管服务，专注研究而非基础设施管理

### 30:00-35:00 AWS定制芯片与实例优化
- HPC优化和加速计算实例
- 三款定制芯片：
  - Graviton：通用计算，能效比同类EC2实例高60%
  - Inferentia：推理优化，每瓦性能持续改进
  - Trainium：训练优化，能效比同类实例高25%
- 三种架构示例展示不同HPC解决方案的部署方式

### 35:00-40:00 牛津大学案例研究
- 印度-巴基斯坦-孟加拉国地区空气污染问题：污染水平超WHO标准10倍，年死亡200万人
- APA项目(空气污染资产检测)：检测1.5百万平方公里范围内的砖窑污染源
- 技术方案：
  - 低分辨率卫星图像(Sentinel 2)初步识别31,000个潜在砖窑
  - 高分辨率图像精确定位实际砖窑位置
  - 创建交互式污染源地图
- 成果：处理120万张卫星图像，节省17,000计算小时，基础设施成本降低80%，监控时间减少90%

### 40:00-41:48 总结与后续步骤
- 关键要点：
  - HPC无需大型热能数据中心，可快速部署可持续解决方案
  - 利用开放数据源减少数据传输和存储成本
  - 根据数据访问模式选择合适的文件系统
  - 根据性能需求选择合适的计算解决方案
- 推荐资源：AWS AI for Good网站、AWS Skills Builder技能提升平台
- 呼吁完成会议调查反馈
