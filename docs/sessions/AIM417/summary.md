# AWS re:Invent 2025 会议总结：可持续计算助力气候解决方案

## 会议概述

本次会议由AWS高级解决方案架构师Guyu和技术客户经理Nitan Paruk主讲，主题聚焦于如何利用可持续的高性能计算(HPC)技术解决气候挑战。会议首先探讨了气候变化的复杂性及其对全球社区的影响，包括极端天气事件、空气污染、粮食安全等问题。演讲者强调，气候系统本身的复杂性（大气、水圈、冰冻圈、岩石圈、生物圈的相互作用）以及人类社会的多样性，使得气候问题的解决极具挑战性。

会议重点介绍了AWS如何通过云基础设施和HPC服务帮助研究人员构建可持续的气候解决方案。AWS的全球基础设施在能效方面表现出色，PUE值达到1.15，比本地数据中心效率提升4.1倍。演讲者详细讲解了可持续HPC工作负载的最佳实践，包括数据管理、存储优化和计算资源选择。会议最后通过牛津大学的APAD项目案例，展示了如何利用AWS服务处理120万张卫星图像，识别印度-巴基斯坦-孟加拉地区的空气污染源，实现了80%的基础设施成本降低和90%的监控时间减少。

### 可持续HPC架构组件 (30:00 - 45:00)

30:00 - 33:00 - HPC架构三层结构

33:00 - 38:00 - 数据管理最佳实践
- 数据对气候研究至关重要，但EB级数据存储会产生大量碳排放

38:00 - 42:00 - 存储优化策略
- 识别数据访问模式，减少存储使用
- Amazon S3生命周期策略：自动将数据从频繁访问层迁移到低频访问层，最终删除不用数据
- Amazon FSx for Lustre优势：
  - 智能分层：自动在存储层间移动数据
  - 自动生命周期管理
  - 数据归档功能
  - 与S3集成
  - 相比标准数据中心，存储占用减少50%

42:00 - 45:00 - 计算服务选择
- **AWS Parallel Cluster**：适合OpenMP和MPI工作负载，可减少57%碳足迹，提供灵活的资源控制

- **AWS PCS**：零运维开销，自动管理集群和资源，适合初学者专注于代码开发

### 牛津大学APAD案例研究 (58:00 - 75:00)

58:00 - 62:00 - 印度-巴基斯坦地区空气污染问题
- 全球污染最严重地区之一，污染水平超WHO标准10倍
- 每年导致超过200万人过早死亡
- 缺乏精确的污染源数据，治理工作受阻数十年
- 主要污染源：砖窑（排放CO2和颗粒物），因房地产快速发展而大量无监管运营

62:00 - 65:00 - APAD项目介绍
- 牛津大学史密斯企业与环境学院的研究项目
- 愿景：清洁空气是基本人权，社区应免受空气污染
- 目标：检测印度-巴基斯坦地区150万平方公里内的所有砖窑
- 创建交互式地图和仪表板，帮助社区做出数据驱动决策

65:00 - 68:00 - 技术挑战与数据选择
- 本地机器无法处理大规模数据存储、处理和机器学习工作负载
- 选择卫星图像作为主要数据源：提供一致的广域和历史地球观测
- 卫星图像类型：公开数据（Sentinel-2、Landsat）和商业数据
- 分辨率差异：低分辨率（10米）vs 高分辨率（0.5米）

68:00 - 71:00 - 两阶段分析方法
- **第一阶段**：使用Sentinel-2低分辨率图像
  - 构建随机森林分类模型识别"橙色椭圆物体"
  - 在巴基斯坦识别出31,000个潜在砖窑（包含大量误报）
- **第二阶段**：仅为识别区域获取高分辨率图像
  - 使用计算机视觉模型精确定位砖窑
  - 大幅降低高分辨率图像采购成本

71:00 - 73:00 - 项目成果
- 创建交互式污染源地图，包括砖窑、锅炉、钢铁厂等
- 显示煤炭、化石燃料等主要能源来源
- 每个污染源可查看具体影响数据
- 帮助社区更好了解本地污染情况
- 项目扩展至非洲地区

73:00 - 75:00 - APAD应用的AWS最佳实践
- **数据**：使用Open Data Registry的Sentinel-2作为分析基础
- **存储**：分析数据访问模式，仅存储必需数据
- **计算**：
  - 在EC2上运行低分辨率和高分辨率工作流
  - 针对不同工作流选择不同实例（低分辨率需要更多计算，高分辨率需要更多内存）
  - 使用正确的工具完成正确的工作

75:00 - 77:00 - 项目成果数据
- 处理120万张卫星图像
- 节省超过17,000计算小时
