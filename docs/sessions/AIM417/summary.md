# AWS re:Invent 2025 可持续计算气候解决方案会议总结

## 会议概述

本次会议由AWS高级解决方案架构师顾宇和技术客户经理Nathan Pathak主讲，主题为"可持续计算气候解决方案"。会议深入探讨了气候挑战的复杂性以及高性能计算(HPC)在解决气候问题中的重要作用。演讲者通过实际案例展示了如何在AWS云平台上构建可持续的HPC工作负载，并分享了牛津大学在空气污染检测项目中的成功实践。

会议强调了气候系统的复杂性，包括大气、水圈、冰冻圈、岩石圈和生物圈五个相互作用的组成部分，以及人类社会在应对气候变化时面临的社会经济、政治和文化挑战。演讲者指出，技术在加速气候解决方案方面发挥着关键作用，特别是在数据获取、存储和大规模计算处理方面。

## 详细时间线与关键要点

- 演讲者自我介绍：顾宇(AWS社会责任与影响现代化高级解决方案架构师)和Nathan Pathak(AWS技术客户经理)
- 互动环节：现场观众分享气候变化影响经历
- Nathan分享印度新德里空气污染问题：冬季雾霾、高AQI指数、需要空气净化器
- 全球极端天气事件增加：高温持续时间延长、飓风强度增强、降水模式改变、野火季节延长

### 5:00-10:00 气候挑战复杂性分析
- 气候系统五大组成部分及其复杂相互作用
- 人类社会复杂性：社会经济、政治、文化因素
- 全球性问题的地区差异化影响
- 长期与短期权衡的紧迫性：1.5°C温升控制目标
- 当前进展：绿色建筑、可再生能源、可持续农业、交通电气化

### 10:00-15:00 技术栈要求与HPC介绍
- 气候解决方案技术栈三大核心需求：
  - 数据：多源数据获取(供应链数据、卫星图像、海洋声音等)
  - 存储：大容量、多类型数据存储与高效检索
  - 计算：大规模数据处理和仿真能力
- HPC在气候研究中的重要性
- 传统HPC挑战：数百台服务器、高能耗、设置复杂、可持续性问题

### 15:00-20:00 AWS可持续性共同责任模型
- AWS负责"云的可持续性"：全球基础设施、数据中心、电力供应、服务器、冷却系统
- 客户负责"云中的可持续性"：工作负载、区域选择、数据使用、ML模型选择
- AWS全球基础设施：120个可用区、245个国家和地区、38个地理区域
- Accenture研究：AWS工作负载比本地部署效率高4.1倍
- 数据中心PUE值1.15，优于本地平均水平和公有云行业标准

### 20:00-25:00 AWS Well-Architected框架可持续性支柱
- 2021年新增可持续性支柱，基于六大基础原则：
  - 区域选择
  - 需求对齐
  - 数据管理
  - 硬件和服务
  - 软件和架构
  - 流程和文化
- 客户碳足迹工具：按区域、服务、账户使用情况分解碳排放
- 可持续HPC构建模块三层架构：基础层(存储和网络)、计算调度层、可视化交互层

### 25:00-30:00 可持续HPC关键组件详解
- 数据层面：AWS开放数据平台，减少数据传输排放，即用即取
- 存储优化：Amazon S3生命周期策略，Amazon FSx for Lustre智能分层
- FSx for Lustre优势：存储占用空间比标准数据中心减少50%
- 计算解决方案：
  - AWS ParallelCluster：碳足迹减少57%
  - AWS Batch：计算时间减少60%
  - AWS Parallel Computing Service：全托管服务

### 30:00-35:00 AWS优化实例与架构模式
- HPC优化实例和加速计算实例
- AWS自研芯片：
  - Graviton：功耗效率比同类EC2实例高60%
  - Inferentia：推理性能功耗比持续改进
  - Trainium：训练能效比标准实例高25%
- 三种HPC架构模式展示：ParallelCluster、Batch、Parallel Computing Service

### 35:00-40:00 牛津大学案例研究
- 印度-巴基斯坦-孟加拉国地区空气污染问题：污染水平超WHO标准10倍，年死亡200万人
- APA项目(空气污染资产检测)：覆盖150万平方公里
- 技术方案：
  - 低分辨率卫星图像初步筛选(Sentinel-2)
  - 高分辨率图像精确识别
  - 随机森林分类模型和计算机视觉模型
- 成果：处理120万张卫星图像，节省17000计算小时，基础设施成本降低80%，监控时间减少90%

### 40:00-41:48 总结与后续步骤

  - 统一数据源，减少传输成本
  - 理解数据访问模式，选择合适文件系统
  - 根据性能需求选择合适工具
