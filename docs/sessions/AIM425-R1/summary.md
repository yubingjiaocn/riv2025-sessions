# AWS re:Invent 2025 - 构建智能事件代理：深入探讨代理式RAG系统

## 会议概述

本次技术会议由AWS解决方案架构师Dani和GenAI数据科学家Akarsha主讲，重点介绍如何构建智能事件代理系统。会议采用400级别的深度技术讲解，通过实际代码演示展示了代理式RAG（Retrieval-Augmented Generation）系统的完整实现过程。

演讲者详细介绍了如何结合Amazon Bedrock知识库、AgentCore Memory和AgentCore Runtime等AWS服务，构建一个能够提供个性化推荐的AWS re:Invent事件助手。该系统不仅能够从知识库中检索会议信息，还能够存储和学习用户偏好，提供真正的超个性化体验。整个会议强调了从概念验证到生产环境部署的完整流程，包括安全性和身份验证的最佳实践。

## 详细时间线与关键要点

### 0:00-10:00 - 会议介绍与基础概念
- 演讲者自我介绍：Dani（Amazon Bedrock团队解决方案架构师）和Akarsha（AgentCore Memory团队GenAI数据科学家）
- 传统RAG系统回顾：查询增强、外部数据源集成、减少模型幻觉
- 代理式RAG的核心概念：上下文工程（Context Engineering）的重要性
- 上下文的三个组成部分：文档信息（传统RAG）、工具策划、记忆组件

### 10:00-20:00 - 项目目标与架构设计
- 构建"AWSome事件代理"的目标：提供超个性化的re:Invent会议推荐体验
- 代理的两大核心能力：从知识库检索会议信息、存储和检索对话历史
- 技术栈选择：Claude Haiku 4.5模型、Amazon Titan文本嵌入v2、Strands Agents框架
- 代码演示环境准备：依赖安装、库导入、唯一资源ID创建

### 20:00-35:00 - 简单代理创建与知识库构建
- 使用Strands Agents创建基础代理：一行代码实现智能事件助手
- Amazon Bedrock知识库详细介绍：完全托管的RAG工作流服务
- 知识库组件：数据源层（S3连接器）、数据存储层、四种分块策略
- 实际操作演示：下载2024年re:Invent会议数据（583个会议）、创建S3存储桶、文件上传

### 35:00-45:00 - 知识库配置与工具创建
- IAM角色创建：为知识库提供嵌入模型和S3访问权限
- S3向量存储和索引创建：1024维度配置对应Titan嵌入v2
- 知识库创建和数据摄取：固定大小分块策略（300令牌，20%重叠）
- 知识库搜索工具开发：使用Strands工具装饰器，语义搜索API调用

### 45:00-55:00 - AgentCore Memory集成与个性化
- 短期记忆与长期记忆概念：原始对话消息vs提取的洞察
- 记忆策略配置：用户偏好、语义事实、会话摘要
- Hook机制实现：代理初始化时加载用户偏好、消息添加时存储对话
- 实际测试：用户信息输入、偏好提取、个性化推荐生成

### 55:00-58:30 - 生产环境部署与安全
- AgentCore Runtime介绍：会话级隔离、微虚拟机架构、自动扩展
- 生产就绪配置：仅需添加四行代码（应用导入、初始化、入口点、运行）
- AgentCore Identity集成：JWT授权器配置、Cognito身份验证
- 安全最佳实践：动态actor ID获取、会话ID管理、IAM策略条件控制
