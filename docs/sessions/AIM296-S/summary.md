# AWS re:Invent 2025 会议总结：使用 Contentful 和 AWS 增强个性化体验

## 会议概述

本次会议由 Contentful 的个性化产品实践负责人 Gabriel Dylan 主讲，重点探讨了如何通过 Contentful 数字体验平台与 AWS（特别是 Amazon Bedrock）的结合来解决现代营销中的"内容崩溃"问题。演讲者指出，尽管 AI 技术能够生成大量内容，但这些内容往往过于通用和乏味，无法有效驱动用户行为。问题的核心在于 AI 工具通常作为独立的侧边栏功能存在，而非深度集成到内容层中。

Contentful 作为原生构建在 AWS 上的平台，通过将 Amazon Bedrock 直接整合到内容层，为营销人员提供了一个既安全又可扩展的解决方案。这种集成方式使得 AI 能够理解内容模型、用户上下文和品牌指南，从而生成真正个性化且相关的内容体验。演讲者强调，成功的个性化需要结构化内容、自动化护栏、上下文感知的内容生成以及 API 优先的交付层。通过这种方法，营销团队可以在保持治理和工作流程的同时，快速创建和发布个性化体验。

## 详细时间线与关键要点

开场介绍 (0:00-1:30)
- 演讲者 Gabriel Dylan 介绍自己是 Contentful 个性化产品实践负责人
- 感谢 AWS 的合作伙伴关系，强调 Contentful 原生构建在 AWS 上
- 会议主题：通过 Contentful 和 AWS（特别是 Amazon Bedrock）增强个性化体验

内容崩溃问题 (1:30-4:00)
- 提出"内容崩溃"（Great Content Collapse）概念：AI 生成的内容过于通用、乏味
- 类比 2011 年 Bootstrap UI 库导致所有网站看起来相同的现象
- 问题影响营销漏斗的各个阶段：顶部、中部和底部
- 营销人员面临的悖论：速度提升了，但相关性丧失了

AI 与内容层的脱节 (4:00-6:00)
- AI 工具作为侧边栏功能存在，未真正集成到内容层
- 应该让营销人员更具人性化和创造力，而非相反
- 引用 CMO Elizabeth Maxon 与 The Atlantic 合作的研究报告
- AI 应该释放营销人员的创造力，而非增加负担

个性化的挑战 (6:00-9:00)
- 个性化并非新概念，但经常失败
- 失败原因包括：
  - 内容孤岛导致跨渠道体验不佳
  - 过于表面的个性化（如仅使用用户名）
  - 不准确的个性化（数据过时或集成不良）
  - 冗余体验和缺乏反馈循环
  - 过度依赖开发人员

乐观与现实的差距 (9:00-10:30)
- 引用 The Atlantic 报告中的"乐观与期望差距"
- 虽然人人都知道个性化是答案，但实施非常困难
- AI 本应解决这些问题，但往往使问题恶化
- 营销人员使用独立的 AI 工具会产生更多低质量内容

优秀个性化的四大要素 (10:30-13:00)
1. 结构化内容：清晰定义内容模型，可跨多个渠道重用
2. 自动化护栏：确保内容符合品牌指南和前端要求
3. 上下文感知的内容生成：AI 理解内容模型、用户上下文和营销组织上下文
4. API 优先的交付层：无论在何处使用都能交付内容

内容作为差异化因素 (13:00-15:00)
- 内容是构建客户体验的关键差异化因素
- 个性化体验是针对目标受众定制内容的机会
- 需要两种主要上下文：
  - 行为信号：用户实时行为
  - 已知客户特征：历史数据、公司信息等

AWS 与 Contentful 的合作 (15:00-18:00)
- Contentful 构建在 AWS 上，提供信任和可扩展性
- Amazon Bedrock 提供特殊的 AI 能力
- Bedrock 允许使用组织确定的生成式 AI 模型，包含所有治理和安全措施
- Contentful 将 Bedrock 直接集成到内容层中

参考架构 (18:00-20:30)
- Contentful 位于中心，提供内容结构、治理和工作流程
- Amazon Bedrock 作为智能引擎，创建、本地化内容并识别用户
- API 优先的可组合架构作为前端展示层
- 三大优势：
  - **速度**：内容和 AI 在同一平台工作
  - **安全**：治理存在于内容层
  - **规模**：AWS 提供基础设施支持

个性化测试程序 (20:30-23:00)
- 需要围绕个性化和测试构建程序
- 采用迭代和小步快跑的方法
- 赋能营销人员进行实验并收集数据
- 衡量影响并与业务目标对齐
- 治理是关键：使用自己的 LLM、角色权限、审计日志和工作流程

客户案例：Craft Hind (23:00-25:00)
- 使用 Contentful 和 AWS 推出数十个体验
- 之前的问题：内容分散在多个系统、治理不一致、缺乏跨品牌一致性
- 实施后的成果：
  - 转化率提升 78%
  - 参与度提升 30%
  - 在全球多个品牌推出超过 180 个个性化体验

总结与行动号召 (25:00-26:30)
- 将 AI 更接近内容层可帮助营销人员摆脱内容崩溃
- 帮助营销人员做出更好的内容决策
- 在工作流程中创建速度和信任
