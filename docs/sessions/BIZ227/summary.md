# AWS re:Invent 2025 QuickSight 嵌入式分析会议总结

## 会议概述

本次会议重点介绍了 Amazon QuickSight 的嵌入式分析能力及其在企业和 ISV 产品中的应用。演讲者 Shakhar Kopuri（QuickSight 产品负责人）和 Jose Banos（QuickSuite 市场专家）展示了如何通过嵌入式模式从 QuickSight 中提取价值，并邀请了 Moody's 保险解决方案的产品经理 Kand Davi 分享了实际实施案例。

会议强调了 QuickSight 作为 AI 原生、代理式平台的能力，能够统一结构化和非结构化数据，提供从洞察到行动的完整分析体验。QuickSight 的嵌入式功能使企业能够将 BI 能力直接部署到用户的日常工作工具中，ISV 则可以通过嵌入分析功能增强产品竞争力。Moody's 的案例展示了如何在短短几个月内成功实施嵌入式 QuickSight，构建了名为"风险数据湖"的应用数据湖解决方案，为保险行业提供灾难风险量化分析。

会议还介绍了 QuickSuite 的最新发展，强调其通过"逆向工作"方法持续适应市场需求，90% 的新功能来自客户反馈。QuickSuite 正在改变传统 BI 工具的工作方式，通过代理式 AI 将数据与智能行动相结合，帮助用户减少重复性任务，更快地做出数据驱动的决策。

- **05:30** - 云原生无服务器架构：无需管理服务器、硬件升级、操作系统补丁
- **06:00** - 按使用付费定价模式，无需多年软件合同

- **12:00** - 选择 QuickSight 的理由：

- **13:30** - 多租户架构支持：命名空间隔离或基于标签/身份的 RLS/CLS 规则
- **14:30** - 强大的 API 和 SDK：一键嵌入、双向交互、参数传递、事件处理

### Moody's 保险解决方案案例 (18:00 - 35:00)
- **18:00** - Kand Davi 介绍 Moody's 保险解决方案背景
- **18:30** - 公司定位：保险行业灾难风险量化分析的市场领导者
- **19:00** - 提供地震、飓风等灾难事件的风险模型和解决方案
- **19:30** - 覆盖风险生命周期：从初级承保到再保险和经纪人
- **20:00** - 以角色为中心的应用：Risk Modeler、Treaty IQ、Underwriting IQ、Exposure IQ
- **20:30** - 基于智能风险平台（Intelligent Risk Platform）构建
- **21:00** - 首个也是唯一的云原生平台，基于 AWS 技术
- **21:30** - 用户角色介绍：
  - Jane（业务用户/承保人/投资组合经理）：需要及时的深度分析
  - John（从业者）：处理大量数据，使用多种工具提供洞察
  - Sarah（研究人员）：对分析进行深度研究，需要可扩展基础设施
- **23:00** - 客户面临的挑战：
  - 数据湖策略失败：仅转储数据无法提供价值
  - 最大客户花费 2 年构建数据湖，1 年后仍无法充分利用
  - 常见问题：缺乏领域特定组织、缺少内置分析工具、与工作流集成困难
- **25:00** - 风险数据湖（Risk Data Lake）解决方案：
  - 应用数据湖概念：数据湖基础 + 领域特定数据组织
  - 数据实体表示为投资组合、保单、位置等领域概念
  - 内置分析工具：拖放报告（QuickSight）、SQL 查询、可编程笔记本（Python/R）
  - 与智能风险平台完全集成
- **27:00** - 实施架构：
  - 应用将数据写入 S3 存储桶
  - 构建风险数据目录，以用户友好的方式组织数据
  - 创建 QuickSight 数据集，执行多表连接，提供可视化就绪的数据
  - 设计师使用这些数据集创建仪表板并发布
- **29:00** - 多租户实施：
  - 数百个客户使用多租户模型
  - 选择命名空间方法：每个租户独立的 IAM 角色、用户和安全组
  - 逻辑文件夹保存数据集、分析和仪表板
  - 通过安全组控制访问
- **30:30** - 角色级安全（RLS）：
  - 同一租户内不同用户访问不同数据资产
  - 将用户身份传递到 QuickSight
  - 使用 QuickSight RLS 功能实现细粒度访问控制
- **31:30** - 嵌入式体验：
  - 设计师体验完全嵌入风险数据湖应用
  - 设计师无需切换应用即可构建仪表板
  - 发布的仪表板在所有平台应用中可用（Risk Modeler、Treaty IQ 等）
  - 业务用户 Jane 无需切换上下文，在常用应用中访问仪表板
- **33:00** - 未来愿景：
  - "与仪表板对话"：业务用户通过自然语言提问获取答案
  - AI 辅助创作：设计师通过自然语言构建可视化和仪表板
  - 这些 GenAI 能力将彻底改变风险分析的开发和消费方式
- **34:00** - 项目时间线：2 月开始集成，10 月（2 个月前）正式发布
- **34:30** - 市场反响：已看到大量市场兴奋度
- **35:00** - 预期影响：
  - 民主化风险分析，Jane、John、Sarah 都能处理大量数据
  - 更好的风险选择和灾难影响理解
  - Jane 及时获得高级风险分析，特别是在续约周期
  - 无需多年 IT 项目即可增强分析能力
