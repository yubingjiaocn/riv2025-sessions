# AWS re:Invent 2025 技术会议总结

## 会议概述

本次技术会议由AWS解决方案架构师Dani和GenAI数据科学家Akarsha Sehwag主讲，重点介绍了如何构建智能代理RAG系统。会议采用400级别的深度技术讲解，通过实际代码演示展示了如何使用Amazon Bedrock、AgentCore Memory等AWS服务构建一个功能完整的事件助手代理。

演讲者详细介绍了从基础RAG概念到生产级部署的完整流程，包括知识库创建、记忆管理、工具集成以及安全身份验证等关键技术组件。整个会议以构建"AWSome事件代理"为实例，展示了如何实现个性化的用户体验和会话持久化功能。

## 详细时间线与关键要点

### 0:00-5:00 会议开场与介绍
- 演讲者自我介绍：Dani（Amazon Bedrock团队解决方案架构师）和Akarsha（AgentCore Memory团队GenAI数据科学家）
- 会议定位为400级别的深度技术讲解
- 使用Claude Haiku 4.5和Amazon Titan文本嵌入v2模型
- 承诺主要进行代码演示而非幻灯片讲解

### 5:00-10:00 RAG基础概念回顾
- 传统检索增强生成(RAG)工作流程说明
- 代理RAG与传统RAG的区别：代理本质上是无状态的
- 引入"上下文工程"概念，包括文档信息、工具策划和记忆组件
- 强调个性化体验的重要性

### 10:00-15:00 项目目标与架构
- 构建"AWSome事件代理"的目标说明
- 代理具备两大核心能力：从知识库检索会议信息、存储和检索对话历史
- 实现超个性化用户体验，支持基于偏好的推荐
- 展示整体架构图和实现章节规划

### 15:00-25:00 简单代理创建
- 使用Strands Agents框架创建基础代理
- 代码演示：一行代码创建代理实例
- 测试基础代理功能，验证无上下文时的响应行为
- 说明需要通过知识库提供上下文信息

### 25:00-40:00 知识库构建与集成
- Amazon Bedrock知识库服务详细介绍
- 完整的RAG工作流程：文档解析、分块、向量化、存储
- 代码演示知识库创建流程：
  - 下载re:Invent 2024会议数据（583个会议）
  - 创建S3存储桶并上传文档
  - 配置IAM角色和权限
  - 创建S3向量存储和索引
  - 执行数据摄取任务
- 知识库检索API测试和工具创建

### 40:00-50:00 记忆管理系统
- AgentCore Memory服务架构说明
- 短期记忆：原始对话消息存储
- 长期记忆：智能洞察提取（用户偏好、语义事实、会话摘要）
- 记忆提取和整合流程：提取相关信息、与现有记忆对比、去重处理
- 代码演示记忆管理器创建和配置

### 50:00-58:00 生产部署与安全
- AgentCore Runtime生产部署方案
- 会话级隔离：计算、内存、存储完全隔离
- 微虚拟机架构：每个会话独立运行环境
- AgentCore Identity身份验证集成
- JWT授权器配置和Cognito集成
- 最终资源分享和问答环节
