# AWS re:Invent 2025 生成式AI与AI代理会议总结

## 会议概述

本次AWS re:Invent 2025分组会议聚焦于生成式AI和AI代理(AI Agents)的最新发展。会议由Yamauchi和Kanasugi两位讲师共同主持,分为两个部分:前半部分介绍了生成式AI的发展历程和AI代理的基本概念,后半部分深入探讨了如何将AI代理融入企业业务运营和服务中的具体方法。

从2023年的探索阶段,到2024年的全面实施,再到2025年AI代理成为业务转型的关键,生成式AI技术经历了快速演进。Amazon Bedrock作为AWS的核心AI平台,为全球数十万客户提供服务,支持多种领先模型包括Anthropic、Meta、Cohere、DeepSeek、OpenAI以及AWS自研的Amazon Nova系列模型。会议特别强调了AI代理与传统生成式AI的本质区别:AI代理能够自主思考、适应环境并独立完成复杂任务,而不仅仅是响应人类指令。这种转变正在汽车、能源、金融、电信等多个行业引发深刻变革。

会议还详细介绍了AWS为支持AI代理开发和部署提供的完整解决方案,包括Amazon QuickSuite用于日常业务自动化、Amazon Bedrock Agent Core用于企业级AI代理部署,以及Kiro等开发工具用于AI驱动的软件开发。AWS强调了其在多供应商支持、成本优化、安全性、数据集成等方面的优势,并通过Sony集团、Amazon Rufus、Amazon Seller Assistant等实际案例展示了AI代理在2C、2E、2B不同场景下的应用价值。

- **18:00** - 讨论AI代理数据处理需求:简单任务连接简单,复杂业务需要治理机制,包括功能访问控制和重复功能整合

- **26:00** - 2B案例:Amazon Seller Assistant - 面向卖家的服务,基于店铺信息提供销售和库存优化建议、业务增长策略、销售分析和新品建议,减轻商品上架工作负担

- **32:00** - 业务规划人员案例:管理者发现产品销售不佳,通过Slack通知负责人,AI代理自动研究产品手册、销售记录和客户反馈,识别改进点并生成研究文档
- **34:00** - IT基础设施管理案例演示:展示如何使用QuickSuite优化AWS服务使用

### Amazon Bedrock与企业级AI代理部署 (40:00 - 55:00)
- **40:00** - 介绍第二种模式:通过AI增强系统,将AI代理融入企业服务,为终端用户提供新体验
- **41:00** - Amazon Bedrock自2023年推出以来,全球数十万客户在生产环境中使用,过去一年客户数量增长约5倍
- **42:30** - 客户选择AWS的六大理由:1)功能丰富 2)多供应商兼容 3)成本优化 4)降低法律风险 5)安全性 6)数据集成
- **43:30** - 讨论AI代理项目挑战:Gartner预测到2027年,超过40%的AI代理项目将被放弃,原因包括成本上升、业务价值不明确、风险管理不足
- **45:00** - 列举生产环境部署AI代理的具体挑战:安全执行和扩展、记忆和存储交互、认证授权、自定义工具和数据源、调试困难、遥测和交互测量

### Amazon Bedrock Agent Core详解 (48:00 - 55:00)
- **48:00** - 发布Amazon Bedrock Agent Core:用于安全、大规模部署和运营AI代理的服务,东京区域已可用
- **49:00** - 介绍Agent Core的构建块(Building Blocks)理念:组合各种服务创建最适合需求的架构
- **49:45** - Runtime(运行时):无服务器执行环境,每个会话在微虚拟机上隔离,支持任何框架(Crew AI、LangChain等)和LLM(Bedrock模型、GPT、Gemini等)
- **51:00** - Gateway(网关):从单一端点发现和连接工具,工具提供者可轻松将现有API转换为工具
- **51:45** - Browser和Code Interpreter:提供浏览器和代码执行环境
- **52:15** - Identity(身份):处理认证和授权
- **52:30** - Memory(内存):分为短期记忆(最近交互)和长期记忆(用户偏好等)
- **53:00** - Observability(可观测性):记录每个步骤和追踪,提供仪表板
- **53:45** - 宣布两项新功能:1)**Gateway Guardrails**:用自然语言定义工具调用策略(如限制超过$1000的退款) 2)**Evaluation**:评估代理行为是否符合人类期望

### AWS选择理由详解 (55:00 - 65:00)
- **55:00** - 多供应商支持:Agent Core支持任何框架和LLM,满足混合需求,构建灵活的AI代理平台
- **56:30** - 持续成本优化:提供高性价比的Amazon Nova、Graviton、Trainium等计算服务,已累计降价161次,今年GPU实例降价45%,Guardrails降价85%
- **58:00** - 降低法律风险:提供知识产权侵权保护保证,如第三方声称输出侵权,AWS将提供保障
- **59:00** - 企业级安全:满足所有合规标准,首个获得AI管理系统认证的主要云服务提供商,不使用客户数据训练模型,Agent Core为每个用户会话分配隔离的微虚拟机,支持细粒度访问控制
- **61:00** - 数据集成:46%的组织认为数据治理是AI采用的重要标准,Bedrock支持企业内部数据利用
- **62:00** - Matt Garman主题演讲中宣布Nova Forge:允许使用自有数据定制Nova模型
- **62:45** - 强调数据基础设施对AI代理的重要性:传统数据基础设施为人类分析优化,AI代理时代需要数据资产易于被AI利用
- **63:30** - AWS解决方案:结合以Bedrock为中心的AI平台和以Amazon S3为中心的湖仓架构,实现数据共享和协作
- **64:15** - 宣布AWS与Oracle战略合作:除RDS for Oracle外,新增Autonomous Database和Oracle Exadata选项,北美已正式可用,未来将在日本区域推出

### AI驱动开发与Kiro工具 (65:00 - 75:00)
- **65:00** - 介绍第三种模式:AI代理提升开发者生产力,软件开发是AI和AI代理带来的最革命性领域
- **66:00** - AI演进软件构建方式:从代码补全→聊天式编程→Vibe Coding(开发者与AI实时协作)→Coding Agents(自主规划、构建、测试、部署)
- **67:30** - 展望未来:人类开发者与编码代理协作的世界即将到来,需要重新定义价值创造和人机交互方式
- **68:15** - 强调AI驱动开发将进一步提升云价值:云抽象基础设施,基于API的平台易于AI使用,通过自动化创造大量价值
- **69:00** - 开发者可用自然语言向AI代理传达需求,AI不仅生成应用代码,还参考AWS提供的MCP服务器,组合AWS服务构建满足业务需求的系统
- **70:00** - 发布**Kiro**:AWS新的集成开发环境(IDE),内置AI代理支持系统开发,可安装在客户端PC上
- **70:45** - Kiro CLI:允许从命令行交互式操作系统,最近向公众发布,企业客户可采用
- **71:15** - Matt Garman主题演讲中宣布Kiro Autonomous Agent
- **71:45** - 介绍**Usage-Driven Development(使用驱动开发)**:解决Vibe Coding中代码可能成为黑盒的问题,通过人机对话达成共识,先生成使用文档,再确定实现方式,显著提高可靠性

### AI人才培养与AIDLC项目 (72:00 - 结束)
- **72:00** - 引用研究数据:到2028年,78%的雇主将演进为AI驱动组织,68%的雇主将招聘AI人才作为优先事项,但82%的公司难以获得必要的AI人才
- **73:00** - 澄清AI人才定义:不是AI专家,而是能够使用AI高效改进业务运营的人员
- **73:30** - 调查显示:仅30%的公司考虑招聘或外包外部人才,大多数考虑对现有员工进行再培训或技能提升
- **74:00** - 强调AI人才培养的战略重要性:技术变革快速,组织变革相对缓慢,需要技术和组织响应两个轴心加速生成式AI和AI代理的引入
- **74:45** - 推出**AIDLC Unicorn Gym项目**:与客户共同推进AI驱动开发的计划
- **75:15** - 定义AI利用的三个阶段:1)AI辅助开发(使用AI编码工具,易于上手但仍需大量手工工作) 2)AI驱动开发/AIDLC(充分利用AI能力重构软件开发,人类保持最终决策权,构建生产级软件) 3)AI管理开发(使用AI自主构建和运营,但可靠性仍有待提高)
- **76:30** - 会议结束
