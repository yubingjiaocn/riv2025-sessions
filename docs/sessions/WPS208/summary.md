# AWS re:Invent 2025 WPS208 会议总结

## 会议概述

本次会议由 Covered California（加州全保）与 AWS 联合呈现，主题为"数据驱动的客户留存策略"。Covered California 是加州的州立医疗保险交易平台，服务于加州居民的医疗保险需求。会议由 AWS 高级销售专家 Chris Carter、Covered California 首席信息官 Kevin Cornish 和首席营销官 Glenn O. Young 共同主讲。

会议重点介绍了 Covered California 如何利用 AWS 技术应对当前医疗保险市场的挑战。面对联邦补贴即将到期、公众信任度下降以及市场复杂性增加等多重压力，Covered California 通过部署先进的身份验证系统和数据驱动的客户留存模型，显著改善了用户体验并提高了注册转化率。组织的核心使命是确保每位加州居民都能获得可负担的高质量医疗保险，目前已成功将加州的无保险率从 17% 降至 6%，服务覆盖六分之一的加州居民。

Glenn 在会议中分享了个人故事，讲述了他父亲作为移民工程师在医疗系统中遇到的困难，强调了医疗公平性对于非英语母语者和弱势群体的重要性。这个故事体现了 Covered California 团队对使命的深刻承诺。会议还展示了全新的品牌宣传片"For the Love of Californians"（为了加州人的爱），彰显了组织对服务对象的深切关怀。

### Covered California 组织介绍 (05:30 - 15:00)
- **05:30** - Glenn 开始介绍 Covered California 作为加州州立医疗保险交易平台的角色
- **06:15** - 展示合作的保险计划，包括 Blue Shield、Kaiser Permanente、Health Net 等主要保险公司
- **07:00** - 阐述组织的四重使命：连接居民与保险计划、提供可负担选择、促进医疗公平、提供卓越服务

- **10:45** - 强调医疗公平对于不会说英语、没有大学学历或缺乏支持系统的人群的重要性
- **12:00** - 介绍组织成就：六分之一的加州居民曾是或将成为 Covered California 会员
- **13:00** - 如果加上 Medi-Cal（加州医疗补助计划）合作伙伴，三分之一的加州居民得到了服务
- **14:00** - 成功将加州无保险率从 17% 降至 6%，意味着 94% 的符合条件的加州居民已获得保险

### 品牌战略与市场挑战 (15:00 - 25:00)

- **15:45** - 讨论当前面临的挑战：政府关闭、联邦补贴即将到期、政策变化和医疗保健领域的广泛变化
- **17:00** - 展示全新品牌平台"For the Love of Californians"（为了加州人的爱）
- **17:30** - 播放品牌宣传视频，展示加州的多样性（60 个国家、200 种语言）和组织承诺
- **19:00** - 视频强调："我们不是医疗保险公司，我们是为你服务的，因为我们就是你"
- **20:00** - Glenn 解释虽然品牌平台是新的，但对加州人的爱一直是组织的核心
- **21:30** - 讨论如何在信任度下降和市场复杂性增加的环境中展现这种关爱

### Kevin 介绍组织的三大关注点 (25:00 - 35:00)
- **25:00** - Kevin 阐述 Covered California 的三大关注点：使命、消费者体验和信息安全隐私
- **26:00** - 强调作为州政府机构，Covered California 对消费者体验的执着追求
- **27:30** - 介绍 CalHEERS 平台作为加州 Medicaid 系统资格确定的门户
- **28:15** - 每年约 1700 万加州居民（接近总人口 3900 万的一半）通过该系统
- **29:00** - 累计收集了近 2500 万加州居民的个人身份信息和健康信息
- **30:00** - 强调对信息隐私和安全的高度重视

### 案例研究一：身份验证挑战 (35:00 - 50:00)
- **35:00** - Kevin 介绍发现的问题：33% 的用户在资格确定流程中退出
- **36:30** - 问题根源：基于信用检查的身份验证方式让用户感到焦虑
- **37:45** - 许多目标人群信用评分不佳或根本没有信用记录
- **38:30** - 用户不愿与政府分享信用信息，导致三分之一的潜在客户流失
- **40:00** - 介绍 Covered California 已在 AWS 云上运行四年，从本地解决方案迁移到云原生微服务架构
- **42:00** - 选择 Socure 解决方案，该方案不使用基于信用的风险评分
- **43:30** - 新方案解决两个问题：减少用户流失和防范欺诈、浪费和滥用
- **45:00** - 展示实施成果：实现无缝部署，用户体验无明显差异
- **46:00** - 立即发现约 5% 的交易存在欺诈行为并成功拦截
- **47:00** - 85% 的用户通过被动验证流程，无需任何额外操作
- **48:00** - 15% 未通过被动验证的用户进入文档验证流程
- **49:00** - 最终 98% 的消费者成功通过身份验证步骤，原先 33% 的流失问题完全解决

### Chris 介绍文档验证流程 (50:00 - 58:00)
- **50:00** - Chris 详细介绍被动验证流程：用户输入个人身份信息后，Socure 通过设备分析等技术被动验证身份
- **51:30** - 84% 的用户在不知情的情况下通过被动验证
- **52:30** - 介绍 DOCV（文档验证）流程：未通过被动验证的 16% 用户需要提供政府颁发的身份证件和生物识别信息
- **53:45** - 系统会检查照片的真实性，包括眼睛是否睁开、是否左右转头等活体检测
- **54:30** - 甚至检查肩膀位置，因为用手机自拍时肩膀会抬起
- **55:00** - 展示技术架构图，列出 Socure 使用的 AWS 服务
- **55:45** - Amazon Textract 用于从文档中提取文本信息
- **56:15** - Amazon Rekognition 用于照片识别和活体检测
- **56:45** - SageMaker 用于开发和改进机器学习模型
- **57:15** - Amazon Pinpoint 用于 SMS 和电子邮件消息传递（正在迁移到终端用户消息服务）
- **57:45** - 所有系统与 CalHEERS 应用程序位于 AWS 同一区域，确保低延迟和安全性

### 案例研究二：客户留存策略 (58:00 - 70:00)
- **58:00** - Glenn 介绍第二个案例研究，聚焦客户留存
- **59:00** - 解释 Covered California 的独特性：平均会员任期仅 14 个月，是过渡性服务
- **59:45** - 理想路径是会员最终获得雇主提供的医疗保险
- **60:30** - 但当前环境要求组织更加关注留存，防止居民失去保险
- **61:30** - 讨论信任危机：无论联邦、州还是市级政府，公众信任度都在下降
- **62:45** - 医疗系统的复杂性、意外账单和就医困难导致信任度持续下降
- **63:30** - Covered California 的品牌认知度虽高，但深入了解后发现认知和好感度呈下降趋势
- **65:00** - 联邦税收补贴将于年底到期，国会对是否延期存在争议
- **66:00** - 这种不确定性造成情绪波动，许多消费者选择延迟行动或放弃
- **67:00** - 内部预测显示如果补贴到期，可能有三分之一的会员选择不续保
- **68:00** - 许多人面临保费翻倍的困境，必须在食物、汽油、房租和医疗保险之间做出选择
- **69:00** - 进行了 21 场焦点小组访谈，覆盖全州，了解真实居民的困境

### 数据驱动的留存模型 (70:00 - 80:00)
- **70:00** - 介绍解决方案：通过技术和合作伙伴挖掘自有数据
- **71:00** - 建立留存模型，分析客户流失风险
- **71:45** - 基于风险评分进行客户细分
- **72:30** - 左侧创建更个性化、更细致的营销信息
- **73:15** - Glenn 提到消费者对 Amazon 等品牌的体验期望同样适用于政府服务
- **74:00** - 人们期望组织了解他们并提供个性化服务
- **74:45** - 这种个性化不仅体现在营销信息中，也体现在客服人员的服务中
- **75:30** - Chris 介绍 Deloitte 团队如何与 Covered California 合作开发风险评分模型
- **76:15** - 风险评分是会员续保或不续保的建模概率
- **77:00** - 考虑因素包括：保费增长、收入方式（固定工资 vs 零工经济）、年龄、是否有被抚养人等
- **78:00** - 通过分析历史数据，预测保费增长对不同受众群体的影响
- **78:45** - 使用 Snowflake 上的 R 语言大规模建模，覆盖数十万甚至数百万人

### 受众细分示例 (80:00 - 结束)
- **80:00** - 举例说明受众细分：26-35 岁年轻成年人，居住在加州城市，从事零工或按小时计酬工作
- **81:00** - 这类人群主要关注保险成本，每月 10-20 美元的增长都会产生重大影响
- **82:00** - 另一个细分群体：45-55 岁的夫妇，抚养被抚养人，可能有慢性健康问题
- **82:45** - 这类人群虽然也关注价格，但持续拥有保险是最重要的
- **83:30** - 强调人们通常不重视健康，直到出现问题（如背部疼痛）
- **84:00** - 基于保费增长影响开发了五个初始受众细分
- **84:30** - 使用 AI 技术为每个细分群体创建定制化的沟通策略
