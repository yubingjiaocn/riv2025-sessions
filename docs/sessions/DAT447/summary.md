# AWS re:Invent 2025 DAT47 会议总结

## 会议概述

本次会议主题为"使用数据湖上的图技术赋能生成式AI应用"，由AWS Amazon Neptune团队的Brad BBE（总监）和Dr. Umit Kalurick（Amazon学者）主讲。会议深入探讨了如何在数据湖上利用图处理技术来提升生成式AI应用的准确性。演讲者强调，图技术的真正价值不在于构建图本身，而在于利用图中的关系来解决实际问题和回答复杂问题。

会议介绍了三大核心策略：第一，使用Neptune Analytics直接在数据湖上进行图查询和算法处理；第二，结合向量搜索和图遍历技术来提升检索准确性；第三，为AI代理添加基于图的记忆功能以改善用户体验。演讲者通过多个实际案例展示了这些技术的应用效果，包括Whiz的云安全态势管理、Trend Micro的安全聊天机器人等，证明了图技术能够显著提升应用准确性（从70%提升到90%）并改善用户满意度。

会议还提供了多个Jupyter notebook示例和QR码链接，让参会者可以亲自实践这些技术，包括航空数据分析、移动支付欺诈检测、向量搜索与图结合等场景。整体而言，本次会议为开发者提供了在数据湖环境中应用图技术的完整方法论和实践指南。

### 图算法详解 (15:00 - 25:00)
- **15:00** - 详细讲解PageRank算法：Google著名的网页排名算法
- **16:00** - PageRank原理：节点重要性等于指向它的节点重要性的平均值
- **17:00** - 算法实现方式：可从目标或源节点角度计算
- **18:00** - 演示如何调用PageRank：使用mutate模式将结果写入图属性
- **19:00** - 算法参数说明：迭代次数（通常10-20次足够）、顶点和边标签选择
- **20:00** - 数据导出：将计算结果导出回数据湖供其他系统使用
- **21:00** - 介绍中心性算法：识别图中的关键节点
- **22:00** - Closeness Centrality（接近中心性）：测量节点到其他节点的平均距离
- **23:30** - 算法优化：采样、向量化和并行化技术提升性能
- **24:30** - 演示调用Closeness Centrality算法，识别Charles de Gaulle机场为中心节点
