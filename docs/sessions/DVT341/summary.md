# AWS re:Invent 2025 技术会议总结

## 会议概述

本次会议由 AWS 下一代开发者体验团队的技术负责人 Brian Beach 主讲，主题聚焦于如何教会 AI 工具理解和使用企业自定义语言和库。演讲者以 Amazon Kiro（一个 AI 代理 IDE）为例，详细展示了如何通过上下文管理、引导文件（Steering）和模型上下文协议（MCP）等技术，让 AI 工具掌握大语言模型从未见过的领域特定语言。

演讲者创建了一个虚构的领域特定语言 MathJSON 作为演示案例，这是一种用 JSON 语法编写数学方程的语言。通过四个递进式的演示，Brian 展示了从 Kiro 完全不了解该语言到能够熟练使用它的完整过程。这个案例虽然是虚构的，但代表了许多企业面临的实际问题——如何让 AI 工具理解组织内部的自定义工具、库或技术栈。整个演示过程强调了一个核心理念：应该让 AI 自己做决策和优化，而不是手动完成所有配置工作。

- **3:00** - 说明将进行四个递进式演示，从失败到成功

- **4:00** - 2023 年：Code Whisperer（代码补全）
- **4:20** - 2024 年：Amazon Q Developer（对话式交互）

### 上下文管理工具介绍 (5:30-8:00)
- **5:30** - 介绍两个核心工具：MCP 和 Steering
- **6:00** - MCP（模型上下文协议）：让代理访问额外信息并操作环境
- **6:30** - 展示 MCP 示例：GitHub 连接器和 PostgreSQL 数据库连接
- **7:00** - Steering（引导文件）：定义 AI 行为的规则集

### MathJSON 介绍 (8:00-9:30)
- **8:00** - 介绍虚构的领域特定语言 MathJSON
- **8:20** - 说明 MathJSON 是用 JSON 语法编写数学方程的语言
- **8:40** - 展示计算圆面积的 MathJSON 代码示例
- **9:00** - 强调这是私有仓库，模型从未训练过该语言
- **9:15** - 指出 GitHub 仓库中有完整文档，类似于新员工入职培训材料

### 演示一：基线测试（失败） (9:30-11:00)
- **9:30** - 开始第一个演示，Kiro 对 MathJSON 一无所知
- **10:00** - 要求创建抵押贷款超额还款计算函数
- **10:20** - Kiro 用 Python 而非 MathJSON 编写代码
- **10:40** - 虽然 Python 代码质量很高且经过测试，但不符合要求
- **11:00** - 总结问题：Kiro 不知道 MathJSON 的存在

### 演示二：添加文档但不够明确 (11:00-13:30)
- **11:00** - 将 MathJSON 函数参考文档（约 1100 行）添加到 Steering 文件
- **11:30** - 再次尝试相同的请求
- **12:00** - Kiro 仍然使用 Python 编写代码
- **12:30** - 修改提示词，明确说明"使用 MathJSON"
- **13:00** - 这次 Kiro 开始创建 JSON 文件，但仍有问题
- **13:20** - 总结问题：文档不够明确，未说明何时使用 MathJSON，文件扩展名错误，文档过于冗余

### 演示三：优化 Steering 文件 (13:30-16:00)
- **13:30** - 使用 Kiro 的"优化"功能精简文档
- **14:00** - Kiro 花费约 90 秒分析并优化文档
- **14:30** - 文档从 1100 行压缩到约 100 行（减少 90%）
- **15:00** - 添加明确的指令：项目使用 MathJSON，必须用于所有数学计算
- **15:20** - 包含 linter 使用说明和测试方法
- **15:40** - 再次测试，Kiro 正确使用 MathJSON 并使用正确的文件扩展名
- **15:50** - Kiro 自动执行 linter 验证代码正确性

### 演示四：集成 MCP 实现动态文档访问 (16:00-18:30)
- **16:00** - 指出问题：创建了文档的衍生副本，可能过时
- **16:20** - 只使用了十几个文档中的一个
- **16:40** - 引入 MCP 的"获取文件内容"工具
- **17:00** - 更新 Steering 文件，指向 GitHub 文档而非嵌入副本
- **17:20** - Kiro 实时从 GitHub 读取文档
- **17:40** - 成功创建 MathJSON 文件，使用正确扩展名
- **18:00** - Kiro 执行 linter 并编写测试用例
- **18:20** - 展示完整的工作流程：从不了解到熟练使用

### 进阶建议 (18:30-20:00)
- **18:30** - 讨论未涵盖的优化方法
- **18:50** - 建议一：让 Kiro 自我更新 Steering 文件
- **19:10** - 当 Kiro 发现任务过于复杂时，可以要求它更新自己的指导文档
- **19:30** - 建议二：使用向量数据库（如 Bedrock Knowledge Bases）或语义搜索（OpenSearch/Elasticsearch）
- **19:50** - 这样可以让 AI 只检索所需的文档片段，而非整个文档
