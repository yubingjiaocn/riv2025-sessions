# AWS re:Invent 2025 Dev 336 会议总结：Kiro 实战 - 构建 IAM 扫描器

## 会议概述

本次会议由 FICO 的红队经理 Nick Gilbert 和高级云安全工程师 Damien 主讲，重点介绍了如何使用 Kiro AI 驱动的 IDE 快速构建生产级 IAM 安全扫描工具。会议针对大型组织在进行红队评估和渗透测试时面临的可见性缺口问题，展示了通过规范驱动开发方法构建自动化安全工具的实践经验。

两位演讲者都是 AWS 社区建设者，具有丰富的云安全实战经验。会议展示了从传统的"氛围编程"转向结构化的规范驱动开发方法，如何将开发时间从 200-350 小时缩减至 25-35 小时，实现 70% 的效率提升。

## 详细时间线与关键要点

### 0:00-2:00 问题定义与演讲者介绍
- 识别大型组织 AWS 环境中的可见性缺口问题
- Nick Gilbert：FICO 红队经理，AWS 社区建设者，安全专家认证 SME
- Damien：FICO 高级云安全工程师，DevSec Blueprint 创始人，内容创作者

### 2:00-4:00 IAM 扫描器功能需求
- 识别高价值攻击路径
- 枚举跨账户访问权限
- 按影响程度优先排序角色
- 发现通配符和信任策略等常见错误
- 提供具体的权限提升路径
- 支持可重复的扫描结果验证
- 使用 Python 多线程技术同时扫描数百个账户

### 4:00-6:00 开发方法论对比
- **氛围编程**：与 AI 随意交互，适合概念验证但不可扩展
- **规范驱动开发**：架构师角色，预先规划需求和实现方案
- Kiro 作为 AI 驱动 IDE 的优势：支持规范驱动开发，用户友好界面，多语言支持

### 6:00-7:30 PROMPT 框架介绍
- **P**urpose：项目目的简述
- **R**equirements：高层次程序概述
- **O**utput：交付物格式定义
- **M**ethod：详细实现方法说明
- **P**rompt：让 AI 提出澄清问题
- **T**estability：单元测试创建能力

### 7:30-9:00 IAM 扫描器工作原理
- 缓存 AWS 托管策略（1000+ 个策略）
- 收集 IAM 数据（用户、组、角色、客户托管策略）
- 使用策略引擎分析高权限策略
- 检查主体的托管和内联策略
- 分析角色信任策略以识别横向移动路径
- 识别未使用的高权限角色
- 分析权限提升路径并提供具体代码示例

### 9:00-11:00 效率对比与实际结果展示
- 传统开发：200-350 小时
- 使用 Kiro：25-35 小时（70% 时间节省）
- 实际扫描结果：发现 40 个未使用角色，其中 21 个存在危险
- 生成 JSON 和 Markdown 两种格式报告
- 支持 PDF 导出用于管理层汇报

### 11:00-11:30 总结与后续步骤
- 关键要点：使用规范驱动设计快速开发安全解决方案
- 通过文本需求转换为可工作代码
- 创建自动化工具发现云环境安全风险
- 提供 GitHub 代码库和社交媒体联系方式
