# AWS re:Invent 2025 代码会话：生成式AI工具的内部使用实践

## 会议概述

本次会话由Amazon首席软件工程师James Hood和AWS高级开发者倡导者Maish Saidel-Keesing主讲，分享了Amazon/AWS内部如何广泛采用生成式AI工具的实践经验。会话分为两个主要部分：Maish介绍了提示工程的最佳实践技术，James展示了Agent SOP（Agent标准操作程序）的实际应用。

两位演讲者强调，Amazon内部已经全面拥抱生成式AI工具，每天在各个场景中使用这些工具。他们通过"机制"（mechanism）的方式将成功的实践标准化并在公司内部推广，体现了Amazon"发明和简化"的领导力原则。会话包含大量实时代码演示，展示了从简单提示到复杂代码实现的完整工作流程。

## 详细时间线与关键要点

### 0:00-10:00 开场介绍与背景
- James Hood自我介绍：Amazon首席软件工程师，16年工作经验，曾是AI怀疑论者
- Maish Saidel-Keesing介绍：AWS高级开发者倡导者，负责营销组织工作
- 说明会话将包含大量代码演示，部分内容需要坐下操作
- 介绍Amazon内部的"机制"概念：将成功实践标准化并推广的工具

### 10:00-25:00 提示工程基础技术
- **零样本提示（Zero-shot）**：直接向LLM提问，获得基本答案
- **单样本提示（One-shot）**：提供一个示例，帮助LLM理解期望的输出格式
- **少样本提示（Few-shot）**：提供多个示例，为复杂任务提供更多上下文
- 演示IAM策略验证脚本的不同提示方式，展示从Python到JavaScript的输出变化
- 最佳实践：简单任务使用零样本，复杂任务使用少样本提示

### 25:00-35:00 高级提示技术
- **系统提示**：为LLM提供基本指令和上下文
- **上下文提示**：提供具体的工作环境和组织结构信息
- **角色提示**：让LLM扮演特定专业角色（如AWS解决方案架构师）
- 演示EC2实例查询脚本的不同提示方式
- 最佳实践：结合多种技术处理复杂任务，先设定上下文，再定义角色，最后指定任务

### 35:00-45:00 自一致性提示和思维树
- **自一致性提示**：要求LLM生成多个独立解决方案并选择最佳方案
- 演示多区域高可用架构设计的三种方案比较
- **思维树方法**：结构化探索多个分支，每个分支代表不同的解决方法
- 演示Lambda性能优化的系统性分析方法
- 适用场景：复杂架构决策、故障排除、安全分析、性能优化

### 45:00-55:00 上下文增强和工作流程
- 展示如何将Amazon内部文档写作风格加载到LLM上下文中
- 演示"探索-计划-编码-提交"工作流程
- 实际代码示例：为现有JavaScript认证系统添加密码重置功能
- 强调提供更多上下文信息可以获得更好的结果
- 介绍开源代码库，包含所有演示示例

### 55:00-63:30 Agent SOP实战演示
- James介绍Agent SOP（Agent标准操作程序）概念和开源项目
- 演示代码库摘要生成：自动分析项目结构、生成架构图、创建文档
- 展示Agent SOP的标准格式：标题、概述、参数、步骤约束、示例、故障排除
- 实时创建GitHub问题分类SOP，展示自动化问题标记和评论功能
- 演示PDD（属性驱动开发）流程：从需求澄清到设计实现的完整开发周期
- 最终展示代码辅助SOP：使用测试驱动开发实现新功能，包括红-绿-重构循环
