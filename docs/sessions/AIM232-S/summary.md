# AWS re:Invent 2025 会议总结：Agentic AI Ops 的未来

## 会议概述

本次 AWS re:Invent 2025 分组会议聚焦于 Agentic AI Ops（智能体 AI 运维）的发展与应用。会议由 Logic Monitor 的产品营销负责人 Andrew Keating 主持，邀请了 NTT Data 美洲区云与安全业务副总裁 Abby Lani 和 Worldwide Technology 全球 AI Ops 实践经理 Mike Cervasio 作为嘉宾。

会议重点讨论了可观测性与生成式 AI 的融合如何推动 IT 运维从被动响应向主动预防转变。Logic Monitor 在会议期间宣布收购 Catch Point，这是一家数字体验监控和互联网性能监控领域的领导者，拥有超过 3,000 个全球监测点。此次收购将 Logic Monitor 在混合环境基础设施监控方面的优势与 Catch Point 在互联网性能监控（IPM）和数字体验监控（DEM）方面的能力相结合，打造了业界最全面的可观测性平台，能够展示从用户到代码的完整体验链路。

三位嘉宾深入探讨了 Agentic AI Ops 如何改变传统的 IT 运维模式，特别是在托管服务提供商（MSP）领域。客户不再满足于传统的 SLA（服务级别协议），而是要求通过智能代理实现主动监控、实时根因分析（RCA）和闭环自动化修复。这种转变正在重塑 MSP 的服务交付模式，从固定成本、基于 SLA 的解决方案转向基于结果的灵活服务模式。会议强调，在当前 AI 时代，可观测性和自主 IT 运维已不再是可选项，而是企业保持竞争力的必要条件。

- Andrew Keating 介绍会议形式和目标：深入探讨 Agentic AI Ops

00:02:30 - Logic Monitor 公司介绍
- 成立于 2007 年，完全基于 SaaS 的原生平台，托管在 AWS 上
- 平台拥有超过 10 万软件用户，每天监控超过 500 万台设备
- 每天收集约 1.5 万亿个指标数据
- 全球员工超过 1,000 人，在 Gartner 魔力象限中被评为挑战者

00:03:45 - 重大收购公告：Logic Monitor 收购 Catch Point
- Catch Point 是数字体验监控和互联网性能监控领域的领导者
- 拥有超过 3,000 个全球监测点
- 此次收购将创建最全面的可观测性平台，覆盖从用户到代码的完整链路
- Catch Point 曾在最近的 AWS 故障发生前 15 分钟检测到问题

00:05:20 - Mike Cervasio 补充 Catch Point 的重要性
- 强调在频繁发生的互联网中断（如 Azure、Cloudflare）背景下，监控不受自己控制的依赖服务的健康状态至关重要
- 需要了解故障域的位置

00:06:15 - 行业趋势分析
- 可观测性、生成式 AI 和 Agentic AI 正在融合
- 需要整合遥测数据、ITSM 中的上下文知识、互联网性能数据和数字体验合成数据
- IT Ops 是 AI 智能体应用的最佳场景之一
- 从关注可见性转向采取行动和主动预防事故

00:07:30 - Agentic AI Ops 的价值驱动因素
- 提高运营效率：需要将 AI 智能体与人类协同工作
- 降低成本：可能意味着角色转变，减少 L1 支持人员
- 提升生产力：让 IT 运维人员专注于更高价值的工作
- 确保业务性能：提高正常运行时间，降低平均恢复时间（MTTR）

00:09:00 - 合作伙伴关系讨论
- Mike Cervasio 分享 Worldwide Technology 的视角：
  - Logic Monitor 的 Edwin AI 使其脱颖而出
  - 重视合作伙伴关系和协作解决客户需求
  - 在高级技术中心和 AI 验证场地中使用 Logic Monitor

00:10:15 - Abby Lani 分享 NTT Data 的合作经验
- NTT Data 定位为全球少数几家全栈转型公司之一
- 全栈可观测性是基础要求
- 在产品制造、SLED（州、地方和教育）等多个行业取得成功
- 与 MSP 模式结合，共同创造有价值的解决方案

00:12:00 - 从监控到可观测性的演进
- 行业长期讨论从被动到主动的转变
- AI Ops 作为概念已存在约十年
- 从机器学习和确定性模型演进到 Agentic 状态
- 这是行业的重要转折点

00:13:30 - Mike 强调紧迫性
- "不再是可选项，而是强制性的"
- 可观测性和自主 IT 运维是差异化竞争的关键
- 与一年前相比，变化速度大大加快

00:14:45 - 为客户创造价值的讨论
- Abby Lani 分享 NTT Data 的客户价值：
  - 托管服务交付方式发生根本性变化
  - 客户质疑是否还需要 L1 支持层级
  - 客户要求部署能够主动跟踪事件、关联事件、实现闭环自动化和实时 RCA 的智能体

00:16:00 - 服务模式的灵活性
- 从传统的固定成本、基于 SLA 的解决方案转向基于结果的模式
- 提供"即服务"选项
- 案例：为孟菲斯郊外的超级工厂项目提供灵活的 NOC 和 SOC 解决方案

00:18:00 - 速度和规模的挑战
- Abby 坦言快速部署让 NTT Data 这样的大型企业感到不适
- 需要合作伙伴帮助实现传统上不具备的速度

00:19:00 - Worldwide Technology 的客户价值方法
- Mike 分享战术工具替换需求
- 更倾向于自上而下的战略方法：构建下一代 NOC、SOC 或企业指挥中心
- 使用 SLO 和变更管理风险来衡量
- Edwin AI 的 Agentic L1 响应能力释放资源用于更具战略性的工作

00:20:30 - 多供应商环境集成
- 展示多 OEM 环境的概念验证
- 利用事件驱动自动化和自主 IT 运维自动化
- 通过警报、事件的海量噪音获取洞察和关联性

00:21:15 - 参考架构方法论
- Worldwide Technology 首先最大化客户已有投资
- 发现盲点和冗余
- 基于高级技术中心的实际测试提供意见

00:22:00 - 数字用户体验的重要性
- Mike 强调用户体验是业务健康的关键
- 以 American Express 为例，测量跨多个托管域的客户旅程
- 真正的健康指标不是网络或应用程序是否正常运行，而是客户是否遇到问题

00:23:30 - Agentic AI Ops 与传统 AI 的区别
- Mike 的观点：
  - 能够采取多项行动
  - 使用 LLM 和 RAG 作为大脑
  - 在几秒钟内做出建议和分析变更
  - 检查变更管理数据库、知识文章、产品手册，甚至 Slack 频道中的非结构化对话数据
  - 快速解读各组织单位的个性化仪表板

00:25:00 - Abby 对 Agentic AI 差异化的看法
- 核心设计原则是模块化和基于单元的基本设计
- 自主独立的智能体在闭环协作 AI 系统中工作
- 快速决策，从持续反馈循环中学习
- 不仅改善智能体提供的服务，还持续改善基础设施和环境健康

00:26:30 - 从 SLA 到环境改进的转变
- Abby 分享客户不再关注 SLA，而是关注环境改进
- 要求跟踪基础设施使用模式和趋势，并与业务事件关联
- Agentic AI 是衡量这些结果的途径

00:28:00 - MSP 模式的转变
- Abby 详细说明 Agentic AI Ops 如何改变 MSP 模式：
  - 降低服务成本
  - 客户要求在三年期内实现 40-50% 的生产力提升
  - 无法通过传统的外包和人员金字塔调整实现
  - 必须找到减少服务环境的方法

00:29:30 - 预测性和主动性监控
- 在事故成为事故之前预防
- 客户已开始期待这种能力
- 实时 RCA 和闭环自动化消除根本原因
- 减少需要服务的环境规模

00:31:00 - MSP 资源优化
- 任何组织能够维持的交付人员数量有限且需保持成本效益
- 需要确保资源分配到项目中，同时服务大量客户群
- AI Ops 技术帮助实现这一平衡

00:32:00 - Mike 对 MSP 和最终客户的观察
- 电信公司（网络即业务）和金融服务公司推动创建 Agentic 网络助手

- Edwin AI 与其他 AI 智能体（如 Dynatrace Davis AI、Datadog Bits AI）对话
- 需要应用程序深度可见性来确定代码问题
- Catch Point 确定互联网上防火墙外的问题

00:33:30 - 主动性和预测性服务
- MSP、电信公司和 GSP 引领潮流
- 从被动响应转向主动甚至预测性和规范性
- 减少"平均无辜时间"（meantime to innocence）
- 涉及人员、流程、技能、文化和技术
- 可观测性和 AI Ops 是团队运动，需要端到端

00:35:00 - 智能体间通信和 MCP 基础设施
- Andrew 提到 Logic Monitor 正在投资智能体间通信
- 使 Edwin 能够与其他智能体（如 ServiceNow）对话

00:36:30 - AWS 合作伙伴生态系统

00:37:00 - Abby 分享 NTT Data 与 AWS 的合作

- 客户 AI 计划的首要问题是如何构建可扩展、灵活的 AI 基础和基础设施
- 使用 AWS Bedrock 设计灵活且具成本效益的基础，客户无需资本投资

00:38:30 - AI 编排层

- 客户在特定领域、安全性和合规性方面面临挑战
- AWS 生态系统提供客户所需的加速能力

00:45:30 - AI 工作流监控
