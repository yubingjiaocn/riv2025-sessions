# AWS re:Invent 2025 技术会议总结

## 会议概述

本次技术会议由 Octopus Deploy 首席解决方案工程师 James Chatmus 主讲，主题聚焦于银行业面临的核心挑战：如何在保持合规性和通过审计的前提下实现更快的软件部署。会议深入探讨了构建既快速又100%合规的CI/CD管道的实践方法，这是当前银行业追求的真正目标。

演讲者基于与全球大型银行合作的丰富经验，分享了在自动化CI/CD管道方面反复验证的成功模式。会议内容涵盖了供应链攻击风险、零信任部署管道的实现，以及通过GitHub Actions和Octopus Deploy实现治理控制的具体方法。整个演示通过实时demo展现了从构建到部署的完整零信任流程。

## 详细时间线与关键要点

### 0:00-2:00 会议开场与议程介绍
- James Chatmus自我介绍，Octopus Deploy首席解决方案工程师
- 确定会议主题：银行业软件快速部署与合规性平衡
- 18分钟议程规划：供应链攻击风险分析、零信任部署管道演示、治理控制实现

### 2:00-5:00 供应链攻击威胁分析
- 关键统计数据：75%的企业组织在过去12个月内遭受供应链攻击
- 攻击者策略转变：从直接攻击生产环境转向在CI/CD管道中植入恶意代码
- 金融机构面临的风险：恶意代码可能导致八位数罚款和高管声誉危机
- 供应链安全已成为组织优先级列表的首位

### 5:00-8:00 现代管道的现实挑战
- 典型构建流程：单元测试→代码扫描→制品创建→包仓库推送
- "漂移窗口"概念：构建与部署之间的时间间隔（数小时到数周）
- 漂移窗口中的风险：配置漂移、新零日漏洞、恶意制品替换
- 多个漂移窗口问题：开发→QA→预发布→生产环境之间的等待时间

### 8:00-11:00 零信任管道的三大核心组件
- **软件物料清单(S-BOM)**：构建制品的完整第三方库和组件列表
- **来源证明(Provenance)**：制品创建的完整信息（提交、工作流文件、运行器等）
- **证明(Attestation)**：加密签名声明，将S-BOM与包关联并验证来源
- 三个组件的自动生成和在软件交付生命周期各阶段的验证

### 11:00-13:00 职责分离与架构设计
- **GitHub职责**：CI侧所有操作（代码构建、单元测试、扫描、S-BOM生成、制品发布）
- **Octopus Deploy职责**：部署相关的单一真实来源和最终网关
- Octopus核心功能：S-BOM验证、证明验证、策略执行、质量门控制
- 部署完成后的审计跟踪和回滚能力

### 13:00-17:00 实时演示：零信任流程实践
- **GitHub Actions工作流**：
  - 并行执行：仓库扫描与Docker镜像构建
  - S-BOM发布和容器扫描
  - GitHub发布创建
  - 签名和证明生成（关键步骤）
  - 向Octopus Deploy移交部署任务

- **Octopus Deploy部署流程**：
  - 包和S-BOM获取
  - S-BOM验证（成分列表完整性检查）
  - 证明验证（SHA256哈希对比）
  - GitHub漏洞扫描（最终安全检查）

### 17:00-19:00 治理控制与合规特性
- **合规策略应用**：使用Rego定义跨实例策略标准化
- **Kubernetes集成**：EKS部署时的对象状态快照和健康检查
- **构建信息追溯**：从GitHub推送的构建信息提供S-BOM、证明和来源的完整访问
- **成果总结**：
  - 建立从构建到部署的信任链
  - 执行部署治理（策略评估、漏洞检查、证明验证）
  - 验证制品完整性（确保部署制品与构建制品完全一致）
