# AWS re:Invent 2025 技术会议总结：利用图洞察加速Amazon Q业务部署

## 会议概述

本次技术会议由AWS合作伙伴Security AI的Chris Joint主讲，重点探讨了如何通过图洞察技术来加速Amazon Q for Business的安全部署。演讲者强调，虽然AI正在颠覆各个行业的工作方式和价值创造模式，但捕获AI价值的能力仍在追赶AI模型的实际能力。Amazon Q作为一个有效的解决方案，能够轻松地基于企业数据为用户提供价值，同时在设计上确保安全性和隐私性。

演讲的核心问题是数据安全性——这是组织部署AI时面临的首要障碍。Amazon Q虽然尊重身份、角色和权限，确保个性化交互基于用户的实际数据访问权限，但AI的强大搜索能力也带来了双刃剑效应：如果底层数据结构没有适当的权限控制和敏感数据管理，可能会导致意外的数据暴露。Security AI通过构建数据图谱来解决这一挑战，提供文件级别的深度智能和控制。

## 详细时间线和关键要点

### 开场介绍 (0:00-2:30)
- **演讲者介绍**：Chris Joint，来自AWS合作伙伴Security AI
- **主题确立**：如何利用图洞察加速Amazon Q for Business部署
- **AI现状分析**：AI正在颠覆工作方式、价值构建和交付方式

### AI部署挑战分析 (2:30-5:00)
- **核心问题**：数据安全是AI部署的首要障碍
- **Amazon Q优势**：
  - 安全和隐私设计
  - 尊重身份、角色和权限
  - 不使用客户数据改进模型
  - 数据保持私有

### 数据暴露风险 (5:00-8:00)
- **AI搜索能力**：能够深度、细粒度地爬取数据
- **潜在风险**：员工可能查询薪资信息、并购评估公司、社会保险号等敏感数据
- **根本问题**：底层数据结构权限控制不当

### 数据复杂性挑战 (8:00-11:00)
- **数据规模**：
  - 数百亿文件
  - 数亿张表
  - 数千个数据流
  - PB级数据量
- **客户案例**：某大型企业客户每日仅日志就产生超过1PB的非结构化数据
- **复杂因素**：不同文件类型、敏感级别、法规要求、权限结构

### Security AI解决方案 (11:00-15:00)
- **核心方法**：文件级深度扫描，提取数千个元数据片段
- **图谱构建**：展示用户、组、权限、策略、数据之间的连接关系
- **功能提供**：
  - 敏感数据保护
  - 策略执行层
  - 细粒度数据访问智能
  - 过度权限识别
  - 问题修复能力

### 与Amazon Q的协作模式 (15:00-18:00)
- **并行伴侣解决方案**：Security AI作为Amazon Q的补充
- **分工明确**：
  - Amazon Q：索引数据，尊重权限和标签
  - Security AI：分类标记、执行最小权限、减少数据暴露、清理冗余数据

### 数据分类和标记 (18:00-20:00)
- **分类器数量**：超过400个分类器
- **敏感数据类型**：识别不同类型的敏感数据
- **策略引擎**：支持自定义标签（如"仅限高管"）

### ROT数据清理 (20:00-24:00)
- **ROT数据定义**：冗余、过时、琐碎数据
- **影响分析**：
  - 破坏信任和准确性
  - 混淆AI结果
  - 影响员工体验和AI采用
  - 造成安全漏洞
- **ROI价值**：仅数据清理就能为整个项目提供资金支持

### 数据访问控制 (24:00-27:00)
- **治理范围**：从结构化数据扩展到非结构化数据（邮件、文档、聊天）
- **访问控制**：跨数十亿文件的细粒度访问智能
- **可视化展示**：图谱中显示财务数据及其对应的策略和权限映射

### 持续数据验证 (27:00-29:00)
- **策略引擎角色**：Security AI数据指挥中心验证新数据
- **API集成**：通过数据指挥图API咨询是否应索引新数据
- **持续运营**：实现数据AI安全的持续运营化

### Gartner AI TISM框架 (29:00-31:00)
- **基础能力**：信息治理层是Gartner AI TISM堆栈的基础
- **四步流程**：
  1. 检测、分类和标记敏感数据
  2. 减少ROT数据使用
  3. 实施最小权限数据访问
  4. 扩展使用新数据和持续数据清理

### 解决方案价值总结 (31:00-33:00)
- **安全采用**：防止意外数据共享
- **智能风险检测**：自动化修复
- **质量改进**：通过减少ROT数据提高AI响应质量
- **全环境保护**：跨环境保护所有数据和AI

- **展位信息**：1966号展位
