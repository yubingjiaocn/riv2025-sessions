# AWS Amazon QuickSight 技术会议总结

## 会议概述

本次会议由 AWS 全球市场负责人 Neil Collie、全球解决方案架构师 Douglas Burquist 以及 Intuit 工程集团经理 Giraj Pakdi 共同主讲，重点介绍了 Amazon Quick Suite（包含 QuickSight）如何通过 AI 代理能力重新定义商业智能。

会议强调了三大职场转型驱动力：AI 的出现使采用 AI 代理的公司生产力提高 33%，到 2028 年约 15% 的业务决策将由代理 AI 完成；使用 AI 的员工生产力提高 40%；企业需要在规模化的同时统一和保护数据安全。Quick Suite 的愿景是提供一套代理队友，能够快速回答工作中的问题并将答案转化为行动，统一数据源，提供企业级 AI 体验。

会议通过实际演示展示了 Quick Suite 的核心功能，包括聊天代理、研究分析、QuickSight 仪表板可视化、工作流自动化等。Intuit 分享了他们使用 Quick Suite 实现大规模民主化洞察的转型案例，通过系统化的评估、启用、教育和扩展方法，成功将仪表板数量翻倍，用户基数显著增长。

### 重新定义商业智能 (08:00 - 25:00)
- **08:00** - Doug 澄清 QuickSight 与 Quick Suite 的关系：QuickSight 是 BI 服务，Quick Suite 是包含代理能力的整体品牌

- **09:30** - 与客户交流发现：数据孤岛是跨行业问题（金融、医疗、零售、制造）
- **10:15** - Quick Suite 的差异化方法：专注 AI 引擎推理能力，而非简单的 AI 功能叠加
- **11:00** - 架构展示：非结构化源（知识库、文档）+ BI（主题、仪表板）通过 Quick Index 索引
- **12:00** - 统一体验：交互式仪表板、像素完美报告、对话式数据分析
- **13:00** - 自然语言构建可视化：专业分析师更快完成工作，非技术用户也能使用
- **14:00** - 操作连接器：直接从仪表板向 Jira、Salesforce、Slack 发送信息
- **15:00** - 像素完美报告：无服务器架构可扩展至数千用户
- **16:00** - 聊天功能：与所有结构化和非结构化数据交互，使用用户身份验证预过滤
- **17:00** - 高级数据准备：无需 SQL 的分步方法，降低使用门槛
- **18:00** - 嵌入功能：可嵌入交互式仪表板、控制台体验、单个可视化、报告和聊天代理
