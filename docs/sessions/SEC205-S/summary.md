# AWS re:Invent 2025 会议总结：云端韧性与共同责任

## 会议概述

本次AWS re:Invent 2025第一天的分组会议由Cohesity技术营销高级总监Teresa Miller主持，特邀Pearl River Community College的IT总监Kevin Lomax作为客户嘉宾分享实战经验。会议核心主题围绕云端数据韧性和共同责任模型展开，深入探讨了企业在AWS环境中如何应对灾难、网络攻击和意外数据损失等挑战。

Kevin Lomax分享了其所在学院经历勒索软件攻击后的宝贵经验，强调了备份策略的重要性。他们在2022年7月成为Cohesity客户，采用了结合本地数据保护和云端Fort Knox解决方案的混合策略。通过与Cisco XDR集成，实现了异常检测和快照自动化，大幅提升了网络安全响应能力。会议还展示了Cohesity的云保险库(Cloud Vaulting)解决方案，该方案提供数据不可变性、空气隔离和多人审批机制(Quorum)等关键安全特性，确保数据在遭受攻击时仍能快速恢复。

整个会议强调了一个核心理念：即使数据存储在云端，企业仍需对数据保护和恢复承担全部责任。通过采用3-2-1备份规则、定期演练和持续加固安全态势，组织可以将灾难恢复时间从数月缩短至数小时，显著提升业务连续性。

## 详细时间线与关键要点

00:00:00 - 会议开场与议程介绍
- Teresa Miller介绍会议主题：云端韧性与共同责任

- 会议分为四个部分：挑战分析、客户经验分享、韧性提升方案、云保险库演示

00:03:30 - 行业挑战分析
- 原生AWS备份工具的局限性：功能有限，仅能满足基本需求
- 安全考量：数据主权问题，可能需要跨云或跨地域备份以满足合规要求(如GDPR、SOX)
- 共同责任模型核心：即使数据在云端，企业仍需负责凭证安全、数据保护和恢复

00:06:00 - 网络安全威胁态势
- Kevin分享密西西比州网络安全趋势：攻击者越来越聪明，利用社交媒体、钓鱼邮件、二维码等手段
- 终端用户是最大的安全漏洞
- 攻击者会等待企业修补漏洞后再次发起攻击

00:08:45 - Pearl River遭遇勒索软件攻击经历
- 攻击者入侵并删除了备份系统
- 幸运的是有一台服务器因做了快照并断网而保留了数据，但损失了三天的数据
- 这次事件促使学院寻找更可靠的备份解决方案

00:10:30 - 选择Cohesity和AWS的原因
- 2022年7月成为Cohesity客户
- 进行了多家供应商的概念验证，第一天5小时内就确定选择Cohesity
- 关键特性：数据不可变性、空气隔离、多人审批机制(Quorum)
- 选择AWS因其全球数据中心分布和行业领导地位

00:13:00 - 网络韧性挑战
- IT部门规模小，需要覆盖网络、无线、软件、学生信息系统等所有领域
- 终端用户是最大的安全漏洞
- 攻击者不分昼夜，善于利用节假日发起攻击

00:15:20 - 备份清洁度验证问题
- Kevin坦承：攻击发生时无法确认备份是否干净
- 攻击者已经入侵并清除了备份系统
- 现在采用本地DataProtect + 云端Fort Knox双重保护策略

00:17:00 - 提升网络韧性的措施
- 集成Cisco XDR与Cohesity平台
- XDR自动检测异常、数据变化和可疑模式，立即拍摄快照并通知团队
- 便于取证团队快速定位问题服务器和触发事件
- 持续的终端用户培训，警惕钓鱼邮件

00:20:15 - 灾难恢复测试
- 每季度进行一次完整恢复测试
- 17TB数据完全恢复仅需17小时(平均每TB 1小时)
- 相比行业内勒索软件攻击后3-9个月的恢复时间，这是巨大优势
- 强调桌面演练和团队协作的重要性

00:23:00 - 3-2-1备份规则实践
- 3份数据副本
- 2种不同介质类型(本地和云端)
- 1份离线副本(空气隔离)
- 加上不可变性(Fort Knox云保险库)
- Pearl River的灾难恢复站点距离主站点仅60英里，龙卷风可能同时摧毁两地，因此云备份至关重要

00:25:30 - 实施成果
- 建立了流畅的灾难恢复计划
- 团队响应速度大幅提升
- 使用第三方SIM/SOC解决方案弥补小型IT团队人力不足
- 集成多种第三方应用和AI功能增强整体防护能力

00:27:45 - 未来12个月的网络韧性优先事项
- 持续加固备份系统
- 定期测试和终端用户培训
- 模拟钓鱼攻击演练
- 将温备站点升级为热备站点(受预算限制)
- 平衡成本与安全需求

- 强调持续加固和权限管理的重要性

00:37:00 - 云保险库功能演示
- Quorum多人审批机制：关键操作需要多人批准
- 数据锁定(WORM)：防止数据被篡改
- 基于角色的访问控制(RBAC)
- 全球多区域数据主权支持

00:38:30 - 演示：云保险库仪表板
- 展示全球云保险库分布
- 配置Quorum审批流程：选择需要审批的操作类型、设置审批人(例如3人中需2人批准)、设置超时时间(4小时)
- 集中查看所有审批请求和状态

00:40:00 - 演示：数据保护策略配置
- 创建和管理保护策略
- 添加云保险库目标(例如选择俄勒冈西部区域)
- 支持同一数据的多个保险库副本，可跨全球不同区域甚至不同云供应商

00:42:00 - 演示：灾难恢复流程

- 从云保险库副本恢复
- 恢复到隔离环境进行取证分析
- Quorum审批流程：自动批准第一个审批人，手动批准第二个审批人
- 恢复任务启动
