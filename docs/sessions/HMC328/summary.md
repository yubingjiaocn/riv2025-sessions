# AWS re:Invent 2025 会议总结：构建弹性低延迟混合电信基础设施

## 会议概述

本次会议由Liberty Latin America波多黎各公司的IT运营总监Jose Flores和云治理与工程经理Juan主讲，重点介绍了他们如何在波多黎各大规模构建具有弹性和低延迟的混合电信基础设施。Liberty Latin America是拉丁美洲和加勒比地区的主要电信运营商之一，在20多个国家拥有业务，拥有超过50,000公里的海底光纤网络连接40多个市场。

2019年，Liberty在波多黎各收购了AT&T的移动网络业务，面临着巨大的技术挑战。当时他们只是一家固定网络运营商，需要在有限的时间内将200多万移动用户从AT&T的IT系统迁移到自己的平台。原有的BSS系统是为固定业务设计的，无法扩展支持移动业务，且高度定制化导致产品上市时间缓慢。此外，波多黎各经常遭受飓风等自然灾害，这要求基础设施必须具备高可用性和弹性。

为解决这些挑战，Liberty采用了AWS混合云解决方案，核心是在波多黎各部署AWS Outposts作为主计费系统，在迈阿密本地区域部署灾备系统，并通过自有的海底光纤电缆建立低延迟连接。整个解决方案采用云原生架构，包括Salesforce作为CRM和产品目录、Aria作为计费系统、容器化的OCS计费平台，实现了10毫秒的网络延迟和无缝故障转移能力。

## 详细时间线

00:00 - 开场介绍
- Jose Flores介绍自己是Liberty波多黎各IT运营总监
- 会议主题：构建弹性低延迟混合电信基础设施

00:30 - 会议议程
- 介绍Liberty Latin America公司背景
- 收购移动网络后面临的挑战
- 解决方案架构
- 经验教训与未来规划

01:00 - Liberty Latin America公司介绍
- 拉丁美洲和加勒比地区主要电信运营商
- 在20多个国家和30多个B2B市场运营
- 拥有超过50,000公里海底光纤网络连接40多个市场
- 年收入44亿美元，近10,000名员工
- 在不同国家使用不同品牌运营（Liberty、Flow、BTC、Mas Movil等）

02:30 - 波多黎各业务背景
- 2019年收购AT&T在波多黎各的移动业务
- 此前仅为固定网络运营商（有线电视和固定服务）
- 通过TSA协议使用AT&T的BSS系统运营

03:30 - 面临的业务挑战
- TSA协议成本高昂且有明确截止日期
- 需要快速迁移200多万用户
- 现有BSS系统仅支持固定业务，无法扩展
- 系统为本地部署，非云原生
- 高度定制化导致产品上市时间缓慢
- 无法与AT&T、Claro等竞争对手竞争

04:30 - 技术挑战
- 需要新的计费系统支持移动业务
- 要求50毫秒以内的延迟以连接所有呼叫
- 计费系统需实时授权数据、语音和短信交易
- 波多黎各频繁遭受飓风等自然灾害（如飓风Maria导致数月断电）
- 需要高弹性和高可用性系统

05:30 - 解决方案概述
- 使用AWS Outposts在波多黎各部署主计费系统
- 在迈阿密部署灾备系统
- 通过本地区域实现高可用性和低延迟

06:00 - 完整架构介绍
- Salesforce作为产品目录和配置中心
- Everett作为支付网关
- Salesforce CRM用于客户服务和订单配置
- Aria作为云计费平台，处理月度账单和发票
- Matrix OCS部署在AWS Outposts上
- OCS实时处理网络交互事件
- 使用Apache Kafka推送交易到Aria
- CDR数据发送到数据湖，使用Athena查询

08:00 - Juan接手讲解基础设施
- 强调混合解决方案的基础是海底光纤网络
- Liberty Networks运营低延迟冗余光纤网络
- 连接加勒比地区与美洲的关键海底电缆

09:00 - 网络连接挑战
- 核心挑战：将波多黎各移动业务与迈阿密AWS本地区域连接
- 需要最高性能
- 将AWS设施视为本地网络的延伸
- 使用Arcos和Americas-2海底电缆的光纤容量
- 创建完全冗余解决方案
- 保证最短路径和最低抖动

10:00 - 混合架构详解：本地部署
- 波多黎各多个数据中心
- 每个位置部署移动网络核心组件
- 包括DRA（直径路由代理）和STP（信令传输点）
- 核心组件需要与计费系统通信

11:00 - AWS Outposts部署
- 在波多黎各部署Matrix计费系统的主实例
- 容器化解决方案，使用EKS
- 使用网络服务和存储服务
- 集成F5管理信令流量
- 使用MongoDB作为数据库层
- 通过AWS本地网关连接移动核心和计费系统
- 实现约10毫秒网络延迟

12:30 - 灾备解决方案
- 在迈阿密本地区域部署被动DR实例
- 使用相同的AWS服务和合作伙伴生态系统
- 主备实例之间建立复制

13:00 - 网络连接层次
- 底层：自有海底电缆
- 传输网络和MPLS
- Direct Connect：两条10Gbps连接提供带宽和完全冗余

14:00 - 区域层
- 使用多个SaaS解决方案（Aria、Salesforce）
- 支付网关、BSS组件、CRM
- 区域内运行内部工作负载、OSS层、中介系统
- 其他IT应用

14:30 - Transit Gateway
- 作为管理所有路由复杂性的"大脑"
- 集中流量检查
- 使用AWS Network Firewall

15:00 - 架构经验总结
- 基础设施使其成为云原生电信运营商
- 创建可复制的模板架构
- 可在不同市场、新收购业务中复制
- 混合服务解决了低延迟、数据驻留、弹性约束
- 实现应用的无缝故障转移和故障恢复

16:00 - 关键经验：接近性、敏捷性优于定制化
- 计费系统靠近网络（EPC和5G核心）至关重要
- 为所有解决方案选择云原生供应商（不仅OCS，还包括BSS、CRM、支付网关）
- 遵循敏捷实践，产品上市时间从数月缩短到数周
- 避免定制化，从遗留系统中吸取教训
- 定制化会拖慢业务速度

17:00 - 会议结束
