# AWS re:Invent 2025 会议总结：Visa 基于 AWS 构建 AI 驱动的反欺诈防护系统

## 会议概述

本次技术分享会议主题为"Secure in Milliseconds"，重点介绍了 Visa 如何在 AWS 上构建 AI 驱动的反欺诈防护系统，专门针对账户到账户（A2A）支付进行安全防护。会议由 AWS 客户解决方案经理 Suraj Sajo 主持，Visa 工程高级总监 Hema Zutshi 和 Visa Protect A2A 产品首席架构师 Aravinda Nagalla 作为主要演讲者。

会议深入探讨了随着数字支付技术发展，A2A 支付方式快速增长所带来的欺诈风险挑战。Juniper Research 预测到 2029 年 A2A 交易量将增长 83%，达到 1 万亿笔年交易量，相应的欺诈损失预计将增长 153%，达到 580 亿美元。为应对这一挑战，Visa 开发了 Visa Protect Account to Account 解决方案，利用 AWS 基础设施实现毫秒级的实时欺诈检测和防护。

## 详细时间线与关键要点

### 0:00-5:00 会议开场与背景介绍
- Suraj Sajo 介绍会议主题和演讲嘉宾
- 概述支付行业发展趋势：从传统信用卡支付到数字钱包，再到 A2A 支付
- A2A 支付定义：资金直接从发送方银行账户转移到接收方银行账户，无需信用卡网络等中介

### 5:00-10:00 A2A 支付市场增长与风险分析
- A2A 支付两种类型：推送支付（发送方发起）和拉取支付（接收方发起）
- A2A 支付流程：消费者/企业请求 → 发送金融机构 → 实时支付网络（RTP）→ 接收金融机构
- 市场增长预测：2029 年交易量增长 83%，2030 年交易价值增长 113% 达到 195 万亿美元
- 欺诈风险：银行业欺诈损失预计增长 153%，达到 580 亿美元

### 10:00-15:00 Visa Protect A2A 解决方案介绍
- Hema Zutshi 介绍 Visa 的信任原则：弹性、高可用性、网络安全、低延迟
- Visa Protect A2A 专门针对非卡支付的欺诈防护解决方案
- 支持 B2B、B2C、C2C、C2B 等各种支付场景
- 在支付流程中的两个拦截点：发送方金融机构和接收方金融机构

### 15:00-20:00 技术要求与挑战
- 严格的网络安全要求：保护内存中数据免受威胁，采用零信任架构和 AWS Nitro Enclaves
- 数据本地化要求：符合 GDPR 要求，在欧洲地区部署
- 超低延迟要求：端到端响应时间必须在 250 毫秒以内
- 高可用性要求：99.99% 正常运行时间（年停机时间不超过 52 分 35 秒）

### 20:00-25:00 架构解决方案
- 使用 Amazon Memory DB with Valkey 实现快速读取和数据持久性
- TLS 连接池机制优化延迟
- 双区域冗余部署：伦敦和爱尔兰地区，距离超过 60 英里
- 多可用区设置：每个地区使用三个可用区

### 25:00-30:00 实时评分 API 技术架构
- Aravinda Nagalla 详细介绍技术架构
- DMZ VPC 安全边界，Amazon Network Firewall 检查所有数据包
- 使用 Transit Gateway、VPC Endpoints、API Gateway 的流量路径
- Nitro Enclaves 中的 A2A 网关应用程序处理敏感数据
- Amazon EKS 上的决策服务和模型推理平台

### 30:00-35:00 AWS Nitro Enclaves 深度解析
- Nitro Enclaves 特性：无外部连接、无 SSH 访问、无持久存储
- 启动时获取加密密钥并缓存在内存中
- 使用信封加密和 KMS 客户管理密钥
- 性能优化：单个 M5A 2xlarge 实例可达到 150-200 TPS

### 35:00-40:00 性能优化策略
- 使用 TLS 1.3 和 HTTP 2.0 提高握手速度
- VPC 端点和 VPC 链接保持流量在 Amazon 骨干网络内
- EKS 优化：负载均衡器控制器、Pod 调度策略
- 拓扑感知路由保持流量在同一可用区内

### 40:00-45:00 多区域架构与总结
- Active-Active 多区域架构满足 Tier 0 应用要求
- 使用 CloudFlare/Akamai DNS 路由到最近区域
- MSK Replicator、S3 跨区域复制、Aurora Global DB 实现数据复制
- 成功案例：英国地区已投入生产使用，计划扩展到巴西圣保罗地区
- 实现目标：P99.5 延迟比 250ms 要求低 40%，支持高达 1000 TPS 的突发流量
