# AWS re:Invent 2025 会议总结：Visa 基于 AWS 构建 AI 驱动的反欺诈防护系统

## 会议概述

本次技术分享会议主题为"Secure in Milliseconds"，重点介绍了 Visa 如何在 AWS 上构建 AI 驱动的反欺诈防护系统，专门针对账户到账户（A2A）支付进行安全防护。会议由 AWS 客户解决方案经理 Suraj Sajo 主持，Visa 工程高级总监 Hema Zutshi 和 Visa Protect A2A 产品首席架构师 Aravinda Nagalla 作为主要演讲者。

会议深入探讨了随着数字支付技术发展，A2A 支付方式快速增长所带来的欺诈风险挑战。Juniper Research 预测到 2029 年 A2A 交易量将增长 83%，达到 1 万亿笔年交易量，相应的欺诈损失预计将增长 153%，达到 580 亿美元。为应对这一挑战，Visa 开发了 Visa Protect Account to Account 解决方案，利用 AWS 基础设施实现毫秒级的实时欺诈检测和防护。

## 详细时间线与关键要点

### 0:00-5:00 会议开场与背景介绍
- Suraj Sajo 介绍会议主题和演讲嘉宾
- 概述议程：支付格局发展、A2A 支付增长趋势、风险挑战
- 强调 Visa 与 AWS 多年合作伙伴关系的重要性

### 5:00-15:00 A2A 支付市场分析与挑战
- 支付方式演进：从信用卡刷卡到数字钱包，再到 A2A 支付
- A2A 支付定义：资金直接从发送方银行账户转移到接收方银行账户
- 两种 A2A 支付类型：推送支付（发送方发起）和拉取支付（接收方发起）
- A2A 支付流程：消费者/企业请求 → 发送金融机构 → 实时支付网络 → 接收金融机构
- 市场增长数据：预计 2030 年交易价值增长 113%，达到 195 万亿美元
- 欺诈风险：银行业欺诈损失预计增长 153%，达到 580 亿美元

### 15:00-25:00 Visa Protect A2A 解决方案介绍
- Hema Zutshi 介绍 Visa 的信任原则：弹性、高可用性、网络安全、低延迟
- Visa Protect Account to Account 专门针对非卡支付的欺诈防护
- 解决方案架构：在发送和接收金融机构两个节点进行欺诈检测
- 支持全球各地区合规要求，特别是英国市场的双重责任保护

### 25:00-35:00 技术要求与解决方案
- 四大核心技术要求：
  - 严格的网络安全准则：零信任架构，使用 AWS Nitro Enclaves 防护内存数据暴露
  - 数据本地化：符合 GDPR 要求，在欧洲地区部署
  - 低延迟：端到端响应时间必须在 250 毫秒以内
  - 高可用性：99.99% 正常运行时间（年度停机时间不超过 52 分 35 秒）
- 解决方案：双区域部署（伦敦和爱尔兰），多可用区设置，使用 Amazon Memory DB with Valkey

### 35:00-45:00 技术架构深度解析
- Aravinda Nagalla 详细介绍技术架构
- 实时评分 API 工作流：DMZ VPC → 网络防火墙 → API Gateway → Nitro Enclaves → EKS 决策服务 → Ray 集群模型推理
- AWS Nitro Enclaves 最佳实践：资源分配、代理调优、健康检查端点
- 性能优化：TLS 1.3、HTTP 2.0、VPC 端点、连接池机制
- EKS 优化：负载均衡器控制器、Pod 调度策略、拓扑感知路由
- 多区域主-主架构：CloudFlare/Akamai DNS 路由、跨区域数据复制
- 性能指标：单个 M5A 2xlarge 实例达到 150-200 TPS，P99.5 延迟比目标低 40%，成功测试 1000 TPS 突发流量

### 45:00-45:10 总结与展望
