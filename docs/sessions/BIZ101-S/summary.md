# AWS re:Invent 2025 会议总结：通过自主企业运营改变业务未来

## 会议概述

本次会议重点介绍了Digitate公司的Igno平台如何通过自主企业运营来解决业务中的技术问题。演讲者Rajie详细阐述了当今企业面临的核心挑战：大多数业务问题的根源实际上是技术问题。无论是零售订单未能按时交付、医疗假肢未能按承诺日期送达手术室，还是快消品公司无法及时补货，这些看似业务层面的失败都源于底层IT系统的性能问题。

Igno平台通过三大核心能力解决这些问题：统一可观测性（从横向业务流程和纵向技术栈两个维度监控）、AI驱动的洞察（预测性分析，在问题发生前提供预警）、以及闭环自动化（以机器速度隔离和修复问题）。该平台采用代理架构（Agentic Architecture），包含五个主要代理，每个代理由感知、推理、行动、学习、控制和LLM模型等六个子代理组成。会议还宣布了Digitate与AWS的深化合作，特别是与BMC ControlM在AWS Marketplace上的首次多产品SaaS发布。

- 部署统一可观测性监控每个数据流转节点

- 能够向医院确认假肢交付日期，确保手术按时进行

5:45-7:30 - 快消品直接店铺配送案例

- 监控从订单采集到制造、配送、补货的全流程
- 每周处理数亿美元订单，确保自主运营

7:30-9:00 - 自主运营的三大目标
1. 消除因IT导致的业务中断
2. 在业务部门发现问题前主动识别并修复
3. 如果问题已发生，尽快恢复，让问题看起来只是短暂波动

9:00-11:30 - Igno平台三大核心能力
1. 统一可观测性 - 横向监控跨应用数据处理流程，纵向监控技术栈，主动发现和避免问题
2. AI驱动洞察 - 分析遥测和日志数据，预测未来故障（例如：预测6周后应用性能下降，或凌晨3点预警6点的报告将延迟）
3. 闭环自动化 - 理解企业IT架构全貌，以机器速度隔离问题，拥有10,000个自动化操作，将修复时间从数小时缩短至几分钟
