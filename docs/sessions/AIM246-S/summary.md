# AWS re:Invent 2025 会议总结：企业级AI与数据分析平台

## 会议概述

本次技术分享会由Qlik公司AI与分析业务总经理Brendan Grady主持，并邀请了来自英格索兰公司(Ingersoll Rand)的客户代表Juan分享实际应用案例。会议主要探讨了在AI快速发展的时代背景下，企业如何利用现有的数据分析基础设施来构建企业级AI解决方案，避免在技术变革中被淘汰。

演讲者强调了"不知道的成本"概念，指出企业决策如果不基于全面的数据分析和正确的技术手段，将面临巨大风险。会议重点介绍了Qlik平台如何帮助企业将传统的商业智能和分析投资转化为AI驱动的智能决策系统，特别是通过数据平台AI和代理框架来实现规模化的AI应用。

## 详细时间线与关键要点

### 0:00-5:00 开场与背景介绍
- Brendan Grady介绍自己作为Qlik AI与分析业务总经理的身份
- 强调"犹豫就会成为历史"的紧迫性，企业必须拥抱AI变革
- 现场调研显示技术人员与业务人员各占50%
- 提出"不知道的成本"概念，建议观众搜索"AI失败案例"和"社交媒体失败案例"

### 5:00-10:00 Qlik平台核心能力介绍
- 强调现代企业面临数据过多而非不足的挑战
- 介绍Qlik三大核心能力：
  - 可信数据产品：无论数据位置，都能提供给代理或人员使用
  - 数据模式发现：找到纯可视化工具无法发现的数据模式
  - 数据行动化：基于可信数据和模式分析采取行动
- 分析传统BI世界正在被颠覆，客户期望分析成为代理智能层

### 10:00-15:00 行业挑战与解决方案
- 识别三大持续挑战：可信数据、成本控制、适应性和可靠性
- 强调数据质量倡议在2025年重新变得重要和令人兴奋
- 介绍英格索兰作为成功客户案例的预告
- 承诺展示如何将现有分析基础设施投资延续到新AI世界

### 15:00-20:00 英格索兰案例分享开始
- Juan介绍英格索兰作为制造业集团的复杂性：90多个品牌，50多个ERP系统
- 描述制造业"数据丰富但洞察贫乏"的典型困境
- 提出AI问题的双重解决策略：
  - 工作场所AI：面向15,000知识工作者的个人使用AI
  - 企业定制AI：复杂数据管道和变革性AI项目
- 识别中间层需求：用户要求更复杂的实时数据问题解答

### 20:00-25:00 数据平台AI解决方案
- 介绍"数据平台AI"概念，利用现有分析资产作为"金发姑娘区域"
- 展示参考架构：数据源→提取层→奖章架构→AI工厂→MCP→前端应用
- 强调MCP(模型控制协议)的重要性，实现数据修改的单点更新
- 分享具体成果：客户经理节省20,000小时，亚太管理团队简化多系统操作

### 25:00-30:00 战术建议与平台能力
- Juan提出两个"非常规技巧"：
  - 分析平台是最接近AI就绪的数据，应充分利用现有投资
  - 分析超级用户成为即时AI军队，赋能数据公民社区
- Brendan总结Qlik的三大贡献：可信数据产品、分析引擎、代理框架
- 介绍Qlik的代理组合：数据任务代理、知识代理、与其他系统的代理间通信

### 30:00-33:00 总结与关键要点
- 强调只有5%的AI项目成功，Qlik目标是提高成功率
- 提出三个关键成功要素：
  - 上下文决策：基于可信数据基础的全数据决策
  - 治理：持续反馈机制和AI军队参与数据治理
  - 嵌入式分析：分析不再是目的地，而是嵌入业务流程
- 预告问答环节，鼓励观众关注英格索兰的实际业务成果
