# AWS re:Invent 2025 会议总结：使用 Rocket 和 GenAI 现代化奔驰全球订购系统

## 会议概述

本次技术分享会由 Capgemini 的 Manuel Breitfield 和奔驰 IT 部门的 Christian Klemme 共同主讲，详细介绍了奔驰如何将其关键业务系统"全球订购系统"(Global Ordering, GO)从大型机迁移到 AWS 云平台的项目。该系统是奔驰销售运营的核心，每一辆销售和生产的汽车都要经过这个系统处理。

项目采用了"Go to Cloud"的策略，结合 Rocket Software 的企业服务器解决方案和生成式 AI 技术，实现了大规模遗留系统的现代化改造。特别值得关注的是，团队成功使用 GenAI 将 130 万行 COBOL 代码转换为 Java 应用程序，展现了 AI 在企业级系统迁移中的巨大潜力。

## 详细时间线与关键要点

### 0:00-5:00 项目背景与系统规模介绍
- 全球订购系统是奔驰销售运营的"生命线"，支持从产品组合设置到生产订单的完整流程
- 系统规模庞大：超过 20,000 个接口，500 万行 Java 和 COBOL 代码
- 占大型机工作负载的 50%，支持 800 多个队列，每年处理 51 亿条消息
- 服务于 150 个国家的 8,000 多名用户，运行 450 个批处理作业

### 5:00-10:00 迁移策略与团队架构
- 项目始于两年前，采用 Rocket Software 的快速评估服务
- 选择重新平台化(re-platforming)而非重构，以降低风险
- Capgemini 作为总承包商，协调 AWS、Rocket Software 等合作伙伴
- 分阶段实施策略：从无状态服务开始，逐步迁移完整应用

### 10:00-15:00 技术架构与平台设计
- 使用 Global Ordering Facade 作为关键组件，基于 MuleSoft 网关产品
- Facade 允许将工作负载路由到云后端或大型机后端，支持平滑测试和快速回滚
- Helios 平台整合了 Rocket Enterprise Server、Java 栈、批处理服务等组件
- AWS 完全托管的着陆区，包含监控、财务管理等完整服务

### 15:00-20:00 GenAI 应用与遗留系统现代化
- 介绍传统的四阶段现代化方法：评估、迁移策略、迁移实施、运营优化
- 使用 CAP360、BREAD 等 AI 工具进行应用结构分析
- 实施多智能体 GenAI 系统，包含软件工程师、审查员、测试员等不同角色
- 结合人工专业知识与 AI 能力，通过"cookbook"训练 AI 进行代码转换

### 20:00-25:00 定价服务案例研究
- 成功将 130 万行 COBOL 代码的定价服务转换为 Java 应用
- 项目时间线：2月开始尝试，3月首次提交代码，5月完成可部署版本
- 使用 Precisely 工具实现实时数据同步，保持与大型机数据一致性
- 通过手动性能调优优化数据库访问，适应云环境特点

### 25:00-30:00 测试策略与生产部署
- 利用集成层进行并行测试，自动比较新旧系统结果
- 实现自动化质量保证，大幅减少手动测试工作量
- 9月成功上线，系统性能优于原大型机版本
- 零事故上线，展示了 AI 辅助迁移的可靠性和有效性
