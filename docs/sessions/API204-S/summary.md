# AWS re:Invent 2025 会议总结：AI 代理的编排力量

## 会议概述

本次会议由来自 Workato 的 Rahul Dja 主讲，主题聚焦于如何将 AI 代理从概念承诺转化为实际业务影响。演讲者指出，尽管 AI 技术本身非常强大，但目前大多数 AI 代理仅被用于前端任务（如撰写邮件、查找信息），而无法真正作用于企业核心应用系统（如 Salesforce、SAP、Zendesk）。这些企业应用才是真正的数据宝库，而编排（Orchestration）正是连接 AI 代理与企业应用的关键"沉默力量"。

演讲深入探讨了编排层如何赋能 AI 代理，使其能够安全、可信地与企业后端系统交互并执行实际操作。Workato 提出了其解决方案 Workato One，该平台将编排能力与代理平台相结合，通过企业级 MCP（Model Context Protocol）使 AI 代理能够利用现有的集成和编排，无需重建基础设施。演讲强调了统一的企业级 MCP 相比供应商特定 MCP 服务器的优势，包括避免工具泛滥、统一安全治理模型，以及加速价值实现。

## 详细时间线

0:00 - 0:45 - 开场介绍
- 演讲者 Rahul Dja 来自 Workato
- 会议主题：如何让 AI 代理从承诺走向实际影响

0:45 - 1:30 - AI 的本质与现状
- AI 本质上是编排与代理的分层结合
- 代理擅长前端任务，编排赋予其操作企业应用的超能力
- 可执行创建订单、添加员工等企业级操作

1:30 - 2:30 - AI 应用的现状分析
- AI 无处不在（Bedrock、Gemini 等云服务）
- 主要用于前端活动：撰写邮件、查找信息
- 缺失环节：无法对核心企业应用（Salesforce、SAP、Zendesk）执行操作
- 主要数据存储在企业应用中，但 AI 无法访问

2:30 - 3:30 - 编排的重要性
- 编排是缺失的关键要素
- 企业客户的四大核心需求：
  - 对数据的实时治理访问
  - 流程编排
  - 角色和权限定义
  - 持续生命周期管理
- AI + 编排 = 真正的行动能力

3:30 - 5:00 - 编排对代理的价值
- **连接应用**：实现应用间通信，代理需要知道连接哪个应用、从哪里提取/写入数据
- **实施防护栏**：安全和信任是企业首要考虑，需要支持各种身份验证方式
- **启用协作**：AI 代理作为员工助手，帮助销售人员激活休眠交易，帮助客服快速解决问题，支持人机切换
- **留下痕迹**：记录代理与应用交互的审计轨迹

5:00 - 6:30 - 业务流程视角
- 企业拥有多个业务流程（订单到现金、采购到付款等）
- 底层是数据系统（SAP、Salesforce 等 ERP）
- 连接层：与应用交互的连接架构
- 代理层：已有的 AI 代理
- **缺失的编排层**：连接代理与底层应用的关键
- 编排层实现深度操作，代理可模拟人类执行任务

6:30 - 7:30 - 重用现有集成
- 许多客户已构建大量编排和集成
- 无需重建：AI 代理可通过 MCP 与现有集成交互
- MCP 使代理更智能，无需额外工作
- 现有集成已使用正确的安全协议和账户

7:30 - 9:00 - Workato 的解决方案
- Workato One 平台介绍
- 三大核心要素：
  - **知识访问**：访问公司文档
  - **编排**：代表用户执行操作
  - **信任**：治理和安全层
- 架构：编排平台（iPaaS Plus）+ 代理平台
- 支持构建自有代理或使用 Workato 企业级 MCP
- 无需使用 Workato Orchestrate 即可使用 Workato Agentic

9:00 - 10:30 - 企业级 MCP 的价值
- 企业级 MCP 连接代理与应用
- 确定性流程可通过 AI 代理以非确定性方式执行
- 利用现有编排层和应用连接
- 将前端任务代理转变为可操作企业应用的代理

10:30 - 11:30 - Workato 企业级 MCP 特性
- 利用 Workato 12,000+ 连接器
- 使用现有治理模型确保安全交互
- 支持用户模拟，代表用户执行操作
- 示例：总结销售管道 - 连接 Salesforce、调用记录，呈现统一视图

11:30 - 13:00 - MCP 的应用场景
- 将现有业务流程通过 MCP 暴露给 AI 代理
- 大规模提升采用率
- 传统方式：需要理解 API 和服务，以确定性方式编排
- MCP 方式：AI 代理自动发现 API 和应用并执行操作

13:00 - 15:00 - 企业级 MCP vs 供应商特定 MCP
- 问题：为何选择企业级 MCP 而非供应商特定 MCP 服务器？
- **工具泛滥问题**：类似 20 年前的集成工具泛滥（SAP、Oracle 各有工具）
  - 每个工具需要独立技能
  - 独立治理模型
  - 独立团队管理
- **MCP 泛滥**：每个供应商有自己的 MCP 工具和安全模型
- **企业级 MCP 优势**：
  - 通过单一统一层与所有应用交互
  - 加速价值实现
  - 遵循统一安全协议
  - 无需依赖供应商特定安全协议
- **核心观点**：没有企业级 MCP，就是将控制权外包给技术栈中的每个供应商

15:00 - 16:30 - 编排层的企业级能力
- 编排层需要强大、可适应
- 构建安全、受治理、可扩展的集成
- Workato 企业级能力：
  - 完全托管服务
  - 始终在线模式部署
  - 零停机升级
  - 日志和审计流
  - 多种监听方式（Salesforce 对象变更、S3 新文件等）
- AI 代理是未来，但编排仍是核心
- 需要与能够提供快速、安全、企业级编排的平台合作
