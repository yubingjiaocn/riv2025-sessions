# MAM-217 AWS Transform 可组合AI代理技术深度解析

## 会议概述

本次技术深度解析会议由AWS Transform团队产品管理负责人Murtaza Chowdhury主持，联合同事Alexis和Ravi，以及合作伙伴代表共同展示了如何使用AWS Transform构建可组合AI代理解决方案。会议重点介绍了AWS Transform从分散的点解决方案发展为统一平台的历程，以及如何通过合作伙伴集成实现更强大的迁移和现代化能力。

AWS Transform于2024年5月正式发布，目前提供四大专业化代理：VMware迁移代理、大型机现代化代理、Microsoft工作负载代理和自定义代理。平台采用工作空间协作模式，支持基于角色的访问控制，并通过持续学习循环不断优化代理性能。会议展示了多个实际应用案例，包括网络文件转换、DORA法规合规性检查以及BI迁移等场景。

## 详细时间线与关键要点

### 0:00-10:00 AWS Transform平台概述
- **服务现状**：2024年5月正式发布，包含四大专业化代理
- **VMware迁移代理**：映射本地数据中心工作负载到云端，识别应用程序分组和网络拓扑
- **大型机现代化**：提取业务规则，生成文档，支持现代语言迁移
- **Microsoft工作负载**：.NET Framework到.NET Core转换，本周扩展至SQL Server现代化
- **自定义代理**：支持Java、Node.js、Python升级，允许用户构建自定义转换

### 10:00-20:00 用户界面和工作流程
- **工作空间概念**：作为安全边界和协作空间，支持管理员、贡献者、只读用户角色
- **作业类型**：基于四大能力创建不同作业类型
- **作业执行流程**：评估需求→创建作业计划→执行任务代理
- **聊天体验**：全程可通过聊天引导，支持人工监督和下一步推荐
- **工件管理**：统一存储迁移现代化数字足迹
- **连接器管理**：支持源端和目标端连接器配置

### 20:00-30:00 可组合性架构设计
- **发展历程**：从AWS内部数十个专用工具整合为统一体验
- **合作伙伴集成**：解决工具重复和摩擦问题，提供统一治理层
- **共同创新模式**：合作伙伴可集成工具、代理和知识库
- **个性化工作流**：基于专业领域经验定制工作流程
- **客户体验**：即插即用，无需访问多个平台

### 30:00-40:00 技术组件深度解析
- **基础代理模块**：提供AWS Transform原语集成能力
- **安全特性**：企业身份提供商集成、操作授权、凭证生命周期管理
- **协作功能**：工作空间角色委派、审批流程、安全工件存储
- **工作流编排**：支持单代理和多代理编排，人机交互和代理间交互
- **部署能力**：Bedrock AgentCore运行时集成，按需扩展，自主模式支持

### 40:00-50:00 实际应用演示
- **网络文件转换代理**：展示本地开发环境到Web应用的集成过程
- **DORA法规合规检查**：结合BRE代理和Kiro实现自动化合规性检测
- **代理通信机制**：展示编排器和子代理间的消息传递
- **知识库集成**：DORA法案YAML文件和Kiro指导文件的自动化处理

### 50:00-61:00 合作伙伴案例展示
- **Wavicle Data Solutions**：BI迁移到QuickSight解决方案
  - 消除采购延迟，增强安全态势
  - 支持快速POC验证，自助服务ROI分析
  - Power BI工作空间分析和报告生成
- **Accenture**：大型机现代化知识库增强
- **Capgemini**：简化大型机现代化流程
- **合作伙伴接入**：通过Partner Central或合作伙伴开发经理联系
