# AWS re:Invent 2025 - Amazon S3 新功能发布会总结

## 会议概述

本次会议由 Paul Megan 和 Anna Boucher 主持，全面介绍了 Amazon S3 团队在过去 12 个月中推出的 35 项新功能。会议涵盖了安全性、耐久性、S3 Express 高性能存储、S3 对象和 API、元数据表以及向量存储等六大主题。

会议的一大亮点是进行了一个复杂的现场演示，展示了如何使用 Amazon Transcribe 将视频转换为文本，上传 700 多个客户案例文件到 S3，并使用批量操作和基于标签的访问控制等新功能进行管理。演示过程中还展示了增强的错误消息功能，帮助快速诊断访问被拒绝的问题。

S3 Tables 作为去年推出的重要服务，在今年得到了大幅改进，目前已有超过 40 万张表被创建。新增的表复制功能和智能分层支持使其成为管理 Apache Iceberg 表的完整解决方案，可为客户节省高达 80% 的存储成本。

## 详细时间线与关键要点

### 00:00:00 - 会议开场与安全功能介绍
- Paul Megan 和 Anna Boucher 介绍会议议程，计划在 60 分钟内介绍 35 项新功能
- 宣布将进行复杂的现场演示，并邀请观众 Matt 和 Indil 作为志愿者参与

### 00:03:00 - 基于标签的访问控制（Tag-based Access Controls）
- 推出基于标签的访问控制，简化多桶多客户端环境下的权限管理
- 通过简单的策略片段和资源标签控制权限，无需为每个桶编写复杂策略
- 新增 API：TagResource、UntagResource 和 ListTagsForResource，支持单标签操作
- 适用于所有 S3 桶类型，包括目录桶、访问点、表桶和向量桶

### 00:06:30 - 组织级控制（Organization-level Controls）
- 将 Block Public Access（阻止公共访问）功能扩展到组织级别
- 403 错误上下文功能现在支持整个组织范围内的请求

### 00:07:30 - 加密更新
- 推出 PQTLS（后量子 TLS）加密，保护传输中的数据免受量子计算威胁
- 新增禁用 SSE-C（服务器端加密-客户提供密钥）的选项
- 2026 年将默认关闭 SSE-C，新创建的桶将不接受 SSE-C 对象

### 00:09:00 - 现场演示第一部分：访问控制调试
- Anna 演示使用 Amazon Transcribe 处理视频转录
- 展示增强的错误消息功能，快速识别访问被拒绝的原因
- 通过修改桶标签解决访问问题，展示基于标签的访问控制的灵活性
- 成功上传 744 个对象到 S3 桶

### 00:12:00 - 耐久性（Durability）
- 介绍 S3 实现 11 个 9（99.999999999%）耐久性的三大机制：
  - 端到端完整性检查
  - 多设备冗余存储
  - 持续的耐久性审计
- 推出新的数据完整性检查功能，允许客户高效验证静态数据的校验和
- 通过批量操作系统运行，无需启动计算资源，不影响智能分层

### 00:15:00 - S3 Express 高性能存储更新
- 价格降低高达 85%
- TPS 限制提升至每秒 200 万次读取和 20 万次写入
- 新增对象重命名 API，支持原子重命名操作
- 增加访问点支持，提供更精细的权限控制

### 00:17:00 - 条件写入（Conditional Writes）扩展
- 去年推出的条件写入功能获得广泛采用，每天执行数百万次操作
- 新增条件复制（Conditional Copy）功能，支持 if-absent 和 if-match 条件
- 新增条件删除（Conditional Delete）功能，确保只删除预期的对象
- 这些功能是耐久性最佳实践，反映了 S3 内部使用的方法

### 00:19:00 - 对象大小限制提升
- 最大对象大小从 5TB 提升至 50TB
- 满足视频、地震数据和基因组学等领域的大对象存储需求

### 00:20:00 - 对象管理功能增强
- 对象标签价格降低 35%
- 批量操作（Batch Operations）重大更新：
  - 新增无清单选项，可直接处理整个桶或前缀
  - 自动创建 IAM 角色
  - 单个作业可处理高达 200 亿个对象
  - 性能提升高达 10 倍

### 00:22:00 - 现场演示第二部分：批量操作
- Anna 演示使用批量操作为 744 个对象批量添加颜色标签
- 展示无清单模式和自动 IAM 角色创建功能
- 根据 Paul 选择的颜色（黑色）为匹配条件的对象添加标签

### 00:24:00 - S3 Tables 概述与进展
- 自去年推出以来，客户已创建超过 40 万张表
- 提供完全托管的 Apache Iceberg 表服务
- 表作为 AWS 一等资源的独特优势

### 00:26:00 - S3 Tables 功能更新
- 推出 Iceberg REST Catalog 端点，支持使用开源 Iceberg API 直接管理表
- 每个表桶的表限制提升至 10,000 张
- 新增排序压缩（Sort Compaction）功能，提高查询效率
- 压缩价格降低 90%
- Create Table API 支持在创建时设置模式

### 00:28:00 - 表级 AWS 资源集成
- 表级 CloudWatch 和 CloudTrail 监控
- 成本管理工具中的表级成本可见性
- 表级 KMS 加密密钥支持

### 00:30:00 - 表复制（Table Replication）
- 昨日推出的重要功能，提供只读副本
- 不仅复制数据，还重写元数据以引用本地数据文件
- 确保目录一致性，查询始终访问最新的完整快照
- 处理异步复制带来的挑战

### 00:34:00 - S3 Tables 智能分层
- 为 S3 Tables 推出智能分层支持
- 存储成本最高可降低 80%
- 三个层级：频繁访问、不频繁访问（30 天后，降低 40%）、归档即时访问（90 天后，额外降低 68%）
- 压缩和表维护系统感知底层层级，仅在频繁访问层进行压缩
- 维护操作不会重置分层计时器

### 00:37:00 - SageMaker Unified Studio 集成
- S3 Tables 集成到 SageMaker Unified Studio
- 支持基于 IAM 的一键式入门
- 从 S3 控制台直接跳转到 SQL 编辑器
- 支持使用 SQL、Python 或自然语言处理表数据

### 00:39:00 - S3 Metadata（元数据服务）
- 去年预览版推出，今年 1 月推出日志表（Journal Table）
- 自动记录桶中所有变更到 Iceberg 表
- 无需额外配置即可自动生成对象元数据
