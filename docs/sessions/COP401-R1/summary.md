# AWS re:Invent 2025 成本与使用报告高级分析会议总结

## 会议概述

本次会议由AWS高级解决方案架构师倡导者Steph Gooch和FinOps高级经理Andy Brown主讲，重点介绍了AWS成本与使用报告(CUR)的高级分析技术。会议围绕三个核心主题展开：成本优化、成本分配和成本数据安全共享。演讲者通过实时编码演示，展示了如何将CUR与其他数据源(如VPC Flow Logs)结合，进行深度成本分析和优化。

会议采用互动式教学方式，包含现场问答游戏和实时查询演示。演讲者强调了CUR 2.0的新功能，并展示了如何利用Amazon Q进行智能分析，以及如何使用AWS Lake Formation实现细粒度的数据访问控制。整个会议内容实用性强，为FinOps从业者提供了具体的技术解决方案。

## 详细时间线与关键要点

### 00:00-05:00 会议开场与CUR基础回顾
- 介绍AWS成本与使用报告11年发展历程
- 确立三大会议主题：优化、成本分配、成本安全
- 演讲者自我介绍：Steph Gooch(高级SA倡导者)和Andy Brown(FinOps高级经理)

### 05:00-10:00 互动问答游戏
- CUR 2.0 vs CUR Legacy功能对比
- 数据存储位置确认(S3)
- 标签默认配置说明

### 10:00-20:00 NAT Gateway成本优化案例分析
- 演示Cost Explorer中NAT Gateway支出增长问题
- 使用CUR查询分析NAT Gateway和数据传输费用
- 创建准透视表分析9月和10月成本对比
- 识别新增资源和成本激增原因

### 20:00-30:00 VPC Flow Logs集成分析
- 利用Amazon Q获取网络优化建议
- 部署VPC Flow Logs作为补充数据源
- 演示Athena中VPC Flow Logs数据结构
- 字节到GB的数据转换技巧(除以1024³)

### 30:00-40:00 深度网络流量分析
- IP地址到资源映射分析
- 使用Amazon Q识别网络资源
- 发现跨可用区数据传输问题
- 计算NAT Gateway和数据传输具体费用

### 40:00-50:00 Amazon Bedrock成本分配
- 介绍第一方、第二方、第三方模型计费差异
- 演示推理配置文件(Inference Profiles)标签应用

- 分析不同模型选择对成本的影响

### 50:00-55:00 单位成本分析与数据安全
- 展示Lambda调用次数作为单位成本计算基础

- 介绍AWS Lake Formation行级安全控制
- 总结三大主题和行动建议
