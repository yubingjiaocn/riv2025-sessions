# AWS re:Invent 2025 智能代理补丁管理会议总结

## 会议概述

本次技术会议由AWS首席解决方案架构师Praveen Bhat和高级云支持工程师Justin Thomas主讲，重点探讨了如何利用智能代理AI来应对类似Log4j这样的关键安全漏洞事件。演讲者通过一个名为"Patchy"的多代理系统演示，展示了如何自动化漏洞评估、合规检查和补丁管理流程。

会议采用了理论与实践相结合的方式，不仅展示了完整的解决方案演示，还深入讲解了底层架构和代码实现。演讲者强调了从简单开始、逐步迭代的开发理念，并分享了他们在构建智能代理系统过程中的经验教训。整个解决方案基于Amazon Bedrock AgentCore平台，使用Strands SDK构建，体现了AWS在企业级AI应用方面的最新进展。

## 详细时间线与关键要点

### 0:00-5:00 开场与问题背景
- 演讲者介绍：Praveen Bhat（澳大利亚首席解决方案架构师）和Justin Thomas（高级云支持工程师）
- 通过举手调查了解观众对Log4j事件的经历
- 强调Log4j事件期间的混乱状况和应对挑战

### 5:00-15:00 问题场景分析
- 介绍虚构公司"Manual Everything Corporation"面临的典型挑战
- 描述紧急安全漏洞响应中的常见问题：
  - 等待合适人员加入紧急会议
  - 缺乏自动化的漏洞评估能力
  - 手动收集分散在各系统中的信息
- 提出智能代理解决方案的价值主张

### 15:00-30:00 Patchy系统演示
- 展示多代理系统架构：一个监督代理控制三个专业代理
- 演示环境配置：开发、测试、生产三个环境，各有不同的补丁计划
- 实时演示主要功能：
  - 环境概览和合规要求查询
  - 生产环境漏洞分析（发现50个高严重性漏洞）
  - 基于SLA要求的补丁时机决策
  - 紧急补丁执行流程

### 30:00-40:00 架构深度解析
- 详细介绍监督代理模式（Supervisor Pattern）
- 核心AWS服务集成：
  - Amazon Inspector：漏洞发现
  - AWS Config：资源依赖关系
  - Systems Manager：补丁合规状态和应用
  - S3：合规报告存储
- Strands SDK的优势：简化代理循环（Agentic Loop）实现
- Amazon Bedrock AgentCore的三大核心功能：运行时、内存、可观测性

### 40:00-50:00 代码实现详解
- 从最简单的EC2代理开始的迭代过程
- 展示Strands SDK的核心用法：
  - Agent类和use_aws工具的基本实现
  - 工具装饰器(@tool)的使用方法
  - 系统提示词的重要性和最佳实践
- 专业代理的详细代码分析：
  - 工具函数的文档字符串规范
  - 必需参数vs可选参数的行为控制
  - 监督代理的路由逻辑实现

### 50:00-52:30 总结与最佳实践
- 三大核心要点：
  - 工具vs提示词：工具更确定性，提示词更具创造性
  - 增强而非替代：AI应该增强现有系统能力
  - 小步构建、测试迭代：从简单开始逐步完善
