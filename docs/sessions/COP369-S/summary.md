# AWS re:Invent 2025 Kubernetes 资源管理与扩展会议总结

## 会议概述

本次技术分享由 ScaleOps 首席架构师 Steven 主讲，重点探讨了 Kubernetes 环境下的响应式扩展挑战与解决方案。会议深入分析了在大规模 Kubernetes 集群中实现可靠性、性能和成本三者平衡的"不可能三角"问题，并介绍了从容器资源管理到节点层扩展的全方位优化策略。

演讲涵盖了资源管理的基础概念、容器大小调整、水平 Pod 自动扩展（HPA）、Spot 实例采用策略，以及如何通过自定义资源和策略在大规模环境中实现智能扩展。Steven 特别强调了传统扩展方法在面对数千个 Pod 时遇到的节点碎片化和资源浪费问题，并提出了 ScaleOps 的综合解决方案。

## 详细时间线与关键要点

### 0:00-5:00 会议开场与核心问题定义
- 介绍"不可能三角"：可靠性、性能、成本三者难以同时实现
- 从 10 个 Pod 扩展到 1000+ Pod 时面临的挑战
- 资源请求与实际使用量差异导致的"幻影浪费"现象

### 5:00-10:00 资源管理基础与碎片化问题
- Kubernetes 中 requests 和 limits 的重要性差异
- 服务质量（QoS）分类：Best Effort、Burstable、Guaranteed
- 节点碎片化现象：60% 利用率下仍有 40% 浪费
- De-scheduler 等现有工具的局限性

### 10:00-15:00 Kubernetes 与 Linux 内核的交互机制
- CPU 时间片分配机制（默认 100 毫秒）
- CGroups V1/V2 在资源边界执行中的作用
- 调度器与 Kubelet 在资源管理中的不同职责
- 优先级类与服务质量在调度和驱逐中的区别

### 15:00-20:00 实际案例分析与性能挑战
- 4 核节点上 4 个 Pod 的 CPU 时间竞争示例
- 开发环境与生产环境性能差异的根本原因
- 饱和阈值的重要性：欠配置 vs 过配置的平衡点
- 大节点策略的利弊：更高利用率 vs 过度打包风险

### 20:00-25:00 扩展资源与 GPU 管理
- Kubernetes 1.33 的就地更新功能
- Pod 级别资源约束的新特性
- GPU 资源管理的三种方法：时间切片、MIG 分区、NPS 动态分配
- 扩展资源在成本优化中的重要作用

### 25:00-30:00 多维度扩展策略与挑战
- VPA、HPA、节点扩展三个维度的协调问题
- VPA 基于历史数据的局限性：无法应对突发流量
- 自定义控制器（GitLab、Airflow、Jenkins）的独特挑战
- VPA 与 De-scheduler 交互导致的频繁中断问题

### 30:00-34:24 HPA 优化与 Spot 实例策略
- HPA 反应式扩展的时延问题：从流量峰值到服务可用需要超过 1 分钟
- 预测性扩展 vs 反应式扩展的性能对比
- Spot 实例采用的复杂性：基础设施层 vs 应用层比例控制
- ScaleOps 解决方案：上下文感知的全栈扩展优化
- 客户案例验证与生产环境实际效果
