# AWS re:Invent 2025 - 使用AWS分析MCP服务器进行代理数据工程

## 会议概述

本次会议由AWS首席专家解决方案架构师Harshida Patel和数据与AI首席解决方案架构师Ram Nottath主讲，重点介绍了如何使用代理AI和模型上下文协议(MCP)来革新数据工程工作流程。会议以AnyCompany零售公司为例，展示了传统数据工程面临的挑战，包括上下文切换、数据质量问题、性能优化等生产力损失问题。

演讲者详细阐述了代理AI的工作原理，特别是代理循环(agentic loop)的概念，以及MCP如何作为"代理AI应用的USB-C"来标准化AI代理与数据源和工具的集成。会议通过实际演示展示了数据工程代理Alex如何通过自然语言指令完成复杂的数据管道任务，从数据发现到最终加载到数据仓库的全流程自动化。

## 详细时间线与关键要点

### 0:00-10:00 - 开场与问题定义
- 以厨师准备菜肴类比数据工程流程的复杂性
- 介绍AnyCompany零售公司的数据转型愿景与现实差距
- 分析数据工程师面临的核心挑战：上下文切换、缺乏文档、从零开始编码、被动数据质量管理、性能问题排查

### 10:00-20:00 - 代理AI解决方案介绍
- 对比传统数据工程与代理数据工程的工作方式
- 详细解释代理循环的工作机制：推理、行动、感知的迭代过程
- 介绍代理AI的核心组件：记忆(短期和长期)、工具访问、大脑(规划和反思)、行动控制

### 20:00-30:00 - 模型上下文协议(MCP)深度解析
- 解释MCP作为通用语言解决N×M集成问题的价值
- 详细介绍MCP架构：主机、客户端、服务器的通信模式
- 展示MCP服务器的三个核心组件：工具(函数调用)、资源(只读数据)、提示(结构化模板)

### 30:00-40:00 - AWS分析MCP服务器概览
- 介绍数据处理MCP服务器(支持Glue、EMR、Athena)
- 展示Redshift MCP服务器的只读功能
- 介绍DataZone、MSK、OpenSearch等专用MCP服务器的功能特性

### 40:00-50:00 - 实际演示：数据工程代理Alex
- 演示在Kiro和SageMaker Unified Studio中配置MCP服务器
- 展示Alex执行六步数据工程任务：数据发现、创建ETL作业、作业运行监控、数据编目、数据验证、加载到数据仓库
- 强调人机协作模式和每步确认机制的重要性

### 50:00-56:30 - 最佳实践与生产部署
- 分享关键最佳实践：合理配置MCP服务器数量、数据和基础设施保护、工具信任级别设置
- 介绍扩展部署选项：从SageMaker Unified Studio内置代理到Bedrock AgentCore企业级部署
- 提供学习资源和QR码，包括MCP服务器列表、技术博客和AgentCore文档
