# AWS re:Invent 2025 - Amazon QuickSuite 与 Agentic Automation 会议总结

## 一、会议概述

本次 AWS re:Invent 2025 分会场主要介绍了 Amazon QuickSuite 及其核心的 Agentic Automation（智能体自动化）能力。演讲者 John Brock（QuickSuite 智能体自动化产品负责人）、Nupur（Quick Flows 产品经理）以及来自 Jable 公司的客户代表 Brian Merlin Brink 共同分享了这一全新服务。

Amazon QuickSuite 是 AWS 在 2024 年 10 月推出的全新统一工作空间，旨在让企业员工能够与 AI 智能体协作，基于安全的企业数据视图完成研究、洞察分析和关键业务流程自动化。该服务的核心理念是让非技术用户也能利用 Agentic AI 的能力，无需编写代码即可自动化复杂的业务流程。QuickSuite 包含多个内置智能体，涵盖聊天助手、研究智能体、BI 分析以及自动化智能体等，所有功能都建立在 Amazon Bedrock 之上，确保企业级的安全性、合规性和数据治理。

会议重点介绍了 QuickSuite 中的两大自动化产品：Quick Flows（面向业务用户的日常任务自动化）和 Quick Automate（面向技术团队的复杂工作流自动化）。通过 Amazon 内部的实际应用案例以及客户 Jable 的深度使用场景，展示了这些工具如何显著提升工作效率，节省大量人力时间，并实现真正的业务流程转型。

## 二、详细时间线与关键要点

### **开场介绍（0:00-5:00）**
- **0:00** - 会议开始，John Brock 介绍自己是 Amazon QuickSuite 中 Agentic Automation 产品的负责人
- **1:30** - 介绍团队成员：Nupur（Quick Flows 产品经理）和 Brian Merlin Brink（Jable 公司代表）
- **2:45** - 会议议程预览：将涵盖 QuickSuite 概述、Quick Flows、Quick Automate，以及 Amazon 和 Jable 的客户案例

### **QuickSuite 产品背景（5:00-12:00）**
- **5:20** - 提出问题：82% 的组织相信将在 1-3 年内将智能体集成到关键业务流程中，但只有 15% 的业务决策预计由智能体做出
- **6:40** - 强调自动化的重要性：帮助员工更有效地与 AI 智能体协作，减少重复性工作
- **7:50** - 介绍知识工作者的三大工作支柱：信息收集、洞察分析与决策、采取行动
- **9:10** - Amazon QuickSuite 定义：统一工作空间，让人类与 AI 智能体在安全的企业数据视图上协作
- **10:30** - 数据来源包括：结构化数据（Aurora、Redshift、Dynamo）和非结构化数据（S3、OneDrive、SharePoint、Google Drive）

### **QuickSuite 核心能力（12:00-20:00）**
- **12:20** - QuickSuite 的三大核心价值：帮助员工更快做出更好的决策、采取行动、确保企业级安全与合规
- **13:45** - 内置智能体类型：
  - 聊天智能体（可自定义）
  - 研究智能体（可进行 20-30 分钟的深度研究，生成 20-30 页综合报告）
  - BI 与分析智能体（增强的 QuickSight 能力，支持动态 SQL 查询和代码解释器）
  - 自动化智能体（Quick Flows 和 Quick Automate）
- **16:00** - 可嵌入能力：仪表板和聊天智能体可嵌入到内部门户和应用中
- **17:30** - 技术架构：基于 Agent Core、支持 50+ 数据连接器、使用多种 Bedrock 模型、支持 OpenAPI 和 MCP 服务器

### **Quick Flows 介绍（20:00-35:00）**
- **20:15** - Nupur 接手演讲，强调传统自动化工具的局限性：人仍然是流程的"粘合剂"
- **21:40** - Quick Flows 的目标：让每个业务用户都能使用自动化，而不仅仅是技术专家
- **22:30** - 案例一：AWS 全球专家组织（Arush 团队）
  - 问题：客户会议准备需要大量研究时间
  - 解决方案：创建会议准备流程，自动拉取 AWS 产品信息和客户背景
  - 成果：每次运行节省 2 小时，团队每年节省数万小时
- **25:50** - 案例二：Amazon 全球合规团队（Donna 团队）
  - 问题：政策审查需要跨 23 个司法管辖区，每次审查耗时 80 小时
  - 解决方案：使用 AI 研究助手自动化繁重的分析工作
  - 成果：审查时间从 80 小时降至 15-20 小时，节省 75% 时间
- **27:30** - UI 智能体演示：可以自动导航网站、执行浏览器任务、提交表单（例如在 Amazon 网站上查找锂电池产品）
- **29:00** - 案例三：Amazon 全球卖家团队（Evan 团队）
  - 问题：卖家国际扩张咨询需要在多个系统间切换，耗时 2-3 小时
  - 解决方案：构建"选品机会导航器"流程
  - 成果：从 3 小时缩短至 10 分钟，每位客户经理每月节省 19 小时，团队整体节省 12% 工作量

### **Quick Flows 演示（35:00-42:00）**
- **35:20** - 现场演示：用自然语言描述流程需求（从 Asana 看板获取客户线索，生成摘要并每周一发送邮件）
- **36:40** - 系统自动生成工作流，用户只需配置时间、输入参数和智能体选择
- **38:10** - 演示流程执行：自动汇总销售线索状态
- **39:30** - 输出选项：可以发送邮件或下载为 PDF/Word 文档（未来将支持 PowerPoint）
- **40:20** - Space（空间）概念：数据边界和范围定义，可以存储仪表板、PDF、操作记录等，支持权限管理
- **41:00** - 与 Space 对话：可以询问历史数据趋势（例如"过去两周某客户的状态如何？"）

### **Quick Flows 核心特性（42:00-45:00）**
- **42:15** - 目标：让每个业务用户都能创建、自定义和共享专用流程，无需技术支持
- **43:00** - IT 治理：提供管理控制、访问管理、审批流程，防止"影子 IT"问题
- **43:50** - 研究智能体集成：可以在流程中调用深度研究智能体（运行 20-40 分钟）
- **44:30** - 流程调度：支持定时执行长时间运行的研究任务

### **Quick Automate 介绍（45:00-55:00）**
- **45:20** - John Brock 回归，介绍 Quick Automate：面向后台复杂业务流程的自动化（如客户支持工单分类、理赔处理、贷款发起）
- **46:40** - 企业现状：数百甚至数千个未良好集成的系统，形成数据孤岛
- **47:30** - 传统工具的局限性：RPA、低代码应用难以自动化需要推理和判断的流程，且维护成本高
- **48:50** - Quick Automate 定义：构建、部署和管理多智能体自动化的解决方案
- **49:30** - 自动化构建智能体：
  - 可以上传 SOP（标准操作程序）文档
  - 智能体自动搭建整个工作流框架
  - 迭代式交互，提出问题并提供最佳实践建议
  - 支持自然语言修改工作流，自动更新所有相关节点
- **51:00** - 三种智能体类型：
  1. **UI 智能体**：行业领先的计算机使用智能体，针对 ERP、表单和数据应用进行了强化学习优化
  2. **自定义流程智能体**：使用 Agent Core 构建，可定义指令、工具和知识库，自动部署并获得内存、护栏等能力
  3. **集成智能体**：可集成 Agent Core、Bedrock Agents 或通过 MCP 引入第三方智能体
- **53:20** - 客户案例：某客户完全自动化了客户支持收件箱分类，智能体评估每封邮件并使用 Jira、ServiceNow、邮件和人工升级等工具自动执行下一步操作
- **54:00** - 自动化编排优势：部署、版本控制、回滚、调度、日志记录、监控和治理控制

### **总结与资源（55:00-结束）**
- **55:30** - 强调 Quick Flows 和 Quick Automate 的互补性：前者面向业务用户的日常任务，后者面向技术团队的复杂工作流
- **56:00** - 提供 QR 码供参会者获取更多信息和资源
- **56:30** - 预告接下来将由 Jable 公司分享 Quick Automate 的深度应用案例
