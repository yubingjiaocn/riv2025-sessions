# AWS re:Invent 2025 会议总结：使用 Clumio 实现应用数据弹性

## 会议概述

本次会议由 Clumio（现为 Commvault Cloud 产品组合的一部分）的产品管理高级总监 Akshai Jooshi 和云销售组织首席工程师 Jeff Cassiana 主讲。会议重点探讨了如何使用 Clumio 解决方案为云应用程序数据提供弹性保护，涵盖从 NoSQL 数据库到数据湖仓库的多种场景。

演讲者指出，52% 的数据领导者表示其数据基础设施尚未为 AI 实施做好准备，这凸显了数据层弹性的重要性。会议通过一个名为"Clumio Final Cut"的虚构电影推荐应用程序演示了三个真实场景，展示了在意外删除、代码推送错误和恶意攻击等常见威胁下如何快速恢复数据。核心解决方案包括 Clumio Backtrack for DynamoDB 和 Clumio Backtrack for S3，这些功能支持精细化恢复、就地恢复和任意时间点恢复，大大简化了传统的繁琐恢复流程。

会议强调了在构建生成式 AI 应用时保护数据层的必要性，并展示了如何在不重新配置应用程序、不重新计算向量嵌入的情况下实现快速恢复，确保最佳用户体验和最小停机时间。

### 场景一：DynamoDB 数据恢复
- **00:06:00** - 分析第一个功能：用户配置文件和观看列表数据存储在 DynamoDB 中
- **00:06:45** - 解释 DynamoDB 的优势：分区性、多租户、高度可扩展
- **00:07:30** - 描述问题场景：金丝雀部署期间代码推送错误导致分区损坏
- **00:08:15** - 可视化事件时间线：下午 1:00 开始部署，下午 2:30 报告错误
- **00:09:00** - 挑战：每个分区在不同时间戳受到影响，没有单一的"最后已知良好副本"

### DynamoDB 恢复需求与解决方案
- **00:09:45** - 四个恢复要求：恢复到已知良好时间点、最小应用重新配置、控制成本、最佳用户体验
- **00:10:30** - 传统恢复方式（困难方式）：对每个分区执行完整恢复、挑选数据、复制回源表、删除临时表
- **00:11:30** - Clumio 方式：两步流程 - 选择时间点和分区，点击恢复；第二步：没有第二步
- **00:12:15** - 介绍 Clumio Backtrack for DynamoDB：2024 年 6 月发布，支持任意时间点恢复、就地恢复、精细化恢复

### 场景二：S3 与 RAG 数据管道
- **00:23:00** - Akshai 介绍第二个功能：礼宾聊天机器人
- **00:23:30** - 解释 LLM 功能与 RAG 数据管道：S3 存储桶中的电影详细信息对象、S3 中的向量存储、LLM 层
- **00:24:30** - 问题场景：如果 S3 数据丢失，向量将指向空处，聊天机器人无法正常工作
- **00:25:15** - 可视化事件：随着对象丢失，LLM 开始"胡言乱语"（例如将《真爱至上》错误分类为动作片）

### S3 恢复需求与解决方案
- **00:26:00** - 四个恢复要求：零数据丢失、最小 LLM 层重新配置、控制成本（避免重新计算向量）、最佳用户体验
- **00:27:00** - 传统恢复方式：完整恢复 S3 存储桶、重新计算向量、删除旧向量存储、清理旧存储桶、将 LLM 指向新向量存储
- **00:28:00** - Clumio 方式：选择时间点和受影响对象，点击恢复；第二步：没有第二步（无需重新计算向量、无需清理、无需更改 LLM 层）
- **00:28:45** - 介绍 Clumio Backtrack for S3：2024 年 re:Invent 发布，支持就地恢复、精细化恢复、时间点恢复

### S3 功能特性
- **00:29:30** - Clumio Backtrack for S3 可与 Secure Vault 配合使用或独立使用
- **00:30:00** - 支持 S3 版本控制，无需备份即可回滚版本
- **00:30:30** - S3 对象的连续变更跟踪，确保零数据丢失
