# AWS re:Invent 2025 日本企业创新案例分享会总结

## 会议概述

本次会议由AWS解决方案架构师Fujikura主讲,时长20分钟,主要探讨日本企业如何利用云计算和生成式AI技术实现创新转型。演讲者强调,创新已不再是可选项,而是企业生存战略。随着云计算(2006年AWS诞生)和生成式AI(2022年普及)两大技术转折点的出现,创新的难度大幅降低,现在正是推动创新的最佳时机。

会议通过五个日本企业的实际案例,展示了如何应对劳动力短缩、技术债务、客户需求多样化和全球竞争力下降等挑战。演讲者总结了创新成功的七大关键要素,并提出"小步快跑、快速迭代"的实施策略。最后强调AWS作为合作伙伴,通过完善的云环境、专家支持体系和企业级安全保障,能够帮助企业实现创新目标。

演讲者鼓励与会者识别自身业务挑战,联系AWS专家团队,从小型概念验证项目开始,通过持续反馈循环逐步扩大规模,最终实现创新突破。

## 详细时间线与关键要点

开场介绍 (0:00-2:30)
- 欢迎参会者,介绍这是re:Invent第三天的日语专场
- 演讲者Fujikura自我介绍,担任AWS解决方案架构师,支持日本企业客户云应用

会议核心信息 (2:30-4:00)
- 三大要点:①创新已成为生存战略而非可选项;②云和AI的出现使创新难度显著降低;③强调从小规模开始逐步扩展的重要性

创新概念回顾 (4:00-6:30)
- 回顾经济学家Peter Schumpeter在100年前首次提出创新概念
- 介绍Clayton Christensen的"持续性创新与颠覆性创新"理论
- 强调创新已融入日常生活,AWS致力于成为企业创新合作伙伴

日本企业面临的挑战 (6:30-9:00)
- 劳动力短缺:预计到2050年劳动人口将减少约30%
- 技术债务:约80%的日本企业受遗留系统困扰,阻碍新技术引入
- 客户需求变化:实时性和个性化推荐成为常态
- 全球竞争力下降:与数字原生企业竞争加剧

创新难度降低的转折点 (9:00-11:30)
- 2006年AWS云计算出现:大幅降低初始投资,实现快速扩展,无需管理基础设施
- 2022年生成式AI普及:使复杂业务流程问题变得可解决,无需专业知识即可实现高级功能
- 强调当前是推动创新的最佳时机

案例一:关西电力智能电表系统 (11:30-14:00)
- 挑战:供应商锁定导致系统黑箱化,电表数量增加带来服务器和网络压力
- 解决方案:将现有系统云迁移,同时构建下一代系统
- 成果:通过硬件替换实现多样化和成本降低,采用微服务和无服务器架构,消除硬件更换需求,显著减轻运维负担,实现电力使用模式分析和异常检测

案例二:RareJob在线英语会话AI反馈系统 (14:00-16:30)
- 挑战:教师需在授课同时撰写反馈,工作负担重,每节课仅能提供1-3条评论
- 解决方案:使用Amazon Bedrock开发AI课程报告Beta版
- 成果:AI自动计算语速、单词数等指标,平均生成10条评论,教师工作量减少,可专注于发挥技能和经验的响应,用户获得教师和AI双重反馈,提升学习动力

案例三:GreenSnap园艺AI (16:30-18:00)
- 解决方案:用户上传原始庭院照片,用自然语言描述期望效果(如"想要池塘"或"种植植物")
- 技术:使用Amazon Bedrock生成式AI服务生成效果图
- 成果:消除订购方与施工方之间的图像差距,仅用2个月即推向市场,开拓外装、建筑、施工等新市场,防止客户满意度下降和商机损失

案例四:SBI生命保险AI客服系统 (18:00-20:00)
- 挑战:业务扩张导致客服台运营压力增大,需提供快速服务
- 解决方案:使用Amazon Connect和Voice ID进行身份验证,结合Amazon Lex聊天机器人和文本转语音服务
- 成果:从项目启动到系统上线仅3个月,用户界面保持电话形式但由AI响应,相关部门运营效率提升约40%

案例五:理光700亿参数大语言模型开发 (20:00-22:30)
- 背景:理光2020年宣布向数字服务公司转型,启动生成式AI业务
- 挑战:开发数百亿参数LLM极其困难,需在成本、碳排放、电力消耗方面实现高效率
- 解决方案:通过AWS生成式AI创新中心支持计划,与Annapurna Labs合作,使用AWS Trainium芯片和Neuron SDK构建学习加速器
- 成果:相比典型GPU学习环境,成本降低约50%,电力消耗减少25%,模型能正确理解工作专业术语,日语回答质量指标表现优异

创新成功的七大要素 (22:30-28:00)
1. 明确目标:设定清晰的任务和业务目标
2. 小步快跑:快速启动和实施(2-3个月内投产)
3. 整合现有业务流程:融入而非从零创建
4. 数据驱动:将数据作为企业差异化要点
5. 安全与治理:确保在安全环境中创新
6. 人机协作:AI作为人类伙伴,人类专注于发挥自身价值
7. 持续改进与学习:建立长期运营和反馈机制

AWS支持创新的三大优势 (28:00-30:00)
1. 合规云环境:Amazon Bedrock提供多种企业级模型,云计算大幅降低创新成本
2. 专家支持体系:AI创新中心、日本生成式AI实践推进计划、技术支持团队、APN合作伙伴网络
3. 企业级安全:利用东京和大阪区域实现国内BCP,符合各项法规,经过验证的安全环境

行动建议 (30:00-33:00)
1. 识别挑战:明确企业运营中的挑战或业务目标(劳动力减少、系统瓶颈、依赖经验而非数据的运营)
2. 联系AWS:与AWS专家团队合作,获取行业成功案例和最佳实践
3. 快速启动:从小型POC项目开始,2-3个月内创建原型,通过持续反馈循环确保持续增长,最小化初始投资,逐步扩大规模

总结 (33:00-结束)
- 创新已成为生存战略,不做就无法生存
- 实现创新比以往任何时候都更容易
- 只要正确实施,每个人都能实现创新
- AWS愿意作为合作伙伴,支持企业迈出创新第一步
