# AWS re:Invent 2025 生成式 AI 策略会议总结

## 会议概述

本次会议主题为"生成式 AI 策略:加速生产之路"(Generative AI Strategy: Accelerating the Path to Production),由 AWS 首席解决方案架构师 Rana Dut 和高级解决方案架构师 Hardik 共同主讲。会议重点探讨了如何将 Agentic AI(智能体 AI)应用从概念验证阶段成功推进到生产环境。

演讲者指出,许多企业在将 AI 智能体应用投入生产时面临重大挑战,包括智能体孤岛问题、性能和可扩展性瓶颈、安全与治理缺失,以及未能实现预期的业务价值。为解决这些痛点,会议提供了一套完整的实施框架,涵盖从创意构思、价值建模、数据策略到原型开发和生产部署的全生命周期。特别强调了在编写任何代码之前必须完成 ROI(投资回报率)计算的重要性,并通过零售客户支持场景展示了如何通过 Agentic AI 实现超过 1000 万美元的年度成本节约。

## 详细时间线与关键要点

### 开场介绍 (00:00 - 02:30)
- **00:00** - 会议开始,介绍主题:生成式 AI 策略与生产路径加速
- **00:45** - Rana Dut 自我介绍:AWS 首席解决方案架构师,驻纽约

### 生产障碍分析 (02:30 - 05:00)
- **02:30** - 现场调查:已构建 AI 智能体应用的参会者举手
- **03:00** - 识别四大主要障碍:
  - 智能体孤岛(不同团队独立开发,缺乏集成)
  - 性能与可扩展性问题(测试不充分,无法处理大规模用户)
  - 安全与治理缺失(未通过安全审查和合规要求)
  - 业务价值不足(未进行 ROI 计算)

### Agentic AI 应用生命周期 (05:00 - 08:30)
- **05:00** - 介绍应用生命周期框架
- **05:30** - 第一阶段:创意构思 - 关键利益相关者头脑风暴,识别痛点和手动流程
- **06:00** - 第二阶段:价值建模 - 强调 ROI 计算的关键性,引用客户案例:"在编写任何代码之前先完成 ROI 计算"
- **06:45** - 第三阶段:数据策略 - 评估数据源是否为 AI 做好准备,包括权限设置
- **07:15** - 第四阶段:构建 MLP(最小可爱产品,而非 MVP)
- **07:45** - 第五阶段:评估与迭代 - 确保答案准确可信,处理 LLM 的非确定性特征
- **08:15** - 强调安全、治理和负责任 AI 贯穿整个生命周期

- **08:30** - 保险行业:自动化承保流程
- **09:00** - 银行业:信用卡申请处理,自动化 W2 和收入验证

### 价值建模与 ROI 计算 (11:00 - 16:00)
- **11:00** - 介绍 TCO(总拥有成本)计算框架
- **11:30** - 成本构成示例(零售客户实际数据):
  - API/推理成本:25%
  - 提示优化:15%
  - CI/CD 基础设施:10%
  - 数据策略:15%
  - 人才培养与开发:20%
  - 运营支持:10%
  - 道德 AI:5%
- **13:00** - 回报指标:
  - 客户满意度评分(CSAT)提升
  - 员工生产力提高
  - 业务运营优化
  - 增量收入增长
- **14:00** - 零售客户案例研究:
  - 250 名客服人员,每天处理 10,000 个查询
  - 人工成本:每小时 30 美元
  - AI 智能体成本:每次查询 0.10 美元
  - AI 解决 50% 查询
  - **年度节约:超过 1000 万美元**
  - 成本降低 50%
- **15:00** - 其他目标:客户满意度提升 10%,转化率提升 15%,任务完成效率提升 22%

### 数据策略 (16:00 - 20:30)
- **16:00** - 强调数据作为差异化竞争优势的重要性
- **16:45** - 数据源类型:
  - 事务型数据库(SQL Server、PostgreSQL、MySQL)
  - 半结构化数据(电子邮件、应用日志、HTML)
  - 非结构化数据(PDF 文档、HR 政策)
  - 第三方 SaaS 数据(Salesforce、Jira、Atlassian)
  - 大数据和数据仓库(Redshift)
- **18:00** - 数据摄取层架构:
  - 将文档导入 Bedrock 知识库(向量存储)
  - CSV 文件导入数据湖以支持 SQL 查询
  - 构建语义层实现自然语言到 SQL 的转换
- **19:30** - 强调数据安全和治理,特别是 S3 存储桶权限配置

### 构建有效 POC - Agentic 工作流 (20:30 - 28:00)
- **20:30** - Hardik 接手演讲
- **21:00** - 智能体工作流组件:
  - 目标/任务定义
  - 关键输入:记忆、工具、数据源
  - 大语言模型作为推理引擎
  - 护栏和观察层确保安全性和可靠性
  - 行动执行和反馈循环
- **23:00** - 渐进式构建方法(零售用例):
  - V1:订单微服务(查询订单状态、取消、退货、追踪)
  - V2:添加知识库(退货和订单政策)
  - V3:产品操作组(产品查找和推荐)
  - V4:添加账户和支持操作
- **25:00** - 问题:智能体单体 - 代码复杂、提示过长、工具过多、难以维护
- **26:30** - 解决方案:监督者-专家智能体模式
  - 监督者智能体协调多个专家智能体
  - 每个专家智能体专注于特定领域
  - 订单专家、产品专家、促销专家等

### 实时演示 (28:00 - 38:00)
- **28:00** - 介绍演示环境:"Every Step"鞋业公司
- **28:30** - 展示多种鞋类产品和多个门店
- **29:00** - 右侧显示 Agentic AI 模式架构图
- **29:30** - 演示 1:门店设施查询
  - 监督者智能体 → 门店智能体 → 知识库工具
  - 返回 Wi-Fi、儿童游乐区等设施信息
- **31:00** - 演示 2:鞋码咨询
  - 监督者智能体 → 商品智能体 → 商品知识库
  - 提供在线购鞋尺码建议
- **32:30** - 演示 3:促销查询
  - 支付和促销专家智能体
  - 调用 MCP 服务器上的列表促销工具
  - 返回当前所有促销活动
- **34:00** - 演示 4:复杂购买流程
  - 用户:"我想订购跑鞋"
  - 智能体列出所有跑鞋类别和价格
  - 用户指定:"Air Running 鞋,男款 10 码,黑色"
  - 转换为结构化 JSON 对象
  - **人工确认环节**(关键步骤)
  - 调用 DynamoDB 下单工具完成订单
- **36:30** - 演示 5:图工作流模式
  - 一键式订单履行工作流
  - 并行执行多个步骤
  - 生成详细的跟踪和审计报告

### 多智能体协作模式 (38:00 - 结束)
- **38:00** - 展示异步工作流执行
- **38:30** - 强调图工作流模式用于订单履行
- **39:00** - 演示完整的端到端集成:从查询到购买到履行

### 关键技术要点
- 使用 Amazon Bedrock 知识库作为向量存储
- 集成 DynamoDB 进行事务处理
- 部署 MCP(模型上下文协议)服务器扩展功能
- 实施 Bedrock Agent Core 进行评估
- 应用护栏确保负责任的 AI 使用
- 构建 CI/CD 管道实现持续部署
