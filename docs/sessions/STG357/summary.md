# AWS S3 Metadata 技术会议总结

## 会议概述

本次 AWS re:Invent 2025 技术分享会由 Amazon S3 产品经理 Roohi Sood 和软件开发经理 Claire 共同主讲，重点介绍了 S3 Metadata 服务如何解决企业数据发现挑战。会议通过实际演示展示了如何利用 S3 Metadata 实现自动化元数据提取和查询，帮助数据科学家和安全团队快速定位所需数据。

演讲者强调，随着数据量的爆炸式增长（Amazon S3 现存储超过 500 万亿个对象）和生成式 AI 革命的到来，传统的数据发现方法已无法满足需求。S3 Metadata 通过提供自动、全面且始终最新的元数据解决方案，让企业能够在数据发现竞赛中获得不可逾越的 AI 优势。

## 详细时间线与关键要点

### 0:00-5:00 - 问题背景与挑战
- 介绍数据发现的常见场景：数据科学家在 2300 万个对象中寻找标记图像，安全团队逐一检查敏感信息
- 强调这不是数据问题，而是数据发现问题
- 阐述数据增长速度和 GenAI 革命对数据价值的重新定义

### 5:00-10:00 - S3 Metadata 解决方案介绍
- 传统元数据解决方案的三大问题：复杂性、难以维护、数据不同步
- S3 Metadata 的核心特性：自动化、全面性、实时更新
- 基于 Apache Iceberg 格式，存储在 S3 table buckets 中

### 10:00-15:00 - 元数据表结构详解
- Journal 表：审计日志功能，记录每个操作的详细信息，几分钟内刷新
- Live inventory 表：数据快照视图，每小时刷新一次
- 21 种元数据类型：请求元数据、系统元数据、自定义元数据

### 15:00-20:00 - 技术架构与查询方式
- Apache Iceberg 开放表格式的优势
- S3 Tables 的性能优势：比通用存储桶高 10 倍的事务处理能力
- 多种查询引擎支持：Athena、Redshift、SageMaker、DuckDB、Spark 等
- AWS Lake Formation 集成提供细粒度访问控制

### 20:00-25:00 - 实际应用场景
- 请求元数据用例：追踪数据变更、识别删除操作、回滚删除
- 系统元数据用例：查找未加密对象、分析存储类别分布
- 自定义元数据用例：区分 AI 生成内容、传感器数据上下文标记

### 25:00-35:00 - 现场演示
- 演示如何在 1 分钟内设置元数据配置
- 使用 SageMaker Unified Studio 查询 journal 表，追踪 7 天内的删除操作
- 使用 Athena 查询 live inventory 表，基于标签筛选对象
- 展示如何将查询结果用于 S3 批处理操作，自动恢复 Glacier 存储的数据
- 使用 Kiro CLI 和 MCP 进行自然语言查询

### 35:00-40:00 - 客户案例与总结
- 医疗影像客户案例：从复杂的 Lambda 处理流程转向基于 S3 Metadata 的批处理，显著提升处理效率
- 数字内容公司案例：利用 SQL 查询管理数百万文件的云迁移过程
- 三大关键收益：实时元数据、智能工作流构建、AI 代理集成的未来准备
- 服务已在 28 个区域可用，其中 22 个区域在过去一周内上线
