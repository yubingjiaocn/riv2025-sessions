# AWS re:Invent 2025 会议总结：从文档到决策 - Arite 的生成式 AI 解决方案

## 会议概述

本次会议由 AWS 商业销售负责人 David Meford 和 Arite 数据科学与 AI 副总裁 Priya 共同主持，主题为"从文档到决策"（Documents to Decisions）。Arite 是一家总部位于芝加哥的全球管理和技术咨询公司，主要服务于医疗保险公司和金融服务行业。会议重点介绍了 Arite 如何利用 AWS 生成式 AI 技术解决文档数据提取的业务挑战。

Arite 面临的核心问题是大量关键数据被锁定在合同文档中，包括医疗服务提供商合同和供应商合同。传统的手工处理方式不仅效率低下、需要大量人力资源和专业知识，而且存在主观性差异，无法规模化处理。通过开发基于生成式 AI 的 Doxy AI 工具，Arite 成功实现了文档元数据的自动提取和解释，将处理能力从每人每周 100 份文档提升至每周 50 万份文档，准确率达到 99%，并为客户节省了 3.3 亿美元。

该解决方案完全构建在 AWS 平台上，采用 Amazon Textract 进行 OCR 处理，通过 Amazon Bedrock 调用多个大语言模型（包括 Nova Pro 和 Anthropic Claude 系列），并将结构化数据存储在 Snowflake 中。Arite 计划将 Doxy AI 打造成 SaaS 平台，使其成为客户业务流程自动化的核心组成部分。

## 详细时间线

0:00 - 0:45 - 会议开场介绍  
David Meford 介绍自己是 AWS 芝加哥地区的商业销售负责人，负责帮助客户构建解决方案。介绍本次会议主题为"从文档到决策"，并欢迎 Arite 的 Priya 参加她的首次 re:Invent 大会。

0:45 - 2:15 - Arite 公司和业务背景介绍  
Priya 介绍自己担任 Arite 数据科学与 AI 副总裁。Arite 是一家全球管理和技术咨询公司，总部位于芝加哥，主要服务医疗保险公司（健康保险支付方），帮助处理理赔、降低医疗和行政成本、提升会员参与度和留存率，并协助护理协调管理。公司还设有金融服务业务部门。

2:15 - 4:30 - 业务挑战说明  
Priya 详细阐述核心挑战：关键数据被锁定在文档中，特别是医疗服务提供商合同（医院和医生与健康保险公司之间的合同）以及会员供应商合同。这些合同包含支付条款、合同条款、SLA、KPI 等重要信息。现有解决方案存在不足：需要大量专业知识和人力资源、无法规模化、现有的合同生命周期管理工具只能配置少数字段，导致下游分析和洞察应用受阻。

4:30 - 6:00 - Doxy AI 解决方案介绍  
介绍 Doxy AI 是一个基于生成式 AI 构建的元数据提取和解释工具。它能像人类一样理解文档，提取信息并存储到结构化存储库中供下游使用。主要优势包括：消除人工处理的主观性和差异性，确保所有文档以统一方式处理；实现规模化处理；提取的数据可聚合、可查询、可搜索，支持下游应用需求。

6:00 - 8:30 - 技术演进历程  
Priya 回顾了 Arite 的技术演进之路：
- **2020 年之前**：完全手工处理，拥有 17 年的本体论经验，但每人每周只能处理 100 份文档，顾问对这种单调工作不满意
- **2020 年**：采用 AWS Textract 作为 OCR 工具，开发基于规则的引擎提取特定位置的数据，但准确率仅 50-60%，且只适用于遵循特定模板的文档
- **2024 年**：随着生成式 AI 的出现，进行快速原型开发，发现这是解决问题的完美技术，取得了显著成效

8:30 - 10:30 - 业务成果和影响  
展示了令人瞩目的业务成果：
- 处理能力从每人每周 100 份文档提升至每周 50 万份文档
- 准确率达到 99%
- 人工工作量减少 97%
- 为客户节省 3.3 亿美元，包括直接节省（通过对比合同和理赔识别超额支付）和间接节省（避免大量人力资源投入）

10:30 - 13:00 - 技术架构详解  
Priya 详细介绍了完全基于 AWS 构建的端到端解决方案架构：
- 前端：Next.js 构建的用户界面
- 身份验证：AWS Cognito
- 存储：文档上传至 Amazon S3
- OCR 处理：Lambda 函数调用 Amazon Textract，将 PDF 转换为文本，识别表格、表单、复选框、签名甚至手写笔记
- 计算：ECS 容器化服务进行文档分块处理
- AI 推理：通过 API 调用 Amazon Bedrock，使用 Nova Pro、Anthropic Claude Sonnet 3.5、3.7 和 4.0 等多个模型，结合 17 年本体论和领域专业知识构建的提示词
- 数据存储：Snowflake 数据库存储结构化输出，供前端用户使用

13:00 - 14:30 - 规模和性能数据  
David 强调了几个关键技术优势：能够灵活切换 4-5 个不同模型以优化成本和性能；在高度监管的行业中，解决方案部署在客户自己的 VPC 中，满足 HITRUST 等合规要求。Priya 分享了处理规模：在过去 22 个月中处理了约 250 万份文档，相当于 4420 亿个 token。这一规模的实现离不开 AWS 团队的支持，不断提升 token 速率限制以满足客户交付需求。

14:30 - 16:00 - 未来发展方向  
Priya 阐述了产品愿景：Doxy AI 不是一次性工具，而是要成为业务流程自动化的核心组成部分。由于合同文档会不断更新、重新谈判和修订，Doxy AI 将嵌入医疗保险公司的生态系统中，在合同签署时自动提取元数据，支持理赔裁决、理赔准确性、发票准确性和定价等下游流程。公司计划年底推出 SaaS 平台，并已提交专利申请。

16:00 - 16:45 - 会议总结  
David 指出该解决方案还为供应商管理和核心健康保险业务之外的其他合同管理场景开启了新机会，Arite 现在能够快速行动，赢得了业务和客户的信心。感谢 Priya 分享挑战、解决方案以及与 AWS 的合作成果，并祝她享受首次 re:Invent 之旅。
