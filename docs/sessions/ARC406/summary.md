# AWS re:Invent 2025 - 使用Amazon Nova进行天气预报的AI解决方案

## 会议概述

本次技术会议由英国气象局（Met Office）的数据科学IT研究员Ed Steele和AWS全球专业服务团队的应用科学家Dinesh共同主讲。会议重点介绍了如何利用Amazon Nova基础模型来革新传统的天气预报服务，特别是具有百年历史的英国海运天气预报。

演讲者展示了一个创新的AI解决方案，该方案能够将复杂的数值天气预报数据转换为结构化的文本预报。通过结合大语言模型（LLM）和视觉语言模型（VLM）方法，他们在仅4周的原型开发期间就实现了52-62%的准确率，而传统方法需要专业气象学家花费数小时才能完成同样的工作。

## 详细时间线与关键要点

### 00:00-05:00 - 开场与背景介绍
- 播放了英国海运天气预报的音频示例，这是世界上最古老、运行时间最长的天气预报
- 介绍海运天气预报的历史起源：1859年皇家宪章号沉船事故导致450多人丧生
- 1861年海军上将Robert Fitzroy建立了英国首个海运风暴预警服务

### 05:00-15:00 - 气象局现状与AI机遇
- 气象局每天处理2150亿个观测数据点
- 独立分析显示气象局未来10年将为英国经济贡献560亿英镑，投资回报率达19:1
- 介绍数值天气预报的工作流程：观测数据→超级计算机建模→专家解释→公众服务
- 物理模型预报准确性每十年提升0.5-1天

### 15:00-25:00 - 机器学习在天气预报中的革命
- 2017年开始，数据驱动模型性能稳步提升
- 2022年12月成为转折点，多个数据驱动模型首次超越物理模型性能
- DeepMind、Nvidia、华为等科技公司推动了这一突破
- 强调"最后一英里"预报的重要性：将数值数据转换为用户可理解的信息

### 25:00-35:00 - 技术挑战与解决方案
- 海运天气预报数据规模：单次预报运行产生45GB大气数据和5GB海洋数据
- 三个月数据总量约85GB，格式为NetCDF（网络通用数据格式）
- 提出两种方法：文本中介的LLM方法和视频编码的VLM方法
- 使用Amazon Nova Lite和Nova Pro 1.0模型进行实验

### 35:00-45:00 - 架构深度解析（Dinesh主讲）
- 介绍两种微调选择：SageMaker训练作业vs SageMaker HyperPod
- 实验使用P5.48xlarge GPU，4个GPU，训练时间3小时
- LLM架构：S3存储→ECS并行处理→Bedrock基础模型→结果存储
- 生产架构增加了AWS Glue Catalog、Lake Formation、SQS解耦等组件

### 45:00-55:00 - VLM方法与实验结果
- VLM数据格式：每天4次预报，31个海域，每个区域5个天气属性
- 将24小时数据转换为24帧图像，创建1秒视频
- 86GB原始数据转换为56GB视频数据
- 比较LoRA微调vs全量微调：全量微调性能提升6.2%，但需要预置吞吐量

### 55:00-59:00 - 评估方法与未来展望
- 使用基于词汇的F1评分进行严格评估
- Nova Pro模型平均F1得分62%，优于Claude 3.7 Sonnet的57%
- 个体模型比组合模型平均提升2.7%
- 分类数据比连续数据模型平均提升25.4%
- 展示了用户界面原型，支持多语言翻译功能
