# AWS re:Invent 2025 数据库节省计划深度解析

## 会议概述

本次会议深入介绍了在AWS re:Invent 2025大会上刚刚发布的数据库节省计划(Database Savings Plan)。这是客户期待已久的功能，自计算节省计划推出六年来一直是最受欢迎的功能请求之一。

会议由两位专家主讲：Adam Driver，AWS解决方案架构团队负责人，专注数据库领域五年以上；Rick Ochs，优化产品团队负责人，负责成本优化中心、节省计划分析器等产品，同时担任FinOps技术咨询委员会成员。

数据库节省计划的核心价值在于提供灵活性和可移植性，允许客户在不同数据库服务、实例类型和部署模式之间自由切换，同时保持节省效果。与传统预留实例的三年承诺不同，该计划采用一年承诺期，无需预付费用，最高可节省35%的成本。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与背景
- 介绍数据库节省计划作为Matt主题演讲的最后一项重要发布
- 演讲者自我介绍：Adam Driver（解决方案架构师）和Rick Ochs（优化产品团队负责人）
- 现场调研：FinOps团队成员、DBA、re:Invent参会经验统计

### 5:00-10:00 议程概览与产品价值
- 会议议程：覆盖范围、工具技术、最佳实践
- 数据库节省计划创建原因：客户反馈驱动，提供可移植性和灵活性
- 核心特点：自动化、一致性定价模型、一年承诺、无预付费、无浪费

### 10:00-15:00 服务覆盖范围详解
- 支持Intel和Graviton实例，涵盖第7代和第8代
- 同时支持无服务器和预置部署模式
- 详细服务列表展示：Aurora PostgreSQL最高35%折扣，RDS SQL Server、DynamoDB等各服务折扣比例

### 15:00-20:00 灵活性与可移植性优势
- 允许在数据库服务间切换、更改引擎、在无服务器和预置模式间转换
- 跨可用区域内的灵活性，适应不断演进的架构需求
- 与预留实例对比：预留实例需要特定预留，节省计划提供更大灵活性

### 20:00-25:00 使用场景与迁移策略
- 节省计划适用场景：需要在数据库服务间切换、快速采用新技术、简化成本管理
- 预留实例适用场景：稳定可预测的工作负载、特定实例类型、长期承诺
- 迁移策略：可同时使用两种模式，但不能在同一工作负载上组合使用

### 25:00-30:00 购买决策流程与推荐算法
- 购买生命周期：确定购买量→跟踪覆盖率和利用率→决定额外购买
- 推荐产品和节省计划购买分析器介绍
- 黄金分割搜索算法：测试不同购买规模，找到最高节省金额的最优点

### 30:00-35:00 控制台新功能展示
- 推荐界面新增数据库节省计划选项
- 三个关键图表：覆盖率图表、利用率图表、成本分析图表
- 折扣应用优先级：按折扣百分比从高到低应用（35% Aurora无服务器优先）

### 35:00-40:00 购买策略与最佳实践
- 阶梯式增量购买策略：可分批购买，立即开始节省
- 从预留实例迁移：等待到期后转换，或混合使用策略
- 80%覆盖率目标建议，为未来优化和现代化留出空间

### 40:00-45:00 实时演示操作
- 计费和成本管理控制台演示
- 推荐生成过程：7天、30天、60天回看期选择
- 购买分析器自定义日期范围功能
- 七天内退货政策保护机制

### 45:00-48:00 总结与行动建议

- 强调无风险试用（七天退货政策）
