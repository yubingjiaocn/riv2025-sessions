# AWS re:Invent 2025 数据库节省计划深度解析

## 会议概述

本次 AWS re:Invent 2025 分会场深入探讨了刚刚在 Matt 主题演讲中宣布的数据库节省计划（Database Savings Plans）。这是客户期待已久的功能，自六年前推出计算节省计划以来，数据库节省计划一直是客户最迫切需要的功能之一。

演讲由 Adam Driver（AWS 解决方案架构团队数据库领域负责人）和 Rick Oaks（优化产品团队负责人）共同主持。数据库节省计划的核心价值在于为客户提供前所未有的灵活性和可移植性，允许在不同数据库服务、实例类型、部署模型（预置实例与无服务器）之间自由切换，同时保持成本节省。与传统的预留实例相比，数据库节省计划采用一年期承诺、无需预付费用的模式，最高可节省 35%（Aurora 无服务器），并且未覆盖的使用量按按需定价计费，真正实现零浪费。

该计划覆盖第 7 代和第 8 代 Intel 和 Graviton 实例，支持 Aurora、RDS、DynamoDB、ElastiCache、MemoryDB、Neptune、DocumentDB、Timestream 和 DSQL 等九大数据库服务。推荐算法采用黄金分割搜索算法，自动计算最优购买金额以实现最大节省。客户可以采用增量购买策略，分阶段建立覆盖率，并通过控制台中的覆盖率和利用率报告持续跟踪效果。
