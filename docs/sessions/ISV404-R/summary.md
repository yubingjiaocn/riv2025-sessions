# AWS re:Invent 2025 - ISV404: 构建弹性SaaS：多账户弹性测试模式

## 会议概述

本次会议由AWS高级解决方案架构师Kanniah VJ独自主讲，重点介绍如何使用AWS故障注入服务(FIS)为多租户SaaS架构构建弹性测试模式。会议针对ISV(独立软件供应商)如何在不影响关键组件的情况下，通过受控实验验证多账户弹性架构设计。

演讲者分享了AWS在构建弹性应用方面的经验和最佳实践，通过实际的多租户控制平面和应用平面架构演示了多种弹性测试模式。会议涵盖了SaaS架构的六大关键支柱：强大的租户隔离、噪声邻居缓解、全面的身份访问管理、租户感知可观测性、战略分层和成本感知跟踪机制。

## 详细时间线与关键要点

### 0:00-5:00 会议开场与背景介绍
- 演讲者自我介绍：Kanniah VJ，AWS英国ISV团队高级解决方案架构师
- 会议主题：ISV404 - 构建弹性SaaS多账户弹性测试模式
- 互动环节：询问参会者是否经历过多租户SaaS注册失败的紧急情况
- 介绍AWS故障注入服务在验证SaaS架构设计中的作用

### 5:00-10:00 SaaS架构核心原则
- SaaS架构六大关键支柱详解：
  - 强大的租户隔离防止安全和数据泄露
  - 噪声邻居缓解维持一致的用户体验
  - 全面的身份访问管理确保安全租户边界
  - 租户感知可观测性
  - 战略分层服务多样化客户群体
  - 成本感知跟踪机制
- AWS Well-Architected可靠性支柱中的故障管理最佳实践

### 10:00-15:00 AWS弹性生命周期框架
- 五个关键阶段介绍：
  - 设定目标：理解工作负载所需的弹性级别
  - 设计和实施：预测各种故障模式
  - 评估和测试：执行弹性测试
  - 运营：关注可观测性工具和指标
  - 响应和学习：基于事件制定缓解策略
- 弹性实验的四个阶段：稳定状态、假设、运行实验、改进

### 15:00-20:00 AWS故障注入服务(FIS)介绍
- FIS作为完全托管服务的优势
- 与IAM集成的编排角色配置
- 实验模板的三个关键组件：
  - 操作(Action)：要执行的故障活动
  - 目标(Target)：AWS资源
  - FIS安全防护：定义实验停止条件
- 场景库和预构建实验模板
- BMW集团案例研究

### 20:00-30:00 真实世界SaaS工作负载架构
- 两个关键角色：SaaS提供商和租户
- 控制平面和应用平面的分离
- 两个SaaS解决方案演示：
  - 电商解决方案：租户特定的产品管理和订单处理
  - RAG(检索增强生成)解决方案：组织数据索引和向量存储
- AWS SaaS参考架构详解
- 多账户结构设置：四个不同账户的特定角色

### 30:00-40:00 弹性测试模式一：多租户噪声邻居
- 噪声邻居场景定义和缓解策略
- 实验架构：两个租户(安全ISV和HR技术ISV)
- API网关使用计划配置：令牌限制和请求限制
- Lambda授权器的JWT令牌验证
- 实验假设：禁用EventBridge规则和删除CloudWatch日志流
- 现场演示：
  - 代码结构展示(CDK和Python模块)
  - 租户配置和稳定状态验证
  - FIS实验执行和结果分析
  - DynamoDB表重置和验证

### 40:00-45:00 其他弹性测试模式概述
- 模式二：租户隔离验证
  - 三种授权策略：基于角色的访问控制、动态IAM策略、基于属性的访问控制

### 45:00-51:00 关键要点和总结

- 将弹性测试集成到CI/CD流程中
