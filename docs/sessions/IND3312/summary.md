# AWS re:Invent 2025 - Coinbase 如何利用生成式 AI 扩展业务运营

## 会议概述

本次会议(Industry 3312)由 AWS 金融服务高级解决方案架构师 Joshua Smith 和 Coinbase 机器学习与人工智能总监 Vara Maharivan 共同主讲,深入探讨了 Coinbase 如何利用 AWS 上的生成式 AI 技术来转型其客户支持、合规性和开发者生产力工作流程。

会议首先回顾了金融服务行业在生成式 AI 领域的演进历程:从 2023 年对数据安全和信任的基本疑问,到 2024 年开始规模化应用并关注架构优化,再到 2025 年构建完全自主的多智能体应用系统。Joshua Smith 详细介绍了 AWS 为智能体 AI 提供的核心服务,特别是 Amazon Bedrock 和新推出的 Bedrock Agent Core,这些服务为企业提供了构建、部署和运营生产级 AI 智能体所需的专业化运行时、身份管理、内存系统和可观测性工具。

Vara Maharivan 随后分享了 Coinbase 的实践经验。作为一家致力于为全球超过 10 亿人扩展经济自由的加密货币交易平台,Coinbase 在安全性、可访问性和全球化扩展三大支柱上都深度应用了 AI 技术。从账户登录的防护、资金转移的风险评估,到个性化搜索和实时价格提醒,机器学习已经渗透到用户旅程的每个环节。Coinbase 还在区块链与 AI 的交叉领域进行了创新,包括基于图神经网络的区块链地址风险评分系统、ERC20 诈骗代币检测,以及预测性机器学习模型来应对加密市场的波动性。

## 详细时间线与关键要点

### 开场与背景介绍 (0:00 - 3:30)
- **0:00** - 会议开始,Joshua Smith 介绍会议主题:Coinbase 如何在 AWS 上使用生成式 AI 扩展支持、合规和开发者生产力工作流程
- **0:30** - 介绍演讲嘉宾:Joshua Smith(AWS 金融服务高级解决方案架构师)和 Vara Maharivan(Coinbase 机器学习与 AI 总监)
- **1:00** - 会议议程概览:AWS 服务介绍、Coinbase AI 愿景、客户支持用例、合规用例、开发者体验以及未来路径

### 生成式 AI 在金融服务领域的演进 (3:30 - 6:00)
- **3:30** - 2023 年:大型语言模型开始进入市场,客户关注数据信任和安全问题,检索增强生成(RAG)成为关键需求
- **4:15** - 2024 年:Amazon Bedrock 发布后,客户开始在客户支持等关键领域应用 AI,关注点转向规模化、容量和数据访问的精细化控制
- **5:00** - 2025 年:随着智能体模式的兴起,客户开始构建完全自主的多智能体应用,问题从"如何构建 AI 聊天"转变为"如何彻底改变业务部门的运作方式",重点转向可衡量的业务影响和 KPI

### AWS 智能体 AI 服务架构 (6:00 - 11:30)
- **6:00** - AWS 提供从预构建智能体到自定义智能体的完整解决方案栈,包括 Q Developer、Q Business、AWS Transform 等
- **7:00** - Amazon Bedrock 介绍:完全托管的服务,通过单一 API 访问 Anthropic、Meta、Mistral 和 Amazon 等领先基础模型
- **8:00** - 生产环境部署智能体的六大挑战:
  - 需要专业化运行时来处理长时间运行和多模态交互
  - 需要内存系统来保留上下文和学习历史
  - 需要细粒度的身份和访问控制
  - 需要工具和数据源的发现与交互能力
  - 需要自主执行多步骤工作流程
  - 需要专门的监控和可观测性工具

### Bedrock Agent Core 核心能力 (11:30 - 14:00)
- **11:30** - Agent Core Runtime:专为部署和扩展动态智能体构建的无服务器运行时,支持长达 8 小时的工作负载,具备检查点和恢复能力
- **12:00** - Agent Core Gateway:轻松集成 MCP 服务器和 API,为智能体提供丰富的工具集
- **12:30** - Agent Core Browser 和 Code Interpreter:允许智能体在浏览器中自主操作或执行代码
- **13:00** - Agent Core Identity:提供基于标准的身份验证、OAuth 支持和安全令牌保管库
- **13:30** - Agent Core Memory:为复杂工作流程提供短期和长期记忆存储
- **14:00** - 可观测性:集中化的日志、追踪和指标,用于大规模衡量性能和准确性

### Coinbase 公司使命与 AI 战略 (14:00 - 18:00)
- **14:00** - Vara Maharivan 开始演讲,介绍 Coinbase 使命:为超过 10 亿人扩展经济自由
- **14:30** - Coinbase 三大核心支柱:
  - 建立信任:通过顶级安全保护用户资产,防范欺诈和账户接管
  - 使加密货币易于访问:通过 AI 和机器学习设计直观和个性化的体验
  - 全球化扩展:高效运营,支持 100 多个国家的数百万用户和数十亿美元的交易量

### 机器学习在 Coinbase 平台中的应用 (18:00 - 22:00)
- **18:00** - 用户旅程中的 ML 应用示例:
  - 登录时:ML 模型防御账户接管
  - 法币转入:ML 模型评估用户合法性和信用违约风险
  - 资金转移到区块链:ML 模型评估逆转风险
- **19:00** - 个性化体验:个性化搜索结果、定制新闻源、智能推荐、实时价格提醒
- **20:00** - 技术基础设施:使用 Anyscale 平台(基于 Ray 框架)在 AWS EKS 集群上运行深度学习模型的训练和推理

### 区块链与 AI 的创新交叉应用 (22:00 - 25:00)
- **22:00** - 区块链地址风险评分系统:基于图神经网络的自适应和高度可扩展系统,检测和阻止恶意区块链地址
- **23:00** - ERC20 诈骗代币检测:结合智能合约审计和机器学习,在资产上市前进行审查
- **24:00** - 预测性机器学习模型:在交易量激增前预先扩展后端数据库,应对加密市场的波动性

### 生成式 AI 的三大应用领域 (25:00 - 27:00)
- **25:00** - LLM 自 2022 年 11 月以来改变了技术格局,Coinbase 在三个关键领域应用生成式 AI:
  - 客户支持:AI 驱动的虚拟助手和人工客服的高级工具
  - 合规性:简化复杂调查和监管合规流程
  - 开发者生产力:提升开发效率

### 客户支持的挑战与解决方案 (27:00 - 31:00)
- **27:00** - 加密货币客户支持的独特挑战:
  - 极端波动性:用户活动可能在一个月内波动 50%
  - 信任至关重要:客户需要在任何市场条件下都感到安全和被支持
  - 全球运营:需要支持不同语言、法规和期望
- **29:00** - Coinbase 的生成式 AI 战略:
  - **广度**:内部生成式 AI 平台集成多个 LLM,通过标准接口(OpenAI API 标准和 MCP 标准)访问数据源
  - **深度**:在客户支持的高影响领域进行针对性投资

### 客户支持聊天机器人的三层架构 (31:00 - 42:00)
- **31:00** - 客户支持的三个用例形成连续循环:
  1. AI 聊天机器人提供即时帮助
  2. 复杂案例升级到配备 AI 辅助工具的人工客服
  3. 接触后 AI 提取模式以改进产品

- **32:00** - 第一层:FAQ 式 RAG 响应
  - 简单的检索增强生成层,处理登录、双因素认证等基本查询
  - 专注于准确性,为更多功能建立坚实基础

- **34:00** - 智能体架构详解:
  - 用户输入和对话历史存储在向量数据库中作为记忆层
  - RAG 检索器使用 Bedrock Knowledge Bases 向量化和存储 Coinbase 帮助文章
  - 使用 Cohere 的重排序模型提高检索准确性
  - 响应生成使用混合 LLM(包括通过 Bedrock 提供的 Claude 模型)
  - 采用 Actor-Critic 架构的子智能体

- **36:00** - 护栏机制:
  - 输入和输出护栏使用 Bedrock Guardrails 防止有害内容和 PII 泄露
  - 自定义领域特定过滤器最小化提示注入风险
  - 自定义规则保持响应的准确性并减少幻觉

- **37:30** - 第二层:业务流程自动化
  - 随着模型能力提升,聊天机器人增强为直接执行业务流程
  - 从对话式收集信息到代表用户直接采取行动
  - 业务流程成为"单一事实来源",人类和 AI 智能体共同使用

- **39:00** - 业务流程分类器:
  - 将查询路由到正确的子智能体
  - 每个业务流程都有对应的子智能体
  - RAG 智能体成为执行专门流程的特殊子智能体之一

- **40:00** - 第三层:主动问题解决
  - 利用用户信号和平台活跃事件
  - AI 智能体在用户明确询问之前预测常见问题
  - 主动解决层是另一个 ReAct 智能体
  - 如果无法找到主动解决方案,则回退到业务流程分类器层
  - 通过标准化 MCP 服务器访问所有数据

### 聊天机器人设计的核心原则 (42:00 - 45:00)
- **42:00** - 模型选择:在准确性、延迟、可扩展性之间取得平衡,并随着模型能力增长持续重新评估
- **43:00** - 工具标准化:通过模型上下文协议(MCP)标准化所有工具访问,为多个领域奠定坚实基础
- **43:30** - 业务适应性:业务流程的单一事实来源同时支持人类和 AI,提供所需的业务适应性
- **44:00** - 响应准确性:确保机器人响应的事实正确性是重要因素

### 质量监控与评估 (45:00 - 46:30)
- **45:00** - 使用 LLM 作为评判者的评估方法
- **45:30** - 每个聊天机器人响应都会评估:相关性、准确性、潜在偏见、幻觉等
- **46:00** - 主动跟踪质量指标和监控趋势,快速发现异常并及时介入

### Agent Assist 工具 (46:30 - 50:00)
- **46:30** - AI 驱动的 Agent Assist 工具帮助人工客服处理复杂问题
- **47:30** - 功能特点:
  - 实时协助诊断和缓解客户问题
  - 从账户信号、事件数据、历史工单等多源获取信息
  - 提供个性化指导
  - 生成多语言精确响应(支持 100 多个地区)
- **49:00** - 结果:更快的解决速度、更满意的客户、支持团队效率大幅提升

### 客户支持的业务影响 (50:00 - 52:30)
- **50:00** - 65% 的客户联系由 AI 系统自动处理
- **50:30** - 每年节省近 500 万员工工时,显著降低成本并提升生产力
- **51:00** - 自动化案例在**单次交互中解决**,用时不到 10 分钟(人工处理可能需要 40 分钟)
- **52:00** - 这些结果展示了 AI 在规模化运营时的强大能力,对业务和客户都产生了重大影响

### 合规领域的 AI 应用 (52:30 - 58:00)
- **52:30** - 合规术语介绍:
  - AML(反洗钱)
  - CFT(反恐怖融资)
  - ABC(反贿赂和腐败)
- **53:30** - Coinbase 实施的合规流程:
  - KYC(了解你的客户)
  - KYB(了解你的业务)
  - TMS(交易监控系统)

- **54:30** - 合规挑战:
  - 人力密集且难以随市场波动快速扩展
  - 监管机构要求彻底调查和完全可解释性
  - 需要适应不同国家的多样化监管要求

- **56:00** - 合规 AI 策略:
  - **广度**:利用相同的内部生成式 AI 平台,通过 MCP 接口访问 LLM 和数据
  - **深度**:构建传统机器学习模型检测高风险合规案例,使用基于 Ray 框架的 Anyscale ML 平台
  - 使用 AI 自动化和加速复杂调查
  - 在 KYC、KYB、TMS 等所有合规工作流程中应用严格的调查方法

### 合规辅助工具 (58:00 - 61:00)
- **58:00** - Compliance Assist 工具展示:类似于客户支持的 Agent Assist,但专为合规人员设计
- **59:00** - AI 系统自动从多个来源收集和综合信息:
  - 内部系统数据
  - 开源情报
- **60:00** - AI 生成叙述性摘要和清晰的风险审查信号
- **60:30** - 合规人员获得完全记录、可解释的调查结果,能够快速、自信地做出明智决策
- **61:00** - 这是 AI 赋能团队并提升合规运营标准的典范

### 合规自动解决引擎架构 (61:00 - 结束)
- **61:00** - 流程从合规风险模型(深度学习模型)触发高风险案例警报开始
- **61:30** - CAR(合规自动解决引擎)作为整个全面审查流程的协调器
- **62:00** - 智能体驱动的工作流程同时调动自动化和人类专业知识
- **62:30** - 协调流程整合多个数据源和智能体,执行全面的合规调查

会议总结:本次会议展示了 Coinbase 如何利用 AWS 的生成式 AI 服务(特别是 Amazon Bedrock 和 Agent Core)构建生产级智能体系统,在客户支持领域实现 65% 的自动化率和每年 500 万小时的工时节省,同时在合规领域通过 AI 辅助工具大幅提升调查效率和决策质量。Coinbase 的成功经验证明了采用模块化、标准化(MCP)和迭代式方法构建企业级 AI 解决方案的有效性。
