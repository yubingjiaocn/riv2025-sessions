# AWS re:Invent 2025 会议总结：Iberdrola 使用 Amazon Bedrock 和 Agent Core 构建企业级 AI 代理

## 一、会议概述

本次 AWS re:Invent 2025 分会场由 Iberdrola（西班牙最大的能源公司）和 AWS 团队共同呈现，重点介绍了如何在全球范围内部署和扩展 Agentic AI 解决方案。

Iberdrola 作为欧洲最大、全球第二大能源公司，服务超过 1 亿客户，资产基础达 1600 亿欧元。面对能源行业电气化增长、可持续能源转型等挑战，该公司将 AI 置于数字化转型的核心位置。他们不仅仅是增量式地优化特定用例，而是采用全面转型的方法，利用 AI 重新设计整个业务流程。

Iberdrola 构建了基于 Amazon Bedrock Agent Core 的最佳实践平台和 Agentic 框架，在英国、美国、巴西和西班牙四个核心市场部署了区域化的 AI 代理解决方案。通过建立卓越中心（Hub-and-Spoke 模式）、标准化交付方法和统一平台，他们成功地在 IT 运营领域实现了三个初始用例，并计划向更复杂的代理系统和代理群（swarms）演进。

该公司强调了三个关键成功因素：1）建立跨组织的能力中心，标准化交付方法；2）构建包含安全、可观测性、监控和审计的完整平台；3）选择 AWS 作为技术合作伙伴，利用其 AI 服务栈实现快速创新。

## 二、详细时间线与关键要点

### **00:00 - 开场介绍**
- Sergio Merchan（Iberdrola 全球 CIO）、Unai（全球专家工程师）和 Jose（AWS 能源行业技术团队负责人）介绍演讲主题
- 会议将涵盖 AWS Agentic AI 解决方案概述、Iberdrola 的 AI 战略、具体用例和经验教训

### **02:30 - Agentic AI 时代概述**
- Jose 介绍 Agentic AI 的定义：基于代理的自主系统，能够像人类一样规划、推理和适应
- Agentic AI 将影响每个业务流程、客户交互、产品和服务
- 企业正从简单任务自动化向复杂工作流和代理群演进

### **04:45 - Agentic AI 的价值主张**
- 生产力提升 10 倍，原本需要数年的项目现在只需数月甚至数周
- 成本从数百万降至数千美元
- Gartner 预测：到 2028 年，33% 的应用将包含 Agentic AI，15% 的企业决策将通过代理自动化

### **06:20 - AWS AI 技术栈介绍**
- 三层架构：应用层（如 Amazon Q、CodeWhisperer）、服务层（Amazon Bedrock）、基础设施层
- Amazon Bedrock Agent Core：框架无关的运行时、内存管理、可观测性、工具网关等核心能力
- 客户越来越多地进行模型微调和训练，AWS 推出 Forge 服务支持自定义模型训练

### **09:15 - Iberdrola 公司介绍**
- Sergio 介绍 Iberdrola：欧洲最大能源公司，市值 1200 亿欧元
- 服务超过 1 亿客户，资产基础 1600 亿欧元，过去 20 年投资 1500 亿欧元
- 业务覆盖发电、输电、配电和零售全产业链，在西班牙、英国、美国、巴西等国家运营

### **11:30 - 能源行业面临的挑战**
- 电气化增长带来的能源需求压力
- 需要创造更可持续、更清洁的能源
- 必须在价值链的每个环节实现卓越运营

### **13:00 - Iberdrola 的 AI 愿景**
- 将 AI 置于核心位置，实现全面转型而非增量改进
- 重新设计和重新思考整个业务流程，而不仅仅是优化特定用例
- 在发电、运营、零售、企业职能等全价值链部署 AI 能力

### **15:45 - AI 应用案例**
- 发电：使用 AI 优化风力涡轮机位置，提高能源生产效率
- 运营：为现场工作人员提供 AI 助手，加快检查和维护，减少停机时间
- 零售：在呼叫中心部署 AI 代理，帮助客服人员更快、更高质量地回答客户问题
- 企业职能：在法务、内审、财务等部门部署 AI 代理

### **17:30 - 三大成功因素**
1. 能力中心：采用 Hub-and-Spoke 模式，标准化交付方法，整合架构师、工程师、项目经理、运营和安全团队
2. 平台：基于 AWS 构建包含安全、可观测性、监控、审计和日志的完整框架
3. AWS 合作伙伴关系：利用 AWS AI 服务快速创新

### **21:00 - 全球部署架构**
- Unai 介绍 Iberdrola 的全球部署策略
- 在四个核心市场（英国、美国、巴西、西班牙）部署 Amazon Bedrock Agent Core
- 确保数据主权，代理保留在所属区域，符合当地法规

### **23:15 - 多账户策略**
- 三个环境：生产、预生产、开发
- 按业务垂直划分：零售、网络、企业、可再生能源、IT
- 实现三个目标：
  1. 每个区域独立扩展
  2. 按设计实现业务隔离
  3. 同一业务内实现可重用性和发现性

### **26:00 - IT 运营的三个初始用例**
1. 变更请求支持：在 ServiceNow 中自动验证和改进变更请求，减少因不合规导致的拒绝和延迟
2. 网络事件丰富化：自动标记、分配和加速事件解决
3. 变更模板选择聊天机器人：帮助用户从 200 多个模板中选择正确的变更模板

### **28:30 - 技术架构详解**
- 每个用例使用独立的 Agent Core Runtime，解耦成本、开发生命周期和扩展
- Runtime 运行 LangGraph 代理，通过 LiteLLM 调用 Bedrock 模型
- 使用 Guardrails 确保模型输出符合预期
- 部署 MCP 服务器访问 RDS PostgreSQL 数据库
- 使用 Core Gateway 与 ServiceNow 集成

### **32:00 - 用例 1：变更请求辅助详解**
- 变更请求经历六个阶段，从草稿到关闭
- 在草稿阶段部署代理群，包括：
  1. 规则结构器：确定需要检查的字段和内容
  2. 内容验证器：验证字段和内容
  3. 变更模型分析器：验证变更模板选择
  4. 修复建议器：自动提出修复建议或转换阶段

### **35:45 - 技术流程**
- ServiceNow 调用 Micro Gateway
- Micro Gateway 路由请求到相应的 Runtime
- Runtime 执行 LangGraph 代理，使用 Claude 模型和 RDS MCP 服务器
- ETL 流程：将 ServiceNow 模板和变更模型加载到 S3，通过 Kubernetes Pod 构建黄金层数据
- 使用 PostgreSQL pgvector 进行向量化，版本化提示词和嵌入

### **39:00 - 用例 2：变更模板选择器**
- 更简单的架构，单一代理
- 通过聊天式交互帮助用户选择正确的变更模板
- 相同的技术栈和 ETL 流程

### **41:00 - 用例 3：网络事件丰富化**
- 最复杂的用例，使用代理群（swarm）
- 编排器根据事件信息决定触发哪些专业代理
- 包括标签分类器、事件相似性分析等多个专业代理
- 使用 Claude 3.5 Sonnet 模型

### **44:00 - 为什么选择 Agent Core Runtime**
1. 框架无关：支持 LangGraph、LangChain、Google ADK 等任何框架
2. 成本效益：仅按实际 CPU 消耗计费，代理等待时（30-70% 的时间）不计费
3. 会话持久化：支持暂停会话，实现有状态代理和人机协作
4. 低延迟：无冷启动，约 200 毫秒响应时间
5. 自动扩展：无需管理基础设施
6. 托管服务：内置内存管理、工具网关等功能

### **47:30 - 代码示例**
- 展示 LangGraph 编排器代码片段
- 使用规划器节点决定运行哪些代理
- 顺序执行代理（也可并行化）
- 组装结果节点汇总输出

### **49:00 - 未来挑战与成熟度提升**
- 面临的挑战：
  1. 代理无法重用
  2. 未能大局思考
  3. 重复开发导致成本增加
  4. 业务逻辑重复导致不可预测行为
  5. 真实数据源重复

### **51:00 - 代理重用的三种场景**
1. 原所有者保留所有权：通过网络和角色授权新用户使用
2. 需要调整的重用：需要额外工具或数据访问
3. 完全转移所有权：新团队接管代理的维护和演进

### **52:30 - 未来展望**
- 从当前的简单用例向更复杂的代理系统演进
- 构建可重用、可发现的代理生态系统
- 利用 Amazon Bedrock Agent Core 的能力实现代理的标准化和规模化部署
- 继续与 AWS、Accenture 等合作伙伴协作，推动平台成熟度提升

### **54:00 - 总结**
- Iberdrola 成功展示了如何在全球范围内部署企业级 Agentic AI 解决方案
- 通过标准化平台、能力中心和 AWS 技术栈实现快速创新
- 初始用例已在 IT 运营中交付价值，为未来更复杂的应用奠定基础
