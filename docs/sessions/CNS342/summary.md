# AWS re:Invent 2025 ECS 会议总结

## 会议概述

本次会议由 AWS ECS 工程总监 Maslaner、产品经理 Alexander 以及来自 QRT（Cube Research and Technologies）的 Ruben 共同主讲。会议深入探讨了 Amazon ECS（Elastic Container Service）的核心功能、最新特性以及实际应用案例。

会议首先介绍了 ECS 作为 AWS 原生容器编排服务的核心价值：无需管理控制平面，专注于应用程序开发。ECS 目前每周在全球范围内启动约 30 亿个任务，65% 的新容器化工作负载选择使用 ECS，其中绝大多数选择 Fargate。会议重点介绍了三种计算选项：Fargate（完全托管）、EC2（完全控制）以及新推出的 ECS Managed Instances（平衡托管与控制）。

在安全合规方面，会议强调了 AWS 共享责任模型的演进，Fargate 和 Managed Instances 大幅减少了客户的运维负担。成本优化策略包括选择合适的硬件（Intel/AMD/Graviton）、利用 Savings Plans 和 Spot 实例，以及通过自动化容量管理降低成本。最后，QRT 分享了他们从本地基础设施迁移到 ECS Fargate 的大规模实践经验，展示了 ECS 在金融量化交易场景下的高性能表现。

## 详细时间线与关键要点

### 开场介绍 (00:00 - 02:30)
- **00:00** - Maslaner（ECS 工程总监）介绍自己及团队成员
- **00:30** - Alexander（ECS Fargate 产品经理）自我介绍
- **01:00** - Ruben（QRT 量化技术专家）登场
- **01:30** - 会议议程概览：ECS 基础、容量配置、安全合规、成本优化、客户案例

### ECS 核心概念 (02:30 - 08:00)
- **02:30** - ECS 定义：为 AWS 构建的容器编排器，承担尽可能多的运维工作
- **03:00** - ECS 四大支柱：大规模启动容器、按使用付费、提升速度和敏捷性、体现 AWS 最佳实践
- **04:30** - ECS 规模数据：每周启动约 30 亿个任务
- **05:00** - 65% 的新容器化客户选择 ECS，大多数使用 Fargate
- **06:00** - Amazon Prime Day 案例：每天在 Fargate 上启动 1800 万个任务
- **07:00** - 客户选择 ECS 的原因：简单性、安全性、效率

### 容量配置详解 (08:00 - 20:00)
- **08:00** - AWS 计算选项概览：EC2、Fargate、Lambda 等
- **09:00** - Fargate 特性：完全托管数据平面，只需指定 vCPU 和内存
- **10:00** - Fargate 任务隔离：每个任务运行在独立 EC2 实例上，实例不重用
- **11:00** - Fargate 计费：仅在任务运行时收费
- **12:00** - EC2 选项：完全控制实例类型和大小，但需管理 AMI、补丁和配置
- **13:00** - EC2 支持预留实例、按需容量预留、Savings Plans
- **14:00** - ECS Managed Instances 发布（两个月前）：平衡托管与控制
- **15:00** - Managed Instances 在客户账户中启动 EC2 实例，但完全由 AWS 管理
- **16:00** - 每两周自动轮换实例以满足补丁 SLA
- **17:00** - 支持自定义实例类型选择（GPU、Graviton、网络优化等）
- **18:00** - 更快的容量扩展和自动成本优化（整合任务、关闭空闲实例）
- **19:00** - 支持特权容器和 eBPF（Fargate 不支持）

### 容量提供者 (20:00 - 26:00)
- **20:00** - 容量提供者概念（2019 年引入）
- **21:00** - Fargate 容量提供者：按需和 Spot 两种
- **22:00** - Managed Instances 容量提供者：与 ECS 任务生命周期紧密集成
- **23:00** - 使用 Bottlerocket 作为容器操作系统
- **24:00** - 支持 EC2 事件窗口控制实例替换时间
- **25:00** - 基于属性的实例类型选择（ABIS）：按 GPU、vCPU、网络等属性选择

### 使用场景建议 (26:00 - 28:00)
- **26:00** - 建议 EC2 用户迁移到 Managed Instances
- **27:00** - Fargate 仍是默认推荐，适合大多数客户
- **27:30** - Managed Instances 适用场景：48 vCPU 以上任务、需要 eBPF、GPU 工作负载

### 安全与合规 (28:00 - 38:00)
- **28:00** - Alexander 接手讲解安全合规主题
- **28:30** - 共享责任模型对比：自管理 EC2 vs Fargate/Managed Instances
- **29:30** - 自管理 EC2：客户负责容量配置、实例选择、AMI、补丁等
- **30:30** - Fargate/Managed Instances：AWS 承担更多责任，客户专注应用
- **31:30** - 隔离边界差异：
  - Fargate：每个任务独立 EC2 实例，强隔离
  - Managed Instances：多任务共享实例，通过 bin packing 优化利用率
- **33:00** - Fargate 安全最佳实践：任务隔离、减少攻击面、定期补丁、VPC 网络模式
- **34:30** - VPC 网络模式优势：每个任务独立 IP、支持安全组、VPC Flow Logs
- **35:30** - Managed Instances 安全特性：完全托管、Bottlerocket OS、每两周补丁
- **36:30** - 客户可见但无法修改 Managed Instances
- **37:00** - 支持事件窗口控制补丁时间
- **37:30** - 合规认证：Fargate 和 Managed Instances 帮助更快达到合规要求

### 成本优化策略 (38:00 - 46:00)
- **38:00** - 成本优化四大策略：硬件选择、购买选项、扩展优化、可观测性
- **39:00** - 硬件选择：Intel、AMD、Graviton（ARM）
- **40:00** - Fargate 支持 x86 和 ARM，Managed Instances 可精确选择 CPU 类型
- **41:00** - 购买选项：
  - 按需：适合突发工作负载，最贵
  - Savings Plans：适合可预测工作负载，显著折扣
  - Spot：容错应用可节省 80%，但容量可能被回收
- **42:00** - Spot 暂不支持 Managed Instances
- **42:30** - Fargate 成本优化：
  - 100% 利用率（精确配置）
  - Seekable OCI 延迟加载加速启动
  - Compute Optimizer 提供右侧调整建议
- **44:00** - Managed Instances 成本优化：
  - 主动 bin packing 和任务整合
  - 关闭空闲实例
  - 容器镜像缓存加速启动
- **45:00** - 使用场景建议：
  - Fargate：大多数工作负载的首选
  - Managed Instances：需要 eBPF、短生命周期任务（<2 分钟）、已有预留实例/ODCR
