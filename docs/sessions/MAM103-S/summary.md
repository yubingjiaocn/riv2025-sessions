# AWS re:Invent 2025 技术会议总结

## 会议概述

本次会议由Pega Systems的Ravesh主讲，主题为"AI驱动的应用现代化"。会议重点讨论了如何利用人工智能技术来解决传统遗留系统向云端迁移的挑战。演讲者指出，目前仍有85%的工作负载尚未迁移到云端，这些主要是30-50年前开发的复杂应用系统，包括COBOL、AS400等主机应用。

Pega公司展示了其Blueprint技术平台，该平台专注于遗留应用的重写问题，而非仅仅分析问题。演讲包含了两个实时演示：一个基于需求文档的应用转换，另一个展示了与AWS Transform集成的COBOL应用现代化过程。会议强调了"从任何系统到Blueprint再到云端"的转换理念。

## 详细时间线与关键要点

### 0:00-5:00 遗留系统现状与挑战
- 85%的工作负载尚未迁移到云端
- 遗留应用特点：30-50年历史、百万行代码、复杂依赖关系
- 传统现代化方法：人力密集、耗时长、风险高
- McKinsey研究显示AI可压缩50%的IT生命周期时间
- 最大压缩发生在设计和发现阶段

### 5:00-10:00 Pega的差异化方法
- 市场上大多数工具专注于分析问题
- Pega专注于重写问题，提供完整价值链：分析-重写-集成-部署
- 三个转换维度：
  - 从旧架构到云原生架构
  - 从编程语言到无代码平台
  - 从模拟流程到AI代理自动化

### 10:00-15:00 Blueprint技术演示
- Matt Healy演示Blueprint平台功能
- 可处理多种输入：视频、文档、需求规格
- AI代理自动提取业务流程、数据模型和相关人员
- 支持协作式业务流程重新设计
- 可导入SQL数据库结构和API规范

### 15:00-20:00 云端部署与AI代理
- 5分钟内生成云原生应用
- 多渠道支持：后台门户、移动应用、Salesforce集成
- 预测性AI代理：自动包装业务流程为对话式界面
- 演示理赔申请的对话式处理流程
- 与AWS Transform集成处理主机应用

### 20:00-21:20 总结与展望
- 主机应用现代化可在两周内完成
- 邀请参观展台366进行深入了解
- 强调"从任何系统到Blueprint再到云端"的核心理念
