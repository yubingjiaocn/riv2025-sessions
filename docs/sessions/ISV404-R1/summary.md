# AWS re:Invent 2025 会话总结：ISV 404 - 构建多账户弹性测试模式

## 会话概述

本次会话由AWS高级解决方案架构师Kane Vijay主讲，重点介绍了如何使用AWS故障注入服务(FIS)为多租户SaaS架构构建弹性测试模式。会话探讨了领先的ISV如何通过受控实验验证其SaaS架构设计，确保在不影响关键组件的情况下进行故障注入测试，从而避免客户可用性问题并满足SLA承诺。

演讲者详细介绍了SaaS架构的六个关键支柱：强大的租户隔离、噪声邻居缓解、全面的身份访问管理、租户感知可观测性、战略分层和成本感知跟踪机制。会话还展示了AWS弹性生命周期框架的五个阶段，以及如何通过实际的多账户参考架构实现弹性测试模式。
