# AWS re:Invent 2025 分会场总结

## 会议概述

本次会议聚焦于AI时代的客户成功策略,探讨如何通过系统化的客户成功方法来弥合AI投资与实际业务价值之间的差距。演讲者指出,尽管AI正在创造前所未有的商业机会,客户投资数十亿美元,但AI也在瓦解传统的竞争优势。小型企业正在成为颠覆性威胁,合作伙伴能够在短时间内建立强大能力。在这个竞争日益激烈的环境中,真正的差异化在于为客户创造可衡量的、切实的业务成果。

会议重点介绍了AWS与合作伙伴共同开发的客户成功框架,该框架系统性地追踪和交付价值。研究数据显示,80%的企业正在部署AI(这一比例在不到一年内几乎翻倍),预计到2030年仅代理AI的支出就将达到1550亿美元。然而,四分之三投资AI的公司未能看到切实的业务成果,预计到2027年将有40%的AI试点项目被取消。解决方案是采用结构化的客户成功方法:当合作伙伴团队和AWS团队共同制定业务案例、明确前期业务价值,并采取系统化方法推动采用和价值实现时,试点项目转化为生产环境的成功率提高了2倍。

演讲者Cynthia随后分享了合作伙伴主导的客户成功试点项目的学习成果,并概述了AWS计划在2025年第一季度推出的价值实现工具包,以及未来将通过自动化遥测数据和行业特定资产来支持合作伙伴的计划。

04:30 - 客户成功作为解决方案
- 当合作伙伴团队和AWS团队共同制定业务案例并采取结构化方法时,效果显著
- 试点项目转化为生产环境的成功率提高2倍
- 客户成功是弥合AI投资价值差距的关键解决方案

05:15 - 行业研究数据支持
- IDC研究显示,客户成功活动是决定合作伙伴未来成功的首要因素
- 排名高于AI赋能转型、行业特定能力和联合销售活动
- 70%的合作伙伴看到部署AI解决方案的巨大潜力
- 合作伙伴需要帮助的领域:提供可证明的ROI路径和在交付过程中证明价值

06:30 - 客户成功的商业价值
- 95%的组织会因为积极的客户成功体验而购买额外的产品和服务
- 客户成功不仅关乎留存,更是可持续增长的驱动力
- 能够帮助赢得更大、更长期的交易并提升收入

07:15 - 启动后客户旅程框架
- **销售与客户成功的清晰交接**:记录用例和明确的30-60-90天计划
- **激活阶段(Activate)**:寻找快速成功案例建立信心,同时奠定坚实的技术基础,启动第一个AI应用和自动化任务
- **消费与采用阶段(Consume & Adopt)**:不仅衡量功能使用和用户采用率,更要追踪切实的业务价值(开发团队生产力、支持团队解决的工单数量、客户满意度改善)
- **规模化阶段(Scale)**:将有效的方案扩展到整个企业

09:00 - 客户案例1: Jima(巴西零售与供应链公司)
- **背景**:为巴西最大500家企业中的300家处理物流和产品目录
- **问题**:决策基于直觉而非数据;目录更新手动且耗时;决策过程缓慢;非技术用户无法访问所需数据
- **解决方案**:AWS高级合作伙伴Flexa构建了Flexa AI平台,扩展Amazon Q Business,实现数据访问民主化
- **实施过程**:激活阶段整合多个数据源并提供自然语言查询能力;采用阶段推动深度赋能,60-90%的销售组织使用该工具
- **成果**:决策时间减少95%;决策速度提高2倍;个别决策从数小时缩短至30秒;预计通过再投资生产力收益实现20%的销售增长

11:30 - 客户案例2: Media Corp(新加坡最大媒体集团之一)
- **背景**:处理500万张图片、15万小时视频、1.6万小时音频和数百万文本文件
- **问题**:不了解AI处理管道的运行情况;内容创建缓慢;内容质量不一致;编辑团队无法按观众需求的速度生产内容;观众难以找到所需内容
- **解决方案**:AWS亚洲顶级合作伙伴Crayon实施CloudWatch,全面端到端地监控AI管道
- **实施过程**:启用跟踪、日志、指标、实时仪表板和健康检查;为不同仪表板和用户提供深度赋能培训
- **成果**:内容生产时间缩短6倍(从20秒降至3秒);AI准确率从75%提升至85%,预计将超过90%;基础设施成本降低85%;保持99%的正常运行时间

13:45 - 客户案例3: Tower Insurance(新Zealand)
- **背景**:约630名客服代表每天接听数千个客户电话,使用老旧的联络中心解决方案
- **问题**:客户电话响应过慢;客户体验不佳;客服代表体验也令人沮丧,无法找到所需信息;响应质量缓慢且不一致;运营效率低下
- **解决方案**:Deloitte与Tower合作部署Amazon Connect和五种不同的AI能力(电子邮件自动化、常规请求的自助服务机器人、Amazon Q帮助客服代表快速找到答案)
- **实施过程**:对客服代表和主管进行深度赋能培训;重新设计日常工作流程;提供变更管理和赋能培训;实现60%的员工采用率
- **成果**:平均处理时间从18分钟以上降至15分钟以上(对630名客服代表和每天数千通电话而言,节省显著);为Tower提供了AI转型的基础能力

17:00 - 试点项目五大调查领域(Cynthia分享)

17:30 - 领域1: 阶段进展衡量
- **发现**:许多团队仍停留在简单的云运营指标测量阶段
- **挑战**:转向业务指标、ROI指标、潜在收入生成等需要重大思维转变
- **解决方案**:AWS正在制定行业基准,根据行业、用例和领域提供正确的指标类型,作为与客户制定业务计划的起点

18:30 - 领域2: 各阶段所需工作量
- **发现**:将其放在幻灯片上比现实简单得多,没有一个简单的答案
- **原因1**:不同合作伙伴在客户成功和业务价值实现方面的成熟度不同
- **原因2**:所需工作因客户而异(例如:Flexa让Jima员工在家用消费级聊天机器人帮助孩子做作业以理解AI;而对于开发人员可以直接进行黑客马拉松)
- **解决方案**:创建框架和详细场景建议,作为赋能计划的一部分

20:00 - 领域3: 商业模式
- **现状**:客户开始关注基于成果的定价,但合作伙伴CFO对缺乏可预测性的模式不感兴趣
- **解决方案**:开发包含基准和最佳实践的遥测数据,提供可预测性(例如:在Connect场景中降低平均处理时间,可以在x个月内承诺20%的改善)

21:00 - 领域4: 框架、手册和工具
- **发现**:框架语言和每个阶段的手册对对话非常有帮助
- **最重要工具**:价值实现计划(业务计划),允许合作伙伴、客户的业务人员和技术人员共同确定目标和衡量方法
- **计划**:价值实现计划将成为任何项目的核心

22:00 - 领域5: 治理
- **预期vs实际**:原以为合作伙伴希望轻量级治理,但实际上希望将其纳入业务节奏(RoB)
- **要求**:定期重新审视价值实现以确保进度;与AWS保持更新
- **需求**:明确角色和职责(AWS、合作伙伴和客户的不同人员)
- **行动**:从PDM(合作伙伴开发经理)开始制定内部角色和职责,确保统一语言

23:30 - 后续计划
- **Q1 2025**:在Partner Central发布价值实现工具包
- **后续阶段**:收集采用模式和最佳实践信息(通过合作伙伴和直接服务团队),先手动分享后自动化为遥测数据
- **未来扩展**:开发领域级资产,可能在明年扩展到行业和用例级资产
