# AWS re:Invent 2025 分组会议总结：代理式AI在数据科学与分析中的应用

## 会议概述

本次会议重点探讨了代理式AI（Agentic AI）在数据科学和分析领域的实际应用。演讲者分享了在多家财富500强企业中部署的四个生产级应用案例，这些案例展示了经典AI、数据科学和生成式AI如何融合成代理式AI系统。演讲者提出了一个核心论点：过去15年建立的大多数决策引擎将在未来几年内从代理式AI的角度重新构想，这些系统将是多模型、多代理的企业工作流系统。

过去一年中，五个关键技术发展加速了这一趋势：推理能力的提升（能够执行复杂的多步骤分析）、代理式工作流的主流化、通过知识图谱和本体论实现的领域融合、MCP服务器和评估系统带来的确定性性能，以及微调模型的应用。这些技术与合规安全层、上下文工程和LLM运维相结合，使得企业级代理应用能够成功部署到生产环境中。演讲者特别强调，这些应用已经在美国和亚太地区（包括日本）的多个财富500强公司投入使用，并产生了实际业务影响。

根据Forrester最近对400个分析组织的调查显示，大多数决策者预期人机协作的代理系统将在未来一年内上线，并在24个月内达到自主水平。他们期望代理式分析能够自动化日常分析工作，将洞察生成时间和成本降低至当前的一半。演讲者预测，未来2-3年将主要是人机协作系统，之后将逐步过渡到更自主的AI分析师系统。

## 详细时间线与关键要点

### 开场与核心论点 (0:00 - 2:30)
- **0:00** - 会议开始，介绍主题：代理式AI在数据科学和分析中的应用
- **0:15** - 提出核心论点：经典AI、数据科学和生成式AI正在融合成代理式AI
- **0:30** - 预测过去15年建立的决策引擎将被重新构想为多模型、多代理系统
- **1:00** - 介绍过去12个月加速这一趋势的五个关键发展

### 技术发展的五大支柱 (2:30 - 5:00)
- **2:30** - 推理能力：能够执行复杂的多步骤分析，识别二阶和三阶问题
- **3:00** - 代理式工作流：从去年的小众应用发展到今年的主流技术
- **3:30** - 领域融合：通过知识图谱和本体论将领域知识和企业特定信息注入代理
- **4:00** - MCP服务器和评估：提供确定性性能，减少LLM幻觉
- **4:30** - 微调模型：大模型触发小型微调模型来回答特定问题

### 企业部署的关键要素 (5:00 - 7:00)
- **5:00** - 强调合规和安全层的重要性，企业法务和合规部门需要批准代理应用
- **5:30** - 医疗保健案例：销售代表、医疗服务联络员和总部都在使用，合规团队配置可控的护栏
- **6:00** - 上下文工程和LLM运维的重要性
- **6:30** - 遥测系统帮助驱动代理应用的性能

### 案例一：按需客户分析 (7:00 - 10:00)
- **7:00** - 介绍四个应用案例，复杂度递增
- **7:30** - 按需客户分析：应用能够思考执行步骤，包括问题是否被批准回答
- **8:00** - 生成完整的可视化图表，而非两年前的简单Python图表
- **8:30** - 集成到CRM系统（Salesforce和医疗保健领域的Viva）
- **9:00** - 结合结构化和非结构化数据进行洞察分析
- **9:30** - 已在美国和亚太地区（日本等）部署

### 多语言和合规挑战 (10:00 - 11:30)
- **10:00** - 日本部署的特殊考虑：语言、文化背景和业务行为差异
- **10:30** - 日语的正式性挑战：LLM的回答风格需要适应当地习惯
- **11:00** - 合规部门监控：过去6-9个月内约10万个问题被提问
- **11:15** - 合规团队需要追踪答案，持续监控潜在问题

### 案例二：营销自动化 (11:30 - 14:00)
- **11:30** - 营销应用：部署在美国第二或第三大房地产公司
- **12:00** - 扫描新信息，生成新的潜在客户线索
- **12:30** - 自动化上下文丰富：通过Apollo或Zoom Info丰富线索信息
- **13:00** - 学习高管的邮件写作风格，起草个性化回复
- **13:30** - 持续学习潜在客户行为，生成包含图像和文本的定制化响应
- **13:45** - 显著提高响应率，大规模扩展营销线索生成

### 案例三：市场细分 (14:00 - 17:00)
- **14:00** - 市场细分：端到端执行完整的细分分析
- **14:30** - 基于最新信息（销售下降或新兴细分市场）进行分析
- **15:00** - 规划引擎分析所有数据，创建新的用户画像
- **15:30** - 推理驱动因素和产品线与细分市场的共鸣
- **16:00** - 使用多模型的工作流设计，由代理触发
- **16:30** - 过去需要3-5人团队工作3-4个月，现在1-2天内完成
- **16:45** - 可跨AWS数据仓库、CRM等多个数据集执行

### 案例四：预测分析 (17:00 - 20:30)
- **17:00** - 预测分析：最复杂和最有争议的应用
- **17:30** - 处理突发事件：价格变动、产品缺货等场景
- **18:00** - 审查上季度预测，查看实际数据，整合多方意见
- **18:30** - 验证假设准确性，必要时进行平滑处理
- **19:00** - 通过知识图谱实现领域融合，捕获预测团队的所有步骤
- **19:30** - 端到端运行场景分析，评估风险和机会
- **20:00** - 人机协作：团队监督代理算法，调整不同步骤
- **20:15** - 提供完全的控制和透明度，这是预测分析的关键需求

### 行业演变与历史视角 (20:30 - 23:00)
- **20:30** - 分析和数据科学处于代理式AI的拐点
- **21:00** - 演讲者的20年行业经验回顾
- **21:15** - 2010年前：使用SAS进行逻辑回归和线性回归
- **21:30** - 2011-2012：第一个拐点 - 大数据集群出现，访问更大规模数据
- **21:45** - 2016-2017：第二个拐点 - 云计算和机器学习兴起
- **22:00** - 未来几年：第三个拐点 - 过去15-20年的决策引擎将从代理式AI角度重做
- **22:30** - 这不仅是个人观点，而是行业趋势

- **23:00** - Forrester对400个分析组织的调查
- **23:15** - 受访者均为决策者或分析部门负责人
- **23:30** - 绝大多数预期人机协作代理将在未来一年内上线
- **24:00** - 许多组织预期在24个月内达到自主水平
- **24:30** - 预期代理式分析在未来两年发挥重要作用
- **25:00** - 自动化日常分析工作，大幅提升洞察生成速度
- **25:15** - 约半数受访者期望时间和成本降低至当前的一半
