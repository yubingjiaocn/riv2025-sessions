# STG 208: 利用Amazon S3 Glacier存储类最大化冷数据价值

## 会议概述

本次AWS re:Invent 2025分组会议由Gayla和Nitish主讲，重点介绍了Amazon S3 Glacier存储类及其在冷数据管理中的价值。会议面向Glacier新用户、偶尔使用者以及存储专家，全面覆盖了从基础概念到高级功能的内容。

演讲者强调了冷数据的重要性正在不断增长，70-80%的数据属于冷数据，这些数据不再仅仅是存储后遗忘，而是成为创新的催化剂和业务差异化因素。通过机器学习、历史分析和合规性应用，冷数据正在转化为可操作的智能资源。会议详细介绍了S3 Glacier的三个存储类别、数据迁移策略、恢复选项，以及两个新功能：计算校验和操作和S3元数据服务。

## 详细时间线与关键要点

### 0:00-5:00 会议开场与冷数据重要性
- Gayla介绍会议结构：三个部分覆盖Glacier基础、新功能和演示
- S3当前存储数百万亿个对象，70-80%的数据为冷数据
- 冷数据价值正在增长，从存储转向创新催化剂
- 应用场景：手写记录机器学习训练、历史分析、金融数据应用开发

### 5:00-15:00 S3 Glacier存储类详解
- 存储类连续体概念：平衡访问速度与成本效率
- **Glacier Instant Retrieval**: 毫秒级访问的冷数据，适用于医疗记录、媒体片段、合规要求
- **Glacier Flexible Retrieval**: 分钟级访问，提供免费批量检索，适用于大规模数据分析、备份归档
- **Glacier Deep Archive**: 最低成本存储，适用于长期保留（年/十年级别）
- S3生命周期策略：自动化数据转换，基于访问模式优化成本
- S3 Intelligent-Tiering：针对不可预测访问模式的自动优化

### 15:00-25:00 数据恢复与访问模式
- 客户恢复数据的新兴模式：媒体内容重新利用、合规性战略价值、机器学习训练
- **Glacier Instant Retrieval**: 标准GET请求，毫秒级访问
- **Glacier Flexible Retrieval和Deep Archive**: 三步流程（发起请求、检查状态、访问数据）
- 性能优化：1000 TPS限制，1000万恢复请求需3小时提交，总计6小时完成
- 事件监控：通过EventBridge、SNS、SQS进行状态跟踪
- S3 Batch Operations：自动最大化TPS，提供重试机制和完成报告

### 25:00-35:00 新功能介绍 - 计算校验和操作
- Nitish介绍数据完整性验证需求：媒体娱乐、生命科学、司法、保存机构
- 传统方法问题：需要下载对象并本地计算，成本高且耗时
- **新的计算校验和操作**：S3 Batch Operations中的新功能，支持原地数据读取和校验和计算
- 支持六种算法：CRC32、CRC32C、CRC64、MD5、SHA-1、SHA-256
- 三个输入组件：对象清单、校验和算法、校验和类型（完整对象或复合）
- 定价：所有存储类统一收费$4/TB

### 35:00-40:00 S3元数据功能增强
- S3元数据自动提取对象元数据，支持SQL查询和自然语言查询
- 两种表类型：日志表（近实时变更）和实时清单表（每小时刷新）
- 查询方式：标准SQL（Athena、Redshift、SageMaker）和自然语言（Amazon Q、Kiro）
- MCP服务器支持：AI助手可直接连接元数据表，支持英语交互
- 应用场景：存储使用分析、未标记对象识别、审计跟踪

### 40:00-45:00 实际演示
- 演示场景：太空技术初创公司使用NASA图像训练自主太空飞行器
- 演示流程：使用清单表查询Project Odin相关对象、创建批处理作业计算校验和、使用Kiro验证结果
- 成功验证25个对象的数据完整性
- 总结要点：AI时代归档数据成为差异化因素、S3提供多样化存储选项、新功能简化归档数据管理并提取更多价值
