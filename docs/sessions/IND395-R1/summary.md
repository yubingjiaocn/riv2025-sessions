# AWS re:Invent 2025 - IND 395 会议总结

## 会议概述

本次会议主题为"使用生成式AI和LLM代理加速游戏设计审查"（IND 395）。会议由AWS解决方案架构师Sam Pzer和Christina主讲，重点介绍了如何利用AI代理技术来改进游戏设计文档的审查流程。

传统的游戏分析师需要从多个维度评估游戏设计文档，包括游戏背景设定（Lore）、质量保证（QA）、游戏玩法设计和商业策略。这个过程通常耗时较长，且容易受到主观偏见的影响。演讲者展示了如何使用AWS的Strands开源框架和Amazon Bedrock Agent Core构建专业化的AI代理系统，通过将复杂的审查任务分解给多个专门的代理来处理，最后由一个协调代理整合所有反馈，从而实现更快速、更客观的设计文档审查。

演示中使用了《New World》游戏作为案例，展示了如何评估在游戏中添加精灵种族这一设计提案。系统通过四个专业代理（Lore、QA、Gameplay、Strategy）分别从不同角度进行分析，并由游戏分析师代理统筹输出最终建议。这种方法不仅加速了创新周期，还提供了协作智能和无偏见的反馈。

### 开场介绍
[00:00 - 02:30]
- 会议开始，确认会场和主题：IND 395 - 使用生成式AI和LLM代理加速游戏设计审查
- 演讲者自我介绍：Sam Pzer和Christina，均为AWS解决方案架构师
- 互动环节：询问现场观众游戏开发和设计文档编写经验

### 游戏分析师角色介绍
[02:30 - 06:00]
- 定义现代游戏分析师的角色：既是解释者也是讲故事的人
- 介绍游戏分析师的四大核心能力：
  - **Lore（背景设定）**：验证更新是否符合游戏世界观
  - **QA（质量保证）**：检查是否破坏游戏平衡
  - **Game Design（游戏设计）**：评估更新是否有趣，能否吸引玩家
  - **Game Strategy（游戏策略）**：判断是否符合商业需求和玩家留存模型

### 当前挑战分析
[06:00 - 08:00]
- 讨论传统审查流程的问题：
  - 迭代周期缓慢，信息不完整导致反复沟通
  - 设计师无法访问分析师拥有的数据源
  - 培养一名合格的游戏分析师需要大量时间
  - 主观反馈和偏见影响审查质量

### 解决方案架构
[08:00 - 10:30]
- 提出构建方案：创建四个专业化代理和一个协调代理
- 每个专业代理连接到特定的知识库（如New World维基）
- 协调代理（游戏分析师代理）整合所有反馈并决定如何呈现
- 方案优势：加速创新周期、协作智能、无偏见反馈

### 工具箱介绍
[10:30 - 15:00]
- **Strands框架**：AWS开源的Python SDK，用于构建AI代理
  - 易于使用、快速原型开发
  - 无缝集成MCP服务器和AWS服务
  - 灵活的模型和工具选择
- **Amazon Bedrock Agent Core**：用于大规模构建、部署和运营AI代理的平台
  - Agent Core Runtime：无服务器托管环境
  - Agent Core Memories：完全托管的记忆基础设施
  - Agent Core Observability：统一的追踪和调试仪表板
- 其他服务：身份管理、代码解释器、浏览器能力、Agent Core Gateway

### 系统架构详解
[15:00 - 20:00]
- 五个代理的分类：
  - 专业代理：Lore、QA、Gameplay、Strategy
  - 协调代理：Game Analyst
- 使用Anthropic Claude Sonnet 4.5模型
- 所有代理部署到独立的Amazon Bedrock Agent Core Runtime
- 数据源：
  - 游戏设计文档和商业策略文档存储在Amazon S3
  - 玩家行为数据存储在Amazon Redshift
- 使用Amazon Bedrock Knowledge Bases提供RAG能力
- 通过AWS Labs Knowledge-based Retrieval MCP服务器标准化数据访问
- 记忆系统：
  - 项目级记忆：语义事实，提高性能
  - 用户级记忆：用户偏好和会话记忆

### 实际演示 - 项目门户
[20:00 - 25:00]
- 展示项目门户界面
- 创建新项目：在《New World》中添加精灵种族
- 配置代理：选择启用Lore、QA、Gameplay三个专业代理（暂不启用Analyst）
- 禁用记忆功能以展示基础性能

### 演示结果分析
[25:00 - 28:00]
- 展示三个代理的独立反馈：
  - **Gameplay代理**：指出引入了前所未有的机制，可能影响PvP平衡和派系系统
  - **Lore代理**：明确指出精灵不符合《New World》的世界观（游戏聚焦中世纪人类）
  - **Strategy代理**：认为从商业角度看这是好的更新
- 展示不同代理之间的冲突：Strategy支持但Lore反对

### 代码实现 - Gameplay代理
[28:00 - 33:00]
- 使用Curo IDE展示代码
- Strands代理的三个核心组件：
  - **Tools（工具）**：自定义工具或MCP服务器
  - **Model（模型）**：Amazon Bedrock模型（Claude Sonnet 4.5）
  - **System Prompt（系统提示）**：定义代理的角色、专业知识和行为
- 展示Gameplay代理的系统提示：定义为游戏机制专家，有权访问知识库
- 使用Bedrock Knowledge-based Retrieval MCP服务器，启用重排序功能
- 通过标签键（inclusion tag key）对资源进行分组

  - 不同的系统提示（定义为策略专家而非游戏玩法专家）

### 记忆功能演示
[47:00 - 52:00]
- 创建启用记忆功能的新项目
- 展示两种记忆类型的效果：
  - **项目级记忆（语义事实）**：代理学习并记住知识库中的关键信息，减少重复查询
  - **用户级记忆**：
    - 用户偏好：记住用户喜欢的输出格式（如项目符号而非段落）
    - 会话记忆：保持对话上下文，支持跨会话引用
- 性能对比：
  - 无记忆：约30秒响应时间
  - 有记忆：约10秒响应时间（提升约3倍）
