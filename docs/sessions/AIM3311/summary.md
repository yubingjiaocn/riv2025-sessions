# AWS re:Invent 2025 - 构建代理工作流程会议总结

## 会议概述

本次AWS re:Invent 2025分组会议主题为"在AWS上使用第三方代理和工具构建代理工作流程"，由AWS Marketplace部署体验产品管理负责人Andrei Okhotnikov主持，全球AI合作伙伴负责人Salman Tahir和Workday Planning AI核心服务负责人Mira Rajaram共同参与。会议重点介绍了AWS代理技术栈的核心服务、架构原则以及客户实践经验，展示了如何利用AWS和合作伙伴生态系统构建代理解决方案。

会议涵盖了从基础理论到实际应用的完整流程，包括AWS AgentCore服务的详细介绍、AWS Marketplace中AI代理和工具的使用方法，以及Workday在构建规划代理方面的实际经验分享。演讲者们通过理论讲解、客户案例和现场演示，为与会者提供了从零开始构建代理工作流程的完整指导。

## 详细时间线与关键要点

### 0:00-5:00 会议开场与背景介绍
- 演讲者介绍和会议议程概述
- 现场互动：调查与会者构建AI代理的经验
- 会议目标：提供AWS代理解决方案的强大基础，帮助参与者准备实际构建代理工作流程

### 5:00-15:00 AWS代理技术栈基础概念
- **代理的核心特征**：具备行动能力、能够接受指令并形成目标、具备推理和规划能力
- **技术演进路径**：从RPA机器人到认知自动化，再到生成式AI助手，最终发展为完全自主系统
- **AWS三大核心关注领域**：
  - 选择性：提供市场领先的能力，支持快速创新
  - 信任：建立控制、治理、护栏和合规机制
  - 规模：提供企业级能力支持生产环境扩展

### 15:00-25:00 AgentCore服务详细介绍
- **AgentCore三大核心领域**：
  - 工具和内存：包括代码解释器和浏览工具
  - 安全规模部署：AgentCore Runtime和Identity服务
  - 端到端执行观察：AgentCore Observability
- **安全部署特性**：
  - AgentCore Runtime：在客户VPC内安全容器化部署
  - AgentCore Identity：解决用户和代理的访问控制挑战
  - AgentCore Gateway：提供工具和能力访问

### 25:00-35:00 AWS Marketplace AI代理和工具
- **Marketplace三大开发者优势**：
  - 采购时间从数月缩短至数天或分钟
  - 统一计费和成本控制透明度
  - 简化的部署体验
- **三类产品分类**：
  - API基础代理和工具（远程ISV托管）
  - 容器化代理和工具（在AgentCore Runtime上运行）
  - 代理应用程序和专业服务

### 35:00-45:00 Workday规划代理实践案例
- **规划代理四个角色**：分析师、建模师、规划师和管理员
- **数据探索技能迭代过程**：
  - 第一次迭代：直接发送数据给LLM（遇到令牌限制、成本高、数学准确性问题）
  - 第二次迭代：数据分块处理（解决令牌限制但仍有延迟和成本问题）
  - 最终解决方案：让LLM编写Python代码进行数据分析
- **AgentCore代码解释器优势**：安全的沙箱环境、有状态多步分析、可扩展性
- **实施成果**：1名软件开发人员、2名工程经理、3天时间从POC到生产就绪

### 45:00-55:00 实际构建代理演示
- **用例场景**：为销售团队构建公司研究代理
- **解决方案组件**：代理执行器、推理LLM、工具（Tavily搜索工具和本地数据搜索）
- **技术实现**：
  - 使用LangChain框架进行编排
  - Amazon Bedrock提供200多个模型选择
  - AgentCore Gateway集成第三方工具
- **代码演示**：展示如何设置LLM、工具和提示的核心代码结构
- **生产化建议**：安全治理、性能可靠性、成本控制

### 55:00-56:30 预构建代理部署演示
- **三步部署流程**：发现代理、订阅、部署
- **Marketplace标准化框架**：供应链保护、ECR私有仓库、权限扫描
- **AgentCore Runtime优势**：VPC内运行、IAM角色配置、安全运行时环境
- **实际演示**：CRM AI销售代理的完整部署过程，从Marketplace订阅到AgentCore控制台部署
