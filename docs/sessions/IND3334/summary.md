# AWS re:Invent 2025 技术分享会总结

## 会议概述

本次AWS re:Invent 2025分享会重点介绍了如何利用AI技术加速软件开发和代理系统构建。演讲者提出了"AI开发生命周期"(AI DLC)的创新理念,将AI从传统的开发工具转变为开发伙伴,帮助组织快速将积压的项目想法转化为生产就绪的代码。核心方法论强调在5天内完成从概念到生产级代码的开发过程,这不是夸张而是经过客户验证的真实案例。

演讲的核心案例来自Nivo公司,他们利用这套方法论成功将一个积压两年的项目在短时间内推向生产环境。该项目的目标是构建一个"代理构建代理"的系统,使广告商能够快速创建面向客户的AI聊天代理,实现品牌声音定制、内容安全管理和大规模部署。整个解决方案基于AWS服务构建,包括Lambda、Bedrock、RDS、S3等,采用无服务器架构确保成本效益和可扩展性。

演讲者强调了AI辅助开发的关键成功因素:明确的方向、领域驱动设计、人机协作分工(让AI处理枯燥的文档和测试编写工作)、以及从一开始就考虑生产环境要求。目前该方法已在三个客户项目中得到验证,所有项目都在向生产环境推进。

## 详细时间线与关键要点

[00:00-01:30] 开场与问题陈述
- 演讲者以热情的方式开场,提出核心问题:组织有大量创意和想法,但多少项目因无法获得批准、资源或时间而积压在backlog中?
- 引出主题:如何快速大规模地构建这些想法?

[01:30-03:45] AI开发生命周期(AI DLC)介绍
- 提出三个核心议题:AI DLC(将AI作为开发伙伴而非工具)、构建能够生成代理的代理系统、模型定制化
- 强调当前AI编码的最大问题:AI缺乏完整上下文信息,导致token耗尽和重复问题
- AI DLC流程从头到尾教会AI理解问题、目标和环境

[03:45-06:30] AI DLC流程详解
- **发现阶段(Discovery)**:教AI了解开发环境,包括CI/CD工具、开发环境、使用的库、首选编程语言、服务白名单/黑名单等
- **需求分析(Requirements Analysis)**:将愿景转化为用户故事,包含足够的技术内容以构建领域模型
- **开发阶段**:使用领域模型和用户故事构建代码、测试框架和CI/CD部署
- **交付阶段**:5天内达到生产就绪代码(虽然会有一些问题需要在第3-5天解决)

[06:30-08:15] AI DLC与敏捷开发的对应关系
- AI DLC清晰映射到现有敏捷软件开发生命周期
- 输出可直接集成到Jira等现有工具中
- 用户故事在开发过程中从backlog推进到进行中、测试、验证、部署等阶段
- 强调这些是文本文件配置,包含丰富的提示词,可根据组织需求定制

[08:15-11:00] 广告技术领域的关键挑战
- **品牌声音**:不同产品需要不同的品牌语调(例如:家用小型车轮胎强调安全舒适,越野卡车轮胎强调坚固耐用)
- **安全性**:防止AI生成不当内容或进行品牌比较
- **资产管理**:标语、文案、视频、图表等广告内容的管理
- **成本控制**:必须实现低成本运营,不能每次聊天都花费巨额费用

[11:00-13:30] Nivo案例介绍
- Nivo的愿景:使广告商能够快速创建与客户互动的代理,讨论广告商产品
- 项目在backlog中积压两年
- 使用AI DLC方法论在5天内完成开发,加上几周的生产打磨和发布准备
- 强调这是真实的生产级代码,不是演示性质的"小猫软件"

[13:30-16:45] 代理构建流程
- **品牌声音分析**:使用AI分析广告商的现有内容(文案、目标、意图声明等)自动生成品牌声音提示词
- **内容选择**:从内容库中为特定代理选择合适的资产(可能管理150个不同广告商)
- **护栏构建**:两层护栏机制
  - Bedrock内置护栏:防止不当内容、提示词注入等
  - 自定义护栏:控制是否允许讨论竞争对手、比较数据来源等
- **资源配置**:配置关系数据库、OpenSearch语义搜索等数据源访问

[16:45-19:30] 配置管理与部署架构
- 将所有信息整合为配置数据而非硬编码:品牌声音规范、内容护栏、数据源配置等
- 存储在关系数据库、JSON数据存储或文档数据库中
- **部署架构**:
  - 三层API Gateway提供安全性
  - React Web应用作为前端
  - Lambda函数提供所有功能(品牌分析、知识库、代理本身、安全服务)
  - S3存储静态资源
  - 自然语言转SQL服务

[19:30-22:00] 数据源集成
- 利用"数据引力"概念:直接使用RDS中现有的PostgreSQL数据源
- 确保自然语言转SQL服务为只读且具有适当安全准则
- S3存储视频、图像、图表等各类内容
- 使用Bedrock Knowledge Base和OpenSearch实现语义搜索
- 提到Agent Core可以封装80%的功能,但客户选择Lambda以保证最低成本

[22:00-24:30] 代理运行时流程
- **初始化**:加载提示词(品牌声音)、初始化Bedrock、创建工具集、初始化框架(使用LangGraph,因为当时Strands不支持TypeScript)
- **运行循环**:
  - 输入:品牌ID、代理配置、消息ID(用于会话管理)
  - 从Agent Core Memory检索对话历史
  - AI分析并决定需要哪些额外信息
  - 调用Lambda工具获取数据
  - 生成最终响应
- **安全控制**:限制循环次数,提供实时反馈(告诉用户"正在搜索数据库"等)防止用户重复点击

[24:30-26:00] 实际应用场景
- 在网站侧边栏放置聊天代理(类似传统广告位)
- 例如:轮胎相关文章旁边显示"想了解更多关于Bob轮胎的信息吗?"
- 引导用户回答问题:轮胎类型、车辆类型、购买时间表等
- 生成销售线索,促进广告商与客户互动
- 强调系统的通用性:只要配置正确,可重复使用,不需要每次都定制开发

[26:00-28:30] 成功因素总结
- **明确方向**:5天内没有时间犹豫不决
- **领域驱动**:从需求和愿景构建领域模型
- **人机协作**:让AI处理人类觉得枯燥的工作(编写用户故事、单元测试、文档)
- **生产级标准**:从一开始就告诉AI生产环境要求
- 演讲者自述45年编程经验,承认自己不擅长写文档,强调让AI做这些工作

[28:30-30:00] 项目成果与建议
- 该方法已在三个客户项目中验证,全部走向生产
- Nivo项目目前处于alpha测试阶段,与广告商一起测试
- 具备护栏、可重复性、可扩展性和成本/时间/能源安全控制
- **建议**:
  - 不要从小型演示项目开始,要投入真实的时间和团队
  - 给予真正的尝试机会
  - GitHub上提供了流程代码库(演讲结尾提供链接)
  - 这些只是文本文件,可以下载并根据组织需求修改

[30:00-30:45] 未来展望与结束
- Nivo正在为生产环境打磨产品,根据客户反馈进行修复
- 已经在规划第二阶段功能扩展
- 鼓励听众尝试这个方法:"你有什么损失?只是一点时间"
- 承认第一次使用时需要大量工作,因为要学习流程
- 演讲被主持人打断(因为时间限制需要准备下一位演讲者)
