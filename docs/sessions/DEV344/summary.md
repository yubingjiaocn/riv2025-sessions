# AWS re:Invent 2025 会议总结

## 会议概述

本次分享来自巴西利亚（Brasília）的AWS用户组社区，由Priscilla Rau（CL Brazil业务开发人员）和Dendi（解决方案架构师及AWS社区领导者）共同呈现。他们介绍了一个专门为巴西AWS用户组创建的社区活动管理平台。巴西拥有超过20个AWS用户组，在组织社区活动时面临诸多挑战：需要处理活动详情、演讲者信息、日期安排、注册管理、设计等超过20项不同的任务和流程，且分散在多个平台上。为了解决这些痛点，他们决定利用AWS服务构建一个集中化的平台来简化流程。

该平台在2025年AWS Summit São Paulo期间首次大规模应用，通过WhatsApp向参会者发送消息，邀请他们参与社区活动。数据显示，使用该平台后，活动浏览量提升了20%以上，社交媒体互动增加了28%。项目从简单的MVP逐步演进为生产级架构，采用了多项AWS服务确保可扩展性、安全性和高可用性。这是一个由多个巴西社区（包括Brasília、Goiânia、São Paulo、Bahia等）协作开发的开源项目，团队成员包括许多年轻的女性开发者志愿者。该平台被命名为"Party Community"，预计每年可为社区组织者节省超过70小时的工作时间。

06:30 - 09:00 - 生产级架构详解
- **Route 53**：域名托管和DNS管理
- **CloudFront**：CDN内容分发，实现全球低延迟
- **WAF**：安全防护，过滤恶意请求
- **S3 + CloudFront**：前端层服务，实现快速和高可用
- **ECS Fargate**：后端API运行环境，跨多可用区部署
- **Application Load Balancer**：分发请求到Fargate任务
- **VPC网络架构**：公有和私有子网跨可用区部署，增强安全性
- **NAT Gateway**：提供互联网连接
- **Amazon Aurora**：数据库层，多可用区部署提升弹性和可用性
- **ECR**：存储容器镜像
- **SNS**：处理HTTP webhook通知
- **SES**：发送事务性邮件（注册确认、活动邀请等）
- **CloudWatch + CloudWatch Logs**：可观测性、指标、告警和日志收集

09:00 - 10:00 - 平台优势总结
- 无需在20多个平台发送消息，统一在控制台操作
- 简化事件前后的消息发送流程
- 过去需要10多个不同平台，现在一个平台完成所有流程
- 提供新的方式连接和通知社区成员
- 预计每年节省超过70小时工作时间
