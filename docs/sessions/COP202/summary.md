# AWS re:Invent 2025 - COP 202: AWS 成本优化新功能

## 会议概述

本次会议由 AWS 成本优化产品管理团队负责人 Rick Oaks 主持，重点介绍了 AWS 成本优化工具的最新功能和发展。会议邀请了达美航空中央 FinOps 团队负责人 Bonnie Fernstall 和储蓄计划购买体验负责人 Mary Naimov 作为嘉宾演讲者。

会议主要围绕三个核心主题展开：首先是 Compute Optimizer 中新推出的 EBS 优化自动化功能，包括闲置卷清理和卷类型升级；其次是新的成本效率指标，为 FinOps 团队提供了统一的优化成功衡量标准；最后是全新的数据库储蓄计划，为多种数据库服务提供高达 35% 的折扣。达美航空的实际案例展示了如何通过系统化的方法实现 27% 的月度成本节省。

## 详细时间线与关键要点

### 00:00-05:00 会议开场与背景介绍
- Rick Oaks 介绍团队背景，负责 Compute Optimizer、成本优化中心、储蓄计划分析器等产品
- 现场调研显示参会者中 FinOps 组织和工程团队人员各占约一半
- 大部分参会者已使用过 AWS 优化工具，如 Compute Optimizer 和储蓄计划

### 05:00-15:00 EBS 优化自动化功能详解
- 新推出的 EBS 优化功能解决客户反馈的"推荐太多难以执行"问题
- 支持两种主要优化：闲置 EBS 卷自动快照和删除；GP2 到 GP3 升级（节省 20%）和 IO1 到 IO2 升级
- 可创建自动化规则，支持组织级别、账户级别或基于标签的精细化控制
- 内置安全网机制：1年内可回滚、实施前重新验证卷状态、失败时自动停止

### 15:00-25:00 现场演示 EBS 自动化功能
- 演示 Compute Optimizer 控制台中的新自动化部分
- 展示多选推荐并一键执行的简便流程（仅需4-5次点击）
- 详细演示自动化规则创建过程，包括标签条件设置、时间窗口配置
- 展示历史执行记录和节省金额追踪功能

### 25:00-30:00 NAT 网关闲置检测新功能
- 新增 NAT 网关闲置检测推荐，基于30天历史使用情况
- 不仅检查零使用率，还验证路由表关联状态，避免误判备用网关
- 在 Compute Optimizer 的闲置推荐列表中显示详细使用情况

### 30:00-40:00 成本效率指标介绍
- 推出全新的成本效率指标，平衡资源优化和承诺优化的价值
- 计算公式：效率 = (1 - 总潜在节省/总可优化支出) × 100%
- 提供90天历史数据，支持按账户、地区查看
- 可识别效率最高和最低的前5名，便于团队间知识分享

### 40:00-50:00 达美航空优化实践案例
- Bonnie Fernstall 分享达美航空在1200个账户生态系统中的优化历程
- 创建机会仪表板解决"在哪里"和"谁来做"的问题，集中化推荐并添加组织上下文
- 开发机会手册，将复杂优化知识浓缩为5分钟阅读材料
- 实施轻量级标签策略，使用10字符代码代表业务能力
- 从纯报告机制演进为包含团队决策的系统，要求提供实施时间线和延迟原因

### 50:00-54:00 数据库储蓄计划新产品
- Mary Naimov 介绍全新的数据库储蓄计划，覆盖9种数据库服务
- 提供高达35%折扣，1年期承诺，无预付费用
- 自动按最高折扣优先级应用：Aurora Serverless (35%) → Elastic Cache → DynamoDB
- 演示购买分析器的使用，支持灵活的回看期选择和自定义承诺金额
- 提供详细的覆盖率和利用率报告，便于购买后监控和优化
