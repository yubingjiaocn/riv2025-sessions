# AWS re:Invent 2025 会议总结：AWS Transfer Family Web Apps

## 概述

本次会议由 AWS 高级解决方案架构师 Ed Laura 主讲，重点介绍了 AWS Transfer Family Web Apps 功能。该服务旨在为存储在 Amazon S3 中的数据提供简单的 Web 浏览器访问方式，无需编写代码即可让非技术用户通过 Web 应用程序访问 S3 数据。这解决了许多企业面临的核心问题：如何让普通用户安全、便捷地访问 S3 中的数据，而不必使用复杂的 S3 控制台或安装专门的客户端软件。

AWS Transfer Family 是一个模块化的托管文件传输服务家族，包括文件传输服务器（SFTP 协议）、文件传输连接器以及 Web UI 三个主要组件。Web Apps 功能提供了浏览器原生的文件管理体验，支持文件上传、下载、搜索、移动等操作，并可通过企业 Logo 和自定义 URL 进行品牌定制。该服务与 IAM Identity Center 集成，支持多种身份提供商（Active Directory、Entra ID、Ping Identity、Okta 等），并通过 S3 Access Grants 实现细粒度的访问控制。会议还重点介绍了刚刚发布的 VPC 端点功能，允许企业将 Web Apps 部署在私有网络中，进一步增强安全性。

01:15 - 核心问题阐述
- 许多客户在 S3 中存储数据，需要为非技术用户提供简单的访问方式
- S3 控制台对普通用户来说过于复杂
- 需要满足严格的安全要求
- 替代方案成本高昂或需要自建定制解决方案

02:30 - AWS Transfer Family 架构介绍
- 文件传输服务器：托管 SFTP 服务器，高可用、高性能，支持 S3 和 EFS 存储
- 文件传输连接器：作为 SFTP 客户端，可向远程 SFTP 服务器推送或拉取文件
- Web UI：为 S3 数据提供浏览器访问界面

04:00 - 事件驱动架构应用
- 支持构建事件驱动的数据工作流
- 第三方数据上传触发后续处理管道
- 与 AWS AI、分析、数据库服务集成

05:00 - Web Apps 核心特性
- 简单易用的浏览器界面
- 易于配置和部署
- 可定制化：支持企业 Logo 和自定义 URL
- 符合 HIPAA、FedRAMP 等合规标准

05:45 - 实时演示开始
- 用户登录界面展示
- 与身份提供商集成（可使用 Active Directory 凭证）
- 支持用户组和权限管理

06:30 - 读写用户演示
- 浏览目录结构
- 创建文件夹功能
- 拖放上传文件到 S3
- 文件上传可触发 S3 事件通知

07:30 - 搜索和文件管理功能
- 全局搜索功能（演示搜索 "Q4"）
- 创建新文件夹
- 移动文件操作
- 所有操作直接反映在 S3 存储桶中

08:45 - 只读用户演示
- 登录财务分析师账户
- 展示受限访问权限
- 只能查看和下载特定目录的文件
- 无法上传或创建文件夹

09:30 - 主要使用场景
- 企业内部文件共享（员工访问）
- 第三方合作伙伴数据共享
- 无需安装客户端，仅需浏览器访问
- 通过 Identity Center 控制身份和访问

10:15 - 设置配置流程
- 利用 IAM Identity Center（可使用现有实例）
- 通过 S3 Access Grants 定义用户访问权限
- 在 Transfer Family 控制台创建 Web App
- 无需编写代码，自动生成公共访问 URL

11:00 - 架构说明
- 用户通过互联网连接
- Identity Center 与企业身份提供商集成
- 支持多因素认证（MFA）
- Transfer Family Web App 在区域内运行
- 通过 S3 Access Grants 控制访问
- 数据存储在 Amazon S3

12:00 - 新功能发布：VPC 端点支持
- 上周刚刚发布的功能
- 增强安全性，适用于内部用户场景

- 保持相同的用户体验

- 自定义 URL 支持
- 相同的最终用户体验
