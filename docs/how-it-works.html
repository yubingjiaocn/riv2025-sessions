工作原理

项目简介

本项目是一个自动化系统，用于从 YouTube 获取 AWS re:Invent 2025 的会议视频，并提供中文翻译和总结服务。通过自动化流程，我们为中文用户提供便捷的会议内容浏览和学习体验。

自动化工作流程

系统通过以下步骤自动处理会议内容：

1. 获取视频信息

系统从指定的 YouTube 播放列表中自动抓取所有 AWS re:Invent 2025 会议视频的基本信息，包括视频 ID、标题、发布时间等元数据。

2. 搜索会议详情

使用视频信息在 AWS Events 官方目录中搜索对应的会议详细信息，包括会议代码、完整标题、摘要、主题分类、涉及的 AWS 服务等结构化数据。

3. 内容翻译

通过 Kiro CLI 工具调用生成式 AI 模型，将会议标题和摘要从英文翻译成简体中文，确保中文用户能够快速理解会议内容。

4. 字幕提取与摘要生成

从 YouTube 视频中提取英文字幕，并使用 AI 模型对字幕内容进行分析和总结，生成结构化的中文会议摘要，帮助用户在观看视频前了解核心内容。

5. 网站生成与发布

将所有处理后的数据生成静态网站文件，包括会议列表页、详情页和搜索过滤功能，并自动发布到 GitHub Pages，使内容能够被公开访问。

技术栈

  • 数据采集：Python + yt-dlp (YouTube 视频下载工具)
  • AI 翻译与摘要：Kiro CLI + 生成式 AI 模型
  • 前端框架：原生 JavaScript + HTML5 + CSS3
  • UI 组件：Choices.js (多选过滤器)、Toast UI Pagination (分页)
  • 托管平台：GitHub Pages

数据结构

项目采用以下目录结构存储数据：

sessions/              # 会议数据目录
└── <SESSION_CODE>/   # 每个会议一个子目录
    ├── metadata.json  # 会议元数据（标题、摘要、分类等）
    └── summary.md     # AI 生成的会议总结

docs/                  # 静态网站文件
├── index.html         # 会议列表主页
├── session.html       # 会议详情页模板
├── how-it-works.html  # 工作原理说明页
├── style.css          # 样式文件
├── app.js             # 主页交互逻辑
├── session.js         # 详情页交互逻辑
├── sessions.json      # 会议数据索引
└── sessions/          # 会议数据副本（用于网站访问）

4. 代码示例

以下是使用不同编程语言访问数据的示例：

Python 示例

import requests

# 获取所有会议列表
response = requests.get('https://yubingjiaocn.github.io/riv2025-sessions/sessions.json')
data = response.json()
print(f"总会议数: {data['metadata']['sessions_count']}")

# 获取特定会议元数据
session_code = 'AIM102-S'
metadata_url = f'https://yubingjiaocn.github.io/riv2025-sessions/sessions/{session_code}/metadata.json'
metadata = requests.get(metadata_url).json()
print(f"会议标题: {metadata['title_cn']}")

# 获取会议摘要
summary_url = f'https://yubingjiaocn.github.io/riv2025-sessions/sessions/{session_code}/summary.md'
summary = requests.get(summary_url).text
print(summary[:200])  # 打印前200个字符

JavaScript 示例

// 使用 Fetch API 获取会议列表
fetch('https://yubingjiaocn.github.io/riv2025-sessions/sessions.json')
  .then(response => response.json())
  .then(data => {
    console.log(`总会议数: ${data.metadata.sessions_count}`);

    // 获取第一个会议的详细信息
    const sessionCode = data.sessions[0].session_code;
    return fetch(`https://yubingjiaocn.github.io/riv2025-sessions/sessions/${sessionCode}/metadata.json`);
  })
  .then(response => response.json())
  .then(metadata => {
    console.log(`会议标题: ${metadata.title_cn}`);
  });

5. 使用场景

  • AI 代理集成：让 AI 助手直接查询和分析 AWS re:Invent 2025 会议内容
  • 自动化工具：构建自动监控、通知或数据分析工具
  • 数据分析：对会议主题、服务、演讲者等进行统计分析
  • 个性化应用：基于这些数据构建定制化的会议推荐或搜索工具
  • 内容聚合：将会议数据整合到其他平台或系统中

6. 注意事项

  • 所有数据通过 GitHub Pages 托管，采用标准 HTTP/HTTPS 协议，无需认证
  • 建议实施合理的缓存策略，避免频繁请求
  • 数据会定期更新，请参考 sessions.json 中的 last_updated 字段
  • 如果某个会议的 summary 字段为 false，表示该会议的摘要文件可能不可用
  • 请遵守 GitHub Pages 的使用条款和合理使用原则
