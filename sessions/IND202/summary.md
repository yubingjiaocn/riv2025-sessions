# AWS re:Invent 2025 会议总结：利用 AI 代理加速真实世界数据分析

## 会议概述

本次 AWS re:Invent 2025 分会场聚焦于医疗保健和生命科学领域的真实世界数据（Real World Data, RWD）和真实世界证据（Real World Evidence, RWE）的应用创新。会议由 AWS 医疗保健与生命科学行业战略解决方案开发项目负责人 Ann Evans 主持，礼来公司（Eli Lilly）真实世界数据高级总监 Greg Cunningham 和 AWS 首席解决方案架构师 Eric Brooks 共同分享了他们在数据访问、分析和 AI 代理工作流方面的实践经验。

会议强调了一个核心挑战：尽管医疗保健数据在过去十年增长了 11 倍（从 2015 年的 16 ZB 增至 2025 年的 181 ZB），但 80% 的医疗数据仍然是非结构化且难以访问的。制药公司在开发每一款突破性药物时需要分析超过 1000 万个患者数据点，而传统方法下从数据获取到分析往往需要 4-12 周甚至更长时间。通过 AWS 与合作伙伴 Datavant 的协作，以及利用 AWS Clean Rooms、AI 代理等技术，这一流程已被压缩至 4 周以内，大幅提升了药物研发效率并降低了数百万美元的成本。

第三段概述了技术实现路径：AWS 与 Datavant 合作构建了基于 AWS Clean Rooms 的隐私保护数据发现平台，使用代币化技术在不泄露个人身份信息的前提下实现跨数据集的患者级数据链接。同时，礼来公司与 AWS 共同开发了基于 AI 代理的自然语言查询系统，让非技术用户也能直接分析复杂的医疗数据集，实现了从"仅限数据科学家使用"到"全员自助分析"的民主化转变。

## 详细时间线与关键要点

### 开场与背景介绍（开始 - 约 5 分钟）
- **关键数据**：每款上市药物需分析超过 1000 万患者数据点；80% 医疗数据仍为非结构化；利用真实世界证据可将临床试验时间缩短 40%，节省数百万美元成本
- **演讲者介绍**：Ann Evans（AWS）、Greg Cunningham（礼来）、Eric Brooks（AWS）

### 真实世界数据的增长与挑战（约 5-10 分钟）
- **数据爆炸**：全球数据从 2015 年 16 ZB 增至 2025 年 181 ZB（11 倍增长）；医疗数据预计 2025 年底达到 10 ZB
- **RWD/RWE 定义**：真实世界数据是日常患者护理中收集的医疗数据（电子健康记录、可穿戴设备、保险理赔等）；真实世界证据是从这些数据中分析得出的洞察
- **应用场景**：预防性医疗、临床试验患者筛选、药物安全测试、医疗干预价值评估、监管合规支持

### 数据访问的核心挑战（约 10-15 分钟）
- **数据碎片化问题**：40% 的数据购买不完整；70% 的研究需要跨多个数据集的特定数据；单一数据提供商无法提供完整患者旅程数据
- **时间成本**：发现和获取第三方数据需 4-12 周；数据协调需额外 4 周；整体药物开发周期超过 10 年，成本达 20-30 亿美元
- **HIPAA 合规**：所有数据必须去标识化，使得跨数据集连接患者数据极具挑战性

### AWS 与 Datavant 的解决方案（约 15-25 分钟）
- **技术架构**：基于 AWS Clean Rooms 构建隐私保护数据发现平台
- **代币化技术**：Datavant 将个人身份信息（姓名、性别、出生日期）转换为唯一加密字符串（token），同一患者在不同数据集中生成相同 token，实现患者级数据链接而不泄露 PII
- **工作流程**：
  - 数据提供商在自己的 AWS 账户中托管代币化数据
  - 数据消费者通过 AWS Clean Rooms 评估多个数据集（数据不移动）
  - 直接与数据提供商协商并通过 AWS Data Exchange 订阅
  - 自动化数据协调并交付至 S3 存储桶
- **成果**：试点客户反馈显示数据访问时间大幅缩短

### 礼来公司的真实世界数据实践（约 25-40 分钟）
- **公司背景**：Greg Cunningham 在礼来工作 39 年，过去 12 年专注于真实世界证据
- **业务需求**：从传统临床试验（金标准但成本高、速度慢）转向实时真实世界数据分析
- **RWE 公式**：科学问题 + 正确的数据 + 统计分析计划 = 真实世界证据
- **应用场景**：
  - 早期药物发现
  - 临床试验设计优化（纳入/排除标准）
  - 上市后观察性研究（经济价值、比较有效性）
  - 患者特征分析
  - 科学期刊发表

### 数据类型与演进（约 40-45 分钟）
- **基础数据**（过去 10-15 年）：
  - 电子健康记录（EHR）：临床变量、实验室数据
  - 理赔与账单数据：长期患者健康交易记录
  - 特点：广泛覆盖所有疾病，但缺乏疾病特异性
- **新兴数据**（近年来）：
  - 可穿戴设备、传感器、健康应用（如偏头痛追踪应用）
  - 生物标志物和基因组数据（肿瘤学团队重点使用）
  - 特点：疾病特异性强，能显示疾病进展
- **代币化链接**：将广泛的纵向数据与深度疾病特异性数据结合，实现更丰富的患者旅程分析

### 数据获取流程与挑战（约 45-50 分钟）
- **传统流程**：
  1. 定义科学问题
  2. 评估内部与外部数据（可能耗时数月甚至一年）
  3. 数据传输、摄取、移动至 Redshift（可能耗时数月）
  4. 数据协调与分析
- **AWS/Datavant 解决方案目标**：将数据获取和协调时间从数月压缩至一个月或更短

### 分析能力的三层演进（约 50-55 分钟）
- **第一层：专业分析师**（复杂编码）
  - 用户：数据科学家、统计学家
  - 工具：R、Python、AWS 云平台
  - 用途：发表论文、监管工作（核心业务）
  - 挑战：人员短缺、学习曲线陡峭（新人需 6 个月才能高效工作）、缺乏统一数据标准
 
- **第二层：低代码环境**
  - 用户：有编程背景的科学家
  - 工具：市场上的分析工具（低代码参数输入）
  - 用途：自助分析、快速回答问题
 
- **第三层：无代码 AI 代理**（即将上线）
  - 用户：所有科学家（无需编程背景）
  - 工具：自然语言查询界面
  - 用途：会议中实时回答问题（如"疾病 X 的患者百分比是多少？""该领域最常用的药物是什么？"）
  - 上线时间：下周（相对于会议时间）

### AI 代理实施的关键经验（约 55-60 分钟）
- **业务上下文至关重要**：LLM 和代理需要领域专家头脑中的业务上下文才能生成准确代码
- **问题越具体越好**：通用问题会导致代理做出更多假设；建议用户明确指定数据源（如"处方理赔数据"）
- **对话式交互**：不是一次性输入大段问题，而是通过多轮对话逐步细化，提高答案准确性
- **建立信任**：科学家天生持怀疑态度；计划通过真实问题测试来验证代理生成的 SQL 代码质量
- **分阶段推广**：先向技术熟练的试点用户软启动，再向技术能力较弱的用户硬启动

### AI 代理架构深度解析（约 60-75 分钟）
- **Eric Brooks 登场**：AWS 首席解决方案架构师，与礼来合作 1.5 年

- **核心挑战**：
  - 这不是简单的销售数据集
  - 不是简单的文本转 SQL 用例
  - 数据量巨大（单个数据提供商每月提供 15 TB 新数据）
  - 数据分散在不同服务、不同模态、不同架构中
  - 用户不是 SAS/R/Python 开发者
  - 用户不了解领域知识（如理赔数据中"连续注册"对质量的重要性）

- **设计目标**：使用 Anthropic 和 Amazon 最新前沿模型，民主化数据访问，确保最高准确性和质量输出

### 代理设计的心智模型（约 75-80 分钟）
- **四个核心问题**：
  1. **目标（Goal）**：用户体验是什么？仅提供 SQL 查询还是提供完整分析（洞察、可视化、高性能体验）？
  2. **工具（Tools）**：人类完成此工作需要哪些工具？开发即时知识需要哪些工具？
  3. **协议（Protocol）**：达成目标需要哪些步骤？这将指导代理提示词设计
  4. **交接（Handoffs）**：多代理系统中如何传递信息？不是数据转储，而是"这是你需要的，这是我要求的，这是指导"

### 架构决策（约 80-85 分钟）
- **单代理 vs 多代理系统**：
  - 单代理：一个文本转 SQL 代理处理所有工具和上下文
  - 多代理：专业化技能、能力和知识集，分解问题有效解决
 
- **礼来案例的选择**：真实世界数据分析涉及多个角色/技能，不是简单的文本转 SQL，因此选择多代理系统

## 关键技术要点总结

1. 隐私保护数据链接：Datavant 代币化 + AWS Clean Rooms
2. 数据民主化：从专业分析师到全员自助分析的三层演进
3. AI 代理最佳实践：业务上下文、具体问题、对话式交互、信任建立
4. 多代理架构：针对复杂真实世界数据分析的专业化分工
5. 时间压缩：从 4-12 个月缩短至 4 周以内
