# AWS re:Invent 2025 - Moody's多智能体系统架构会议总结

## 会议概述

本次技术分享会由AWS首席解决方案架构师Samuel Baruffi和Moody's数字内容与创新部门工程与架构总监Dennis Climent共同主讲，详细介绍了这家百年金融机构如何在AWS上构建生产级多智能体系统。会议重点展示了Moody's从2023年简单RAG应用发展到复杂多智能体协作系统的完整技术演进历程。

Moody's作为服务于全球1500多家客户、覆盖165个国家的信用评级机构，面临着处理大量非结构化金融数据的挑战。他们需要为商业银行、资产管理公司、保险公司等不同类型客户提供高精度的风险分析服务，这些决策往往涉及数十亿美元的资金。因此，系统的准确性要求极高，99%的准确率都不够，必须达到接近100%的精度。

## 详细时间线与关键要点

### 00:00-05:00 开场介绍与背景
- 会议在拉斯维加斯周三早上8:30举行，主题为IND3303
- 介绍演讲嘉宾和Moody's公司背景
- 现场调研显示大多数参会者正在构建智能体系统，且来自金融服务行业

### 05:00-15:00 Moody's公司概况与挑战
- Moody's是拥有100年历史的信用评级机构，服务财富100强中97%的公司
- 公司数据宇宙包含四大支柱：评级、研究洞察、数据信息、决策解决方案
- 运营Orbis数据库，包含6亿个实体的信息
- 面临客户多样性挑战：2600家商业银行、1900家资产管理公司、800多家保险公司
- 金融服务行业80%的数据为非结构化数据，但只有20%的组织能有效利用

### 15:00-25:00 技术演进历程
- 2023年12月：发布研究助手RAG应用，用户满意度高
- 2024年8月：推出PDF上传功能，开始处理客户自有数据
- 2025年底：实现完整多智能体系统，包含定制编排器和专业化工作流
- 研究助手为用户节省60%时间，任务完成效率提升30%

### 25:00-35:00 AWS Bedrock服务介绍
- Amazon Bedrock作为完整生态系统，支持多模型选择
- 全球跨区域推理：自动路由到有容量的区域，提高系统弹性
- 地理跨区域推理：在特定地理区域内路由，满足合规要求
- Knowledge Base for Amazon Bedrock：提供端到端RAG工作流管理

### 35:00-45:00 非结构化数据处理挑战
- Bedrock Data Automation：专门处理复杂文档的多模态数据提取
- 支持音频、视频、图像和文档等多种格式
- 相比传统大语言模型，在价格和准确性方面都有显著提升
- 解决金融文档中复杂表格、图表提取的技术难题

### 45:00-55:00 架构深度解析
- 系统架构五大支柱：无服务器优先、工具为核心、简单与复杂智能体分离、定制编排器、多模型选择
- 当前系统规模：80个工具、100多个工作流、每日处理超过100万个令牌
- 工作流设计器支持从20步到400步的复杂工作流
- PDF处理的四种失败尝试：基础Python库、自定义解析算法、多模态基础模型、大上下文窗口

### 55:00-58:00 未来发展方向与AgentCore介绍
- 智能页面分类系统：文本页面使用Bedrock LLM，表格页面使用Bedrock Data Automation
- 智能体检索系统：模拟人类文档阅读方式，支持复杂查询分解
- AWS AgentCore八大原语：Runtime、Gateway、Identity、Memory等
- 向客户开放MCP和智能API，实现工具共享
