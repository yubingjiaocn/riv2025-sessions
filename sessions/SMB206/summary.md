# AWS re:Invent 2025 技术分享会总结

## 会议概述

本次分享会由AWS高级解决方案架构师Deepti Ventur Milli和Slick Deals工程副总裁Mike Lively共同主讲,重点介绍了Slick Deals如何在AWS平台上完成迁移、现代化改造和创新的完整历程。Slick Deals是一个社区驱动的优惠发现平台,拥有1200万月活跃用户,在过去25年中为用户节省了100亿美元。

会议首先介绍了人工智能的演进过程,从传统AI到机器学习,再到分类AI、预测AI,最后到生成式AI。随后详细讲解了Amazon SageMaker AI作为综合性机器学习服务的能力。Mike Lively分享了Slick Deals面临的业务挑战,包括数据可扩展性问题、僵化的本地基础设施、缺乏实时交互数据,以及无法有效运行个性化推荐平台等问题。为解决这些挑战,Slick Deals进行了全面的云迁移和现代化改造,包括数据平台迁移、应用程序迁移、实时数据收集增强,以及将个性化推荐和搜索功能内部化。

通过这些改造,Slick Deals实现了显著的业务成果:测试周期从6周缩短到1周,优惠评分延迟从3小时降低到30秒,商家外链点击和收入增长了7%。Mike强调了现代化改造的关键要素:关注快速扩展能力、聚焦业务差异化优势、建立强大的数据基础,以及建立快速反馈循环机制。

## 详细时间线

### 开场介绍 (00:00 - 02:30)
- **00:00** - Deepti Ventur Milli自我介绍,AWS高级解决方案架构师,在AWS工作4年
- **00:30** - 介绍演讲嘉宾Mike Lively,来自Slick Deals客户
- **01:00** - 说明本次分享主题:Slick Deals在AWS上的迁移、现代化和创新故事

### AI和机器学习基础概念 (02:30 - 05:00)
- **02:30** - 介绍人工智能的定义:能够复制需要人类智能的任务的系统
- **03:00** - 讲解AI的不同子集:传统AI、机器学习、分类AI、预测AI
- **03:45** - 介绍生成式AI:深度学习的子集,可以创建新内容(对话、故事、图像、视频)
- **04:30** - 介绍Amazon SageMaker AI:综合性机器学习服务,支持业务分析师、MLOps工程师和数据科学家

### Slick Deals公司介绍 (05:00 - 08:30)
- **05:00** - Mike Lively开始介绍Slick Deals
- **05:30** - 核心定位:社区驱动的优惠发现网站,社区是核心差异化优势
- **06:00** - 关键数据:1200万月活跃用户,25年内为用户节省100亿美元
- **06:45** - 业务重点:改进个性化推荐、实时优惠评分平台、优惠分享平台
- **07:30** - 市场定位分析:位于优惠聚焦和娱乐性的象限,结合优惠发现和社交元素
- **08:15** - 广告解决方案:每年产生14.5亿美元的合作伙伴销售额

### 业务挑战 (08:30 - 11:00)
- **08:30** - 数据可扩展性问题:本地数据平台无法处理展示数据(impressions)
- **09:00** - 基础设施僵化:20-25年前的本地基础设施难以创新和测试
- **09:30** - 缺乏实时交互数据:基于ETL的分析平台,数据延迟3小时
- **10:00** - 个性化能力不足:不得不将个性化功能外包给第三方
- **10:30** - 最大障碍:无法及时判断优惠质量,优惠生命周期通常24小时,但交易数据延迟2-3天

### 解决方案策略 (11:00 - 13:30)
- **11:00** - 建立更强大的数据基础,能够处理更多数据(特别是展示数据)
- **11:30** - 扩展Slick Deals的娱乐和社交价值,增加用户评论和投票
- **12:00** - 将个性化推荐和智能推荐功能内部化
- **12:30** - 开发预测分析能力:利用投票、评论、浏览、点击等先导指标预测优惠转化
- **13:00** - 构建优惠评分机制,增强对消费者参与度的理解

### 实施方案 (13:30 - 16:00)
- **13:30** - 数据平台迁移:从本地SQL Server数据仓库迁移到Databricks
- **14:00** - 应用程序迁移:从本地VM的LAMP架构迁移到EKS编排
- **14:30** - 增强数据收集:构建更好的实时数据采集平台
- **15:00** - 内部化个性化和搜索功能:构建专用解决方案,更好地利用业务理解

### 个性化架构详解 (16:00 - 19:30)
- **16:00** - 介绍两阶段个性化架构:检索阶段和排序阶段
- **16:30** - 检索阶段:使用Siamese模型基于文本嵌入理解相似优惠
- **17:15** - 过滤阶段:基于启发式规则过滤,例如根据展示次数移除已多次展示的优惠
- **18:00** - 排序阶段:使用XGBoost模型,基于优惠类别、价格点等特征进行排序
- **19:00** - 输出个性化优惠列表,在整个网站中使用

### 优惠评分框架 (19:30 - 22:00)
- **19:30** - 介绍两种评分:参与度评分和内容评分
- **20:00** - 参与度评分:替代延迟的收入数据,跟踪投票、评论、点击等多种参与类型
- **20:45** - 内容评分:不考虑参与度,基于优惠上下文(类别、折扣、品牌、描述、图片)预测表现
- **21:30** - 应用场景:运营工具、优惠质量保证、优惠采购、个性化排序、性能分析

### 技术架构 (22:00 - 25:00)
- **22:00** - 数据采集架构:基于像素的平台,客户端和服务器端跟踪
- **22:30** - 数据流:通过Amazon API Gateway → Lambda → Kafka → Databricks
- **23:15** - 搜索和个性化:基于EKS,使用Elasticsearch
- **24:00** - 使用SageMaker进行查询分类等功能
- **24:30** - 未来计划:探索Nova基础模型应用于优惠评分

### 实时演示 (25:00 - 28:00)
- **25:00** - 展示Slick Deals首页和"Just For You"个性化推荐区域
- **25:30** - 演示实时数据流:在Databricks中查看结构化流事件
- **26:00** - 模拟用户行为:浏览家居改善类优惠,30秒内在Databricks中看到浏览事件
- **26:45** - 投票互动:对优惠进行投票,实时数据流中显示投票事件
- **27:30** - 外链点击:点击优惠跳转到商家网站,这是实时收集的最后一个数据点
- **28:00** - 刷新页面:个性化推荐已更新,显示更多家居改善类优惠

### 业务成果 (28:00 - 30:00)
- **28:00** - 创新能力提升:测试周期从6周缩短到1周
- **28:45** - 优惠评分延迟:从3小时降低到30秒
- **29:15** - 业务指标:商家外链点击和收入增长7%

### 关键要点和总结 (30:00 - 32:30)
- **30:00** - 关注快速扩展能力:减少基础设施关注,增加创新关注
- **30:45** - 聚焦差异化优势:利用业务独特之处,使其成为每个项目的核心
- **31:15** - 数据基础:建立能够实时响应和做出运营决策的数据平台
- **31:45** - 反馈循环:利用短周期测试和学习,快速迭代
- **32:00** - 核心总结:现代化改造释放了创新能力,增强了差异化竞争力
