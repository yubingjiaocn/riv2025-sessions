# AWS re:Invent 2025 技术会议总结：使用代理AI现代化信贷风险应用

## 会议概述

本次会议由Cloudera全球AI解决方案总监Adrian主讲，重点探讨了如何利用代理AI（Agentic AI）技术帮助银行现代化信贷风险申请流程，特别是针对商业银行和中小企业贷款领域。根据麦肯锡的研究，生成式AI在银行业有望创造3400亿美元的潜在价值，但实现这一目标仍面临诸多挑战，包括与现有系统的集成、人员培训、以及AI安全防护措施等。

演讲者强调，与个人抵押贷款相比，中小企业贷款流程的自动化程度较低，存在大量人工干预环节，这导致运营成本高昂且效率低下。通过部署代理AI系统，银行可以实现文档审核自动化、信贷备忘录与政策合规性检查、以及贷款管理等关键流程的优化，预计可带来50%的流程改进。会议展示了一个基于Cloudera AI平台的实际应用案例，该系统能够自动提取信贷政策规则、创建智能代理、并对信贷申请进行合规性评估。

- 麦肯锡预测：3400亿美元的潜在价值创造
- 三大挑战：与现有流程集成、人员培训、AI安全防护措施

00:03:00 - 私有AI的优势
- 所有服务部署在客户自己的VPC内

- 开源技术栈，避免供应商锁定

00:04:00 - 代理AI在商业银行的应用价值
- 商业银行和中小企业贷款流程改进潜力达50%
- 个人抵押贷款已高度自动化，但企业贷款仍需大量人工处理
- 中小企业贷款的运营成本高，利差大

00:05:00 - 理想与现实的差距
- 理想状态：完美信息、完美信任、完美决策
- 现实挑战：文档质量参差不齐、信任度不足、决策不确定性高

00:06:00 - 贷款发放流程的摩擦点
- 流程包括：申请、验证、承销、审批
- 主要障碍：物理文档、人工编写的政策文件、大量人工干预
- 机器学习模型在中小企业贷款中仍需大量专家输入

00:07:30 - AI驱动的解决方案
- AI驱动的贷款发放：文档审核自动化、信贷备忘录与政策合规性检查
- AI驱动的贷款管理：自动催收电话、风险审查自动化
- 风险审查案例：董事会成员变更时的自动风险评估

00:09:00 - 核心用例：政策合规性检查
- 重点解决：信贷政策与信贷备忘录的匹配问题
- 信贷政策由人类编写，需要AI系统理解并执行
- 示例：贷款价值比（LTV）必须达到80%的要求

00:10:00 - 代理创建流程
- 通过读取政策文档，AI系统自动创建代理
- 每个政策规则对应一个代理
- 示例中创建了数百个代理

00:11:00 - 技术架构
- 基于Cloudera平台构建
- 使用Neo4j图数据库管理代理关系
- 使用Docling进行文档处理
- 流程：提取代理→评估需求→存储到图数据库→检索代理→评估合规性

00:12:00 - 现场演示开始
- 展示Cloudera AI Workbench应用界面

00:13:00 - 代理提取示例
- 示例规则：信用报告年龄限制（申请必须在30天内）
- 系统将规则转换为JSON格式
- 支持自动选择相关代理

00:14:00 - 上传抵押贷款申请
- 上传包含收入等信息的抵押贷款申请文档
- 系统构建知识图谱，建立规则之间的关系

00:15:00 - 合规性评估结果

- 首次购房者最低首付：通过（实际9%，要求3%）
- 系统能够进行自动计算和推理

00:16:30 - 可观测性工具展示
