# AWS re:Invent 2025 会议总结：AI 代理 - 特权机器身份的新阶段

## 会议概述

本次会议由 CyberArk 的 Venu Shastri(平台和 AI 解决方案产品营销高级总监)和 Anatan Dvidi(产品管理高级经理)主讲,重点探讨了 AI 代理作为特权机器身份的新兴安全挑战。会议强调,随着企业大规模采用 AI 代理技术,这些代理正在成为一种新型的身份类别,它们既具有人类身份的特征(如推理和决策能力),又具有机器身份的特性(如高速运行和规模化操作)。

演讲者指出,虽然 AI 代理具有巨大的商业潜力,可以为各行业创造数万亿美元的价值,但它们也带来了前所未有的安全风险。这些风险不仅来自外部威胁行为者,还源于 AI 代理本身可能出现的幻觉或误解上下文的情况。CyberArk 宣布将在年底前正式推出其安全 AI 代理解决方案,该方案结合了人类身份和机器身份的安全控制措施,旨在为 AI 代理提供全面的特权控制。会议还介绍了与 AWS Bedrock 的集成,以及多个开源安全工具包,帮助开发者在构建 AI 代理时采用安全最佳实践。

## 详细时间线

00:00 - 开场介绍
- 演讲者自我介绍:Venu Shastri 和 Anatan Dvidi
- 会议主题:AI 代理作为特权机器身份的新阶段

01:30 - CyberArk 公司介绍
- CyberArk 是身份安全领域的领导者
- 提供全面的平台保护人类、机器和 AI 代理身份
- 提供特权访问管理(PAM)、身份访问管理、威胁检测等功能
- 宣布 Palo Alto Networks 收购 CyberArk 的意向

03:45 - AI 代理的潜力与风险
- AI 代理可为各行业创造数万亿美元价值
- 早期实验显示真实可衡量的收益
- 但也带来前所未有的安全风险,特别是在获得企业资源访问权限时
- 风险不仅来自外部威胁,还包括 AI 代理的幻觉和误解

05:20 - AI 代理成熟度级别
- 当前阶段:AI 助手(如 Copilot)
- 中级阶段:基于事件触发的自主代理
- 高级阶段:多层编排代理系统,执行跨系统的多步骤任务
- 随着复杂性增加,风险呈指数级增长

07:10 - 模型上下文协议(MCP)标准
- Anthropic 推出的 MCP 标准
- 第一版未充分解决身份验证和授权问题
- 下一版本将重点使用 OAuth 标准来访问资源

08:00 - OWASP AI 代理十大风险
- 不同风险针对代理系统的不同组件
- 代理行为风险:针对 LLM 的操纵和越狱
- 代理供应链风险:针对应用程序
- 共同主题:利用代理的特权访问权限

09:30 - 身份安全的必要性
- AWS 的 Swami 在主题演讲中强调身份是关键控制平面
- AI 代理是新的身份类别,结合人类和机器特征
- 需要将 AI 代理视为特权机器身份

11:00 - CyberArk 安全 AI 代理解决方案
- 宣布年底前正式推出
- 结合人类和机器身份安全控制
- 包括凭证管理、零常驻特权、会话监控等功能
- 支持强身份验证、密钥轮换、证书管理、PKI、工作负载访问等

12:45 - 解决方案的四大支柱
- 发现和上下文:识别组织中运行的代理
- 安全访问:确保代理安全运行,实施零常驻特权
- 威胁检测和响应:快速应对异常或被攻陷的代理
- 治理和生命周期:确保合规性

13:30 - AWS Bedrock 集成
- AWS Bedrock 是目标代理生态系统之一
- 支持发现和上下文功能

14:00 - Anatan 接手:安全访问详解
- 三大安全挑战:控制代理行为、防止未授权操作、监控和审计

15:30 - 访问控制复杂性示例
- 报告代理场景:不同角色(HR、营销、财务)访问不同数据库
- 需要基于用户、代理、风险和上下文的动态访问策略

17:20 - CyberArk AI 代理网关
- 作为 AI 代理与工具/目标之间的执行点
- 控制访问、实现人工参与、确保最小特权、提供审计追溯

18:45 - 动态上下文感知策略
- 传统访问控制对 AI 代理过于静态
- 需要考虑用户、代理、风险、上下文等多种因素

20:00 - 最小特权原则
- 过度权限是人类和机器的巨大问题
- AI 代理被攻陷后,损害可在几分钟内扩大
- 实施即时授权(JIT)和事后撤销

21:30 - MCP 服务器解决方案
- 已在 AWS Marketplace 提供
- 为 AI 代理提供最小特权和即时访问

22:15 - 恢复可见性和审计
- 当前审计追踪不足以应对 AI 领域
- 需要了解是用户还是代理执行了操作
- AI 代理网关提供完整的可见性

23:30 - 开源安全工具包
- 解决开发者将 API 密钥加载到环境变量的常见做法
- 减少内存暴露和意外泄漏风险
- 支持 AWS Secrets Manager 和 CyberArk Conjur
- 已在 GitHub 上提供

25:00 - 总结与行动建议
- AI 代理将长期存在,需要立即开始安全规划
- 使用身份安全作为基础
- 实施最小特权、即时访问和足够访问

26:30 - 后续步骤
- 解决方案将于本月底正式推出
- 提供与 McKinsey 合作的白皮书,研究代理 AI 采用和风险
- 提供技术白皮书,介绍安全管理身份的关键要求
- 开发者工具:CyberArk MCP 服务器和 CyberArk Agent Guard
- 所有工具均可在 AWS Marketplace 下载

28:00 - 会议结束
- 感谢参会者,邀请到展位进一步交流
