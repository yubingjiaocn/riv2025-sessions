# AWS re:Invent 2025 Agentic Factory 会议总结

## 会议概述

本次会议由来自AWS的Goro Aurora、Aaron Sam和Slalom的Stefan Buckman共同主持,重点探讨了如何从构建少量AI代理扩展到数百甚至数千个代理的挑战与解决方案。会议指出,根据Gartner预测,到2028年行业将拥有13亿个AI代理,这意味着每个3-5人的团队将与50-100个代理协同工作,这是工作方式的根本性转变。为应对这一趋势,AWS推出了Agentic Factory计划,这是一个面向合作伙伴和客户的创新框架,旨在帮助组织以可重复、安全且高效的方式大规模部署AI代理系统。

会议深入分析了企业在AI代理部署过程中面临的六大核心挑战:技术基础薄弱、数据孤岛、安全与治理缺失、领导层战略不一致、价值实现困难以及缺乏可重复模式。针对这些挑战,Agentic Factory提供了完整的解决方案,包括评估(Assess)、规划(Plan)、实施(Implement)和迭代(Iterate)四个核心流程,每个流程都由后端代理支持。该工厂框架内置了安全治理机制,涵盖身份访问、数据安全、模型安全、运营安全、治理合规和运行时控制六大支柱,确保代理系统从设计之初就符合最佳实践和监管要求。

会议还展示了Agentic Factory平台的实际演示,包括代理目录、自动制造(Fabricator)功能、集成策略以及用户角色管理等核心能力。该平台支持与AWS Bedrock、各类数据库、API、事件总线以及MCP服务器的无缝集成,使代理能够访问所需的工具和数据。通过提供蓝图、加速器和平台入门套件,AWS希望帮助组织像使用"宜家平板包"一样快速搭建自己的代理工厂,实现从概念验证到规模化生产的跨越。

## 详细时间线与关键要点

00:00:00 - 开场与观众调研
- 主持人Goro Aurora进行现场调研,了解与会者构建AI代理的规模
- 少数组织已构建10-15个代理,极少数构建了100+个代理
- 强调未来组织将需要管理数百甚至数千个代理

00:01:30 - 团队介绍
- Goro Aurora:AWS澳新地区合作伙伴解决方案架构团队负责人
- Aaron Sam:亚太及日本地区下一代技术负责人
- Stefan Buckman:Slalom解决方案负责人

00:02:15 - 会议目标设定
- 行业背景与机遇分析
- 挑战识别与解决方案探讨
- Agentic Factory深度解析与演示
- 合作伙伴实践经验分享

00:03:00 - 行业趋势分析
- AI代理市场将从2025年的80亿美元增长到2032年的980亿美元,年复合增长率45%
- 2025年前三季度,50%的风险投资流向AI优先企业
- 当前5%的企业软件嵌入AI代理,2026年将增长至40%

00:04:30 - 从生成式AI到Agentic AI的演进
- 第一阶段:生成式AI聊天机器人(ChatGPT类应用)
- 第二阶段:简单代理(能够分解任务、调用API)
- 第三阶段:多代理编排系统(处理复杂多步骤业务流程)

00:05:45 - 代理崛起的规模预测
- Gartner预测:到2028年将有13亿个代理
- 3-5人团队将与50-100个代理协同工作
- 需要新的基础设施和思维方式来支持代理生态

00:06:30 - Stefan分享客户面临的六大挑战

挑战1:技术基础薄弱(00:07:00)
- 零售客户案例:缺乏强大的技术基础导致代理部署受阻
- 需要具备治理能力的可扩展部署平台

挑战2:数据孤岛(00:07:45)
- 制造业客户案例:运输记录和库存数据分散在不同数据库
- 数据不可访问导致AI缺乏正确上下文

挑战3:安全、信任与治理(00:08:30)
- 金融服务客户案例:监管控制要求严格
- 需要可观测性框架和可信答案验证机制

挑战4:领导层战略不一致(00:09:15)
- AI已扩散到整个组织,不再局限于单一部门
- 需要统一方向和技能提升策略
- 关键思维能力的转变至关重要

挑战5:价值实现困难(00:10:00)
- 许多代理项目停留在POC阶段,无法进入生产
- 必须明确ROI和业务价值
- 缺乏价值衡量机制将阻碍项目推进

挑战6:缺乏可重复模式(00:10:45)
- 制造业客户案例:每次从零开始构建AI工作负载
- 扩展到数百个代理时需要可重复的蓝图和模式

00:11:30 - Agentic Factory解决方案介绍
- 定位:AWS面向合作伙伴和客户的创新计划
- 目标:从业务用例概念到规模化生产
- 提供加速器、资产、蓝图、框架和playbook

00:12:15 - Factory核心组件
- 平台入门套件:内置代理、蓝图和集成
- 安全治理:原生内置,而非事后添加
- 类比"宜家平板包":提供构建自有工厂的所有组件

00:13:00 - Factory四大核心流程

流程1:评估(Assess)(00:13:30)
- 接收用例输入(如贷款发起流程)
- 根据业务、技术、商业和治理标准进行资格审查
- 通过问答和文档上传生成评估报告

流程2:规划(Plan)(00:14:15)
- 详细设计和资源规划
- 时间线和冲刺规划
- 环境准备和合规要求确认

流程3:实施(Implement)(00:14:45)
- 在工厂环境或外部AWS账户中部署代理系统
- 包含部署、QA测试和上线流程
- 支持非生产和生产环境

流程4:迭代(Iterate)(00:15:15)
- 递归学习机制(关键差异化特性)
- 收集幻觉率、错误率、正常运行时间等指标
- 持续优化成本和性能

00:16:00 - 集成策略
- 代理间通信(Agent-to-Agent)
- API和工具集成
- 数据访问集成
- 支持Bedrock、GraphQL、REST、A2A等协议

00:17:00 - 事件驱动架构
- EventBridge作为核心消息中心
- 支持SNS、SQS、Kafka、MQTT
- 实现实时响应和工作流触发

00:17:45 - 数据连接器
- S3集成
- 多种向量数据库支持
- SQL和JDBC连接
- 安全数据访问机制

00:18:30 - 外部系统集成
- SAML和OAuth 2.0身份认证
- MCP服务器支持(自有、AWS和第三方)
- 为代理提供必要的工具和数据访问

00:19:15 - AI代理与代理蓝图概念
- AI代理:新AI世界的微服务,具有明确范围、上下文和目的
- 代理蓝图:业务代理工作流的编排(如贷款发起流程)
- 蓝图包含多个代理、对象和数据资产的协调

00:20:30 - 可重复性价值
- 代理作为基础构建块
- 蓝图作为抽象业务构建块
- 跨用例复用,节省时间、成本和资源

00:21:15 - 安全与治理六大支柱

支柱1:身份与访问(00:21:30)
- 人员操作工厂的身份管理
- 代理和代理蓝图的访问控制

支柱2:数据安全(00:22:00)
- 最小权限数据访问
- PII和敏感数据过滤
- 加密机制

支柱3:模型安全(00:22:30)
- 与Bedrock和Bedrock Agent Core集成
- 防止提示注入的护栏
- 输入输出验证

支柱4:运营安全(00:23:00)
- 代理沙箱隔离
- 工具访问监控和审计
- 安全执行环境

支柱5:治理合规(00:23:45)
- 实施AWS负责任AI最佳实践
- 路线图包含ISO和NIST标准对齐
- 内置合规控制

支柱6:运行时控制(00:24:15)
- 代理门控(Agent Gating):敏感操作需人工审批
- 反馈循环完整性:防止偏见重新引入系统
- 人机协同机制

00:25:00 - Aaron技术架构深度解析
- 工厂需要标准化模式和可重复治理
- 利用非确定性AI实现工作流的涌现行为
- 通过护栏控制幻觉和非预期输出

00:26:30 - 可扩展性与互操作性
- 工厂和代理解决方案的可扩展性
- 代理间互操作(平台内和第三方平台)
- 与内部系统集成能力

00:27:15 - Factory技术蓝图
- 基于AWS Well-Architected Framework
- 整合一年的代理解决方案研发成果
- 包含Supervisor、Arbiter、Fabricator等前沿模式

00:28:00 - 合作伙伴生态系统价值
- 合作伙伴经验和资产
- 客户环境理解
- 治理和约束要求知识

00:29:00 - 知识库与记忆层
- 整合AWS、合作伙伴和客户知识
- 作为代理系统的共享记忆层
- 支持推理周期和规划

00:30:15 - Factory生命周期代理
- 评估代理:需求获取和差距分析
- 规划代理:设计、资源和时间线规划
- 实施能力:规范驱动开发、项目管理集成、自动制造

00:32:00 - 开发集成选项
- AI辅助开发(Kirao、Q Developer)
- 工厂内自动制造
- 合作伙伴定制模块开发

00:33:00 - 编排与可观测性
- 工作流执行编排
- 指标、日志和行为监控
- 持续改进和迭代

00:34:00 - 统一UI界面
- 单一接口管理整个平台
- 减少跨系统查询的安全层
- 安全优先原则

00:35:00 - 组织级管理
- 用户、角色、团队和组织管理
- 支持数百到数千代理的治理
- 部门和团队级别的控制

00:36:30 - 平台演示开始
- 展示预览版本功能
- 身份验证和授权流程
- 注册和登录机制

00:37:30 - 仪表板概览
- 指标、日志和可观测性数据
- 代理健康状态监控
- 工厂接收和部署状态

00:38:15 - 组织与角色管理
- 新用户进入"大厅"(Lobby)等待分配
- 管理员分配组织、团队和角色
- 开发者、架构师、项目负责人等角色

00:39:30 - 代理目录(Agent Catalog)
- 内置代理展示
- Fabricator:核心仲裁模式代理
- 用于监督工作流的工作代理创建

- 即时创建代理:读取S3文档、生成摘要、批判性分析、上传到文档存储
- 工具选择:Strands SDK工具或自定义工具

00:42:30 - 自动制造流程

00:43:45 - 代理生命周期管理
- 初始状态为停用,需人工审核
- 检查合规性和完整性

00:44:30 - 集成重要性强调
- 代理必须与外部系统集成才有价值
- 支持内部和第三方系统连接

00:46:30 - 思维链可视化
