# AWS re:Invent 2025 - AWS 欧洲主权云技术会议总结

## 会议概述

本次会议由 AWS 副总裁兼杰出工程师 Colin McCarthik 和 EC2 团队首席产品经理 Addie 主讲，深入介绍了 AWS 欧洲主权云（AWS European Sovereign Cloud）的设计理念、技术架构和运营模式。Colin 拥有 18 年 AWS 工作经验，同时具有欧盟和美国双重公民身份，他强调主权云项目结合了他在公共服务、隐私保护、加密和虚拟化等多个领域的专业背景。

AWS 欧洲主权云是一个完全独立的云基础设施，专为欧盟公共部门和受监管行业设计。该项目投资 78 亿美元，位于德国勃兰登堡州，由欧盟公民独立运营，确保数据和元数据完全保留在欧盟境内。会议详细阐述了 AWS 的"主权设计"（Sovereign by Design）理念，以及欧洲主权云如何在提供完整 AWS 服务能力的同时，满足最严格的数字主权要求，包括运营自主性、持续运营能力和独立治理结构。

## 详细时间线与关键要点

### **开场介绍** (00:00 - 03:30)
- **00:00** - Colin McCarthik 欢迎参会者，介绍自己是 AWS 副总裁兼杰出工程师，在 AWS 工作 18 年
- **01:15** - Colin 分享个人背景：拥有欧盟和美国双重公民身份，曾在爱尔兰公共部门工作，参与过 GÉANT 和 CERN 等泛欧项目
- **02:20** - Addie 自我介绍：EC2 团队首席产品经理，负责数字主权和欧洲主权云项目
- **03:00** - 会议议程预告：讨论主权重要性、主权设计模式、欧洲主权云差异化特点

### **主权的重要性** (03:30 - 08:00)
- **03:30** - Colin 阐述主权国家和国际联盟（如欧盟）是人类最伟大的成就，减少了战争、冲突和饥荒
- **04:45** - 强调主权客户面临的增强要求：安全性、自主性和持续性
- **05:30** - AWS 理念：主权客户应获得所需的全部控制权，但不做任何妥协
- **06:20** - 指出传统方案的问题：主权客户常被困在小规模本地部署或功能受限的系统中
- **07:10** - 现实风险：政府机构面临的日常安全威胁（数据泄露、钓鱼攻击、勒索软件），有时由外国行为者支持

### **AWS 的完整能力** (08:00 - 11:30)
- **08:00** - 强调 AWS 希望为主权客户提供完整的超大规模服务，无需妥协
- **08:45** - 数据中心规模对比：AWS 可用区的规模相当于宜家商店大小，充满服务器
- **09:30** - 使用医院类比：以莱顿大学医学中心为例，说明 AWS 服务的综合性和协同性
- **10:20** - 每个部门（如 S3、EC2）都是世界级设施，但整合在一起能提供更强大的能力
- **11:00** - 目标：让客户能够构建像 Amazon.com 一样复杂的工作负载或运行先进的机器学习模型

### **主权设计原则** (11:30 - 17:00)
- **11:30** - 大多数公共部门业务实际上在 AWS 常规商业区域运行
- **12:00** - 合规性挑战：主权和政府客户需要遵守各自的国家或国际法规
- **12:45** - 监管机构反馈：欧洲各地监管机构普遍人手不足，有大量待认证工作积压
- **13:20** - AWS 从第一天起的设计理念：构建"主权设计"系统
- **14:00** - 2008 年关键决策：在爱尔兰建立第二个区域时，选择显示"接缝"而非无缝扩展
- **15:00** - 数据驻留原则：客户在特定 AWS 区域存储的数据会保留在那里，AWS 不会移动数据
- **15:45** - 数据隔离理念：AWS 将客户数据视为"放射性"，不希望接触或查看
- **16:30** - 核心架构：Nitro 系统和 KMS 确保 AWS 无技术手段访问客户数据

### **区域独立性** (17:00 - 20:30)
- **17:00** - 每个 AWS 区域几乎完全独立运营
- **17:30** - 技术实现：每个区域都有独立的 EC2 服务副本，自主运行
- **18:15** - 可用性优势：一个区域的问题不会级联影响其他区域
- **18:45** - 全球规模：目前有近 40 个公开商用区域，持续增加中
- **19:20** - 现有区域已满足大多数主权工作负载需求
- **20:00** - 特殊需求：某些情况需要超越数据主权的保证，涉及运营者身份和法律治理

### **专用主权云经验** (20:30 - 22:30)
- **20:30** - AWS 在美国拥有 GovCloud，专为美国政府客户设计，由美国公民独家运营
- **21:00** - 中国专用云：为中国客户和业务建立，独立运营
- **21:30** - 成功经验：AWS 知道如何构建完整的超大规模基础设施，同时实现最高级别的主权分离
- **22:00** - 欧洲主权云目标：不仅实现数据主权，还包括运营主权和政府控制

### **欧洲主权云核心特性** (22:30 - 28:00)
- **22:30** - Addie 介绍欧洲主权云：2023 年 10 月预发布，基于 AWS 数十年运营经验
- **23:00** - 地理位置：位于德国勃兰登堡州，在德国母公司下成立
- **23:30** - 技术架构：全新分区，物理和逻辑上与其他 AWS 区域完全分离
- **24:00** - 独立系统：拥有独立的身份访问管理（IAM）和计费系统
- **24:45** - 账户分离：客户需要创建新账户才能访问欧洲主权云服务
- **25:15** - 增强数据驻留：不仅客户内容，客户创建的元数据也保留在欧盟内
- **26:00** - 元数据定义：资源标签、标记、配置等客户创建的管理数据
- **26:45** - 运营自主性：欧盟以外零运营控制，所有运营由欧盟公民完成
- **27:30** - 运营范围：从数据中心运营到服务运营、日常运营和客户支持

### **治理结构与关键特性** (28:00 - 33:00)
- **28:00** - 过渡期：目前采用欧盟居民和欧盟公民混合工作模式，逐步过渡到完全由欧盟公民运营
- **28:45** - 独立治理结构：建立新实体和独立治理体系
- **29:15** - 无限期运营：确保在地缘政治危机、技术故障或自然灾害情况下持续运营
- **30:00** - 无关键依赖：基础设施层面不依赖欧盟以外的资源
- **30:45** - 源代码副本：运行欧洲主权云所需的服务源代码副本存储在欧盟境内
- **31:30** - 主权参考框架：将主权要求转化为可审计的控制措施
- **32:15** - 框架来源：基于客户和监管机构反馈，遵循行业领先标准
- **32:45** - 专用安全运营中心：将在后续详细介绍

### **信任服务与定价** (33:00 - 37:00)
- **33:00** - 欧洲信任服务提供商：首次在欧盟内控制密钥证书材料和证书的发放
- **33:45** - 自主能力：在断开连接等情况下，仍可在欧盟内自主发放证书
- **34:30** - 核心原则：运营欧洲主权云所需的一切（人才、技术、基础设施、领导层）都在欧盟内
- **35:00** - 治理结构：母公司为 AWS European Sovereign Cloud GmbH
- **35:30** - 管理团队：Stefan Israel 担任董事总经理，配备安全官，均为欧盟公民
- **36:00** - 咨询委员会：包括 4 名成员，其中 3 名亚马逊员工和 1 名独立成员
- **36:45** - 定价体验：端到端欧元定价，包括所有服务、储蓄计划和支持计划

### **技术实现细节** (37:00 - 43:00)
- **37:00** - Colin 详细介绍后端系统：门禁系统、电子邮件等企业系统都在欧洲主权云内自主运营
- **37:45** - 信任服务提供商：世界首个专用欧洲根证书颁发机构
- **38:30** - 证书服务：负载均衡器、CloudFront 分发或 EC2 实例的 SSL 证书直接来自欧洲主权云
- **39:00** - 专用 DNS 服务：在欧洲主权云内构建 Amazon Route 53 专用版本
- **39:45** - 域名策略：使用欧洲域名（.de、.fr 等）而非 .com、.net、.org、.co.uk
- **40:30** - 专用网络和骨干网：连接数据中心和可用区的链路在欧洲主权云内管理
- **41:00** - Direct Connect：专用的 Direct Connect 位置
- **41:30** - 互联网连接：连接到全球 AWS 骨干网，同时拥有来自欧洲提供商的专用备份连接
- **42:15** - 安全运营中心：专用安全团队和领导，全面监督所有安全监控

### **合规性与投资** (43:00 - 47:00)
- **43:00** - 合规目标：简化客户工作负载认证流程
- **43:30** - 元数据驻留优势：客户不再需要琐碎的管理数据例外
- **44:00** - 主权控制文档：与欧洲监管机构合作，详尽记录所有主权控制措施
- **44:45** - 文档内容：包括控制措施的实施方式、原因和基于证据的流程
- **45:15** - AWS 对欧洲的承诺：欧盟拥有 AWS 在美国以外最大的工程团队
- **45:45** - 开发中心：位于德累斯顿、都柏林和柏林
- **46:15** - 核心服务：AWS Nitro 和 CloudWatch 等核心服务在欧洲开发
- **46:45** - 投资规模：78 亿美元投资，致力于欧盟的社会经济发展

### **合作伙伴与目标客户** (47:00 - 50:00)
- **47:00** - 合作伙伴价值：合作伙伴最接近客户，了解工作负载和用例
- **47:30** - 合作伙伴产品：欧洲主权云将提供大量合作伙伴产品
- **48:00** - 目标客户：公共部门和受监管行业客户
- **48:30** - 明确定位：针对因主权问题尚未上云的客户和工作负载
- **49:00** - 最佳实践一：建立并理解用例，确定哪些工作负载需要迁移到欧洲主权云
- **49:45** - 重要提醒：不需要将现有区域的所有内容迁移到欧洲主权云，现有区域仍提供行业领先的合规标准

### **工作负载评估建议** (50:00 - 结束)
- **50:00** - 最佳实践二：仔细考虑需要额外主权控制的工作负载类型
- **50:30** - 评估标准：基于实际需求或要求决定是否迁移到欧洲主权云
- **50:45** - 会议总结：Colin 将返回进行最后总结
