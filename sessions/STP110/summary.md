# AWS re:Invent 2025 会议总结

## 会议概述

本次会议由Rocks公司联合创始人兼CTO Shriram主讲,重点介绍了AI原生收入代理系统如何变革企业收入团队的工作方式。Shriram曾在AWS工作7年,参与构建了Amazon Aurora数据库,并于2014年在re:Invent上首次发布该产品。Rocks公司专注于为全球2000强企业构建收入代理,旨在通过AI技术实现收入团队生产力的10倍提升。

公司自2024年从隐身模式推出以来,已与5000多个组织合作,涵盖金融服务、能源、医疗、半导体等多个垂直领域,并获得了Sequoia、GC和GV的投资支持。演讲深入探讨了传统收入系统的局限性、AI原生架构的优势,以及Rocks如何帮助企业从试点项目过渡到大规模生产部署。公司目前每月处理4000亿个token,是OpenAI"万亿token俱乐部"仅有的20家组织之一,展示了其在企业AI领域的实际规模化应用能力。

Rocks构建了一个四层架构的AI原生收入操作系统,包括数据层、知识图谱层、代理集群层和交付层,通过多代理协同工作实现从潜在客户到成交、续约、扩展的全生命周期管理。公司采用前置部署工程团队的方式,而非依赖客户IT团队自建代理,确保快速交付可衡量的业务价值。

## 详细时间线与关键要点

00:00:00 - 开场介绍
- 演讲者Shriram自我介绍,Rocks公司联合创始人兼CTO
- 曾在AWS工作7年,参与构建Amazon Aurora数据库
- Rocks致力于为全球2000强企业构建收入代理

00:01:30 - 公司背景与使命
- 公司使命:保护和增长全球收入
- 团队成员来自AWS、Google、Stanford、Berkeley等
- 2024年从隐身模式推出,已服务5000+组织
- 获得Sequoia、GC和GV投资支持

00:02:45 - 收入系统的历史演变
- 本地部署时代:Siebel作为单一整体CRM系统,功能强大但僵化、昂贵、难以操作
- 云时代:Salesforce和HubSpot成为记录系统,但记录系统与行动系统分离
- 云2.0时代:数据迁移到Snowflake、Databricks、Redshift等数据仓库
- AI原生时代:记录系统与参与系统重新整合

00:04:15 - 收入团队面临的挑战
- 数据碎片化:分散在CRM、支持系统、产品使用数据、通话记录、邮件等多个系统
- 工作流程手动且不一致:销售代表需要花费大量时间进行账户研究和邮件撰写
- BCG研究显示:销售代表仅20%时间用于实际销售,80%时间用于行政工作

00:05:30 - Rocks的AI原生架构
- 四层架构设计:
  1. 数据层:整合结构化和非结构化数据
  2. 知识图谱/数据织物层:包含实体和关系的元数据
  3. 代理集群层:企业级即用型代理
  4. 交付层:通过iOS、Slack、Web等多渠道交付

00:07:00 - 代理集群(Agent Swarm)概念
- 每个销售代表的每个账户都配备AI代理集群
- 代理协同工作,覆盖整个收入生命周期
- 从洞察到行动的自动化转换

00:08:30 - 平台技术架构
- 完全构建在AWS之上
- 客户数据平面与Rocks数据平面清晰分离
- 支持安全数据共享或数据导入
- 使用AWS全套服务:智能、计算、内容摄取、编排等

00:09:45 - 多代理协同机制
- 不使用单一通用代理
- 编排代理将工作流分解为步骤序列
- 为每个步骤选择合适的专用代理
- 共享工作记忆和统一知识图谱
- 确保可预测、一致、可重复的执行

00:11:30 - 实际案例演示
- 复杂查询处理:整合财报电话会议、新闻稿、职位发布、产品信号等
- 守护代理:检查权限和内容安全性
- 查询扩展代理:优化搜索查询
- 并行网络搜索和推理合成
- 格式化和验证代理确保输出质量

00:13:00 - 规模化挑战
- 模型故障转移和可靠性保障
- 跨不同提供商的提示工程
- 评估和守护机制
- 成本优化,避免token浪费

00:14:15 - 规模化运营数据
- 每月处理4000亿token
- OpenAI"万亿token俱乐部"成员(仅20家组织)
- 这不是实验,而是收入团队的日常运营依赖

00:15:30 - 企业AI采用现状
- 收入领导者期望Gen AI在未来几年成为关键技术
- 愿景:在不增加人员的情况下提高每个销售代表的收入
- 现实:AI仍是附加功能,未集成到核心工作流
- MIT案例研究:95%的Gen AI试点项目在生产中失败

00:17:00 - 现有方案的局限性
- 传统CRM:功能强大但僵化,AI策略依赖DIY代理构建器
- 附加工具:需要购买多个供应商产品,集成负担重
- Rocks方案:从零开始构建,作为长期转型合作伙伴

00:18:30 - Rocks的差异化方法
- 不依赖IT团队构建代理或调优提示
- 前置部署工程团队(混合McKinsey、BCG顾问和软件工程师)
- 快速交付可衡量的业务影响

00:19:15 - 实施流程
1. 启动有明确成果的试点项目
2. 培训现有团队和培训师
3. 复制顶尖销售人员的工作流(销售遵循幂律:10%的人带来90%的收入)
4. 跨团队和职能推广
5. 在整个组织中制度化新的收入运营模式

00:21:00 - 客户案例:RAMP
- RAMP是史上增长最快的SaaS公司
- 成果:
  - 发现15%更多的六位数交易
  - 会议准备时间减少90%
  - 预订会议增加20%
- 关键差异:一致性执行,每个销售代表获得相同质量的研究和优先级排序

00:22:30 - 跨客户的共同模式
- 更快的会议准备
- 更高的转化率
- 更好的目标定位
- 更短的新员工培训时间
- 规模化的一致执行

00:23:15 - 总结与展望
- AI代理将从根本上改变收入团队的工作方式
- 行业仍处于转型的早期阶段
- 就像编码代理将工程团队生产力提高10倍一样,收入代理将为企业收入团队带来10倍生产力提升
- 通过为每个账户提供具有完整上下文的全天候代理来实现
