# AWS re:Invent 2025 - AWS Transform 会议总结

## 会议概述

本次会议由AWS Transform副总裁Asa Khalawade主讲,重点介绍了AWS Transform这一首个用于迁移和现代化的代理AI服务。会议以一个生动的比喻开场:企业IT转型就像使用纸质地图跨国驾驶一样低效,而AWS Transform正是为了解决这一问题而生。目前美国有2.41万亿美元的技术债务,70%的工作负载仍在本地部署,财富500强企业中70%的传统IT软件已有20年以上历史。

AWS Transform于2025年5月推出,最初支持三大关键工作负载:VMware迁移、IBM z/OS大型机现代化和.NET框架到.NET Core的现代化。该服务采用多代理、多任务、多用户协作模式,通过专家任务代理、基于目标的编排和持续学习机制,帮助企业实现自动化迁移。自推出以来,已有数百家客户使用Transform,处理了超过10亿行COBOL代码,估计节省了超过80万小时的人工工作。

会议还特别展示了宝马集团(BMW)的大型机现代化案例,以及最新发布的全栈Windows现代化和AWS Transform Custom功能。Transform Custom允许客户创建自定义转换,处理软件更新、API升级等技术债务问题。此外,AWS还宣布了可组合性(Composability)功能,使合作伙伴能够基于自己的知识库和专业经验创建个性化的转型工作流。
