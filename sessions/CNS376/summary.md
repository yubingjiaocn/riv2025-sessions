# AWS re:Invent 2025 - AWS Lambda 产品发布会总结

## 会议概览

本次会议由 AWS Lambda 产品管理负责人主持，重点介绍了 2025 年 Lambda 和无服务器技术的重大变革。演讲者强调，无服务器技术已经发展了十年，2025 年对 Lambda 来说是变革性的一年，因为 AWS 推出了一系列新功能，使得全新类别的工作负载能够在无服务器架构上运行。

会议的核心主题是"速度"——不仅是代码生成的速度，更是整个软件交付周期的速度。Lambda 的价值主张在于让应用程序默认具备良好架构，通过接管所有基础设施管理工作（可扩展性、可用性、可靠性等），大幅减少运维负担，从而让开发者能够更快地迭代和交付产品。演讲者将新发布的功能分为三大类：新原语（支持全新工作负载类型）、默认良好架构（提升应用质量）、以及开发者加速工具（提升开发效率）。

整个演讲节奏紧凑，在 20 分钟内覆盖了十多项重要功能更新，展示了 AWS Lambda 在 2025 年的全面进化，特别是在性能优化、成本控制、开发体验和企业级功能方面的显著提升。

## 详细时间线与关键要点

00:00 - 01:30 | 开场与背景介绍
- 演讲者介绍自己是 AWS Lambda 产品管理负责人
- 强调无服务器技术已存在十年，2025 年是变革性的一年
- 新功能使得全新类别的工作负载可以使用无服务器架构

01:30 - 03:00 | Lambda 的核心价值主张
- 核心差异化优势是"速度"——不仅是代码生成，更是整个交付周期
- 通过默认良好架构减少运维负担（可扩展性、可用性、可靠性等）
- 无需管理基础设施，大幅减少运维工作

03:00 - 06:30 | 新原语：Lambda 托管实例（Managed Instances）
- 解决稳态流量场景的性能和成本优化需求
- 允许用户选择运行 Lambda 函数的具体实例类型
- 支持多并发（Multi-concurrency）：同一执行环境可处理多个请求
- 结合 EC2 定价激励（储蓄计划、预留实例）实现成本大幅节省
- 引入"容量提供者"（Capacity Provider）概念来配置实例偏好

06:30 - 08:00 | 托管实例的使用场景
- 稳态流量应用：始终有用户需要服务的场景
- 性能关键型应用：需要专用实例满足 SLA 要求
- 合规性需求：可指定特定可用区或本地区域运行
- 支持从媒体处理、数据处理到 Web 应用等多种工作负载

08:00 - 09:30 | 新原语：租户隔离（Tenant Isolation）
- 针对 SaaS 构建者和多租户场景
- 自动提供干净的隔离执行环境，无需自定义清理工具
- 只需传入租户唯一标识（租户 ID 或 JWT token）
- 提升 CI/CD 效率，减少自定义工具开发

09:30 - 11:30 | 新运行时支持
- 发布 Python 3.14、Java 25、Node.js 24 运行时
- 目标是在社区发布后 90 天内提供新运行时支持
- 自动进行安全补丁和性能优化
- 提升开发者生产力，更快交付安全软件

11:30 - 13:00 | AWS Transform Custom - 运行时升级工具
- 基于生成式 AI 的运行时升级工具
- 在预览测试中，85% 的函数可以直接接受 AI 建议的升级
- 可集成到 CI/CD 流水线中
- 大幅减少技术债务

13:00 - 14:30 | SnapStart 扩展支持
- 冷启动优化功能，现已支持 Python 和 .NET（此前仅支持 Java）
- 通过快照和恢复机制显著减少初始化时间
- 无需代码更改，仅需勾选复选框启用
- 提升终端用户体验

14:30 - 16:00 | 故障注入服务（FIS）集成
- 集成 AWS Fault Injection Service 进行韧性测试
- 可模拟延迟、下游服务不可用等压力场景
- 帮助规划和预防生产环境故障
- 无需自定义测试工具

16:00 - 18:00 | 事件源增强：CloudWatch 指标
- 为 SQS 和 Kafka 等事件源提供额外的 CloudWatch 指标
- 包括轮询计数、延迟和限流等指标
- 解决了之前需要拼凑多个服务指标的痛点
- 即时检测问题，缩短故障恢复时间

18:00 - 19:30 | Schema Registry 支持
- 支持 Kafka Avro 格式的自动序列化和反序列化
- 无需在每个函数中手动添加样板代码
- 支持 Schema 演进
- 减少代码量和错误

19:30 - 20:30 | SQS 预配置模式（Provision Mode）
- 为 SQS 事件源提供预热容量，即时处理流量峰值
- 继去年 Kafka 预配置模式后的后续功能
- 帮助满足 SLA 要求并优化成本

20:30 - 23:00 | 开发者加速：控制台到 IDE 无缝转换
- 一键将控制台创建的应用打包并在本地 IDE 中打开
- 自动处理所有依赖项
- 无需手动下载或配置

23:00 - 24:30 | 本地测试：LocalStack 集成
- 与 AWS 合作伙伴 LocalStack 深度集成
- 支持在本地完全离线开发和测试
- 模拟存储、数据库、网络等 AWS 服务
- 加快业务逻辑迭代速度

24:30 - 26:00 | 远程调试（Remote Debugging）
- 两次点击即可在云端运行的代码上设置断点
- 像本地调试一样分析变量和源代码
- 可测试 IAM 策略、VPC 网络连接等云端特定场景
- 在部署到生产环境前验证代码

26:00 - 27:30 | MCP 服务器更新
- Model Context Protocol 服务器集成最佳实践
- 生成的代码包含输入验证、错误处理、状态码等
- 提升代码质量和一致性，减少代码审查周期
- 新增对事件驱动应用的支持（如 Kafka 触发器）

27:30 - 29:00 | 其他会议和路线图
- 提及持久化函数（Durable Functions）将在次日会议中详细介绍
- Lambda 路线图现已在 GitHub 上公开
- 欢迎社区参与功能请求和讨论
- 强调 AWS 的客户至上理念

29:00 - 29:30 | 总结与致谢
- 演讲者承认演讲节奏很快，20 分钟内压缩了大量内容
- 邀请参会者参加其他无服务器相关会议
