# AWS re:Invent 2025 Coralogix 可观测性成熟度模型会议总结

## 会议概述

本次技术分享由 Coralogix 的布道师主讲，重点探讨了可观测性的价值实现问题。演讲者指出，许多组织认为可观测性成本过高、价值不明确，但实际问题在于缺乏将技术信号转化为业务洞察的能力。通过一个简单的日志分析案例，演讲者展示了如何将基础的非结构化数据转换为可以向 C 级高管汇报的业务价值信息。

Coralogix 作为一家成立于 2017 年的可观测性平台公司，目前拥有数千客户，在全球 8 个高可用区域部署，包括 GovCloud，并正在申请 FedRAMP 认证。公司以其卓越的客户支持著称，中位响应时间少于 30 秒，问题解决时间少于 1 小时，这在行业中处于领先地位。演讲的核心是介绍 Coralogix 开发的四级可观测性成熟度模型，旨在帮助企业从基础的遥测数据收集发展到深度的业务洞察分析。

## 详细时间线与关键要点

### 0:00-5:00 开场与问题定义
- 提出可观测性的核心问题：成本高昂但价值不明确
- 强调问题本质不是价格过高，而是价值不清晰
- 介绍演讲目标：展示如何将简单数据转化为 C 级高管可理解的业务价值

### 5:00-10:00 Coralogix 公司介绍
- 公司成立于 2017 年，四年内客户数量从 2000 增长到 4000+
- 8 个高可用区域部署，支持 GDPR 合规和数据驻留要求
- 客户支持团队 80 人（公司总员工 600 人），中位响应时间从 5 分钟提升到 30 秒以内
- 全面支持 OpenTelemetry，APM 和追踪功能原生兼容 OTel

### 10:00-15:00 信息流动问题分析
- 批评传统的"技术侧 vs 业务侧"分离模式
- 指出技术事件对整个公司的影响，如 API 延迟会影响营销页面加载速度
- 强调需要通过统一工具实现信息的横向流动，而非仅依赖临时协作

### 15:00-20:00 实践案例：日志数据转换
- 展示如何将非结构化日志转换为业务洞察
- 通过解析和提取关键字段（如产品 ID）进行数据索引
- 使用 DataPrime 查询语言进行数据丰富化，关联产品详情查找表
- 将"RPC 未找到错误"转换为"15 分钟内因 bug 损失 1500 美元"的业务影响描述

### 20:00-25:00 可观测性成熟度模型
- **第一级：遥测覆盖** - 获得日志、指标、追踪、性能分析等全栈遥测数据
- **第二级：工程智能** - 围绕数据库、基础设施定义 SLO，实现团队间一致性度量
- **第三级：生产环境智能** - 将工程知识置于生产环境上下文中，关注客户影响
- **第四级：业务洞察** - 主动挖掘数据发现问题，将技术信号转化为业务影响

### 25:00-30:00 客户案例与总结
- **TradeWeb**：每日 130TB 数据，审计查询时间从两周缩短到秒级
- **Soft2Bet**：每日 140TB 数据，90% 的仪表板被全公司使用，包括业务人员
- **Kaltura**：通过优化 OTel 配置节省 25% 的遥测基础设施成本
- 强调行业正从"响应问题"向"主动挖掘洞察"转变，可观测性平台需要支持跨业务的智能分析
