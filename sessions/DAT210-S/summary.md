# AWS re:Invent 2025 Oracle 多云数据库服务会议总结

## 会议概述

本次会议由Oracle OCI多云产品副总裁Kami Ziggily主持,并邀请了Equifax公司的代表Ain分享客户实践经验。会议重点介绍了Oracle数据库服务在AWS数据中心的原生部署方案,这是Oracle首次将完整的Exadata技术栈和数据库服务直接部署在AWS、Azure和GCP等第三方云服务商的数据中心内。

这项多云战略的核心价值在于让客户能够在AWS环境中使用Oracle的企业级数据库技术,同时享受AWS的云服务和AI能力。通过微秒级延迟的网络连接,应用层和数据库层可以在同一数据中心内无缝协作。Oracle在简化管理运维、数据保护和安全集成三个方面持续创新,特别是新推出的AWS KMS集成功能,让客户可以使用AWS密钥管理服务来保护Oracle数据库。

Equifax作为主要信用机构和数据公司,分享了他们从RDS迁移到Oracle Autonomous Database at AWS的实践。他们的计费系统处理数十亿用户记录,需要高性能OLTP数据库支持。通过采用部署在AWS的Exadata技术,Equifax解决了存储限制问题(从64TB扩展到PB级),提升了性能,并为AI应用做好了数据准备。

## 详细时间线

00:00 - 00:45 - 会议开场与议程介绍
- Kami Ziggily介绍自己是Oracle OCI多云产品副总裁
- 说明会议将包含Oracle概述和Equifax客户案例分享,并预留问答时间

00:45 - 02:30 - Oracle多云战略背景
- 强调Oracle作为关系数据库技术领导者已有40年历史
- 首次将完整的Exadata技术和数据库服务原生部署到AWS数据中心
- Oracle现已在所有主流云平台(AWS、Azure、GCP)提供原生服务

02:30 - 04:45 - 三大创新领域介绍
- **简化管理运维**: 提供基础设施即代码工具,支持API、CLI、SDK、CloudFormation和Terraform
- **数据保护**: 推出Autonomous Recovery Service,提供近实时的自动化数据保护
- **安全集成**: 上周发布AWS KMS集成功能,客户可使用AWS密钥管理服务,同时支持OCI Vault、Oracle Key Vault和TDE

04:45 - 06:30 - 多云服务核心价值
- Oracle数据库服务运行在原生Oracle硬件基础设施上,但位于AWS数据中心
- 与AWS服务实现原生互操作性
- 支持与AWS Bedrock等AI服务的深度集成,可构建智能应用
- 联邦式运维治理和安全数据保护服务

06:30 - 08:15 - Exadata技术优势
- Exadata是运行数据库服务的最佳硬件平台
- 支持区域内零RTO/RPO(恢复时间目标/恢复点目标)
- 提供跨区域的近零延迟异步Data Guard和主-主配置
- 微秒级网络延迟连接应用层和数据库层

08:15 - 09:30 - 服务可用性与采购体验
- 服务已正式发布(GA),拥有数百家企业客户
- 目前在US East和US West 2区域的多个可用区提供服务
- 本月底将新增3个区域,未来9-10个月内将扩展到20多个区域
- 客户可通过AWS Marketplace采购,使用AWS PPPA等财务工具支付

09:30 - 11:00 - Equifax公司介绍
- Ain代表Equifax发言
- Equifax不仅是信用机构,更是数据公司,拥有全面的消费者数据
- 提供三大核心服务:雇主劳动力解决方案、美国信息系统解决方案(抵押贷款服务)、就业验证服务

11:00 - 12:30 - Equifax数据架构
- 构建了内部云平台"数据织物层"(Data Fabric Layer)
- 整合所有消费者数据并提供实时洞察
- 强调多数据资产规模化处理、速度和性能的重要性

12:30 - 15:00 - Equifax使用ADB的原因
- 支持企业数据和财务系统,特别是计费系统
- 计费系统是高度OLTP型数据库,处理数十亿用户记录
- 有三个主要计费周期,需要及时评级、创建账单和发送发票
- 高性能要求至关重要,否则会造成收入损失
- 需要大容量存储,因为数据无法立即删除

15:00 - 18:00 - 从RDS迁移到ADB的关键收益
- **优化性能和吞吐量**: 满足高OLTP系统需求,确保按时完成计费处理
- **可扩展存储**: Exadata提供PB级存储容量,解决了RDS 64TB限制问题(虽然昨天宣布RDS支持256TB)
- 无需每季度清理和归档数据
- Exadata性能随数据增长保持稳定

18:00 - 19:30 - ADB的其他优势
- **统一云工作负载**: 与AWS服务无缝集成,无延迟问题
- 如果使用OCI的Exadata服务,需要处理AWS和OCI之间的延迟
- 在AWS数据中心部署是"两全其美"的方案
- 减少维护活动

19:30 - 20:30 - AI就绪能力
- Oracle 23ai和26ai版本支持AI功能
- 虽然应用层(Oracle计费系统)可能在AI路线图上滞后
- 通过迁移到ADB,数据已为AI做好准备
- 可使用APEX等工具构建自有AI解决方案

20:30 - 21:30 - 会议总结与资源
- Kami提供学习资源链接,建议搜索"Oracle multicloud"
- 会议录像将在几周内发布到AWS网站
- 预留4-6分钟问答时间

21:30 - 22:00 - 会议结束
- 无观众提问
- 演讲者致谢并结束会议
