# AWS re:Invent 2025 技术会议总结

## 会议概述

本次会议由 Trianz 公司主讲，主题为"从数据到企业级智能代理 AI 的零迁移路径"。Trianz 是 AWS ISV 合作伙伴，专注于通过其核心平台 Concierto 提供"服务即软件"的转型方法。会议重点讨论了如何在不迁移现有数据的情况下，快速实现从传统数据平台到智能代理 AI 的转换。

演讲者强调了当前企业面临的两个关键挑战：如何在 AI 竞争中保持领先地位，以及如何在有限时间内完成转型。传统的数据平台现代化需要 18-24 个月，而行业变革周期已缩短至 11 个月以内，这种时间差距迫使企业寻求更快速的解决方案。Trianz 提出了一个 90 天内完成数据到智能代理 AI 转换的创新方法。

## 详细时间线与关键要点

### 0:00-2:00 - 公司介绍与平台概述
- Trianz 作为 AWS ISV 合作伙伴的定位和"服务即软件"方法
- Concierto 平台的五大核心解决方案：现代化、迁移、管理、最大化和洞察智能代理 AI
- 平台托管在 AWS 上，过去 5 个月已有 60-70 个客户采用

### 2:00-4:00 - 联邦数据连接经验
- Trianz 在联邦数据和连接模型方面的丰富经验
- 4-5 年前成为首家连接 AWS 云与全球数据源的公司
- AWS 收购了相关软件，Trianz 成为战略合作伙伴
- 为 AWS 构建了十几个第三方 ISV 连接器

### 4:00-8:00 - 核心挑战分析
- 企业面临的两大存在性问题：如何在 AI 领域保持领先，以及完成转型的时间限制
- 传统数据平台部署需要 18-24 个月，而行业变革周期仅 11 个月
- 企业实际上在"借用时间"生存
- 数据本身"不愿意移动"的现实：合规问题、SaaS 部署、并购等因素

### 8:00-10:00 - 二元化结果预测
- AI 转型的结果将是二元化的：成功或失败，中间地带将消失
- 失败者将面临缓慢淘汰、客户流失、收入波动等问题
- 成功者将获得收入激增，因为客户会从落后公司转移过来
- 各行业都将经历这种变化模式

### 10:00-13:00 - 解决方案设计原则
- 重新设计方法论，专注于三个目标：快速从数据到洞察、业务易用性、企业级可扩展性
- 解决六个核心问题：现有资产 AI 启用、数据产品化、业务-IT 瓶颈、治理效率、团队协作、速度优化
- 设定 90 天完成目标

### 13:00-15:00 - 技术解决方案
- 实时联邦数据连接，支持全球任意位置的结构化和非结构化数据
- 基于使用方式而非存储方式的数据产品化
- 零代码平台，无需编写查询
- 在源头使用现有治理规则并增强
- 4-5 周内完成全面评估，统一各团队认知

### 15:00-17:00 - 平台功能展示
- 联邦数据连接：支持本地、云端、SaaS 等任意位置的数据
- AI 驱动的数据产品化和编目
- 类似 Netflix 的数据市场界面，按业务领域直观组织数据
- 原生生成式 BI 和对话式 AI，同时支持现有工具集成

### 17:00-19:00 - 企业级功能
- 基于 AWS 技术栈构建：Anthropic、Claude、Bedrock、SageMaker
- 数千条智能规则提升查询精度和速度
- 对话式 AI 支持自然语言数据查询
- 企业智能代理市场，支持代理设计和流程自动化
- DALAI 评估工具生成数据就绪度报告

### 19:00-20:00 - 实施方案与合作模式
- 90 天实施周期：选择一个业务单元创建模板并培训团队
- AWS 资助评估项目，客户无需承担评估费用
- 无义务继续使用，评估后可自由选择
- 支持传统数据迁移和云原生湖仓构建
- Trianz 展位 1381 提供现场咨询
