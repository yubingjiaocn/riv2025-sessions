# MAM334: 使用代理AI能力扩展现代化项目

## 会议概述

本次AWS re:Invent 2025分会场会议聚焦于如何利用代理AI技术加速应用程序现代化项目。会议由AWS企业解决方案架构师Pratip Bagchi主持，Thomson Reuters的AI解决方案架构师Lalit Kumar分享了实际案例，AWS技术主管Yogi Barot深入介绍了AWS Transform for .NET服务的技术细节。

会议主要探讨了传统.NET Framework应用程序向.NET Core平台迁移的挑战与解决方案。Thomson Reuters作为早期采用者，成功利用AWS Transform for .NET将数百个应用程序进行现代化改造，涉及超过5亿行代码。会议强调了通过AI驱动的自动化工具，企业可以将现代化速度提升4倍，同时显著降低成本并提高代码质量。

## 详细时间线与关键要点

### 00:00-05:00 开场与互动
- 主持人Pratip Bagchi进行现场互动，了解参会者背景
- 三个关键问题：现代化项目经验、AI采用需求、成本优化目标
- 介绍会议议程：挑战分析、Thomson Reuters案例分享、AWS服务深度解析

### 05:00-15:00 现代化挑战与动机分析
- **技术挑战**：传统.NET Framework应用程序复杂且单体化，现代化过程缓慢
- **技术动机**：
  - 性能提升：.NET Core比.NET Framework 4.7快600倍
  - 跨平台支持：支持Windows、Mac OS和Linux容器部署
  - 安全性和可支持性显著改善
- **业务动机**：
  - 节省高达40%的运营成本
  - 消除昂贵的商业许可证费用
  - 利用轻量级容器降低存储和计算成本

### 15:00-30:00 Thomson Reuters案例分享
- **企业背景**：Thomson Reuters致力于成为全球领先的内容驱动AI技术公司
- **现状挑战**：
  - 运营超过400个.NET应用程序
  - 涉及5亿多行代码
  - 复杂的依赖关系网络
- **解决方案选择**：选择AWS Transform for .NET的三个原因
  - 可靠性（Reliability）
  - 智能化（Intelligence）  
  - 速度（Velocity）
- **转换示例**：
  - XML配置文件转换为代码配置
  - 自动识别遗留模式并进行现代化改造
  - 保持功能一致性的同时提升安全性

### 30:00-45:00 AWS Transform技术深度解析
- **服务优势**：
  - 加速转换速度提升4倍
  - 免费使用，节省转换工具成本
  - 基于生成式AI和LLM的高质量代码转换
- **两种体验模式**：
  - Web体验：适合大规模批量处理，支持多用户协作
  - IDE体验：适合开发者单个项目POC测试
- **转换流程**：五阶段循环
  - 发现阶段：确定代码行数和应用类型
  - 分析阶段：依赖关系映射和兼容性评估
  - 规划阶段：制定转换策略和时间线
  - 执行阶段：自动代码转换
  - 验证阶段：性能评估和Linux就绪性报告

### 45:00-48:00 最新功能发布与总结
- **新功能发布**（会议当天11:00 AM生效）：
  - 支持6个新区域，包括加拿大中部
  - 多用户协作功能
  - 支持Okta和Microsoft Entra ID认证
  - 支持.NET 10作为目标版本
  - Windows全栈现代化：同时支持.NET和SQL Server到Aurora PostgreSQL的迁移
- **Thomson Reuters成果**：
  - 项目周期从数月缩短至数周
  - 每月现代化150万行代码
  - 为开发者释放时间专注于未来构建而非维护过去
- **最佳实践建议**：
  - 从小项目开始，逐步扩展
  - 准备面对挑战，转换不会100%成功
  - 使用AI辅助工具处理剩余差异
  - 始终保持人工参与循环
