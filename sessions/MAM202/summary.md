# AWS re:Invent 2025 - MAM 202: 使用 AWS Transform 进行 VMware 迁移的 Agentic AI

## 会议概述

本次会议介绍了 AWS Transform for VMware 服务,这是一个利用 Agentic AI 技术加速 VMware 工作负载迁移到 AWS 的创新解决方案。会议由 AWS 基础设施迁移和现代化总监 Sean Lewis 主持,并由 Rajes Rothod 和 Harpit Verk 共同演示。

会议以一个虚构的金融服务客户 Ultimate Bank 为例,该客户拥有分布在四个数据中心的 4,000 个 VMware 虚拟机。AWS 团队通过这个案例展示了企业在迁移过程中面临的技术和组织挑战。根据 Gartner 数据,目前仍有 70% 的 IT 工作负载运行在本地环境,而 Fortune 500 公司运行的 70% 传统 IT 软件已有超过 20 年历史。传统迁移项目平均需要 1.5 年时间完成,这促使 AWS 开发了 AWS Transform 来解决这些痛点。

AWS Transform for VMware 是一个多智能体系统(multi-agent system),采用"人在回路"(human-in-loop)设计理念,将 AI 智能体作为团队成员而非替代人类。该服务在 2025 年 5 月首次发布,并在本次 re:Invent 大会上宣布了重大更新,包括目标导向工作流、业务上下文集成、大规模迁移规划、网络转换增强以及多区域支持等功能。服务现已在 8 个区域可用,支持迁移到 16 个目标区域。

### 迁移路径选择
- **00:09:00 - 00:12:00**: 介绍 AWS 提供的多种迁移路径
  - Relocate(重新定位):使用 Amazon EVS(Elastic VMware Service)
  - Rehost(重新托管):迁移到 Amazon EC2
  - Replatform(重新平台化):迁移到容器(EKS/ECS)或托管服务
  - Refactor(重构):使用无服务器和微服务架构
  - Ultimate Bank 选择使用 AWS Transform 进行 Rehost 迁移

- **00:12:00 - 00:15:00**: Rajes Rothod 接手演讲,深入分析迁移挑战
  - **技术挑战**:旧系统、单体架构、文档不足、复杂依赖关系、缺乏自动化和测试
  - **组织挑战**:不同所有权、业务线分散、审批流程复杂、资源优先级冲突、技能人才短缺、多方协作困难

- **00:18:00 - 00:21:00**: 定义 Agentic 系统

  - Agentic 系统:多智能体系统,面向业务目标
  - 采用"人在回路"设计,智能体作为团队成员而非替代人类
