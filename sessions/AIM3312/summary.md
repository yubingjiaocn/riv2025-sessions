# AWS re:Invent 2025: Making Agents Remember Using Amazon Bedrock AgentCore Memory

## 会议概述 (Session Overview)

本次会议由AWS首席生成式AI解决方案架构师Mani Khanuja主持，产品经理Jay和Experian高级副总裁兼工程主管Imran Shah共同参与。会议重点介绍了Amazon Bedrock AgentCore Memory服务，这是一个完全托管的服务，旨在解决AI代理应用中的记忆管理挑战。

会议通过实际演示和企业案例，展示了如何让AI代理具备短期和长期记忆能力，从而提供更好的用户体验。Mani通过构建幻灯片生成代理的对比演示，直观地展现了有记忆和无记忆代理之间的差异。Experian分享了他们从产品特定的短期记忆实现转向统一记忆架构的企业级应用经验，展示了AgentCore Memory如何简化实现并提高可扩展性。

- 介绍代理的基本组成：模型、工具、数据API、知识库、系统提示

### 5:00-10:00 记忆缺失的问题与解决方案
- 用聚会忘记姓名的类比说明代理记忆的重要性
- 展示幻灯片生成代理案例：基础代理vs记忆代理的对比
- 说明用户偏好（技术演示风格、负责任AI角度）需要重复输入的问题
- 预告现场演示环节

### 10:00-15:00 演示架构预览
- 介绍演示中使用的技术栈：Strands Framework、Memory Hooks
- 解释Memory Hooks vs Memory as Tool的区别
- 短期记忆：每条消息自动添加（确定性方式）
- 长期记忆：关键事实提取（工具方式调用）
- 技术组件：Amazon Bedrock、Claude Sonnet 4.5、IAM权限集成

### 15:00-20:00 自建记忆系统的复杂性
- 短期记忆需求：原始消息存储、K轮对话历史、键值数据库
- 长期记忆需求：语义检索、向量存储、基于查询意义的信息提取
- 数据管理挑战：记忆刷新、自动化管道、重复偏好整合逻辑
- 安全性、可观测性、调试功能的集成复杂度

### 20:00-30:00 Jay介绍AgentCore Memory工作原理
- 人脑记忆类比：近期对话vs一年前对话的记忆差异
- 记忆资源架构：短期记忆（7天到1年）+ 长期记忆（持久化洞察）
- 短期记忆：事件捕获（Actor ID、Session ID）、对话历史、元数据、二进制载荷
- 金毛犬演示案例：PowerPoint构建代理的会话恢复功能
- 分支功能：消息编辑、并发事件流的逻辑分离

### 30:00-35:00 长期记忆策略与配置
- 三种内置策略：Summary（摘要）、User Preferences（用户偏好）、Semantic（语义）
- 高级控制选项：Override策略（自定义LLM和提示）、Self-managed策略（完全控制）
- 语义检索能力：基于查询搜索的记忆记录检索
- 多代理支持：单一记忆资源支持多个代理读写

### 35:00-42:00 Imran分享Experian企业应用经验
- Experian背景：全球数据技术公司，15亿消费者，2.01亿企业客户
- 从短期记忆到统一记忆架构的转型历程
- 短期记忆限制：对话历史持续性、性能成本、跨产品召回、合规要求
- 统一记忆的两个上下文平面：权威上下文（知识库、API）+ 对话上下文（短期/长期记忆、代理状态）
