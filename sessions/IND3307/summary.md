# AWS re:Invent 2025 会议总结：利用 AWS 云和 AI 解决方案推动供应链创新

## 一、会议概述

本次 AWS re:Invent 2025 分会场聚焦于餐饮行业供应链的数字化转型，由 AWS 全球餐饮业务负责人 Deborah Simeone 主持，邀请了两位行业领军企业的供应链专家分享实践经验。Sysco 作为全球最大的食品服务分销商，其高级供应链工程总监 Samir Patel 分享了公司如何通过 AWS 实现仓库管理系统的现代化改造。McDonald's 全球供应链技术负责人 Stephanie Koig 则展示了麦当劳如何利用 AWS 构建全球供应链可追溯性解决方案。

两家企业虽然处于供应链的不同位置，但都面临着规模化运营的挑战。Sysco 每天服务 75 万家餐厅客户，拥有 50 万个 SKU 和 340 多个配送中心；McDonald's 在 120 个国家运营超过 45,000 家餐厅，每天服务全球 1% 的人口。两家公司都成功利用 AWS 云服务和 AI 技术解决了供应链中的关键痛点，实现了运营效率的显著提升，为餐饮行业的数字化转型提供了宝贵的实践案例。

## 二、详细时间线与关键要点

### **00:00 - 03:30 会议开场与行业背景介绍**
- Deborah Simeone 介绍会议主题和嘉宾背景
- 强调餐饮行业规模化运营的复杂性，以 McDonald's 全球一致性体验为例
- 介绍供应链作为餐饮行业"隐形支柱"的重要性

### **03:30 - 06:00 企业规模展示**
- McDonald's：在 120 多个国家运营，面临区域性供应商、监管环境和物流网络的差异化挑战
- Sysco：每天服务 75 万家餐厅客户，处理新鲜食品配送，时间和温度控制至关重要
- 两家企业的共同点：利用 AI 技术释放供应链潜力

### **06:00 - 09:30 Sysco 公司介绍**
- Sysco 成立 56 年，在标普 500 指数中排名第 56 位
- 关键数据：73 万客户、50 万 SKU、810 亿美元营收、90 个国家、340+ 配送中心
- 仓库管理系统（SWIMS）的两大目标：优化仓库劳动力效率、改善工作流程

### **09:30 - 12:00 Sysco 仓库管理系统规模**
- 在 110 个地点部署定制化解决方案
- 25-30 年的持续创新和增强
- 15,000 名仓库员工使用该系统
- 每年处理超过 20 亿件商品

### **12:00 - 15:30 现代化改造面临的挑战**
- 技术债务：150 万行代码、Oracle 技术栈（Oracle Forms UI、Oracle 数据库、PL/SQL）
- 部署困难：单租户系统，需要逐个站点部署，每次 2-4 小时
- 110 个站点全部部署需要数周时间
- 难以创新和集成新技术（如 AI）

### **15:30 - 18:00 现代化改造的动机**
- 业务层面：为仓库员工提供更直观的用户体验，提高生产力，加快新员工培训
- 技术层面：采用云原生硬件和软件，加快部署和升级速度
- 运营挑战：员工在不同温区（干货、冷冻、冷藏）工作，使用多种设备（浏览器、RF 扫描枪、移动设备）

### **18:00 - 21:00 现代化改造策略**
- 用户体验优先：浏览器端采用 React 响应式 UI，移动端从 Windows CE 迁移到 Android
- 架构转型：从单租户转向区域化多租户部署
- 成本优化：降低 Oracle 许可成本、硬件成本和人力成本

### **21:00 - 24:00 技术方案选择**
- 采用"绞杀者模式"（Strangler Pattern）而非"边车模式"（Sidecar Approach）
- 保持单一代码库，逐步构建现代技术栈，新旧系统共存
- 支持蓝绿部署和金丝雀部署
- 数据库从 Oracle 企业版迁移到 AWS RDS 标准版

### **24:00 - 27:00 选择 AWS 的原因**
- 成本优化：降低双轨运行成本
- 快速创新：利用 AWS 先进技术加速开发
- 数据库优势：使用 RDS 标准版替代 Oracle 企业版，显著降低成本

### **27:00 - 30:00 云架构设计**
- 高可用性：单区域内跨多个可用区部署
- 灾难恢复：多区域主备架构
- 旧系统：使用 EC2 实例
- 新系统：采用无服务器架构（Fargate、Lambda）
- 监控：集成 Datadog 企业监控工具
- 架构在最近的 AWS 区域故障中经受住了考验

### **30:00 - 33:00 现代化改造成果**
- 创纪录地在一年内完成 110 个站点迁移，零回滚
- RTO/RPO 显著改善：同区域高可用性指标优异，灾难恢复指标比本地基础设施提升 50 倍
- 部署速度：从每站点 2-4 小时缩短到全部站点 8 小时内完成

### **33:00 - 36:00 安全性和监控改进**
- 传输中加密和静态加密
- 数据库级加密、HTTPS 协议
- 密码轮换机制
- 使用 AWS Certificate Manager 和 Secrets Manager
- 采用 Sysco 定制的"黄金 AMI"进行部署
- 集成 ServiceNow 和 Datadog 实现主动监控

### **36:00 - 39:00 成本节约**
- 利用 AWS 迁移加速计划（MAP）获得积分支持
- 非生产环境实施基础设施停放策略
- 生产环境使用预留实例和 Savings Plans
- 总体运营成本降低至本地环境的一半
- 从 Oracle 企业版迁移到 RDS 标准版节省大量支持成本

### **39:00 - 42:00 AI 愿景和用例概述**
- 构建智能供应链的蓝图
- 三大类用例：自动化、计算机视觉、机器人技术
- 目标：为仓库提供互联体验

### **42:00 - 45:00 AI 用例 1：根因分析（Root Ripple）**
- 问题：110 个仓库的故障响应从被动转为主动
- 解决方案：使用 GitHub Copilot 和 LLM 分析来自 ServiceNow、Datadog 和运维手册的数据
- 部署：基于 Kubernetes
- 成果：平均检测时间（MTTD）减少 30 分钟，恢复时间从 6 小时减少到 3 小时，故障排查人员生产力提升 10 小时

### **45:00 - 48:00 AI 用例 2：路线优化（Router IQ）**
- 背景：黑客马拉松获奖项目，针对运输优化
- 问题：路线规划人员需要在截单时间后手动规划最优配送路线
- 解决方案：使用 Claude LLM、LangChain 框架和混合 VRP（车辆路径问题）算法
- 部署：ECS 或 EKS 集群，使用 Kafka 进行数据流处理
- 保持人工参与：AI 提供建议，最终决策由路线规划人员做出
- 效益：提高仓库员工效率，加快客户交付速度

### **48:00 - 51:00 AI 用例 3：库存优化**
- 问题：50 万个 SKU 的库存管理，电商渠道可能出现缺货情况
- 解决方案：使用 AWS SageMaker 构建 AI/ML 驱动的库存预测系统
- 目标：减少缺货，提高客户满意度，优化库存水平

### **51:00 - 54:00 Sysco 经验教训**
- 转型充满挑战，需要坚持和导航能力
- 建立强大的工程团队，与业务伙伴和产品团队保持良好关系
- 自动化是关键：迁移、备份、基础设施即代码、部署流水线、灾难恢复测试
- 成本管理：避免现代化项目因成本过高而被降低优先级
- 创新思维：不仅仅是迁移到云端，更要思考下一步发展
- 与 AWS 作为战略合作伙伴紧密合作

### **54:00 - 57:00 Sysco 总结与 McDonald's 介绍**
- Samir 将在展厅提供 2-3 点的答疑时间
- Stephanie Koig 开始介绍 McDonald's 供应链技术实践

### **57:00 - 60:00 McDonald's 公司规模**
- 发明了 Big Mac 和 Chicken McNugget
- 120 个国家的 45,000+ 家餐厅
- 每天服务全球约 1% 的人口
- 220 万餐厅员工和数千名技术开发人员

### **60:00 - 63:00 McDonald's 供应链规模展示**
- 在 120 个国家本地化采购食材
- 全球供应商网络分布图展示复杂性
- 美国每年采购超过 20 亿枚无笼鸡蛋
- 鸡蛋首尾相连可绕地球三圈，制成煎蛋卷可填满 40 个奥运会标准游泳池

### **63:00 - 66:00 汉堡规模估算**
- 使用 AI 估算：全球每年销售约 23.6 亿个汉堡
- 每秒售出约 75 个汉堡
- 每天全球销售 640 万个汉堡
- 需要大量原材料：牛肉、奶酪、洋葱、面包

### **66:00 - 69:00 行业关键挑战**
- 监管报告：FSMA、EUDR 等法规合规要求
- 食品安全：确保全球标准得到支持
- 可持续发展：满足并报告全球承诺
- 供应链可见性：最重要的基础能力，支持其他三项挑战

### **69:00 - 72:00 可见性的重要性**
- 可见性是所有其他优先事项的基础
- 支持验证可持续采购地点
- 改善监管报告的准确性
- 快速识别食品质量问题的来源
- 优化供需预测准确性
- 加速实时数据驱动决策
- 需要追踪超过 10,000 家供应商的原材料

### **72:00 - 75:00 优先用例选择**
- 可追溯性：可见性的关键
- 销售与运营计划（S&OP）：对餐厅需求支持产生重大影响
- 营销活动和促销影响：极大影响供应链功能和系统需求

### **75:00 - 78:00 可追溯性挑战规模**
- 每年需要追踪 12 亿个箱子
- 1 英尺的箱子首尾相连几乎可以到达月球
- 虽然 McDonald's 还未在月球开店，但追踪量巨大

### **78:00 - 81:00 数据分布挑战**
- 数据分布在全球约 120 个国家
- 存储在 McDonald's 拥有和不拥有的实体中
- 与 Sysco 等大型企业不同，McDonald's 不拥有供应链
- 供应商、配送中心、配送服务全部外包
- 本地化采购导致一半数据由本地供应商持有

### **81:00 - 84:00 McDonald's 的所有权范围**
- 拥有餐厅及其库存
- 决定如何采购食品和服务以及与谁合作
- 负责质量保证
- 负责全球和本地法律法规合规

### **84:00 - 87:00 解决方案指导原则**
- 直接从源系统获取数据，不经过调整
- 主数据来自 McDonald's 自有系统
- 不通过第三方建立连接，采用点对点连接
- 近实时数据更新
- 安全性和可扩展性是首要关注点

### **87:00 - 90:00 架构原则**
- 采用 AWS Well-Architected Framework
- 以数据分析为视角，确保消费者易于使用
- 便于利用 AI 能力
- 遵循 McDonald's 最佳实践和标准
- 安全性是重中之重

### **90:00 - 93:00 MCD Track 解决方案诞生**
- 基于 Amazon Supply Chain 解决方案构建
- 结合指导原则与 AWS 合作
- 利用 AWS 两小时配送包裹的技术
- 用于追踪制作汉堡、麦乐鸡和薯条所需原料的配送

### **93:00 - 96:00 MCD Track 技术架构 - 三层结构**
- 第一层（数据生产者层）：来自 10,000+ 供应商、配送中心、50,000 家餐厅和 McDonald's 主数据的原始数据标准化
- 第二层（精炼层）：合并来自多个数据源的近实时数据，使用 Amazon Supply Chain 解决方案获得追踪可见性
- 第三层（洞察/策划层）：创建报告、提供用户访问、用于 AI 和 ML 处理

### **96:00 - 99:00 MCD Track 架构详解**
- 使用 Amazon Supply Chain 解决方案
- 集成广泛的服务，包括供应链规范数据模型的强大数据服务
- 提供数据治理以确保准确性和及时更新
- 基于 LLM 的外部数据摄取和映射
- 利用 AWS 先进的数据和 AI/ML 能力监控供应链并生成改进建议

### **99:00+ 会议结束**
- 演讲在介绍 Amazon Supply Chain 解决方案的高级功能时结束
