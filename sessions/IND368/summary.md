# AWS re:Invent 2025 会议总结：Apollo轮胎利用AWS高性能计算加速工程仿真

## 会议概述

本次会议由AWS北美产品工程业务开发负责人Alex Fronasier主持，Apollo轮胎的Shailender和Gautam分享了他们如何利用AWS云平台上的高性能计算(HPC)解决方案来加速轮胎工程开发流程。会议重点介绍了Apollo轮胎从本地数据中心迁移到AWS云平台的完整数字化转型历程。

Apollo轮胎是一家在亚太地区知名的轮胎制造商，年收入35亿美元，拥有20,000名员工，在全球设有7个制造工厂。面对日益复杂的产品设计需求、电动汽车市场的兴起以及激烈的市场竞争，Apollo轮胎需要缩短产品上市时间，同时控制研发成本。传统的本地HPC基础设施存在资本投入大、资源利用率低、扩展性差等问题。通过采用AWS云平台，Apollo轮胎实现了按需付费的弹性计算模式，显著提升了工程仿真效率，并将成本降低了60%。

会议详细展示了Apollo轮胎的云架构设计，包括使用Amazon FSX作为共享存储、采用多种EC2实例类型优化性价比、以及通过Rescale Tachyon进行作业调度管理。此外，Apollo还建立了完整的数据湖架构，利用AI和LLM技术进行数据分析，为研发团队提供了强大的决策支持工具。

## 详细时间线与关键要点

00:00 - 会议开场与议程介绍
- Alex Fronasier介绍自己作为AWS北美产品工程业务开发负责人的身份
- 会议议程：工程卓越愿景、工程仿真实施、Apollo轮胎挑战、解决方案细节、经验教训与未来规划
- 问答环节将在会议结束后进行

03:30 - 工程卓越的三大关键要素
- 第一要素：在数字优先的制造世界中竞争，传统制造商面临数字原生竞争对手的挑战，需要通过云技术和自动化实现运营现代化
- 第二要素：掌握数字主线(Digital Thread)，连接产品全生命周期的数据，实现需求、配置和多学科设计的无缝管理
- 第三要素：通过掌握质量和效率实现规模化卓越，利用AI、物联网和计算机视觉等先进技术

07:00 - Amazon的工程实践经验
- Amazon设计自有硬件并从全球供应链采购，拥有应对这些挑战的第一手经验
- 采用"从客户出发反向工作"的方法论，推动快速原型开发和创新
- 全球分布式团队负责数据中心设计、消费电子设备(Echo、Kindle)、机器人和AI服务开发
- 云平台是加速产品开发的关键推动力

10:00 - 仿真工作流程与产品生命周期
- 典型仿真工作流程：设计概念 → 模型准备 → 仿真研究规划 → 作业提交 → 结果后处理
- 仿真在产品生命周期各阶段的应用：探索设计空间、提高稳健性和可靠性、准备制造选项和降低技术风险
- 仿真可以虚拟测试数千个实验，节省时间并实现仅依靠物理原型无法实现的创新

13:30 - 云端工程仿真的四大优势类别
- 容量优势：大规模容量访问、按使用付费模式、弹性扩展能力
- 全球优势：全球团队无缝协作、内置灾难恢复和业务连续性、支持"追随太阳"工作模式
- 经济优势：无需资本投资即可访问最新技术、提高软件许可证利用率和投资回报率
- 环境优势：通过共享基础设施实现可持续发展目标

17:00 - AWS HPC架构的三大组件
- 前端：包括作业提交和管理的Web门户、运行CAD/CAE应用的虚拟桌面基础设施(VDI)、用于高性能远程可视化的DCV技术
- 计算：核心计算集群、集群管理功能、多种专用计算实例选择
- 数据：基于数据使用需求的仿真数据存储、数据管理以实现可追溯性、数字主线集成

20:00 - AWS实例类型优化策略
- 每个作业可以匹配最优化的实例类型(多物理场、CFD、有限元分析等不同求解器)
- 实例采购选项：Spot实例用于成本敏感的批处理作业、按需实例用于关键作业、Savings Plans用于基线常驻计算容量
- 未来将利用AI实现智能配置，自动推荐每个工作负载的最优实例

23:00 - GPU实例与AI集成
- ANSYS、Siemens等独立软件供应商重新设计求解器以在GPU上运行，加速结果时间
- AWS提供多种GPU实例系列，包括最新发布的P6实例(配备NVIDIA B300芯片)
- GPU实例不仅适用于计算密集型仿真，还可运行AI和机器学习工作负载

25:30 - Apollo轮胎公司介绍
- 在亚太地区是知名品牌，在美国主要以Vredestein品牌销售高端和运动轮胎
- 全球业务：35亿美元收入，20,000名员工，7个制造工厂(5个在印度，2个在欧洲)
- 产品线复杂：乘用车轮胎、农用轮胎、卡车轮胎、越野轮胎、两轮车轮胎，以及夏季、冬季、全季节和镶钉轮胎

28:00 - Apollo轮胎面临的挑战
- 产品设计日益复杂，电动汽车对轮胎有特殊要求
- 竞争激烈的市场需要缩短产品上市时间
- 需要性价比高的解决方案来设计优质产品
- 两个研发中心(印度和欧洲)需要合适的资源进行化学配方和轮胎设计仿真

30:00 - Apollo的数字化转型历程(2021年开始)
- 第一步：从制造工厂、机器和PLC实时流式传输物联网数据
- 第二步：使用S3和Redshift创建数据湖，后来扩展使用Glue、DMS和Aurora服务
- 目前数据湖规模：超过400TB数据(包括X光图像、视频分析)，8TB结构化关系数据(来自SAP等系统)
- 第三步：将整个SAP ERP解决方案从本地迁移到云端

35:00 - 工业4.0与轮胎谱系(Genealogy)系统
- 连接更多物联网设备：混合机、挤出机、带式切割机、轮胎成型机、硫化机
- 轮胎制造复杂性：一个简单轮胎可能由200多个组件粘合而成，整个生命周期1-2小时，硫化过程20-40分钟
- 谱系系统追踪：使用了哪些组件、供应商信息、装配线、操作员、天气条件等
- 支持从任意点开始的双向追溯(从成品到原材料，或从组件到成品)

40:00 - 基于LLM的数据分析解决方案
- 传统BI报表无法满足不断变化的问题需求(公司已有500多个报表)
- 在数据湖之上部署LLM，用户可以用自然语言提问
- LLM将自然语言转换为SQL语句，获取数据并提供答案
- 可进行根因分析，识别故障原因

43:00 - 轮胎设计与测试的复杂性
- 轮胎设计是昂贵且复杂的过程，现场测试需要大量投入
- 测试项目：制动条件、雪地/雨天/湿滑路面性能、制动距离、噪音水平
- 电动汽车轮胎需要特殊设计(例如需要增加噪音以提醒行人)
- 虚拟环境仿真可以避免制造模具、硫化轮胎和物理测试的成本

46:00 - HPC挑战与使用的软件
- 每月数百次仿真、多个作业、多个用户、老化的硬件
- 使用行业标准软件：ANSYS用于仿真，Siemens用于设计

47:30 - 本地HPC vs 云HPC的对比
- 本地方案：大量资本支出、6-7年ROI规划、购买大型计算机(256/512 CPU)、大容量存储(10-20TB SAN)、资源利用率不确定
- 云方案：按使用付费、改善现金流、夜间无人使用时自动关闭、按需启动和终止
- 计算成本持续下降：GPT-2在2019年训练成本45万美元，现在不到600美元

51:00 - AWS云架构设计
- 用户通过Site-to-Site VPN从本地连接到AWS云
- 创建登陆区：使用Windows终端服务，多用户可同时登录、查看作业状态、运行作业
- 后端作业仿真服务器控制作业分配和资源配置
- 可根据配置启动多个集群
- 使用FSX存储共享数据，这是关键的游戏规则改变者

54:00 - Amazon FSX的关键优势
- 共享存储，可自动扩展和缩减容量(EBS只能单向增长)
- 根据使用率自动调整：超过90%时扩展，低于60%时缩减
- 可以像S3一样在热、冷、温存储层之间自动移动数据
- 支持快照功能，用户误删文件可恢复，无需依赖完整备份
- 对于多用户HPC环境中的数据共享至关重要

57:00 - 性能测试与实例选择
- 管理层和用户都需要了解性能提升和成本效益
- 在POC阶段进行大量仿真测试，比较不同实例类型
- 测试了多种组合：Linux、Graviton、AMD、Intel、GPU等
- Linux实例在高负载场景下提供最佳性价比
- 针对两种不同软件分别进行测试，确定最适合的实例类型

60:00 - 作业调度工具选择
- 最初使用Slurm作为作业调度工具，但过于原始
- 选择Rescale公司的Tachyon专有解决方案
- 与Rescale长期合作定制产品，确保用户良好采用
- 用户对UI界面和作业控制方式满意

62:00 - 操作系统选择策略
- 前端使用Windows终端服务作为登陆区(用户熟悉Windows)
- 后端对用户是黑盒，用户无需知道运行环境
- Linux提供更好的性价比，后端作业在Linux上运行
- **相同作业在Windows和Linux之间实现60%的成本节省**
