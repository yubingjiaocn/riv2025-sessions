# AWS re:Invent 2025 - 曼彻斯特机场集团应用代理AI优化航班准点率

## 会议概述

本次技术分享会由AWS高级AI策略师Tom Chester、团队成员Margarita以及曼彻斯特机场集团(MAG)的Steve共同主讲，重点介绍了MAG如何应用代理AI技术解决机场运营复杂性问题，确保航班准点起飞。作为英国最大的机场集团，MAG每天处理超过1000个航班，面临着巨大的运营挑战。

演讲者详细阐述了MAG的愿景——创建由代理驱动的统一数字化同事工作场所，通过消除运营复杂性为公司创造价值。他们选择了"大胆思考、小步开始、快速扩展"的框架，首先从员工意外缺勤报告这一关键业务需求入手，构建了基于Amazon Bedrock Agent Core的技术解决方案。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与背景
- 欢迎致辞，介绍演讲团队成员
- MAG作为英国最大机场集团的基本情况：3个机场、9000名员工、每日1000+航班
- 强调机场安全和客户体验的重要性
- 提出核心挑战：如何在复杂的运营环境中保持航班准点

### 5:00-10:00 MAG运营挑战分析
- 详细介绍机场运营的复杂性：涉及MAG员工、边境管制、航空公司合作伙伴、地勤处理等多方协作
- COVID-19期间的影响：航班量降至1-2%，主要为货运
- 安全优先原则导致的高运营成本
- 40000名工作人员在机场园区内的协调管理挑战

### 10:00-15:00 AI愿景与策略
- 提出创建统一数字化同事工作场所的愿景
- 介绍多个潜在应用场景
- 解释选择代理AI而非传统自动化工具的原因
- 强调"大胆思考、小步开始、快速扩展"的实施框架

### 15:00-20:00 首个用例：意外缺勤报告
- 选择意外缺勤报告作为首个实施场景的原因分析
- 当前流程的痛点：第三方报告热线、人工重新排班、多系统更新
- 业务价值量化：减少加班费用、提高安检效率、降低第三方服务成本
- 关键考虑因素：用户认证、数据准确性、系统可靠性

### 20:00-25:00 代理AI技术选择
- 生成式AI解决方案的自主性演进：从AI助手到AI代理再到完全自主系统
- 代理的核心能力：目标导向、工具访问、记忆功能、推理规划
- 解释为何选择代理AI：动态环境、非确定性流程、多系统协调需求
- 与传统自动化工具的对比分析

### 25:00-30:00 技术架构深度解析
- 当前工作流程分析：员工联系、信息收集、HR政策检查、系统记录、人员通知
- 目标解决方案架构展示
- 工具适配的重要性：详细错误捕获、人类可读输出、连续工具合并
- Agent Core Runtime的应用优势

### 30:00-35:00 扩展性与安全考虑
- MCP服务器的引入：工具与代理的关注点分离
- Agent Core Gateway的自动工具发现机制
- 安全审查要求：防止社会工程攻击
- 自定义护栏开发：多轮对话上下文感知、最小权限原则

### 35:00-40:00 语音交互实现
- Amazon Nova Sonic语音到语音模型的集成
- 实时流媒体复杂性处理：WebSocket连接、ECS缓冲管理
- 语音模型提示优化：避免思维链、枚举模式、拼写考虑
- 代理层次结构：Nova Sonic调用文本代理作为工具
- 语音认证解决方案

### 40:00-45:00 用户体验与成果展示
- UI设计的重要性：保持用户参与度、提供后台处理透明度
- 实际演示：用户报告缺勤的完整流程
- 项目成果：99%缺勤报告一致性、90%处理时间减少
- 未来规划：自动排班、航站楼智能管理、更多代理和工具集成

### 45:00-45:48 总结与联系方式
- 四个关键经验总结：适配代理工具、多轮对话护栏、语音提示优化、UI至关重要
- AWS生成式AI创新中心介绍
- 问卷调查提醒和后续交流安排
