# AWS re:Invent 2025 会议总结：使用 AWS Managed Microsoft AD Hybrid Edition 简化 Active Directory 迁移

## 会议概述

本次技术分享会由 AWS 专家解决方案架构师 Rodney Undercuffler 和 Deloitte 数字控制审计与保障实践顾问 Alex Gunther 共同主讲。会议重点介绍了 AWS Managed Microsoft AD Hybrid Edition 这一新服务如何解决企业在 Active Directory 迁移过程中面临的复杂挑战。Rodney 拥有超过 25 年的 Active Directory 工作经验，几乎每天都在与客户讨论 Active Directory 的实施、迁移和优化问题。

会议深入探讨了三种主要的 Active Directory 与 AWS 集成路径：AD Connector（代理服务）、EC2 上的自管理 AD 扩展，以及 AWS Managed Microsoft AD。传统方案虽然解决了部分问题，但仍存在信任管理复杂、需要重新授权、迁移周期长等挑战。AWS 在 2024 年 8 月推出的 Hybrid Edition 彻底改变了这一局面，它允许企业直接扩展现有域，无需迁移用户、无需设置信任关系，将迁移时间从数月缩短至 2-4 周。Deloitte 作为 AWS 的长期合作伙伴，在服务正式发布前进行了为期 9 个月的深度测试，验证了该方案在 AD 最小化、风险控制和快速部署方面的显著优势。

## 详细时间线与关键要点

[00:00 - 02:30] 开场介绍
- 演讲者介绍：Rodney Undercuffler（AWS 专家解决方案架构师，25 年以上 AD 经验）和 Alex Gunther（Deloitte 数字控制审计顾问）
- 会议主题：AWS 和 Deloitte 如何通过 AWS Managed Microsoft AD Hybrid Edition 简化 Active Directory 迁移

[02:30 - 04:00] Deloitte 的角色与合作背景
- Deloitte 专注于自动化风险控制，将手动流程转化为云原生服务实现持续合规
- Deloitte 与 AWS 建立长期合作关系，共同推广 Hybrid Edition
- Deloitte 的审计保障团队和网络安全团队对该服务进行了广泛的 Beta 测试和预评估

[04:00 - 07:30] Active Directory 集成的三种路径
- **AD Connector**：作为代理服务连接 AWS 原生服务与本地 AD，无需部署硬件，适合概念验证或有限服务场景
- **EC2 上的自管理 AD**：在 AWS 上扩展域控制器，消除对本地环境的依赖，但仍需手动管理补丁、备份等运维工作
- **AWS Managed Microsoft AD**：独立的森林/域，AWS 负责运维管理，通过信任关系集成多个域和森林，支持 RDS、FSX、WorkSpaces 等服务集成

[07:30 - 11:00] Active Directory 身份管理的演进
- **25 年前的 AD**：解决了集中式身份验证、多主复制、组策略标准化等问题
- **持续存在的挑战**：缺乏云集成、需要硬件更新、手动灾难恢复、物理边界限制
- **EC2 扩展方案**：消除了硬件更新问题，但仍需手动补丁、备份和运维管理
- **Managed AD（10 周年）**：通过信任关系集成多域/森林，AWS 负责灾难恢复、备份、补丁，但仍需管理信任关系和独立森林

[11:00 - 13:30] 传统迁移面临的挑战
- **复杂的规划**：迁移规划时间往往超过实际迁移时间
- **访问权限保留**：迁移到独立森林/域需要重新授权，无法保留原有 AD 森林名称
- **SID 历史和双跳问题**：使用第三方工具（如 ADMT）需要建立信任关系，导致双重复杂性
- **信任管理**：需要设置和维护单向或双向信任关系

[13:30 - 16:00] Hybrid Edition 的突破性解决方案
- **2024 年 8 月发布**：基于成熟的 Managed AD 服务构建
- **核心优势**：
  - 直接扩展现有域，无需迁移用户、组或 SID
  - 无需设置信任关系，使用相同的域和森林
  - 保留现有身份验证机制
  - AWS 负责域控制器的运维管理，客户保留自管理端的控制权

[16:00 - 18:00] 可扩展性优势
- **横向扩展**：通过控制台或自动化方式将域控制器从 2 台扩展到 3、4、5 台或更多
- **跨账户共享**：可将 Hybrid Edition 共享给 10、100 或更多 AWS 账户，无需部署额外实例
- **弹性伸缩**：根据负载指标自动扩展或缩减，这在传统物理硬件环境中难以实现

[18:00 - 22:00] Deloitte 的 Beta 测试经验
- **测试周期**：产品发布前进行了约 9 个月的 Beta 测试
- **测试视角**：
  - **AD 最小化趋势**：随着云身份提供商（如 Entra ID）的成熟，减少直接连接 AD 的技术组件，战略性缩减 AD 基础设施占用空间
  - **联盟关系**：帮助客户应对身份现代化和云迁移战略
  - **风险与控制**：评估新服务对组织控制框架的影响，包括攻击面、IAM、网络控制等
  - **技术验证**：测试权限、域扩展简化程度、管理体验、安全边界维护等

[22:00 - 25:00] Beta 测试的关键发现
- **快速部署**：非域管理员在一周内搭建完整模拟环境，几天内成功启动 Hybrid 服务
- **迁移速度**：显著提升迁移速度和效率
- **风险控制优势**：
  - 平台多样化
  - 消除额外信任关系的维护负担
  - 降低灾难恢复事件中的连接中断风险
- **RDS 集成简化**：无需建立额外森林、同步身份或为每个数据库进行权限映射

[25:00 - 27:00] 实际应用场景
- **并购整合**：快速、高效地将新组织纳入现有环境，保留所有访问控制
- **多账户策略**：满足领导层对多账户战略或环境扩展的要求
- **云转型**：支持 AWS 环境的分布式部署

[27:00 - 29:00] 与传统方案的对比优势
- **安全团队开销最小**：无需监控身份同步、自动化配置/取消配置
- **无需额外基础设施**：不需要启动新的基础设施或增强网络控制
- **迁移时间对比**：从传统方案的数月缩短至 2-4 周（包括测试）
- **双向门决策**：支持可控回滚，可以取消配置并在需要时来回切换，降低决策风险

[29:00 - 30:30] RDS 集成案例
- 许多客户最初使用 RDS 本地登录，导致手动工作量大、离职员工访问权限未及时清理
- 通过 Hybrid Edition 与 RDS 集成，大幅简化身份管理流程
