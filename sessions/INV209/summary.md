# AWS re:Invent 2025 金融服务创新演讲总结

## 会议概述

本次AWS re:Invent 2025金融服务行业创新演讲由AWS金融服务业务董事总经理Scott Mullins主持,重点探讨了AI代理(Agentic AI)如何重塑金融服务行业的未来。演讲回顾了AWS与金融机构过去13年的合作历程,强调了行业从传统IT架构向智能代理系统演进的关键转折点。会议展示了金融机构如何通过Amazon Bedrock等AWS服务,构建安全、合规且高效的多代理系统,以应对从客户服务到风险评估等各类业务挑战。

演讲邀请了两位重量级嘉宾分享实践经验:安联科技(Allianz Technology)首席技术官Axel Shell介绍了全球最大保险公司之一如何建立AI解决方案的生产框架;Coinbase开发者平台工程负责人Eric Reppel则展示了X402协议如何通过稳定币实现代理间的原生互联网支付。两位嘉宾的分享揭示了一个共同主题:要实现自主AI系统的规模化应用,技术能力固然重要,但更关键的是建立信任机制、治理框架和开放标准,以确保代理系统能够安全、透明地代表用户执行复杂任务并相互协作。

## 详细时间轴与关键要点

[00:03:30 - 00:06:45] AI代理时代的到来

- 强调代理正在承担越来越关键的职责,直接与客户互动并处理敏感数据

[00:06:45 - 00:10:20] 未来场景演示:个人金融代理

- 代理主动分析房产披露文件,发现地下室曾被淹的历史
- 代理访问气象数据库,评估未来洪水风险
- 代理实时分析财务状况,评估资产配置并建议资金调整以确保购房能力
- 代理协助税务优化和教育基金规划

[00:10:20 - 00:12:15] 代理对保险行业的影响
- 展示人寿保险承保场景:代理自动提取文件数据、执行风险评估
- 未来代理将能够与其他代理协商价格
- 强调实现这一愿景需要建立信任机制:安全运营、合规性、数据保护
- 介绍Amazon Bedrock新增18个基础模型,以及Policy和Evaluations两项新功能

[00:12:15 - 00:16:30] Axel Shell开场:多重危机世界
- 介绍安联:150,000名员工,74个国家运营,2.2万亿资产管理规模,135年历史
- 描述当前"多重危机世界":IT故障增加、数据泄露、自然灾害、深度伪造、全球恐怖主义

[00:16:30 - 00:20:45] 代理浏览演示:宠物保险购买
- 演示使用单一提示词为宠物猫购买保险
- 上传动物护照,代理自动提取信息(姓名、出生日期、性别等)
- 代理浏览多个网站进行比较,整个过程4-5分钟
- 强调代理浏览能力:从语法约束解码到可靠的工具调用,从浏览器API到真实Web UI导航
- 代理具备规划、行动、观察和推理能力,可处理完整的Web工作流

[00:20:45 - 00:23:00] 代理间通信的未来
- 指出不希望成千上万的代理爬取网站并点击按钮(计算密集)
- 展望未来:安联自己的代理将与用户代理直接交互,使用协议进行通信
- 引出代理间支付的重要性(为Eric的演讲铺垫)

[00:23:00 - 00:27:15] 安联的多代理框架开发
- 与AWS合作开发模型无关的多代理框架
- 目标:将风险团队的"不喜欢的工作"(数据收集)时间减少,提高生产力
- 技术架构:使用Amazon Bedrock、Claude大语言模型、S3存储、VPC端点、Fargate、Langraph
- 强调开放模型选择和MCP(模型上下文协议)的重要性

[00:27:15 - 00:31:30] 从单体到微服务再到代理的演进
- 回顾技术演进:单体架构→微服务(优点:开发效率,缺点:复杂性、治理困难)
- 本地部署→云迁移(优点:成本降低、韧性提升,缺点:技术锁定、过渡复杂)
- 警告:代理系统将带来巨大价值,但也会有许多缺点
- 强调核心原则:"你可以扩展一切,但无法扩展复杂性"

[00:31:30 - 00:35:45] 从孤立POC到Gen AI网格
- 批评当前状态:孤立的POC项目(会议助手、协同工具、HR机器人等)
- 问题:低复用性、与用例紧密耦合、增量价值实现、无法扩展复杂性
- 提出解决方案:Gen AI网格(Gen AI Mesh)
- 特点:代理发现能力、注册和编排、跨用例复用、灵活的即插即用系统、规模化价值实现

[00:35:45 - 00:39:00] 安联的代理分层架构
- L0层:系统记录(数据库、保单、案例数据库等)
- L3层:实用代理(文档处理、视频、图像、语音、电子邮件)- 高度可复用
- L2层:业务代理(通知、解决方案、案例管理、保单保险)- 业务特定
- L1层:规划代理(编排工作流)
- 所有层级连接到客户旅程,提高复用性、减少复杂性、易于扩展

[00:39:00 - 00:42:30] 代理架构框架(Agentic Architecture Framework)
- 四大组件:
  1. 核心栈:前端编排、AI工作流工具
  2. DevOps工具:自动化、持续部署
  3. AI模型:灵活性
  4. 基础服务层:容器编排、API管理、全栈可观测性

[00:42:30 - 00:46:15] 代理控制平面与市场
- 开发内部代理市场:收集、展示和使用代理
- 功能:配置管理、监控、消费跟踪、成本优化(Agentic Ops)
- 代理提示词游乐场:实验新的代理应用
- 演示理赔处理代理:MCP连接、REST端点、工具可用性、测试用例执行、完整的追踪和审计日志

[00:46:15 - 00:50:00] 实际用例:停电理赔处理
- 场景:停电导致冰箱食物变质
- 涉及7个代理:规划代理、网络安全代理、保障代理、天气代理、欺诈代理、支付代理、审计代理
- 处理时间减少80%(从约100天缩短)
- 演示移动应用交互流程:身份验证→地址验证→事件时间→照片上传→后台代理协作→支付决策→审计追踪

[00:50:00 - 00:51:30] Axel总结
- 强调需要为未来1-2年的大量代理用例做好准备
- 必须建立适当的治理和控制机制

[00:51:30 - 00:54:00] Scott过渡与Eric介绍
- 强调信任主题:需要信任代理负责任地行动和相互协作
- 介绍Eric Reppel,Coinbase开发者平台工程负责人
- Coinbase开发者平台(CDP)类似于AWS之于Amazon的角色

[00:54:00 - 00:57:30] 互联网的演进与支付问题
- 互联网为人类设计,基于"人在回路中"的标准
- Web 1.0(80年代):信息共享标准(HTML、HTTP、浏览器)
- Web 2.0(90年代):读写能力(JavaScript、Ajax、JSON)
- 核心问题:从未解决"如何传递价值"的问题,没有开放的互联网原生支付标准
- Marc Andreessen名言:"原罪是我们无法在互联网核心构建经济学,即货币"

[00:57:30 - 01:00:15] HTTP 402状态码的历史
- HTTP状态码:200(OK)、404(未找到)、402(需要支付)
- 402状态码定义:"保留供未来使用"- 从未真正实现标准化
- 支付成为互联网最困难但最重要的部分
- 每个支付API集成略有不同,没有"一次集成,到处支付"的体验

[01:00:15 - 01:03:00] 结账体验的挑战
- 传统结账表单:14个字段(信用卡号、到期日、安全码、地址等)
- 对人类:体验尚可;对代理:体验糟糕
- 代理是概率系统:85%成功率×14次任务=整体成功率大幅下降
- 用户不愿将信用卡无限制地交给AI代理

[01:03:00 - 01:05:30] 稳定币与代理商务
- Coinbase论点:稳定币非常适合AI代理和代理商务
- 稳定币定义:1:1美元储备支持的区块链代币(如USDC)
- 优势:24/7可用性、秒级结算(T+2秒而非T+2天)、极低费用(约0.1美分,无论金额大小)

[01:05:30 - 01:08:45] X402协议介绍
- X402:402状态码的扩展,互联网原生支付的开放标准
- 由Coinbase创立,将转移到独立基金会治理
- 定位:连接支付和互联网的桥梁
- 代理标准栈:MCP(工具定义)、X402(支付)、A2A(代理通信)、EIP-8004(代理声誉和发现)

[01:08:45 - 01:11:00] X402的采用数据
- 过去30天:超过7000万笔交易
- GitHub:4000+ star,近100名贡献者
- 交易量:超过3000万美元(实际约3800万美元)
- 用户:超过25万人购买商品或服务
- 常见用例:数据访问、信息获取、API使用

[01:11:00 - 01:14:30] X402实际演示
- 演示:让LLM研究X402的使用和采用情况
- 代理执行网络搜索获取公开信息
- 代理支付5美分访问社交媒体趋势的付费资源
- 展示隔离资金(约56美分)和支出规则(每次25美分或每天1美元限制)
- 强调:数字原生货币使规则编码变得简单

[01:14:30 - 01:17:15] 动态代理的概念
- 传统代理:在预定义的工具箱中操作
- 动态代理:通过开放标准,可以发现并支付任何接受X402的服务
- 微支付成为可能:0.1美分转账成本,支持1美分到100万美元的支付
- 平均交易额不到1美元(7000万笔交易/3800万美元)
- X402标准本身零费用

[01:17:15 - 01:20:30] 为什么是现在?
- AI是显而易见的原因
- 稳定币已找到产品市场契合度
- 监管和合规日益明确
- 区块链用户体验大幅改善,用户甚至意识不到在使用加密技术
- AI代理迫使非人在回路支付的出现

[01:20:30 - 01:23:45] AI打破互联网经济模式
- 传统模式:用户看内容和广告→商家付费投放广告→用户概率性转化购买
- AI的问题:LLM抓取网站时主要关注内容,不关注广告
- 结果:发布者无法获取其创造的价值
- X402解决方案:在同一请求中直接为内容付费
- 示例:AI访问"拉斯维加斯最佳餐厅"博客,网站要求25美分访问费

[01:23:45 - 01:26:30] X402技术架构
- 流程:代理↔服务器↔促进者(Facilitator)
- 促进者作用:抽象加密复杂性,使其像Web开发一样简单
- 大多数语言和框架可用一行代码集成X402
- AWS支持:通过Bedrock、Lambda、EC2等服务支持X402服务器和代理构建
- CDP钱包:基于AWS Nitro和KMS构建,优化用于代理和稳定币操作

[01:26:30 - 01:28:30] 集成示例
- 使用Bedrock + Coinbase Agent Kit(代理构建工具包)
- 为代理提供钱包,连接发现和支付工具
- 约10行代码即可创建能够购买任何支持X402的互联网商品的代理
- 一次集成,永久受益于开放访问

[01:28:30 - 01:31:00] X402的价值
- 解锁内容货币化、按使用付费API、微支付
- 货币成为互联网的一等公民
- 核心价值:更有用的AI(更多能力=更多可能性)
- 避免围墙花园:防止各平台竞争标准导致的碎片化
- 保持互联网开放和大部分Web免费

[01:31:00 - 01:32:30] 下一步与资源
- Coinbase使命:增加全球经济自由
- 愿景:货币全球24/7流动,无参与限制
- 资源:X402.org(标准和开源代码,Apache 2.0许可)、CDP.coinbase.com(开发者平台)

- 不能用昨天的技术或教条构建明天的系统
- 机会:以全新方式构建和消费金融服务
- 愿景:更多选择、更好的资本和机会获取、更好的风险保护
