# AWS re:Invent 2025 可持续性会议总结

## 会议概述

本次会议由AWS技术客户经理Nitan Patuk主讲，重点探讨了云工作负载增长对环境的影响以及如何通过AWS可持续性最佳实践来优化能源效率。会议指出，云工作负载每年增长约25%，但许多组织在扩展过程中往往忽视了可持续性审查。随着工作负载的增长，碳足迹、运营成本也随之增加，同时还面临监管合规和品牌声誉的挑战。

特别值得关注的是生成式AI对可持续性的巨大影响。生成式AI工作负载的年增长率高达300-500%，其能源消耗惊人：训练一个标准GPT模型需要约1200兆瓦时的能量，相当于为美国150户家庭、欧洲300户家庭或印度1000户家庭提供一整年的电力。演讲者强调，AWS通过六大可持续性最佳实践、定制芯片（Graviton、Inferentia、Trainium）以及各种优化工具，帮助客户在保持业务增长的同时实现可持续发展目标。

会议最后鼓励参会者使用AWS客户碳足迹工具和CloudWatch指标来监测和优化其可持续性表现，并强调可持续性本质上是成本优化与环境责任的结合。

## 详细时间线

开场与问题提出 (00:00-01:30)
- 演讲者Nitan Patuk自我介绍，担任AWS技术客户经理
- 现场调查：询问有多少人进行过AWS可持续性架构审查，以及进行过两次以上的人数
- 指出问题：大多数人不会持续进行可持续性审查

云工作负载增长的环境影响 (01:30-03:45)
- 云工作负载每年增长约25%
- 四大关键影响：
  - 更高的碳足迹
  - 运营成本增加
  - 监管合规挑战（各国政府对碳报告、水资源使用、排放范围有标准要求）
  - 品牌声誉影响（客户、利益相关者和政府都关注可持续性）

生成式AI的环境影响 (03:45-06:00)
- 生成式AI工作负载年增长率达300-500%
- 标准GPT训练消耗约1200兆瓦时能量
- 标准GPT推理消耗约550兆瓦时能量
- 能源消耗对比：
  - 1200兆瓦时可为美国150户家庭供电一年
  - 可为欧洲300户家庭供电一年
  - 可为印度1000户家庭供电一年

AWS可持续性架构六大最佳实践 (06:00-08:30)
- AWS于2021年添加可持续性架构支柱
- 六大基础最佳实践：
  1. 区域选择
  2. 根据需求进行适当扩展
  3. 数据策略
  4. 硬件和服务的充分利用
  5. 软件架构实践（如考虑Python代码的能源消耗）
  6. 公司流程和文化（可持续性优先方法）

低成本优化措施 (08:30-11:00)
- 引用AWS CTO名言："最清洁的能源是未使用的能源"
- 四大快速见效措施：
  - 自动扩展：利用云的灵活性，按需配置资源
  - 使用Spot实例：重复利用即将浪费的资源
  - 使用无服务器服务（如Lambda）
  - 使用AWS Compute Optimizer获取资源调整建议
- 效果：通过适当的资源调整可减少20-30%的碳排放和成本

AWS定制芯片的能效优势 (11:00-13:00)
- Matt Garman在前一天主题演讲中提到的芯片组
- Graviton处理器：比同等规格的标准EC2实例节能60%
- Inferentia 2：推理工作负载能效提升50%
- Trainium：训练工作负载能效提升25%

可持续的生成式AI架构 (13:00-15:30)
- 三大关注领域：
  1. 训练优化
  2. 推理优化
  3. 数据管理和数据效率
- 建议使用SageMaker中的预训练和预优化模型
- 强调数据管理策略的重要性：
  - 清理未使用的训练和推理数据
  - 使用数据生命周期管理策略
  - 训练数据通常达到PB级别，需要积极管理

数据传输优化 (15:30-17:30)
- 常见问题：在US East 1构建工作负载，但用户基础在欧洲或亚洲
- 跨区域数据传输会增加碳足迹
- 建议：选择靠近用户基础的区域，减少数据传输产生的排放

监测和测量工具 (17:30-19:30)
- AWS客户碳足迹工具：
  - 位于成本管理器仪表板中
  - 可查看工作负载的能源效率和排放情况
- CloudWatch指标作为可持续性KPI：
  - 实例和资源利用率
  - 数据传输（网络进出）
  - 内存消耗
- 核心理念：可持续性 = 成本优化 + 环境责任

总结与资源 (19:30-20:30)
- 鼓励参会者确保工作负载在业务增长的同时保持可持续性
- 推荐查看AWS Skill Builder课程
- 演讲者将在可持续性展位提供进一步咨询
- 会议结束
