# AWS re:Invent 2025 CDL AI 之旅：从概念验证到生产环境

## 会议概述

本次技术分享会由 Dynatrace 赞助，展示了英国保险科技公司 CDL 如何在 AWS 和 Dynatrace 支持下，成功将 AI 技术从早期开发阶段推进到生产环境。CDL 作为英国保险市场的核心基础设施提供商，处理着英国约 60% 的保险交易，年交易量达万亿级别。

演讲嘉宾包括 Dynatrace 现场 CTO Jon Griffiths、Amazon Bedrock AgentCore 团队高级数据科学家 Ishan Singh，以及 CDL 架构负责人 Matthew Eisengruber 和运营负责人 Chris Buckley。会议重点分享了 CDL 在高度监管的保险行业中如何建立 AI 治理框架、技术架构选择，以及如何通过可观测性确保 AI 系统的可靠性和合规性。

## 详细时间线与关键要点

### 0:00-10:00 - Dynatrace 平台介绍与 AI 可观测性
- **Dynatrace 核心能力**：专注于可观测性和安全性，通过 AI 和自动化推动边界突破
- **AI 可观测性四大用例**：
  - 成本和性能控制（令牌使用、提示缓存）
  - 可解释性和调试追踪
  - 合规性监控
  - 护栏和安全防护
- **全栈 AI 监控**：覆盖应用层、编排层、语义层、模型代理层和基础设施计算层

### 10:00-20:00 - CDL 公司背景与市场地位
- **CDL 简介**：总部位于英国曼彻斯特，600 名员工的 B2B 软件公司
- **市场地位**：被称为"英国保险市场的血管"，处理 60% 的保险交易
- **业务范围**：涵盖报价、购买、保单管理、支付、续保和政府合规报告
- **技术规模**：2024 年 Dynatrace 观察到 1 万亿次交易，管理 700 万份保单

### 20:00-30:00 - AI 转型的组织准备
- **CEO 驱动**：自上而下的 AI 战略推动，提出"AI inside"概念
- **技术基础**：
  - 完成云优先到云唯一的迁移
  - 采用 API 优先的产品思维
  - 250 名工程师全员获得 AWS 云从业者认证
- **文化变革**：通过孵化器和黑客马拉松营造安全的探索环境

### 30:00-40:00 - AI 治理框架建立
- **AI 指导小组**：包含高管、合规法务、网络安全和技术专家
- **流程优化**：新模型审批从 3 个月缩短到 48 小时
- **合规标准**：
  - 获得 AWS FTR（基础技术审查）验证
  - 获得 ISO 42001 AI 系统认证
  - 遵循 OWASP LLM 前 10 大漏洞防护

### 40:00-50:00 - 技术架构与实施
- **模型选择策略**：采用"被动投资"方法，避免绑定单一模型
- **Amazon Bedrock 架构**：
  - 多代理方法：监督代理 + 领域专用子代理
  - 强大的护栏功能：输入和输出双重保护
  - OpenTelemetry 集成实现端到端追踪
- **关键技术发现**：通过可观测性及时发现令牌耗尽问题，避免生产发布延迟

### 50:00-56:00 - 生产部署与未来展望
- **生产就绪标准**：AWS FTR 验证、ISO 认证路径、明确的价值切片定义
- **客户采用策略**：专注于低风险变更（如电话号码更新、营销偏好设置）
- **技术演进**：
  - 正在开发 V2 引擎室，集成 Amazon Bedrock AgentCore
  - 计划 MCPify 现有 API 接口
  - 探索 Kiro 和 Dynatrace MCP 在服务台的应用
- **成功要素**：明确业务价值主张，建立可衡量的成功标准，保持客户体验为核心
