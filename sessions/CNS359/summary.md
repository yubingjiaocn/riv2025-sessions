# AWS re:Invent 2025 CNS 359 会议总结

## 会议概述

本次会议由AWS首席专家解决方案架构师Diraj Mapatra和解决方案架构师AJ Park主讲，主题为"使用AWS无服务器服务构建、部署和运营智能体应用程序"。这是一个300级别的技术会议，重点讨论了从单一智能体系统到多智能体系统的架构演进，以及如何在生产环境中大规模运行智能体应用。

会议以一个实际的客户服务场景为例：客户在黑色星期五购物后遇到重复扣费、商品缺失和紧急配送需求的问题。通过这个案例，演讲者展示了如何从传统的无服务器API架构演进到智能体驱动的解决方案，并介绍了MCP（模型上下文协议）和A2A（智能体到智能体）等开放标准的应用。

## 详细时间线与关键要点

### 0:00-10:00 会议开场与问题定义
- 介绍演讲者和会议议程
- 提出客户服务场景：重复扣费、商品缺失、紧急配送需求
- 展示传统无服务器架构：订单API、库存管理、支付系统、客户支持API
- 分析当前挑战：支持人员需要查看多个分散系统，客户等待时间长

### 10:00-20:00 从传统架构到智能体的演进
- 介绍两种上下文提供方式：搜索驱动和工具调用驱动
- 解释增强提示的概念和JSON文档结构
- 展示智能体循环的工作原理：提示→模型调用→工具执行→结果返回→重复
- 对比传统API与智能体架构的差异

### 20:00-30:00 单一智能体到多智能体系统
- 讨论智能体微服务的概念
- 展示如何将单体智能体分解为领域特定的智能体
- 介绍扩展能力的方法：添加产品推荐等功能
- 演示Strands Agent SDK的使用方法和代码示例

### 30:00-40:00 智能体开发实践
- 详细展示订单智能体的Python代码实现
- 对比Step Functions工作流与智能体方法的差异
- 强调声明式编程相对于命令式编程的优势
- 介绍Strands Agent的特性：内置错误处理、模型和计算无关性

### 40:00-50:00 开放标准与协议
- 介绍MCP（模型上下文协议）解决智能体与工具集成的标准化问题
- 展示MCP客户端和服务器的代码示例
- 介绍A2A（智能体到智能体）协议用于智能体间通信
- 演示智能体卡片和能力发现机制

### 50:00-60:00 部署模式与AWS服务
- 介绍三种部署选项：Lambda函数、ECS Fargate、Agent Core
- 详细说明Agent Core套件：运行时、网关、托管工具、身份管理
- 展示多智能体架构模式：串行管道、编排器模式、群体架构
- 讨论安全考虑和治理最佳实践

### 60:00-62:10 总结与资源
- 强调无服务器最佳实践在智能体应用中的适用性
- 提供相关会议推荐和学习资源
- 鼓励参会者填写调查问卷并保持联系
