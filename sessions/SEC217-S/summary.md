# AWS re:Invent 2025 会议总结：使用代理工作流实现自动化云安全修复

## 会议概述

本次会议由Dynatrace的首席解决方案工程师Matt Gibbs和Shannon，以及AWS的代表Raj共同主讲，主题聚焦于通过代理工作流（Agentic Workflows）实现自动化云安全修复。会议展示了Dynatrace与AWS Security Hub的深度集成，演示了如何利用AI驱动的自动化技术来解决当今云安全面临的挑战。

核心内容围绕三个关键技术的整合：Dynatrace可观测性平台、AWS Security Hub统一安全解决方案，以及基于MCP（Model Context Protocol）协议的代理工作流。通过这种整合，企业能够在同一平台上统一管理性能数据和安全数据，并利用Davis Copilot AI助手自动检测和修复AWS环境中的安全漏洞。会议通过现场演示展示了从发现安全问题到自动修复的完整流程，整个过程仅需约2分钟即可完成，无需人工干预。

演讲者强调，Dynatrace的Grail架构是实现这一自动化的关键，它能够将来自不同源的遥测数据（应用工作负载、基础设施、日志、AWS Security Hub等）存储在统一的上下文中，从而避免自动化错误决策。这种方法不仅提高了响应速度和效率，还能随着云系统和AI系统的复杂度增加而扩展，最终加速企业的安全态势管理。
