# AWS re:Invent 2025 技术会议总结

## 会议概述

本次会议由 Atangi Ranganatan 主讲，重点介绍了 AWS 软件合作伙伴计划如何加速软件合作伙伴的业务增长。会议深入探讨了当前 SaaS 市场的发展趋势和挑战，指出尽管 SaaS 市场以 20% 的速度增长，预计到 2029 年全球 IT 支出将达到 1.7 万亿美元，但目前仍有超过 50% 的企业工作负载尚未迁移到云端。客户在云转型过程中面临诸多挑战，包括对云的犹豫态度、安全顾虑、并行代码库维护困难、第三方集成复杂性、合规性问题以及成本考量。

为了应对这些挑战，AWS 推出了完整的合作伙伴盈利框架，通过"构建、营销、销售、增长"四大支柱为技术合作伙伴提供全方位支持。会议详细介绍了四个核心计划：Marketplace List and Sell 计划、Marketplace Private Offer Promotions (MPOP) 计划、ISV Accelerate 计划以及 ISV Workload Migration Program (WMP)。这些计划通过财务激励、销售支持、技术赋能和市场推广等多维度手段，帮助软件合作伙伴加速客户的云迁移进程，实现业务增长和更好的客户成果。

## 详细时间线

00:00:00 - 会议开场与市场趋势分析
- 演讲者 Atangi Ranganatan 介绍会议主题：AWS 软件合作伙伴计划
- SaaS 市场以 20% 的速度增长
- 全球 SaaS IT 支出预计 2029 年达到 1.7 万亿美元，2025 年底将达到 3000 亿美元
- 目前仍有超过 50% 的企业工作负载未运行在云端

00:02:30 - 客户云转型面临的挑战
- 对云的普遍犹豫和感知风险
- 安全顾虑
- 本地软件与 SaaS 版本的并行代码库维护问题
- 大型企业解决方案与第三方集成的复杂性
- 特定行业的合规性和监管问题
- 成本仍是影响云转型的主要因素

00:04:00 - 成功迁移的关键模式
- 明确且有说服力的商业案例
- 强大的 SaaS 价值主张
- 成本透明度
- 避免客户因挑战而变得被动并延迟转型

00:05:00 - AWS 合作伙伴盈利框架介绍
- 提供可扩展和可靠的方式最大化合作伙伴价值
- 不同业务模式提供不同盈利水平
- 核心理念：为客户提供更好的价值，实现更高的盈利能力

00:06:30 - 四大核心支柱：构建、营销、销售、增长
- **构建 (Build)**：在云端构建解决方案和能力
- **营销 (Market)**：通过专业化认证、市场支持和 Marketplace 列表推动需求
- **销售 (Sell)**：销售支持、联合销售支持以及配套计划和激励措施
- **增长 (Grow)**：战略合作机会、额外资金机制和增强计划

00:08:00 - 技术合作伙伴的特定支持
- 技术合作伙伴提供市场就绪的解决方案
- 专注于特定业务成果而非仅销售技术
- 帮助客户淘汰技术债务并超越竞争对手

00:09:00 - Marketplace List and Sell 计划
- 财务激励计划，连接 AWS 技术合作伙伴与其他合格 AWS 合作伙伴
- 帮助在 AWS Marketplace 上构建、营销和销售
- 加速联合销售，增加客户采用率和收入增长
- 提供财务激励解锁产品主导增长
- 连接第三方系统集成商简化 Marketplace 体验
- 合作伙伴体验更快的上市时间、更高的项目成功率和增加的收入流

00:11:30 - Marketplace Private Offer Promotions (MPOP) 计划
- AWS Marketplace 的旗舰激励计划
- 基于每个机会计算激励，按 Marketplace 交易的 TCV 的 1-2% 计算
- 信用额度直接发放给最终客户
- 2025 年 8 月推出增强功能，简化激励接收流程
- 目前参与 MPOP 的合作伙伴数量是两年前的 5 倍

00:13:00 - MPOP 计划的影响数据
- 过去 12 个月接收 MPOP 的客户下游软件销售额增加 50%
- 下一个 Marketplace 交易机会的滞后时间更短
- 接收 MPOP 优惠的客户 TCV 支出比未接收的客户高 1.7 倍

00:14:00 - 联合销售 (Co-Sell) 的定义
- AWS 与合作伙伴携手合作的协作方式
- 通过互补服务和计划、集成工具和流程、配套计划和激励措施实现
- 帮助客户更快实现 AWS 投资价值
- 合作伙伴增长 AWS 业务并增强客户信任

00:15:30 - ISV Accelerate 计划介绍
- 全球联合销售计划，面向寻求与 AWS 深度合作的合作伙伴
- 合作伙伴拥有市场就绪的专业化解决方案
- 适用于商业和公共部门合作伙伴
- 最近向拥有 SaaS 解决方案的服务合作伙伴开放
- 关键要求：在 AWS 上运行或集成的 SaaS 解决方案，并通过 AWS Marketplace 销售

00:17:00 - ISV Accelerate 计划的核心优势
- 为内部销售团队提供销售补偿，激励他们更紧密合作
- 包括 SaaS Co-Sell Benefit 计划，为客户账户经理提供配额退休激励
- 解决方案在合作伙伴解决方案查找器和推荐引擎中获得更高可见性
- 在 CRM 系统中突出显示计划参与情况
- 优先访问联合销售指南、网络研讨会和活动

00:19:00 - ISV Workload Migration Program (WMP) 介绍
- 面向在 AWS 上拥有 SaaS 解决方案的软件合作伙伴
- 适用于迁移大型传统安装基础或竞争解决方案到 AWS SaaS 解决方案
- 目标：推动可重复和可扩展的迁移模式，降低迁移风险并抵消迁移成本

00:20:30 - WMP 的三大价值支柱
- **技术赋能**：访问 AWS 解决方案架构师，审查迁移方法并提供优化建议
- **财务激励**：基于每个机会计算，帮助创建更好的客户资金优惠
- **市场推广**：联合现场消息传递、集成营销活动、客户参考和电子书等

00:22:00 - WMP 的合格标准
- 从非 AWS 来源迁移（本地软件、开源软件或其他云/私有数据中心的第三方解决方案）
- 遵循 AWS 迁移 7R 框架
- 导致 AWS 外部系统的弃用
- 增加 AWS 上 SaaS 解决方案的消费

00:23:30 - WMP 适用的合作伙伴类型
- 传统软件提供商
- 云原生合作伙伴
- 希望将 AWS 外部运行的开源解决方案货币化的合作伙伴

00:24:00 - WMP 激活后的成果
- 软件合作伙伴收入流增加
- 迁移时间框架加快
- 客户体验改善
- 可重复和可扩展的模式结合强大的市场推广方法，创建加速客户迁移的飞轮效应

00:25:00 - WMP 激励机制详解
- 基于每个机会计算
- 根据第一年年度经常性收入 (ARR) 计算
- 采用分层激励模型：ARR 的 15% 或 25%
- 激励以 AWS 促销信用形式提供，可用于所有 AWS 服务
- 信用直接发放到运行 SaaS 解决方案的 ISV 账户

00:26:30 - WMP 2024 年 10 月重大增强
- 允许信用直接发放给最终客户
- 消除 ISV 合作伙伴将信用转移给客户的负担
- 适用于所有通过 WMP 和 Marketplace 交付的合格交易

00:27:30 - 如何开始使用这些计划
- 确定自己在"构建、营销、销售、增长"旅程中的位置
- 从 APN 合作伙伴网络提供的计划中选择适合的计划
- 与 AWS 客户经理或合作伙伴发展专家/代表合作开始实施
