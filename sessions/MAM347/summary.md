# AWS 主机现代化技术会议总结

## 会议概述

本次 AWS re:Invent 技术会议重点介绍了主机现代化(Mainframe Modernization)的创新解决方案。会议由 AWS 团队主持,邀请了西联汇款(Western Union)和 Unum 保险公司的代表分享他们使用 AWS Transform 和合作伙伴(Accenture、Pega)的可组合式 AI 代理框架进行主机现代化的实践经验。

AWS Transform 是首个用于大规模迁移和现代化的代理服务,通过生成式 AI 和代理式 AI 技术,将传统需要数年完成的转型项目缩短至数月甚至数周。该服务涵盖评估、VMware、主机和 .NET 四大支柱,能够自动化整个转型流程——从评估阶段的代码分析、文档生成,到代码转换(如 COBOL 转 Java)、测试自动化和数据迁移。

会议的核心亮点是"可组合性(Composability)"概念的提出。AWS 通过与合作伙伴共同创新,整合各方最佳工具和代理,避免了市场上工具选择的混乱,为客户提供量身定制的解决方案。西联汇款和 Unum 的案例充分展示了这种方法的实际效果:代码转换时间从数月缩短至数周,文档生成从数周缩短至数小时,显著提升了转型速度和效率。

## 详细时间线与关键要点

### 00:00 - 会议开场与主题介绍
- 主持人介绍主机现代化主题,强调虽然听起来不够"激动人心",但内容将非常精彩
- 介绍嘉宾:西联汇款的 Will Holloway 和 Unum 的 Jay Moody
- 核心议题:使用 AWS Transform 和可组合式 AI 代理框架加速主机现代化

### 02:30 - AWS Transform 服务概述
- AWS Transform 是首个用于大规模迁移和现代化的代理服务
- 四大支柱:评估、VMware、主机、.NET
- 利用生成式 AI 和代理式 AI 实现快速转型(年→月,月→周)

### 04:15 - 主机现代化的挑战
- 数十年积累的 COBOL 代码成为"黑盒",知识流失严重
- 技术债务层层叠加,缺乏文档
- 客户无法评估风险和制定现代化计划

### 06:00 - AWS Transform 的核心能力
- 自动分析数百万行代码
- 生成技术文档、功能文档和业务规则提取
- 提供代码转换代理(COBOL 到 Java)
- 集成行业知识库(与 Accenture 合作)
- 自动化测试和数据迁移

### 09:20 - 市场面临的三大挑战
- 工具混乱:市场上存在大量重叠的工具,客户难以选择
- 利益相关者不一致:不同工具和模式的提案导致决策困难
- 集成挑战:多个孤立工具难以协同工作

### 11:00 - 可组合性(Composability)解决方案
- 整合市场上所有最佳工具,实现"1+1>2"的效果
- AWS 迁移和现代化代理 + 第三方代理注册
- 代理编排层:支持并行和线性工作流
- 客户也可以引入自己的代理

### 14:30 - 可组合性的价值
- 对合作伙伴:市场差异化,共同销售机会
- 对客户:量身定制的解决方案,而非强制适配
- 对 AWS:通过合作伙伴扩展技术组合,共同创新

### 16:45 - 技术架构与安全性
- 代理注册能力和 API 控制
- MCP(模型上下文协议)服务器支持代理间通信
- A2A(代理到代理)协议实现编排
- VPC 隔离运行时确保数据安全
- 细粒度 IAM 策略控制访问权限
- 日志记录和可观测性

### 20:00 - 西联汇款案例开始
- Will Holloway 介绍西联汇款背景
- 公司历史:1861 年首创电报,1871 年开始汇款服务
- 现状:覆盖 200 多个国家,约 50 万个零售网点

### 22:30 - 西联汇款的战略目标
- "超越汇款"战略:三年内将消费者服务收入从 15% 提升至 25%
- 目标:未来三年实现 10 亿美元收入增长
- 70% 的收款人对额外金融服务产品感兴趣

### 25:00 - 汇票(Money Order)业务现状
- 美国和加拿大每年销售超过 1.1 亿张汇票
- 主要服务于无银行账户人群
- 现有系统无法支持 5 亿美元的增长目标

### 27:15 - 汇票业务面临的挑战
- 工程专业知识日益减少
- 需要以成本效益和动态方式准备容量增长
- 需要推出新解决方案和功能
- 后台运营效率低下:15 个团队使用终端屏幕处理数千万张汇票

### 30:00 - 现代化重点
- 一个产品,三个角色:客户、代理商、后台团队
- 需要为所有角色提供现代化界面
- 提高账户清算速度,匹配银行处理速度
- 人性化设计,摆脱终端窗口

### 33:45 - 西联汇款试点项目进展
- 2024 年第四季度正在进行试点
- 重点关注人性化设计和用户体验
- 利用聊天机器人简化工程复杂性
- 保留现有术语和命名规范

### 36:20 - 试点项目成果
- 250 万行代码分析
- 仅用 1.5 个月完成基础设施搭建、代码转换和测试启动(传统方法需 3 个月以上)
- 53,000 行 COBOL 代码在 1.5 小时内完成转换
- 31 个元素的技术文档在 19 小时内完成(使用 Accenture 代理)
- 业务需求文档在 5 小时内完成
- 成功迁移 21 个 VSAM 文件和 25 个 DB2 表,无准确性问题

### 40:00 - 西联汇款的目标与愿景
- 提高所有用户的速度和可用性
- 将传统产品带入现代化时代
- 通过新功能在市场中保持主导地位并吸引竞争对手的业务

### 42:00 - Unum 案例开始
- Jay Moody 介绍:Unum IT 交付和数字流程自动化 AVP
- 互动环节:现场调查显示很少有人完成成功的转型项目

### 44:30 - Unum 公司背景
- 保险公司,提供雇主赞助的福利(人寿、伤残、意外、牙科、视力等)
- 成立 175 年以上,积累了大量遗留技术
- 总部位于田纳西州查塔努加,全球设有办事处
- 服务 370 万人,90,000 个组织和公司
- 在终身寿险和万能寿险领域排名第二,意外和住院赔偿领域排名第三

### 47:00 - Unum 面临的挑战
- 理赔审查员需要同时打开 7 个不同窗口处理一个理赔
- 工作流程碎片化,手动处理,系统分散
- 9 名理赔经理每天花费 3 小时手动分配工作(每年 7,000 小时)
- 主机系统性能高但缺乏灵活性
- 文档不完善,难以维护和改进

### 50:15 - Unum 的转型方法
- 标准流程:评估、发现研讨会、代码获取、分阶段分析
- 使用 AWS Transform 和 Pega Blueprint
- 构建、验证和确认

### 52:00 - 第一个挑战:找到代码
- 遇到"地下室的 Bob"问题:维护 30 年主机的工程师对代码非常保护
- 强调同理心的重要性:理解可能正在替换 Bob 30 年前编写的代码

### 54:00 - 代码分析过程
- 将代码上传到 Amazon S3 存储桶
- 连接 AWS Transform 到 S3
- 要求代理分析代码并从主机系统中提取业务规则
- 分析了 150 万行代码
- 生成业务规则提取文档,包含代码路径、角色、集成和数据模型

### 56:30 - Pega Blueprint 的应用
- Pega 是世界级的低代码/无代码工作流自动化和 AI 决策工具
- Blueprint 是免费在线工具
- 可以上传各种文档,包括 Transform 业务规则提取或屏幕操作录像
- 自动分析并提取工作流,组织成阶段和步骤
- 可视化界面,业务专家可直接参与流程重新定义
- 成为 IT 和业务之间的合同

### 59:00 - 会议总结
- 强调从发现到测试的快速转型(6 周)
- 两个客户案例都展示了显著的时间和成本节省
- 可组合式 AI 代理框架是成功的关键
