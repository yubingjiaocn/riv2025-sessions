# AWS re:Invent 2025 SAP 工作负载迁移会议总结

## 会议概述

本次会议重点介绍了Intel与AWS在SAP工作负载优化方面的深度合作,以及丰田汽车北美公司(TMNA)成功将SAP系统迁移至AWS云平台的案例。会议由Intel团队Amazon业务负责人Akong Sha Balani主持,AWS首席技术客户经理Neil Readyi和丰田汽车的Ashish分享了实际迁移经验。

Intel与AWS的合作关系已超过18年,从2006年EC2服务推出至今,Intel一直为AWS提供底层芯片支持。目前AWS EC2上约85%的实例由Intel处理器驱动。会议特别强调了第八代Intel实例的发布,这些实例基于Intel Xeon 6处理器,相比上一代性能提升高达20%。对于SAP这类关键企业工作负载,Intel与AWS共同开发了专门优化的U7i实例,提供高达32TB内存和896个CPU核心,是云端最密集的实例之一。

丰田汽车的迁移案例被称为"奇迹般的迁移"。从2016年开始SAP绿地实施,到2019年首个制造工厂上线,再到2021年迁移至AWS云平台,丰田克服了硬件采购困难、COVID-19疫情等多重挑战。通过采用AWS的U7i实例,丰田不仅解决了可扩展性问题,还将关键财务结算流程从3天缩短至24小时以内,系统稳定性显著提升,生产中断事件大幅减少。

- 专为SAP HANA等大型内存数据库设计

00:15:45 - U7i实例的技术优势
- 专为SAP构建的高内存实例
- 通过SAP认证

- 无需考虑物理基础设施的电力、环境控制、灾难恢复等问题

00:18:20 - SAP工作负载的四大关键需求
- 可扩展性:支持业务增长
- 性能:满足终端用户体验要求
- 可靠性:确保系统稳定运行
- 运营卓越:持续优化配置

00:20:00 - R8i实例规格介绍
- 第八代计算优化实例,同样通过SAP认证
- CPU核心数等技术规格展示
- 丰田计划未来采用R8i实例

00:22:30 - AWS Well-Architected框架在SAP中的应用
- 企业支持团队帮助客户关注运营卓越、可靠性、性能
- 定期进行灾难恢复测试和业务连续性检查
- SAP专用架构审查(SAP Lens)
- 持续迭代优化,而非一次性项目

00:25:00 - 可靠性支柱案例
- 挑战:手动流程、配置不一致、技能差距导致SAP弹性能力有限
- 解决方案:定期审查高可用性和灾难恢复架构
- 成果:增强审计合规性,实现HA和DR目标

00:27:15 - 运营卓越案例
- 挑战:配置复杂性和配置漂移影响性能和事件响应
- 解决方案:使用Systems Manager for SAP进行配置管理和最佳实践检查
- 成果:实现一致的SAP配置,提升系统稳定性

00:29:00 - 丰田汽车公司背景介绍
- Ashish上台分享丰田案例
- TMNA负责北美地区的制造、销售和研发
- 支持13个制造工厂和企业职能部门
- 运行混合SAP应用:本地部署(AWS上)和SaaS应用(Ariba、SuccessFactors)
- 正在利用SAP BTP套件服务,如集成套件、数据球等

00:31:30 - 丰田SAP迁移旅程时间线
- 2016年:开始SAP绿地实施项目
- 2016-2019年:规划阶段,整合企业职能(财务、采购、发票等)
- 2019年4月1日:首个制造工厂模板上线(愚人节)
- 面临采用SAP到制造环境的诸多挑战

00:34:00 - 2019-2023年持续推广阶段
- 四年内多次上线和转换项目重叠进行
- 期间经历6个月COVID-19疫情,但时间表未改变
- 团队远程工作仍按时完成13个制造单元的转换
- 同时进行多次重大版本升级

00:36:15 - SAP版本升级历程
- 2019年:从ECC 6.0升级到S/4 HANA 1709
- 2020-2021年:升级到S/4 HANA 2020
- 期间还完成BW/4HANA升级
- 在推广SAP到各制造工厂的同时保持系统更新

00:38:00 - 可扩展性挑战的出现
- 首个制造工厂上线后,快速意识到平台扩展性不足
- 数据增长迅猛:从1TB增长到3TB再到6TB
- 2019-2020年一年内数据量翻倍多次
- 每次需要硬件都要经历漫长的采购流程
- COVID期间芯片短缺,无法及时采购硬件

00:40:30 - 2020-2021年云迁移决策
- 2020年开始云转型之旅
- 迁移期间未暂停SAP转换工作
- 最初计划6TB实例,但2021年上线时直接部署12TB生产实例
- 6个月内硬件容量翻倍,AWS快速响应需求
- 2024年升级到24TB实例

00:43:00 - 第一个问题解决:可扩展性
- 高内存实例问题得到解决
- 部署速度更快
- 环境弹性更好
- 但出现新问题:CPU限制

00:44:30 - CPU瓶颈问题
- 第一代U系列实例CPU仅448核心
- 2020-2022年间,生产系统报告11次中断
- 生产系统宕机直接影响制造单元
- 系统不稳定导致制造效率下降和经济损失

00:46:00 - U7i实例预览与测试
- 丰田向AWS TAM团队反馈需要更多CPU核心
- 2024年2月获得U7i实例预览访问权
- U7i提供896核心,是上一代的两倍
- 正值S/4 HANA 2023升级项目期间

00:48:15 - 零停机升级(ZDO)体验
- 使用零停机选项进行S/4 HANA升级
- 在生产系统中运行预升级步骤
- 仅需短暂停机即可完成切换
- 在U7i上运行效率提升超过30%
- 旧一代系统预升级耗时更长,U7i显著加快

00:50:00 - 快速迁移到U7i生产环境
- 2024年2-4月参与预览测试
- U7i正式发布后,从签署协议到实例分配仅10天
- 从U6 bare metal迁移到U7i生产工作负载仅7-10天
- AWS EC2工程团队、TAM、企业支持团队全力配合
- 在生产升级关键时期完成迁移,无任何停机或业务影响

00:52:30 - 财务月结流程优化案例
- 月度财务账簿关闭涉及大型劳动力和间接费用计算流程
- 多个批处理作业和团队协作
- 旧一代系统:整个流程端到端需要3个完整工作日
- 通过实施SAP HANA工作负载类来限制CPU使用
- U7i迁移后:移除资源争用限制,流程缩短至24小时以内
- 节省大量人工时,避免因作业失败导致的重跑

00:55:00 - 应用层实例升级
- 数据库层面的改进之外,应用层也进行优化
- 从R5实例迁移到R6i实例
- 应用层性能提升10-15%

00:56:30 - R8i实例评估与未来计划
- 当前正在评估迁移到R8i实例
- R8i提供38,000 SAPS,而R6i仅18,000 SAPS(SAPS是SAP运行时性能测量单位)
- SAPS性能翻倍以上
- 预期效率和性能提升至少50%
- 预计EC2成本降低约20%
- 计划2026年re:Invent分享R8i实施成果
