# AWS re:Invent 2025 会议总结：从用户体验到智能体体验的电商革命

## 会议概述

本次会议由AWS解决方案架构师Adi Pendiala和Martin Sakowski主讲，聚焦电商领域正在经历的重大转型——从传统的用户体验(UX)向智能体体验(Agent Experience)的转变。演讲者开场通过一个简单的调查揭示了当前的现状：许多人使用ChatGPT或Perplexity搜索产品，但没有人让AI直接完成购买。这个差距正是本次会议要探讨的核心问题。

会议指出，51%的Z世代消费者已经开始在AI平台上进行产品搜索，而不是直接访问电商网站。预计到2030年（仅4年后），通过智能体完成的年销售额将达到5000亿美元，Gartner预测25-30%的电商交易将通过智能体完成。更值得关注的是，OpenAI报告其平台上的产品搜索量同比增长340%，而传统的Google产品搜索流量下降了23%，点击率下降了30%。这些数据表明，流量并未消失，只是消费者的购物方式发生了根本性改变。

演讲者强调，过去二十年电商行业在用户体验上的所有努力——精美的视觉设计、流畅的导航、优化的页面加载时间——对智能体来说毫无意义。智能体关注的是良好的API设计、结构化数据和可靠的交互协议。会议重点介绍了Model Context Protocol (MCP)作为解决方案，以及AWS Bedrock Agent Core Gateway如何帮助企业无缝过渡到智能体时代，而无需重建整个技术栈。

## 详细时间线与关键要点

00:00 - 开场与现状调查
- 通过现场调查揭示差距：许多人用AI搜索产品，但无人让AI完成购买
- 这个差距代表了自移动革命以来电商领域最大的转变

02:30 - 电商现状数据分析
- 51%的Z世代消费者在AI平台上开始产品搜索
- 预计到2030年通过智能体实现5000亿美元年销售额
- Gartner预测25-30%的电商将通过智能体完成
- OpenAI平台产品搜索量同比增长340%
- Google产品搜索流量下降23%，点击率下降30%

04:45 - 从用户体验到智能体体验的转变
- 传统电商专注于视觉设计、导航优化、页面加载速度
- 智能体不关心这些，它们需要：良好的API设计、结构化数据、可靠的交互协议
- 如果网站不适配智能体体验，将失去三分之一的电商流量

06:20 - 智能体协议概览
- 介绍四种主要协议：
  - ACP (Agent Commerce Protocol)：OpenAI和Stripe发布，专注产品搜索和结账流程
  - AP2：Google支付流程协议
  - A2A (Agent-to-Agent)：Google的智能体间通信协议
  - MCP (Model Context Protocol)：Anthropic发布的"瑞士军刀"协议

08:15 - MCP深度解析
- MCP的三大核心能力：
  1. Resources（资源）：可读数据源，如产品目录、库存、订单历史
  2. Tools（工具）：可执行操作，如加入购物车、处理支付、更新库存
  3. Prompts（提示）：预构建的交互模式，帮助智能体有效使用工具和数据

10:00 - 本地MCP vs 远程MCP
- 本地MCP：在本地计算机运行，适合本地自动化，不适用于电商场景
- 远程MCP：通过网络连接，是电商智能体交互的必需方案

11:30 - 实施挑战
- 安全性：身份验证和授权管理
- 可观测性：从用户意图到系统执行的完整追踪
- 可扩展性：智能体可并行执行数千到数万次请求，远超人类点击速度
- 演进性：协议和能力持续变化，需保持兼容性

13:45 - Amazon Bedrock Agent Core Gateway介绍
- 无需从头编写MCP服务器，自动将现有API和Lambda函数转换为MCP兼容工具
- 提供单一安全端点处理所有智能体交互
- 内置身份验证、授权和安全管理
- 上下文感知发现：根据智能体意图智能展示相关工具和权限

16:00 - 工作原理演示
- 架构说明：智能体 → Agent Core Gateway → 现有API/Lambda函数
- Gateway作为翻译层，暴露单一MCP端点
- 后端基础设施无需改变，保持现有API和Lambda函数不变
- 自动处理安全性、可扩展性、可观测性和演进性问题

18:20 - 实施四步骤
1. 识别要暴露的API能力，准备OpenAPI规范
2. 配置身份验证：入站（智能体到Gateway）和出站（Gateway到后端API）
3. 设置Agent Core，使用基础设施即代码（如CDK）进行生产部署
4. 获取MCP端点URL，连接到Claude、ChatGPT或自定义智能体框架进行测试

20:30 - 最佳实践
- **设计智能体优先的API**：确保结构化数据、数据语义和丰富元数据；智能体是字面化的，数据类型必须一致准确
- **渐进式披露**：从小规模开始，先暴露产品元数据、定价、结账等高影响API，测试后再逐步扩展
- **多协议支持**：保持灵活性，为未来可能成为标准的新协议做好准备，通过Agent Core抽象复杂性

22:45 - 总结与行动呼吁
- 从用户体验到智能体体验的转变是真实且不可避免的
- MCP提供强大基础，避免重新发明轮子
- Bedrock Agent Core Gateway消除实施复杂性
- 关键问题不是"是否"要为智能体构建电商，而是"何时"开始
- 提供博客文章二维码供深入学习

24:00 - 会议结束
- 演讲者留下来回答问题
