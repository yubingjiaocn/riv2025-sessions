# AWS re:Invent 2025 边缘架构弹性优化会议总结

## 会议概述

本次会议是一场400级别的代码演示会议，主题为"使用AI构建和优化边缘架构的弹性"。会议由AWS开发者倡导者Robert Belson和首席解决方案架构师Jesus Federico共同主持。两位演讲者都曾在电信垂直领域工作，现专注于生成式AI和代理AI在混合边缘服务中的应用。

会议采用全新的代码演示格式，允许现场编程和实时互动。演讲者强调这是一个小型亲密的环境，鼓励观众提问和参与。会议的核心理念是展示如何使用生成式AI工具来帮助设计和优化AWS混合边缘基础设施的弹性，而不是传统的在边缘运行AI模型的方式。

## 详细时间线与关键要点

### 0:00-10:00 会议介绍与背景设定
- 介绍400级别代码演示会议的新格式
- Robert Belson自我介绍：纽约的开发者倡导者，专注混合边缘服务和生成式AI
- Jesus Federico自我介绍：首席解决方案架构师，专注生成式AI，曾在电信垂直领域工作
- 强调两人在相关主题上合作超过一年，将分享客户反馈和最佳实践

### 10:00-20:00 AWS混合边缘服务概述
- 回顾2019年re:Invent首次发布的AWS混合边缘服务
- 介绍云连续体概念：Regions、Local Zones、Outposts、Wavelength Zones
- 现场调查：只有2-3位观众体验过Local Zone或Outpost
- 详细解释各服务定义：Local Zones扩展到都市区域，Outposts为42U机架式计算设备
- 介绍EKS Hybrid Nodes和远程边缘设备支持服务

### 20:00-30:00 弹性架构理论基础
- 定义弹性：工作负载从基础设施或服务中断中恢复的能力
- 弹性的三个层面：高可用性、灾难恢复、备份管理
- 混合边缘环境的弹性考虑因素：
  - 站点级弹性：地理多样性
  - 网络弹性：多路径连接
  - 计算弹性：冗余计算单元
  - 数据弹性：集群和复制
  - 运营弹性：自动扩展、健康检查、监控

### 30:00-40:00 AI代理解决方案架构
- 提出使用MCP服务器代表各领域专家的概念
- 介绍代理循环：目标设定→工具使用→数据收集→目标评估→迭代
- 展示从静态配置文件到高级AI代理的演进路径
- 介绍Strands Agents作为构建代理的开源SDK
- 说明多模式策略：Agent as Tool、Swarm、工作流

### 40:00-50:00 实际演示环境设置
- 演示环境：法兰克福区域，5个Outposts（4个活跃，1个配置中）
- 介绍Kiro CLI作为Q Developer CLI的演进版本
- 现场调查：60-70%的观众接触过Strands或Kiro
- 开始使用Kiro CLI创建静态发现脚本
- 生成edge_discovery.py文件，使用boto3发现边缘资源

### 50:00-59:48 高级代理实现与多代理模式
- 展示从静态报告到动态MCP集成的演进
- 介绍AWS知识MCP服务器获取最新文档
- 演示Strands Agent的创建，仅需3-4行核心代码
- 展示多代理模式：
  - Agent as Tool：将代理作为工具暴露给协调代理
  - Swarm模式：无单一协调者的多代理协作
  - 工作流模式：结构化任务协调
- 强调可在AgentCore中运行Strands Agent实现安全隔离
- 总结：展示Well-Architected Framework审查的未来形态
