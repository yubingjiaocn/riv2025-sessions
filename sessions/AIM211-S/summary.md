# AWS re:Invent 2025 技术会议总结：Atlassian Rovo Dev - AI驱动的开发者助手

## 会议概述

本次技术会议由Atlassian高级技术布道师Jovana主讲，重点介绍了Atlassian的AI驱动开发者助手Rovo Dev。会议深入探讨了开发者生产力和代码质量的重要性，以及AI工具如何解决开发者在非编码任务上浪费时间的问题。

Jovana分享了Atlassian 2025年开发者体验报告的关键发现：99%的开发者报告使用AI节省了时间，但开发者仍然只将16%的时间用于编码，其余84%的时间都花费在非编码任务上。Rovo Dev正是为了解决这一痛点而设计，它是一个理解组织上下文的智能团队成员，能够在终端、IDE、CI/CD管道和Jira等各种工作环境中提供帮助。

## 详细时间线与关键要点

### 0:00-2:30 - 开场介绍与问题定义
- 演讲者自我介绍：Jovana，Atlassian高级技术布道师
- 强调开发者生产力投资的重要性：投资开发者 → 更好的产品 → 公司收益
- 分享2025年DevEx报告核心数据：
  - 99%开发者报告AI带来时间节省
  - 开发者仅16%时间用于编码
  - 84%时间浪费在非编码任务上

### 2:30-4:00 - Rovo Dev产品介绍
- 介绍Atlassian的愿景：为每个软件开发团队提供理解组织上下文的AI队友
- Rovo Dev核心特性：
  - AI驱动的开发者代理
  - 加速整个软件开发生命周期
  - 支持代码规划、生成、审查和自动化
  - 基于Atlassian Teamwork Graph提供上下文感知

### 4:00-6:30 - CLI演示
- 展示Rovo Dev CLI的实际应用
- 演示场景：修复网站上错误的演讲者头像
- 关键功能展示：
  - 直接从Jira问题链接获取上下文
  - 图像识别和智能修复能力
  - 自动检测和纠正图像中的错误元素

### 6:30-9:00 - 代码审查功能
- AI代码审查器自动功能：
  - 在创建拉取请求后自动运行
  - 提供bug、可维护性、可读性、安全性等方面的反馈
- 验收标准功能：
  - 自动比较代码与Jira需求
  - 验证是否按照预期构建功能
- 自定义指令：
  - 支持团队特定的编码标准
  - 使用自然语言描述组织原则

### 9:00-11:30 - Jira集成与VS Code扩展
- Rovo Dev在Jira中的应用：
  - 即将进入开放测试版
  - 直接从Jira问题触发AI工作流
  - 提供类似VS Code的交互式沙盒环境
  - 自主工作并创建拉取请求
- VS Code扩展功能：
  - 连接Atlassian云端
  - 显示分配的Jira票据
  - 提供实时通知功能

### 11:30-13:30 - 自动化与管道集成
- 自动化规则创建：
  - 跨团队和代码库执行代码质量标准
  - 自动化重复性任务
  - 支持自定义编码、安全和设计标准
- Bitbucket Cloud管道集成：
  - 诊断和解决CI/CD管道故障
  - 分析失败构建并提供根本原因
  - 通过使用不断学习和改进

### 13:30-14:30 - 差异化优势与总结
- Rovo Dev的独特优势：
  - 连接Jira、Confluence、Bitbucket和GitHub的完整工具链
  - 基于Teamwork Graph的上下文感知
  - 覆盖软件开发生命周期的AI自动化
  - 终端优先的丰富体验
  - 企业级安全性和定制化能力
