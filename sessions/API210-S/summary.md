# AWS re:Invent 2025 Redis 现代化缓存技术分享会总结

## 会议概述

本次技术分享会由Redis产品管理团队负责人Pieter Cailliau和ifood机器学习工程师Heber Scachetti联合主讲，主题为"使用现代化缓存驱动实时应用"。会议深入探讨了Redis从传统缓存到现代化数据平台的演进历程，重点介绍了Redis在解决热键问题、数据同步、向量搜索等方面的最新技术突破。

Pieter首先回顾了Redis的发展历程，从最初解决关系数据库读取性能问题的简单缓存，发展为支持多种数据结构和复杂查询的现代化数据平台。他强调Redis的演进完全由用户需求驱动，每当用户遇到性能瓶颈时，Redis团队就会开发相应的解决方案。Heber则从实际应用角度分享了ifood如何在大规模生产环境中使用Redis的各项新功能，包括GenAI平台的速率限制、RAG系统的向量搜索以及特征存储等关键应用场景。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与Redis概述
- Pieter介绍Redis基本概念：内存数据结构存储，支持缓存、会话存储、消息代理等多种用途
- Redis每日Docker拉取量达225万次，总计超过100亿次
- 连续被Stack Overflow评为最受欢迎的非关系型数据库
- 团队专注三个核心领域：Redis开源核心、开发者体验、数据迁移

### 5:00-15:00 热键问题分析与解决方案
- 引用微软和Oracle数据：80-90%的OLTP工作负载为纯读查询
- 热键问题的两种典型场景：高频访问键（如社交媒体影响者资料）和索引查询热键
- 客户端缓存解决方案：在客户端本地缓存热键副本，避免网络往返
- Redis查询引擎：支持复杂查询，包括全文搜索、地理位置、向量搜索等

### 15:00-25:00 向量搜索与语义缓存
- Redis向量搜索性能基准测试：在各种数据集和工作负载下均优于竞争对手
- LangCache语义缓存：缓存LLM响应，对语义相似问题返回缓存结果
- 性能提升可达15倍，显著降低LLM调用成本
- Asurion客户案例：70%缓存命中率，响应时间减半，客户服务门户参与度提升4倍

### 25:00-35:00 缓存模式演进与数据集成
- 传统Cache-aside模式的局限性：数据不一致、雷群效应、冷启动问题
- Refresh-ahead模式：主动数据同步，实现读写分离
- Redis数据集成(RDI)：基于Apache Flink的低代码数据同步解决方案
- Axis Bank案例：6百万日活用户，响应时间从173ms降至29ms，吞吐量从500 RPS提升至数万RPS

### 35:00-40:00 Redis Flex与成本优化
- Redis Flex自动分层存储：结合内存和SSD/NVMe存储
- 支持高达50TB数据库，成本降低75%
- 热数据保持在内存中，冷数据存储在较慢存储介质上
- 特别适用于特征存储等大数据量场景

### 40:00-45:00 ifood实际应用案例
- Heber介绍ifood规模：月订单量1.6亿，日活用户6千万，覆盖1500个城市
- GenAI平台：月处理1万亿token，月请求量4亿次，使用Redis进行速率限制
- RAG系统：使用Redis向量存储实现文档检索和语义搜索
- 特征存储：支持每秒500万次请求，存储近2000个不同特征，响应时间低于10毫秒
