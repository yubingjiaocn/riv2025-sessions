# AWS re:Invent 2025 会议总结：使用 AWS Agentic AI 实现数字化转型卓越

## 会议概述

本次 AWS re:Invent 2025 分会场主题为"使用 AWS Agentic AI 从愿景到胜利实现数字化转型卓越"。会议由 AWS 专业服务团队的 Brad Senko 和 Oscar Rodriguez 主讲，并邀请了丹斯克银行（Danske Bank）的高级副总裁 Ashish Shakar 分享实际案例。

会议核心内容聚焦于 AI 代理技术如何革命性地加速企业迁移和现代化进程。演讲者强调，通过 AWS 的 Agentic AI 工具，企业可以实现比传统方法快 10 倍的迁移速度，并降低 80-90% 的项目实施成本。这不是未来趋势，而是正在发生的现实。会议详细介绍了三大核心工具：AWS Transform（专为大规模迁移设计的 AI 代理）、Amazon Q Suite（无代码业务流程自动化平台）和 Amazon Q Developer（原 Curo，AI 驱动的集成开发环境）。

演讲者特别指出，当前 70% 的企业工作负载是 20-30 年前构建的遗留系统，面临技能短缺、文档缺失等挑战。而 Agentic AI 技术的出现，使得企业能够系统性地解决这些问题。到 2028 年，预计 15% 的技术和业务决策将由 AI 自主完成。丹斯克银行的案例展示了如何将这些技术应用于实际的主机迁移、应用现代化和开发效率提升项目中。

## 详细时间线与关键要点

### 开场与背景介绍（0:00-5:00）
- **演讲者介绍**：Brad Senko（AWS 专业服务迁移和现代化团队负责人）和 Oscar Rodriguez（AWS 专家服务全球实践总监）
- **核心主张**：使用 Agentic AI 实现 10 倍速度的迁移，成本降低 80-90%
- **嘉宾介绍**：丹斯克银行的 Ashish Shakar 将分享实际进展

### 行业挑战与机遇（5:00-10:00）
- **遗留系统困境**：70% 的工作负载建于 20-30 年前，面临遗留软件、硬件和流程问题
- **人才短缺**：许多原始开发人员已不在岗，文档缺失
- **AI 必要性**：需要 AI 和自动化工具来加速云迁移
- **行业拐点**：到 2028 年，15% 的日常技术和业务决策将由 AI 自主完成
- **采用现状**：82% 的组织刚开始利用 AI，早期采用者将获得竞争优势

### 客户需求与解决方案（10:00-15:00）
- **双重目标**：客户既想迁移到云端，又想利用 AI 能力
- **工具介绍**：AWS Transform（专用工作负载）、Q Suite（无代码解决方案）
- **开发者技能转变**：开发者需要掌握两项技能——编写完美提示词和调试 AI 生成的代码
- **迁移时间对比**：主机迁移传统需要 10-15 年，企业基础设施迁移需要 24-36 个月

### 传统方法 vs AI 驱动方法（15:00-20:00）
- **传统流程**：人工评估、构建示范 Pod、制定波次计划、扩展团队
- **超自动化**：去年引入的概念，今年叠加生成式 AI 自动化
- **工具套件**：
  - AWS Transform：评估基础设施、构建网络、制定波次计划、一键迁移服务器
  - Q Suite：无代码体验，询问预算、进度、人员需求等问题
  - Amazon Q Developer：使用 AI 完全重构遗留应用

### 实施框架与最佳实践（20:00-25:00）
- **三阶段方法**：评估（Assess）、执行（Execute）、扩展（Scale）
- **框架重要性**：需要治理框架来管理角色、责任、合作伙伴和培训
- **谨慎引入 AI**：建议"测量两次，切割一次"，避免过于激进或过于缓慢
- **实际成果**：3 倍交付速度提升，实施成本减半

### 实用工具与产品映射（25:00-30:00）
- **客户常见需求**：
  - 加速迁移至少 50%
  - 提高工作场所生产力
  - 快速原型开发
- **工具对应**：
  - AWS Transform：专用 AI 能力，用于迁移
  - Q Suite：自动化业务流程，无需编码
  - Amazon Q Developer：快速原型开发和应用重构

### Oscar Rodriguez 深入讲解（30:00-35:00）
- **AWS Transform 四大核心能力**：
  1. VMware 迁移：帮助客户退出 VMware
  2. 全栈 Windows 现代化：处理 .NET、SQL Server 和 Windows 许可证
  3. 主机现代化：处理大规模遗留系统
  4. 自定义转换：支持 COBOL 到 Java/.NET 等跨语言迁移

### Windows 现代化案例（35:00-40:00）
- **典型场景**：Microsoft .NET + SQL Server + Windows 虚拟机
- **转换能力**：
  - .NET Framework 迁移到 .NET Core（开源）
  - SQL Server 迁移到 Amazon Aurora PostgreSQL
- **数据迁移四步骤**：
  1. 分析：依赖关系、应用使用情况
  2. Schema 分析
  3. 数据迁移
  4. 存储过程转换和验证部署

### 主机现代化能力（40:00-45:00）
- **高需求领域**：过去 6 个月主机迁移需求激增
- **系统化方法**：
  - 代码分析
  - 逆向工程：自动生成文档（针对 10-20 年前的代码）
  - 代码分解：将单体应用拆分为业务域或微服务
  - 迁移规划：确定波次数量
  - 重构 vs 重新构想决策
  - 测试和部署

### 丰田汽车案例研究（45:00-48:00）
- **挑战**：
  - 大型主机系统
  - 主机技能短缺
  - 超过 4000 万行代码
- **成果**：
  - 比传统方法快 50% 以上
  - 识别技术债务和需要现代化的内容
  - CTO Brian Corser 评价："Transform 完成了许多人认为不可能的事情"

### Amazon Q Suite 介绍（48:00-52:00）
- **三大核心功能**：
  1. 统一数据源：整合 Excel、Salesforce 等数据，实时决策
  2. 教练和导师：帮助评估决策选项、利弊分析
  3. 触发行动：自动化工作流，发送邮件、修复问题等

### 业务流程自动化演示（52:00-57:00）
- **实际案例**：大型银行的贷款文档验证自动化
- **演示流程**：
  - 拖放驾驶执照和贷款申请表
  - 系统自动解析数据、识别缺失信息、应用业务规则
  - 欺诈检测：验证驾照真伪
  - 识别差异和匹配项
- **可扩展性**：可处理数百万条记录并行运行
- **持续学习**：系统随数据增长不断学习
- **人机协作**：保留人工审核环节

### Amazon Q Developer（Curo）介绍（57:00-62:00）
- **使用率调查**：现场约 30% 的人使用过 Q Developer
- **用户规模**：几个月内已有近 30 万订阅者
- **独特优势**：首个与 AWS 完全集成的 Agentic AI IDE
- **覆盖全生命周期**：需求、设计、实施全流程支持
- **三大应用场景**：
  1. 构建新应用：从原型到生产
  2. 优化现有应用：识别技术债务、重构代码
  3. 结合 AWS Transform：自动化迁移 + 人工精修

### Q Developer 实际演示（62:00-65:00）
- **演示案例**：为银行创建西班牙语银行应用
- **输入**：简单的自然语言描述 + 样式表链接
- **过程**：多个 AI 代理并行工作，分析规则、参考应用
- **结果**：几秒钟内生成可运行的原型
- **实际案例**：某客户从 3 个月计划缩短到 3 周交付

### AWS Agentic AI 完整产品组合（65:00-68:00）
- **三层架构**：
  1. **即用型工具**：Q Developer、Q Suite、AWS Transform
  2. **自建代理**：使用 Bedrock Agents、开源框架
  3. **自建模型**：通过 SageMaker 构建基础设施
- **建议**：根据组织成熟度选择合适层级

### 丹斯克银行案例介绍（68:00-72:00）
- **银行背景**：
  - 150 年历史的金融机构
  - 300 万客户，21,000 名员工
  - 高度监管行业
- **起始状态**：
  - 约 18,000 台虚拟机
  - 80,000 个容器
  - 25 PB 数据
- **挑战**：与大多数企业类似，既不太大也不太小，但遗留系统复杂
