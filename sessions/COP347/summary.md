# AWS re:invent 2025 - COP347 会话总结

## 会话概述

本次会话主题为"改善治理和合规性的可操作控制措施"（Actionable Controls for Improving Governance and Compliance），由AWS首席云解决方案架构师Rodolfo Fenes和AWS Control Tower团队首席产品经理Arnold Kandre共同主讲。会话重点介绍了如何在AWS环境中实施、评估和监控安全控制措施，以满足不同的合规性需求。

演讲者首先阐述了云治理的核心挑战：如何将抽象的法规要求转化为具体可执行的技术控制措施。他们强调了平衡创新与安全的重要性，并介绍了AWS将云治理分为四个关键领域：安全、运营、合规和成本管理。会话详细展示了AWS Control Tower的控制目录（Control Catalog）功能，该目录包含超过1000个预配置的AWS托管控制措施，并已预先映射到17个主流合规框架（如PCI DSS、CIS、NIST等）。

演讲的核心内容围绕基于风险的控制措施生命周期展开：选择、实施、评估和持续监控。通过实际演示，展示了如何使用Control Catalog快速查找、筛选和部署符合特定合规框架要求的控制措施，大大简化了传统的手动映射流程。

## 详细时间线

### 开场与背景介绍
00:00:00 - 00:03:30
- 会话开始，介绍主题COP347：改善治理和合规性的可操作控制措施
- 使用建筑比喻：控制措施是网络安全的隐藏架构，如同支撑墙体的螺栓和横梁
- 提出核心挑战：如何将"组织应确保加密控制措施到位"等抽象法规要求转化为实际实施

### 利益相关方分析
00:03:30 - 00:06:00
- 介绍多个关键利益相关方及其需求：
  - 业务人员：追求快速市场采用和创新机会
  - 开发人员：需要工作自主权和灵活性，避免最后时刻的合规变更
  - 安全团队：希望实施主动而非被动的控制措施
  - 风险管理者：确保组织遵守风险偏好
- 强调需要高层领导支持以平衡这些竞争性需求

### AWS云治理框架
00:06:00 - 00:09:30
- 介绍AWS云治理的四大支柱：
  1. 安全：遵循组织安全计划
  2. 运营：确保按预期执行和弹性
  3. 合规：符合内外部政策
  4. 成本：在成本效益下成功运营
- 讨论常见挑战：
  - 在创新与控制之间取得平衡
  - 跟上不断变化的监管环境（如EU数据法案、生成式AI法规）
  - 满足业务变化的敏捷性需求
  - 支持并购活动
  - 实现成本和运营效率

### 基于风险的方法论
00:09:30 - 00:12:00
- 介绍风险管理框架的必要性
- 展示基于风险的控制措施方法的核心步骤：
  1. 资产分类
  2. 选择控制措施
  3. 实施控制措施
  4. 评估控制措施有效性
  5. 持续监控
- 强调这是一个循环过程，而非一次性活动

### 控制措施类型详解
00:12:00 - 00:18:00
- Arnold接手讲解，介绍AWS Control Tower控制目录
- 详细说明三种控制措施类型：
 
 预防性控制措施：
  - 服务控制策略（SCPs）：组织级边界保护
  - 资源控制策略（RCPs）：资源级权限和限制
  - 声明性策略：服务级配置强制执行
 
 检测性控制措施：
  - 基于AWS Config规则
  - 监控不合规资源并发出警报
 
 主动性控制措施：
  - 基于CloudFormation Hooks
  - 在资源部署前进行验证

### AWS服务审批策略
00:18:00 - 00:24:00
- Rodolfo介绍三种服务审批方法：
 
 1. 全面审批方法：
  - 服务一旦批准即对所有团队开放
  - 难以适应不同工作负载的特定需求
 
 2. 上下文服务审批（推荐）：
  - 通过威胁建模练习评估服务
  - 仅在特定工作负载上下文中批准
  - 允许灵活性和演进
 
 3. 混合两阶段审批：
  - 适用于高度监管行业
  - 利用AWS共享责任模型
  - 第一阶段：验证AWS服务符合所需合规框架
  - 第二阶段：审查客户责任部分的上下文实施

### 共享责任模型
00:24:00 - 00:26:00
- 解释AWS共享责任模型：
  - AWS负责"云的安全"（基础设施、硬件）
  - 客户负责"云中的安全"（访问控制、数据加密、配置）
- 介绍AWS Artifact：下载合规报告和第三方审计证据的服务

### Control Catalog功能演示
00:26:00 - 00:35:00
- Arnold演示Control Catalog的核心功能：
 
 无需预先设置：
  - 新的"仅控制"功能允许直接访问控制目录
  - 无需部署完整的Control Tower Landing Zone
  - 仅需启用AWS Organizations的可信访问
 
 目录特性：
  - 超过1000个AWS托管控制措施
  - 通过控制台、API和CLI访问
  - 每个控制措施具有全局唯一标识符（ARN）
  - 丰富的元数据：域、通用控制、目标
  - 预映射到17个合规框架
 
 实际演示场景：
  - 需求：符合PCI DSS和CIS标准，确保DynamoDB加密
  - 演示多维度筛选：
    - 按框架筛选（PCI DSS 4.0 + CIS 8.0）
    - 按目标筛选（静态数据加密）
    - 按服务筛选（DynamoDB）
  - 结果：找到5个相关控制措施
  - 选择检测性和主动性控制措施配对使用
  - 一键部署到整个工作负载OU

### 控制措施关系和最佳实践
00:35:00 - 00:38:00
- 展示控制措施详情页面包含的信息：
  - 描述、域、目标、框架映射
  - 服务、资源类型、区域可用性
  - 相关控制措施（如检测性和主动性配对）
- 强调"纵深防御"策略：同时使用预防性、主动性和检测性控制措施

### 自定义控制措施
00:38:00 - 00:40:00

- 介绍AWS Config支持的两种检测性控制措施：
  - AWS托管规则
  - 自定义规则（用于组织特定基线）
