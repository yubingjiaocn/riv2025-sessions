# AWS re:Invent 2025 - Security Hub 演进：统一安全运营 (SEC 228)

## 会议概述

本次会议重点介绍了AWS Security Hub的重大演进，该服务已从云安全态势管理(CSPM)工具发展为统一的安全运营平台。演讲者Himanshu(AWS全球安全专家团队负责人)和Scott Ward(首席解决方案架构师)详细阐述了Security Hub如何通过整合多个AWS原生安全服务(如GuardDuty、Inspector、Macie)以及合作伙伴工具的安全信号，为客户提供统一的安全视图。新版本于re:Invent 2025主题演讲中宣布正式发布(GA)，核心改进包括：实时关联分析、暴露发现(Exposures)、攻击路径可视化、以及采用开放网络安全模式框架(OCSF)标准化所有安全发现。

该服务解决了安全团队面临的关键挑战：碎片化的可见性、缺乏上下文的海量告警、以及响应自动化的复杂性。通过AI和机器学习驱动的关联引擎，Security Hub现在能够将来自不同来源的孤立安全信号整合为具有丰富上下文的暴露发现，帮助团队快速识别最高优先级的风险。此外，新的定价模型基于资源而非事件量，提供了更可预测和简化的成本结构。服务保持原有名称，现有CSPM用户可通过30天免费试用轻松升级到新体验。

- **26:15** - Reachability：互联网开放网络路径
- **26:30** - Sensitive Data：敏感数据指标
- **26:45** - Vulnerability：可利用的软件漏洞
- **27:00** - 严重性等级：Critical、High、Medium、Low
- **27:30** - 严重性评估方法论：发现难易度、利用难易度
- **28:00** - 考虑因素：利用可能性、公众认知度、潜在影响
- **28:30** - 使用EPSS评分和AWS内部威胁情报
