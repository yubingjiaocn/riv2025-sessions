# AWS re:Invent 2025 会议总结：通过 AWS Marketplace 实现产品驱动的国际化增长

## 会议概览

本次 AWS re:Invent 2025 分会由 AWS Marketplace 增长营销负责人 Alex Jones 和国际开发负责人 Patrick Lwick 主讲，主题聚焦如何通过 AWS Marketplace 利用产品驱动增长（PLG）策略实现国际化扩张。

会议深入探讨了产品驱动增长的核心理念：让产品本身成为客户获取、转化和收入增长的主要驱动力。这种策略特别适合希望在无需先期投资建立全球销售团队的情况下，快速扩展到国际市场的软件公司。演讲者强调了三个关键要素：专注于最终用户体验、提供自助服务能力、以及将产品作为销售渠道。通过 AWS Marketplace 这一平台，软件供应商可以从第一天起就接触到 190 个国家的潜在客户。

会议还详细介绍了 AWS Marketplace 如何在运营层面支持国际化交易，包括自动化的合规处理、本地化的发票体验、多货币支持以及简化的税务管理。演讲者通过 Databricks、Pinecone、Veronis 和 Snowflake 等实际案例，展示了成功实施 PLG 策略的企业如何实现流量、订阅和收入的显著增长。

### 开场介绍（0:00-2:30）

- **0:30** - 介绍 Patrick Lwick，国际开发工作负责人

- **1:30** - 会议结构说明：Alex 讲解 PLG 概念和激活方式，Patrick 讲解国际销售运营

### 产品驱动增长（PLG）核心概念（2:30-8:00）
- **2:30** - PLG 定义：一种以产品本身驱动客户获取、转化和收入的市场策略
- **3:00** - PLG 三大关键要素：
  - 专注于最终用户（开发者和实践者）
  - 自助服务体验（用户无需与人交流即可发现、试用和使用产品）
  - 产品作为销售渠道（特别是与云市场结合时）
- **4:00** - 市场趋势分析：
  - 自助服务需求增长：80% 的云市场用户计划在未来两年增加采购
  - AWS Marketplace 超过 80% 的订阅是自助服务
- **5:00** - 开发者影响力上升：IDC 2021 年调查显示 70-79% 的开发者对组织采购决策有完全或重大影响
- **6:00** - 云市场支出加速：预计到 2030 年将达到 1630 亿美元
- **7:00** - AWS Marketplace 介绍：覆盖 37 个区域，支持国际化扩张，提供 70 多个软件类别

### 传统销售 vs 产品驱动增长（8:00-12:00）
- **8:00** - 传统销售主导增长的局限：
  - 需要在目标市场建立法律实体
  - 需要本地销售团队
  - 高客户获取成本
  - 较长的销售周期
- **9:00** - PLG 策略优势：
  - 产品在 AWS Marketplace 上架后即可覆盖 190 个国家
  - 产品成为全球销售团队
  - 客户位置无关紧要（新加坡或西雅图体验一致）
  - 较低的客户获取成本
- **10:00** - 经常性收入模型解析（基于 Winning by Design 的 Dave Boyce 模型）：
  - 发现和获取层：通过 AWS 控制台和 Marketplace 实现产品可发现性
  - 中间层：免费试用/免费层 → 体验价值 → 按使用付费消费
  - 扩展层：销售团队专注于帮助用户扩大使用并升级到私有优惠（承诺合同）
- **11:00** - PLG 数据驱动决策：
  - 流量来源分析
  - 访客转化为订阅的比率
  - 免费用户转化为付费客户的信号
  - 支持工单和用户扩展需求分析

### AWS Marketplace Seller Prime 计划（12:00-15:00）
- **12:00** - Seller Prime 计划介绍：帮助 ISV 合作伙伴入门的项目
- **12:30** - 计划提供的支持：
  - 最佳实践和路线图
  - 转化优化技巧
  - 产品列表页面优化策略
- **13:00** - 营销支持：
  - 激活自有营销活动的支持
  - 提供启动资金
  - 优先纳入 AWS 主导的营销活动
- **13:30** - PLG 市场报告：帮助了解访客来源和优化转化策略
- **14:00** - 重要提醒：不要只是上架产品就期待用户自动发现，需要将 Marketplace 整合到整体市场策略中

### 成功案例分析（15:00-17:00）
- **15:00** - Databricks 案例：
  - Marketplace 产品列表页面流量增长 9 倍
  - 订阅量提升 5 倍
  - 驱动自助服务消费收入
  - 销售团队将自助订阅用户升级为私有优惠
- **16:00** - Pinecone（向量数据库初创公司）案例：
  - 较短时间内实现类似增长模式
  - 流量和订阅增加
  - 收入增长 3 倍
- **16:30** - 成功模式总结：高质量流量 → 自助订阅 → 销售团队升级为承诺合同

### 国际化运营支持（17:00-25:00）
- **17:00** - Patrick 接手，介绍 AWS Marketplace 如何在运营层面支持全球交易
- **17:30** - "流畅性"概念：交易顺利进行，私有优惠可在几分钟内完成，客户几次点击即可订阅
- **18:00** - 国际化常见问题：
  - 是否需要翻译产品列表？
  - 现有实体能否向全球客户销售？
  - 如何处理本地货币？
  - 如何设置收入运营流程？
  - 如何满足本地法规和税务要求？
- **19:00** - AWS Marketplace 愿景：成为首选的全球市场路径，在每个运营国家提供本地化体验

### 本地化功能详解（20:00-25:00）
- **20:00** - 自动翻译功能：
  - 无需手动翻译 100 次
  - 支持日语、法语、西班牙语等多种语言
  - 标准化许可条款也提供翻译版本
- **21:00** - 实体注册灵活性：
  - 支持来自 40 多个司法管辖区的企业注册
  - 最新增加印度支持
  - 可设置多个本地实体（如德国实体支持德国客户交易）
- **22:00** - 自动化合规：
  - 自动评估、收取和汇缴本地税款
  - 全球运营商模型：美国、印度、澳大利亚、日本和韩国（最新）的本地 AWS 实体
  - 客户从同一 AWS 实体收到软件和 AWS 服务的发票
  - 支持本地支付系统（如欧洲的 SEPA 支付）
- **23:00** - 货币和银行账户设置：
  - 注册时设置支持的货币
  - 可添加多个本地银行账户并关联货币
  - 默认公开产品列表以美元计价
  - 私有交易可使用欧元、日元、澳元等货币
- **24:00** - 交易流程：
  - 供应商以选定货币报价
  - 客户按其 AWS 账户设置的格式收到发票
  - 客户通过 AWS 发票支付
  - AWS 以报价货币向供应商付款至指定银行账户

### 最佳实践与案例研究（25:00-29:00）
- **25:00** - 销售主导增长最佳实践：
  - 将本地实体也设置为 Marketplace 卖家
  - 启用本地货币交易
  - 在交易发生地区设置银行账户
- **25:30** - 产品驱动增长最佳实践：
  - 优化产品列表（知道会自动本地化）
  - 使用 Seller Prime 计划资助市场推广活动
- **26:00** - Veronis 案例：
  - 将本地实体设置为 Marketplace 卖家
  - 启用本地货币交易
  - 结果：交易和收入运营团队像处理其他交易一样处理 Marketplace 交易
  - 消除摩擦，加速交易
- **27:00** - Snowflake 案例：
  - 2024 年通过 Marketplace 交易超过 10 亿美元
  - 利用 AWS Marketplace 的自动化合规
  - 将区域收入运营流程与 Marketplace 镜像
  - 实现更高的本地交易效率，是全球增长的关键贡献因素
