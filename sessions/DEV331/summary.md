# AWS re:Invent 2025 - 使用 Kiro、MCP 和 AgentCore 构建 AI 智能体

## 会议概述

本次技术分享会由 AWS 高级开发者倡导者 Erik Hanchett 和专注于智能体 AI 的开发者倡导者 Du'An Lightfoot 主讲，主题为"使用 Kiro、MCP 和 AgentCore 构建 AI 智能体"。这是一场 300 级别的深度技术讲座，旨在帮助开发者了解如何使用 AWS 的最新工具链来构建和部署 AI 智能体。

会议涵盖了 AI 智能体的基本概念、AgentCore 平台的核心功能、Kiro 智能体 IDE 的使用方法，以及模型上下文协议（MCP）的实际应用。演讲者通过一个完整的食物追踪智能体演示，展示了如何将这些工具整合在一起，创建一个具有记忆功能、多层架构的实用智能体系统。整个演示包括前端界面、多个 MCP 服务器、以及完整的可观测性监控。

## 详细时间线与关键要点

### 0:00-5:00 - 开场介绍与议程概览
- Erik Hanchett 和 Du'An Lightfoot 自我介绍
- 现场调研：AgentCore、Kiro 和 MCP 服务器的使用情况
- 会议议程：AI 智能体定义、AgentCore 深入讲解、Kiro 介绍、MCP 概述、实际演示

### 5:00-15:00 - AI 智能体基础概念
- **智能体定义**：利用 AI 进行推理、规划和执行操作的自主系统
- **核心能力**：推理能力、软件系统架构、API 和工具访问、记忆管理
- **技术演进**：从 2022 年 ChatGPT 发布到 2024 年智能体框架的发展历程
- **智能体架构组成**：中央 LLM、API 工具访问、短期和长期记忆系统

### 15:00-25:00 - AgentCore 平台深入解析
- **核心挑战**：安全部署、上下文记忆管理、身份认证、工具管理、系统可观测性
- **AgentCore 解决方案**：
  - 运行时环境：支持任何框架和模型，最长 8 小时工作负载
  - 记忆系统：短期和长期记忆的自动管理
  - 身份管理：入站和出站连接的身份验证
  - 网关服务：将现有 API 转换为 MCP 服务器
  - 浏览器和代码解释器：安全的隔离执行环境

### 25:00-35:00 - AgentCore 核心功能详解
- **运行时部署**：支持 Docker 容器和 ZIP 文件两种部署方式
- **记忆策略**：语义搜索、用户偏好、摘要、自定义策略
- **网关功能**：
  - 支持 OpenAPI 和 Smithy 规范
  - Lambda 函数集成
  - 第三方 API 认证
  - 语义搜索优化工具选择
- **可观测性**：CloudWatch 集成，支持第三方监控工具导出

### 35:00-45:00 - Kiro 智能体 IDE 介绍
- **Kiro 特性**：基于 VS Code 的智能体 IDE，支持规范驱动开发
- **开发模式**：
  - Vibe 模式：直接解决问题的快速开发
  - Spec 模式：基于需求文档的结构化开发
- **MCP 集成**：支持本地和远程 MCP 服务器
- **Kiro Powers**：新发布的功能，支持 MCP 服务器、引导文件和上下文增强

### 45:00-55:00 - 实际演示：食物追踪智能体
- **架构展示**：多层智能体架构，包含前端、食物追踪智能体、配方智能体和两个 MCP 服务器
- **MCP 服务器开发**：
  - 使用 FastMCP 库创建 Python MCP 服务器
  - MCP 检查器工具进行本地测试
  - Lambda 网关集成现有函数
- **记忆系统演示**：短期和长期记忆的实际工作效果
- **智能体交互**：展示智能体如何处理复杂的膳食规划请求

### 55:00-60:30 - 可观测性与问答环节
- **CloudWatch 集成**：日志流查看、错误排查
- **生成式 AI 可观测性**：
  - 会话和跟踪管理
  - 详细的 span 分析
  - 令牌使用量和延迟监控
  - 输入输出数据的完整可见性
- **现场问答**：涵盖记忆配置、令牌优化、工具管理等技术细节
