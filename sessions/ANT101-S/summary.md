# AWS re:Invent 2025 会议总结：EverCommerce 数据现代化之旅

## 会议概述

本次技术分享会议由 5TR 企业销售副总裁 John Sedlak 和 EverCommerce 数据与分析副总裁 Kalpit Patel 共同主讲，重点介绍了 EverCommerce 的数据现代化转型历程。会议深入探讨了如何通过 5TR 的托管数据湖服务和 AWS S3 基础设施，将分散在 50+ 个独立运营组织中的数据整合到统一的"数据工厂"平台中。

EverCommerce 作为一家服务商务平台公司，在 40+ 个国家为 70 万客户提供服务，面临着典型的数据孤岛挑战。通过与 5TR 的战略合作，公司成功构建了基于 Apache Iceberg 的开放数据平台，实现了从"数据饥渴"向"数据驱动企业"的转型，为未来的 AI/ML 应用奠定了坚实基础。

## 详细时间线与关键要点

### 0:00-3:00 - 开场介绍与公司背景
- John Sedlak 介绍 5TR 公司概况：1500 名员工，专注自动化、安全、可扩展的数据移动
- 5TR 拥有 700 个预构建连接器，服务 7000 个客户
- 支持从 SaaS 应用到大型运营数据存储的广泛数据源组合
- 全球业务覆盖北美、欧洲和亚太地区

### 3:00-6:00 - 5TR 技术架构展示
- 展示 5TR 数据管道的工作原理：从源系统到 AWS S3 存储
- 强调自动化管道的优势：减少管理开销、消除架构漂移
- 支持实时分析、业务运营、云迁移和 AI/ML 应用场景
- Kalpit Patel 介绍 EverCommerce：服务商务平台，40+ 国家，70 万客户

### 6:00-9:00 - 数据现代化挑战分析
- Kalpit 分享加入公司后的发现：通过 60 次访谈了解到公司是"数据饥渴"型企业
- 面临的核心挑战：数据孤岛、数据重复、缺乏统一治理
- 提出"数据工厂"概念：三个阶段 - 建设工厂、部署机械设备、准备投产
- 强调从集中化向赋能中心转变的理念

### 9:00-12:00 - 技术实施与架构决策
- 选择基于 Apache Iceberg 的开放数据平台架构
- 利用 AWS S3 作为基础存储层，确保可靠性和可扩展性
- 通过 5TR 托管数据湖服务实现数据整合加速
- 实现"构建一次，多次消费"的数据资产管理模式

### 12:00-15:00 - 业务价值与未来规划
- 实现跨品牌可视性提升，支持交叉销售和向上销售
- 构建企业级报告能力，标准化数据定义
- 规划 AI/ML 就绪的基础设施，支持客户流失预测和保留分析
- 集成 DBT 生态系统，实现"左移"数据质量管理策略

### 15:00-17:48 - 生态系统整合与总结
- 讨论 5TR、DBT、Census 整合后的 360 度数据旅程
- 介绍 DNA 飞轮概念：关注数据生产者、消费者和应用场景
- 强调以业务转型为导向，而非技术优先的方法论
