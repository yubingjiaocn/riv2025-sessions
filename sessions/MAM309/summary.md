# AWS re:Invent 2025 Microsoft工作负载迁移会议总结

## 会议概述

本次会议由AWS高级解决方案架构师Sassan Hajrasooliha主讲，联合Mangesh Budkule和Adi Simon共同展示了如何使用数据驱动的方法将Microsoft工作负载迁移到AWS云平台。会议重点介绍了AWS Transform等新一代AI驱动的迁移工具，涵盖了从发现评估到实际迁移的完整流程。

演讲者强调，尽管70%的工作负载仍在本地运行，但随着生成式AI的兴起，企业迁移到云端的需求日益迫切。会议通过实际演示展示了如何利用AWS的各种服务和工具，包括传统的MGN（应用程序迁移服务）以及最新的AWS Transform智能代理，来简化复杂的迁移过程。整个会议采用了理论讲解与实际演示相结合的方式，为与会者提供了全面的Microsoft工作负载迁移指导。

## 详细时间线与关键要点

### 00:00-05:00 开场介绍与迁移类比
- 演讲者通过旅行规划类比引入迁移概念，强调需要明确目标、时间、预算等关键问题
- 介绍演讲团队：Sassan Hajrasooliha（基础设施现代化专家）、Mangesh Budkule和Adi Simon
- 概述会议议程：发现、工作负载分类、演示展示

### 05:00-15:00 迁移背景与发展历程
- 分析当前70%工作负载仍在本地运行的现状
- 回顾IT基础设施发展历程：从服务器机房到商业数据中心，再到云计算时代
- 重点介绍生成式AI发展阶段：2023年手动提示、2024年RAG增强、2025年智能代理时代
- 传统发现方法：优化和许可评估（OLA）流程介绍

### 15:00-25:00 AWS Transform发现演示
- 演示vCenter环境扫描过程，包括ESXi主机和虚拟机清单
- 展示AWS发现工具的使用，包括依赖关系映射和SQL版本识别
- AWS Transform工作空间创建和配置流程
- 实际演示清单数据上传和分析过程，生成PDF报告和Excel分析表

### 25:00-35:00 服务器和虚拟机迁移
- 介绍7R迁移策略：重新托管、重新平台化、重构
- Amazon Elastic VMware Service (EVS)简介
- 应用程序迁移服务（MGN）详细说明，包括代理安装和块级复制
- Mangesh演示AWS Transform for VMware的AI驱动迁移流程

### 35:00-45:00 Active Directory迁移方案
- 三种AD部署选项对比：自管理EC2域控制器、AWS托管AD、AD连接器
- 新服务发布：AWS托管AD混合版本，支持现有域扩展
- 详细演示混合AD部署过程，包括健康检查、评估和域控制器部署
- 展示共同管理模式下的OU结构和域控制器配置

### 45:00-54:00 数据库和应用程序现代化
- SQL Server迁移选项：EC2重新托管、RDS托管服务、RDS Custom
- 四种迁移方法：MGN、原生备份恢复、集群环境同步、DMS连续复制
- AWS Transform for SQL Server现代化演示，包括模式转换和代码修改
- .NET应用程序容器化和现代化选项介绍
- 文件服务器迁移方案：自管理EC2、FSX服务、数据同步工具

### 54:00-54:30 总结与资源
- 强调AI驱动工具如何简化传统迁移挑战
- 介绍可用的迁移计划和激励措施
- 提供详细的学习资源清单，包括技术文档、培训材料和相关会议录像
