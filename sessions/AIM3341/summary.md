# AWS re:Invent 2025 Bedrock Data Automation 会议总结

## 会议概述

本次会议由AWS的两位产品经理Anushrithia和Nene，以及初创公司Air的联合创始人兼CTO Tyler Strand共同主讲，重点介绍了Amazon Bedrock Data Automation (BDA)服务及其在多模态内容处理方面的应用。

会议深入探讨了企业在处理非结构化多模态内容时面临的挑战，包括文档、图像、视频和音频等多种格式。据统计，80%的企业内容属于非结构化数据，但只有20%的组织能够有效利用这些数据。传统方法需要为不同模态构建独立的处理管道，维护复杂的编排系统，并不断跟进最新的AI模型。Amazon Bedrock Data Automation通过提供统一的API接口，彻底简化了这一流程。用户只需通过一个API调用，就能处理所有类型的多模态内容，并通过自然语言定义所需的输出格式和字段。

Tyler Strand分享了Air公司的实际应用案例。Air是一个创意运营平台，管理着8PB的数据和数亿个创意资产，服务于包括Sweet Green、NBCU、JP Morgan等在内的200多家企业的10万名创意专业人士。通过采用BDA，Air成功将原本复杂的多管道架构简化为统一的处理流程，大幅降低了维护成本，提高了处理效率，同时确保了数据安全性和成本效益。

### 多模态内容挑战 (05:30 - 10:15)

05:30 - 07:20 - 核心问题阐述
- 80%的内容是非结构化数据
- 只有20%的组织能够大规模利用这些数据
- 需要将非结构化内容转换为可用格式以支持下游应用

07:20 - 10:15 - 具体挑战分析
- 多模态内容类型极其多样（保险理赔涉及文档、表单、视频、照片等）
- 单个组织可能需要管理300多种不同类型的文档
- 需要在大规模场景下保证准确性（处理数百万份文档和视频）
- 缺乏标准化工具，需要学习多个服务和接口
- 数据转换和整合工作繁琐耗时

### Amazon Bedrock Data Automation介绍 (10:15 - 20:45)

10:15 - 12:30 - BDA核心概念
- 单一API处理图像、文档、视频和音频
- 用户只需指定想要的输出，BDA自动完成处理
- 无需手动编排或选择模型

12:30 - 15:00 - BDA的三大优势
- 实现简化：所有模态通过同一管道处理
- 输出定制：支持自定义schema和数据转换
- 可审计性：提供置信度分数和内容溯源

15:00 - 17:30 - 五大应用场景
- 智能文档处理（保险理赔、贷款处理）
- 智能搜索分析（呼叫中心分析、情感分析）
- 媒体分析和内容发现（广告素材检索）
- 代理工作流（作为agent的工具调用）
- 文档搜索

17:30 - 20:45 - BDA工作原理
- 用户提供资产和项目配置
- 可选择标准输出或自定义blueprint
- BDA自动识别资产类型并应用相应处理
- 返回结构化输出和线性化内容

### 传统方法对比 (35:20 - 38:45)

35:20 - 37:00 - 传统方法的复杂性
- 需要构建特定任务的机器学习模型
- 需要拼接多个模型
- 需要分别处理音频、文本和视觉信号
- 需要手动添加摘要功能
- 需要检测多种实体（logo、对象、人物、场景情绪）

37:00 - 38:45 - 维护成本
- 开发时间可能需要数月
- 需要持续维护和更新
- 需要跟进每个新模型和新功能
- 需要不断平衡性价比

38:45 - 40:30 - 智能文档处理

45:00 - 47:30 - 互动游戏：服务选择

### Air公司案例分享 (47:30 - 58:45)

47:30 - 49:45 - Air公司介绍

- 服务10万名创意专业人士

49:45 - 51:30 - 产品核心功能
- 资产管理：存储、组织、搜索
- 创意工作：反馈、编辑、外部共享

51:30 - 53:15 - The Infatuation客户案例
- 餐厅评论平台，业务遍布全球

55:00 - 58:45 - 采用BDA前的挑战
- 每种文件类型需要特定处理方式
- 不同服务返回的元数据碎片化且不一致
- 需要维护复杂的编排系统
- 每种内容类型需要独立的处理管道和第三方服务

### BDA解决方案实施 (58:45 - 结束)

58:45 - 60:30 - 选择BDA的原因
- 需要更简单、易扩展、安全且成本效益高的解决方案
- 8PB媒体数据需要大规模AI增强处理

60:30 - 结束 - BDA带来的改进
- 简化架构：统一API替代多个碎片化管道
- 一次性调用：单次请求获取所有所需信息
- 异步处理：等待通知后获取所有元数据
- 抽象模型选择：无需追踪最新最佳模型，专注于构建用户体验
- AWS原生集成：确保安全性和成本效益
