# AWS re:Invent 2025 - NET211: The State of Edge 会议总结

## 会议概述

本次会议由AWS Edge服务专家Etienne（来自悉尼）、AWS网络服务总监Nishit Sawhney以及Atlassian云网络工程师Ambrose Phey共同主讲。会议重点探讨了AWS Edge服务的现状和发展趋势，特别是Amazon CloudFront、Shield和WAF在应对新兴AI驱动的"代理网络时代"中的演进。

会议深入分析了互联网从静态网页到AI代理网络的演变历程，以及这种演变对性能、安全性和开发复杂性带来的挑战。演讲者们分享了AWS如何通过技术创新和服务优化来应对这些挑战，并通过Atlassian的实际案例展示了大规模Edge服务部署的最佳实践。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与会议概述
- 三位演讲者自我介绍：Etienne（AWS Edge专家）、Nishit（AWS网络服务总监）、Ambrose（Atlassian工程师）
- 会议定位为200级别技术分享，聚焦CloudFront、WAF和Shield服务
- 强调AWS Edge服务的核心使命：让应用更快速、安全和具有弹性

### 5:00-15:00 Edge服务定义与互联网演进历程
- Edge服务定义：将网络、存储、计算能力部署到更接近用户的位置以降低延迟
- 互联网发展的五个阶段演进：
  - 静态网页时代：简单的GET请求和IP阻断安全策略
  - 交易电商时代：API普及，PUT/POST请求增加，DDoS攻击出现
  - 社交移动时代：用户规模爆发，机器人活动增加，延迟要求提升
  - 短视频直播时代：延迟敏感度达到情感层面，毫秒级差异影响用户参与度
  - 物联网时代：设备数量激增，被攻击设备成为DDoS攻击源

### 15:00-25:00 AI代理网络时代的挑战
- 当前每日数十亿请求来自AI机器人（ChatGPT、Claude、Perplexity等）
- 传统robots.txt协议已不适用于现代AI训练机器人和代理AI机器人
- 三大定义趋势：
  - 流量模式变化：突发流量、双工连接、LLM令牌传输
  - 安全需求升级：既要保护AI应用，也要防范AI驱动的攻击
  - 开发周期加速：基础设施不应成为快速开发的阻碍

### 25:00-35:00 性能与弹性增强
- AWS全球网络规模：750+个PoP、1140个嵌入式PoP，覆盖200+城市50+国家
- 网络处理能力：每日万亿级请求，峰值数百TB/秒
- 安全数据：去年处理620万次DDoS攻击（同比增长150%），最大攻击规模25TB/秒
- 新功能发布：Global Anycast BYOIP，与VPC IPAM集成

### 35:00-45:00 性能优化技术详解
- QUIC/HTTP3协议：25%的平台请求已采用，默认启用
- HTTPS DNS记录支持：减少握手往返时间，降低DNS查询成本
- TLS 1.3到源站：握手时间改善36%
- TCP Fast Open：TCP连接时间改善50%
- 所有性能增强功能均默认启用且免费

### 45:00-55:00 Atlassian实际案例分享
- Atlassian背景：服务数百万全球客户，每日处理数十亿请求
- 迁移目标：将Jira和Confluence迁移到CloudFront以增强安全性和性能
- 关键挑战解决：
  - 使用Anycast静态IP满足企业防火墙白名单需求
  - 部署分层防御策略：Shield Advanced + WAF托管规则
  - 实现数据驻留优化：使用CloudFront Functions进行区域路由
- 显著成果：
  - 大幅降低数据传输费用
  - 消除跨区域费用
  - 提升100-150毫秒的性能
  - 成功阻止数十万恶意请求和大规模DDoS攻击
