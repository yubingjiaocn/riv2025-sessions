# AWS re:Invent 2025 会议总结：构建安全的边界保护

## 会议概述

本次会议由AWS网络团队的产品经理Shoan Das、CloudFront团队负责人Mega Kande以及首席解决方案架构师Sher Tahir共同主讲，主题聚焦于如何在AWS上构建全面的安全边界保护体系。

会议以一个虚拟客户"Secure Shop"为例，展示了如何保护公共应用（如零售网站secureshop.com）和私有应用（如财务系统financ.com）。演讲者强调，构建应用只是工作的第一部分，保护这些应用同样重要。完整的边界保护需要覆盖四个流量方向：互联网入站流量、应用访问互联网的出站流量、应用之间的东西向流量，以及员工访问应用的流量。

会议深入介绍了AWS边缘安全服务（CloudFront、AWS WAF、Shield）、网络防火墙以及零信任访问解决方案。特别值得一提的是，演讲者展示了如何使用AWS的AI服务Kiro CLI通过自然语言提示快速部署安全配置，大大简化了安全实施的复杂度。整个会议强调了规模化、全面覆盖和易用性三个核心原则。

- **01:20** - 介绍会议结构：边缘保护、网络流量检测、用户访问控制和演示

### 边界保护需求分析 (02:30-06:00)
- **02:30** - 介绍两类应用：公共应用（零售网站、API网关）和私有应用（HR系统、财务仪表板）
- **03:45** - 阐述四个流量方向的保护需求：互联网入站、互联网出站、应用间通信、员工访问
- **05:00** - 介绍虚拟客户Secure Shop的架构：VPC托管应用、通过TGW连接、使用CloudFront加速

### CloudFront边缘安全 (06:00-20:00)
- **06:00** - Mega Kande接手，介绍CloudFront在安全中的角色
- **07:15** - 分析互联网流量构成：超过50%是机器人流量，DDoS攻击量两年增长400%
- **09:00** - 介绍AWS边缘基础设施：750个接入点，分布在50多个国家的100多个城市
- **10:30** - 详解流量检测层次：Shield在网络层阻止恶意IP、检测SYN洪水攻击、数据包校验
- **12:00** - TLS安全：支持高级加密套件、后量子计算密钥、双向TLS认证
- **14:00** - AWS WAF功能：速率限制、机器人防护、第7层DDoS管理
- **15:30** - 介绍CloudFront与源站的安全连接：支持私有VPC源站，通过VPC对等连接保护
- **17:00** - AWS WAF托管规则：OWASP前10大风险防护、机器人控制规则、购物车填充防护
- **18:30** - 新推出的第7层DDoS防护规则：响应速度更快（几秒内）、准确率更高、可配置性更强

### Kiro CLI安全部署演示 (20:00-28:00)
- **20:00** - Sher Tahir开始演示，介绍Kiro CLI（AWS的AI代理工具）
- **21:00** - 演示架构：S3托管静态网站，通过CloudFront分发，配置WAF规则
- **22:30** - 展示通过自然语言提示部署安全配置的过程
- **23:45** - 部署的安全功能：SQL注入防护、跨站脚本防护、速率限制（每5分钟200请求）、地理位置封锁
- **25:00** - 展示Secure Shop零售网站和流量监控指标：1400个请求中51个被阻止
- **26:30** - 展示CloudFront控制台中的托管规则和自定义规则配置
- **27:30** - 强调配置的安全特性：仅支持HTTPS、WAF策略、HTTP到HTTPS重定向、源访问控制

### 网络防火墙 (28:00-36:00)
- **28:00** - Shoan Das回归，介绍网络防火墙用于东西向和南北向流量保护
- **29:00** - 网络防火墙三大功能：基于Suricata的规则引擎、有状态规则、协议检查（IDS/IPS）
- **30:30** - 网络防火墙作为"线路中的凸起"：与Transit Gateway、IGW、Direct Connect、VPN原生集成
- **32:00** - 通过AWS Firewall Manager和Organizations实现集中管理
- **33:00** - 日志功能：支持S3、Kinesis Firehose、CloudWatch Logs导出
- **34:00** - 托管规则和威胁情报：AWS蜜罐网络（MADS）持续检测威胁
- **35:00** - 蜜罐网络规模：全球部署数万个传感器，每天分析超过1亿个签名模式

### 用户访问控制 (36:00-40:00)
- **36:00** - 介绍两类用户访问场景：通过浏览器访问和SSH/Telnet访问
- **37:00** - 三种网络访问模式：通过本地VPN连接、AWS Client VPN、基于浏览器的零信任访问
- **38:00** - Client VPN三种认证方式：证书认证、Active Directory集成、身份提供商（Okta、Ping）集成
- **39:00** - 介绍AWS Verified Access作为零信任解决方案：基于身份、设备状态的细粒度访问控制
