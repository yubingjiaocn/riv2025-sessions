# AWS re:Invent 2025 分组会议总结

## 会议概述

本次会议由Salesforce的Shivani Huja和Capita的Samir共同主讲,重点探讨了企业如何通过Salesforce Agent Force和AWS Bedrock的集成来实现智能代理(Agentic AI)的规模化部署。Shivani在过去一年中与520位企业CIO进行了深度交流,分享了她观察到的AI实施策略演变趋势。会议展示了Capita作为英国最大的公共部门业务流程外包公司,如何成功转型为智能代理企业的实际案例。

Samir详细介绍了Capita在接触英国70%人口的业务中,如何利用Salesforce和AWS的协同优势,在数月内完成了原本需要数年和数千万英镑投资的项目。会议强调了"更好的协同"(Better Together)战略的价值,即通过原生智能代理集成(Agentic Native Integration)来整合现有技术栈,而非推倒重建。核心信息是:AI不是为了取代人力,而是赋能员工,提升客户和员工满意度。

## 详细时间线

00:00:00 - 开场介绍
- Shivani Huja自我介绍,分享在Salesforce工作一年的背景
- 提到之前在加拿大TD银行负责数字银行和全球技术
- 幽默地讲述如何因为在Mark Benioff面前直言不讳而被Salesforce招募

00:03:30 - 职责说明
- 介绍当前角色:帮助Salesforce制定面向CIO、CEO、CFO等高管的市场策略
- 过去一年与520位企业CIO进行了深度交流

00:04:45 - AI实施趋势演变
- 第一阶段:CIO坚持自建(DIY),声称要构建自己的AI解决方案
- 许多CIO并不真正理解预测式AI、生成式AI和智能代理AI之间的区别

00:06:15 - 策略转变
- 第二阶段:从"构建vs购买"转向考虑SaaS平台
- 一些企业选择Salesforce利用现有客户数据
- 另一些企业选择AWS等云服务提供商,追求无限扩展性和安全性

00:07:30 - 最新趋势:原生智能代理集成
- 最近一个半月观察到的新趋势
- CIO开始关注如何整合现有技术栈(例如90个营销技术工具)
- 用Wolverine的金属骨骼比喻智能代理架构的复杂性
- 企业寻求已有合作伙伴提供的原生智能代理集成方案

00:09:00 - 引入Samir
- 介绍Capita的Samir,强调其一年前还在AWS工作
- Capita成为智能代理企业的案例

00:10:15 - Samir的背景和选择
- 一年前的今天还在AWS工作,现在以客户身份回到re:Invent
- 引用Jeff Bezos的话:在不确定时期,构建在不会改变的事物之上
- 选择AWS和Salesforce作为长期合作伙伴

00:11:30 - Capita的四大用例
- 使用MuleSoft整合65个不同应用系统
- 通过Amazon Connect和Agent Force构建语音智能代理
- 使用Mission Force开发国防应用
- 构建"盒装地方政府"解决方案
- 原本需要数年和数千万英镑的项目,现在数月内完成

00:13:00 - Capita公司介绍
- 英国最大的公共部门业务流程外包公司
- 每周接触英国70%的人口
- 业务范围:公务员养老金、伦敦交通、英国军队招募等
- 流程密集型业务,适合智能代理化

00:14:45 - 人力与AI协同策略
- 引用Nike的口号"Just Do It"
- 消除市场对AI取代工作的恐惧
- 客服代表实际上比客户更沮丧,因为缺乏解决问题的工具
- 提供智能代理工具后,10名试点客服的成功引发了1000多名客服主动要求使用

00:16:30 - 员工满意度提升
- 员工不再需要手写记录、自我评分或接受主管监督
- 员工更满意、更快乐,转化为客户NPS(净推荐值)提升

00:17:15 - 三个衡量维度
- KPI改善:消除了数千万英镑的合同罚款,100%利润率直接落到底线
- 提高中标率:在每个投标中嵌入AI,更便宜、更快、更灵活
- 民主化智能代理开发:为每个员工分配虚拟货币衡量智能代理构建和行动
- 员工NPS作为采用度和满意度的良好指标

00:19:30 - 衡量标准的重要发现
- Shivani强调:Samir的所有衡量标准都不是关于减少FTE(全职员工)
- 这是思考用例的健康方式,关注增长和扩展机会

00:20:15 - 联络中心案例
- 原本9-5工作的联络中心
- 实施Agent Force后,智能代理处理常规任务
- 联络中心立即变成24/7运营

00:21:30 - 时间分片多路复用
- 历史上客服代表需要专门负责一个账户
- 现在通过按需知识库和实时协助(生成式AI和转录)
- 高能力的客服可以跨多个账户工作
- 下一步:提供智能代理实现首次呼叫解决率的大幅提升

00:23:00 - 更好的协同故事
- Shivani请Samir从客户角度阐述Salesforce和AWS合作的优势
- Samir自称极客,但不能再写汇编代码

00:24:00 - 低代码民主化
- Salesforce实现了让组织中的每个人用自然语言描述需求
- 无需IT团队介入即可为客户测试
- IT团队从构建者转变为顾问和指导者(Yoda)
- 每个员工都是学徒(Padawan),有望成为绝地武士

00:25:30 - 财务和技术优势
- 在AWS Marketplace购买Salesforce,可以抵扣AWS承诺消费
- 利用Salesforce生态系统中数万个已智能代理化的用例
- 无需重新创建,直接使用Agent Force Marketplace
- AWS提供地球上最好的安全平台进行安全治理

00:27:00 - 三大关键要点
1. 构建在长期稳定的平台上:选择AWS和Agent Force这样有全球覆盖的平台,可以适应任何用例
2. 自己构建:没有人比你更了解自己的流程和工作,为自己构建然后与员工分享
3. 享受乐趣:Samir分享了一个SI(系统集成商)报价200万英镑的项目,他在半小时内用Salesforce原型完成并展示给SI

00:29:00 - 会议结束
- 邀请观众访问展位亲手构建智能代理
- 了解更多Salesforce和AWS合作伙伴关系
- 鼓励联系客户代表或直接咨询
