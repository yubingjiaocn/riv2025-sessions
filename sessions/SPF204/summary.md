# AWS re:Invent 2025 - PGA巡回赛使用Agentic AI生成内容的会议总结

## 一、会议概述

本次会议由PGA巡回赛数字架构副总裁David Provin和AWS解决方案架构师Murali Bakta共同主讲,重点介绍了PGA巡回赛如何利用Agentic AI技术为数字平台自动生成内容。PGA巡回赛作为一个球员所有的体育组织,每周在全球范围内运营高尔夫锦标赛,面临着巨大的数据收集和内容生成挑战。他们在150英亩以上的场地上追踪156名球员的31,000次击球,需要实时为球迷提供有价值的内容。

为了解决内容生成的规模化问题,PGA巡回赛采用了产品优先的方法,专注于为球迷、业务和品牌创造价值。他们从2023年开始探索AI应用,最终在2025年推出了击球评论系统和基于Agentic AI的内容生成平台。该系统每周可生成800篇文章,包括球员投注档案、锦标赛预览、赛后总结等多种类型,大幅提升了内容生产效率,降低了95%的成本,并实现了在赛事结束后5-10分钟内发布文章的速度优势。

通过使用AWS Bedrock和Agent Core等服务,PGA巡回赛构建了一个包含研究、写作、编辑和验证等多个智能代理的系统,确保生成的内容既准确又符合品牌规范。这些AI生成的文章每年获得数十亿次页面浏览量,成为非比赛日吸引球迷的重要手段,特别是在SEO方面取得了领先地位。

## 二、详细时间线与关键要点

### **00:00 - 开场介绍**
- David Provin介绍自己是PGA巡回赛数字架构副总裁
- Murali是AWS解决方案架构师,负责PGA巡回赛账户5年
- 会议主题:使用Agentic AI为数字平台生成内容

### **01:30 - PGA巡回赛组织介绍**
- PGA巡回赛是球员所有的组织,与其他体育联盟不同
- 运营四个不同的巡回赛:PGA巡回赛、Corn Ferry巡回赛、PGA巡回赛冠军赛和PGA巡回赛美洲赛
- 每周举办锦标赛,全年仅有三个周末休息

### **03:00 - 数据收集挑战**
- 比赛场地达150英亩以上,所有区域都是可打球区域
- 标准锦标赛有156名球员,产生31,000次击球
- 采用多种计分格式:团队击球赛、比洞赛、稳定杆计分等

### **05:00 - 数字战略架构**
- 网站是基础层,包含所有详细信息
- 移动应用的用户参与度是网站的7倍
- 内容策略从核心平台向外扩展
- 与USGA和PGA of America合作提供技术

### **07:00 - 数据采集技术**
- 每个球场部署140台4K摄像头
- 每个球洞配备两个军用级雷达(发球台和果岭各一个)
- 使用太阳能板为摄像头供电,实现可持续运营
- 步行记分员实时捕捉每一次击球

### **10:00 - 数据湖基础设施**
- 与AWS合作建立大规模数据湖
- 存储击球数据可追溯到2012年
- 计分和排行榜数据可追溯到1864年
- 高度结构化的数据存储系统

### **12:00 - 媒体湖**
- 每场标准赛事运行7个直播视频流
- 与ESPN、Golf Channel、NBC、CBS合作
- 存储数千小时的高尔夫比赛录像
- 提供"购物车"功能,可按条件检索特定镜头

### **15:00 - AI应用的产品优先方法**
- 2-3年前开始探索AI应用
- 采用产品团队的方法:评估对球迷、利益相关者和品牌的价值
- 不是所有功能都需要满足三个价值点,有时只满足一个

### **17:00 - 生产优先思维**
- 体育行业经常面对边缘案例(如球员将球打到会所顶部)
- 31,000次击球不统一也不一致
- 必须是第一个发布,不能是第二
- 使用评分和验证技术确保准确性

### **19:00 - 运营监控**
- 保持相同的支持团队规模
- 扩展AWS CloudWatch仪表板和集成
- 目标是在4天比赛中"无所事事"就是成功
- 比喻:表面像天鹅优雅,水下腿在疯狂划动

### **21:00 - AI应用演进历程**
- 2023年初:所有人都想要聊天机器人(David不喜欢聊天机器人)
- 2024年:原型开发击球评论系统
- 从聊天机器人转向更有价值的应用

### **23:00 - 击球评论系统**
- 2025年球员锦标赛推出
- 生成事实:Scottie Scheffler击球164码,距球洞5英尺
- 添加背景:他有96%的机会推进这个推杆,如果成功将登上排行榜第一
- 强调是"评论"而非"叙述"
- 有时只报告事实,不添加背景

### **26:00 - ExploreCase功能**
- 每个球场的3D数据模型
- 显示每次击球和雷达曲线
- 显示推杆方向和距离
- 集成投注档案

### **28:00 - 投注档案内容**
- 针对非比赛日(周一至周三)的内容策略
- 比赛日(周四至周日)流量高,非比赛日流量低
- 幻想和博彩用户对统计数据高度参与
- 156名球员需要156篇类似文章

### **30:00 - 内容生成效率**
- 不可能要求内容作者每天写156篇相似文章
- 周一早上9点开始生成,9点30分完成
- 快速进入SEO索引
- 非比赛日参与度最高的内容

### **32:00 - SEO领先地位**
- PGA巡回赛在高尔夫运动SEO方面领先
- 用户首先访问PGA巡回赛网站查看比分
- 通过此类内容努力推动SEO结果

### **34:00 - AI应用的下一步**
- 从球迷参与转向运营改进
- 每周生成约180篇不同类型的文章(8种类型)
- 奖金分配文章:比赛结束后搜索量垂直上升
- 积分分配文章:基于排名计算

### **37:00 - 思维转变**
- 2025-2026年:不再做"AI项目",而是在合适的地方使用AI
- 避免"必须有更多AI"的陷阱
- 在有价值和意义的地方使用AI

### **39:00 - 开发和运营工具**
- 探索Cursor、Claude等AI编码工具
- 图像生成是一个挑战(球员权利和IP问题)
- 在编码、AWS账户分析、支出异常检测等方面看到价值

### **41:00 - AI工具使用的洞察**
- AI工具对初级开发者:生成看起来不错的代码
- AI工具对高级开发者:能识别问题并改进模式
- AI代理像小狗一样想让你满意
- 高级开发者提供更直接的提示指导

### **44:00 - Murali开始技术深入讲解**
- 规模化内容生成的挑战
- 每周生成约800篇文章
- 使用AI生成实时体育赛事内容很困难

### **46:00 - 内容生成的挑战**
- LLM通常没有实时数据
- 需要向LLM提供实时体育数据
- 必须保护品牌价值
- 验证内容的事实准确性
- 遵守品牌和风格指南

### **48:00 - 文章类型多样性**
- 投注档案文章(156篇,每位球员一篇)
- 投注档案摘要
- 锦标赛预览
- 锦标赛总结
- 球员总结(156篇)
- 回合总结(每轮比赛后)

### **50:00 - 内容格式多样性**
- 网站和移动应用的长篇文章
- 社交媒体的段落格式
- 应用通知的短格式内容
- 架构需要适应所有这些格式

### **52:00 - Agentic AI架构概述**
- 内容请求进入系统
- 研究阶段:从PGA巡回赛API获取结构化数据
- 数据代理:从媒体指南(PDF)获取非结构化数据
- 创建工作订单

### **55:00 - 编辑和写作流程**
- 编辑代理从工作订单获取任务
- 要求写作代理为特定受众和格式写作
- 写作代理基于提供的信息生成内容
- 从Getty库选择合适的图像

### **57:00 - 图像选择**
- 图像必须与文章类型相关
- 必须以积极的角度展示球员或PGA巡回赛
- 避免使用球员生气等负面图像
- 图像与文章一起返回给编辑代理

### **59:00 - 编辑审核循环**
- 编辑代理检查文章质量
- 如果不符合风格和品牌指南,返回给写作代理修改
- 编辑满意后发送验证请求

### **61:00 - 验证流程**
- 从内容中提取事实
- 数据代理从巡回赛API获取数据进行比对
- 例如:验证Scottie Scheffler在18号洞打出小鸟球
- 如果匹配,验证通过
- 如果不匹配,返回编辑代理修复

### **63:00 - 发布流程**
- 验证完成后发送发布请求
- 发布到不同渠道

### **65:00 - 技术架构细节**
- 使用AWS Bedrock Agent Core实现
- 请求进入系统后存储在DynamoDB表中
- 作业转换器将请求放入SQS队列
- 避免同时处理所有请求导致令牌限制

### **67:00 - Agent Core运行时**
- Lambda从SQS队列获取消息
- Agent Core Lambda调用器调用Agent Core运行时
- 调用巡回赛API获取数据
- 调用Getty API获取图像
- 调用Bedrock生成内容

### **69:00 - 成本优化**
- Agent Core运行时在等待响应时不收费
- 节省大量计算成本
- 使用Agent Core可观测性监控延迟

### **71:00 - 内容存储和发布**
- 生成的内容写入S3存储桶
- 触发Lambda将内容推送到内容摄取工作流

### **73:00 - 成本节省数据**
- 每篇文章生成成本仅25美分
- 成本降低95%
- 目前每周生成140-180篇文章
- 年底将达到每周800篇

### **75:00 - 使用Nova模型**
- 使用Nova模型进行图像审核
- 检查图像是否适合文章
- 使用Nova后成本降低75%
- Nova模型性价比更高

### **77:00 - 文章浏览量**
- AI生成的文章每年获得数十亿次页面浏览
- 每周锦标赛期间浏览量达到峰值
- 锦标赛结束后浏览量下降

### **79:00 - 速度优势**
- 比赛结束后5-10分钟内生成文章
- 例如:下午5点比赛结束,5点05分或5点10分发布文章
- 比其他体育媒体更快进入市场
- Google搜索中获得最高点击量

### **81:00 - 会议结束**
- 总结了PGA巡回赛使用Agentic AI的完整实施过程
- 展示了从概念到生产的完整旅程
