# AWS re:Invent 2025 NET323 会议总结

## 会议概述

本次会议主题为"大规模网络运维自动化"（Networks at Scale: How to Automate Operations），由AWS高级网络专家解决方案架构师Pablo Sanchez和高级解决方案架构师Claudia Izquierdo主讲。会议采用代码演示形式，重点展示如何通过自动化、测试和策略驱动部署来解决大规模网络运维中的三大挑战：无法扩展的手动流程、造成安全风险的不一致部署，以及过晚的验证。

演讲者通过实际编码演示，展示了如何使用AWS Organizations、AWS Cloud WAN、VPC IPAM、Step Functions等服务构建自动化网络解决方案。整个会议强调将开发团队已经采用的最佳实践应用到网络运维中，实现网络基础设施的代码化管理和自动化运维。

## 详细时间线与关键要点

### 0:00-10:00 会议开场与挑战分析
- Pablo和Claudia介绍会议议程，强调这是一个代码密集型的互动会议
- 分析大规模网络运维面临的三大挑战：
  - 手动流程无法扩展
  - 不一致的部署造成安全风险
  - 验证发生得太晚
- 描述企业网络增长过程中的痛点：从小规模VPC到指数级增长带来的管理复杂性

### 10:00-15:00 解决方案架构概述
- Claudia介绍三大解决策略：
  - 采用多账户环境和护栏机制
  - 使用AWS托管网络服务构建全球网络
  - 采用无服务器架构实现自动化
- 重点介绍AWS Organizations、AWS Cloud WAN和AWS Step Functions的作用

### 15:00-25:00 用例场景与现有架构
- Pablo详细介绍演示用例的需求：
  - 通过VPC IPAM控制IP地址管理
  - 按环境实现流量分段（dev、prod等路由域）
  - 实现路由域间和出口流量检查
  - 降低运维开销
- 展示已构建的组织架构和网络基础设施

### 25:00-35:00 第一个需求：VPC IPAM作为唯一IP分配源
- Pablo开始编码演示，使用Kiro IDE和Terraform
- 演示创建不合规VPC的过程
- 实施服务控制策略（SCP）阻止非IPAM池的VPC创建
- 观众提问关于IPAM池分配和动态账户创建的问题

### 35:00-45:00 第二个需求：构建全球网络
- 介绍AWS Cloud WAN的"send to"和"send via"概念
- Pablo编码实现Cloud WAN策略文档
- 配置路由域间检查、出口检查和跨区域检查
- 实现附件自动关联到相应路由域的逻辑

### 45:00-55:00 第三个需求：强制路由域关联
- 创建额外的SCP阻止手动设置域标签
- 使用EventBridge捕获Network Manager事件
- 实现Step Functions自动化工作流：
  - 等待VPC附件创建
  - 从账户ID获取OU信息
  - 从网络账户创建域标签
- 演示自动化流程的实际运行

### 55:00-59:30 总结与关键要点
- 强调三个关键原则：
  - 根据用例而非习惯进行架构设计
  - 从业务成果出发实现架构敏捷性
  - 超越网络层面的编排自动化
- 提供相关资源链接：AWS Samples仓库、Cloud WAN蓝图、AWS Skill Builder课程
