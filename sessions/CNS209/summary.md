# AWS re:Invent 2025 - 简化 Amazon EKS 集群中有状态工作负载的备份

## 会议概述

本次会议由 AWS 高级解决方案架构师 Sandosh Walur Pali 主讲，重点介绍了如何使用 AWS Backup 简化 Amazon EKS 集群中有状态工作负载的备份。会议在 re:Invent 2025 的周四午餐时间举行，主要面向已经在使用 Amazon EKS 的用户群体。

演讲者首先介绍了 AWS Backup 作为完全托管服务的核心功能，它能够集中化和自动化跨所有支持的 AWS 服务以及混合工作负载的数据保护需求。该服务采用简单的策略驱动方式，帮助用户从数据保护需求出发进行反向工作。AWS Backup 支持去中心化和集中化两种部署方式，可以通过 AWS Organizations 在组织级别统一管理备份策略。

会议的重点是宣布 AWS Backup 对 Amazon EKS 的原生支持，这是上个月在 CubeCon 上发布的新功能。该功能使客户能够备份和恢复各种 Kubernetes 对象，包括集群范围和命名空间范围的资源，如部署、配置映射、密钥、集群角色等。同时还支持运行在 EBS、S3 和 EFS 上的有状态工作负载及其对应的持久卷的备份。
