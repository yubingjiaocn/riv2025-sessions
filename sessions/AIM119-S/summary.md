# AWS re:Invent 2025 会议总结：云安全如何演变为AI安全

## 会议概述

本次会议由Wiz公司的产品管理联合创始人Inon Costa主讲，深入探讨了AI技术如何从根本上改变了云安全的攻击面。演讲者指出，AI不仅仅是炒作，而是已经成为现实——超过90%的云环境至少使用一个托管AI模型，80%的环境使用了AWS Bedrock等托管AI服务。这种快速采用带来了全新的安全挑战，包括新的攻击向量、AI驱动的恶意软件，以及通过低代码平台生成的系统性漏洞。

演讲的核心观点是，AI既扩展了攻击面（引入了模型、提示词、MCP等新组件），也加速了攻击者的能力（从漏洞武器化到钓鱼攻击的自动化）。Costa提出了一个四象限模型来理解这些威胁：现有攻击面vs新攻击面，以及现有技术vs新技术。解决方案包括三个支柱：使用Wiz的CNAPP平台扩展传统安全工具以覆盖AI工作负载、通过攻击面管理主动发现暴露点，以及利用AI代理加速安全运营。演讲强调，虽然AI带来了新挑战，但通过正确的工具和方法论，组织今天就可以开始保护其AI转型。

## 详细时间线

### 开场与议程介绍
[00:00 - 01:30]
- Inon Costa自我介绍，担任Wiz产品管理职务及联合创始人
- 会议主题：云安全如何演变为AI安全
- 议程预告：攻击面变化、AI安全实践、攻击面管理、AI代理在安全运营中的应用

### AI采用现状分析
[01:30 - 04:00]
- 调查数据显示AI已成为现实：90%的环境至少使用一个托管AI模型
- 80%的云环境使用托管AI服务（如AWS Bedrock）
- 超过50%的环境已在使用MCP（模型上下文协议）
- 80%的开发环境安装了至少一个AI IDE扩展
- 近20%的GitHub应用生态系统已运行AI应用

### 攻击者如何利用AI
[04:00 - 07:30]
- AI加速攻击者的能力：更快生成钓鱼邮件、自动化漏洞武器化
- 漏洞利用时间从天缩短到小时甚至分钟
- AI辅助侦察和自动化攻击流程
- Wiz发起的安全竞赛：针对20多种AI和云原生技术寻找RCE漏洞
- AI不仅帮助攻击者，也帮助研究人员更快发现漏洞

### AI驱动的恶意软件新趋势
[07:30 - 10:00]
- 新型恶意软件使用AI提示词作为命令（如LummaStealer案例）
- 乌克兰CERT报告的案例：恶意软件通过纯英文提示词执行命令
- Amazon Q开发者扩展攻击中利用AI提示词
- Singularity攻击：通过供应链漏洞在开发者机器上运行恶意提示词
- IOC（入侵指标）从IP地址和哈希值演变为英文恶意语句

### 新型攻击技术
[10:00 - 12:30]
- "Living off the LLM"：利用本地AI代理进行横向移动和侦察
- AI组件供应链攻击：恶意模型、提示词、MCP的投毒
- 代理妥协：Salesloft Drift攻击案例，通过代理代码获取敏感凭证
- 低代码平台的系统性漏洞：认证逻辑在客户端、密钥嵌入等模式性问题
- 攻击者也可以使用低代码工具快速开发针对性攻击工具

### 威胁分类四象限模型
[12:30 - 16:00]
- 横轴：现有攻击面 vs 新攻击面（模型、训练数据、代理等）
- 纵轴：现有技术 vs 新技术
- 四个象限的应对策略：
  - 现有技术+扩展攻击面：改进现有平台，扩展CSPM到AISPM
  - 现有技术+现有攻击面：保持良好的安全基础
  - 新技术+现有攻击面：利用AI工具增强防御
  - 新技术+新攻击面：需要创新解决方案（如模型扫描）

### AI可见性挑战
[16:00 - 20:00]
- AI应用的四层架构：
  1. 基础设施层（容器、网关、存储桶、数据库）
  2. AI组件层（模型、训练管道、护栏、提示词、MCP）
  3. 数据层（用户输入输出、训练数据、代理可访问数据）
  4. 应用层（代码、配置、暴露方式）
- 架构决策的复杂性：模型可以是自托管、PaaS（Bedrock）或SaaS（OpenAI）
- 代理身份可以是云角色、应用角色或SaaS角色
- 需要统一视图才能理解完整的AI应用

### AI安全解决方案：可见性
[20:00 - 23:30]
- 连接所有云环境、SaaS环境和代码仓库
- 跨开发生命周期检测AI使用：代码中的SDK、云中的Bedrock服务、存储桶中的训练数据
- 服务目录功能（GA）：自动聚合资源到服务视图，可按"是否包含AI"筛选
- 解决"影子AI"问题：DeepSeek发布一周内被10%的云环境使用
- 可见性扩展到整个AI管道，包括训练数据集

### AI安全解决方案：扫描与配置
[23:30 - 26:30]
- 将现有扫描器扩展到AI工作负载：
  - 身份扫描：代理的权限
  - 漏洞扫描：模型和AI技术的CVE
  - 网络扫描：互联网暴露面
  - 新扫描器：模型恶意软件检测
- 发布数十条AI资源配置规则（Bedrock、SageMaker等）
- 开箱即用，无需额外配置

### 低代码平台安全集成
[26:30 - 28:30]
- 与Lovable等低代码平台合作
- 在Lovable门户中直接提供Wiz安全扫描
- 使用与传统代码相同的安全策略
- 开发者可在Lovable和Wiz中查看发现
- 将安全思维注入低代码开发流程

### 风险优先级排序
[28:30 - 31:00]
- 利用安全图谱进行攻击路径分析
- 真实案例检测：
  - 外部暴露的MCP具有过度云权限，可访问敏感数据
  - 暴露的应用，其代理可访问敏感数据
- 通过图谱从暴露点追踪到身份再到敏感数据
- 优先处理"有毒组合"（toxic combinations）

### 运行时监控
[31:00 - 33:00]
- 监控AI行为以检测恶意活动
- 检测恶意模型、模型妥协、模型逃逸
- 通过主机运行时传感器监控模型
- 监控控制平面以识别LLM资源的异常活动
- 检测LLM劫持攻击

### AI安全总结
[33:00 - 35:00]
- 四步法：
  1. AI可见性（基础）
  2. 扫描AI风险（已知和新风险）
  3. 优先级排序和修复（关注"三要素"：输入+敏感数据访问+输出）
  4. 运行时监控

### 攻击面管理新能力
[35:00 - 40:00]
- 本周GA的新功能，与安全图谱深度集成
- 工作流程：
  1. 分析云环境的潜在暴露点（公共配置、网关、负载均衡器、公共仓库）
  2. 从外部自动扫描（模拟攻击者视角）
  3. 指纹识别技术栈（数据库类型、Jenkins等）
  4. 测试多种风险（默认凭证、错误配置、可利用漏洞）
- 关键优势：
  - 验证暴露和影响
  - 自动识别资源所有者（通过代码分析）
  - 生成可操作的工单和PR
  - 零部署，一键启用

### AI代理在安全运营中的应用
[40:00 - 45:00]
- Wiz的云运营模型演进：
  1. 民主化：让开发团队修复问题
  2. 左移：在SDLC中预防风险（Wiz Code）
  3. SecOps：清理后专注威胁监控和响应
- 安全图谱是基础：实现优先级排序和可视化解释
- 教AI代理理解图谱，使其成为"同事"
- MCP优先策略：类似API优先，使上下文可供任何团队使用
- Wiz外部MCP服务器：将Wiz上下文流式传输到任何角色的原生体验
- Grammarly等客户案例（演讲在此处被截断）
