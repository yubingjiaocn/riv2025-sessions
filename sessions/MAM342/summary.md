# AWS re:Invent 2025 - 使用代理AI在企业规模自动化代码现代化

## 会议概述

本次会议由AWS Transform Custom团队的高级产品经理Elio Damaggio和Morgan Lunt主讲，重点介绍了如何使用代理AI技术在企业规模自动化代码现代化。会议深入探讨了技术债务的挑战，展示了AWS Transform Custom产品如何帮助企业解决大规模代码现代化问题。

演讲者强调，技术债务是一个普遍存在的问题，影响着每个组织。传统的手动方法和基于规则的自动化工具存在局限性，而AWS Transform Custom通过代理AI技术提供了更智能、更灵活的解决方案。该产品不仅支持AWS管理的转换（如Java、Python、Node.js运行时升级），还允许用户创建自定义转换来解决组织特定的挑战。

## 详细时间线与关键要点

### 0:00-5:00 - 开场与技术债务挑战
- 介绍演讲者和会议主题
- 统计数据显示技术债务的严重性：
  - 20%的IT预算用于解决技术债务
  - 23%的开发者时间因技术债务而损失
  - 美国每年技术债务成本达2.4万亿美元
- 技术债务的多种形式：安全合规风险、生命周期结束的运行时、维护负担、性能限制等

### 5:00-10:00 - 现有解决方案的局限性
- 手动方法的问题：将技术债务视为普通软件开发任务
- 基于规则的自动化工具局限性：
  - 需要专业知识
  - 脆弱且不灵活
  - 启动成本高
  - 难以定制
- 通用编程AI的限制：主要用于交互式执行，不适合自主执行

### 10:00-15:00 - 客户案例研究
- Air Canada：Lambda升级和Graviton迁移项目
- Twitch：900多个代码库的AWS SDK v1升级
- QAD：制造业客户平台迁移
- MongoDB：帮助客户迁移旧版SDK

### 15:00-25:00 - AWS Transform Custom产品介绍
- 2024年5月GA发布AWS Transform，支持VMware迁移、Windows现代化和大型机
- AWS Transform Custom的主要组件：
  - 自然语言定义转换
  - 大规模执行能力
  - 多代码库活动支持
  - 从开发者反馈中学习
- AWS管理的转换：Java、Python、Node运行时升级，AWS SDK升级
- 早期访问功能：技术债务分析、x86到Graviton的Java迁移

### 25:00-35:00 - 快速入门演示
- 安装过程：单行命令安装
- 依赖要求：Node.js和Git
- 支持平台：Linux、macOS、Windows（通过WSL）
- 交互式和机器驱动模式
- 实际演示Python代码库升级过程

### 35:00-45:00 - 企业规模扩展策略
- 扩展模式：
  - 规划阶段：确定转换目标和代表性目标
  - 试点阶段：测试初始转换定义
  - 规模化阶段：推送和拉取活动模式
- 最佳实践：
  - 大量试点测试
  - 学习项目管理
  - 脚本和自动化集成
- AWS Transform Web应用程序用于活动管理

### 45:00-53:00 - 定制化和价值定价
- 定制化需求：组织特定模式、内部库、验证标准
- 创建自定义转换的方法：
  - 提示和优化
  - 基于指南的方法
  - 交互式"与我一起做"模式
- OpenTelemetry追踪添加的实际演示
- 定价模型：按使用付费，每代理分钟$0.035
- 客户价值：直接时间节省、启用优先级项目、质量一致性
- 渐进式学习建议：从文档开始，然后AWS管理转换，最后自定义转换
