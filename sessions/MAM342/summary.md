# AWS re:Invent 2025 - 使用 Agentic AI 实现企业级代码现代化自动化

## 会议概述

本次会议介绍了 AWS Transform Custom，这是一款利用 Agentic AI 技术帮助企业大规模自动化代码现代化的新产品。会议由 AWS Transform Custom 的高级产品经理 Eric Deajjo 和 Morgan Land 主讲。

技术债务是当今企业面临的普遍挑战。数据显示，20% 的 IT 预算用于处理技术债务，23% 的开发人员时间因技术债务而损失，仅在美国每年就造成 2.4 万亿美元的成本。技术债务包括安全合规风险、生命周期结束的运行时、易受攻击的库、维护负担、性能限制以及战略不一致等多个方面。传统的解决方案要么是手动处理，要么使用基于规则的自动化工具（如 OpenRewrite），但这些工具通常需要专业知识、缺乏灵活性且启动成本高。通用编码 AI 助手虽然有用，但更适合交互式执行，难以实现自主执行和知识复用。

AWS Transform Custom 提供了一个低门槛、专为自动化设计的解决方案。它允许用户使用自然语言定义转换，支持大规模执行，能够从开发人员反馈中持续学习，并可以在整个组织中复用转换定义。产品提供了 AWS 托管的转换（如 Java、Python、Node 运行时升级和 AWS SDK 升级），同时也支持用户创建自定义转换。在 Beta 测试期间，Air Canada 实现了 90% 的转换成功率和 80% 的时间成本节省；Quad 将原本需要两周的项目缩短到三天；Twitch 在 913 个代码库的迁移中实现了 70% 的加速。
