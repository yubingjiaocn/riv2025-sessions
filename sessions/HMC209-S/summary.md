# AWS re:Invent 2025 技术会议总结

## 会议概述

本次会议由 AWS 网络市场负责人 Ranga Tatai 和 Equinix 产品经理 Brandon Wright 共同主讲，主题聚焦于为混合云和多云环境构建高性能安全网络。会议深入探讨了大规模数据迁移的挑战，以及如何通过 AWS Direct Connect 和 Equinix Fabric 等解决方案来应对这些挑战。

演讲者首先分析了当前企业面临的数据传输需求，特别是在生成式 AI、模型训练、RAG（检索增强生成）应用、自动驾驶、媒体制作等场景下的海量数据迁移需求。这些用例往往需要在 6-9 个月内迁移数百 PB 的数据到 AWS，这对网络的可靠性和带宽提出了极高要求。会议重点介绍了 AWS Direct Connect 作为专用网络连接的优势，包括避免互联网不稳定性、提供一致的性能、广泛的全球覆盖以及 MACsec 加密支持。

Equinix 作为会议赞助商，详细展示了其 Fabric Cloud Router 网络即服务平台的能力。该平台覆盖全球 270 多个数据中心、36 个国家、75 个城市，提供 56 个 AWS Direct Connect 接入点，并承诺 99.999% 的全球正常运行时间 SLA。通过实际测试数据和客户案例，演讲者证明了私有网络连接在延迟敏感型应用中的显著优势，以及在大规模数据传输场景下的可靠性。

## 详细时间线

0:00 - 0:45 - 开场介绍
- Brandon Wright（Equinix 产品经理）和 Ranga Tatai（AWS 网络市场负责人）介绍自己
- 概述会议议程：客户用例、数据迁移考量、混合多云私有连接案例研究

0:45 - 2:30 - 大规模数据迁移的重要性
- 讨论生成式 AI 应用场景：模型训练、基础模型开发、RAG 向量存储构建
- 行业应用案例：自动驾驶车辆、互联制造、云端媒体制作
- 企业迁移场景：6-9 个月内迁移数百 PB 数据的需求

2:30 - 3:45 - 生成式 AI 的网络需求
- 模型训练和微调需要从混合环境和多云环境迁移大量数据
- 分布式推理场景：混合 RAG、混合 EKS 部署
- 强调实时交互场景下高性能、确定性网络的重要性

3:45 - 5:00 - AWS Direct Connect 介绍
- 专用网络路径，避免互联网不稳定性
- 提供可用性 SLA，确保关键项目的持续连接
- 覆盖 30 多个国家、100 多个位置
- 支持 MACsec 第二层加密，满足合规要求

5:00 - 6:15 - Direct Connect 性能对比
- 案例：10 PB 数据迁移
- 使用 25 Gbps 托管 Direct Connect：1 小时完成
- 使用 VPN over Internet：同样时间仅完成 5%
- 对于 10 PB 迁移项目，可靠性和速度优势显著累积

6:15 - 7:00 - 合作伙伴生态系统
- AWS 合作伙伴提供即插即用的连接解决方案
- 基于用例的弹性连接模型
- 可根据供应商选择进行定制

7:00 - 8:30 - Equinix 公司背景
- 成立 26 年，起源于互联网交换中心
- 从网络服务提供商互联发展到公有云连接
- 作为中立互联合作伙伴，连接 AWS 和其他云提供商
- 目前服务超过 11,000 家企业客户

8:30 - 10:00 - Equinix Fabric 平台介绍
- 覆盖 270 多个数据中心、36 个国家、75 个城市、6 大洲
- 全球软件定义网络
- 220 多个云接入点，其中 56 个为 AWS Direct Connect
- 使用 2,000 多个光纤和电信网络
- 提供 99.999% 全球正常运行时间 SLA

10:00 - 12:00 - Fabric Cloud Router 产品特性
- 2024 年 1 月推出的完全基于服务的解决方案
- 无需虚拟设备、许可证或客户硬件
- 按需在指定城市部署实例，几分钟内完成
- 原生支持高可用性，提供完全独立的主备数据路径
- 多种套餐和层级选择
- 支持 50 Mbps 到 100 Gbps 的单连接吞吐量
- 单个云路由器实例支持超过 1 Tbps 的聚合吞吐量

12:00 - 13:30 - 混合多云连接场景
- 客户可在多个城市部署云路由器
- 通过 Fabric Port 与托管基础设施互联，最高 100 Gbps
- 连接不同云提供商
- 优势：本地路由替代传统的回程路由（hairpin routing）
- 极低延迟的云间互联

13:30 - 16:00 - 性能测试数据展示
- 测试环境：西雅图和阿什本之间的云路由器
- 连接 AWS（西海岸）和 GCP（东海岸）
- 使用 10 Gbps 虚拟连接（AWS 托管 Direct Connect）
- 平均延迟：63 毫秒（跨越美国东西海岸）
- 固定 4 跳路径：Direct Connect 网关 → 云路由器 → 第二个云路由器 → Google 云路由器
- 相比互联网路由，路径高度可预测，抖动极低

16:00 - 18:00 - 吞吐量测试结果
- 使用小型虚拟机（仅要求 10 Gbps 出站能力）
- TCP 测试（100 个并行线程）：9.5 Gbps 和 9.26 Gbps
- UDP 测试：接近线速率，63 毫秒延迟下达到 9.89 Gbps
- 单流限制：受 AWS 网络单流限制影响，非云路由器限制

18:00 - 20:30 - 客户案例研究 1：金融服务公司
- 行业：汽车贷款金融服务
- 需求：通过降低响应延迟提高竞争力
- 部署在 AWS 东海岸，需要连接新收购公司的 Azure 环境
- 时间线：首次接触到部署完成仅 4 小时
- 客户从未使用过 Equinix，通过 AWS Marketplace 快速上手
- 成功实现 AWS 与 Azure 的互联

20:30 - 22:00 - 客户案例研究 2：自动驾驶公司
- 需求：每天传输 500 TB 数据
- 数据来源：多个车队的自动驾驶车辆
- 目标：传输到 S3 和其他公有云
- 解决方案：在车队所在区域的数据中心部署物理设备
- 使用多个 25 Gbps AWS Direct Connect 虚拟连接
- 灵活应对车队位置变化，利用 Equinix 全国数据中心网络

22:00 - 23:30 - 免费试用推广

- 包含完整标准付费版本功能
- 提供两个 200 Mbps 虚拟连接（可降至 50 Mbps）
- 适合测试多云和混合多云连接场景
