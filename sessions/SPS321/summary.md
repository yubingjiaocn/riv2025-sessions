# AWS re:Invent 2025 会议总结：使用 Amazon Bedrock Agent Core 构建多智能体金融研究系统

## 一、会议概述

本次 AWS re:Invent 2025 分会场由 AWS GenAI 创新中心的应用科学家 Rahul Gosh、Yahoo Finance 的 Deepen 以及 AWS GenAI 创新中心应用经理 Vidya Saga 共同主讲。会议聚焦于企业如何采用多智能体系统（Multi-Agent Systems）来解决复杂的金融研究和问答应用场景。

演讲者首先介绍了 AI 智能体的基础概念和核心原理，特别是在深度研究场景中的应用。随后深入探讨了 Amazon Bedrock Agent Core 的架构和能力，展示了其如何为金融研究智能体提供独特的价值主张。最后，Yahoo Finance 团队分享了他们在实际生产环境中实施多智能体系统的真实经验，包括遇到的挑战和解决方案。

会议强调了从概念验证到生产部署的关键转变，特别是在处理多模态数据、协调分布式智能体、管理异构数据源以及评估多智能体架构性能等方面的实践经验。这些内容对于希望将生成式 AI 应用于企业级金融分析场景的团队具有重要的参考价值。

## 二、详细时间线与关键要点

### **0:00-2:30 - 开场介绍**
- Rahul Gosh 介绍自己是 AWS GenAI 创新中心的应用科学家，已在亚马逊工作两年多
- 主要工作是与 AWS 客户合作，解决将生成式 AI 引入生产流程的挑战
- Deepen 介绍自己在行业有 17 年经验，在 Yahoo Finance 工作 3 年，团队构建 GenAI 和数据科学功能
- Vidya Saga 介绍 GenAI 创新中心是 AWS 的一个项目，帮助客户快速构思和构建生成式 AI 解决方案

### **2:30-4:00 - 会议议程概览**
- 探讨企业如何采用 Amazon Bedrock 和 Bedrock Agent Core 的多智能体系统
- Yahoo Finance 分享在金融研究和问答应用中实施多智能体系统的真实经验
- 涵盖关键方面：跨分布式环境编排智能体、访问多模态数据、智能体间协调、工具和 API 调用
- 讨论管理异构数据源和评估多智能体架构性能

### **4:00-6:30 - AI 智能体基础概念**
- AI 智能体定义：能够推理、规划并通过迭代思考和改进完成任务的自主系统
- Agentic AI 是 LLM 的重大进步，通过工具、记忆和智能体循环赋予真正的自主性
- 四个关键技术推动因素：模型推理能力、数据和知识集成、基础设施和智能体协议、智能体开发工具
- 介绍智能体成熟度等级：从基于规则的自动化到生成式 AI 助手，再到目标驱动的 AI 智能体

### **6:30-9:00 - 智能体推理策略**
- 核心智能体循环：用户交互 → 智能体决策 → 工具调用 → 推理 → 响应
- ReAct 模式（推理然后行动）：交织的思考-行动-观察循环
- ReWOO 模式：无观察推理，预先生成复杂执行计划，并行执行工具调用
- Reflexion 模式：促进自我反思和迭代改进，评估报告质量并识别研究空白
- 金融研究中结合使用多种策略以获得最佳效果

### **9:00-12:00 - 单智能体 vs 多智能体系统**
- 单智能体的局限性：工具过多会导致选择混淆，提示词膨胀，性能下降
- 多智能体优势：通过智能体专业化提高性能，每个子智能体专注于特定领域
- 两种主要模式：监督者-子智能体模式（层级化控制）和智能体群模式（分布式协作）
- 混合模式：结合结构化层级流程和协作智能
- 金融研究案例采用监督者-子智能体范式以保持结构化控制

### **12:00-15:00 - 金融分析的关键需求**
- 复杂推理需求：多步推理、假设生成、数据收集、验证和改进循环
- 多模态数据集成挑战：处理财务报告文本、结构化财务数据、可视化内容、音频和视频
- 数据来源于数百个全球来源，更新频率和可靠性标准各异
- 规模与质量权衡：海量数据中容易遗漏关键数据点
- 需要完整的审计追踪和重要信息检测，同时保持研究质量一致性

### **15:00-17:30 - 金融研究架构示例**
- 采用监督者-子智能体模式
- 监督者智能体与用户交互，协调整体研究工作流
- 结构化数据库智能体：连接 SQL 数据库，执行查询
- 新闻智能体和 SEC 文件智能体：连接非结构化数据，通过 MCP 服务器访问向量存储
- 内部和合作伙伴 API 智能体：与外部 API 交互
- 每个智能体独立运作，监督者确保全面分析并返回综合报告

### **17:30-21:00 - 生产部署的挑战**
- 从个人工具到企业规模部署的复杂性递增
- 五个关键挑战：性能（从少数用户扩展到多用户）、可扩展性（动态资源管理）、安全性（身份验证和身份管理）、上下文保持（跨长期会话）、审计和合规性
- 这些是所有 POC 扩展到企业级必须解决的关键挑战
- 基础设施和运营开销会减缓创新速度

### **21:00-24:00 - Amazon Bedrock Agent Core 介绍**
- Agent Core 是一个智能体工具链，专门解决生产部署挑战
- 模块化解决方案：可以只使用特定组件（如可观测性或内存功能）
- 三大核心价值主张：加快业务用例的价值实现时间、支持任何框架或模型（不锁定技术栈）、提供安全、可扩展和可靠的企业级平台
- 无需担心基础设施、运营开销、吞吐量和自动扩展问题

### **24:00-27:00 - Agent Core 组件详解**
- Agent Core Runtime：安全可信的智能体部署运行时环境
- Agent Core Gateway：托管所有工具（MCP 服务器或 API）的单一位置
- Agent Core Identity：提供智能体间的安全身份验证和访问控制
- Agent Core Memory：跨会话持久化，保持上下文
- Agent Core Observability：监控智能体行为、性能指标、延迟和令牌使用情况
- 支持 OpenTelemetry 标准，可集成到现有监控工具

### **27:00-30:00 - Agent Core Runtime 特性**
- 支持多模态：文本、图像、音频和视频的大型负载
- 内置自动扩展：处理从 10 个用户到黑色星期五级别的流量变化
- 支持实时和批处理工作负载：同步和异步推理机制
- 会话隔离：为多租户提供安全和隐私保护
- 简单集成：只需添加 10 行代码即可将现有智能体部署到 Agent Core

### **30:00-33:00 - Agent Core 可观测性**
- 提供标准化的指标，无需构建单独的可观测性工具
- 审计日志和监控数据直接输出到 OpenTelemetry 标准
- 可集成到现有的监控和可观测性工具
- 清晰观察智能体性能、采取的行动、延迟和令牌使用情况
- 能够识别多智能体架构中的具体问题位置

### **33:00-36:00 - 多智能体架构中的 Agent Core 应用**
- 所有子智能体托管在 Agent Core Runtime 上
- 所有工具（MCP 服务器或 API）托管在 Agent Core Gateway 上
- Agent Core Identity 提供身份验证
- Agent Core Memory 启用长期和短期记忆持久化
- Agent Core Observability 提供跨智能体的细粒度信息：调用了哪些子智能体、使用了哪些工具、令牌使用情况、延迟位置

### **36:00-39:00 - Agent Core 对深度研究的支持**
- 持久化长时间运行的工作流：提供 8 小时运行时，支持多市场分析、全面回测和模型改进
- 高级内存管理：双重内存架构（长期记忆保存历史关联和研究上下文，短期记忆保存当前工作流信息）
- 生产级企业平台：以安全可信的方式部署，自动扩展，包含所有必需组件
- 使智能体能够像人类一样利用历史知识处理新信息

### **39:00-42:00 - Yahoo Finance 案例介绍开始**
- Yahoo Finance 使命：赋能投资者做出自信的财务决策
- 全球排名第一的金融新闻和信息网站
- 覆盖全球超过 1.5 亿用户
- 除美国外，在 10 个优先全球市场运营
- 提供实时市场更新、深度访谈和全面的投资者工具
- 可通过网页和移动应用（iOS 和 Android）访问
