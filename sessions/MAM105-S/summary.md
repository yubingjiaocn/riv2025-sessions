# AWS re:Invent 2025 Cribl 会议总结

## 会议概述

本次会议由 Cribl 公司的 Kam Amir 和现场 CISO Ed Bailey 主讲，主要介绍了 Sophos 如何使用 Cribl 平台来现代化其 AWS 数据和其他数据源的安全管理。会议涵盖了 Sophos 面临的挑战、采用的解决方案、取得的成果以及最佳实践。

Cribl 是一家成立约七年的后期创业公司，拥有超过 1000 名员工，专注于数据管道和遥测数据解决方案。Sophos 作为全球领先的 MDR 安全厂商，拥有 60 万客户，面临着数据量指数级增长（60% 年增长率）和成本控制的双重挑战。通过部署 Cribl 解决方案，Sophos 实现了显著的成本节约和数据管理现代化。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍
- 介绍演讲者和会议议程
- Sophos 公司背景：全球 MDR 专家，60 万客户，数据量巨大
- Cribl 公司简介：七年历史，1000+ 员工，数据管道解决方案先驱

### 5:00-10:00 面临的挑战
- **数据增长挑战**：行业平均 28-30% 增长率，Sophos 达到 60%
- **数据源多样性**：多云环境、多终端、多种数据类型
- **成本压力**：传统工具无法跟上增长速度，工程成本、SIEM 成本、存储成本均不可持续
- **预算与数据增长不匹配**：30% 年复合增长率 vs 有限预算增长

### 10:00-15:00 解决方案架构
- **渐进式现代化**：避免一次性替换所有系统的高昂成本
- **数据分层策略**：根据数据价值匹配存储成本，SIEM 约 1 美元/GB
- **Cribl Stream 部署**：先部署数百台 Stream 服务器，后逐步推出 Edge 代理
- **成本与价值优化**：区分必需数据、可能需要的数据和不需要但必须接收的数据

### 15:00-20:00 SIEM 迁移策略
- **无风险迁移**：通过 Cribl Stream 同时向新旧 SIEM 发送数据副本
- **并行验证**：在新 SIEM 上构建内容并验证检测效果
- **软切换**：避免传统的硬切换风险，可保持 30 天并行运行
- **加速价值实现**：从传统的 8-12 个月缩短到 5 个月

### 20:00-25:00 Cribl 产品组合
- **Cribl Edge**：替代传统代理，集中管理复杂性，基于策略的管理
- **Cribl Stream**：核心产品，数据收集、转换、路由、富化和 PII 处理
- **Cribl Lake**：对象存储解决方案，内置基于角色的访问控制
- **Cribl Search**：跨组件查询能力，包括与 Security Hub 的新集成

### 25:00-30:00 实施成果与最佳实践
- **显著成本节约**：首月即实现 48% 成本节约
- **数据独立性**：避免供应商锁定，数据可移植到任何平台
- **安全设计**：从设计阶段就考虑安全性，持续监控和检测
- **企业数据共享**：防火墙日志可同时服务 SOC、NOC、DevOps 和网络工程团队

### 30:00-37:00 演示与总结
- **实时监控仪表板**：展示数据消费、负载状态和系统健康度
- **数据路由配置**：演示如何根据用户认证事件进行数据分流
- **数据富化示例**：为防火墙数据添加上下文信息
- **免费资源推广**：Cribl Cloud 免费租户（每日 1TB）、沙盒环境、免费培训认证

### 关键学习要点
- 制定长期数据治理计划，逐步推进
- 建立集中化数据可见性和监控
- 早期过滤和优化数据，移除 SIEM 中 90 天未使用的数据
- 采用分阶段迁移方法，降低风险
- 利用 AI 辅助工具提高运维效率
