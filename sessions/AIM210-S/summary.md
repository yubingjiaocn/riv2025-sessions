# AWS re:Invent 2025 Dynatrace 日志分析与管理会议总结

## 会议概述

本次技术会议聚焦于通过 Dynatrace 和 Davis AI 的强大功能实现大规模日志分析和日志管理。会议由三位演讲者主持：Dynatrace 欧洲、中东和非洲地区首席技术官 Jon Griffiths，Storio Group 工程总监 Alex Hibbitt，以及 AWS 合作伙伴专家全球可观测性用例负责人 Frank Schwarzenau。

会议重点探讨了如何在 PB 级规模下进行日志摄取和精确根因分析，展示了 Dynatrace 与 AWS 的深度合作关系，以及客户如何通过现代化的可观测性方法实现创新加速、缩短故障解决时间。Storio Group 作为欧洲最大的照片礼品公司，分享了他们从传统日志管理向 AI 驱动的现代可观测性平台转型的实际案例，展现了如何通过文化变革、技术整合和 AI 自动化实现显著的业务价值。

## 详细时间线与关键要点

### 0:00-10:00 会议开场与 Dynatrace 平台介绍
- **演讲者介绍**：Jon Griffiths（Dynatrace 欧洲、中东、非洲地区首席技术官）、Alex Hibbitt（Storio Group 工程总监）、Frank Schwarzenau（AWS 合作伙伴专家）
- **Dynatrace 核心定位**：专注于以 AI 为核心的可观测性和安全性，整合日志、指标、链路追踪等所有信号
- **市场领导地位**：Gartner 魔力象限中执行能力最强，在六项关键能力中四项排名第一，连续 15 年保持领导者象限地位
- **客户覆盖**：服务于航空、银行、零售、公用事业等各个行业，遍布全球各大洲

### 10:00-20:00 AWS 与 Dynatrace 合作关系
- **合作伙伴地位**：AWS 拥有约 12 万个合作伙伴，其中仅 8 个 ISV 可观测性合作伙伴获得专家认证
- **能力认证**：Dynatrace 获得多项 AWS 能力认证，包括 AI 用例的深度集成
- **技术集成**：与 AWS Bedrock、AgentCore 的端到端可观测性集成，以及 Kiro 集成用于 IDE 中的 AI 辅助故障排除
- **客户价值**：500 个 AI 用例实施案例，涵盖应用层、基础设施层、编排层和语义层的全方位可观测性
- **Well-Architected 框架对齐**：在弹性、资源优化、成本优化、性能优化和 SLA 管理等方面与 AWS 核心原则保持一致

### 20:00-35:00 Storio Group 公司背景与挑战
- **公司规模**：欧洲最大的照片礼品公司，在十几个国家运营八个不同品牌（albelli、Photobox、Hofmann、posterXXL 等）
- **业务数据**：年服务超过 700 万客户，2022 年交付超过 1000 万订单，拥有超过 11PB 的客户数据
- **合并挑战**：2021 年由 albelli 和 Photobox 合并而成，面临五个不同技术平台的整合难题
- **平台统一**：花费两年半时间将五个平台整合为一个统一平台，但在可观测性方面做出了妥协
- **业务风险**：黑色星期五期间每小时交易额达 150 万美元/欧元，平均故障解决时间 1.5 小时意味着超过 200 万欧元的潜在损失

### 35:00-45:00 可观测性三大支柱：文化变革
- **传统问题**：工程师习惯于通过查看日志来判断系统故障，每天需要人工关联分析 11 亿条日志记录
- **文化转型挑战**：需要让工程师认识到传统方法的局限性，而不是简单的自上而下强制改变
- **12 步可观测性项目**：建立由 5 名有影响力工程师组成的工作小组，让他们参与市场调研和成熟度模型构建
- **成熟度模型**：从 Level 1-2（监控到基础可观测性）提升到 Level 3-4（因果可观测性和主动可观测性）
- **北极星目标**：通过框架和方向指引，为团队提供清晰的发展路径

### 45:00-55:00 技术实施与数据统一
- **日志摄取架构**：
  - ECS 容器化工作负载通过 AWS FireLens 和 Fluent Bit 直接接入 Dynatrace Active Gate
  - Lambda 工作负载（占 40%）通过 CloudWatch Log Groups 订阅到 Kinesis Firehose，再通过 OpenPipeline 处理
  - 传统 EC2 工作负载通过现有 OTel 实现直接发送到 Dynatrace API
- **数据湖存储**：所有数据统一存储在 Dynatrace Grail 数据湖中，支持实时查询和长达 10 年的数据保留
- **信号优化**：从单纯依赖日志转向使用合适的信号类型（日志、指标、链路追踪），将日志量从 11 亿条降至数亿条
- **服务拓扑可视化**：首次实现分布式服务间调用关系的可视化，打破各服务孤岛状态

### 55:00-61:40 AI 驱动的自动化与业务价值
- **Davis AI 能力**：
  - 预测性 AI：自动基线设定，无需手动配置阈值和告警
  - 因果 AI：理解 200+ 微服务间的互联关系，自动构建服务依赖图
- **实际案例价值**：
  - 黑色星期五当天晚上 8 点，Davis AI 预测并阻止了三个 Severity 1 级别事件，避免了总计 450 万欧元的损失
  - DynamoDB 税率配置错误案例：20 分钟内自动定位到具体问题行并完成修复，团队仅接手该服务两周时间
- **根因分析增强**：Davis Copilot 提供自然语言问题解释、可执行的修复步骤建议，以及相关日志的精确定位
- **故障排除指南**：基于历史问题自动推荐相似故障的解决方案，减少重复性工作
- **预测性运维**：基于任何时间序列数据进行预测，包括日志数据，实现问题预防和自动修复
