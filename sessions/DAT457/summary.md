# AWS re:Invent 2025 Web3 代理技术会议总结

## 会议概述

本次技术会议由 Forrest 和 David 主讲，重点介绍如何使用 Amazon Bedrock AgentCore 构建 Web3 代理助手。会议采用代码演示的形式，展示了从基础代理到具备区块链交互能力的完整构建过程。

演讲者强调这是一个交互式技术会议，旨在通过实际代码演示来展示 AgentCore 的核心功能在 Web3 场景中的应用。他们构建的代理具备三个核心能力：获取区块链和市场数据、具备长期记忆功能、以及能够调用其他专门化代理。会议特别强调该代理定位为助手而非交易执行工具，不会自动执行交易或提供投资建议。

## 详细时间线与关键要点

### 0:00-10:00 会议介绍与需求分析
- Forrest 介绍会议目标和 Web3 代理的三个核心需求
- 展示 GitHub 仓库二维码，提供完整代码和工作坊指南
- David 介绍技术架构和 Docker 构建方法
- 强调使用 AgentCore runtime 的会话隔离特性，确保用户数据安全

### 10:00-20:00 基础代理构建
- 演示使用 UV 包管理器初始化 Python 项目
- 创建 AgentCore 内存配置，包括短期和长期记忆策略
- David 展示内存策略的实现，包括用户偏好存储和对话历史管理
- 介绍 Strands Agent 框架的基本结构和系统提示配置

### 20:00-30:00 Docker 部署流程
- 展示 Dockerfile 配置，使用 ARM64 架构和跨区域推理配置
- 演示 ECR 仓库创建和 Docker 镜像构建过程
- 配置 IAM 角色和权限策略，遵循最小权限原则
- 部署 AgentCore runtime 并获取运行时 ARN

### 30:00-40:00 代理交互演示
- 使用流式客户端与基础代理进行交互
- 演示记忆功能：代理记住用户姓名和加密货币偏好
- 回答观众关于内存命名空间、用户隔离和自定义策略的问题
- 展示如何动态更新用户偏好（添加 Solana 到投资组合关注列表）

### 40:00-50:00 浏览器工具集成
- 添加 AgentCore 浏览器工具以获取非结构化数据
- 更新 IAM 权限以支持浏览器功能
- 演示从 CoinMarketCap 获取实时市场数据
- 展示代理如何分析多种加密货币的市场表现和趋势

### 50:00-60:00 代理间通信与区块链交互
- 介绍 A2A（Agent-to-Agent）通信架构
- 演示与预部署的加密货币代理的集成
- 展示 KMS（密钥管理服务）在区块链交易签名中的应用
- 实际演示查询以太坊 Sepolia 测试网账户余额
- 解释 KMS 公钥到以太坊地址的映射计算过程
