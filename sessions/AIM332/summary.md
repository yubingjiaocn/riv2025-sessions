# AWS re:Invent 2025 会议总结：利用客户碳足迹工具推动可持续发展

## 一、会议概述

本次AWS re:Invent 2025分组会议（Session 332）聚焦于云计算可持续发展，由AWS可持续发展技术团队负责人Alexis Baitman主持，并邀请了Salesforce技术可持续发展总监Eric Zman和Adobe云端FinOps高级项目经理Ralph Charagla共同参与。会议核心主题是如何通过AWS客户碳足迹工具（Customer Carbon Footprint Tool, CCFT）帮助企业实现可持续发展目标。

会议展示了AWS在可持续发展方面的承诺，包括到2040年实现净零碳排放的目标，以及如何通过共享责任模型与客户合作推动整个行业的绿色转型。Salesforce和Adobe两家企业分享了各自的可持续发展战略，以及如何利用AWS提供的碳足迹数据来优化运营、降低排放并满足监管报告要求。

三位演讲者强调了数据透明度的重要性，展示了从数据收集、分析到报告的完整流程，以及如何将可持续发展目标与业务运营、成本优化和技术创新相结合。会议还涉及了生成式AI对可持续发展的影响，以及未来在碳核算、水资源管理等方面的发展方向。

## 二、详细时间线与关键要点

### **开场与背景介绍 (0:00-5:30)**
- **0:00** - 会议正式开始，主持人Alexis Baitman欢迎参会者
- **0:30** - 现场调查：询问参会者工作领域（可持续发展、运营、软件开发、FinOps等）
- **1:45** - 调查显示仅有3人了解客户碳足迹工具，会议目标是让所有人了解该工具
- **2:15** - 确认会议编号为Session 332，介绍演讲嘉宾团队
- **3:00** - Alexis介绍自己领导AWS可持续发展技术团队，负责净零碳、水资源正平衡、可再生能源等目标
- **3:30** - 介绍Salesforce的Eric（技术可持续发展总监）和Adobe的Ralph（云端FinOps高级项目经理）

### **会议议程说明 (5:30-7:00)**
- **5:30** - 概述会议流程：AWS可持续发展目标 → CCFT工具介绍 → Salesforce案例 → Adobe案例 → 生成式AI与可持续发展Q&A
- **6:15** - 强调会议将涵盖建设者、FinOps专业人士和运营人员的不同需求

### **AWS可持续发展战略 (7:00-15:00)**
- **7:00** - AWS是气候承诺签署方，承诺2040年实现净零碳排放
- **7:45** - AWS可持续发展三大支柱：
  - 无碳能源（太阳能、风能、核能）
  - 高效运营（数据中心效率优化）
  - 循环经济（2023年回收或二次销售2350万个组件）
- **9:00** - 水资源正平衡目标：2030年前实现，目前已完成53%
- **9:45** - AWS基础设施规模：120个可用区，覆盖38个地理区域，245个国家和地区
- **10:30** - 研究显示AWS运营效率比本地部署高4.1倍

### **共享责任模型 (15:00-18:30)**
- **15:00** - 介绍可持续发展的共享责任模型
- **15:45** - AWS负责：基础设施建设、服务器组件、冷却系统、废物处理、能源采购
- **16:30** - 客户负责：如何使用AWS服务、优化资源利用、选择合适的服务和区域
- **17:15** - 该模型是AWS Well-Architected框架可持续发展支柱的基础

### **客户反馈与工具演进 (18:30-24:00)**
- **18:30** - 2022年推出客户碳足迹工具（CCFT）
- **19:15** - 2024年开展客户调研，收集可持续发展领先客户的反馈
- **20:00** - 客户主要需求：更多透明度、运营可见性、方法论验证
- **20:45** - 2024年发布：各区域电力使用效率（PUE）评级
- **21:15** - 2024年发布：水资源使用效率（WUE）数据
- **21:45** - 2025年发布：经第三方验证的方法论、改进的数据导出功能
- **22:30** - 2025年新增：基于位置的排放数据（Location-based emissions）
- **23:15** - 最近发布：范围3排放数据（Scope 3 emissions），完善了完整的碳足迹画像
- **23:45** - 2026年计划：提升客户体验、增加数据粒度

### **碳核算范围说明 (24:00-27:30)**
- **24:00** - 解释碳核算的三个范围（类似财务会计）
- **24:45** - 范围1：直接运营排放（如备用发电机）
- **25:15** - 范围2：能源相关排放（电网供电来源）
- **25:45** - 范围3：供应链排放（硬件、组件、建筑材料、水泥等）
- **26:30** - 所有范围的排放数据现已包含在CCFT工具中

### **碳足迹计算方法 (27:30-32:00)**
- **27:30** - 数据来源：硬件组件、处理器、建筑材料、设备、能源消耗、电网来源、水资源使用
- **28:30** - 数据处理流程：
  - 摄取：从全公司收集数百万数据点
  - 编排：协调不同频率的数据（小时、日、周、月）
  - 转换：应用排放影响因子或科学建模
  - 治理：持久化存储并管理数据
- **30:00** - 自上而下的分配模型：区域 → 服务器机架 → 基础服务（EC2、S3）→ 非基础服务 → 客户账户
- **31:00** - 展示CCFT工具界面截图，可查看市场基础排放和位置基础排放

### **Salesforce可持续发展战略 (32:00-45:00)**
- **32:00** - Eric Zman开始介绍Salesforce的可持续发展之旅
- **32:45** - Salesforce可持续发展四大支柱：
  1. **排放减少**：2040年实现净零碳，范围1和2长期目标，范围3碳强度目标
  2. **能源转型**：2021年起实现100%可再生能源（范围1和2）
  3. **自然正向**：相信商业健康依赖自然健康
  4. **利益相关者参与**：客户、监管机构、投资者
- **35:30** - 经济价值：世界经济论坛研究显示，到2030年自然正向政策可创造10万亿美元商业价值和4亿个工作岗位
- **36:45** - 水资源关注：数据中心和电网的水资源韧性，投资上游流域
- **37:30** - 1亿美元自然与可持续发展基金，支持"一万亿棵树"计划（1t.org）
- **38:15** - 支持生态创业者和小型使命驱动型公司

### **Salesforce基础设施可持续发展项目 (39:00-45:00)**
- **39:00** - 基础设施可持续发展重点领域：
  - 向公共云过渡（从托管数据中心快速部署到AWS）
  - 优化利用率（避免盲目扩容）
  - 成本驱动优化（FinOps与碳核算协同）
- **40:30** - 具体优化措施：
  - 选择合适的实例类型
  - 使用S3智能分层
  - 升级到EBS GP3
  - 暂停或释放按需容量预留（ODCR）
- **41:45** - 客户部署效率：帮助客户更高效部署，基准对比相似客户
- **42:30** - 区域选择考虑：碳强度和水资源压力
- **43:00** - 提供客户选项：从托管迁移到清洁区域，或在AWS区域间迁移
- **43:45** - 软件代码优化：开发更高效、优雅的代码，避免反模式（包括人工和AI生成的代码）
- **44:30** - 跟踪水资源、电子废物，设定性能预期，平衡碳和水的权衡

### **Salesforce数据需求与应用 (45:00-50:00)**
- **45:00** - 强调不同类型数据服务于不同目的
- **45:45** - 数据类型需求：
  - 基于位置的电网排放
  - 基于市场的排放
  - 范围1、2、3排放
  - 按主付款账户、子账户、区域、服务级别细分
  - 水资源提取和消耗数据
- **47:00** - 数据应用的四大类别：
  1. **足迹核算**：跟踪历史数据，内部追踪目标进度，外部报告（利益相关者影响报告、CDP、10-K文件）
  2. **预测与趋势设定**：基于历史数据预测未来，追踪科学基础目标（2040年）
  3. **内部分析**：按业务单元分析（CRM、营销、商务、Slack、Tableau、Mulesoft等），识别模式和机会
  4. **利益相关者参与**：客户询问其使用的碳足迹，供应商参与，行业基准对比

### **Adobe可持续发展战略 (50:00-60:00)**
- **50:00** - Ralph Charagla开始介绍Adobe的可持续发展之旅
- **50:45** - Adobe理念：可持续发展不是目的地而是旅程，融入企业DNA
- **51:30** - Adobe可持续发展三大支柱：
  1. **产品**：为客户开发数字工作流程
  2. **运营可持续性**：优化内外部运营效率
  3. **倡导**：与合作伙伴和客户协作
- **52:45** - Adobe五大指导原则：
  1. 通过产品赋能客户（Adobe Acrobat节省数十亿张纸，Adobe Sign减少物流碳排放）
  2. 为员工建设可持续创新工作空间（80%设施获得LEED认证）
  3. 提高数据中心生产力，减少范围1和2排放（降低PUE）
  4. 员工参与（绿色团队，90%废物回收）
  5. 透明度（年度可持续发展报告，包括道德AI和数据透明度）

### **Adobe承诺与里程碑 (55:00-60:00)**
- **55:00** - 北极星目标：2050年实现净零碳
- **55:45** - 关键里程碑：
  - 2030年前运营排放（范围1和2）减少42%
  - 2030年前供应链排放（范围3）减少52%
- **56:30** - 四大运营里程碑：
  - 2025年前实现100%可再生能源电力
  - 90%全球废物回收转化率
  - 80%建筑保持LEED认证
  - 减少25%水资源消耗（数据中心和人均）
- **57:45** - 强调可持续发展是共享责任，需要合作伙伴关系

### **Adobe数据驱动方法 (58:00-结束)**
- **58:00** - 三个关键要素：
  1. **数据与洞察**：从数据中获取可操作的见解
  2. **报告**：多种报告类型（可持续发展披露、内部深度分析、产品碳足迹）
  3. **行动**：基于数据和报告采取优化措施
- **59:00** - 会议准备进入Q&A环节，讨论生成式AI与可持续发展的关系
