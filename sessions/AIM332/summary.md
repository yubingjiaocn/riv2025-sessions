# AWS re:Invent 2025 可持续发展会议总结

## 会议概述

本次会议是AWS re:Invent 2025的第332号分组讨论会，主题为"通过客户碳足迹工具实现可持续发展目标"。会议由AWS可持续发展技术团队负责人Alexis Bateman主持，并邀请了Salesforce技术可持续发展总监Eric Gertsman和Adobe云端FinOps高级项目经理Rao Chargondla作为嘉宾分享各自企业的可持续发展实践。

会议重点介绍了AWS的可持续发展承诺，包括2040年实现净零碳排放目标，以及客户碳足迹工具(CCFT)的发展历程和功能。通过与企业客户的合作案例，展示了如何通过数据透明度和共同责任模型推动整个行业的可持续发展进程。会议还探讨了生成式AI对可持续发展的双重影响，以及未来在数据粒度和可访问性方面的发展方向。

## 详细时间线与关键要点

### 0:00-5:00 开场与观众调研
- 主持人Alexis Bateman进行观众背景调研
- 询问参会者在可持续发展、运营、软件开发和FinOps方面的工作经验
- 调查对客户碳足迹工具的了解程度，仅有3人举手表示了解

### 5:00-15:00 AWS可持续发展目标介绍
- AWS作为气候承诺签署方，承诺2040年实现净零碳排放
- 三大支柱战略：
  - 无碳能源：通过太阳能、风能、核能等实现可再生能源目标
  - 高效运营：优化数据中心运营效率
  - 循环经济：2023年以来回收或二次销售超过2350万个组件
- 水资源正效益目标：2030年实现水资源正效益，目前已完成53%
- 覆盖120个可用区、245个国家和地区、338个地理区域
- 相比本地部署，AWS运营效率提高4.1倍

### 15:00-25:00 共同责任模型与CCFT发展历程
- 介绍可持续发展的共同责任模型概念
- 2022年推出客户碳足迹工具
- 2024年通过客户调研收集反馈，发布功耗使用效率(PUE)和水资源使用效率(WUE)数据
- 2025年发布经第三方验证的方法论，增加基于位置的排放数据和Scope 3排放数据
- 计划2026年提升客户体验和数据粒度

### 25:00-30:00 碳排放范围与计算方法
- 详细解释Scope 1、2、3排放范围：
  - Scope 1：直接运营排放（如备用发电机）
  - Scope 2：电网能源排放
  - Scope 3：供应链排放（硬件、建筑材料等）
- 介绍CCFT的数据处理流程：数据摄取、编排、排放因子应用、数据治理
- 展示自顶向下的碳排放分配模型：从区域到服务器机架，再到基础服务和客户账户

### 30:00-40:00 Salesforce可持续发展实践
- Eric Gertsman介绍Salesforce四大可持续发展支柱：
  - 减排目标：2040年实现净零排放
  - 能源转型：2021年已实现100%可再生能源
  - 自然正效益：投资1亿美元自然与可持续发展基金
  - 基础设施可持续发展项目
- 重点关注向公共云迁移、优化资源利用、成本驱动优化
- 强调数据透明度的重要性，包括基于位置和基于市场的排放数据需求

### 40:00-45:00 Adobe可持续发展战略
- Rao Chargondla介绍Adobe三大支柱：产品、运营可持续性、协作倡导
- 五大指导原则：
  - 通过产品赋能客户（如Adobe Acrobat节省纸张）
  - 建设可持续创新工作空间（80%设施获得LEED认证）
  - 提升数据中心生产力
  - 员工参与（绿色团队实现90%废物回收）
  - 透明度报告
- 承诺2050年实现净零排放，2030年运营排放减少42%，供应链排放减少52%

### 45:00-50:00 生成式AI与未来展望
- 讨论生成式AI对可持续发展的双重影响：
  - 机遇：数据洞察、ESG披露、基础设施智能自动化
  - 挑战：GPU能耗增加、电子废物、碳足迹增长
- 未来合作需求：
  - 提高数据可访问性和粒度
  - 实现月度数据可用性
  - 产品级碳足迹分配
  - 更好的API和实时数据
- 强调可持续发展是持续的商业价值创造过程，不仅仅是环保行为
