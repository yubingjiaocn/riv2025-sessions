# AWS SAP 转型会议总结

## 会议概述

本次会议由 AWS 的 Solid Khan 主持,邀请了三位来自不同行业领先企业的嘉宾分享他们在 AWS 上运行 SAP 工作负载的转型经验。三位嘉宾分别来自亿滋国际(Mondelēz International)、高露洁棕榄(Colgate-Palmolive)和澳大利亚联邦银行(Commonwealth Bank of Australia)。

这些企业都面临着 SAP ECC 2027 年终止服务的挑战,需要迁移到 S4/HANA。他们选择 AWS 作为云平台合作伙伴,不仅是为了完成技术迁移,更是为了实现业务转型和创新。会议重点讨论了选择 AWS 的原因、迁移过程中的经验教训、以及如何利用 AWS 服务实现现代化和创新。

每位嘉宾都强调了 AWS 在规模、性能、弹性和创新方面的优势,以及 AWS 团队提供的深度技术支持和合作伙伴关系。他们的成功案例展示了即使是最关键、最复杂的 SAP 工作负载也可以安全、高效地在 AWS 上运行,并通过云原生服务实现业务价值的最大化。

### 选择 AWS 的原因

时间戳: 嘉宾介绍后
Simon Davies (联邦银行):
- 系统处理澳大利亚 40% 的经济交易,是国家经济的支柱
- 核心银行转型项目 70% 失败率,决策压力巨大
- 选择 AWS 的三大原因:
  1. **规模和弹性** - AWS 理解关键工作负载的责任,有运行大型 SAP 系统的经验
  2. **工程合作伙伴关系** - 需要为高吞吐量在线交易处理系统解决复杂问题,AWS 协助开发了 EC2 新功能
  3. **生态系统和创新** - 不仅是技术迁移,更是运营模式转型,从 8 个孤立团队转向统一的全栈方法

时间戳: Simon 发言后
Pete Rockenstein (亿滋国际):
- 公司成立仅 16 年,由 Kraft 北美和 Cadbury 业务合并而成
- 系统分散在不同数据中心和云平台(2017 年开始云迁移)
- 4 年前建立内部云工程组织,采用基础设施即代码(IaC)和 Terraform 账户工厂
- 所有 AWS 部署都通过 IaC 完成,可在数小时内从需求到实现
- 老板曾在儿子足球比赛现场用手机部署服务器
- 正在进行 S4/HANA 和 Rise 迁移,同时解决历史遗留的系统异构问题
- 团队拥有 425 年的综合经验,AWS 帮助降低风险并加速交付

时间戳: Pete 发言后
Punachan Naropi (高露洁棕榄):
- 250 年历史的公司,服务 200 个国家和地区
- 已有 6 个生产环境在 AWS Rise 上运行,计划 2027 年中期前迁移另外 8 个系统
- 选择 AWS 的关键因素:
  1. **性能** - 通过基准测试验证 AWS 满足或超越现有 IBM 系统性能
  2. **SAP 路线图对齐** - AWS 与 SAP 的紧密合作提供信心
  3. **云多元化战略** - 符合公司战略
- 2019 年就已迁移到 S4/HANA,业务案例包含云迁移

### AWS 项目和经验教训

时间戳: 选择原因讨论后
Pete Rockenstein:
- **AWS 专业服务** - 从第一天起就提供支持,每周办公时间持续 4 年
- **re:Invent 合作伙伴关系** - 4 年前在 re:Invent 遇到 Lemongrass,建立长期合作
- 寻找在系统工程、自动化和 IaC 方面理念一致的合作伙伴
- 专注于规模化、可重复性和自动化,让团队专注于转型价值

时间戳: Pete 分享后
Simon Davies (量化改进):
- **性能提升** - 从 DB2 主机迁移到 S4/AWS 后性能提升 30%
  - Web 服务响应时间改善,余额更新实时显示
  - 支持实时欺诈检测
  - 批处理作业更快,支持基于客户的定价(每天数百万次计算)
- **弹性改进** - RTO 从 1.5 小时降至 16 分钟
  - 与 SAP、Red Hat 和 AWS 深度工程合作
  - 优化 Pacemaker 配置和 Red Hat 内核
  - 开发 EC2 跳过关机功能,现已向所有客户开放
- **效率提升** - 基础设施成本降低 30%
  - 从专用计算转向 x86 商品硬件
  - 全栈自动化(配置、扩展、重启、故障转移等)
  - 每月减少数百个手动操作点
  - 团队从"保持运行"转向"构建未来",专注 AI 运维

时间戳: Simon 量化分享后
Punachan Naropi:
- 2025 年第一季度开始 AWS 之旅,8 个月内迁移 6 个生产环境
- 使用 AWS Immersion Days 和 Well-Architected Framework 进行团队赋能
- **连接性研讨会** - AWS 团队现场研讨,讨论连接模式,快速部署 Direct Connect
- **定期支持** - 即使使用 Rise,仍与 AWS 团队定期通话
- **安全态势** - AWS 团队提供多次安全会议,帮助建立 AWS 安全架构
- **概念验证** - AWS 团队能够用客户语言沟通,理解企业特定需求

### 现代化和创新

时间戳: 经验教训后
Simon Davies (核心银行创新):
- **AI 辅助开发** - 使用 Amazon Q、Cursor、Klein、GitHub Copilot 进行代码修复和测试
  - 当前工作流程笨拙(VS Code 到 Eclipse 复制粘贴)
  - 期待改进,最近 AWS 宣布完全支持 ABAP 和 Amazon Q Developer Eclipse 插件
- **实时数据流** - 系统包含 90% 的客户和交易数据
  - 从 HANA 到 Amazon MSK 实时流式传输,每秒 5000 笔交易
  - 通过 Kinesis 到 SageMaker 和 Bedrock
  - 为下一个十年的代理银行奠定基础
  - 使用 Bedrock 和 Agent Core 实验超个性化行为银行体验

时间戳: Simon 创新分享后
Punachan Naropi:
- AWS 作为 SAP Rise 的核心平台
- 使用 AWS 补充服务:存储、计算、NetApp、打印服务
- 逐步将组件从本地迁移到云
- **数据收集** - 收集网络、存储、延迟、出入站数据传输指标
- 评估哪些组件应更靠近 AWS,计划将部分系统迁移到自有 AWS 账户
- **品牌网站** - 多个品牌网站已在 AWS 上运行,提供全球覆盖和强大性能
- "AWS 无处不在,很难避开 AWS"
