# AWS re:Invent 2025 Metronome 会议总结

## 会议概述

本次会议由 Metronome 解决方案工程师 Austin 主讲，主题为"为什么基于使用量的计费必须被视为基础设施"。这是 Metronome 首次参加 AWS re:Invent，会议重点讨论了现代软件行业从传统订阅模式向基于使用量计费模式的转变，特别是在 AI 时代背景下的实时货币化需求。

Metronome 作为一家专注于基于使用量计费的基础设施公司，为包括 OpenAI、Anthropic、Nvidia、Confluent 等知名 AI 和软件公司提供服务。Austin 强调了在当前代理式工作流和混合定价模型时代，实时货币化已成为产品体验的核心组成部分，而不再是后台办公室的简单账单处理。

## 详细时间线与关键要点

### 0:00-2:30 - 开场介绍与背景
• Austin 介绍自己作为 Metronome 解决方案工程师的身份
• 强调这是 Metronome 首次参加 AWS re:Invent
• 介绍 Metronome 服务的知名客户：OpenAI、Confluent、Anthropic、Nvidia、LangChain
• 提出核心议题：为什么基于使用量的计费现代化必须成为基础设施

### 2:30-5:00 - 软件定价模式的演进
• 回顾软件行业定价模式发展：从许可证销售到订阅模式，再到当前的代理式结果工作流
• 强调新定价模型几乎每天都在被发明
• 讨论混合定价模型的兴起：从纯席位制到席位加配额制
• 指出 LLM 的纯使用量特性使传统订阅模式难以匹配价值

### 5:00-7:30 - 实时货币化的重要性
• 对比传统计费（批处理、月度发票、后向查看）与现代实时货币化
• 强调货币化现在包括：实时工作流通知、访问控制、产品体验核心
• 以 ChatGPT 的 token 概念为例说明货币化与产品体验的深度融合
• 将货币化定义为产品开发和客户匹配的持续飞轮

### 7:30-10:00 - Metronome 的事件驱动方法
• 介绍 Metronome 的核心概念：业务事件（API 调用、推理、服务器启停等）
• 强调实时属性和门控访问的重要性
• 讨论超额使用通知和追加销售机会的实时识别
• 说明为什么这是基础设施问题：发票和收入数据必须实时理解

### 10:00-12:00 - 技术挑战与解决方案
• 分析传统事务可观察性的"三选二"问题：正确性、低延迟、高可用性
• 详细解释每个维度的重要性：
  - 正确性：财务审计要求、事件时序处理
  - 低延迟：实时服务警报、欺诈检测、PLG 产品需求
  - 高可用性：计费系统永不宕机的要求
• 强调 Metronome 实现了全部三个维度

### 12:00-14:00 - 基础设施原则与案例研究
• 介绍应用于货币化的四大基础设施原则：可扩展性、模块化、编排能力、可扩展性
• 详细说明每个原则的实际应用
• 展示 Metronome 平台架构：从使用事件到 CRM、计费门户和数据湖的完整流程
• 介绍 OpenAI 案例研究：从"OpenAPI"小研究实验室到现在的规模

### 14:00-14:42 - 结尾与推广
• 强调 AI 时代的准备：85% 软件公司采用基于使用量的定价
• 介绍货币化工程师这一新兴职位
