# AWS re:Invent 2025 - 使用生成式AI和LLM代理加速游戏设计评审

## 会议概述

本次技术分享会由AWS解决方案架构师Sam Patzer和Christina主讲，主题为"使用生成式AI和LLM代理加速游戏设计评审"（IND395）。会议重点介绍了如何构建AI代理系统来自动化游戏分析师的工作流程，通过多个专业化代理协同工作来评估游戏设计文档。

演讲者展示了一个完整的解决方案，该方案使用AWS的Strands开源框架和Amazon Bedrock AgentCore来构建生产就绪的AI代理。系统包含四个专业化代理（背景设定、质量保证、游戏玩法、策略）和一个协调器代理，能够从不同角度评估游戏设计提案，并提供客观、无偏见的反馈。

## 详细时间线与关键要点

### 00:00-05:00 开场介绍与背景
- Sam和Christina自我介绍，确认听众背景
- 介绍会议议程：游戏分析师角色、构建概览、工具介绍、现场编码演示
- 解释现代游戏分析师的作用：既是解释者又是故事讲述者，负责将业务目标和玩家目标转化为可执行的故事

### 05:00-10:00 游戏分析师的四大核心能力
- **背景设定（Lore）**：验证更新是否符合游戏世界观
- **质量保证（QA）**：确保更新不会破坏游戏平衡
- **游戏设计**：评估更新是否有趣，能否吸引玩家持续参与
- **游戏策略**：判断是否符合业务需求和玩家留存模型
- 讨论当前挑战：评审周期长、信息不完整、培训成本高、主观偏见

### 10:00-15:00 解决方案架构概述
- 构建四个专业化代理，每个代理专注于一个核心能力
- 连接不同的知识库，包含详细的游戏背景和玩法信息
- 使用协调器代理（游戏分析师代理）整合所有反馈
- 优势：加速创新周期、协作智能、消除偏见

### 15:00-25:00 技术工具栈介绍
- **Strands框架**：AWS开源Python SDK，专为构建AI代理设计
- **Amazon Bedrock AgentCore**：生产就绪的代理平台，包含运行时、内存管理、可观测性
- **系统架构**：五个代理（四个专业化+一个协调器），使用Anthropic Claude Sonnet 4.5模型
- **数据源**：Amazon S3存储设计文档，Amazon Redshift存储玩家行为数据
- **RAG能力**：通过Amazon Bedrock Knowledge Bases提供向量数据存储

### 25:00-35:00 现场演示：项目门户和代理评估
- 演示创建新项目：为《新世界》游戏添加精灵种族
- 展示设计文档内容：角色特征、文化背景、种族能力等
- 配置代理选项，禁用内存功能进行基础测试
- 展示三个代理的不同评估结果：玩法代理发现机制问题，背景设定代理指出与游戏世界观不符，策略代理认为有商业价值

### 35:00-45:00 代码实现详解
- 使用Kiro IDE演示Strands代理的三个核心组件：工具、模型、系统提示
- 展示MCP服务器集成，用于Bedrock Knowledge Base检索
- 详细解释系统提示的重要性：定义代理个性、角色和指令
- 演示如何部署到Amazon Bedrock AgentCore Runtime
- 展示策略代理代码，说明不同代理间的相似性

### 45:00-51:00 内存集成和性能优化
- 演示启用内存功能后的性能改进
- 对比分析：响应时间减少80秒以上，token使用量显著降低
- 解释两级内存架构：项目级语义事实存储，用户级偏好和会话记忆
- 展示Amazon Bedrock Prompt Manager工具，自动优化提示词
- 总结三大关键要点：AgentCore简化生产就绪代理开发、内存增强性能和用户体验、将主观反馈转化为客观洞察
