# AWS re:Invent 2025 - AWS Transform 代理式 AI 服务会议总结

## 会议概述

本次会议由 AWS 高级解决方案架构师 Lakshmi Dasuri 和 Val Paidi 主讲,重点介绍了 AWS Transform——AWS 首个代理式 AI 服务,专门用于加速云迁移和现代化进程。演讲者强调,虽然 80% 的企业将在 2026 年前部署生成式 AI 应用,但目前仍有 70% 的工作负载运行在传统数据中心,超过 70% 的遗留 IT 软件已有 20 多年历史。这些技术债务难以维护和现代化,成为企业数字化转型的主要障碍。

AWS Transform 通过代理式 AI 技术,能够将传统需要 12-18 个月的迁移周期缩短 4 倍。该服务基于 AWS 19 年的迁移专业知识和数千次迁移经验训练的专业代理,支持 VMware、大型机、Windows 等多种技术栈的端到端迁移。服务采用统一的 Web 体验界面,提供丰富的聊天功能和团队协作能力,并在自主性和人工监督之间保持适当平衡。会议还分享了新西兰能源公司 Vector Limited 的成功案例,该公司通过 AWS Transform 实现了 34% 的迁移时间缩短和未来 5 年 35% 的总体拥有成本节省。

## 详细时间线

00:00 - 开场介绍
- 演讲者 Lakshmi Dasuri 分享个人经历,使用 Kiro(AWS 代理式 AI IDE)为孩子创建个性化旅行活动手册
- 介绍代理式 AI 如何提升个人和企业生产力

02:30 - 行业现状与挑战
- 数据点:80% 的企业将在 2026 年前部署生成式 AI 应用(两年前这一比例不到 5%)
- 到 2028 年,超过 15% 的工作决策将由代理式 AI 自动化(从 0% 起步)
- 技术债务现状:70% 工作负载仍在传统数据中心,70% 遗留软件超过 20 年历史

05:00 - 云迁移方法论
- 介绍典型云采用旅程的三个阶段:评估、规划动员、迁移和现代化
- 评估阶段:创建业务案例、深度发现 IT 状态、确定迁移策略
- 规划阶段:构建迁移计划、确定服务器迁移波次、建立 AWS 着陆区架构
- 强调传统迁移流程需要 18 个月以上

08:00 - AWS Transform 服务介绍
- Val Paidi 接手演讲,强调 AWS Transform 可将迁移速度提升 4 倍
- 核心特性:专业代理(VMware、大型机、Windows 等)、端到端迁移能力、统一 Web 体验
- 基于 AWS 19 年迁移专业知识和数千次迁移经验训练
- 提供自主性和人工监督的适当平衡

10:00 - 产品演示开始
- 展示 AWS Transform 工作区界面
- 创建迁移作业:选择 VMware 端到端迁移
- 系统自动生成迁移路线图,包含三个阶段:发现与规划、基础设施设置、迁移与现代化

12:00 - 库存导入与发现
- 演示导入库存功能:支持发现代理、RV 工具导出、CMDB 等多种数据源
- 展示 48 台服务器的库存预览和网络连接分解

14:00 - 迁移计划构建
- 演示使用聊天功能自定义迁移计划
- 使用 "@" 符号针对特定任务提供指令(如 @build migration plan)
- 系统根据 VM 信息自动创建自定义波次计划
- 展示四个波次计划,建议顺序执行

16:00 - 目标账户连接
- 介绍 Transform 连接器功能
- 演示如何将连接器输入到目标 AWS 账户

17:30 - 网络迁移
- 强调网络迁移是简化复杂源网络拓扑的关键环节
- 支持 RV 工具、NSX、Cisco、Palo Alto、Fortigate 等配置文件
- 提供两种网络拓扑选项:Hub-and-Spoke 或隔离 VPC
- 演示 CIDR 编辑和自定义功能

19:00 - 审批流程
- 展示内置审批功能,支持基于角色的访问控制(RBAC)
- 允许不同团队协作处理不同迁移计划
- 审批后自动部署 VPC 到目标应用程序

20:30 - 执行迁移计划
- 底层使用 MGN(Application Migration Service)连接器
- 流程包括:安装复制代理、持续复制、测试环境、最终切换

22:00 - 客户成功案例
- Vector Limited(新西兰能源公司)案例
- 从 VMware Cloud on AWS 迁移到原生 AWS 架构
- 成果:迁移时间缩短 34%,未来 5 年 TCO 节省 35%
- 波次规划自动化是加速迁移的关键因素

24:00 - AWS Well-Architected Framework
- Lakshmi 介绍迁移后的持续优化工具
- Well-Architected Tool:AWS 控制台中的自助服务工具
- 六大支柱评估框架,包含 57 个问题
- 工作流程:定义工作负载 → 选择支柱 → 回答问题 → 获取风险报告 → 实施改进

27:00 - 总结与行动号召

- 推荐参加 EBA(Experience-Based Acceleration)项目
- 鼓励填写会议调查问卷
