# AWS re:Invent 2025 会议总结：Tempo Labs 的云迁移与 AI 创新之旅

## 一、会议概述

本次 AWS re:Invent 2025 分会场聚焦于 Tempo Labs 的客户案例，展示了这家加拿大快速成长的初创公司如何利用 AWS 服务实现云迁移和生成式 AI 创新。会议由 AWS 全球迁移技术负责人 Vidi 主持，Tempo Labs 联合创始人兼 CEO Kevin 和增长负责人 Brandon 分享了他们的实践经验。

Tempo Labs 是一个革命性的设计工程平台，它将可视化设计工具与生产级代码库无缝集成，使设计师和产品经理能够直接编写和提交生产级代码。该公司于 2024 年 12 月发布产品后，迅速获得超过 16 万用户，月增长率达到 22%。通过迁移到 AWS 和采用 Amazon Bedrock，Tempo 不仅解决了成本和扩展性挑战，还将云基础设施成本降低了 5 倍，同时实现了团队交付速度提升 40%。

会议深入探讨了 Tempo 从另一个云服务商迁移到 AWS 的完整历程，包括技术架构现代化、LLM 推理成本优化、以及与 AWS 团队共同开发的创新 PRD Agent 项目。这个案例充分展示了 AWS 不仅提供技术资源，更作为战略合作伙伴深度参与客户的创新和成长。

## 二、详细时间线与关键要点

### **00:00:00 - 会议开场与背景介绍**
- Vidi 介绍自己作为 AWS 全球迁移技术负责人的身份
- 说明本次会议是客户聚焦会议，将展示 Tempo Labs 的创新和迁移历程
- 强调会议将涵盖 Amazon Bedrock 创新、云迁移和持续现代化三个主题

### **00:02:30 - 迁移与现代化趋势演变**
- Vidi 回顾了大规模迁移的驱动因素演变：从最初的成本和运营效率，到如今的业务转型和客户体验改善
- 强调董事会现在期望的不仅是成本节省，更要求数字化转型和现代化成果
- 企业希望将成本节省再投资于创新，特别是通过生成式 AI 实现 10 倍速度和敏捷性提升

### **00:04:00 - 生成式 AI 发展历程**
- 2023 年：探索概念和基础，了解生成式 AI 的潜力
- 2024 年：进入实施阶段，提出正确的问题并实现实用性
- 2025 年：将 AI 推向生产环境，使其成为业务卓越的核心要素

### **00:05:00 - AWS Transform 案例**
- AWS 自身使用生成式 AI 升级遗留 Java 技术栈的例子
- 2023 年探索低接触升级能力，2024 年实现产品化，每小时可升级 Java 应用
- 2025 年推出 AWS Transform 代理服务，支持 Java、.NET 和大型机现代化
- 已节省 80 万小时开发时间，处理了 10 亿行代码

### **00:07:00 - Tempo Labs 公司介绍**
- Brandon 介绍 Tempo 是加拿大增长最快的初创公司之一
- Tempo 是下一代设计工程平台，能将规格转化为生产就绪的应用程序
- 展示了充满活力的产品介绍视频

### **00:09:30 - Tempo 产品核心功能**
- Tempo 是面向设计工程的最佳设计工具，类似 Figma 和 IDE 的结合体
- 在生产级 React 代码库之上运行，完全消除设计师、PM 和开发团队之间的传统交接
- 使用 Tempo 的团队平均加速 40% 的功能交付速度

### **00:11:00 - Tempo 公司数据**
- 2024 年 12 月发布产品
- 超过 16 万用户使用产品
- 月增长率约 22%
- Y Combinator 孵化公司
- 创始团队曾创立 Perpeta 公司并以 1.5 亿美元出售

### **00:13:00 - Tempo 与传统 Vibe Coding 工具的区别**
- 强调 Tempo 不是另一个"氛围编码"工具
- 不仅仅是构建表面级的 SaaS 仪表板
- 是设计师、开发者和 PM 真正协作并交付生产级代码的解决方案

### **00:15:00 - 现场产品演示开始**
- Kevin 进行高风险的现场演示（没有录制备用视频）
- 展示 Tempo 的两个关键差异化特性：与生产代码库集成、为设计师提供熟悉的工具

### **00:17:00 - 演示：云端协作开发环境**
- 展示 Tempo 运行在云端 Docker 容器中的实时 React 应用
- 类似 Figma 的协作体验，多人可同时编辑同一代码库
- 展示从 GitHub 一键导入现有代码库的功能

### **00:19:00 - 演示：从组件市场添加功能**
- 访问 React Components 市场（Tempo 旗下另一个产品）
- 复制粘贴 testimonials 组件到 Airbnb 示例代码库
- 像 Figma 一样拖放组件，Tempo 自动在正确位置添加代码

### **00:21:00 - 演示：手动精确控制**
- 展示设计师如何手动控制每个像素
- 编辑文案、调整内边距、修改颜色为琥珀色、添加模糊效果
- 所有手动更改都自动反映在底层代码中，无需使用 AI 或提示词

### **00:23:00 - 演示：代码提交流程**
- 所有代码更改自动暂存，可直接推送到 GitHub
- 或者分享链接给工程师继续完善
- 约 60-70% 的 PM 和设计师生成的代码可被前端团队直接重用

### **00:25:00 - 演示：Storybook 集成**
- 展示与企业设计系统的深度集成
- Storybook 中的每个 story 都可以像 Figma 资源一样拖放使用
- 演示添加动画标签组件的流程

### **00:27:00 - 演示：Tempo on Tempo（递归演示）**
- 展示 Tempo 编辑器在自身内部递归运行
- 证明 Tempo 可以处理复杂的企业级代码库
- Tempo 团队自己使用 Tempo 为产品贡献代码

### **00:29:00 - AWS 迁移背景**
- Kevin 解释 Tempo 需要云端开发环境来实现类似 Figma 的协作体验
- 最初在 GCP 上构建，因为之前的公司使用 GCP（主要为 BigQuery）
- 早期采用"不可扩展"的方法：使用单个巨型机器托管所有容器

### **00:31:00 - 迁移危机时刻**
- 2024 年 3 月产品走红后，基础设施成本失控
- 使用 5 台巨型机器在用户之间路由，成本极高且不可扩展
- 2024 年 3-4 月收到 25 万美元的 LLM 账单，而当月收入远低于此
- 大部分 LLM 支出用于 Anthropic 模型

### **00:33:00 - 选择 AWS Bedrock 的原因**
- Anthropic 与 AWS 有深度合作关系
- 面临成本和速率限制两大挑战
- 作为 YC 公司获得各云服务商 25-35 万美元积分
- Bedrock 是唯一允许将积分用于 Anthropic LLM 推理的平台

### **00:35:00 - Bedrock 迁移的技术优势**
- Anthropic 是企业 API 用例的最佳 LLM
- AWS 团队提供出色的速率限制支持
- 新模型发布时确保平滑过渡和充足的速率限制
- 快速响应技术需求（如 prompt caching API 支持）

### **00:37:00 - Prompt Caching 支持案例**
- Tempo 使用 Vercel AI SDK 开源框架
- Bedrock API 最初不支持 Vercel AI SDK 所需的 prompt caching API
- 联系 Amazon 后几天内就解决了问题
- Prompt caching 可显著降低 LLM 推理成本

### **00:39:00 - Brandon 谈 AWS 合作伙伴关系价值**
- 从初创公司视角看，25 万美元 LLM 账单是巨大压力
- 云计算已成为商品，但 AWS 的差异在于真正的合作伙伴关系
- AWS 不仅提供积分和资源，更投资于 Tempo 的未来和技术
- 晚上 10 点发消息给 AWS 代表，当晚就能得到回复

### **00:41:00 - Go-to-Market 支持**
- AWS 启动团队在市场拓展方面提供重要支持
- 帮助 Tempo 向企业级客户发展
- 提供 AWS Marketplace 访问权限
- 促成与四大咨询公司之一的大规模部署合作

### **00:43:00 - 基础设施现代化项目**
- AWS 通过迁移计划投资帮助 Tempo 现代化架构
- 不仅提供迁移激励，还投入资源帮助构建可扩展架构
- 从路由巨型机器转向基于 Kubernetes 的容器化基础设施

### **00:45:00 - OpsGuru 合作伙伴项目**
- AWS 投资 OpsGuru 合作伙伴帮助 Tempo 完成迁移
- 6-8 周内完成整个项目
- 实现自动扩展、自愈、Pod 级权限和安全性、高可用性
- OpsGuru 编写的代码，Tempo 团队现在完全拥有和维护

### **00:47:00 - 成本降低 5 倍**
- 月度云基础设施成本从 2.5 万美元降至 5000 美元
- 同时提升了可扩展性、安全性和可用性
- 选择手动构建基础设施而非使用 Fly.io 等托管解决方案
- 虽然需要更多投入和维护，但已看到独特优势

### **00:49:00 - 自定义基础设施的优势**
- 在 Pod 上运行 TypeScript 语言服务器和 Tailwind 语言服务器
- 为设计师提供类似 Figma 的所见即所得属性编辑体验
- 使用语言服务器提供 IntelliSense 功能支持 React 组件属性编辑
- 这些功能在完全托管的解决方案中难以实现

### **00:51:00 - PRD Agent 创新项目介绍**
- AWS 与 Tempo 共同开发的 R&D 项目
- 超越迁移，专注于创新
- 双方共同撰写研究论文
- 针对产品经理的新型 AI 代理

### **00:53:00 - PRD Agent 的工作原理**
- 每个产品专家脑中都有产品知识图谱
- 包括功能、范围、限制、支持的和不支持的内容
- 大多数组织没有将这些知识以 LLM 可解析的方式记录下来
- 知识存在于代码中，但被实现细节的噪音掩盖

### **00:55:00 - PRD Agent 的技术实现**
- 对代码库进行大量 LLM 预处理
- 从代码库中提取和解析每个 PRD（产品需求文档）
- 测试大型开源代码库，输出 15-20 个详细 PRD
- 在正确的抽象层次解释已实现的功能和限制

### **00:57:00 - PRD Agent 的应用价值**
- 输出作为 PM 代理的上下文
- 代理了解产品的所有信息，如同从一开始就在的资深 PM
- 加速 AI 辅助 PRD 编写的细微度和任务类型
- 计划 2026 年初发布，欢迎参与 Beta 测试

### **00:59:00 - AWS GenAI IC 团队的深度参与**
- AWS R&D 团队的深度远超 Tempo 作为初创公司能达到的水平
- AWS 不仅帮助实现项目，还将其推向更远
- 展示了真正的技术合作伙伴关系

### **01:01:00 - Vidi 总结与 MAP 计划介绍**
- 希望每个客户都能有类似 Tempo 的旅程
- 介绍迁移加速计划（Migration Acceleration Program, MAP）
- AWS 拥有 19 年以上的迁移经验
- 提供加速器帮助客户开始迁移、持续现代化和大规模创新之旅
