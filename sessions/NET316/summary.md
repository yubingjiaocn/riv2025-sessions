# AWS re:Invent 2025 CloudFront SaaS Manager 会议总结

## 会议概述

本次会议深入探讨了 Amazon CloudFront SaaS Manager 如何解决多租户 SaaS 平台在边缘交付中面临的挑战。演讲者 Saga Desarda（AWS SaaS 客户团队负责人）和 Bhagat（CloudFront 产品经理）详细介绍了该服务的核心功能，并邀请 Netlify 的首位工程师 Ryan 分享了实际应用案例。

会议重点讨论了 SaaS 平台在快速扩展时遇到的运营复杂性问题。当 SaaS 平台获得大型企业客户时，往往需要在短时间内支持数千个域名，同时满足严格的安全和正常运行时间要求。CloudFront SaaS Manager 通过提供多租户分发、租户隔离策略、参数化配置和自动化证书管理等功能，使 SaaS 架构师能够在不牺牲性能的前提下实现快速、可靠和可扩展的服务交付。

Netlify 的案例展示了该服务的实际价值：他们每天服务 600 万个不同的网站域名，通过 CloudFront SaaS Manager 实现了租户的秒级自动化入驻，大幅降低了运营开销并提升了平台的可扩展性。

### SaaS 边缘交付的挑战
[05:30 - 12:00] 阐述在边缘运行 SaaS 的困难
- 多租户交付在全球扩展时面临的挑战
- 控制平面和数据平面同时扩展的复杂性
- 单个配置错误可能影响数千个租户
- 使用家庭共享流媒体账号的类比说明租户隔离的重要性：如果不隔离，会导致跨租户数据泄露、缓存污染和性能争用

### 租户隔离策略
[12:00 - 18:30] 详细介绍三种租户隔离模式
- **孤岛隔离（Siloed Isolation）**：每个租户独立的分发，提供最高安全性但成本高、难以扩展，适合高价值客户
- **池化隔离（Pool Isolation）**：租户共享资源但通过命名空间、租户 ID、配额等实现逻辑隔离，高效可扩展但需注意嘈杂邻居问题
- **分层隔离（Tier-based Isolation）**：结合两种方法，高价值客户获得独立资源，其他客户共享资源，平衡成本和性能

### CloudFront SaaS Manager 核心概念
[18:30 - 28:00] 介绍 CloudFront SaaS Manager 的关键构造
- **多租户分发（Multi-tenant Distribution）**：单个 CloudFront 分发同时服务多个租户
- **分发租户（Distribution Tenants）**：在共享分发中的独立客户或逻辑实体
- **参数（Parameters）**：租户特定的键值对，用于动态自定义路由和行为，无需修改底层代码
- **缓存策略（Cache Policies）**：通过包含租户标识符实现租户间缓存隔离
- **连接组（Connection Groups）**：控制多租户环境中终端用户连接的管理方式，实现连接池隔离

### 租户入驻流程
[28:00 - 35:00] 详细说明租户入驻的四个步骤
- **步骤 1**：租户入驻服务启动，配置租户特定资源（计算、存储、数据库）
- **步骤 2**：配置 CloudFront 规则，通过参数实现租户差异化
- **步骤 3**：从 AWS Certificate Manager (ACM) 请求证书，支持 HTTP 验证方式实现完全自动化
- **步骤 4**：设置 DNS 记录，将租户自定义域名指向 CloudFront
