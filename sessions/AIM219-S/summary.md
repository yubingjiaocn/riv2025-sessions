# AWS re:Invent 2025 会议总结：AI驱动的现代运维自动化

## 会议概述

本次会议由PagerDuty产品开发副总裁Nora Jones和华纳兄弟探索公司(Warner Bros. Discovery)站点可靠性工程副总裁Tom Leaman共同主持，探讨了AI驱动的现代运维自动化。会议采用对话形式，重点讨论了如何在运维和事件管理中有效利用AI和自动化技术。

Tom分享了华纳兄弟探索公司在合并Discovery Communications和Warner Media后，九个月内构建全新流媒体应用Max的经验。他们建立了标准化的运维元数据架构和关键用户旅程(CUJ)概念，为AI应用奠定了坚实基础。会议强调了通过AI代理减少运维负担、提高事件响应效率，以及如何在不同复杂程度的运维场景中应用AI技术。

## 详细时间线与关键要点

### 0:00-5:00 开场与背景介绍
- 会议开场，主持人与观众互动拍摄合影
- Nora Jones介绍自己为PagerDuty产品开发副总裁
- Tom Leaman介绍华纳兄弟探索公司SRE团队职责范围
- 涵盖HBO Max、Discovery Plus、TNT等直接面向消费者的产品

### 5:00-15:00 华纳兄弟的技术转型之旅
- 2022-2023年Discovery Communications与Warner Media合并
- 九个月内构建全新流媒体应用Max的挑战
- 建立运维元数据架构(Operational Metadata Schema)标准化方法
- 创建关键用户旅程(Critical User Journey)概念
- 实现从代码库创建到事件处理的完整可追溯性

### 15:00-25:00 运维场景的三种类型
- 新颖情况：需要人工专家注意的全新场景
- 部分理解情况：似曾相识但需要额外上下文的场景  
- 充分理解情况：不应中断人工工作流程的常规场景
- AI代理如何在不同场景中提供支持和上下文信息

### 25:00-35:00 事件严重性分类与AI应用
- 华纳兄弟的四级严重性分类系统(SEV 1-4)
- 最近部署的事件严重性分类AI代理
- 解决事件严重性争议的常见原因：文档缺失、复杂性、不同产品线影响差异
- 通过AI代理快速解析复杂的严重性分类文档

### 35:00-45:00 事件响应流程与自动化
- SEV 1-4不同级别的响应流程和SLA要求
- "你构建，你运行"的文化理念
- 站点可靠性运维团队(SRO)的协调作用
- 技术呼叫领导轮值制度处理SEV 1事件
- AI代理在状态更新和执行层沟通中的应用

### 45:00-55:00 PagerDuty功能应用与未来展望
- 噪音减少功能实现40-50%的事件创建减少
- 运维控制台在重大发布期间的实时监控价值
- 自定义字段用于增强事件元数据和分析能力
- 高级工作流程控制SEV 1升级流程
- 未来完全AI驱动的事件管理生命周期愿景

### 55:00-57:30 总结与问答
- 强调指向历史数据的LLM应用价值
- 基础数据集投资对AI应用质量的重要性
- 利用AI代理减少上下文切换和协调成本
- 介绍正在推出的新服务健康框架
- 会议结束，感谢参与者
