# AWS re:Invent 2025 生成式AI规模化部署专题会议总结

## 会议概述

本次AWS re:Invent 2025闪电演讲由Jacqueline Crane（AWS全球增长计划负责人）和Tamore Rasheed（AWS生成式AI创新中心负责人）共同主讲。会议聚焦于企业如何将生成式AI从实验阶段推进到规模化生产部署。演讲者指出，虽然实施规模化生成式AI的客户已经看到高达40%的生产力提升，但目前只有15%的企业成功地从实验阶段过渡到全面部署。成功与失败的关键区别在于是否拥有正确的路线图。

AWS生成式AI创新中心成立于2023年，其根源可追溯到2017年AWS推出SageMaker时期。该团队由战略专家、应用科学家和部署工程师组成，在全球范围内为客户提供从用例识别、解决方案构建、部署到运营的全方位支持。创新中心还建立了合作伙伴创新联盟（Partner Innovation Alliance），将经过筛选的GenAI能力合作伙伴纳入其中，共享方法论、工具和解决方案加速器。会议重点介绍了四大成功支柱：数据作为竞争差异化因素、生产就绪思维、通过Agentic AI实现规模化，以及重新定义价值衡量标准。

## 详细时间线与关键要点

### 开场与背景介绍
[00:00 - 02:30] 
- 会议开场，介绍演讲嘉宾
- 关键数据：实施规模化生成式AI的客户生产力提升达40%，但仅15%的企业完成从实验到部署的转变
- Jacqueline Crane介绍AWS增长计划的战略定位

[02:30 - 04:45]
- Tamore Rasheed介绍生成式AI创新中心的成立背景（2023年成立，根源追溯至2017年）
- 团队构成：战略专家、应用科学家、部署工程师
- 服务范围：从简单聊天机器人到深度模型定制和微调

### 第一支柱：数据作为竞争差异化因素
[04:45 - 07:15]
- 介绍合作伙伴创新联盟（Partner Innovation Alliance）
- 强调数据是企业在大语言模型时代的核心竞争优势
- 数据平台三大关键要素：
  - 数据基础（安全性、数据质量、数据清洗）
  - 开放且安全的数据访问
  - 选择合适的定制化方法（RAG检索增强生成或微调）

[07:15 - 08:30]
- 案例分享：LOCA合作伙伴与医疗保险支付方的合作
- 在符合HIPAA合规的数据源上进行领域特定定制
- 成果：构建生产就绪的、经过策划的领域特定数据集和模型

### 第二支柱：生产就绪思维
[08:30 - 11:00]
- 四个关键方面：
  1. 不断演进的基础工作（基础设施、数据安全、数据治理）
  2. 敏捷性至关重要 - 创新中心的口号"45天上线"（Live in 45）
  3. 模型选择 - Bedrock提供多种模型选择（专有、第一方、第三方、开源）
  4. 关注变革性应用而非仅仅运营效率提升

[11:00 - 12:30]
- 案例分享：SonicWall与合作伙伴Cybage的合作
- 3周内构建开源框架，配备安全防护措施
- 创建多智能体系统用于防火墙配置和威胁检测
- 成果：代码可维护性提升40%

### 第三支柱：通过Agentic AI实现规模化
[12:30 - 15:00]
- Agentic AI的三阶段方法：设计、构建、运营
- Kira：规范驱动的IDE，AI原生代码生成（包括文档和单元测试）
- Strands：开源SDK用于构建智能体
- Agent Core：8个核心原语（运行时环境、内存、网关、身份、浏览器工具、可观测性、评估）
- 支持多种框架（LangGraph、CrewAI、Strands）和协议（MCP、A2A）

[15:00 - 16:30]
- 案例分享：泰国第五大银行Krungthai与合作伙伴1x0的合作
- 1x0构建了首个合作伙伴智能体工厂（Partner Agent Factory）中的智能体
- 专门针对ETL的智能体，解决迁移过程中60%的数据整合瓶颈
- 成果：云迁移加速50%，迁移时间减半

### 第四支柱：重新定义价值
[16:30 - 17:45]
- 强调需要重新思考和定义Agentic AI时代的价值衡量标准
- 介绍与新兴创业公司Payi的合作
- Payi提供FinOps和Agentic AI解决方案，实现成本分配、成本预测的精细化管理
- 帮助企业回答核心问题：AI投资的价值是什么

### 平台能力与总结
[17:45 - 19:30]
- 三种进入平台的方式：
  1. 购买方式：QuickSight、Amazon Connect、AWS Transform等预构建产品
  2. 构建方式：使用SageMaker、Bedrock、Agent Core自定义解决方案
  3. 合作方式：利用创新中心和合作伙伴网络的专业知识
