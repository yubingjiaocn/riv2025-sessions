# AWS re:Invent 2025 NET208 会议总结

## 会议概述

本次会议是AWS re:Invent 2025的NET208技术分会，主题为"AWS网络基础：连接、安全和扩展"。会议由两位AWS专家主讲：Rohit Aswani（AWS全球公共部门首席网络解决方案架构师）和Dave DeRicco（企业支持团队网络专家技术客户经理）。

这是一个200级别的基础会议，旨在为参会者提供AWS云网络的基本构建模块。会议涵盖了从全球基础设施到具体网络服务的完整网络知识体系，包括VPC基础、连接选项、DNS服务、流量检查、网络监控等核心主题。会议要求参会者具备CIDR表示法、DNS、IP路由等基础网络概念的基本了解。

## 详细时间线与关键要点

### 0:00-10:00 开场介绍与全球基础设施
- **会议开场**：介绍演讲者和会议议程
- **AWS全球基础设施概览**：
  - 38个区域、43个本地区域、146个以上Direct Connect接入点
  - AWS骨干网络连接所有基础设施
  - 区域和可用区的概念解释
  - 数据中心间的高速、高带宽、低延迟连接

### 10:00-20:00 IP地址选项与VPC基础
- **IPv4和IPv6地址选项**：
  - 公共IP地址：在互联网上广告的地址
  - 私有IP地址：不能在互联网上广告的地址
  - 支持自带IP地址(BYOIP)和连续IP块
- **IPv6支持**：
  - 双栈和仅IPv6配置模式
  - 150+个AWS服务支持IPv6
  - 公共服务端点和私有服务端点支持

### 20:00-30:00 Amazon VPC详细介绍
- **VPC构建基础**：
  - VPC是区域级构造，需要定义CIDR范围
  - 支持IPv4和IPv6的双栈配置
- **VPC IPAM集成**：
  - 自动化IP地址分配和管理
  - 支持多账户、多区域的IP地址规划
  - 私有和公共作用域的概念
- **子网配置**：
  - 支持IPv4-only、dual-stack、IPv6-only子网
  - IPv4子网大小：/28到/16
  - IPv6子网大小：/44到/64（以/4递增）

### 30:00-40:00 网络安全与连接选项
- **基础安全控制**：
  - 网络访问控制列表(NACL)：无状态，子网级别
  - 安全组：有状态，ENI/实例级别
- **IPv6向后兼容性**：
  - DNS64支持，用于IPv6-only子网访问IPv4资源
  - 合成IPv6地址的十六进制格式转换
- **互联网连接**：
  - 公共子网：通过互联网网关连接
  - 私有子网：通过NAT网关连接
  - 仅出站互联网网关用于IPv6

### 40:00-50:00 高级连接与应用网络
- **VPC间连接**：
  - VPC对等连接：区域内和跨区域支持
  - Transit Gateway：第三层路由服务
  - AWS Cloud WAN：全球WAN连接，支持动态路由
- **混合连接**：
  - AWS Direct Connect：专用连接，支持虚拟接口
  - Site-to-Site VPN：IPsec加密隧道
  - SD-WAN集成支持
- **应用网络**：
  - AWS PrivateLink：基于身份的访问控制
  - Amazon VPC Lattice：大规模服务连接管理

### 50:00-58:30 DNS、防火墙与监控
- **Route 53 DNS服务**：
  - VPC内置DNS解析器
  - 私有托管区域和跨VPC DNS
  - 入站/出站解析器端点
- **流量检查**：
  - AWS Network Firewall：完全托管的防火墙服务
  - Gateway Load Balancer：第三方防火墙设备支持
  - 分布式和集中式检查模型
- **远程访问**：
  - AWS Client VPN：网络级访问
  - AWS Verified Access：应用级访问
- **网络监控与故障排除**：
  - VPC Flow Logs：流级别可见性
  - CloudWatch网络监控工具
  - VPC可达性分析器：逐跳路径分析
  - 网络访问分析器：合规性检查
