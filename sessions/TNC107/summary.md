# AWS re:Invent 2025 分会场总结：AI技能培训与企业转型

## 会议概览

本次AWS re:Invent 2025分会场由AWS培训产品总监Umesh Sut和桑坦德银行代理CTO Julio Blanco Perez共同主持，重点探讨了AI技能培训在企业数字化转型中的关键作用。会议分为两个主要部分：首先介绍了当前AI技能培训面临的挑战与机遇，以及AWS Skill Builder平台如何通过AI驱动的学习体验帮助组织实现规模化的AI技能提升；随后，桑坦德银行分享了其在巴西市场实施的"AWS Level Up"项目的实践经验，展示了如何将传统培训与现代微学习方法相结合，成功培养了数千名员工的云计算和AI技能。

会议强调了一个核心观点：到2030年，40%的核心工作技能将发生变化，而AI技能的半衰期可能缩短至仅两年。这要求企业必须建立持续学习的文化。同时，数据显示拥有AI技能的个人薪酬可提高47%，而92%的组织计划在2028年前部署AI项目，但75%的组织面临人才短缺问题。AWS通过Skill Builder平台提供了超过330个学习资源，包括Cloud Quest、SimLearn、Meeting Simulator等创新产品，将学习、实践和认证紧密结合。桑坦德银行的案例证明，通过混合学习模式、游戏化机制和社区驱动的方法，可以在两年内将AWS认证人数从120人增加到700人，并显著提升业务成果。

## 详细时间线与关键要点

### 开场介绍 (00:00 - 03:30)
- **00:00** - Umesh Sut介绍自己为AWS培训产品总监，欢迎桑坦德银行代理CTO Julio Blanco Perez共同分享AI技能培训主题
- **01:15** - Julio表示很高兴分享过去两年与AWS合作的成果
- **02:00** - Umesh概述会议议程：挑战与机遇、Skill Builder平台演进、桑坦德银行实践案例
- **02:45** - 说明前20-25分钟讨论问题空间和方法，然后由Julio分享实际应用案例

### AI技能培训的挑战与机遇 (03:30 - 08:00)
- **03:30** - 引用世界经济论坛2025年就业报告：到2030年，40%的核心工作技能将发生变化
- **04:15** - AWS研究显示：92%的组织计划在2028年前实施AI项目，但75%找不到所需人才
- **05:00** - 技能半衰期数据：1980年代为10年，2024年为4年，未来2-4年可能缩短至2年
- **05:45** - 持续学习不再是可选项，而是必需品
- **06:30** - Skillsoft研究表明：拥有AI技能的个人薪酬可提高47%
- **07:15** - 强调AI技能培训既是挑战也是机遇

### 现代学习者旅程的演变 (08:00 - 11:30)
- **08:00** - 传统技术技能仍然重要，但人本技能（沟通、批判性思维、关系建立）变得同等重要
- **08:45** - 生成式AI技术使人本技能培训更容易融入学习体验
- **09:15** - 微学习成为新常态：15分钟到几小时的学习片段，而非2-3天的集中培训
- **09:45** - 学习需要有趣且引人入胜
- **10:15** - 学习周期重构：学习和实践阶段变得更小、更连续，支持快速迭代
- **11:00** - 微学习支持在实际工作中即时学习和应用

### AWS Skill Builder学习阶段 (11:30 - 15:00)
- **11:30** - 学习阶段：Skill Builder提供330+资源，涵盖技术技能和人本技能
- **12:00** - 介绍AI驱动的沉浸式学习体验：SimLearn和Meeting Simulator
- **12:30** - 实践阶段：多种产品支持角色和行业专业知识培养
- **13:00** - Cloud Quest：3D游戏化学习体验，学习云概念并在真实AWS环境中实践
- **13:45** - Builder Labs和Jam Sessions：在真实AWS环境中实践概念
- **14:15** - Code Clash：基于游戏的团队学习产品，支持一对一竞技
- **14:45** - 学习和实践体验深度集成到Skill Builder产品中

### 创新学习产品详解 (15:00 - 21:00)
- **15:00** - Cloud Quest Generative AI Practitioner：3D虚拟城市中的引导式学习
- **15:30** - 在游戏中直接使用Amazon Bedrock、SageMaker、Q Developer等服务
- **16:00** - 学习基础模型、提示工程、RAG实现等概念
- **16:30** - SimLearn：模拟真实场景，与AI虚拟代理互动
- **17:00** - SimLearn提供技术技能和沟通技能的双重反馈
- **17:30** - 数据显示Cloud Quest和SimLearn的用户参与度远高于传统数字学习
- **18:00** - Meeting Simulator：11月12日新推出，专注人本技能培养
- **18:30** - 通过语音或文本模拟高风险对话场景
- **19:00** - 可模拟与CFO讨论成本等实际场景，获得实时沟通反馈
- **19:30** - 适合技术销售或利益相关者角色的准备
- **20:00** - 准备阶段：通过考试风格问题复习和练习
- **20:30** - 自我评估是否准备好参加AWS认证考试

### AWS认证的重要性 (21:00 - 24:30)
- **21:00** - 认证帮助获得47%的薪酬提升
- **21:30** - 认证在简历上验证AI技能，帮助克服就业障碍
- **22:00** - 超过100万人拥有AWS认证，部分人拥有多个认证
- **22:30** - 91%的组织表示AWS认证员工提高了公司创新能力
- **23:00** - 90%以上的组织表示能够降低成本
- **23:30** - 认证带来创新和生产力双重收益

### AI/ML认证体系 (24:30 - 27:00)
- **24:30** - AWS共有12个认证，其中4个专注AI/ML
- **25:00** - AI Practitioner Foundational：验证AI基础概念
- **25:20** - Machine Learning Engineer Associate：验证机器学习和深度学习概念、模型调优
- **25:45** - Data Engineer Associate：验证数据管道能力，支持构建代理应用
- **26:15** - Generative AI Professional：11月18日发布Beta版，验证构建代理应用能力
- **26:45** - Machine Learning Engineer和Data Engineer认证已更新，包含更多代理能力相关问题

### 微认证创新 (27:00 - 29:00)
- **27:00** - 推出微认证（Micro Credentials），降低认证门槛
- **27:30** - 微认证在真实AWS环境中进行实时评估，而非Q&A形式
- **28:00** - 验证实际操作技能，而非仅理论知识
- **28:20** - 11月12日推出两个微认证：AWS Agentic AI和AWS Serverless
- **28:45** - 微认证与传统认证互补，支持微学习概念

### 个人学习旅程总结 (29:00 - 30:30)
- **29:00** - 学习和实践阶段更加紧密结合
- **29:20** - 考试准备在多次学习-实践循环后进行
- **29:40** - 微认证可在学习-实践阶段进行部分验证
- **30:00** - 通过微认证或传统认证完成最终验证
- **30:15** - 这是个人AI职业发展和成功的路径

### 组织层面的团队学习 (30:30 - 34:00)
- **30:30** - 协作式团队学习是建立文化的最有力方式
- **31:00** - Cloud Quest Tournaments：团队版3D游戏，团队竞赛解决问题
- **31:30** - AWS Jams：开放式实验室问题，团队解决真实挑战，设有排行榜
- **32:00** - AWS Code Clash：游戏化应用，一对一知识测试
- **32:30** - Escape Rooms：团队认证考试准备，类似物理密室逃脱
- **33:00** - Skill Builder Trivia：通过大规模团队竞赛强化知识
- **33:30** - 这些形式在re:Invent现场多个场馆可体验

### Cohort Studio发布 (34:00 - 36:00)
- **34:00** - 11月12日推出Cohort Studio
- **34:30** - 解决客户反馈：难以将多个学习体验整合到单一项目中
- **35:00** - 任何人可在15分钟内启动队列项目，AI驱动
- **35:20** - 可选择不同学习体验，推动参与度、团队合作和认可
- **35:40** - 支持认证考试、AI技能训练营、团队竞赛等场景
- **35:50** - 感谢桑坦德银行帮助构想和可视化该产品

### 桑坦德银行介绍 (36:00 - 39:00)
- **36:00** - Julio开始分享桑坦德银行案例
- **36:20** - 桑坦德银行拥有180年历史，是全球性传统银行
- **36:45** - 产品组合覆盖个人到大型企业的所有银行业务
- **37:10** - 拥有近2亿客户，管理资产近2万亿欧元
- **37:30** - 自2018年起与AWS合作实现规模化
- **37:50** - 定位为"有分支机构的数字银行"，技术嵌入所有业务
- **38:20** - 采用混合方式：全数字银行+传统分支机构关系
- **38:45** - 本次分享聚焦两年前在巴西启动的项目

### AWS Level Up项目概述 (39:00 - 42:30)
- **39:00** - 传统学习方式不再有效：强制性培训、3天课堂培训无法规模化
- **39:30** - 不同水平的人在同一课堂效果不佳
- **40:00** - 完全自主学习也不现实：员工自己管理培训计划困难
- **40:30** - Level Up混合两种方式：有规则、有路径、有节奏
- **41:00** - 根据课程复杂度，预期3-9个月完成
- **41:20** - 与AWS共同设计，结合AWS平台知识和桑坦德需求
- **41:45** - 将认证路径与实际需求相匹配

### Level Up运作机制 (42:30 - 46:00)
- **42:30** - 入职时分配到学习小组，匹配知识水平和目标认证
- **43:00** - 在Skill Builder上注册，获得清晰的学习路径
- **43:20** - 可自主选择路径外的课程，但有明确的学习节奏
- **43:45** - 团队成员互相询问进度，相互推动
- **44:15** - 意外发现：团队成员开始互相推动学习进度
- **44:40** - 通过徽章实现轻度游戏化，可看到小组其他成员进度
- **45:00** - 不设置排行榜和冠军，避免竞争过度
- **45:20** - 希望所有人都参与并推动培训完成
- **45:45** - 为完成认证的人创建特殊体验：类似re:Invent的AWS体验活动

### AWS体验活动 (46:00 - 48:00)
- **46:00** - 认证人员可参加VIP午餐和特殊活动
- **46:20** - 他们是"派对的VIP"，可在组织内展示认证成就
- **46:40** - 创建了进阶系统框架
- **47:00** - 不同认证路径（AI、云从业者、解决方案架构师）但流程相同
- **47:30** - 固定时间表：3个月内完成培训有多个预定培训节点
- **47:50** - 期望自主学习，但小组成员可看到你的进度

### 有效沟通与社区精神 (48:00 - 50:00)
- **48:00** - 更有效的沟通开始在所有团队中出现
- **48:20** - 创建了之前不存在的社区精神
- **48:40** - 之前也有AWS培训和大量投资，但效果不同
- **49:00** - 直到采用这种混合系统才达到最佳平衡点
- **49:30** - 在强制性要求和自由学习之间找到平衡

### 技术员工学习旅程 (50:00 - 54:00)
- **50:00** - 巴西桑坦德技术员工的学习旅程
- **50:20** - 入职后在学习门户注册，同时注册AWS Skill Builder
- **50:45** - 分配参加Cloud Quest锦标赛，团队一起游戏解决场景
- **51:15** - 为特定团队同步Skill Builder培训课程
- **51:40** - 传统的大型课堂培训（葡萄牙语称为"aulões"）
- **52:00** - 保留传统学习方式，因为有些人不适应微学习策略
- **52:30** - 混合所有方式以确保每个人都能前进
- **52:50** - 使用SimLearn和其他平台练习模拟考试
- **53:15** - 通过评估后获得AWS考试券参加认证
- **53:40** - 3、6或9个月项目结束时举行闭幕式

### 激励机制与成果 (54:00 - 57:30)
- **54:00** - 闭幕式有奖品抽奖，只有获得认证的人可以获奖
- **54:20** - 不是大额奖金，但每个人都想上台领取设备
- **54:45** - 从组织角度看，数千人参加培训，数百人完成认证
- **55:10** - 项目开始时只有约120个认证
- **55:30** - 从120增长到700个认证，增长巨大
- **55:50** - 培训观念发生改变：从强制性HR要求到主动学习
- **56:20** - 人们开始一起讨论如何最好地使用工具
- **56:40** - 工具被用于其他用例
- **57:00** - 特定团队要求更高级的ILT课堂培训，因为路径中未涵盖某些日常工作需要的内容
- **57:20** - 过去两年培训管理方式发生根本改变

### AWS体验大会 (57:30 - 61:30)
- **57:30** - 三年前，高管从re:Invent回到巴西后提议组织类似活动
- **58:00** - 与AWS和其他合作伙伴合作，创建"桑坦德AWS体验"
- **58:20** - 这改变了培训思维方式
- **58:40** - 去年10月举办，有21场200-300级别的演讲
- **59:00** - 由于很多人已接受培训，不需要100级别课程
- **59:20** - 提高标准，提供更高质量的路径
- **59:40** - 不同路径：AI、数据库、架构等
- **60:00** - 虽不完全等同re:Invent，但同时有3-4场不同演讲
- **60:20** - 今年在开放空间中设置了DeepRacer赛车
- **60:40** - 进入决赛的团队吸引了大量观众，因为都在同一栋楼
- **61:00** - 三天内还有密室逃脱和考试准备活动

### 游戏化与VIP体验 (61:30 - 63:30)
- **61:30** - 密室逃脱通常是数字化的，但这次让所有人在同一房间
- **61:50** - 团队合作完成挑战"逃出"房间
- **62:10** - 开玩笑说要锁门，只有完成挑战才能出去
- **62:30** - 最后举办大型派对，认证人员进入VIP区
- **62:50** - VIP区在派对中间，有不同的展位和活动
- **63:10** - 通过游戏化机制激励，但不设排行榜
- **63:20** - 排行榜只激励前排人员，中后排会失去动力，希望所有人都参与

### 业务成果：投资顾问案例 (63:30 - 67:00)
- **63:30** - 随着更多人接受云和AI培训，可以更轻松地完成以前无法完成的事情
- **64:00** - 投资顾问组织遇到问题：为客户准备正确推介花费太多时间
- **64:20** - 准备对话的时间多于实际与客户对话的时间
- **64:40** - 在AWS帮助下使用AWS技术栈整合所有内容
- **65:00** - 彻底改变了顾问的工作方式
- **65:20** - 现在只需30秒到1分钟准备推介
- **65:40** - 知识库中准备了所有银行产品目录信息
- **66:00** - 包含相关回报、风险等信息
- **66:20** - 可根据不同客户调整推介深度：深度对话或正式方法
- **66:40** - 可在项目中调整多个参数
- **66:50** - 结果显示客户接触量增加1.2倍
