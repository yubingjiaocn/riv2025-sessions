# AWS 量子计算会议总结：Amazon Braket 从探索到企业应用

## 会议概述

本次 AWS re:invent 会议由 Amazon Braket 总经理 Eric Kesla 和量子计算市场负责人 Michael Brett 共同主讲，深入探讨了量子计算技术的现状与未来发展方向。

会议首先澄清了一个关键误区：量子计算机并非运行更快的经典计算机，而是能够运行更丰富算法类别的计算设备。量子计算机的优势在于能够以指数级更少的操作解决某些特定问题，特别是在模拟量子系统方面。正如物理学家 Richard Feynman 在 1981 年所预言的，自然界本质上是量子力学的，要模拟自然就需要量子计算机。目前，量子计算技术仍处于早期阶段，尚无法在实际工业问题上超越经典计算机，但 AWS 正通过多个维度推动这项技术的发展。

AWS 的量子技术战略包括四大支柱：Amazon Braket 量子计算服务、Amazon 高级解决方案实验室、AWS 量子计算中心（与加州理工学院合作开发自有量子硬件），以及后量子密码学团队。Amazon Braket 目前提供来自五家供应商的不同类型量子计算机访问，包括离子阱、超导和里德堡原子技术。会议还重点介绍了量子-经典混合工作负载的实际应用案例，特别是与阿斯利康合作的药物设计项目，展示了如何将量子计算集成到端到端的高性能计算工作流程中。

## 详细时间线与关键要点

00:00 - 会议开场与演讲者介绍
- Eric Kesla（Amazon Braket 总经理）和 Michael Brett（量子计算市场负责人）介绍会议议程
- 会议将涵盖量子计算基础、AWS 服务概览、混合工作负载和客户案例研究

02:30 - 澄清量子计算的核心概念
- 强调量子计算机不是"更快的经典计算机"，而是运行不同类别算法的设备
- 量子计算机实际上比经典计算机时钟周期更慢，但能解决经典计算机需要指数级资源的问题
- 模拟 50 个量子粒子需要超级计算机规模的经典计算资源，100 个粒子则完全超出现有计算能力

05:00 - Richard Feynman 的预言与量子计算的指数优势
- 引用 Feynman 1981 年的著名论断：自然不是经典的，模拟自然需要量子力学方法
- 量子计算提供的不是 2 倍速度提升，而是 2 的 x 次方的指数级提升

07:15 - 量子计算的三个重要启示
- 量子计算不会取代经典计算机，两者将共存服务于不同类型的问题
- 许多量子计算用例目前是"隐藏的"，因为它们解决的是经典计算机无法处理的问题
- 量子应用不是简单的"迁移"，需要全新的算法和大量专业知识

09:00 - 量子计算的应用领域
- 核心应用：物理和化学模拟、分子反应、材料科学
- 经典问题的量子优势：密码学（Shor 算法可指数级加速大数分解）
- 研究中的领域：优化问题和机器学习

11:30 - AWS 的量子计算愿景
- AWS 将量子计算视为基础计算技术，目标是使其成为计算架构的一部分
- 长期目标是让量子计算像 GPU、ASIC、FPGA 一样成为"无聊"的加速器选项

13:00 - 量子计算技术的现状警示
- 明确指出：目前没有任何量子计算机能在实际工业问题上超越经典计算机
- 当前客户主要用量子计算机来研究量子计算本身

15:00 - 量子计算的发展阶段类比
- 将量子计算发展比作 GPU 的演进：从图形处理的小众应用，到数学计算，最终通过 AI/ML 实现广泛采用
- 当前阶段：从研究工具向科学应用工具过渡

17:30 - AWS 量子技术四大支柱介绍
- Amazon Braket 量子计算服务
- Amazon 高级解决方案实验室（专业服务团队）
- AWS 量子计算中心（自研硬件）
- 后量子密码学和量子网络团队

19:00 - AWS 量子计算中心与 Cat Qubit 技术
- 位于加州理工学院的研究中心，采用量子纠错优先方法
- Cat Qubit 技术结合超导量子比特和微波谐振器，可抑制特定类型噪声
- Ocelot 原型芯片展示：纠错资源需求比传统方法降低 10 倍

22:00 - Amazon Braket 服务发布背景
- 2020 年推出，目标是民主化量子计算访问
- 提供多种量子计算机类型，让客户能够对比不同技术路线

24:00 - Amazon Braket 的硬件合作伙伴
- 离子阱技术：IonQ 和 Alpine Quantum Technology (AQT)
- 超导技术：Rigetti 和 IQM
- 里德堡原子技术：QuEra

26:30 - Amazon Braket 服务架构三层模型
- 底层：基础设施和访问模式（QPU、量子电路模拟器、经典计算资源）
- 中间层：接口和能力（Jupyter IDE、编程工具、编译器、混合作业功能）
- 顶层：工具集成（支持 Qiskit、PennyLane、NVIDIA CUDA Quantum 等主流框架）

29:00 - 访问模式介绍
- 按需付费模式
- 预留时段模式（获得设备专用访问时间）

31:00 - Amazon Braket Program Sets 新功能
- 允许在单次 API 调用中提交多个量子程序
- 优化编译和 I/O 流程，减少每个程序的开销
- 某些工作负载可实现高达 24 倍的速度提升

34:00 - 量子计算的首批用例：科学应用
- 强调科学发现的重要性，不仅是通往工业应用的跳板
- 引用 NERSC（美国能源部超算中心）研究：50% 的生产工作负载有潜力被量子计算指数级加速

37:00 - 国家量子计划与 Amazon Braket
- 多国政府启动国家量子计划以加速科学突破
- AWS 与新加坡 NQCH、英国 NQCC、爱尔兰 QCoIR、澳大利亚 PsiQuantum 等合作
- 澳大利亚刚推出 Quantum Hub Portal 和 Schrödinger's Q 试点项目

40:00 - Michael Brett 接棒：企业客户战略
- 介绍 AWS 如何帮助企业客户为量子计算未来做准备
- 当前大多数客户是研究人员，但目标是推动工业科学用例和大规模企业应用

42:30 - JP Morgan Chase 案例引用
- 作为量子计算领先投资者的典型代表
- Marco Pistoia 的观点："如果公司现在不采取行动，当量子优势成为现实时可能为时已晚"
- 预告次日有 JP Morgan Chase 专场分享会

45:00 - 企业量子计算采用路径
- 用例发现阶段
- 研究和基准测试（量子与量子对比、量子与经典对比）
- 混合机会探索（跨多平台编排）
- 为生产应用奠定基础

48:00 - Quantum Embark 项目介绍
- 2023 年 re:invent 推出的企业入门项目
- 12 周固定价格、固定范围的项目
- 包括用例发现、培训赋能和深入解决一个具体问题
- 目标是提供无炒作的可操作成果

51:00 - 高级解决方案实验室（Advanced Solutions Lab）
- 专业服务团队进行长期研发合作
- 与 JP Morgan Chase 进行了三年的多问题研究项目
- 涵盖设备优化、编译策略、应用基准测试等各个层面
- 帮助客户开发知识产权和专业能力

54:00 - 量子计算在加速计算组合中的定位
- 量子计算是 AWS 加速计算产品组合中的另一种实例类型
- 与 CPU、GPU、Trainium、Inferentia 等并列
- 正在将 Braket 集成到 AWS Batch、Parallel Cluster 等 HPC 服务中

57:00 - 容量消费模式对比
- 经典基础设施：按需、容量块、Spot、预留实例
- Braket 正在构建相同的消费模式
- 当前支持按需访问和容量块预留

59:30 - 端到端工业级工作流程架构
- 典型 HPC 工作流程：用户操作 → 执行计划 → I/O → 编排层 → 多类型计算
- 量子计算的目标：在工作流程中插入适合量子计算的部分，实现端到端加速

62:00 - 阿斯利康药物设计案例介绍
- 四方合作：阿斯利康、IonQ、NVIDIA、AWS
- 探索算法问题、编排问题和端到端工作流程集成

64:00 - 过渡金属催化问题
- 现代药物设计的核心计算挑战
- 问题复杂度随分子增长呈指数级增加
- 目标：在湿实验室实验前准确模拟合成路线

66:30 - 氧化加成和 Suzuki-Miyaura 偶联反应
- 模拟催化过程中的能量势垒
- 势垒高度决定反应是否可行
- 需要在实验前了解势垒高度和最终状态

68:00 - 经典工作流程 vs 量子混合工作流程
- 经典流程：活性空间计算 → 试验波函数（最困难部分）→ 蒙特卡洛分析
- 量子混合方法：经典 → 量子 → 经典
- 用量子算法替换试验波函数准备步骤
- 在 IonQ Aria 机器上运行量子部分

关键技术术语
- QPU (Quantum Processing Unit): 量子处理单元
- Cat Qubit: 猫量子比特（一种量子纠错架构）
- NISQ (Noisy Intermediate-Scale Quantum): 噪声中等规模量子
- VQE (Variational Quantum Eigensolver): 变分量子本征求解器
- 量子-经典混合算法：结合量子和经典计算优势的算法
