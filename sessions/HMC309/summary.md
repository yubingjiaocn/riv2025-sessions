# AWS re:Invent 2025 - AMC 309: 从VMware工作负载迁移到AWS混合边缘解决方案

## 会议概述

本次会议重点介绍了AWS如何帮助企业将VMware工作负载迁移到AWS混合边缘解决方案，特别是AWS Outposts和Local Zones。会议由AWS的Frankie和Scott Rose主讲，并邀请了Caesars Entertainment的Travis McVeigh分享实际迁移经验。

会议强调了传统本地基础设施面临的挑战，包括高昂的运营成本、技术债务积累以及日益增长的数据主权要求。AWS提出了"AWS where you need it"策略，通过Outposts和Local Zones为客户提供渐进式迁移路径，既能满足数据驻留要求，又能享受云原生服务的优势。Caesars Entertainment作为早期采用者，成功将30多个司法管辖区的VMware环境迁移到Outposts，实现了4倍性能提升和显著的运营简化。

## 详细时间线与关键要点

### 0:00-5:00 - 会议开场与挑战概述
- 介绍会议主题：VMware工作负载迁移到AWS混合边缘解决方案
- 主讲人介绍：Frankie、Scott Rose (AWS)、Travis McVeigh (Caesars Entertainment)
- 传统本地基础设施面临的核心挑战：
  - 难以准确计算TCO（总拥有成本）
  - 人员成本、设施成本、技术债务难以量化
  - 业务和技术驱动因素推动云迁移需求

### 5:00-10:00 - 迁移挑战与AWS策略
- 迁移过程中的三大挑战：
  - 迁移时间长短和价值展示问题
  - 数据主权和监管要求日益严格
  - 生成式AI应用的本地推理需求
- AWS "where you need it"策略介绍：
  - AWS Outposts：部署在客户数据中心
  - Local Zones：部署在大都市区域
  - 提供渐进式迁移路径，最终目标是迁移到AWS区域

### 10:00-15:00 - 迁移方法与商业考量
- 多种迁移路径：
  - Lift and shift：快速迁移现有工作负载
  - 现代化改造：采用托管数据库、容器服务
  - 重构：完全无服务器部署
- Scott Rose介绍商业支持：
  - AWS提供迁移资金支持计划
  - 避免重复许可成本
  - 与合作伙伴协作的资金模式

### 15:00-25:00 - AWS Outposts第二代与技术优势
- Outposts Gen 2关键特性：
  - 高计算密度EC2实例
  - 并行网络基础设施
  - 支持大规模迁移需求（数万CPU、PB级存储）
- 新实例类型：
  - 高密度CPU和内存配置
  - 裸金属实例（金融服务、电信）
  - 即将推出的GPU实例（边缘推理和训练）

### 25:00-35:00 - AWS Transform AI工具
- 迁移评估面临的复杂性：
  - 传统环境缺乏优化，架构不合理
  - 数千台VM的评估需要数月甚至数年
  - 网络配置复杂，需要专业分析
- AWS Transform功能：
  - 基于AI的环境分析和优化
  - 网络拓扑转换（NSX到AWS网络实体）
  - 波次分析和迁移规划
  - 支持VMware、大机、传统应用迁移

### 35:00-45:00 - Caesars Entertainment案例分享
- 公司背景：
  - 美国最大的赌场娱乐公司
  - 50,000+员工，管理多个知名品牌
  - 在线博彩业务快速增长（2018年以来增长10倍）
- 迁移前的挑战：
  - 开发和生产环境不匹配
  - Wire Act法规要求交易必须在州内完成
  - 复杂的多厂商环境管理
  - VMware许可成本上涨压力

### 45:00-55:00 - 迁移实施与经验教训
- 原有架构：
  - 基于CAPI（VMware上的开源Kubernetes）
  - 大部分工作负载容器化
  - 数据库VM和存储网关
- 迁移后架构：
  - EKS on Outposts替代CAPI
  - C节点用于数据库，M节点用于应用
  - Route 53和存储网关保持连续性
- 迁移时间线：
  - 7月：4个机架，迁移2个州
  - 8月：1个机架，迁移2个州  
  - 9月：5个机架，每周迁移1个州
  - 停机时间缩短至2小时
- 关键经验教训：
  - 插槽规划至关重要
  - 备用电源必不可少（重启需要数小时）
  - 服务链路故障测试应超过2小时
  - 性能提升3-4倍（1:1 CPU比率）
