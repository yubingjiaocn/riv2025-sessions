# AWS re:Invent 2025 技术会议总结

## 会议概述

本次会议由英国普华永道（PWC）健康团队总监Catherine主持，与会者包括PWC英国公共部门团队的Andy以及英国国家医疗服务体系（NHS）伦敦郊外医院的客户代表Nicola。会议重点讨论了AWS产品如何变革英国公共部门，特别是NHS的数字化转型。

NHS作为欧洲最大的雇主，面临着巨大挑战：尽管拥有数百万员工，仍有10%的人口（约600万人）在医疗等待名单上。技术基础设施陈旧复杂，数字化、AI和云技术应用有限，这为改善公民体验提供了巨大机遇。会议展示了从地方政府到NHS的成功技术迁移案例，重点介绍了Amazon Connect、Bedrock、Lex和Lambda等AWS服务的实际应用效果。

## 详细时间线与关键要点

### 0:00-3:00 会议开场与背景介绍
- Catherine介绍与会嘉宾和会议主题
- NHS现状分析：欧洲最大雇主，10%人口在等待名单
- 技术挑战：基础设施陈旧，数字化程度低
- 机遇识别：巨大的改善空间和转型潜力

### 3:00-6:00 地方政府成功案例分享
- Andy分享地方议会Amazon Connect部署经验
- 关键成果：
  - 10-15天完成完整联络中心迁移
  - 10周内释放30%联络中心容量
  - 快速决策和治理机制的重要性
- 可复用资产的价值：质量、速度、跨行业适用性
- 地方议会现代化需求：资金紧张、公民期望提升

### 6:00-9:00 NHS面临的具体挑战
- Nicola介绍Medway医院情况：
  - 患者常规预约等待时间长达一年
  - 服务区域社会经济困难，药物滥用、酒精滥用问题严重
  - 人口平均阅读年龄仅6岁，沟通挑战巨大
- 联络中心问题：
  - 年均12万通电话，50%无人接听
  - 预约变更和取消信息无法及时传达
  - 资源浪费导致等待时间进一步延长
  - 员工面临患者投诉和辱骂

### 9:00-12:00 AWS解决方案实施与效果
- Andy详述技术解决方案：
  - 使用Lex和Lambda自动化简单事务性联系
  - 伯明翰案例：55%入站电话实现自动化
  - 80%电话得到有效回应，客户满意度高
  - 24/7全天候服务，支持边缘化社区
  - 通过Lambda连接后台系统，实现端到端流程自动化

- Nicola分享实施成果：
  - 6周内完成部署
  - 80%电话实现自动化（约8-9万通/年）
  - DNA（未到诊）率下降4%
  - 多语言支持，提升移民群体可及性
  - 患者反馈从投诉转为高度满意
  - 保留人工服务选项，照顾老年和特殊需求患者

### 12:00-15:00 实施挑战与关键成功因素
- Andy强调的关键挑战：
  - 准确理解用户意图，而非仅关注用词
  - 医疗专业术语处理：结合Bedrock进行复杂医学语言查找
  - "不要让完美成为良好的敌人"：80% MVP策略
  - 持续改进方法：快速迭代和调整能力
  - 业务流程与技术配置的融合

- 变更管理策略：
  - 员工参与设计过程，避免"AI取代工作"恐惧
  - 从项目开始就获得团队支持
  - 渐进式实施，建立信心

### 15:00-17:30 未来发展规划与愿景
- Nicola的扩展计划：
  - 服务5个监狱的特殊人群医疗需求
  - 建立Kent和Medway地区中心，支持其他医院信托
  - Canterbury等地区的推广部署
  - 后台职能自动化：HR、财务等部门
  - 临床服务集成：术前问卷自动化

- Andy的技术发展愿景：
  - Nova Sonic语音到语音技术应用
  - 纯生成式环境中的丰富对话体验
  - 双维度方法：客户支持和业务支持并行
  - 代理式开发：主动服务而非被动响应
  - 个人代理与组织系统的无缝集成

### 17:30-17:54 会议总结
- 强调心态转变的重要性：愿意尝试新技术
- 成功经验的可复制性和规模化潜力
- 跨地理区域的雄心勃勃愿景
- 对未来技术发展的期待和信心
