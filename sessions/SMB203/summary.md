# AWS re:Invent 2025 分会场总结：使用 AI 革新高级视频工作流

## 概述

本次会议由 AWS 解决方案架构师 Stefano Sandrini 和 CHS 21 公司的 CTO David Gonzalez 共同主讲，重点介绍了 CHS 21 如何利用 AWS 上的 AI 技术革新高级视频工作流程。Stefano 首先阐述了媒体娱乐行业客户选择 AWS 的三大核心原因：AWS 在该行业的深厚专业知识、专门构建的技术服务和解决方案，以及庞大的合作伙伴生态系统。这些优势帮助客户在内容制作、媒体供应链、直播、流媒体等六大解决方案领域实现数字化转型。

会议特别强调了生成式 AI 在媒体娱乐行业的五大应用场景：档案修复、增强广播、本地化、超个性化以及视频语义理解。David 随后详细介绍了 CHS 21 开发的 Media Copilot 平台，这是一个统一的 AI 驱动平台，旨在解决客户在使用碎片化应用生态系统时面临的内容交付缓慢和成本高昂的问题。该平台采用无服务器架构，集成了多种 AI 模型能力，并通过智能代理进行编排，实现了更快的内容交付和更低的运营成本。

  - 专门构建的服务和预构建解决方案
  - 庞大的专注于媒体娱乐的合作伙伴生态系统
- **4:30** - 介绍媒体娱乐的六大解决方案领域：内容制作、媒体供应链、档案管理、广播、直播制作、直接面向消费者和流媒体

### 生成式 AI 在媒体娱乐中的应用 (6:00-8:00)
- **6:00** - 介绍生成式 AI 对媒体娱乐行业的影响
- **6:30** - 五大常见生成式 AI 应用场景：
  - 档案和修复
  - 增强广播
  - 本地化（自动配音、自动字幕）
  - 超个性化
  - 视频语义理解
- **7:30** - 视频语义理解的应用：语义搜索、基于历史内容创建新内容（如社交媒体内容）

### CHS 21 公司背景 (8:00-10:00)
- **8:00** - David Gonzalez 开始介绍 CHS 21
- **8:30** - 公司历史：近 20 年前成立，专注于西班牙的直播流媒体服务
- **9:00** - 主要客户：广播公司
- **9:30** - 两年前开始开发自有 AI 管道

### Media Copilot 平台的诞生 (10:00-12:00)
- **10:00** - 发现客户痛点：使用碎片化的应用生态系统
- **10:30** - 客户面临的问题：内容交付缓慢、成本高昂、大量重复工作
- **11:00** - Media Copilot 解决方案：统一平台，集成多种 AI 模型能力
- **11:30** - 平台特点：由智能代理编排，提供更快的内容交付和更低的成本

### 技术架构详解 (12:00-16:00)
- **12:00** - 采用经典的无服务器架构
- **12:30** - 选择无服务器的原因：更快的上市时间、可靠性和可扩展性
- **13:00** - 核心技术栈：
  - API Gateway 和 Lambda
  - Step Functions 作为核心编排器
  - SageMaker
  - Amazon Cognito（双因素认证、单点登录）
  - CloudFront 和 S3（全球内容分发和保护）
- **14:00** - AWS Media Services 的应用：
  - MediaConvert：转码视频资产
  - MediaLive 和 MediaPackage：接收直播源
  - Harvest Jobs：直播剪辑
- **15:00** - 自研视频编辑器：支持直播剪辑创建、字幕添加、品牌和样式定制

### AI 管道和模型 (16:00-18:00)
- **16:00** - 自定义 AI 管道介绍
- **16:30** - 音频处理能力：
  - 自动语音识别（ASR）
  - 语音活动检测
  - 说话人分离
  - 场景检测
- **17:00** - 视频处理能力
- **17:30** - 使用 Amazon Bedrock 创建复杂元数据：字幕、精彩片段、摘要等

### API 集成和使用场景 (18:00-19:30)
- **18:00** - 所有功能通过 API 提供
- **18:30** - 大多数客户使用 API 而非 UI
- **19:00** - 集成场景：CMS、媒体资产管理系统、直播剪辑服务
- **19:30** - Media Copilot 作为客户系统的 AI 层

### Agent Core 的应用 (19:30-23:00)
- **19:30** - 两个月前开始开发智能代理
- **20:00** - AWS Agent Core 的优势：
  - 运行时网关
  - 身份管理
  - 可观测性
  - 内存管理
  - 可扩展、安全
- **21:00** - 开发 MCP 服务器：提供不同工具连接数据库、API 和代码
- **21:30** - 使用 Gateway 部署 MCP 服务器：只需提供 OpenAPI 规范，几分钟内完成部署
- **22:00** - Agent Core Runtime 的特性：
  - 无状态
  - 自动处理会话
  - 为每个会话部署专用资源
  - 封装会话上下文
- **22:30** - 内容保护和隐私是核心原则

### 智能代理应用示例 (23:00-25:00)
- **23:00** - 复杂任务示例：
  - 找到访谈中最精彩/最具病毒传播潜力的时刻
  - 创建适合社交媒体的竖屏剪辑
  - 烧录字幕
  - 自动导出带字幕的内容和元数据
- **24:00** - Memory 服务的应用：
  - 短期记忆：恢复过往会话上下文
  - 长期记忆：存储用户偏好（样式、文本生成偏好）
  - 代理可以学习用户偏好
- **24:30** - Observability 服务：监控代理会话中的所有活动，识别瓶颈，优化工作流

### 经验教训 (25:00-27:30)
- **25:00** - 开发 Media Copilot 的挑战和经验
- **25:30** - 在 SageMaker 中选择正确的部署模型至关重要：
  - 异步推理端点适合点播（VOD）
  - 直播需要实时或无服务器服务
- **26:00** - 分段视频处理：
  - 将视频分段并行处理
  - 减少推理时间
- **26:30** - 无服务器架构是关键成功因素

### 未来规划 (27:30-28:30)
- **27:30** - 直播实时处理：在直播进行时做出决策
- **28:00** - 开发专业化 AI 代理：
  - 使用更少工具的代理可以节省更多 token
  - 多个具有专业能力的代理效果更好
- **28:30** - 集成新模型，特别是视觉模型，为代理添加更多上下文

- **29:00** - 会议结束
