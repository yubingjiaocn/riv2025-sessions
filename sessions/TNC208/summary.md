# AWS re:Invent 2025 - 使用 AWS Transform 加速云迁移之旅

## 会议概述

本次技术分享会由 AWS 高级技术讲师 Ryan Forger 主讲,重点介绍了 AWS Transform 这一创新服务。AWS Transform 是 AWS 推出的首个代理式 AI（Agentic AI）服务,专门用于大规模的云迁移和现代化改造。该服务旨在解决企业在云迁移过程中面临的主要痛点:迁移过程缓慢、可扩展性受限、遗留系统知识库稀缺等问题。

Ryan 强调了迁移（Migration）和现代化（Modernization）同等重要的理念。迁移是将应用从本地环境转移到云端的过程,而现代化则是确保应用充分利用云原生特性、实现云优化的关键步骤。AWS Transform 通过生成式 AI 和代理式 AI 技术,能够自动化处理代码转换、依赖关系分析、迁移规划等复杂任务,显著提升迁移效率并减少技术债务。

会议展示了多个客户成功案例,包括 Thomas Reuters 每月转换 150 万行代码、The Hartford 迁移 .NET 应用程序、Toyota Motors 处理 4000 万行 COBOL 遗留代码等。这些案例充分证明了 AWS Transform 在实际生产环境中的强大能力和实用价值。

## 详细时间线与关键要点

00:00 - 开场介绍
- 讲师 Ryan Forger 自我介绍,担任 AWS 高级技术讲师和培训客户成功经理
- 专注于生成式 AI 和机器学习课程教学
- 说明这是一场"静音会议"(Silent Session),参与者通过耳机收听

02:30 - 迁移挑战分析
- 询问现场观众迁移经验,大多数人在迁移过程中遇到过困难
- 指出传统迁移的三大挑战:转型缓慢、可扩展性有限、遗留知识库稀缺
- 强调需要工具来弥合本地环境与 AWS 之间的差距

05:15 - AWS Transform 服务概述
- 介绍 AWS Transform 作为首个用于大规模迁移和现代化的代理式 AI 服务
- 强调迁移和现代化同等重要:迁移是转移过程,现代化是云优化过程
- 介绍三种主要代理类型:VMware 代理、.NET 代理、大型机代理

08:00 - AI 技术背景讲解
- 解释 AI 技术层次:人工智能 → 机器学习 → 生成式 AI → 代理式 AI
- 基础模型(Foundation Model)是在互联网规模数据上训练的大型机器学习模型
- 代理式 AI 通过结构化对话帮助用户更好地与 AI 交互,提供上下文以获得更准确的答案

12:00 - Transform 工作原理
- 通过聊天机器人进行结构化对话,引导用户完成迁移流程
- 针对不同迁移场景(VMware、大型机、.NET)提供专门的代理
- 代理收集必要信息后自动执行代码转换、依赖分析等任务

15:30 - 迁移三大领域
- 基础设施迁移:决定应用在云端的运行环境(EC2、容器、Lambda 等)
- 应用现代化:将单体架构重构为微服务架构
- 软件更新:保持系统安全、打补丁、版本升级

18:00 - Transform 核心能力
- 简化迁移规划:自动评估和识别需要迁移的内容
- 降低成本:提供财务预测,识别成本优化机会(如从 Windows 迁移到 Linux 减少许可费用)
- 提高效率:使团队更高效,减少重复性工作

20:45 - VMware 代理介绍
- 支持从本地 VMware 迁移到 AWS VMware 或 EC2
- 消除第三方许可费用
- 迁移后可利用云优化功能:CloudWatch 监控、Auto Scaling、Elastic Load Balancer

23:30 - 大型机代理介绍
- 传统大型机迁移平均需要 18 个月或更长时间
- Transform 可在几分钟内转换数千甚至数百万行代码
- 支持将 COBOL 代码转换为现代 Java 代码

25:00 - .NET 代理介绍
- 支持 .NET Framework 版本升级
- 本周更新:扩展为全栈 Windows 转换代理,不仅支持应用代码,还支持数据库迁移
- 自定义代码代理已在 US East 1 区域正式可用

27:15 - 开始使用 Transform
- 步骤 1:在控制台激活 Transform,配置 IAM 权限
- 步骤 2:需要 AWS Identity Center 进行身份验证
- 步骤 3:连接源代码仓库(如 GitHub)
- 步骤 4:Transform 执行评估、波次规划和代码转换
- 步骤 5:验证更改后自动执行

30:00 - 最佳实践
- 从小规模开始:先用 MVP 项目测试,积累经验后再扩大规模
- 不要跳过评估阶段:评估帮助构建技术和财务层面的业务案例
- 始终审查和测试:Transform 会自动生成单元测试

32:30 - 客户成功案例:Thomas Reuters
- 自 5 月以来每月转换 150 万行代码
- AWS Transform 团队成为其内部团队的延伸
- 减少了 70% 的技术债务

34:00 - 技术债务概念

- 旧系统越久越难迁移,需要更多人力和时间
- 减少技术债务是 AWS Transform 的最重要目标之一
- re:Invent 大会上象征性地炸毁了一个退役服务器机架

36:00 - 客户成功案例:The Hartford

- 已成功迁移 4 个核心项目

38:00 - 客户成功案例:Toyota Motors
