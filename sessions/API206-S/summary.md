# AWS re:Invent 2025 会议总结：利用 AI 优化供应链管理

## 会议概述

本次会议由 Chris Bingham 主讲，他是 AWS 大使兼富士通瑞士区的国家 CTO。会议重点介绍了富士通如何将 AI 技术应用于供应链管理这一实际业务场景，超越了传统的代码生成应用。供应链管理面临三大核心挑战：波动性（Volatility）、中断（Disruption）和转型（Transformation）。供应链管理涉及处理来自供应商、客户、海关、监管机构、制造到销售等多个维度和来源的快速变化数据，这种高复杂性、高维度的场景正是 AI 擅长的领域。

富士通的 Uvance 决策智能解决方案通过结合富士通的技术专长和客户的领域专业知识，为供应链团队提供 AI 伴侣式支持。第一代解决方案虽然能够帮助客户在地震等突发事件发生后 2 小时内量化影响并开始缓解措施（而竞争对手需要 3 天时间），但面临着"代理漂移"（Agent Drift）的技术挑战。为解决这一问题，富士通采用了 AWS Bedrock Agent Core 平台，并创新性地开发了"守护代理"（Guardian Agent）来监控和纠正 AI 代理的漂移问题。

通过部署 Uvance SCM 解决方案，客户的供应链团队效率得到显著提升，对供应链事件的响应速度大幅加快。借助 Bedrock Agent Core 的能力，新数据源（如新供应商）的集成速度提高了约一个数量级，从而增强了企业应对日常问题和重大不利事件的韧性。

06:30 - 08:00 - 第一代解决方案介绍
- 利用 AI 代理帮助平衡库存和管理不利影响及中断

- 客户能够在 2 小时内量化地震对供应链的影响并开始缓解

- 核心问题：速度
- 缺乏统一的代理生命周期管理平台
- 拼凑式架构导致"代理漂移"问题，影响响应质量和业务成果

- Agent Core 提供统一的代理生命周期管理平台

13:00 - 15:00 - Agent Core 架构组件详解
- Agent Core Runtime：为所有 AI 代理提供运行时环境
- Agent Core Gateway：集成所有供应链数据源和工具
- Agent Core Identity：集中管理访问控制
- Agent Core Observability：提供 AI 代理性能的深入洞察

15:00 - 17:30 - 守护代理（Guardian Agent）创新
- 利用 Agent Core Observability 创建守护代理
- 守护代理负责识别和纠正代理漂移
- 结合可观测性数据和供应链管理者反馈

17:30 - 19:00 - 人机协作机制

- 捕获业务绩效变化，独立于技术指标
- 守护代理结合业务反馈和思维链数据分析漂移点

19:00 - 20:00 - 自动化漂移修正流程
- 守护代理制定提示词修改方案
- 更新存储在 DynamoDB 提示词表中
