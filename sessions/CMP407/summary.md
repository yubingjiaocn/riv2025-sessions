# AWS re:Invent 2025 机密计算会议总结

## 会议概述

本次会议由AWS的Arvind和首席产品经理William Yap共同主持，深入探讨了AWS在机密计算领域的创新和发展。会议重点介绍了AWS如何通过Nitro系统实现"始终开启"的机密计算，以及针对AI工作负载推出的全新EC2实例证明功能。

演讲者详细阐述了机密计算的核心概念——保护使用中的数据，并展示了从传统的Nitro Enclaves到最新EC2证明功能的技术演进。会议特别强调了在AI时代，如何在GPU和AI加速器上实现机密计算，满足模型权重保护、多方计算等新兴需求。整个演讲涵盖了技术架构、实际应用案例以及客户成功故事，为与会者提供了全面的机密计算解决方案指南。

## 详细时间线与关键要点

### 0:00-8:00 会议开场与机密计算基础概念
- 演讲者介绍：Arvind和William Yap共同主持
- 会议议程概览：机密计算定义、工作负载需求、AWS解决方案
- 数据保护三个维度：静态数据、传输中数据、使用中数据
- 机密计算核心定义：保护内存中正在处理的数据
- 典型应用场景：解密操作、交易签名、令牌化、数据脱敏

### 8:00-15:00 客户需求分析与保护目标
- 敏感数据类型：个人身份信息(PII)、受保护健康信息(PHI)、金融资产、数字资产
- 客户核心需求：防止未授权访问、仅允许授权代码处理数据、最小化攻击面
- 资源灵活性要求：支持CPU密集型和内存密集型工作负载
- 两个关键考虑因素：保护什么数据（敏感数据）、防范谁（云提供商操作员、客户自身操作员）

### 15:00-25:00 AWS Nitro系统架构革新
- 传统虚拟化架构局限性：hypervisor承担过多功能，资源利用率低
- Nitro系统创新：将虚拟化功能抽象到专用硅芯片(Nitro卡)
- 架构优势：轻量级Nitro hypervisor、接近裸机性能、天然隔离
- "始终开启"机密计算：所有Nitro EC2实例默认提供操作员零访问保护
- 无需代码更改、无额外成本的机密计算能力

### 25:00-35:00 Nitro Enclaves深度解析
- 额外隔离需求：保护数据免受客户自身操作员访问
- Nitro Enclaves特性：无外部网络连接、无持久存储、无管理员访问、仅安全本地通道
- 身份证明机制：构建时获得参考哈希、运行时生成证明文档
- 证明流程：Nitro hypervisor签名、加密哈希比较、建立信任关系
- KMS集成：简化密钥管理和证明流程

### 35:00-42:00 客户成功案例与应用场景
- 多行业应用：广告技术、金融服务、Web3区块链、加密货币交易所
- 重点客户案例：Visa和MasterCard的实时支付处理、Coinbase的安全钱包解决方案
- 其他应用：Fireblocks托管解决方案、Stripe密钥管理、Brave奖励计算
- 本周相关会议推荐：支付处理、代理支付等专题讨论

### 42:00-48:00 AI工作负载新需求与EC2证明功能
- AI工作负载特殊需求：GPU/AI加速器支持、模型权重保护、更大带宽、存储访问
- 多方敏感数据：用户数据、模型权重、推理输出
- 三个保护目标：基础设施提供商零访问、服务提供商零访问、加密数据环境验证
- EC2实例证明功能：将Nitro Enclaves能力扩展到常规EC2实例

### 48:00-52:00 总结与后续资源
- 技术能力总结：Nitro系统、Nitro Enclaves、EC2证明功能
- 零操作员访问保证、接近裸机性能、全实例类型支持
- KMS集成简化、全商业区域和GovCloud支持
- 后续学习资源：代码讲座、实践工作坊、生成式AI可信执行环境创建
