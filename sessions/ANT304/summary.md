# AWS re:Invent 2025 数据分析会议总结

## 会议概述

本次会议由AWS数据分析技术负责人Taz和产品负责人Shika Verma共同主持，重点探讨了如何利用AWS数据基础设施构建AI代理应用程序。会议通过一个旅行预订代理系统的实际演示，展示了数据工程和数据分析如何成为AI应用的关键差异化因素。

演讲者强调，构建AI应用不需要全新的数据基础设施，而是应该充分利用现有的数据仓库、数据湖等资源。他们详细介绍了如何通过AWS的数据分析服务（如Amazon Redshift、S3、OpenSearch等）来管理上下文、实现事件驱动架构，以及确保数据治理和元数据管理。核心观点是：数据和数据工程是AI成功的关键，而AWS提供的AI就绪数据基础设施可以帮助开发者在不影响业务底线的情况下加速AI创新。

会议还特别介绍了模型上下文协议(MCP)服务器、Amazon Bedrock Agent Core、向量存储等新技术，展示了如何通过这些工具优化代理的性能、降低成本并提高响应速度。最后，演讲者强调了元数据管理在AI时代的重要性，指出元数据正在成为连接数据生产者和AI消费者的关键桥梁。

## 详细时间线

### 开场介绍
时间戳：开始
- Taz（AWS数据分析技术负责人）和Shika Verma（AWS数据分析产品负责人）介绍会议议程
- 议程包括：数据基础定义、AI代理应用的数据基础、AWS MCP服务器、上下文管理、事件驱动架构、数据治理和元数据管理

### 数据基础概念
时间戳：约5分钟
- 定义AWS数据基础：包括数据存储、数据处理、数据集成、数据治理等基本构造
- 强调数据工程是AI的关键差异化因素
- 观众互动：询问是否认为数据工程是AI的差异化因素

### 旅行代理演示 - 场景一
时间戳：约10分钟
- 演示航空公司预订代理应用
- 用户请求：查询从纽约JFK到洛杉矶的所有可用航班
- 代理功能展示：
  - 显示不同航空公司的航班时刻表和票价
  - 从第三方旅行社获取数据
  - 显示同事的用户评价
  - 检查公司差旅政策合规性
  - 提供符合政策的推荐航班

### 旅行代理演示 - 场景二
时间戳：约15分钟
- 用户选择预订美国航空航班（符合公司政策）
- 代理确认预订并使用用户偏好（餐食类型、座位偏好等）
- 执行后端操作（发送电子机票等）

### 参考架构介绍
时间戳：约18分钟
- 展示演示应用的参考架构
- 用户提示与代理交互，代理调用工具/API
- 集成第三方旅行社、内部数据仓库（Amazon Redshift）、数据湖（S3）
- 使用Claude和Strands Agents SDK构建

### React循环（代理循环）
时间戳：约22分钟
- 解释代理AI应用的核心：React循环（Reason + Act）
- 工作流程：用户指令 → 事件循环 → 迭代直到任务完成
- 挑战：循环意味着更慢的响应、不可预测的模型调用次数、更高的成本

### AWS MCP服务器
时间戳：约28分钟
- 介绍模型上下文协议(MCP)服务器的价值
- MCP让模型通过自然语言决定使用哪些工具
- 演示中使用的MCP服务器：Redshift MCP服务器和S3 Tables MCP服务器
- MCP的局限性：不适合大规模数据处理、复杂数据转换、需要确定性结果的操作

### 上下文管理
时间戳：约35分钟
- 上下文是模型的"工作记忆"
- LLM的两个关键限制：无状态、有限的上下文窗口
- 上下文窗口包含三类数据：
  1. 指令集（用户提示、系统提示、对话历史）
  2. 知识库（文档、FAQ、产品规格、最佳实践）
  3. 工具/MCP服务器的数据

### 代理记忆
时间戳：约40分钟
- 使用Amazon Bedrock Agent Core实现代理记忆
- 演示：代理记住航班可用性、预订信息、用户偏好
- 其他选项：ElastiCache for Valkey、DynamoDB、Neptune Analytics
- 代理记忆的作用：保留知识、提供个性化交互

### 知识库和向量存储
时间戳：约45分钟
- 演示使用公司差旅政策文档（PDF、HTML）构建知识库
- 使用Amazon S3存储，Amazon OpenSearch作为向量存储
- Amazon Bedrock Knowledge Bases提供端到端RAG工作流
- 宣布：OpenSearch向量引擎的GPU加速正式可用（性能提升10倍）
- 其他向量存储选项：Amazon S3 Vectors（本次re:Invent宣布正式可用）

### 实时数据处理
时间戳：约50分钟
- 变更数据捕获(CDC)事件结合流式处理
- 保持用户偏好等信息的最新状态
- AWS选项：Amazon Kinesis Data Streams、AWS Glue、Amazon Managed Apache Flink

### 多代理架构
时间戳：约52分钟
- 演示场景二：预订违反政策的航班需要经理批准
- 后端操作涉及多个代理：
  - 确认邮件通知代理
  - 公司政策规则触发代理
  - 交易日志记录代理

### 事件驱动架构
时间戳：约58分钟
- 多代理类似微服务架构：松耦合、异步
- 使用Amazon MSK（Managed Streaming for Apache Kafka）
- 优势：解耦代理任务、实现低延迟、内置可扩展性、持久化事件、多可用区部署

### 数据就绪性和元数据管理
时间戳：约62分钟
- Shika回到舞台讨论数据就绪性
- 强调：AI需要元数据，就像人类需要地图
- 元数据不是新概念，但在AI时代变得至关重要
- 数据来源：数据仓库、数据湖、湖仓一体等

### 元数据的作用
时间戳：约65分钟
- 元数据连接数据生产者和数据消费者（包括AI代理）
- 在数据丰富、数据质量检查、数据血缘追踪过程中生成元数据
- 元数据为AI代理提供必要的上下文

### Amazon SageMaker平台
时间戳：约68分钟
- 介绍Amazon SageMaker AI平台
- 扩展到包含所有数据分析和AI能力
- 成为数据分析和AI的中心平台
