# AWS re:Invent 2025 技术会议总结

## 会议概述

本次会议由Martin（应用产品负责人）和Jeff（资深软件开发工程师）主讲，主题为"在事件驱动架构中集成AI和智能代理"。会议以Unicorn Gaming Shop为案例，展示了如何通过Amazon EventBridge和Amazon Bedrock Agent Core来构建智能化的事件驱动系统。

Martin作为新加入的产品负责人，面临着50倍用户增长的挑战，需要自动化客户交互和后端问题解决流程。Jeff作为经验丰富的开发工程师，负责使用事件驱动架构现代化应用程序，并集成AI能力来应对规模化挑战。会议通过两个实际演示案例，展现了如何利用智能代理实现客户服务自动化和系统可靠性工程自动化。

## 详细时间线与关键要点

### 0:00-5:00 会议开场与角色介绍
- Martin和Jeff介绍各自角色和面临的挑战
- Unicorn Gaming Shop面临50倍用户增长压力
- 需要自动化客户交互和后端问题解决流程
- 目标：降低故障排除响应时间，减轻工程师负担

### 5:00-10:00 现有架构介绍
- Jeff展示基于EventBridge的事件驱动架构
- EventBridge三个核心组件：事件生产者、事件总线、规则
- 采用分层架构模式（Leave-in Layer Architecture）
- 现有系统可扩展但存在性能瓶颈

### 10:00-15:00 AI集成方案
- 引入Amazon Bedrock提供AI能力
- Bedrock提供第一方和第三方模型的API访问
- 构建研究助手聊天功能增强客户服务
- 系统仍保持无服务器、事件驱动、异步特性

### 15:00-20:00 智能代理架构
- 介绍三种代理类型：部门级、组织级、外部代理
- Amazon Bedrock Agent Core三大功能：
  - 工具和内存管理
  - 运行时环境和身份管理  
  - 可观测性和审计能力
- 代理具备自主理解、决策和执行任务的能力

### 20:00-25:00 集成模式
- 模式一：EventBridge触发代理
  - 事件流入EventBridge，触发Bedrock代理执行任务
  - 适用于游戏推荐等客户服务场景
- 模式二：代理触发EventBridge事件
  - 代理执行决策后产生事件，通过EventBridge扇出处理
  - 适用于系统监控和自动化运维场景

### 25:00-30:00 游戏推荐系统设计
- 智能代理处理客户聊天请求
- 情感分析功能：识别正面、中性、负面情绪
- 负面情绪自动转接人工客服
- 代理具备记忆功能，支持连续对话
- 节省处理时间和成本

### 30:00-35:00 站点可靠性工程自动化
- CloudWatch日志和告警集成
- SRE代理自动分析告警严重程度
- 低/中等严重度问题自动修复
- 高严重度问题通过SNS通知工程师
- 实现无人值守的故障处理

### 35:00-45:00 实际演示
- 演示一：客户游戏推荐系统
  - 展示Amplify前端界面和Python后端
  - 代理Gus处理客户查询和情感分析
  - 动态生成SQL查询，无硬编码查询语句
  - 支持拼写错误识别和上下文记忆

- 演示二：自动化运维系统  
  - CloudWatch告警触发EventBridge规则
  - Agent Core分析日志并确定问题严重程度
  - 自动化修复或升级至工程师处理
  - 完整的可观测性和版本管理

### 45:00-50:00 总结与问答
- 强调事件驱动架构与AI代理的完美结合
- 无需从零构建，可在现有架构基础上增强
- 实现客户、工程师和客服团队的多方满意
- 开放现场问答环节
