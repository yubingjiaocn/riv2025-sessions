# AWS re:Invent 2025 WhatsApp 业务消息会议总结

## 会议概述

本次会议由 Meta 的 Afi Ahmed 以及 AWS 团队的 Bipin Nepani 和 Veni Krishnan 共同主讲，重点介绍了如何通过 WhatsApp 平台扩展客户互动规模。WhatsApp 作为全球领先的消息平台，拥有 30 亿月活跃用户，每天有 6 亿次企业与客户之间的对话。会议强调了 WhatsApp 业务消息的独特优势——持久化对话线程，这种特性能够在企业与客户之间建立信任关系，从而提高消息的阅读率和转化率。

会议详细介绍了 WhatsApp 支持的四种主要消息类型：服务消息（用户发起）、实用消息（交易通知）、营销消息（促销优惠）和认证消息（一次性密码）。演讲者特别强调，使用多种消息类型的企业能够获得更好的客户互动效果，因为多样化的沟通方式有助于建立更深层次的客户信任。此外，会议还展示了 WhatsApp 与 AWS End-User Messaging 服务的原生集成，使 AWS 客户能够在 10-15 分钟内快速部署 WhatsApp 业务账户，无需编写代码即可实现复杂的消息自动化流程。

## 详细时间线

开场介绍 (0:00-2:30)
- 三位演讲者介绍：Afi Ahmed (Meta)、Bipin Nepani 和 Veni Krishnan (AWS)
- 会议主题：如何在 WhatsApp 上扩展客户互动规模

业务消息的演变 (2:30-5:45)
- 客户与企业沟通方式的历史演变：从面对面到电话、网站、应用，再到消息平台
- 个人案例：通过 WhatsApp 与理发师预约，展示持久化对话线程的优势
- 业务消息通过持久化线程丰富了所有传统沟通方式

WhatsApp 平台数据 (5:45-7:20)
- 全球 30 亿月活跃用户
- 每天 6 亿次企业与客户之间的对话
- 美国市场超过 1 亿月活跃用户，增长速度快于其他地区
- 企业需要在客户已经使用的渠道上与他们沟通

客户生命周期全覆盖 (7:20-10:15)
- 展示客户旅程飞轮：从认知、发现、考虑、购买、售后到再互动
- WhatsApp 支持营销、销售和客户服务等所有场景
- 多种消息类型的重要性：认证消息、实用消息、营销消息和客户服务消息
- 使用多种消息类型的企业获得更高的阅读率和行动率

平台互动指标 (10:15-11:30)
- 80% 的消息阅读率
- 50% 的点击率
- 30-35% 的转化率（客户行动率）
- 高互动性使 WhatsApp 成为企业触达客户的理想平台

数据隐私与安全 (11:30-12:45)
- 端到端加密
- 多层安全协议和程序确保最高级别的隐私保护
- 适用于企业与客户之间以及消费者之间的所有通信

AI 赋能对话 (12:45-14:30)
- WhatsApp 是 AI 对话的理想渠道，三大原因：
  1. 广泛的用户基础（30 亿用户）
  2. 丰富的功能特性（行动按钮、PDF 附件等）
  3. 丰富的合作伙伴生态系统
- AI 代理可以帮助企业大规模处理客户互动

四种消息类型详解 (14:30-16:00)
- 服务消息：用户发起的对话（如改签航班、查询包裹）
- 实用消息：企业发起的交易性消息（包裹到达、预约提醒）
- 营销消息：促销和优惠信息
- 认证消息：一次性密码验证

实用消息深入分析 (16:00-18:15)
- 应用场景：订单管理、支付提醒、预约提醒、欺诈警报
- 支持富媒体内容和附件
- 与 RCS 和 SMS 对比：支持富媒体、更长字符数、端到端加密

营销消息详解 (18:15-21:30)
- 客户需要选择接收（opt-in）
- 因为客户同意接收，互动可能性更高
- 新功能：营销消息优化
  - 考虑客户打开消息的可能性
  - 智能决策何时向谁发送消息
  - 印度研究显示：消息送达率提高 9%
- 类似广告引擎的优化机制

认证消息详解 (21:30-23:00)
- 一次性密码的多种应用场景：
  - 新账户创建
  - 账户恢复
  - 新交易验证
  - 现有交易确认
- 实例：英国 Deliveroo 通过 WhatsApp 发送验证码
- 与 RCS/SMS 对比：单一线程、复制代码、自动填充、无需切换应用、低延迟高可靠性

AWS 集成介绍 (23:00-25:30)
- Veni Krishnan 介绍 AWS End-User Messaging 服务
- AWS 的两大通信服务 API：
  1. Amazon Simple Email Service
  2. AWS End-User Messaging（支持 SMS、推送通知和 WhatsApp）
- 原生集成优势：使用单一 API 栈构建应用

多渠道通信策略 (25:30-28:00)
- 企业考虑消息传递的三个维度：
  1. 特定渠道的专属用户群体
  2. 消息的富媒体保真度
  3. 不同市场的最佳触达方式
- WhatsApp 优势：
  - 单一号码覆盖全球客户
  - 无需像 SMS 那样为每个市场配置长号码和短号码
  - 高保真度通信
  - 端到端加密的单线程对话

AWS 集成的技术优势 (28:00-30:15)
- 统一的 IAM 安全策略
- 应用 API 安全策略自动应用于消息基础设施
- 可交付性指标集成到统一基础设施
- 支持 AI 代理能力
- 构建客户信任的可信渠道
- 用户可以屏蔽企业，确保渠道的信任基础

集成实施步骤 (30:15-34:00)
- Bipin Nepani 演示集成过程
- 在 AWS 控制台中打开 End-User Messaging
- 选择社交消息服务
- 嵌入式注册流程，无需离开 AWS 生态系统
- 向导式流程：
  1. 连接 Facebook 账户（OAuth 认证）
  2. 选择或创建 Business Manager 账户
  3. 选择或创建 WhatsApp Business 账户
  4. 配置电话号码
  5. 验证号码
- 整个过程 5-10 分钟完成

编程集成选项 (34:00-36:00)
- 可与 Amazon SNS 或 Amazon Connect 集成
- 使用 AWS Lambda 实现事件驱动自动化
- 基于实时事件触发消息
- 像乐高积木一样灵活编排用例流程

监控与分析能力 (36:00-37:30)
- 开箱即用的分析和可观测性
- 自动扩展能力
- 实时更新自动化（电商事件、取消、调度更新）
- 使用 Lambda 和 SNS 实现"如果-那么"编排
- 完整的消息生命周期跟踪
- 细粒度的消息审计能力

案例研究 1：印尼 Siloam 医院 (37:30-40:30)
- 背景：WhatsApp 是印尼主要通信协议
- 之前通过电子邮件和电话进行客户沟通
- 将所有客户沟通迁移到 WhatsApp
- 成果：
  - 预约量增加 8%
  - 患者自助签到增加 36%
  - 客户服务响应时间从一周缩短到 24 小时内解决 80% 的案例
- 主要使用实用消息和营销消息

案例研究 2：巴西 PicPay 数字银行 (40:30-42:30)
- 背景：作为纯数字银行，需要在线完成客户入职和安全验证
- 问题：通过 SMS 发送 OTP 时客户完成率低
- 解决方案：将 OTP 迁移到 WhatsApp
- 成果：
  - 客户入职率提高 3.5%
  - 验证率提高 6-12%
- 专门使用认证消息类型

总结与行动号召 (42:30-44:00)
- 多种消息类型组合使用的企业获得最佳投资回报
- 行动建议：开始使用 WhatsApp Business
- AWS 客户可在控制台中 10-15 分钟内完成配置
- 展位 875 提供互动演示和更多案例研究
- 可体验不同消息类型的实际流程
