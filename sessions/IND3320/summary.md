# IND 3320: 加速AI - Sun Life如何降低延迟并提升用户参与度

## 会议概述

本次AWS re:Invent 2025分组会议由AWS专业服务首席顾问Salman Moghul主持，邀请了Sun Life的三位技术领导者分享他们在构建生成式AI应用方面的实际经验。Sun Life团队包括：产品负责人Robert Chen（Gen AI模式的开创者）、联络中心总监Dennis Wolowic（构建创新的多智能体协作工作流）以及高级首席工程师Walter Kurzat（负责在严格监管环境中设计和运营Gen AI模式）。

该团队最近使用Amazon Connect和Bedrock构建了名为"Iris"的全渠道Gen AI解决方案，并获得了CIO奖项。会议重点探讨了企业AI的演进如何带来独特挑战，特别是在复杂工作流中保持用户参与度的问题，以及如何通过实时流式传输和AWS AppSync等技术解决这些挑战。

## 详细时间线与关键要点

### 0:00-10:00 - 开场介绍与企业AI演进
- **0:00**: Salman Moghul开场，介绍Sun Life团队成员及其背景

  - 自主智能体（多步骤智能体工作流）

  - AWS API Gateway原生支持流式传输

  - 通过私有API实现灵活安全性

  - API模型的灵活演进

### 20:00-30:00 - AppSync安全性与架构模式
- **20:30**: 高度监管客户的安全要求：私有API访问、VPC端点、IAM策略

  - 简单Q&A机器人：直接调用Bedrock
  - 复杂异步模式：适用于长时间运行的工作流
- **27:00**: 异步模式架构：用户请求→AppSync变更→Lambda编排→MCP服务器→实时流式传输

- **43:00**: MCP服务器容器化选择：避免Lambda的无状态特性，使用EKS实现水平扩展
- **45:00**: 确定性智能体设计：
  - 身份验证场景需要可预测输出
  - 降低温度值、结构化JSON输出
  - 多智能体编排：礼宾智能体→身份验证智能体
