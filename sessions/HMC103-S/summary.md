# AWS re:Invent 2025 Nutanix 混合云解决方案技术会议总结

## 会议概述

本次技术会议重点介绍了 Nutanix 在 AWS 上的混合云解决方案，旨在帮助企业现代化基础设施并加速 AI 应用部署。演讲者强调了 Nutanix 如何通过统一的平台管理本地数据中心、边缘计算和云环境，解决企业在云迁移过程中面临的互操作性、安全性和成本控制等挑战。

会议涵盖了 Nutanix 在 AWS 上的全球可用性、多种实例类型支持、AI 推理优化、数据主权保护以及集群休眠等核心功能。演讲者特别强调了混合云架构的重要性，指出虽然 80% 的企业正在采用 AI 和 Kubernetes 等现代技术，但 80-90% 的工作负载仍运行在虚拟机上，因此需要一个能够无缝连接传统和现代工作负载的平台。

## 详细时间线与关键要点

### 0:00-2:00 开场与背景介绍
- 演讲者询问现场 Nutanix 和 VMware 用户情况
- 强调企业关注重点：AI 应用、工作负载现代化、成本优化
- 介绍 Nutanix 在现代混合云、应用构建和 AI 启用方面的价值主张

### 2:00-5:00 全球可用性与核心功能
- Nutanix 在 AWS 上提供 11 种不同实例类型
- 支持 GPU 模型、存储密集型和计算/内存密集型工作负载
- Intel AMX 技术支持 AI 推理优化
- Nutanix Sovereign 解决方案确保数据主权，适用于公共部门、金融和医疗客户

### 5:00-8:00 集群休眠与灾难恢复
- 集群休眠功能允许将环境存储在 S3 中，降低计算成本
- 灾难恢复场景下可在一小时内恢复完整集群
- 覆盖所有 AWS 区域和可用区，提供多种实例类型选择

### 8:00-12:00 技术架构详解
- 基于 Amazon EC2 Metal 实例运行，最小集群规模为三节点
- 在客户 VPC 内运行，自动应用现有安全策略
- 提供计算层（虚拟机和容器）、存储层和网络安全层
- 使用 NVME 存储，避免传统 IOPS 收费模式
- 支持 EBS 扩展和第三方存储集成

### 12:00-16:00 企业挑战与解决方案
- 引用 "80-2026 规则"：80% 企业使用 AI，但 80-90% 工作负载仍在虚拟机上
- 指出 50% 企业无法在 2029 年前实现云和 AI 愿景（Gartner 数据）
- 60% 采用 Kubernetes 的组织完全停滞
- 关键挑战：互操作性、安全性、双模式运营、成本控制、复杂性

### 16:00-19:00 产品组合与客户案例
- NC2（Nutanix Cloud Clusters）：混合云基础
- NCM（云管理）：统一管理 AWS 原生服务和本地环境
- NKP（Nutanix Kubernetes Platform）：企业级 Kubernetes 管理
- NAI（Nutanix AI）：企业 AI 解决方案
- NDB（Nutanix Database Service）：类似 RDS 的数据库服务
- 客户案例：52 分钟内完成集群部署，24 小时内完成工作负载迁移
