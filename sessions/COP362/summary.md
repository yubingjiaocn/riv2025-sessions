# AWS DevOps Agent 技术会议总结

## 会议概述

本次AWS re:Invent 2025分会场介绍了AWS DevOps Agent，这是一个在公开预览阶段发布的AI驱动的运维助手。会议由产品经理Bill和高级首席工程师David主讲，David在AWS拥有近20年经验，参与构建了DynamoDB、Lambda、IoT、CloudWatch等服务，并是AWS Builder Library的主要贡献者。

AWS DevOps Agent旨在帮助运维团队更快地解决事故并预防未来问题。该工具能够自动分析指标、日志和追踪数据，识别根本原因，并提供缓解步骤。通过与多种观测工具集成，包括Dynatrace、Datadog、New Relic和Splunk，该代理能够作为团队成员自主工作，响应告警和支持工单。

## 详细时间线与关键要点

### 0:00-10:00 开场介绍与产品概述
- Bill介绍David的背景，强调其在AWS DevOps文化建设中的重要作用
- 播放演示视频，展示AWS DevOps Agent如何在David收到告警时自动进行根本原因分析
- 宣布AWS DevOps Agent于昨日进入公开预览阶段
- 询问现场观众的运维经验，约半数参与者有值班经历

### 10:00-20:00 核心功能说明
- **事故解决**：分析指标、日志、追踪数据，查找相关部署，生成根本原因分析和缓解步骤
- **预防功能**：定期扫描历史事故，基于AWS最佳实践提供改进建议
- **四大特性**：
  - 作为团队成员运作，可响应ServiceNow工单和Slack通知
  - 遥测专家，集成多种观测工具
  - 管道专家，理解部署流程和代码仓库
  - 了解应用和组织结构

### 20:00-30:00 应用拓扑与客户案例
- 自动生成应用拓扑图，构建知识图谱映射应用环境
- 支持自定义运行手册和指导文件
- 内部使用数据：处理超过1000个真实事故，根本原因分析成功率达86%
- 客户案例：澳大利亚联邦银行用15分钟完成了原本需要5小时的网络事故分析

### 30:00-45:00 实际演示 - 缓存序列化问题
- David演示"机器人即服务"系统的故障排查
- 展示代码变更导致的缓存序列化不兼容问题
- 代理自动检测到Dynatrace告警，进行并行调查
- 识别CloudFormation部署、实例替换和代码变更的关联
- 生成详细的回滚计划，包括预验证、执行和后验证步骤

### 45:00-55:00 高级功能演示
- **自定义MCP服务器**：展示如何集成自定义日志存储系统
- **交互式调查**：通过聊天界面询问事故详情，如受影响的机器人数量
- **环境探索**：发现EC2实例异常流失问题
- **预防性建议**：分析19个历史事故，提供基础设施、治理和可观测性改进建议

### 55:00-57:30 总结与使用指南
- 公开预览阶段免费使用，每账户每月提供20小时调查时间和15小时预防时间
- 设置简单，仅需几次点击即可在AWS账户中部署
- 未来计划增加自动执行操作的能力
- 鼓励用户试用并提供反馈
