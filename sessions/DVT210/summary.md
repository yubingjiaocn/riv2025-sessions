# AWS re:Invent 2025 Java 应用现代化会议总结

## 会议概述

本次会议重点讨论了如何使用 AI 技术安全、高效地现代化传统 Java 应用程序。会议由 AWS 下一代开发者体验团队的专家以及 ADP 公司的实际案例分享组成。

会议首先识别了 Java 应用现代化过程中的六大挑战，被形象地称为"现代化怪兽"：文档龙、测试巨魔、模块怪物、版本吸血鬼、依赖恐龙和时间吸血蜘蛛。这些挑战使得传统的手动升级方式耗时且容易出错，往往需要数月甚至数年才能完成一个生产应用的升级。

为了解决这些挑战，AWS 推出了基于 AI 的解决方案。从最初的 Amazon Q Developer 中的 Java 转换代理，到本周刚刚发布的 AWS Transform Custom，这些工具能够自动化处理代码分析、依赖升级、测试修复等复杂任务，大幅提升开发者生产力。

## 详细时间线和关键要点

### 开场介绍 (00:00-05:00)
- **演讲者介绍**：Ben Coburn (AWS 首席市场专家)、Venu Vasu Devan (AWS 高级解决方案架构师)、Vijay Golola (ADP 应用开发总监)
- **现状调研**：约25%的与会者运行着超过20年的 Java 应用程序，这些应用诞生于云计算时代之前
- **核心议题**：现代化已不再是"是否要做"的问题，而是"如何安全、高效地实施"

### Java 现代化挑战分析 (05:00-15:00)
- **文档龙**：升级过程需要大量文档更新，传统应用往往缺乏高质量文档
- **测试巨魔**：新版本 Java 的测试方式发生变化，如 Java 9 后移除的内部 sun 类
- **模块怪物**：企业应用的复杂生态系统，一个模块升级可能引发连锁反应
- **版本吸血鬼**：版本间的破坏性变更，如 Java 11 中移除 JAXB
- **依赖恐龙**：需要找到与新 Java 版本兼容的第三方依赖版本
- **时间吸血蜘蛛**：整个过程极其耗时，导致项目积压和技术债务累积

### AI 解决方案介绍 (15:00-25:00)
- **Amazon Q Developer 中的 Java 转换代理**：
  - 本地项目验证和构建
  - 分析 JDK 运行时和依赖关系
  - 创建自定义转换计划供审查
  - 服务器端代码生成，本地验证构建
  - 提供文件差异审查功能
- **分块策略**：支持将大型项目分解为多个补丁进行处理

### ADP 实际案例分享 (25:00-45:00)
- **公司背景**：ADP 拥有75年经验，服务110万客户，为4200万员工发放薪资
- **项目规模**：全球薪资门户包含40多个 Java 服务，处理60万行代码
- **实施效果**：
  - **时间节省**：从手动需要3-4天缩短到3-4小时，总体节省60%时间
  - **安全提升**：85%的高危 CVE 漏洞得到修复
  - **准确率**：Spring Boot 应用几乎无缝升级，复杂 Tomcat 应用达到80-85%准确率
- **扩展策略**：通过卓越中心团队在组织内推广，最终升级了数百个应用

### AWS Transform Custom 发布 (45:00-70:00)
- **发布背景**：本周在 AWS re:Invent 2025 上正式发布
- **核心特性**：
  - 支持任意代码模式和编程语言转换
  - 命令行界面，可集成到 CI/CD 流水线
  - 持续学习能力，从用户反馈中改进
  - 支持自定义转换需求
- **工作流程**：
  - 规划代理：创建详细的转换计划
  - 执行代理：逐步执行并创建增量提交
  - 验证和自调试代理：确保目标达成并自动修复问题
- **MCP 集成**：支持与现有工具和系统集成

### 现场演示 (70:00-85:00)
- **演示一**：使用 AWS 管理的转换定义进行 Java 8 到 21 的升级
  - 展示了交互式命令行体验
  - 自动依赖分析和升级
  - 实时错误检测和自修复
- **演示二**：创建自定义转换定义
  - Java 21 现代化重构（使用 records、pattern matching、virtual threads）
  - Docker 文件更新
  - 交互式定义创建过程

### 总结和未来展望 (85:00-90:00)
- **技术优势**：从传统的特定用例代理转向通用可定制代理
- **组织影响**：显著提升开发者生产力，释放时间用于创新
- **未来方向**：支持更多编程语言和架构转换，如 x86 到 Graviton 迁移
