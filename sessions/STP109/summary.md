# AWS re:Invent 2025 技术会议总结

## 会议概述

本次会议由 Physics X 的首席研究科学家兼数学家 James Ley 主讲，重点介绍了如何为物理学（特别是汽车空气动力学）构建基础模型。Physics X 是一家专注于工业应用的 AI 公司，已完成超过 1.55 亿美元的 B 轮融资，拥有 200 多名工程师团队，涵盖 AI 研究人员、仿真工程师和软件工程师。公司与航空航天、国防、半导体、材料、能源、可再生能源、工业和汽车等领域的领先企业合作。

演讲展示了物理 AI 如何将传统计算流体动力学（CFD）仿真从 4 天的计算时间缩短到 6 秒的推理时间，这是一个根本性的变革。Physics X 正在构建预训练基础模型，通过大规模数据集训练，实现了更好的零样本性能和更低的客户前期数据需求。会议还展示了明确的扩展定律：更多数据带来更好的模型性能，特别是在汽车空气动力学领域的应用中，模型在分布外数据上的表现随着训练数据集的增长而显著改善。

整个系统依托 AWS 基础设施构建，利用 SageMaker HyperPod 进行仿真和模型训练，使用 S3 和 FSx for Lustre 进行数据存储和管理，并通过主动学习循环不断优化模型性能。这不仅是研究基础设施，更是生产级软件平台，旨在完全支持工程和物理领域的 AI 生命周期。

2:30 - 4:00 - 计算流体动力学（CFD）介绍
- 展示 CFD 仿真示例：模拟飞机周围的纳维-斯托克斯方程
- CFD 是从物理原型到数字仿真的重大转变
- 传统方法需要在风洞中测试物理原型
- 风洞仍然重要，但仿真大幅提升了创新速度
- 仿真仍是瓶颈，有进一步创新空间

4:00 - 5:00 - 物理 AI 的突破
- 传统仿真案例：7 秒瞬态仿真，超过 10 亿网格元素，需要 4 天计算
- 使用物理 AI：6 秒推理时间完成
- 从 4 天到 6 秒，不是渐进式改进，而是根本性变革
- 这将改变制造商的创新和设计方式

5:00 - 6:00 - 代理模型概念
- 模型被称为"代理"，因为它们替代昂贵的物理仿真
- 训练良好的代理模型可以提供足够准确的答案用于工程决策
- 计算成本仅为传统方法的一小部分
- 技术应用跨多个工程领域：车辆空气动力学、热管理的传热、反应化学、电子电磁学
- 不同领域的物理学不同，但基本方法相同：从仿真和实验数据中学习，做出快速准确的预测

6:00 - 7:30 - 构建 AI 代理的要求
- 需要前期进行昂贵的仿真以生成训练数据
- 需要存储、处理、清理数据并转换为 ML 模型可使用的格式
- 需要训练、验证和部署模型
- Physics X 构建的平台完全支持工程和物理的 AI 生命周期
- 包括数据管理、模型训练、微调，全部部署为智能体工作流

7:30 - 8:30 - Physics X 平台架构
- 仿真工作台：编排仿真
- AI 工作台：从 AI 模型目录开发和部署模型（部分预训练，部分未预训练）
- 统一数据骨干：企业安全性和多云可扩展性
- 强调这是生产级软件，不仅是研究基础设施

8:30 - 10:00 - 汽车空气动力学预训练模型
- 数值仿真是工程的第一次重大变革
- 仿真设置复杂，需要数小时或数天，但比物理原型更便宜快速
- 传统 ML 模型和代理：需要训练数据集、ML 模型和管道，可实现大规模设计空间探索
- 问题：每个客户、每个新几何形状都需要从头收集新数据集

10:00 - 11:30 - 预训练方法的优势
- 生成大型预训练数据集
- 结合基础架构
- 需要严肃的基础设施：从仿真到数据加载器，使用动态数据并行进行大规模训练
- 预训练基础模型的优势：客户前期数据需求更低，零样本和微调的开箱即用性能更好

11:30 - 14:00 - 数据集挑战
- 缺乏大型多样化的数据语料库
- 最大的公共数据集：DrivAerNet++（MIT Decode Lab 的 Faez Ahmed 贡献）
  - 8,000 个雷诺平均纳维-斯托克斯（RANS）仿真
  - 稳态时间平均近似，是最快的仿真类型
  - 基于 8 辆汽车的变形（拉伸和拉动不同部分）
  - 拓扑结构非常相似
- Luminary Shift SUV 数据集：
  - 2,000 个分离涡模拟（DES）
  - 比 RANS 保真度更高
  - 仅 2 个基线汽车，各变形 1,000 次
- 总计约 10 辆汽车，1,000 次变形，多样性不足

14:00 - 15:00 - 其他挑战
- 不同保真度的仿真
- 不同的湍流闭合模型
- 是否应该在多样化的仿真集上预训练模型仍是开放研究问题
- 本次会议仅关注几何多样性

15:00 - 17:30 - 多样性的重要性实验
- 在基于注意力的架构上进行预训练（Transolver、Point Cloud Attention、Linear Attention 等变体）
- 网格非常大，需要使用适合的注意力变体
- 在 Luminary 和 DrivAerNet 两个公共数据集上训练
- 在代表性客户数据集（CustomerNet）上测试分布外性能
- CustomerNet：一个基线仿真，具有客户常见的复杂变形

17:30 - 19:00 - 性能指标
- 阻力系数：轿车的空气动力学阻力系数在 0.25-0.3 之间
  - 理解这个数字帮助汽车制造商优化燃油效率和电动车续航
- 压力系数：汽车表面的压力
  - 显示车辆表面每个点的压力变化
  - 前部高压区，尾流低压区
  - 这些差异产生阻力
  - 准确预测压力场可以理解设计的阻力原因

19:00 - 21:00 - 实验结果分析
- 阻力系数图：
  - Y 轴：Spearman 秩相关系数（衡量设计排名的准确性）
  - 绿色：在 DrivAerNet 上训练的模型
  - 橙色：在 Luminary 上训练的模型
  - 分布内性能相当好
  - 在 CustomerNet 上的分布外性能差几个数量级
- 压力系数图：
  - 较低值更好
  - 查看汽车表面压力系数的平均绝对误差
  - 显示类似的模式

21:00 - 23:00 - 多样化数据集的必要性
- 结论：需要生成非常多样化的数据集
- 数据集仍在增长中
- 将讨论如何通过主动学习获得更多新的多样化设计
- 当前数据：在 18,000 个仿真上训练，超过 120 个基线，固定入口速度
- 模型在 DrivAerNet 和 Luminary 公共数据集上仍表现良好
- 零样本分布外性能显著提升
- 这仅是零样本结果，不包括微调

23:00 - 25:00 - 扩展定律
- 研究问题：随着数据集增长，分布外性能是否改善？
- 图表分析：
  - Y 轴：分布外压力系数的平均绝对误差
  - X 轴：训练集大小
  - 观察到性能下降（误差减少）
- 问题：训练集大小是指变形还是基线？
  - 需要理解：应该对一辆车进行多少次变形？这是否能提高性能？
  - 当前：沿 X 轴每个基线约 100 次变形
- 主动学习将增加这些曲线的斜率

25:00 - 27:00 - 不确定性量化的重要性
- 准确性本身不足以支持工程应用
- 工程师需要知道何时信任模型
- 预测阻力系数时，0.01 的差异很重要
- 需要置信区间
- 右侧图表：显示阻力如何沿汽车长度从前到后累积
  - 通过积分表面的压力和剪切应力获得
  - 图表最右侧的最终数字是阻力系数
  - 蓝色带是不确定性区域
  - 最右侧的最终点代表最终阻力系数周围的置信区间

27:00 - 28:00 - 不确定性量化的双重作用
- 这是对模型的期望不确定性量化类型
- 不确定性量化不仅对最终使用至关重要
- 还需要知道模型在哪些设计点不确定
- 用于确定下一步应该选择哪些设计
- 这就是主动学习循环

28:00 - 31:00 - AWS 仿真和训练架构
- 起点：多样化几何形状
- 空气动力学仿真：
  - 使用 Siemens Star-CCM+
  - 使用基于 GPU 的求解器
  - 在 SageMaker HyperPod 上运行（与模型训练相同的集群）
- 输出：多样化的解析流场
  - 非常大的场数据集，数 TB
  - 存储在 S3（长期存储）
- 预处理：
  - 使用 AWS Batch 并行处理
  - 转换为模型可摄取的格式
- 模型训练：
  - 在 SageMaker HyperPod 上训练
  - 使用数据并行
  - 存储使用 FSx for Lustre

31:00 - 33:00 - 主动学习循环
- 模型具有不确定性量化
- 去年在 AWS 展示了大型生成几何模型
  - 可以生成不同设计的模型
- 不确定性 + 生成几何机制 = 驱动主动学习循环
- 流程：
  - 有一组基线几何形状来微调几何模型
  - 模型可以告诉我们设计空间的哪些部分最不确定
  - 获取这些几何形状
  - 反馈到循环中进行空气动力学仿真

33:00 - 35:00 - 瞬态仿真的未来
- 当前空气动力学仿真使用 Siemens Star-CCM+ 的 RANS（非瞬态仿真）
- 有大量证据表明在瞬态数据上训练更好
- 但要在瞬态数据上训练，需要在线数据加载器
- 这使得仿真器和模型在同一位置训练变得更加重要
- 即使为 RANS 构建此基础设施也为下一步发展奠定了基础

35:00 - 37:30 - AWS 架构详细说明
- 核心：SageMaker HyperPod
  - 使用 Slurm 进行作业管理
  - 控制器机器跨多个计算节点和仿真节点工作
- FSx for Lustre：
  - 文件系统层
  - 与 S3 双向同步
  - 数据加载器所在位置
- 网络架构有两个子网：
  - HyperPod 私有子网（HyperPod 集群）
  - 公共子网（托管 MLflow 仪表板，研究人员查看结果）
- 研究人员通过 AWS Session Manager 进入

37:30 - 39:00 - 总结与回顾
- 讨论了仿真以及 ML 代理如何实现重大飞跃，降低推理时间
- 讨论了预训练模型如何通过减少前期数据需求来加速客户工作流程
- 扩展定律正在出现

39:00 - 40:30 - 未来方向
- 从汽车空气动力学开始，将继续创新
  - 转向瞬态仿真等
- 将扩展到其他领域：
  - 雷达散射截面
  - 声学
  - 结构力学
- 问题：这将是一个模型还是多个独立模型？
  - 答案：两者都会做
  - 将训练同类最佳模型
  - 将研究扩大模型范围以实现多物理场时会发生什么
  - 与客户合作了解哪些模型最适合他们
