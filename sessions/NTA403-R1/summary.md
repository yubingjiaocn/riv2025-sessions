# AWS re:Invent 2025 高级RAG架构会议总结

## 会议概述

本次会议是AWS re:Invent 2025的400级技术深度会议，主题为"高级RAG架构"。会议由两位AWS解决方案架构师主讲：Vivek Mittal（来自坦帕，专注于AI/ML领域3.5年）和Pallavi Nargund（主要解决方案架构师，IT行业25年经验，AWS工作7.5年）。两位讲师都是AI/ML专家组成员，专门研究检索增强生成（RAG）架构。

会议采用代码演示为主的形式，重点解决RAG准确性问题。讲师们通过实际的Jupyter Notebook演示，展示了从基础RAG到自纠正智能RAG的完整技术演进过程。会议使用了虚构的Octank金融公司10-K报告作为演示数据，该数据完全由LLM生成，确保了演示的安全性和实用性。

## 详细时间线与关键要点

### 0:00-5:00 会议开场与背景介绍
- 讲师自我介绍和专业背景
- 现场互动：了解观众对RAG的熟悉程度和生产环境使用情况
- 确认观众在RAG准确性方面面临的挑战

### 5:00-10:00 会议议程与Amazon Bedrock Knowledge Bases概述
- 明确这是400级代码实战会议，主要进行代码演示而非PPT讲解
- 介绍Amazon Bedrock Knowledge Bases作为端到端RAG工作流的托管服务
- 强调会议目标：提升RAG架构准确性，特别是针对企业数据和自主系统查询

### 10:00-15:00 RAG基础概念回顾与挑战分析
- RAG三大组件：数据摄取、检索、生成
- 重点关注非结构化数据（文档、图像、视频）
- 标准RAG vs 高级RAG的适用场景分析
- 复杂应用场景下的挑战：大量文档、复杂查询、专有术语

### 15:00-25:00 高级RAG架构模式介绍
- **条件分支模式**：根据查询类型选择不同向量存储（如产品查询vs政策查询）
- **并行分支模式**：从多个数据源获取信息并综合处理（如制造业错误代码分析）
- **查询重构技术**：将复杂查询分解为多个子查询
- Amazon Bedrock Knowledge Bases的编排功能介绍

### 25:00-35:00 代码演示环境设置与基础RAG实现
- Jupyter Notebook环境配置
- 使用Octank金融公司10-K报告作为演示数据
- AWS凭证配置和Knowledge Base创建
- 基础RAG流程演示：retrieve API和retrieve_and_generate API
- 三个基础问题测试：HTM投资组合公允价值、主要业务部门、员工数量

### 35:00-40:00 基础RAG局限性展示
- 复杂查询测试：Octank Tower和举报人丑闻
- DCM（专有缩写）查询失败分析
- 识别标准RAG在处理复杂查询和专有术语时的不足

### 40:00-45:00 查询分解技术实现
- 配置编排功能中的查询转换参数
- 演示query_decomposition类型的应用
- 成功处理多部分查询的效果展示

### 45:00-50:00 查询扩展技术与自纠正RAG架构
- 使用DynamoDB存储术语词汇表
- Strands Agent框架实现查询扩展工具
- 自纠正智能RAG架构核心组件：
  - 中央代理协调多种技术
  - 上下文相关性检查
  - 响应质量评估（相关性、完整性、事实准确性）
  - 自动策略选择机制

### 50:00-52:30 其他高级技术概述与总结
- **数据摄取优化**：语义分块、基础模型解析、多模态解析、元数据标记
- **检索优化**：元数据过滤、重新排序、混合搜索
- 强调上下文相关性检查和检索相关性是提升RAG准确性的关键
- 会议资源分享和后续交流安排
