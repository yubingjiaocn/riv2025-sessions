# AWS re:Invent 2025 会议总结：使用企业数据为生成式AI提供基础支撑

## 会议概述

本次技术分享由Coveo产品关系团队的Nick Bordeleau主讲，重点探讨了如何通过企业数据为生成式AI提供基础支撑，特别是Coveo与AWS AgentCore的集成方案。Coveo作为企业搜索公司，已帮助Dell、Nvidia、Intuit、Vanguard等知名客户将生成式AI解决方案投入生产环境。

演讲涵盖了为什么LLM需要基础数据支撑、优秀检索器的特征、以及通过MCP（模型上下文协议）与AgentCore集成的技术架构。核心观点是，无论是传统的生成式AI还是智能体AI，都强烈依赖于数据和工具来确保准确性和相关性。

## 详细时间线与关键要点

### 0:00-2:30 开场介绍与公司背景
- 介绍演讲主题：使用企业数据为生成式AI提供基础支撑
- Coveo公司定位：企业搜索解决方案提供商
- 客户案例展示：Dell、Nvidia、Intuit、Vanguard等大型企业的成功应用

### 2:30-5:00 产品架构与AWS集成策略
- Coveo提供完整解决方案：内容索引、基础数据支撑、UI组件
- AWS集成选项：作为检索器与Bedrock、AgentCore、Q Business、Quick Suite集成
- 重点关注AgentCore集成方案

### 5:00-8:00 LLM基础数据支撑的重要性
- **事实准确性**：LLM需要可信数据源避免过时信息
- **上下文相关性**：需要用户历史、项目信息等个性化数据
- **可追溯性与信任**：通过源归属建立用户信任
- **动态知识更新**：弥补训练数据的时效性限制
- **减少幻觉**：企业数据对于准确回答至关重要

### 8:00-12:00 优秀检索器的核心特征
- **知识深度**：支持大规模数据检索，避免数据死角
- **上下文感知**：基于用户身份和权限个性化检索
- **相关性质量**：确保检索信息的准确性和相关性
- **执行速度**：快速检索以保证用户体验
- **格式支持**：支持文档片段、完整文档、链接等多种格式
- **简洁精确**：减少token成本，提高LLM处理效率

### 12:00-15:00 MCP工具集与技术架构
- **MCP工具集**：段落检索、答案生成、搜索和文档检索、完整文档检索
- **推荐架构**：Agent + LLM + 长短期记忆 + Gateway + Coveo MCP服务器
- **身份集成**：利用AgentCore身份提供商确保数据安全访问

### 15:00-17:30 实施要点与最佳实践
- **提示工程核心**：全局指令、基础数据支撑说明、记忆与检索区分、源引用要求
- **MCP描述优化**：
  - 使用标准工具命名（避免点号，使用下划线）
  - 简洁精确的工具描述（1-2句话）
  - 前置关键信息（如认证要求）
  - 区分模式与描述的不同用途
- **会议结束**：提供展位信息（1529号）、AI大师课程系列、调查反馈
