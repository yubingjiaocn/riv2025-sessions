# AWS re:Invent 2025 会议总结：使用企业数据为生成式AI奠定基础

## 会议概述

本次会议由Coveo公司产品集成团队的Nick Baldo主讲，主题聚焦于如何使用企业数据为生成式AI（GenAI）提供基础支撑，特别是与Amazon Bedrock Agent Core的集成应用。Coveo是一家企业搜索公司，已帮助Dell、Nvidia、Intuit和Vanguard等多家大型企业将GenAI解决方案投入生产环境。

会议深入探讨了为什么大语言模型（LLM）需要基于企业数据进行"接地"（grounding），以及如何通过检索增强生成（RAG）技术来减少幻觉、提高准确性。演讲者强调，无论是简单的GenAI应用还是复杂的AI代理系统，都离不开对企业数据的依赖。Coveo提供了完整的解决方案，同时也通过MCP（Model Context Protocol）服务器的形式，将其平台作为检索器（retriever）与AWS Bedrock、Q Business等服务集成，为客户提供灵活的构建选项。

会议的核心内容围绕"秘密武器"——提示工程（prompting）展开，详细讲解了如何通过精心设计的提示词和MCP工具描述，让AI代理能够正确、高效地使用企业数据检索工具。这对于构建可靠的企业级AI应用至关重要。

03:45 - Coveo平台架构说明
- 左侧：完整解决方案（索引、接地、提示构建、UI组件）
- 右侧：作为检索器的平台集成方案
- 提供与Bedrock Agent Core、Q Business和QuickSight的集成点

05:20 - AI代理技术的未来趋势
- 引用AWS的观点：代理技术是LLM发挥全部潜力的关键
- 强调无论何种AI应用形态，都对数据和工具有强依赖性

06:30 - 为什么LLM需要接地（Grounding）
- **事实准确性**：LLM训练数据有时效性，需要提供可信的信息源
- **上下文相关性**：需要提供项目信息、用户历史等个性化数据
- **可追溯性和信任**：通过源引用让用户验证信息来源
- **动态知识更新**：提供最新的企业数据，而非依赖过时的训练数据
- **减少幻觉**：这是接地的"圣杯"目标，对企业应用至关重要

10:15 - 优秀检索器的特征
- **知识深度**：能够检索大量数据，支持多轮对话和问题细化
- **上下文感知**：了解用户身份、权限和历史，实现个性化检索
- **相关性质量**：这是最重要的特征，因为检索结果直接影响LLM的输出质量
- **执行速度**：检索发生在推理之前，需要快速返回结果以改善用户体验
- **格式支持**：支持段落、链接、完整文档等多种格式
- **简洁精确**：返回最相关的信息，减少token消耗，降低成本

15:40 - Coveo的MCP检索工具集
- 段落检索（Passage Retrieval）：从文档中提取相关段落
- 答案生成（Answer Generation）：将Coveo作为问答代理使用
- 搜索和文档检索：提供结果列表供用户探索
- 完整文档检索：用于深度研究和复杂问题

17:20 - 推荐的代理架构
- 中心：使用LLM的代理
- 顶部：长期和短期记忆
- 通过Gateway连接到Coveo MCP服务器
- 侧面：通过MCP服务器提供的各种工具
- 集成身份提供者，确保基于用户权限的安全访问

19:45 - 秘密武器：提示工程
- **元提示（Meta Prompt）设计**
  - 全局指令：定义代理的角色和主要任务
  - 明确说明接地、记忆和源引用的使用方式
  - 区分记忆和检索器提供的新鲜信息
  - 如果需要源引用，必须明确说明

22:30 - MCP工具描述最佳实践
- 使用标准命名约定（避免使用点号，使用下划线或短横线）
- 提供简洁的工具描述（1-2句话）
- 前置关键信息（如"需要身份验证"）
- 使用明确的动词和对象类型
- 区分Schema和Description的作用：Schema用于运行时调用，Description用于LLM决策
