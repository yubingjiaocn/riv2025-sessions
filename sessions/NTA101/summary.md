# AWS re:Invent 2025 - AWS Marketplace 会议总结

## 会议概述

本次 AWS re:Invent 2025 分会场由 AWS Marketplace 团队的高级客户专家 Carla Bogs 和国际扩展团队负责人 Phil Sloan 主持，特邀嘉宾 Riot Games 战略采购总监 Ma Olam 分享了实际应用案例。会议重点探讨了 AWS Marketplace 如何帮助企业加速创新，在保持控制力的同时提升软件采购速度。

演讲者强调了传统软件采购流程中的痛点：采购周期过长、软件支出可见性不足、管理开销大、合规审查成为瓶颈等问题。AWS Marketplace 通过提供精选的数字化目录，整合了软件、数据、服务和 AI 解决方案的发现、测试、购买和部署全流程。根据 Forrester 研究，客户在产品发现和采购流程中可节省 70% 的时间，部署效率提升使上市时间缩短 30%。云市场正成为增长最快的市场渠道，预计到 2030 年将达到 1630 亿美元规模。

会议还发布了多项重要功能更新，包括 Agent Mode（AI 智能搜索助手）、AI Agents and Tools 新类别，以及 Multi-Product Solutions（多产品解决方案）功能。Riot Games 的案例展示了如何通过 Marketplace 将采购周期从超过 100 天缩短至 30 天以内，并在一年内实现了 900% 的支出增长。

- 支持灵活支付和自定义合同条款
- 快速、安全的部署，直接集成到 AWS 环境
- 集中化治理，为财务、采购和安全团队提供可见性

- 适应客户在 AWS 旅程中的不同阶段

[00:27:15 - 00:29:00] - 灵活的部署模式
- 支持快速启动的解决方案
- 完全托管的服务

[00:29:00 - 00:31:00] - 增强的安全、治理和控制

- 跟踪软件使用、支出和合规性

### 灵活的交易方式
[00:31:00 - 00:35:30] - Phil 介绍不同角色的使用方式
- 约 80% 的交易是自助服务

- 2017 年推出 Private Offers（私有报价）功能
- 支持采购团队协商定制条款和多年合同
- 随后推出 Channel Partner Private Offers（渠道合作伙伴私有报价）
- 支持通过可信渠道合作伙伴和系统集成商购买

[00:35:30 - 00:38:00] - 本地化采购体验
- 今年在日本和瑞士推出税务自动化
- 在韩国和印度引入 Marketplace 运营商协助本地税务发票
- 推出新语言支持，帮助用户以母语发现产品
- 扩展端到端货币支持至五种货币，消除外汇风险

### 本周重要功能发布
[00:38:00 - 00:42:00] - Agent Mode（AI 智能搜索助手）
- 解决软件采购信息碎片化和流程复杂的问题
- 在 Marketplace 中心提供类似聊天的简单交互界面
- 用户可描述需求或业务成果（如零信任安全解决方案、AI Agent 构建工具）
- 可上传技术需求文档或业务需求文档
- AI 会搜索 30,000 个产品列表并提供推荐
- 整合外部信息、评论和合规记录
- 提供动态并排比较功能
- 可在聊天窗口中进一步提问和细化需求
- 生成提案供内部利益相关者传阅
- 显著减少内部讨论周期

[00:42:00 - 00:45:30] - AI Agents and Tools 新类别
- 几个月前在纽约峰会推出
- 响应客户向 Agentic 世界转型的需求
- 包含来自 Asana、Automation Anywhere、IBM、Perplexity、PWC、ServiceNow 等的数千个产品
- 支持多种部署选项：SaaS、API 访问、容器化
- 与 Amazon Bedrock Agent 集成
- 不仅包含 Agent，还包含构建工具（如 MCP 服务器、护栏）
- 以及系统集成商的服务
- 购买行为显示灵活性：从 1 万美元的自助服务工具到数百万美元的 Agent 私有报价

[00:45:30 - 00:49:00] - Multi-Product Solutions（多产品解决方案）
- 本周推出的重大功能
- 业务领导者从单一产品采购转向综合业务线解决方案
- 改变了 Marketplace 的交易方式
- 之前每个产品需要单独交易
- 现在可将多个产品打包成一个简化的交易
- 整合渠道合作伙伴、私有报价、专业服务等核心功能
- 单一合同实现明确的业务成果
- 发布时已有丰富的产品支持
- 特别适合系统集成商提供成熟的解决方案

### Riot Games 客户案例
[00:49:00 - 00:52:00] - Riot Games 公司背景
- 2006 年成立，使命是"让玩家体验更好"
- 2009 年推出《英雄联盟》，革新了在线游戏
- 现在是多游戏公司（Valorant、云顶之弈等）
- 每月支持全球数亿玩家
- 制作获奖动画作品如《Arcane》
- 支持 20 多种语言的多个游戏标题
- 需要最好的软件工具且要快速获取

[00:52:00 - 00:55:30] - 传统采购流程的挑战
- 团队识别需求后提交采购工单
- 采购团队开始审核和识别供应商
- 信息安全团队介入，验证过程需 4-6 周
- 法务团队介入进行合同谈判
- 最终工程师才能使用工具
- 完成流程时业务需求可能已改变
- 更糟的情况是工程师找到变通方法，产生影子 IT 问题

[00:55:30 - 00:58:00] - 采购团队面临的具体问题
- 花费大量时间在供应商识别上
- 追踪供应商、检查合法性、初步评估
- 这些是战术性工作，非战略性增值
- 需要翻转这个比例
- 应专注于验证工具是否解决痛点和业务需求
- 应专注于高价值活动如谈判获得最佳条款
- 不想扮演侦探角色验证 SaaS 供应商是否合法

[00:58:00 - 01:01:30] - AWS Marketplace 带来的改变
- 显著缩短预审供应商的识别时间
- 不需要在互联网上搜索可能满足需求的供应商
- 通过筛选器和 Agent 工具快速找到符合要求的供应商
- 例如找到支持 SOC 2 认证的 Live Ops 供应商
- 解锁工程团队启动试用
- 在试用完成时就准备好进行更广泛的搜索
- 将采购周期从超过 100 天加速到 30 天以内

[01:01:30 - 01:04:00] - 实施策略
- 没有强制要求必须通过 Marketplace 购买
- 而是让 Marketplace 成为阻力最小的路径
- 对于新软件请求：如果在 Marketplace 上有，就在那里购买
- 对于续约：采取务实方法，能在 Marketplace 购买就购买，不能就记录原因
- 关键是展示价值
- 让团队看到可以找到工具、启动试用、开始 POC
- 一旦展示了价值，采用就自然发生了

[01:04:00 - 01:06:30] - 采用成果
- 没有只是试水，而是像孩子跳水一样"炮弹式"全面投入
- 一年内通过 Marketplace 的支出增长超过 900%
- 2021 年开始旅程，只有小部分软件支出通过 Marketplace
- 2022 年有了"顿悟时刻"
- 2023 年实现三位数增长
- 2024 年又增长 20%（处理最后的落后者）
- 这不仅是采用，而是 Riot 软件采购方式的转型

[01:06:30 - 01:09:30] - 为不同团队带来的价值
- 采购团队：供应商已预审，不浪费时间验证供应商是否真实
- 专注于验证工具是否满足需求
- 验证业务案例是否合理
- 进行战略谈判，确保为 Riot 获得最佳条款
- 对于小型/早期公司：更大的价值
- 可以利用 Marketplace 的预审供应商
- 将原本投资于 TPRM 工具的资金重新分配到追逐市场机会
- 等公司发展到需要多种工具时再投资 TPRM 平台

[01:09:30 - 01:11:00] - 未来展望
- Agent Mode 看起来非常令人兴奋
- 未来 12 个月目标：90% 的软件支出通过 Marketplace
- 不仅是 SaaS 软件，还将购买机器学习模型
- 将购买容器化镜像
- 通过 Marketplace 利用的技术资产类型将扩展
- 优化采购等运营流程使 Riot 能够专注于为玩家带来下一个激动人心的游戏
