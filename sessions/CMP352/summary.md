# AWS re:Invent 2025 - CMP352: 使用 AWS Graviton 节省成本仪表板实现成本效率

## 会议概述

本次会议主题为"使用 AWS Graviton 节省成本仪表板实现成本效率"（CMP 352），由 AWS 高级云 FinOps 架构师 Rosa Corley 和高级技术客户经理 Reji Guptan 主讲。会议重点介绍了 Graviton 节省成本仪表板这一可视化工具，帮助企业发现和最大化 AWS Graviton 处理器带来的成本优化机会。

Graviton 处理器相比 x86 实例提供了 40% 更好的性价比和高达 60% 的能效提升。该仪表板作为中央指挥中心，集成在云智能仪表板框架中，利用 Amazon QuickSight 提供清晰可操作的洞察。它支持多个 AWS 服务，包括 EC2、RDS、ElastiCache 和 OpenSearch，为 FinOps 专业人员、开发人员等多种角色提供全面的可见性。仪表板提供三个关键视角：已实现的节省、潜在机会和详细分解，帮助用户优先考虑优化工作。

会议通过实际演示展示了仪表板的四大核心功能：摘要视图、Graviton 代际比较、基于性能的成本建模工具和标签可见性。演示案例显示，一个组织当前 Graviton 覆盖率仅为 26.3%，每月已节省 2 万美元，但通过迁移剩余工作负载，基础节省潜力可达 6.15 万美元，如果考虑 15% 的性能提升，总节省潜力可增至 10.3 万美元。

10:00 - 摘要标签页演示
- 提供跨不同服务的 Graviton 已实现节省和成本优化机会的高级概览

- 演示数据显示 EC2 具有最高的节省潜力

11:00 - EC2 标签页深入分析
- **当前使用和节省部分**：
  - 显示已实现的 Graviton 节省：每月 2 万美元
  - 当前覆盖率仅为 26.3%
  - 意味着近四分之三的工作负载仍可从迁移到 Graviton 中受益

12:00 - Graviton 机会部分
- 潜在的 Graviton 基础节省：每月 6.15 万美元
- 这是通过迁移到相同大小和数量的可比 Graviton 实例获得的节省

12:45 - Graviton 代际选择功能
- 使用顶部的单选按钮在不同 Graviton 代际之间切换
- 选项包括：默认（下一代 Graviton）、Graviton 2、Graviton 3 或 Graviton 4
- 切换代际时，节省数据会自动更新和重新计算
- 适用于已在特定 Graviton 实例上测试应用程序或希望跨基础设施标准化的情况

14:00 - 性能增益建模
- 引入滑块以输入标准化实例小时数（NIH）的减少
- NIH 是 AWS 跨不同实例大小标准化计算的方式
- 演示案例：假设 Graviton 性能提升 15%
- 输入 15% 后，总潜在节省从 6.1 万美元增加到 10.3 万美元
- 额外的性能节省：每月 4.2 万美元

16:00 - Graviton 节省详细表格
- 用于定位工作负载和识别可重点实施节省的实例类型
- 每行代表一个节省机会
- 包含关键指标：账户信息、购买选项、操作系统、简单的节省实施工作量估算
- 可以使用顶部过滤器对所有属性进行过滤

17:00 - 过滤功能
- 表级过滤
- 使用顶部控制选项进行过滤
- **基于标签的过滤**（重点功能）：
  - 允许将自己的成本分配标签引入仪表板
  - 可以按特定工作负载、业务单元或成本分配标签进行过滤
  - 演示示例：环境、业务单元和公司标签

19:00 - 其他服务特定标签页
