# AWS re:Invent 2025 - STG339 会话总结

## 会话概述

本次会话主题为"使用事件驱动的 SFTP 帮助客户现代化其托管文件传输（MFT）系统"，由 AWS Transfer Family 产品管理团队的 Smith Shriram、Sue 和首席解决方案架构师 Pierre 共同主讲。

会话重点介绍了如何使用 AWS Transfer Family 和 AWS B2B Data Interchange (B2BI) 服务来构建现代化的托管文件传输解决方案。演讲者强调了 MFT 在各行业中的重要性，包括支付对账、医疗保健理赔处理、供应链管理、监管文件提交等场景。AWS Transfer Family 作为一个完全托管的服务，支持 SFTP、FTP、FTPS 和 AS2 等行业标准协议，帮助客户消除基础设施管理负担，实现高可用性和实时扩展。

会话还展示了多个客户成功案例，如 FICO 通过迁移到 AWS Transfer Family 降低了 TCO 并减少了基础设施开销；BMW 集团利用该服务将生产线的摄像头数据传输到 S3 进行 AI 分析。此外，演讲者详细介绍了 2025 年发布的多项新功能，包括 IPv6 支持、VPC 端点增强、FSx for NetApp ONTAP 集成、Terraform 模块支持等，并通过实际演示展示了如何使用 Agentic AI 构建智能化的保险理赔处理系统。

- **00:00 - 02:30** - 开场互动，Smith Shriram 介绍自己和团队成员，询问观众使用 SFTP、MFT、AS2 和 EDI 的情况
- **02:30 - 04:45** - 介绍会话议程：MFT 和 EDI 概述、行业趋势、功能深入讲解、演示和总结

### MFT 的业务价值与应用场景
- **04:45 - 07:20** - 阐述数据移动性的重要性，介绍 MFT 在不同行业的应用：支付对账、医疗保健理赔、数据集销售、内部流程自动化（HR、财务、工资）、监管文件提交（FDA、SEC）、供应链和物流
- **07:20 - 09:15** - 介绍 MFT 的四大支柱：行业标准协议支持、身份验证和访问控制、处理和自动化、强大的审计功能

### 行业趋势与 AWS Transfer Family 介绍
- **09:15 - 11:30** - 讨论六大趋势：安全性、治理工具、简化基础设施管理、自动化、实时分析、AI 就绪
- **11:30 - 14:00** - 介绍 AWS Transfer Family 的核心特性：完全托管、高可用性、实时扩展、支持无缝迁移、事件驱动架构、内置合规性（HIPAA、PCI 等）
- **14:00 - 15:30** - 介绍 AWS B2B Data Interchange 服务：托管 EDI 服务、X12 文档的翻译和验证、使用生成式 AI 生成映射代码

### 服务资源与客户案例
- **15:30 - 17:00** - 介绍 Transfer Family 的三种资源类型：SFTP 服务器、SFTP 连接器、Web 应用程序
- **17:00 - 19:30** - FICO 客户案例：作为全球信用评分和分析领导者，FICO 通过迁移到 AWS Transfer Family 消除了大量基础设施、缩短了部署时间、降低了 TCO
- **19:30 - 21:00** - BMW 集团案例：使用 Transfer Family 将生产线摄像头数据通过 SFTP 传输到 S3，支持视觉和音频分析服务，存储了 130 万个文件（1.3 PB）

### 合作伙伴案例
- **21:00 - 23:00** - 介绍服务交付合作伙伴计划，**Scale Capacity** 帮助洛杉矶市迁移 MFT 并节省大量成本
- **23:00 - 25:30** - BisCloud Experts 案例：帮助供应链和物流客户迁移 EDI 系统，管理 100+ 贸易伙伴和每月 25 万条消息，年节省超过 100 万美元，缩短了贸易伙伴入驻时间

### 功能深入讲解（Sue 主讲）
- **25:30 - 27:00** - Sue 接手，介绍 2025 年的新功能发布（以粉色标注）

#### 服务器端点与网络控制
- **27:00 - 30:00** - Transfer Family 服务器介绍：完全托管端点、支持 SFTP/FTP/FTPS/AS2 协议
- **30:00 - 33:00** - 网络控制选项：公共端点（简单设置、高可用性）、VPC 端点（内部访问、安全组控制）、面向互联网的 VPC 端点（结合两者优势）
- **33:00 - 34:30** - 新功能：IPv6 支持，适用于 VPC 和公共端点，支持双栈端点

#### 身份验证模式
- **34:30 - 37:00** - 三种身份验证模式：服务托管（基于密钥）、AWS Directory Services 集成、自定义身份提供商（Custom IDP）
- **37:00 - 39:00** - Custom IDP 的两种部署模型：直接调用 Lambda、通过 API Gateway（支持 WAF）
- **39:00 - 40:30** - 多方法身份验证支持：仅密钥、仅密码、密钥或密码、密钥和密码
- **40:30 - 42:00** - 新功能：动态切换身份提供商模式，零停机时间

#### 自定义身份提供商解决方案
- **42:00 - 45:00** - 介绍开源的 Transfer Family Custom IDP 解决方案：使用 SAM 模板部署、支持多种流行 IDP（Cognito、Okta、Entra 等）、内置安全最佳实践（每用户 IP 白名单）
- **45:00 - 46:30** - 新功能：Terraform 支持，包括 Custom IDP 的 Terraform 模块

#### 存储集成
- **46:30 - 48:00** - 存储选项：Amazon EFS、Amazon S3、S3 访问点（细粒度权限控制）
- **48:00 - 51:00** - 重大新功能：S3 访问点支持 FSx for NetApp ONTAP，实现混合访问模式（外部用户使用 SFTP，内部用户使用 NFS/SMB）
- **51:00 - 52:00** - 支持的操作：上传、下载、删除、复制；限制：不支持重命名和追加，上传文件大小限制 5GB

#### SFTP 连接器
- **52:00 - 54:30** - SFTP 连接器介绍：完全托管的 SFTP 客户端，连接到远程 SFTP 服务器
- **54:30 - 56:00** - 连接器操作：发送文件（start file transfer）、检索文件、目录列表、远程文件操作（移动、删除、重命名）
- **56:00 - 58:30** - 新功能：VPC 支持，可连接到私有 SFTP 服务器（共享 VPC、本地或合作伙伴环境）
- **58:30 - 60:30** - VPC 连接器设置：使用 VPC Lattice 创建资源网关和资源配置，无需修改现有 VPC 配置

#### Transfer Family Web Apps
- **60:30 - 63:00** - Web Apps 介绍：为非技术用户提供托管 Web UI，通过 IAM Identity Center 进行身份验证，使用 S3 Access Grants 控制权限
- **63:00 - 65:00** - 设置简单：在控制台几次点击即可完成，支持自定义（公司 Logo、Favicon、页面标题）
- **65:00 - 67:00** - 合规性：HIPAA、FedRAMP、PCI 等，浏览器访问，拖放式操作

#### Web Apps 使用案例与架构
- **67:00 - 69:30** - 使用案例：财务团队交换报告、营销团队上传媒体资产、研究科学家协作数据集
- **69:30 - 72:00** - 工作流程：管理员创建 Web App → 分享 URL → 用户 SSO 登录 → 访问授权文件 → 直接传输到 S3 → CloudTrail 审计
- **72:00 - 74:30** - 新功能：VPC 端点支持，所有流量保持在 VPC 内，支持通过 Direct Connect 或 VPN 访问，可限制客户端 IP
- **74:30 - 75:30** - 两种部署选项：公共端点（外部协作者）、私有端点（敏感工作负载）

#### B2B Data Interchange (B2BI)
- **75:30 - 78:00** - B2BI 与 Transfer Family 结合使用的典型架构：贸易伙伴通过 SFTP/AS2 传输 EDI 文件 → Transfer Family 存入 S3 → B2BI 监听事件 → 验证和转换 EDI 文件为 JSON/XML → 输出到 S3 → 触发 EventBridge 事件 → 后处理操作
- **78:00 - 79:30** - 新功能：B2BI 扩展到欧洲首个区域（都柏林），现支持四个区域

### 演示环节（Pierre 主讲）
- **79:30 - 81:00** - Pierre 介绍演示内容：使用 Agentic AI 构建现代 MFT 系统
- **81:00 - 83:00** - 核心构建块：AWS Transfer Family（安全文件传输）、Amazon GuardDuty（恶意软件防护）、AI 代理（智能处理）、Amazon EventBridge（事件驱动架构）

#### 保险理赔处理用例
- **84:30 - 87:00** - 传统流程的三个阶段：摄取（文档上传到 SFTP）、提取（基础 OCR，但僵化）、分析（手动处理 + 规则引擎，容易出错且不可扩展）
- **87:00 - 89:00** - 现代化方法：安全文件传输 + 恶意软件扫描 + Agentic AI，实现端到端自动化

#### 现代化架构详解
- **89:00 - 91:30** - 第一阶段 - 现代化基础：用 Transfer Family 替代传统 SFTP 服务器（无基础设施管理、自动扩展、高可用）、使用 Custom IDP 验证外部用户（维修店、合作伙伴）
