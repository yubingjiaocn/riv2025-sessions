# AWS re:Invent 2025 数据安全会议总结

## 会议概述

本次会议由 Varonis 的现场 CTO Brian Veti 主讲,主题为"超越态势管理的数据安全"。Brian 拥有 16 年的网络安全从业经验,他在演讲中强调了一个核心观点:攻击者不是"闯入"系统,而是通过窃取身份凭证"登录"系统。数据显示,88% 的数据泄露事件都涉及被盗或被破坏的身份凭证。攻击者的最终目标始终是数据本身——无论是窃取、加密还是勒索数据。

Brian 指出,传统的数据安全态势管理(DSPM)和配置安全态势管理(CSPM)工具虽然重要,但已经不足以应对现代威胁。这些工具无法检测授权变化、横向移动、异常行为或内部威胁。他提出了一个完整的数据安全框架,包括身份保护、数据和配置安全态势管理、以数据为中心的用户和实体行为分析(UEBA),以及完整的取证能力。Varonis 平台正是基于这一理念构建,能够在 AWS 及多云环境中提供全面的数据可见性、自动化修复和威胁检测能力。

演讲最后,Brian 介绍了 Varonis 如何与 AWS 安全工具栈集成,包括 Security Hub、CloudTrail、IAM Identity Center 和 GuardDuty,并邀请与会者进行风险评估和产品演示。

## 详细时间线

00:00 - 开场介绍
- Brian Veti 自我介绍,Varonis 现场 CTO
- 拥有 16 年网络安全经验,之前从事 IT 和 IT 安全工作

00:30 - 核心观点:攻击者通过身份登录而非闯入
- 强调攻击者不会"闯入银行偷笔",他们的目标是数据
- 88% 的数据泄露涉及被盗凭证
- 窃取身份是进入基础设施最快、最简单的方式

01:15 - 攻击者的行为模式
- 使用被盗凭证登录后,利用爆炸半径进行横向移动
- 提升权限,获取管理员账户访问权限
- 建立持久性,创建新身份和账户
- 最终目标:访问、窃取和加密数据

02:00 - 真实威胁案例
- 提到针对 AWS 的勒索软件"Codefinger"
- 不利用 AWS 漏洞,而是利用身份漏洞
- 信息窃取恶意软件案例:通过 GitHub 上的游戏破解程序传播

03:30 - 多种攻击向量
- 钓鱼攻击
- 供应链妥协
- 高级持续性威胁(APT)
- 所有攻击的最终目标都是数据访问

04:15 - DSPM/CSPM 的局限性
- 无法检测授权变化
- 无法检测横向移动(跨 AWS、其他云平台或 SaaS 应用)
- 无法检测异常行为
- 无法检测当前活跃威胁或内部威胁

05:30 - 完整的安全框架需求
- 身份保护:监控所有账户(内部/外部、用户/机器、全局/应用身份)
- 数据和配置安全态势管理
- 以数据为中心的 UEBA(用户和实体行为分析)
- 完整的取证能力

06:45 - 安全团队的四大目标
- 阻止数据泄露
- 避免罚款
- 减少人力需求
- 证明合规性

07:15 - 身份保护的重要性
- 需要了解哪些是人类身份,哪些是机器身份
- 移除过度权限
- IAM 只是拼图的一部分

08:00 - 数据风险评估发现
- Varonis 在客户 AWS 环境中发现:高达 40% 的数据对所有能够认证的账户开放
- 类比:如果银行 40% 的钱对任何进门的人开放,会怎样?

09:00 - 安全修复的挑战
- 需要安全地实施最小权限原则
- 避免"瓷器店里的公牛"问题
- 必须了解身份的访问模式,避免在修复时破坏业务

10:00 - 以数据为中心的 UEBA
- 信用卡类比:信用卡公司知道你的位置、购买习惯和金额
- 当异常交易发生时(如在蒙特利尔加油),立即识别为欺诈
- 同样原理应用于数据访问监控

11:30 - 取证层的必要性
- 需要快速完整地回答"发生了什么?"
- 事件响应中的首要问题:哪些数据被触及?
- 次要问题:攻击者如何进入?哪个身份被破坏?权限如何变化?

13:00 - Varonis 平台介绍
- 20 年数据安全平台开发历史
- 覆盖文件、数据库、所有云平台、S3、RDS、EC2
- 支持本地数据中心和混合环境

14:30 - Varonis 的核心能力
- 提供可观测性,发现问题
- 跨云和混合环境准确分类数据
- 支持结构化/非结构化数据、SaaS 应用、数据库、文件、协作平台
- 分析所有访问控制、深层链接、权限和身份权限

16:00 - 自动化修复能力
- 不仅发现问题,还提供自动化修复
- 避免"发现疲劳"——数十万个问题清单无法解决
- 基于遥测数据安全地部署自动化
- 在修复前了解可能的影响

17:30 - 威胁检测能力
- 提供有用的 UEBA,减少噪音
- 生成少量高质量告警,包含丰富的上下文信息
- 锁定 AWS IAM,确保身份和基础设施正确配置
- 自动执行最小权限原则

18:45 - 与 AWS 安全工具集成
- AWS Security Hub:导入信号、遥测、分析和威胁,提供额外上下文
- AWS CloudTrail:丰富遥测流,添加额外信息和上下文
- IAM Identity Center:提供数据访问和权限应用的上下文
- AWS GuardDuty:增强以数据为中心的威胁检测

20:00 - Varonis 的覆盖范围
- 保护 AWS 生态系统中的数据
- 支持 SaaS 应用(Salesforce、GitHub)
- 支持数据存储(Snowflake、Databricks、RDS、EC2 中的数据库、S3)
- 跨云单一管理平面和统一策略

21:30 - 风险评估流程
- 15-20 分钟即可接入 AWS 环境
- 第一天即可看到结果
- 一周内识别真实风险
- 几周内提供降低风险的实际计划

22:30 - 结束语
- 邀请参会者到展位(430 号)了解更多
- 提供演示和风险评估
- 强调网络安全的最终目标是保护数据
- Varonis 能够跨云、准确、自动地实现数据保护
