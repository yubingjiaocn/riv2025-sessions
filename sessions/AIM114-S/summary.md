# AWS re:Invent 2025 - Agentic Commerce 会议总结

## 会议概述

本次会议由 Stripe 主讲,重点探讨了 AI 代理商务(Agentic Commerce)的新兴趋势。演讲者提出了一个核心观点:AI 代理正在成为新的店面,消费者的购物发现方式正在从传统搜索转向 GenAI 辅助代理。会议展示了 AWS 与 Stripe 如何合作,为这种新型商务模式提供即时、安全且可信的支付解决方案。

Stripe 作为互联网的金融基础设施平台,不仅提供支付处理,还涵盖订阅计费、税务自动化、市场支付、银行即服务等多项功能。通过与 AWS 云基础设施的深度合作,Stripe 实现了 99.999% 的可用性,在刚刚过去的黑色星期五和网络星期一期间处理了近 400 亿美元的交易。Stripe 的优化结账套件平均可为商家带来 12% 的收入增长,并通过创新的 API 和开发者工具将产品上线速度提升 30%。

会议深入介绍了 Stripe 今年推出的全球首个支付基础模型(Payments Foundation Model),该模型基于数百亿个关键信号进行训练,利用高维向量空间对支付数据进行聚类分析。这使得 Stripe 能够主动预防欺诈,而非被动应对。该模型在整个支付生命周期中发挥作用,从结账、身份验证、欺诈预防到退款和争议处理。实际应用中,95% 的卡测试攻击被实时检测,比之前的方法提升了 21%。DoorDash 通过使用智能争议功能将退单成本降低了 10%,Vimeo 和 Squarespace 的退单追回率提升了 13%。

## 详细时间线与关键要点

00:00 - 01:30 | 开场与核心论点
- 提出核心观点:AI 代理正在成为新的店面
- 消费者发现方式正在转向 GenAI 辅助代理
- 将展示两种代理商务模式以及 AWS + Stripe 如何确保支付的即时性、安全性和可信度

01:30 - 03:45 | Stripe 平台介绍
- Stripe 定位为互联网的金融基础设施,提供可编程的资金流动
- 服务范围:全球支付接受、订阅计费自动化、税务计算与报告、市场支付(DoorDash、Instacart、Lyft)
- 银行即服务产品:Stripe Issuing、Capital,以及稳定币领域的领先地位
- 在零售、媒体、娱乐、旅游、保险等多个垂直领域拥有 AWS 能力认证

03:45 - 05:15 | AWS + Stripe 合作成果
- 通过 AWS 云基础设施和 Stripe 工程团队实现 99.999% 的可用性
- 黑色星期五/网络星期一处理近 400 亿美元,保持 99.9999% 可用性
- 服务超过 400 万客户
- Stripe 优化结账套件平均带来 12% 的收入增长
- 通过创新 API 和开发者 SDK 实现 30% 更快的产品上线速度
- 通过欺诈数据模型降低风险

05:15 - 07:30 | 支付基础模型(Payments Foundation Model)
- 今年早些时候推出全球首个支付基础模型
- 基于数百亿个关键信号训练,利用全球经济数据
- 使用高维向量空间对支付进行聚类分析(相同国家、相同卡、相同地理位置、相同银行、相同邮箱等)
- 2024 年处理 1.4 万亿美元年度支付量
- 92% 的卡片曾被该模型识别过

07:30 - 09:00 | 支付基础模型的应用范围
- 覆盖整个支付生命周期:结账、身份验证、欺诈预防(Radar)、卡测试、退款和争议
- 实时检测 95% 的卡测试攻击,比之前方法提升 21%
- 智能争议功能:使用 GenAI 自动生成 10-20 页的争议文档
- DoorDash 实施后退单成本降低 10%
- Vimeo 和 Squarespace 退单追回率提升 13%

09:00 - 10:30 | 优化结账套件
- 通过创建无摩擦结账体验提升收入
- Stripe Link:一键式数字钱包,设备和浏览器无关
- 不同于 Apple Pay 和 Google Pay,可在任何设备和浏览器上使用

10:30 - 12:45 | 代理商务的三种模式
- B2A(企业对代理):企业参与代理渠道
- A2C(代理对消费者):AI 驱动的顾问/造型师
- A2A(代理对代理):未来模式,买方代理与卖方代理自主协商,无需人工介入
- 现场调查:超过一半的观众使用过 AI 进行购物(行业统计为 1/4)

12:45 - 14:30 | B2A 模式:参与代理渠道
- AI 正在成为新的搜索引擎和店面
- 新兴生态系统:买家、AI 代理/编排器、商家/企业
- 企业希望产品在 AI 聊天服务中可被发现
- 买家需要无摩擦、安全的结账体验
- 支付是代理商务中不可妥协的信任层
- 与所有主要 AI 公司合作:OpenAI、Perplexity、Gemini、Microsoft Copilot、X、Meta 等

14:30 - 17:45 | 代理商务协议(Agentic Commerce Protocol, ACP)
- 9 月底在纽约发布,与 OpenAI 共同制定
- 开放标准,Apache 2 许可,社区驱动,可被任何 AI 平台使用
- 解决的问题:代理不擅长处理人类可读的结账流程(概率性结果不一致)
- 解决方案:创建抽象层,将人类可读的结账转换为机器可读格式
- 代理擅长确定性结果,可以一致地调用 API
- 包含产品目录规范、委托支付规范等文档

17:45 - 19:00 | ACP vs AP2 对比
- Google 在 ACP 发布前两周推出 AP2
- AP2 是原则框架,包含思想领导力
- Stripe 的方法:构建实际产品,与设计合作伙伴创新,快速上线,在实际应用中学习和迭代
- ACP 已在 Etsy 上线,正在与 Walmart、Shopify 等大型零售商实施

19:00 - 20:45 | 共享支付令牌(Shared Payment Tokens)
- 解决场景:防止代理误解指令(如将"模型飞机"理解为"真实飞机"并清空银行账户)
- 类似俄罗斯套娃的多层逻辑结构
- 包含预算限制:代理不能超过设定金额
- 包含欺诈信号:如卡片是否曾用于卡测试攻击
- 即时生成,确保交易安全

20:45 - 22:00 | 实时演示
- 现已在 ChatGPT + Etsy 上线
- 用户可以在 ChatGPT 中搜索 Etsy 商品
- 部分商品显示"访问"(未采用 ACP),部分显示"购买"(已采用 ACP)
- 可在聊天会话中直接完成购买,无需离开 ChatGPT
- 提供超个性化推荐,在灵感产生时完成支付

22:00 - 24:30 | A2C 模式:AI 造型师/顾问
- 部分零售品牌不想将所有发现模式交给外部 AI 代理
- 奢侈品牌、酒店等更适合站内 AI 助手
- 案例:Amazon Rufus、Walmart Sparky、Ralph Lauren Ask Ralph
- 使用 GenAI 优化体验,提供超个性化推荐,但保持品牌化体验

24:30 - 26:15 | Stripe + AWS 参考架构
- 5 月在 Stripe Sessions 发布站内购物顾问
- 与 AWS 共同构建参考架构,允许任何企业实施自己的 AI 助手
- 使用 AWS Bedrock 作为 GenAI 前端
- Bedrock 在知识库、产品目录、护栏方面训练良好
- 内置护栏防止代理产生不可预测的行为

26:15 - 27:30 | 技术架构细节
- 展示 AI 造型师的参考架构
- Stripe 交互通过清晰的 API 调用完成
- 站内购物助手无需使用共享支付令牌
- Stripe 提供 MCP 服务器,可直接调用生成支付链接
- 使用 Lambda 函数
- 可通过 Stripe API、MCP 服务器或代理商务 SDK 构建站内支付体验

27:30 - 28:45 | 总结与资源
- 访问 agenticcommerce.dev 查看与 OpenAI 共同发布的协议
- GitHub 仓库开放,鼓励开源社区参与
- Stripe 强调"我们还没有赢":协议刚发布,需要更多用例和开发者社区参与
- 协议将持续成熟和演进
- 演讲者会后可进行深入交流
