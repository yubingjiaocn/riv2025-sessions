# AWS re:Invent 2025 云安全会议总结

## 会议概述

本次会议由CrowdStrike的首席产品经理Vanayak Shastri主讲，主题为"重新构想云安全：云检测与响应的智能化体验"。演讲深入探讨了当前云安全威胁的演变趋势，特别是Scattered Spider和Genesis Panda等高级威胁组织如何利用AI技术加速攻击效率。会议指出，2024年云威胁的中位突破时间已降至48分钟，最快记录仅为51秒，这对传统的安全运营响应能力提出了严峻挑战。

演讲者详细介绍了CrowdStrike如何重新架构其Falcon云安全平台，通过实时流式检测引擎、统一检测框架和自动化响应机制来应对现代云威胁。新平台整合了云控制平面、数据平面、流日志和审计日志等多维度数据，利用机器学习和智能代理(Agentic AI)技术实现从检测到响应的秒级处理。会议还展示了一个实战演示，演示了如何在一分钟内检测并自动阻止攻击者通过社会工程获取AWS凭证后建立持久化访问的攻击链。

整个解决方案强调"对手视角"和"预防优先"的理念，通过Charlotte AI提供智能化威胁分析和响应建议，并通过Falcon Fusion实现自动化工作流编排。这种端到端的方法旨在帮助安全团队将平均响应时间从60分钟缩短至60秒，同时解决安全人才短缺和技能差距的问题。

## 详细时间线

[00:00-02:30] 开场与议程介绍
- 演讲者Vanayak Shastri自我介绍，担任CrowdStrike云检测与响应产品线的首席产品经理
- 会议议程包括：最新威胁趋势、对手行为分析、SOC团队面临的挑战、现有CDR解决方案的不足、CrowdStrike的重新架构方案、产品演示和智能代理体验

[02:30-05:00] 当前威胁态势分析
- Scattered Spider在2023年占云威胁的三分之一，2024年降至13%（并非活动减少，而是其他威胁组织增加）
- 成熟的威胁组织正在云基础设施内构建完整的威胁生态系统
- 攻击者利用云原生构建块对抗防御者

[05:00-07:30] Genesis Panda攻击演变
- 2024年初：窃取凭证（通过实例元数据服务等方式）
- 4月：突破客户云账户，访问云基础设施和虚拟机
- 横向移动并在客户生态系统内建立自己的基础设施，连接到C2域
- 2025年3月：建立SSH密钥实现持久化，无需接触端点即可登录

[07:30-10:00] AI对威胁的增强作用
- 增强侦察能力：从小时/天级缩短至分钟/秒级
- 漏洞利用：通过LLM快速生成利用脚本，缩短暴露到攻击的时间窗口
- 恶意软件进化：快速构建跨基础设施的工具
- 技术支持：降低攻击门槛，使云突破更容易

[10:00-12:30] 防御者面临的挑战
- CrowdStrike 2025威胁报告显示中位突破时间降至48分钟，最快记录51秒
- 云环境特有挑战：缺乏内联传感器、依赖CloudTrail事件
- 复杂且孤立的调查：身份、云检测、传感器检测分散在不同产品中
- 响应困难：无法像传统工作负载那样终止进程或隔离主机，如何隔离身份角色或S3存储桶成为难题

[12:30-15:00] 重新构想云检测与响应的三大支柱
- 实时可见性：实时查看控制平面活动
- 统一调查：整合数据平面、控制平面、流日志、审计日志、Kubernetes日志，提供完整上下文
- 自动化响应：能够隔离存储桶、撤销恶意访问令牌的自动化能力

[15:00-17:30] CrowdStrike解决方案核心理念
- 对手视角思维：从Scattered Spider和Crazy Panda的角度思考
- 通过攻击指标(IoA)早期检测侦察活动
- 理解攻击时间线
- 利用工作负载传感器和融合技术阻止突破
- 使SOC团队能够大规模调查和响应

[17:30-20:00] 新架构的四大核心能力
1. 实时可见性和检测：从摄取和检测两个维度实现实时性
2. 关联与上下文化：整合资产风险、容器信息、命名空间、各类日志，通过机器学习和智能代理分析
3. 智能化分析：利用Charlotte AI理解日志含义
4. 自动化响应：通过API执行自动响应策略，先扑灭火焰再制定长期修复策略

[20:00-23:00] 新检测引擎技术细节
- 流式摄取：日志流式传输，检测在流中实时应用
- 检测时间从分钟级降至秒级
- 经过Overwatch和MDR团队十年验证，每天处理5万亿事件
- 统一检测：整合身份、云检测、传感器检测，避免孤岛效应
- 为AI提供完整反馈而非孤立数据

[23:00-25:00] 加速响应机制
- 运行时提供完整上下文
- Charlotte AI自动分类检测，提供事件摘要、原因分析和响应建议
- 智能代理做出专家级决策
- 将SOC响应时间从60分钟缩短至60秒
- 集成Falcon Fusion和SOAR实现实时检测与响应

[25:00-30:00] 实战演示
- 模拟场景：攻击者通过社会工程获取开发者AWS凭证
- 攻击链：使用CloudShell创建SSH密钥对 → 建立具有管理员权限的IAM角色 → 启动附加特权角色的EC2实例
- 攻击完成时间：不到1分钟
- Falcon检测：实时图形化展示攻击展开过程
- 异常识别：开发者从未使用CloudShell进行部署，创建管理员配置文件并启动带有新SSH密钥的非托管EC2实例
- 自动响应：拒绝被入侵凭证的进一步访问，停止EC2实例启动同时保留磁盘用于取证调查
- 结果：攻击者再次尝试登录时被拒绝访问

[30:00-33:00] 智能代理(Agentic AI)体验
- 提高SOC团队生产力：自动化手动分类工作
- 专家AI代理节省数周时间
- 降低平均响应时间(MTTR)
- 解决技能短缺问题：SOC管理员需要跨平台、身份、本地、云、SaaS工作，难以成为所有领域专家

[33:00-35:00] 预训练智能代理类型
- 分类代理：跨板块完整上下文分类
- 威胁狩猎代理
- 漏洞代理
- 响应代理
- 基于MDR和事件响应团队(Overwatch、Falcon Complete)多年研究和情报手册训练
- 持续扩展更多代理能力

[35:00-37:30] 统一安全平台架构
- 单一统一传感器：覆盖Linux、Windows、Mac、Kubernetes和容器
- 企业级图谱：关联风险和资产详情
- 跨板块检测：端点检测、云检测、SaaS检测、身份检测
- 云威胁检测与响应(CDR)作为单一组件
- MDR和CDR团队协同工作
- 智能AI和AI代理获得完整上下文，避免身份、云、SaaS孤岛
- 从整体视角实现即时修复和长期预防
