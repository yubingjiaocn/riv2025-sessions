# AWS re:Invent 2025 会议总结：三星电子如何使用 AI Agent 优化大规模基础设施

## 会议概述

本次会议由三星电子的 Kota Tyler Gim 主讲，他是公司 CSR 部门的 redo 工程师，负责运营 Bixby、Galaxy AI 等主要 AI 服务，以及三星钱包、三星账户等关键任务项目。会议主题聚焦于三星电子如何利用 AI Agent 结合 Cast AI 解决方案来优化大规模 Kubernetes 基础设施。

三星电子面临着管理超过 1000 个 EKS 集群、数万个虚拟机节点以及 50 多个应用程序的挑战，这些应用运行在 AWS、Google Cloud、Azure 和三星云等多云环境中。为应对规模化运营、CPU/GPU 工作负载混合管理以及成本优化等挑战，三星开发了 Agent AI 解决方案，并集成 Cast AI 实现基础设施的智能自动化管理。通过 AI 驱动的自动化、智能装箱优化和 Spot 实例的组合使用，三星实现了超过 30% 的成本节省，同时大幅减少了运维团队的人工干预需求。

会议还展望了未来发展方向，包括实现完全自主的云运营、建立行业效率标准、统一 CPU 和 GPU 工作负载优化，以及在多云环境中提供一致的优化体验。

### 三星电子公司背景 (02:30 - 04:00)
- **02:30** - 公司概况：成立于 1969 年，全球员工超过 26 万人
- **03:00** - 业务范围：Galaxy 品牌移动设备、显示器、家电、B2B 业务、半导体制造
- **03:30** - 全球 R&D 中心：专注于本地化和 AI、SR 等专业领域

### 面临的挑战 (04:00 - 08:30)
- **04:00** - 挑战一：大规模 Kubernetes 运营的复杂性
  - 运行在多云环境（AWS、Google Cloud、Azure、三星云）
  - 需要考虑基于多云的基础设施即代码
  - 细粒度的安全策略管理
- **05:00** - 挑战二：应用规模
  - 超过 50 个应用运行在 AWS 上
  - 超过 1000 个 EKS 集群
  - 数万个虚拟机节点
- **06:00** - 挑战三：CPU 和 GPU 工作负载混合管理
  - 训练团队需要简化运营（单一 Kubeflow 集群）
  - 推理团队需要更多性价比高的 GPU
  - 传统后端和前端开发团队需要统一需求
- **07:00** - 挑战四：成本节省限制
  - 过去仅关注 RI 和 Savings Plans
  - 今年开始考虑 Spot 实例
  - Spot 实例无需承诺即可获得更低成本

### 解决方案：Agent AI 架构 (08:30 - 12:00)
- **08:30** - Agent AI 项目介绍：包含多个类别
  - 基础设施管理
  - 安全和变更管理
  - FinOps 和监控
  - Karpenter 集成管理 GPU 和 CPU 工作负载
  - 智能工作负载放置的装箱优化
- **09:30** - Cast AI 的核心作用：同时控制 AI Ops 和工作负载，支持 AWS EKS
- **10:00** - 关键挑战详解：
  - 可扩展性：如何高效管理数万个多样化工作负载
  - 性能瓶颈：以三星钱包为例，韩国用户依赖移动支付，服务中断影响重大
  - 集成复杂性：50 多个应用需要简单架构，Cast AI 作为单一组件实现 AI 代理架构

### 技术解决方案 (12:00 - 16:00)
- **12:00** - AI 驱动的自动化：使用三星 AI Agent 实时持续优化云资源
  - 多个 Agent 协同工作（变更管理、安全、FinOps 等）
  - Supervisor Agent 监督整体运行
- **13:00** - Cast AI 集成：利用智能自动化功能快速扩展并实时调整工作负载大小
  - 无需运维工程师手动操作
  - 自动控制基础设施
- **14:00** - 装箱优化（Bin-packing）：
  - 仅使用装箱自动化即可节省超过 30% 成本
  - 结合 Spot 实例预期效果更佳
- **15:00** - 监控能力：Cast AI 提供美观且实用的监控仪表板，轻松掌握状态和进度

### 实施成果 (16:00 - 20:00)
- **16:00** - 减少运营开销：AI 驱动的自动化简化基础设施管理
- **16:30** - 降低手动工作：减少扩展、配置和优化的人工操作
  - 50 多个团队实施 Cast AI 后无需每天早晨监控
- **17:30** - 提升应用效率：实时优化持续调整资源大小
  - Cast AI Agent 持续检查工作负载
  - 自动缩减或扩展节点数量
- **18:30** - 消除过度和不足配置：充分利用物理负载中的所有资源
- **19:00** - 显著降低成本：结合 AI 自动化、智能装箱和 Spot 实例
  - 在 Kubernetes 环境中实现大幅节省
- **19:30** - 释放团队精力：专注于创建其他 Agent 任务和 AI 运营环境
