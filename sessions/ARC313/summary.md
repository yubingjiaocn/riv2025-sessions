# AWS re:Invent 2025 - Architecture 313: 面向构建者的智能体架构

## 会议概述

本次会议由AWS高级首席解决方案架构师Andrew Baird和WEX解决方案架构师Dan DeLauro共同主讲，主题为"面向构建者的智能体架构"。会议重点探讨了如何将传统的面向服务架构(SOA)设计原则应用到智能体系统开发中，帮助软件工程师利用现有技能构建生成式AI应用。

Andrew Baird分享了从技术角度理解智能体的方法，强调智能体本质上仍然是基于Docker容器的分布式系统，遵循服务导向架构的核心原则。Dan DeLauro则详细介绍了WEX公司如何在三个月内将智能体系统从试点推向生产环境，目前已服务超过2000名内部用户，展示了智能体在运维支持场景中的实际应用价值。

## 详细时间线与关键要点

### 0:00-10:00 开场介绍与智能体概念解析
- Andrew Baird介绍自己在AWS 13.5年的经验，强调当前是技术职业生涯中最激动人心的时期
- 阐述会议议程：前三分之一讲解智能体与SOA的关系，后续由WEX分享实际案例
- 解构智能体的技术本质：LLM + Docker容器应用 + MCP协议集成
- 强调智能体系统本质上仍是面向服务的架构，软件工程师现有技能完全适用

### 10:00-20:00 设计原则的演进与应用
- **传统SOA原则的延续**：松耦合、模块化、可重用性、可发现性在智能体系统中同样适用
- **演进的设计原则**：
  - 状态管理：从无状态转向需要持久化上下文记忆的多轮对话
  - 编排方式：从确定性工作流转向运行时自主协调
  - 服务契约：从静态API契约转向运行时能力涌现
- **全新设计原则**：
  - 目标导向：从CRUD操作转向目标实现的设计思维
  - 推理透明度：需要新的可观测性方法来跟踪非确定性推理过程
  - 自我纠错：智能体具备多路径目标实现能力
  - 非确定性：需要新的测试和运维方法

### 20:00-25:00 AWS服务工具介绍
- **Strands SDK**：简化智能体应用开发的框架
- **KIRO CLI和IDE**：支持云端模型推理的开发工具
- **Bedrock AgentCore Runtime**：无服务器智能体基础设施平台
- 强调采用成熟的智能体框架而非从零构建

### 25:00-35:00 WEX公司背景与挑战
- Dan DeLauro介绍WEX：全球商务平台，年处理交易额超过2300亿美元
- 全球技术服务团队年处理超过40,000个支持请求
- 构建ChatGTS系统的动机：自动化重复性工作，提升自助服务能力
- 选择高频次、高摩擦、已有流程的场景作为AI应用切入点

### 35:00-42:00 网络故障排查智能体案例
- **场景描述**：跨AWS、Azure、Google Cloud的复杂网络环境故障排查
- **智能体能力**：
  - 自动使用AWS Reachability Analyzer分析网络路径
  - 并行检查流日志和近期网络变更
  - 识别已知问题并提供自然语言解释
- **价值体现**：将需要专业知识的复杂排查工作简化为分钟级自助服务

### 42:00-48:00 EBS存储扩容自动化案例
- **事件驱动架构**：智能体响应CloudWatch告警而非被动等待
- **多智能体协作**：
  - 发现智能体：执行分类分析，检查历史记录
  - 维护智能体：选择预存的SSM文档执行操作
- **闭环管理**：自动更新Jira工单，通过SNS发布状态，向上游基础设施提交修复请求

### 48:00-52:00 架构设计与实现细节
- **前端集成**：Google Chat + WAF + API Gateway的安全架构
- **编排层**：Step Functions提供智能体工作流的纪律性约束
- **身份认证**：Google身份 + Active Directory权限的混合模式
- **知识管理**：Amazon Kendra构建统一的企业知识库
- **可观测性**：Dynamo存储关系数据，S3存储推理轨迹，Splunk提供实时监控

### 52:00-55:00 经验总结与展望
- **三大核心经验**：
  1. 架构设计仍然至关重要，现有技能完全适用
  2. 可以从小处着手但要有大局思维
  3. 保持平衡，不要被技术浪潮淹没
- 强调在快速技术变革中保持工程师的主导地位
- 鼓励与会者通过调查反馈帮助改进未来的会议内容
