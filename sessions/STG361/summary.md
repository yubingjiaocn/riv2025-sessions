# AWS re:Invent 2025 - 现代化托管文件传输会议总结

## 会议概述

本次会议由AWS数据同步团队的产品经理Jeff Bartley和负责Transfer Family等服务的产品管理负责人Smitha共同主讲，重点介绍了AWS如何帮助客户现代化其托管文件传输(MFT)解决方案。

会议核心聚焦于AWS DataSync和AWS Transfer Family两大服务，展示了它们如何帮助企业快速、安全、可靠地大规模移动文件数据。这些服务自2018年re:Invent推出以来，已经服务了全球数万客户，涵盖金融服务、医疗保健、生命科学、制造业等多个行业。演讲者强调，AWS的方法是消除繁重的基础设施管理负担，让客户能够专注于核心业务能力。

会议通过真实客户案例、架构演示和现场Demo，展示了如何利用这些服务解决迁移、定期传输和B2B数据交换等关键用例。特别值得一提的是，演讲者展示了DataSync和Transfer Family Web Apps如何协同工作，为生命科学和医疗保健等行业提供端到端的数据管道解决方案。

### 托管文件传输的挑战
[05:30 - 10:00]
- 非结构化数据（文件、电子表格、AI模型、报告、图像、视频等）无处不在且快速增长
- 客户面临的传统MFT解决方案挑战：
  - 硬件和软件更新管理耗时
  - 安全问题响应困难
  - 难以找到熟悉传统解决方案的技术人才
- 数据规模和全球分布使快速可靠的数据交换变得前所未有的重要

### AWS MFT服务概览
[10:00 - 15:00]
- DataSync和Transfer Family均于2018年re:Invent推出，目标是让数据移动变得轻松
- 2024年推出B2B Data Interchange (B2BI)，用于自动化EDI文档交换和处理
- AWS MFT核心原则：
  - **安全性**：传输中和静态数据加密，使用校验和验证每个字节
  - **自动化**：与Step Functions、Lambda、EventBridge无缝集成
  - **合规性**：提供详尽的日志记录以满足监管需求
- 数万客户跨多个行业使用这些服务

### 用例一：数据迁移
[17:30 - 35:00]

AWS DataSync介绍
- 专为快速数据移动设计，使用自定义并行传输协议
- 核心特性：
  - 快速：高速并行传输
  - 安全可靠：TLS加密，全程校验和验证
  - 可扩展：支持PB级数据、数十亿文件
  - 易用：内置过滤、调度和审计日志

支持的传输场景
- 本地到AWS：支持NFS、SMB、S3协议、Hadoop集群
- 跨云传输：支持Google Cloud、Azure Blob、Oracle等
- AWS内部：S3、FSx（所有文件系统类型）、EFS之间的传输

Enhanced Mode增强模式
- 速度提升高达8倍
- 支持几乎无限数量的文件
- 提供详细指标跟踪文件传输过程
- 简化跨云传输，无需部署基础设施

迁移模式
- 初始传输：大量数据的首次复制
- 增量传输：定期同步变更数据
- 切换：应用程序切换到新数据集
- DataSync在每个阶段都提供关键能力：传输数据和元数据、过滤、带宽最大化、详细指标

实时演示展示
- 初始传输：约20万个文件，接近1TB数据，30分钟内完成，吞吐量达600MB/s
- 增量传输：仅传输581个变更文件，3分钟内完成，大部分文件被跳过

高级模式：并行任务
- 将数据集分区为多个子集
- 运行多个并行任务以最大化网络带宽利用率
- 客户使用此模式每天移动PB级数据

客户案例：伦敦证券交易所集团
- 从其他云迁移30PB历史市场数据到AWS
- 与合作伙伴Data Art合作
- 3个月内完成迁移，每天移动近1PB数据
- 使用S3 Glacier存储成本降低约80%

### 用例二：定期传输
[35:00 - 50:00]

本地数据保护
- 许多客户仍有本地应用程序和数据
- 使用DataSync定期将数据备份到S3
- 部署DataSync代理，配置定期调度
- 自动、安全地移动数据

客户案例：Santos
- 每天产生TB级备份文件
- 需要将数据复制到S3以满足合规要求
- 最初尝试自建解决方案遇到困难（脚本管理、错误处理、数据验证）
- 采用DataSync后快速部署，大幅简化流程

生成式AI和数据湖
- DataSync在AI训练和推理数据管道中日益重要
- 支持数十亿文件和PB级数据集
- 将数据移动到FSx for Lustre或S3等可扩展存储服务

机器生成数据
- 基因测序仪、制造系统、实验室仪器、医院设备产生大量数据
- 客户希望在云中处理和分析数据
- 典型工作流：
  1. 数据在本地临时存储
  2. DataSync在作业开始时触发传输到S3
  3. 作业完成后通过EventBridge发送通知
  4. 清理本地存储为新作业腾出空间

客户案例：默克(Merck)
- 生命科学领域，面临制造过程中高缺陷率问题
- 使用DataSync将缺陷图像从多个站点传输到S3
- 利用Amazon SageMaker进行分析
- 成功将产品线缺陷率降低50%

### DataSync与Transfer Family Web Apps集成演示
[50:00 - 65:00]

Transfer Family Web Apps介绍
- 2023年re:Invent推出
- 为最终用户提供安全、简便的S3数据访问
- 无需访问S3控制台或复杂的第三方应用
- 使用AWS Identity Center进行身份验证
- 使用S3 Access Grants进行授权
- 支持自定义品牌
- 简单界面，易于用户上手

实际演示场景
- 模拟生命科学/医疗保健工作流
- 实验室仪器在本地生成数据
- 数据在云中处理
- 全球科学家需要查看生成的报告

演示步骤
1. 展示NFS服务器上的源数据（/mount/nfs/demo目录）
2. 包含5个文件夹，需要复制2个archive文件夹
3. 配置DataSync任务：
   - 源位置：NFS服务器（通过DataSync代理）
   - 目标位置：S3存储桶（jb-web-apps）
   - 使用包含过滤器：只复制archive*文件夹
   - 使用排除过滤器：排除*.temp文件
4. 运行DataSync任务，成功传输14个文件
5. 在Transfer Family Web Apps中访问数据：
   - 用户Jeff通过Identity Center登录
   - 通过S3 Access Grants获得存储桶访问权限
   - 在Web界面中浏览和下载文件
   - 验证MD5校验和文件内容

演示亮点
- 展示了DataSync的过滤功能
- 演示了Web Apps如何简化最终用户访问
- 无需给用户S3控制台访问权限
- 利用现有身份验证机制

### 用例三：B2B文件交换
[65:00 - 结束]

Transfer Family核心能力
- 当无法控制文件传输的两端时使用
- 支持行业标准协议：SFTP、AS2
- 完全托管的文件传输服务

主要用例
1. 监管文档交换：PII、PHI、HIPAA等敏感数据（金融服务、医疗保健、制药、油气行业）
2. ERP集成：连接跨业务边界的SAP等系统，如供应商接收制造商的采购订单
3. 内容分发：分发增值数据集（分析、媒体、软件），细粒度访问控制保护收入来源
4. 制造数据采集：将制造设施数据捕获到S3，用于错误检测和预防性维护

架构组件
- 支持多种数据源：
  - 手动编写的SFTP/FTP脚本
  - 非技术用户通过Web Apps
  - 远程SFTP服务器（如医疗保健理赔处理、支付对账）
  - AS2协议合作伙伴集成
- Transfer Family资源：服务器、Web UI、连接器
- 目标存储：S3存储桶或EFS文件系统
- 下游集成：AWS B2B Data Interchange用于托管EDI集成

事件驱动自动化
- Transfer Family设计为事件驱动
- 支持工作流自动化
- 与其他AWS服务集成

## 总结

本次会议全面展示了AWS在托管文件传输领域的解决方案，通过DataSync、Transfer Family和B2BI三大服务，帮助客户解决从数据迁移、定期传输到B2B交换的各种场景。演讲者通过丰富的客户案例和实际演示，证明了这些服务在安全性、可扩展性和易用性方面的优势，为企业数字化转型和云迁移提供了强有力的支持。
