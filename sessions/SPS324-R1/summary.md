# AWS re:Invent 2025 开发容器与 Amazon Q Developer 会议总结

## 会议概述

本次技术分享会由 AWS 专业服务顾问 Kris Nielander 和 Matteo Palazzi 主讲，主题为"使用开发容器和 Amazon Q Developer 加速代码交付"。会议重点介绍了如何通过开发容器（Dev Containers）创建一致的开发环境，集成质量控制和安全检查，并结合 Amazon Q Developer 实现智能代码重构和优化。演讲者强调了在咨询工作中"第一次就做对"的重要性，并展示了如何通过标准化开发环境来减少返工和提高团队效率。

会议采用互动式代码演示形式，现场展示了从基础开发容器配置到集成 Amazon Q Developer，再到创建可重用特性（Features）的完整流程。演讲者还介绍了他们构建的 CI/CD 流水线解决方案，用于自动化构建和分发开发容器镜像，实现团队间的标准化开发环境共享。

## 详细时间线与关键要点

### 0:00-5:00 开场与问题设定
- 演讲者介绍：AWS 专业服务顾问 Kris Nielander 和 Matteo Palazzi
- 互动调研：了解现场参与者背景（顾问、工程师、安全人员）
- 问题场景：描述典型的开发困境 - 周一的 PR 被拒绝，安全问题需要紧急修复，导致冲刺承诺受到威胁
- 统计数据：45% 的生产部署需要返工或回滚

### 5:00-10:00 DORA 指标与精英团队标准
- 介绍 Google DORA 研究项目的四个关键指标
- 现场调研变更前置时间：从提交到生产的时间（月、周、天）
- 部署频率：生产环境部署的频率
- 故障恢复时间：从故障到恢复正常运行的时间
- 变更失败率：精英团队约 15%，一般团队 45%-60%

### 10:00-15:00 开发容器基础概念
- 开发容器的定义和作用：创建一致的开发环境，嵌入质量控制和安全检查
- 必需组件：容器运行时（Docker、Podman 等）和兼容的 IDE（VS Code、IntelliJ 等）
- 配置文件位置：.devcontainer/devcontainer.json
- 基础配置示例：单行镜像配置到复杂的多组件配置

### 15:00-25:00 实际演示：CDK 应用开发容器配置
- 演示基础开发容器配置：从简单的镜像引用开始
- 复杂配置展示：包含 Dockerfile、生命周期命令、IDE 设置、扩展安装
- Python 开发环境配置：虚拟环境、安全工具（pip audit、pip licenses）
- 代码质量工具集成：black 格式化器、pylint、flake8
- 实时容器重建和环境加载演示

### 25:00-35:00 Amazon Q Developer 集成
- Amazon Q 扩展安装和配置
- 创建自定义规则文件：.amazonq/rules.md
- 规则内容：Python 文件头部版权信息、函数文档字符串、PEP 8 合规性
- 实时代码重构演示：使用 Amazon Q 根据预定义规则重构 Python 代码
- 容器重建过程中的技术问题处理（保存文件的重要性）

### 35:00-40:00 特性（Features）创建与流水线部署
- 开发容器特性的概念：可重用的构建块
- 特性文件结构：install.sh 和 devcontainer-feature.json
- Kiro CLI 特性创建演示：从安装脚本到配置文件
- CI/CD 流水线架构：S3 触发、EventBridge、多架构镜像构建（AMD64/ARM）
- 最终镜像演示：单行配置实现完整开发环境
- AWS Samples 仓库介绍和 QR 码分享
