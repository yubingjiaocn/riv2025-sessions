# AWS re:Invent 2025 - AWS 生成式 AI 快速入门会议总结

## 会议概述

本次会议由 AWS 高级解决方案架构师经理 Shimizu 主讲，主题为"AWS 生成式 AI 快速入门"。Shimizu 首先介绍了自己负责社会基础设施客户的团队，并分享了他在东京 AWS Summit 上展示的 AI 驱动自动演奏钢琴项目，该项目通过摄像头识别乐谱并由机械臂自动演奏，在展会中获得了第三名的参观量。

会议系统地介绍了 AWS 在人工智能领域的服务架构，重点讲解了生成式 AI 的基础模型概念、Amazon Bedrock 服务以及 AI Agent 的应用。与传统机器学习需要标注数据并针对特定任务训练模型不同，生成式 AI 的基础模型通过海量未标注数据训练，拥有数十亿参数，能够处理文本、语音、图像、视频等多种格式的输入输出，实现更接近人类的处理能力。会议特别强调了 AI Agent 作为下一代 AI 应用的重要性，它能够自主思考、规划和执行任务，减少人工干预，处理复杂的业务流程。

Shimizu 还介绍了多个实用的 AWS AI 服务，包括用于商业智能的 Amazon QuickSight、用于软件开发的 Amazon Q Developer 和 AWS Transform 等。根据 Gartner 报告，到 2028 年，33% 的企业软件将具备 AI Agent 功能，15% 的日常工作决策将由 AI 自主完成，显示出这一技术的巨大发展潜力。

## 详细时间线

开场介绍 (0:00-2:30)
- 演讲者 Shimizu 自我介绍，担任 AWS 高级解决方案架构师经理
- 分享在东京 AWS Summit 展示的 AI 自动演奏钢琴项目
- 该项目在展会参观量中排名第三
- 提供社交媒体联系方式供观众关注

AI 基础概念讲解 (2:30-5:00)
- 介绍人工智能框架下的机器学习和深度学习发展
- 解释生成式 AI 使用 TB 级数据训练基础模型
- 基础模型拥有数十亿参数，构成 AI 的"大脑"

传统机器学习 vs 生成式 AI (5:00-7:30)
- 传统方法：需要标注数据，功能局限于特定任务（如文本摘要、物体检测）
- 生成式 AI：使用未标注数据训练，不限于特定任务，更接近人类处理能力
- 支持多种模态：文本、语音、程序代码、图像、音频、视频

生成式 AI 应用示例 (7:30-8:30)
- 演示文本到视频生成：输入"多汁的芝士汉堡、融化的奶酪、薯条和可乐"等自然语言描述
- 系统自动生成相应的视频内容

AWS AI 服务三层架构 (8:30-11:00)
- **底层**：基础设施层 - Amazon SageMaker、AWS Inferentia、Trainium、GPU 等硬件
- **中间层**：模型和工具层 - Amazon Bedrock 提供构建 AI 应用的模型和工具
- **顶层**：应用层 - Amazon QuickSight、AWS Transform 等提高生产力的应用服务

Amazon Bedrock 详细介绍 (11:00-14:30)
- 用于构建生成式 AI 应用的核心服务
- 四大特点：
  1. 通过单一 API 选择业界最先进的模型
  2. 使用组织数据定制模型，适应行业术语和专业术语
  3. 负责任的 AI（Guardrail 功能）确保安全性
  4. Agent 功能处理多个连续任务

Bedrock 模型选择 (14:30-16:00)
- 提供多家公司的最新模型供选择
- Marketplace 功能提供医疗、研究等专业领域模型
- Amazon Nova：AWS 独有的基础模型系列，支持图像、音频、视频
- 之前展示的汉堡视频即由 Amazon Nova 生成

从生成式 AI 助手到 AI Agent (16:00-19:00)
- 传统生成式 AI 助手：处理单一任务（文本摘要、图像生成），需要人工干预
- AI Agent（Agentic AI）：减少人工干预，提高自主性，处理高度复杂任务
- AI Agent 将直接影响业务运营

AI Agent 核心概念 (19:00-22:00)
- 定义：能够在数字和物理环境中独立思考、规划和行动以实现目标
- 基于生成式 AI 基础模型，结合 API 和知识库处理各种任务
- 与传统工作流对比：
  - 传统：按预定逻辑处理，遇到异常需人工支持
  - AI Agent：通过与环境交互，自主识别必要任务并继续处理

AI Agent 应用示例 (22:00-23:30)
- 餐厅预订场景：
  - 根据天气情况（晴天/雨天）灵活决定座位选择
  - 连接天气预报 API，自主判断最佳餐厅和座位并完成预订

Gartner 报告数据 (23:30-25:00)
- 2024 年：不到 1% 的企业软件具备 AI Agent 功能
- 2028 年预测：33% 的企业软件将具备 AI Agent 功能
- 2028 年预测：15% 的日常工作决策将由 AI 自主完成

Amazon QuickSight 介绍 (25:00-28:00)
- 利用组织数据做出更好的商业决策
- 内置 AI Agent 功能，作为团队成员和同事协同工作
- 确保私有数据安全
- 四大功能：
  1. 定制 AI Agent 以适应组织需求
  2. QuickSight Research 和 QuickSight Site：在新领域提供专家级工作能力
  3. QuickFlow 和 QuickAutomate：自动化日常任务和复杂业务流程
  4. Spaces 功能：加速团队协作

软件开发支持 (28:00-31:00)
- 传统开发（2-3 年前）：人工编写所有代码、设计和测试
- AI 辅助开发：生成代码样本、填补代码空白，显著减少工作量
- AI Agent 开发：自主设计和开发程序代码

传统 AI vs AI Agent 开发对比 (31:00-33:00)
- 传统 AI：提供代码示例，人工决策并在项目中实施
- AI Agent：分析现有项目、添加新功能、独立分析行为、生成测试、生成文档

AWS 软件开发支持服务 (33:00-38:30)

1. Amazon Q Developer
   - AI 驱动的高效代码生成和问题解决
   - 对话式界面支持代码生成、调试、优化和更新

2. Amazon Q Developer (IDE 集成)
   - 集成 Agentic AI 功能的开发环境
   - 将代码从原型级别提升到生产级别
   - 基于提示独立设计需求、系统设计、识别任务
   - 促进需求驱动开发
   - Hook 功能：修改保存时自动生成文档或运行测试，防止代码与文档不一致

3. AWS Transform
   - 现代化遗留工作负载（大型机、VMware 等）
   - 自动分析并转换为新版本项目
   - 显著降低成本和人力成本
