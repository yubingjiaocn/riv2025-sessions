# AWS re:Invent 2025 会议总结：使用代理AI改善会员金融机构体验

## 会议概述

本次技术会议由AWS高级解决方案架构师Murali Ramanathan和首席解决方案架构师Mrudhula Balasubramanyan主讲，重点展示了会员制金融机构如何利用代理AI技术改善会员体验。会议围绕一个完整的贷款审批系统演示展开，该系统基于AWS Bedrock AgentCore平台构建，使用Strands Agents框架实现多代理协作。

演讲者详细介绍了AgentCore的核心组件，包括无服务器运行时、代理内存管理、MCP网关和身份认证服务。整个解决方案采用Agent-to-Agent (A2A)协议实现代理间通信，通过文档代理、信用风险代理、合规代理等专门化代理的协作，实现了从文档上传到最终决策的全自动化贷款处理流程。

## 详细时间线与关键要点

### 0:00-5:00 开场介绍与背景
- 演讲者自我介绍，确认参会者背景（主要来自保险公司和信用合作社）
- 介绍会议目标：展示代理AI在贷款审批系统中的应用
- 强调这是一个企业级、接近生产就绪的解决方案

### 5:00-15:00 AWS Bedrock AgentCore平台介绍
- **AgentCore Runtime**: 无服务器计算组件，支持容器化代理应用部署
- **Agent Memory**: 管理短期和长期记忆，支持多轮对话和多代理协作
- **AgentCore Gateway**: 管理MCP服务器，提供统一接口连接各种工具和API
- **AgentCore Identity**: 集中管理入站和出站身份认证授权
- **AgentCore Browser和Interpreter**: 提供网页浏览和代码执行能力
- **AgentCore Observability**: 支持OpenTelemetry，与CloudWatch集成提供全链路追踪

### 15:00-25:00 解决方案架构详解
- **文档上传流程**: 申请人上传文档至S3，触发Lambda调用监督代理
- **监督代理**: 发现可用代理，创建工作流程，协调整个处理过程
- **文档代理**: 验证文档完整性，调用Bedrock Data Automation进行数据提取
- **信用风险代理**: 使用代码解释器调用ML模型进行信用评估
- **合规代理**: 执行合规检查，包括债务收入比计算
- **决策逻辑**: 基于预定义标准自动批准、拒绝或转人工审核

### 25:00-40:00 现场演示
- 展示完整的贷款申请处理流程
- 通过AgentCore Observability实时查看代理执行日志
- 演示文档验证、数据提取、信用评估和合规检查的自动化过程
- 展示最终生成的PDF报告和邮件通知功能
- 信用风险模型预测98.3%的成功还款概率

### 40:00-50:00 代码实现详解
- **Kiro Agentic IDE**: 使用规格驱动开发方法构建解决方案
- **代理结构**: 四个主要部分 - 库导入、配置、提示工程、代理定义
- **A2A协议**: 实现代理间的双向通信和服务发现
- **OAuth认证**: 确保代理间通信的安全性
- **提示工程**: 详细的角色定义、标准操作程序和决策逻辑

### 50:00-60:30 技术问答与最佳实践
- **安全性**: 讨论防止提示注入攻击的方法，包括Bedrock Guardrails的使用
- **数据准确性**: 通过S3持久化存储确保数据可审计性
- **模型评估**: 使用Bedrock模型评估功能比较不同模型性能
- **工具管理**: 通过语义搜索解决代理工具选择问题
- **最佳实践**: 将代理视为微服务，只提供必要的工具和权限
- **开源资源**: 提供Strands Agents和Bedrock AgentCore的GitHub代码库
