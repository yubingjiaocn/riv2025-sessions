# AWS re:Invent 2025 文件存储服务更新总结

## 会议概述

本次AWS re:Invent 2025的分组会议由Christian Smith(AWS全球存储专家团队负责人)和Andy Crudge(FSX服务团队负责人)主讲,重点介绍了AWS文件存储服务的最新功能和创新。

会议强调文件存储是AWS增长最快的存储服务类型,主要服务于三大类应用场景:传统企业应用(如数据库、文件共享、备份恢复)、机器生成数据处理(如基因组学、芯片设计、金融数据分析)以及机器学习和高性能计算工作负载。AWS提供了包括FSX for Windows File Server、FSX for OpenZFS、FSX for NetApp ONTAP、FSX for Lustre和Amazon EFS在内的五种文件存储服务,每种服务针对不同的工作负载进行了优化。

本次会议的核心内容围绕四个投资方向展开:优化性价比、增强数据处理能力、提升系统韧性以及简化应用迁移到AWS的过程。特别值得关注的是智能分层存储(Intelligent Tiering)的推出,以及FSX文件系统与Amazon S3的深度集成,使用户能够通过S3 API访问文件数据,无缝对接AWS的分析和AI/ML服务生态。

## 详细时间线与关键要点

### 00:00 - 开场介绍
- 会议主讲人介绍:Christian Smith和Andy Crudge
- 会议主题:文件存储服务的新功能发布

### 02:30 - 文件存储的应用场景
- **企业应用场景**:组文件共享、Oracle/SQL数据库、备份和灾难恢复
- **机器生成数据**:股票交易数据、基因组学数据、芯片设计数据(特别提到Nvidia芯片设计需要大量文件存储支持)
- **机器学习和研究**:需要高速、高吞吐量的共享存储,支持数据科学家和量化分析师的工作

### 05:00 - AWS文件存储服务组合介绍
- **FSX系列**(Windows File Server、OpenZFS、NetApp ONTAP):为企业NAS迁移提供同类替代方案
- **FSX for Lustre**:世界上最流行的高性能文件系统,支持基因组学、地震处理和机器学习训练
- **Amazon EFS**:完全弹性的文件系统,为云原生工作负载优化,与ECS、EKS和Lambda集成

### 08:00 - 性能优化更新
Amazon EFS性能提升:
- 读取吞吐量从30 Gbps提升至60 Gbps
- 写入吞吐量从3 Gbps提升至5 Gbps
- IOPS性能10倍提升:读取从25万提升至250万IOPS,写入从3万提升至5万IOPS
- 单客户端吞吐量提升至1.5 Gbps(需安装EFS utils工具包)

### 10:30 - Amazon EFS访问点扩展
- 2025年2月发布:每个文件系统的访问点数量从1000个增加到10000个(10倍提升)
- 支持更大规模的多租户基础设施部署

### 12:00 - 智能分层存储(Intelligent Tiering)介绍
FSX for OpenZFS智能分层(去年re:Invent发布):
- 完全弹性存储,无需预配置容量
- 三层存储架构自动分层:频繁访问层(2.3美分)到归档层(低于0.5美分)
- 可选的SSD读取缓存,提供亚毫秒级延迟

### 15:00 - FSX for Lustre智能分层
- 2025年5月发布:将智能分层功能扩展到高性能FSX for Lustre
- 存储成本1-2美分,同时支持TB级吞吐量和数百万IOPS
- 适用于HPC工作负载和机器学习训练

### 18:00 - FSX for Lustre性能持续改进
- 18个月前:元数据IOPS提升15倍
- 去年re:Invent:支持Elastic Fabric Adapter和GPU Direct Storage,单客户端吞吐量提升12倍,达到1.2 TB/s

### 20:00 - 机器学习研究用例
- ML研究人员需要高吞吐量和简单易用的文件系统界面
- 研究人员通过SSH访问主目录,使用Python脚本迭代处理数据

### 22:00 - LS命令性能优化
- 2周前发布:优化Lustre的IO模式,目录列表(ls)性能提升5倍
- 改善了研究人员与文件系统交互的响应速度

### 25:00 - FSX for NetApp ONTAP存储池动态调整
- 2025年8月发布:支持在线动态缩减SSD存储池
- 用户可以根据工作负载需求灵活扩展和缩减SSD层,无需中断应用
- 适用于批处理作业和月末处理等场景

### 28:00 - 数据处理能力增强:S3集成
核心功能:FSX文件系统与Amazon S3的集成
- 2025年6月:首次为FSX for OpenZFS添加S3访问
- **本周re:Invent发布**:FSX for NetApp ONTAP也支持S3访问

### 30:00 - S3访问点工作原理
- 通过S3访问点(Access Point)为FSX文件系统提供S3 API访问
- 数据保留在FSX文件系统中,不复制到S3
- 可以在任何需要S3 bucket名称的地方使用访问点别名

### 32:00 - S3集成的应用场景
- 与AWS分析服务集成:Amazon Athena、EMR、Bedrock、QuickSight等
- 灾难恢复场景:将FSX作为DR站点,通过S3访问点对副本数据进行只读分析
- 无需迁移应用即可使用云原生服务处理本地数据

### 35:00 - Bedrock知识库创建示例
- 使用FSX数据创建知识库的步骤:
  1. 创建S3访问点
  2. 在Bedrock控制台使用访问点别名创建知识库
  3. 指定分块策略和嵌入模型
  4. 选择向量数据库(可使用本周发布的S3 Vectors)
- 成本优化:创建知识库的成本从数千美元降至数百美元

### 38:00 - AWS Transfer Family集成
- 支持SFTP、FTPS等传统协议
- 企业间数据交换(市场数据、监管数据、医疗数据)的安全可审计方式
- 通过S3访问点,Transfer Family可以直接访问FSX文件系统
