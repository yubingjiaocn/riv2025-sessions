# AWS re:Invent 2025 会议总结：使用 Kiro 加速多步骤 SDLC

## 会议概述

本次会议由 AWS 开发者代理和体验团队的 Derek 和 Kieran 主讲,主题是如何使用 Kiro 加速多步骤软件开发生命周期(SDLC)。会议的核心理念是超越简单的"vibe coding"(使用 AI 生成代码),而是将 AI 应用于软件开发生命周期的每一个步骤,包括需求分析、系统设计、规划、构建、测试、打包和部署运维。

演讲者强调,虽然使用 AI 生成代码很有用,但专业的软件开发不应跳过关键步骤。他们介绍了 AIDLC(AI-Driven Development Life Cycle)框架,这是一个开源方法论,包含启动阶段(需求、设计、规划)、构建阶段(编码、打包)和运维阶段。会议通过现场演示构建一个名为"Flappy Kiro"的浏览器游戏,展示了如何从需求收集到最终实现的完整流程,使用 Kiro CLI 和自定义代理配置来指导整个开发过程。

演讲者特别强调了代理组合的重要性,包括正确配置 LLM 模型、上下文窗口管理和工具选择(通过 MCP - Model Context Protocol)。通过预先配置好的自定义代理,团队可以确保 AI 助手遵循特定的工作流程,提出正确的问题,并生成符合组织标准的代码和文档。
