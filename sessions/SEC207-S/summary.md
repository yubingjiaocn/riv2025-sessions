# AWS re:Invent 2025 技术会议总结

## 会议概述

本次会议由Dynatrace开发者体验团队负责人Sean O'Dell和AWS首席解决方案架构师Simone共同主持，重点讨论了Amazon Bedrock Agent Core服务以及如何使用Dynatrace进行智能代理的可观测性监控。会议强调了AWS与Dynatrace之间的强大合作伙伴关系，以及两家公司如何协作解决企业级AI代理从概念验证到生产部署的挑战。

会议内容涵盖了构建智能代理应用时面临的主要挑战，包括基础设施管理、内存能力、第三方工具访问、身份管理和可观测性等方面。Simone详细介绍了Bedrock Agent Core如何通过运行时、身份管理、网关、内存、浏览器和代码解释器等组件来解决这些挑战。Sean则从开发者体验角度阐述了Dynatrace如何通过端到端可观测性平台，结合OpenTelemetry等开源技术，为开发者提供更全面的监控和调试能力。

## 详细时间线与关键要点

### 0:00-2:00 会议开场与介绍
- Sean O'Dell介绍自己为Dynatrace开发者体验团队负责人
- Simone自我介绍为AWS首席解决方案架构师
- 展示会议议程，包括AWS和Dynatrace的最新公告
- 强调AWS与Dynatrace的合作伙伴关系

### 2:00-5:00 智能代理应用挑战分析
- 讨论构建智能代理应用的主要难点
- 强调从概念验证到生产部署的复杂性
- 引用Gartner预测：到2027年，超过40%的智能代理AI项目将被取消
- 分析代理需要访问多个工具和其他代理的复杂场景

### 5:00-10:00 Bedrock Agent Core服务详解
- 介绍六大核心挑战：专用基础设施、内存能力、第三方工具访问、工具发现与交互、高级功能、可观测性
- 详细说明Agent Core的三大服务类别：部署、增强、监控
- 运行时服务：支持任何框架和模型的无服务器运行时
- 身份管理：提供代理与各种服务交互所需的身份认证
- 网关服务：作为中央MCP服务器管理所有可用工具
- 内存服务：提供短期和长期内存功能
- 浏览器服务：完全托管的浏览器，支持数十到数百个并行会话
- 代码解释器：执行LLM生成的代码进行确定性计算

### 10:00-15:00 Dynatrace可观测性解决方案
- Sean强调开发者体验的重要性，希望在IDE中直接获取信息
- 介绍Dynatrace如何扩展CloudWatch的功能
- 讨论端到端可观测性的价值，包括云原生数字服务和AI代理的整合
- 提及95%的AI试点项目失败的统计数据
- 强调OpenTelemetry作为开源解决方案的重要性
- 展示Dynatrace如何支持多种LLM和第三方AI服务

### 15:00-19:00 实际应用展示与总结
- 展示Dynatrace仪表板的实际界面
- 演示如何在IDE中使用自然语言搜索
- 强调与MCP集成的重要性，支持Quiro、VS Code等开发环境
- 讨论基础设施层面的可观测性需求
- 强调开发者体验和减少复杂性的重要性
