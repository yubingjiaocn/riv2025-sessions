# AWS re:Invent 2025 会议总结：使用 Amazon Bedrock Agent Core Gateway 扩展 AI 代理

## 会议概述

本次会议重点介绍了如何使用 Amazon Bedrock Agent Core Gateway 在企业级规模上安全地部署和管理 AI 代理。会议由 AWS 全球生成式 AI 解决方案架构团队负责人 Daval Patel 和首席工程师 Nick Aldridge 主讲，并邀请了 Sumo Logic 的客户代表分享实际应用案例。

会议强调了 AI 代理在生产环境中面临的核心挑战：当代理数量从几个扩展到数百个，工具数量从几十个增长到数千个时，如何管理代理与工具之间的集成、安全性、细粒度访问控制和多租户场景。Amazon Bedrock Agent Core Gateway 作为统一的通信层，提供了一站式解决方案，支持模型上下文协议（MCP）、自动化 API 转换、无服务器架构和企业级安全功能。

Sumo Logic 展示了他们如何利用 Agent Core Gateway 构建 Dojo AI 多代理系统，实现安全运营中的自动化威胁分析、事件响应和跨平台集成，证明了该技术在实际生产环境中的有效性。

## 详细时间线

### 开场与背景介绍
[00:00:00 - 00:02:30]
- 主持人 Daval Patel 进行现场调查：几乎所有参会者正在构建 AI 代理，大部分已部署到生产环境，约 25% 拥有超过 10 个 MCP 工具
- 介绍会议目标：学习如何从少量工具扩展到数千个工具和数百个代理，同时保持企业级安全性

### AI 代理面临的扩展挑战
[00:02:30 - 00:06:45]
- 说明 AI 代理需要访问真实数据：知识库、私有数据存储、企业 API 和本地数据
- 案例分析：零售客户支持代理从原型到生产的演进过程
- 核心问题：当代理数量和工具数量增长时，出现 M×N 指数级复杂度增长
- 需要解决的关键问题：基础设施扩展、细粒度访问控制、多租户管理、动态权限分配

### Amazon Bedrock Agent Core Gateway 解决方案
[00:06:45 - 00:09:30]
- 介绍 Agent Core Gateway 作为统一通信点的核心功能
- 一键式 MCP 化现有企业 API 和 AWS Lambda 函数
- 完全无服务器、按使用付费的架构
- 支持从私有 VPC 进行安全通信

### 技术架构深入讲解（Nick Aldridge）
[00:09:30 - 00:15:20]
- Gateway 作为中间层提供标准 MCP 操作：列出工具、搜索工具、调用工具
- 支持多种集成方式：
  - Lambda 函数集成
  - OpenAPI 规范（支持任何云平台的 API）
  - Smithy 模型规范（可访问 400+ AWS 服务）
  - MCP 服务器作为目标

### 工具缓存与搜索功能
[00:15:20 - 00:18:45]
- Gateway 缓存所有工具模式，无需额外下游调用即可响应列表请求
- 语义搜索功能：从数百个工具中返回最相关的 10 个工具
- 显著降低延迟、成本并提高准确性
- 支持隐式和显式两种同步方式

### 安全与网络集成
[00:18:45 - 00:21:30]
- 支持 PrivateLink 集成，流量不经过公共互联网
- 通过 Lambda 集成实现安全的出站连接到 VPC 内的 API
- 完整的请求生命周期可在 AWS 安全网络骨干上完成

### 跨云与跨框架能力
[00:21:30 - 00:23:15]
- 展示 Gateway 统一接口的强大能力：连接 AWS 服务、私有 MCP 服务器、私有代理、其他 Gateway、公共 MCP 服务器和各种 API
- 协议将成为像 TCP/UDP 一样的基础设施层，开发者无需深入了解协议细节

### 授权机制
[00:23:15 - 00:26:00]
- 入口授权支持两种方式：
  - OAuth：与企业身份提供商集成
  - IAM：支持机器对机器通信
- 提供无授权模式用于公共访问（如 AWS 文档 MCP 服务器）
- 出口端通过 Agent Core Identity Service 进行安全凭证交换和缓存

### 拦截器功能（Interceptors）
[00:26:00 - 00:30:45]
- 上周发布的突破性功能，允许在代理调用和工具调用之间运行自定义逻辑
- 基于 Lambda 实现，支持：
  - 细粒度访问控制
  - 参数过滤和预填充
  - 模式转换
  - 令牌传播
- 可在列表工具、搜索工具和调用工具时应用
- 支持请求和响应的双向拦截

### 工具集设置辅助代理
[00:30:45 - 00:31:30]
- AWS 构建的自定义代理帮助简化工具集设置过程
- 可在 GitHub 上获取，协助生成适合代理使用的有效模式

### Sumo Logic 客户案例介绍
[00:31:30 - 00:34:00]
- Sumo Logic 介绍：云规模智能运营平台，10 年前诞生于 AWS
- 规模：每天摄入数 PB 新数据，扫描数 EB 数据，运行在全球 10 个 AWS 区域
- 服务数千家企业级客户

### Sumo Logic 演示 - 安全运营
[00:34:00 - 00:39:15]
- 演示 Dojo AI 在 Cloud SIEM 中的自动化威胁分析
- Sock 分析师代理自动对每个洞察进行分类：良性、可疑或恶意
- AI 调查详情提供简洁的上下文和基于证据的严重性调整建议
- 支持基于假设的自然语言调查，代理自动执行搜索查询并总结发现

### Sumo Logic 演示 - Slack 集成
[00:39:15 - 00:43:00]
- 展示通过 Slack 直接与 Dojo AI 交互
- 后台架构：Slack 应用 → Agent Core Runtime 上的编排代理 → Agent Core Gateway（通过标准 MCP 协议）→ Sumo Logic 平台 API 和 Dojo AI 代理
- 演示多工具调用：获取分类详情、更新洞察状态、列出相关实体、运行日志搜索、添加评论
- 展示如何在日常应用中无缝使用 AI 代理

### Sumo Logic 架构解析
[00:43:00 - 00:47:30]
- Sumo Logic 平台架构：
  - 数据摄入层：从任何云无缝摄入遥测数据
  - 分析工具集：将原始数据转换为有意义的洞察和建议
  - 集成层：数百个开箱即用的集成用于报告、工单和修复
  - Dojo AI 层：多代理系统

### Dojo AI 设计原则
[00:47:30 - 00:49:00]
- 四大核心原则：
  1. 将代理视为数字队友，与人类和其他代理无缝协作
  2. 完全上下文感知，代理间自动共享和更新上下文
  3. 自动工具发现和选择，动态访问领域知识
  4. 安全设计

### Dojo AI 技术架构
[00:49:00 - 会议结束]
- 三大资源支柱作为 MCP 资源暴露：
  - 平台 API：数百个公共 API
  - Dojo AI 代理：新的代理家族
  - 领域知识：安全和可观测性专业知识
