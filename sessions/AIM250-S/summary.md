# AWS re:Invent 2025 Heroku AI 平台服务演讲总结

## 会议概述

本次演讲由 Heroku 首席开发者倡导者 Julian Duki 主讲，重点介绍了 Heroku 作为 Salesforce 旗下公司如何将自身打造成"AI 平台即服务"(AI PaaS)。Heroku 自 2010-2011 年起就是业界首个支持 Ruby on Rails 的平台即服务提供商，现已支持九种以上编程语言。面对 AI 应用开发的复杂性，Heroku 决定构建"AI 领域的 Heroku"，为开发者提供与其传统平台一致的卓越开发体验。

演讲深入探讨了构建 AI 应用和智能代理所面临的挑战，包括模型选择、数据安全、基础设施管理等问题。Heroku 通过提供集成化平台、精选的 AI 模型库、模型上下文协议(MCP)支持以及简化的部署流程，帮助开发者专注于应用构建而非基础设施运维。演讲通过两个实际演示展示了 Heroku 的 AI 能力：一个是使用托管推理和代理服务构建的太阳能监控仪表板，另一个是使用 Heroku Vibes 工具通过自然语言构建 AWS re:Invent 2025 议程管理应用。

## 详细时间线

### 开场与背景介绍
- **00:00:00 - 00:01:30**: 演讲者自我介绍，Julian Duki 介绍自己是 Heroku 首席开发者倡导者，提醒听众 Salesforce 是上市公司，不要基于未正式发布的功能做采购决策

- **00:01:30 - 00:02:45**: Heroku 平台历史回顾，强调自 2010-2011 年起作为首个 Ruby on Rails PaaS 的地位，现支持九种以上编程语言，以卓越的开发者体验著称，许多开源产品试图模仿"Heroku 体验"

### AI 战略与挑战
- **00:02:45 - 00:03:30**: 提出核心问题："谁将构建 AI 领域的 Heroku？"答案是 Heroku 自己将打造 AI 平台即服务

- **00:03:30 - 00:05:00**: 阐述构建 AI 应用和代理的挑战：需要选择合适的大语言模型(文本生成、图像生成、嵌入等)、与其他应用和服务集成、安全访问数据(防止数据泄露)、管理 GPU 等基础设施的复杂性

- **00:05:00 - 00:06:15**: 介绍 Heroku 的解决方案：集成平台、卓越的开发者和运维体验、精选的 AI 模型库、通过模型上下文协议(MCP)实现可扩展性，让 LLM 通过提示资源和工具访问更多上下文

### 生产部署的复杂性
- **00:06:15 - 00:07:30**: 讨论将应用部署到生产环境的复杂决策：选择部署位置、底层操作系统和网络配置、CPU 和内存分配、数据存储、日志记录、可观测性等

- **00:07:30 - 00:08:30**: Heroku 如何简化这些选择：提供有主见的平台，让开发者专注于应用构建，使用开发工具部署，通过提供的工具进行扩展、监控和可观测性，Heroku 负责其余部分(指标、支持等)

- **00:08:30 - 00:09:00**: 强调这种方式提高开发者生产力、降低 DevOps 和基础设施运维成本、提高应用投资回报率

### Heroku AI 产品线
- **00:09:00 - 00:10:30**: 介绍今年发布的 AI 产品，使 Heroku 成为 AI 平台即服务：
  - **托管推理和代理**：一键或一条命令为应用配置 AI 模型
  - 精选模型库支持文本生成、图像生成和嵌入

- **00:10:30 - 00:11:45**: 最新模型支持列表：
  - Claude 3.5 Sonnet
  - Claude 3.5 Haiku
  - Amazon Nova Lite
  - Amazon Nova Pro
  - GPT-4o
  - GPT-4o mini

- **00:11:45 - 00:13:00**: MCP(模型上下文协议)支持的两种方式：
  - Heroku MCP 服务器：供开发者和 DevOps 人员管理 Heroku 资源
  - MCP 部署平台：安全部署和托管自定义 MCP 服务器，通过 HTTP 接口远程访问或直接从代理端点访问

- **00:13:00 - 00:14:00**: 其他 AI 功能：
  - **PG Vector 支持**：PostgreSQL 扩展，用于创建向量数据库、执行相似性搜索和构建检索增强生成(RAG)管道
  - **Heroku Vibes**：10 月发布的试点工具，允许通过自然语言从 Web 构建应用，自动部署到 Heroku，从想法到生产只需几分钟

### 演示一：太阳能监控仪表板
- **00:14:00 - 00:15:30**: 介绍第一个演示应用：太阳能公司仪表板，用于监控能源生产、消耗、节省情况并获取洞察

- **00:15:30 - 00:17:00**: 演示如何使用 AI 代理查询数据：
  - 应用连接到部署在 Heroku 上的 PostgreSQL 数据库
  - 使用代理而非传统 RAG 管道
  - 通过 Heroku 端点和工具(托管的 MCP)让代理访问数据库
  - 只提供只读访问权限给数据库副本，确保安全

- **00:17:00 - 00:19:00**: 第一个查询演示："本月产量最低的一天是哪天？"
  - 代理执行数据库查询工具
  - 生成 SQL 查询(代理已知数据库结构，也有工具检索模式以避免幻觉)
  - 运行查询并返回结果
  - 推理模型生成最终答案

- **00:19:00 - 00:21:30**: 第二个查询演示："绘制今天每小时的生产数据图表"
  - 执行数据库查询获取数据
  - 使用代码执行工具(Python)生成图像
  - 使用 Matplotlib、Pandas、NumPy 等库
  - 在 Heroku 的虚拟机(Dyno)上安全运行代码沙箱
  - 将图像上传到 Amazon S3 并返回预签名 URL
  - 只按应用运行的秒数计费

- **00:21:30 - 00:23:30**: 第三个实时查询演示："过去七天的峰值产量是多少？"
  - 首次查询失败，展示代理的自我修复能力
  - 代理自动尝试另一个查询
  - 成功获取数据并生成报告
  - 强调这是未经脚本编排的实时演示

### 演示一：技术实现细节
- **00:23:30 - 00:25:00**: 展示如何在 Heroku 上配置 AI 模型：
  - 应用是部署在 Heroku 上的 Node.js API
  - 在 Heroku 仪表板的"资源"部分搜索"Heroku 托管推理和代理"
  - 选择模型(如 Claude 3.5)
  - 点击"配置"即可为应用提供模型

- **00:25:00 - 00:26:30**: 访问已配置的模型：
  - 系统提供 API URL 和 API 密钥
  - 可使用 SDK 或直接 HTTP 请求访问推理端点
  - 所有操作在 Heroku 基础设施内完成，数据不会传输给第三方
  - 在应用设置中可查看配置变量，包含已配置模型的 API 密钥和 URL

- **00:26:30 - 00:28:30**: 代码实现演示：
  - 展示代理的系统提示：定义代理角色(太阳能代理)、可用工具和库(S3 上传、Matplotlib、Python 绘图等)
  - 使用 Heroku 工具(运行在 Heroku 上的工具，无需自己编写)
  - 授予工具访问数据库的权限
  - 通过 HTTP 请求调用 API
  - 使用与 OpenAI API 兼容的接口，可与 LangChain、Vercel AI SDK、Pydantic、LlamaIndex 等框架配合使用

### 演示二：Heroku Vibes
- **00:28:30 - 00:30:00**: 介绍第二个 AI 工具 Heroku Vibes：

  - 允许构建应用并部署到 Heroku
  - 从提示开始构建应用

- **00:30:00 - 00:32:00**: 实时演示创建 AWS re:Invent 2025 议程构建器：
  - 提示内容：创建议程构建器应用、跟踪会议、将信息存储在本地存储(也可选择 Heroku 上的 PostgreSQL)、从 CSV 数据源获取数据
  - 系统进入计划模式，创建逐步构建计划
  - 可批准计划或建议修改
  - 批准后开始构建

- **00:32:00 - 00:34:30**: 展示完整的构建过程：
  - 显示与代理的完整对话历史
  - 从相同的初始提示开始
  - 代理提供计划，用户批准
  - 代理实现应用并直接部署到 Heroku
  - 最终应用运行在 Heroku 域名上，可立即使用
  - 演示添加会议到议程的功能

- **00:34:30 - 00:36:00**: 迭代改进过程：
